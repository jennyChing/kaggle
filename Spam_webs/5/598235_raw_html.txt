       <div class="entry-content">
        <p>
         As you read the title, you are probably wondering isnt the wp-admin directory already password protected. You are required to login right. Well that is true, but to add an additional layer of security popular sites often add an extra layer of authentication. Few days ago, we started seeing some suspicious activity on WPBeginner, so our host
         <a class="thirstylink" href="http://www.wpbeginner.com/refer/hostgator/" rel="nofollow" target="_blank" title="HostGator">
          HostGator
         </a>
         advised us to password protect our WordPress admin directory. Apparently popular sites like Mashable do the same. In this article, we will show you a step by step guide on how to password protect your WordPress admin (wp-admin) directory.
        </p>
        <p>
         To keep things easy and simple, we will only cover
         <a href="http://www.wpbeginner.com/wordpress-hosting/" title="cPanel Web Hosting">
          cPanel web hosting companies
         </a>
         here just because cPanel has an easy enough interface to add password protected directories.
        </p>
        <p>
         Login to your cPanel. Scroll down till you see the Security Tab. Click on the Password Protect Directories icon.
        </p>
        <p>
         <img alt="Password Protect Directories" class="alignnone size-full wp-image-7954" height="107" src="http://cdn2.wpbeginner.com/wp-content/uploads/2012/07/passwordprotectdirectories.jpg" title="Password Protect Directories" width="502"/>
        </p>
        <p>
         When you click on that, a lightbox popup will show up asking for directory location. Just click on web root. Once you are there, navigate to the folder where your WordPress is hosted. Then click on the /wp-admin/ folder. You will see a screen like this:
        </p>
        <p>
         <img alt="Security Settings for a Folder" class="alignnone size-full wp-image-7955" height="427" src="http://cdn.wpbeginner.com/wp-content/uploads/2012/07/securitysettingsforafolder-e1371840680572.jpg" title="Security Settings for a Folder" width="520"/>
        </p>
        <p>
         Simply check the box to password protect the directory. Then create a user for the directory. That is it. Now when you try to access your wp-admin directory, you should see an authentication required box like this:
        </p>
        <p>
         <img alt="Authentication Required" class="alignnone size-full wp-image-7956" height="226" src="http://cdn.wpbeginner.com/wp-content/uploads/2012/07/authenticationrequired.jpg" title="Authentication Required" width="356"/>
        </p>
        <h4>
         Manual Method
        </h4>
        <p>
         First create a .htpasswds file. You can do so easily by using
         <a href="http://www.htaccesstools.com/htpasswd-generator/" rel="nofollow" target="_blank" title="htpasswd generator">
          this generator
         </a>
         . Upload this file outside your /public_html/ directory. A good path would be:
        </p>
        <p>
         home/user/.htpasswds/public_html/wp-admin/passwd/
        </p>
        <p>
         Then, create a .htaccess file and upload it in /wp-admin/ directory. Then add the following codes in there:
        </p>
        <pre class="brush: php; title: ; notranslate" title="">
         AuthName "Admins Only"
AuthUserFile /home/yourdirectory/.htpasswds/public_html/wp-admin/passwd
AuthGroupFile /dev/null
AuthType basic
require user putyourusernamehere
        </pre>
        <p>
         You must update your username in there. Also dont forget to update the AuthUserFile location path.
        </p>
        <h4>
         I have a 404 Error or a Too many redirects error
        </h4>
        <p>
         Well this can happen depending on how your server is configured. To fix this issue, open your main WordPress .htaccess file and add the following code there before the WordPress rules start.
        </p>
        <pre class="brush: php; title: ; notranslate" title="">
         ErrorDocument 401 default
        </pre>
        <p>
         Well there you have it. Now you have double authentication for your WordPress admin area. This is a good alternative to
         <a href="http://www.wpbeginner.com/wp-tutorials/protect-your-admin-folder-in-wordpress-by-limiting-access-in-htaccess/" title="Limiting WP-admin Access by IP Address">
          limiting wp-admin access by IP address
         </a>
         .
        </p>
        <p>
         <strong>
          Update: Here is how to fix the Admin Ajax Issue
         </strong>
        </p>
        <p>
         If you password protect your WordPress Admin directory, then it will break the Ajax functionality in the front-end (if it is being used). In our case, we dont have any plugins that is using ajax in the front-end. But if you do, then here is how you fix that issue.
        </p>
        <p>
         Open the .htaccess file located in your
         <strong>
          /wp-admin/
         </strong>
         folder (This is NOT the main .htaccess file that we edited above).
        </p>
        <p>
         In the wp-admin .htaccess file, paste the following code:
        </p>
        <pre class="brush: php; title: ; notranslate" title="">
         &lt;Files admin-ajax.php&gt;
    Order allow,deny
    Allow from all
    Satisfy any 
&lt;/Files&gt;
        </pre>
