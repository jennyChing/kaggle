  <h2 id="descr">
   Description
  </h2>
  <p>
   This is a bash script meant to create video contact sheets (previews) of
videos. Any video supported by mplayer and ffmpeg can be used.
   <br/>
   A note of warning:
Unlike most similar tools it, by default, makes screenshots the same size as
the video, see below for details on how to change this.
  </p>
  <p>
   Take a look at the
   <a href="../dox/vcs">
    documentation
   </a>
   and/or use the
   <kbd>
    --help
   </kbd>
   and
   <kbd>
    --fullhelp
   </kbd>
   options to learn how to use it.
  </p>
  <h3 id="notes">
   Notes
  </h3>
  <ul>
   <li>
    Using -i or -n with values that lead to the same number of vidcaps
doesn't necessarily yield exactly the
    <em>
     same
    </em>
    vidcaps, you'll
have to play with them for best results.
    <br/>
    Example: With a video that lasts for 22min 14secs,
    <kbd>
     -n 6
    </kbd>
    will create the six vidcaps:
    <br/>
    <samp>
     3:22, 6:44, 10:04, 13:28, 16:50 and 20:12
    </samp>
    ,
while
    <kbd>
     -i 3m
    </kbd>
    will create
    <br/>
    <samp>
     3:00, 6:00, 9:00, 12:00, 15:00 and 18:00
    </samp>
    .
    <br/>
    As you can see,
    <kbd>
     -n
    </kbd>
    distributes the vidcaps all over the video
while
    <kbd>
     -i N
    </kbd>
    starts at
    <samp>
     N
    </samp>
    and goes in increments
of it up to the end.
   </li>
   <li>
    You can set the aspect ratio of the video with
    <kbd>
     -a
    </kbd>
    . It is only
required for anamorphic material and discouraged for the rest.
    <br/>
    It can take either a floating point number (like
    <samp>
     1.33
    </samp>
    or
    <samp>
     1.778
    </samp>
    ) or a fraction (like
    <samp>
     4/3
    </samp>
    or
    <samp>
     16/9
    </samp>
    ).
The latter is slightly preferable.
   </li>
  </ul>

   Default behaviour
  </h2>
  <p>
   By using configuration files and/or command line options the default behaviour
can be tweaked to your needs, this are the defaults, followed by a link with
the explanation on how to change it:
  </p>

   <li>
    Captures are made every 5 minutes. See
    <em>
     <a href="../dox/vcs:usage">
      Basic usage
     </a>
    </em>

   </li>
   <li>
    Captures are the same size as the input video. See

     <a href="../dox/vcs:usage#capture_size_and_aspect_ratio">
      Size and aspect ratio
     </a>
    </em>

   </li>
   <li>
    Captures are arranged in 2 columns (if there's more than one capture). See
    <em>
     <a href="../dox/vcs:usage#number_of_columns">
      Number of columns
     </a>
    </em>
    .
   </li>
  </ul>

  <h2 id="issues">
   Known Issues
  </h2>
  <dl>
   <dt>
    Getting stuck on first frame
   </dt>
   <dd>
    This happening is a sign that the decoder (FFmpeg or MPlayer) has problems dealing with the input file. Interrupt
with CTRL+C and re-run with
    <kbd>
     -O stderr=/dev/stderr
    </kbd>
    to display decoder errors. It might be due to unsupported codecs or
broken files, there's little more you can do, except trying with the other decoder (
    <kbd>
     -M
    </kbd>
    to use MPlayer,
    <kbd>
     -F
    </kbd>
    to
use FFmpeg, which is the default).
   </dd>
   <dt>
    Getting gray frames
   </dt>
   <dd>
    This is a decoder error (either due to a broken file or some codec bug). Again little to be done other than switching decoders.
   </dd>
  </dl>

   Remember this is a (neverending) work in progress, please report any bugs you encounter, either through e-mail or the
