       <code>
        &lt;head&gt;
       </code>
       tag/in your normal workflow.
      </p>
      <p>
       You can include the please-wait.js file at the end of your body, but above your other
       <code>
        &lt;script&gt;
       </code>
       tag(s) if you like. This allows you to use PleaseWait immediately and show a loading page while the browser downloads the rest of your Javascript. Of course, this is optional. Feel free to roll the PleaseWait library into your normal, minified JS file if you prefer.
      </p>

      </h4>
      <p>
       Optional: If your initial template has some styled HTML by default (like a toolbar), you'll probably want to hide that, or your users could see a quick flash of your normal template, followed by the loading screen. This is easy to do by wrapping the HTML in your body in a div, and hiding/showing that when things are ready. PleaseWait adds a class of 'pg-loaded' to your
       <code>
        &lt;body&gt;
       </code>
       when it's done.
      </p>
      <script src="https://gist.github.com/jhdavids8/26a4bf10cf3c59366c7c.js">
      </script>
     </section>
     <section>
      <h4>
       Step 4: Show the loading screen
      </h4>
      <p>
       As soon as possible in your app, call
       <code>
        pleaseWait
       </code>
       to trigger the loading screen. Right now, the API is quite simple. Valid options are:
      </p>
      <ul>
       <li>
        <code>
         backgroundColor
        </code>
        - Needs to be a valid CSS color.
       </li>
       <li>
        <code>
         logo
        </code>
        - The URL to a logo to display. The
        <code>
         src
        </code>
        attribute of an
        <code>
         img
        </code>
        tag will be set to this value.
       </li>
       <li>
        <code>
         loadingHtml
        </code>
        - HTML to display below the logo. Can be a message, spinner, or any HTML you want. See
        <a href="http://tobiasahlin.com/spinkit/" target="_blank">
         SpinKit
        </a>
        for some great HTML/CSS3 spinners.
       </li>
      </ul>
      pleaseWait returns the loading screen when called. You'll want to save this in order to close the loading screen later.
      <script src="https://gist.github.com/jhdavids8/885fc10cf53e50c4425a.js">
      </script>
     </section>
     <section>
      <h4>
       Step 5: Close the loading screen
      </h4>
      <p>
       Whenever you're app is ready for the user, call
       <code>
        finish()
       </code>
       on the returned object from the
       <code>
        pleaseWait
       </code>
       call. This will fade the screen out.
      </p>
     </section>
     <section>
      <h4>
       Bonus Section! Update the HTML
      </h4>
      <p>
       If you want to update the
       <code>
        loadingHtml
       </code>
       displayed, call
       <code>
        updateLoadingHtml(html)
       </code>
       on the returned object from
       <code>
        pleaseWait
       </code>
       . This, by default, will fade the current HTML out and the given HTML in. This is useful if you want to cycle through messages (in case your app takes a good bit of time to load), show a login form within the loading screen, etc.
      </p>
     </section>
