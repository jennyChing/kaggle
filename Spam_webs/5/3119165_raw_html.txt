         <p>
          The average size of a web page has been growing at an accelerating rate over the last few years. Research carried out by
          <a href="http://httparchive.org/interesting.php#bytesperpage" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://httparchive.org']);" target="_blank">
           HTTP Archive
          </a>
          in January reports that the average page size has now hit 1.25mb.For users with a decent broadband connection this is not too troublesome, but for users that still rely on slow dial-up connections (like in many developing countries) or crippled mobile data networks this poses a real issue.
         </p>
         <p>
         </p>
         <div class="wp-caption aligncenter" id="attachment_21239" style="width: 410px">
          <a href="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/http_archive.png">
           <img alt="Average Web Page Size 2013 [Source: HTTP Archive]" class="size-full wp-image-21239" height="225" src="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/http_archive.png" width="400"/>
          </a>
          <p class="wp-caption-text">
           Average Web Page Size 2013 [Source: HTTP Archive]
          </p>
         </div>
         In 2011, KISSmetrics created an
         <a href="http://blog.kissmetrics.com/loading-time/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://blog.kissmetrics.com']);" target="_blank">
          infographic
         </a>
         showing how page load times can have a direct impact on user engagement. The data is most likely a little out of date now but the general principle is still valid.
         <p>
          In this blog post you are going to learn about a number of techniques that you can use to help reduce the size of your web pages and speed up your page load times.
         </p>
         <h2>
          Optimize Images
         </h2>
         <p>
          One of the easiest ways to decrease the size of your web pages and increase load times is to properly resize and optimize your images. Using images that are larger than they will be displayed in your web page means that there is a lot of unnecessary data being transferred.
         </p>
         <p>
          Once you have resized your images there are a number really handy tools like
          <a href="http://www.smushit.com/ysmush.it/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://www.smushit.com']);" target="_blank">
           Smush.it
          </a>
          and
          <a href="http://tinypng.org/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://tinypng.org']);" target="_blank">
           TinyPNG
          </a>
          to reduce the size of the files further. These tools work by removing unnecessary data from the images and optimizing the use of colors.
         </p>
         <h2>
          Minify CSS &amp; JavaScript Code
         </h2>
         <p>
          Minifying your CSS and JavaScript code can help to reduce the size of these assets by removing unneeded whitespace and optimizing your code. Minifying your code does make it incredibly difficult for humans to read, so I would only minify when you are ready to launch a site.
         </p>
         <p>
          <a href="http://www.minifycss.com/css-compressor/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://www.minifycss.com']);" target="_blank">
           CSS Compressor
          </a>
          and
          <a href="http://www.minifyjavascript.com/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://www.minifyjavascript.com']);" target="_blank">
           JavaScript Compressor
          </a>
          are two handy web apps that can minify your code for you.
         </p>
         <h2>
          Avoid using @import
         </h2>
         <p>
          The
          <code>
           @import
          </code>
          CSS rule allows you to load other CSS files into your stylesheet. Whilst this can sometimes be useful, files loaded through
          <code>
           @import
          </code>
          cannot be downloaded at the same time as the mainstylesheet which increases the overall load time of the page.
         </p>
         <p>
          Instead of using
          <code>
           @import
          </code>
          consider copying the CSS from the external file into your mainstylesheet or use multiple
          <code>
           &lt;link&gt;
          </code>
          tags.
         </p>
         <h2>
          Reduce The Number of HTTP Requests
         </h2>
         <p>
          Another great way of speeding up your web pages is to simply reduce the number of files that need to be loaded. There a few ways that you can do this.
         </p>
         <h3>
          Combine Files
         </h3>
         <p>
          Combining multiple stylesheets into one file is a really useful way of eliminating extra HTTP requests. This strategy can also be adopted for your JavaScript files. A single larger CSS or JavaScript file will often load quicker because more time can be spent downloading the data rather than establishing multiple connections to a server.
         </p>
         <h3>
          Use CSS Sprites
         </h3>
         <p>
          Using CSS sprites can be a great way of reducing the number of HTTP requests your page needs to load. A CSS sprite is a collection of images within a single image file. They are particularly useful for instances where you have a lot of small icons. You set the background image of an HTML element to the sprite using CSS and then use some clever CSS code to position the background image so that only the image within the sprite that you wish to use is visible.
         </p>
         <p>
          For more information on using CSS sprites check out
          <a href="http://css-tricks.com/css-sprites/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://css-tricks.com']);" target="_blank">
           this article
          </a>
          on CSS-Tricks.
         </p>
         <h2>
          Leverage Browser Caching
         </h2>
         <p>
          Your website probably contains files that dont change very often. Things like images, CSS files and JavaScript assets. You can help to speed up your page load times by instructing the browse to cache a copy of these files on the hard drive so that next time the user visits your web page these resources will not need to be downloaded again.
         </p>
         <p>
          To leverage browser caching you need to update your web server configuration to serve up resources with
          <code>
           Expires
          </code>
          headers. This is what tells the browser how long it should keep the cached files for.
         </p>
         <div class="wp-caption aligncenter" id="attachment_21240" style="width: 339px">
          <a href="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/expires-headers.png">
           <img alt="Inspecting Response Headers in Chrome Dev Tools" class="size-full wp-image-21240" height="220" src="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/expires-headers.png" width="329"/>
          </a>
          <p class="wp-caption-text">
           Inspecting Response Headers in Chrome Dev Tools
          </p>
         </div>
         <p>
          The easiest way to update your server configuration for Apache servers is to use a
          <code>
           .htaccess
          </code>
          file. The code below shows an example configuration that would set all image, CSS and JavaScript resources to be cached for 1 month.
         </p>
         <pre>
          &lt;IfModule mod_expires.c&gt;

# Enable Expirations
ExpiresActive On

# Default Expiration Time
ExpiresDefault "access plus 1 month"

# Expiration for Images
ExpiresByType image/gif "access plus 1 month"
ExpiresByType image/png "access plus 1 month"
ExpiresByType image/jpg "access plus 1 month"
ExpiresByType image/jpeg "access plus 1 month"

# Expiration for CSS
ExpiresByType text/css "access plus 1 month

# Expiration for JavaScript
ExpiresByType application/javascript "access plus 1 month"

&lt;/IfModule&gt;
         </pre>
         <p>
          For more information on browser caching check out
          <a href="https://developers.google.com/speed/docs/best-practices/caching" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://developers.google.com']);" target="_blank">
           this page
          </a>
          on the Google Developers website.
         </p>
         <h2>
          Tools for Testing Page Load Times
         </h2>
         <p>
          There are some fantastic tools available that can help you test the performance of your websites. In this section you are going to learn about three of my favorites.
         </p>
         <h3>
          PageSpeed
         </h3>
         <div class="wp-caption aligncenter" id="attachment_21241" style="width: 594px">
          <a href="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/pagespeed.png">
           <img alt="Analysing a Web Page with PageSpeed (Web App)" class="size-large wp-image-21241" height="365" src="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/pagespeed-1024x640.png" width="584"/>
          </a>
          <p class="wp-caption-text">
           Analysing a Web Page with PageSpeed (Web App)
          </p>
         </div>
         <p>
          PageSpeed is a tool from Google that can analyze your web pages and give you pointers on what to do to improve performance. The tool will provide you with an overall PageSpeed Score that allows you to quickly see how well optimized a page is.
         </p>
         <p>
          You can either install PageSpeed as an
          <a href="https://developers.google.com/speed/pagespeed/insights_extensions" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://developers.google.com']);" target="_blank">
           extension for Chrome or Firefox
          </a>
          , or use the
          <a href="https://developers.google.com/speed/pagespeed/insights" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://developers.google.com']);" target="_blank">
           web app
          </a>
          .
         </p>
         <h3>
          Speed Tracer
         </h3>
         <div class="wp-caption aligncenter" id="attachment_21242" style="width: 594px">
          <a href="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/speed-tracer.png">
           <img alt="Speed Tracer Chrome Extension" class="size-large wp-image-21242" height="365" src="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/speed-tracer-1024x640.png" width="584"/>
          </a>
          <p class="wp-caption-text">
           Speed Tracer Chrome Extension
          </p>
         </div>
         <p>
          <a href="https://developers.google.com/web-toolkit/speedtracer/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://developers.google.com']);" target="_blank">
           Speed Tracer
          </a>
          is another tool from Google that allows you to take a more in-depth look into the performance of your web pages. Instead of making recommendations this tool just provides you with the data that you need to identify performance bottlenecks in your web pages.
         </p>
         <h3>
          YSlow
         </h3>
         <div class="wp-caption aligncenter" id="attachment_21243" style="width: 594px">
          <a href="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/yslow.png">
           <img alt="YSlow Extension in Chrome" class="size-large wp-image-21243" height="365" src="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/yslow-1024x640.png" width="584"/>
          </a>
          <p class="wp-caption-text">
           YSlow Extension in Chrome
          </p>
         </div>
         <p>
          <a href="http://developer.yahoo.com/yslow/" onclick="javascript:_gaq.push(['_trackEvent','outbound-article','http://developer.yahoo.com']);" target="_blank">
           YSlow
          </a>
          is a browser extension from Yahoo! that is very similar to PageSpeed. Running YSlow against your web pages will give you a score and a grade that can be used to track the impact of your performance optimizations. Like PageSpeed, YSlow also gives you recommendations of techniques to use to increase performance.
         </p>
         <h3>
          Developer Tools
         </h3>
         <div class="wp-caption aligncenter" id="attachment_21244" style="width: 594px">
          <a href="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/chrome-dev-tools-network.png">
           <img alt="Using Chrome Dev Tools to Examine Asset Load Times" class="size-large wp-image-21244" height="365" src="http://blog.teamtreehouse.com/wp-content/uploads/2013/02/chrome-dev-tools-network-1024x640.png" width="584"/>
          </a>
          <p class="wp-caption-text">
           Using Chrome Dev Tools to Examine Asset Load Times
          </p>
         </div>
         <p>
          As well as using the tools mentioned above, many browsers like Chrome and FireFox come with built-in developer tools that you can use to help find areas where performance could be increased. The Network tab in the Chromedeveloper tools for example can be really useful for identifying large files that take a long time to load.
         </p>
         <h2>
          Summary
         </h2>
         <p>
          In this post we have covered a number of techniques that you can adopt to increase the performance of your web pages. However this has by no means been an exhaustive list. Using the tools that are outlined above you will be able to identify loads more tips for enhancing the performance of your web pages. Take a look at some of the links below for more information on enhancing theperformanceof your websites.
         </p>
