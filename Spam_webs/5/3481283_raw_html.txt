        <h3 class="hed">
         Setting Up Shares
        </h3>
        <p>
         You've created a basic storage volume on your FreeNAS-powered machine. Now you need to set up your sharing arrangement so that other users can find and access your drive's contents.
        </p>
        <p>
         CIFS shares are the way to go if your office uses a blend of Mac, Linux, and Windows systems. If you have a Mac-only setup, you might be better served (no pun intended) by configuring the drive for AFP shares. In the spirit of inclusivity, we'll walk through a CIFS setup.
        </p>
        <p>
         <figure class="image right small">
          <figcaption class="caption" href="http://images.pcworld.com/images/article/2012/05/share1-11360178.jpg" title="">
           Choose CIFS sharing if your network is PC-only or a mix of Macs and PCs.
          </figcaption>
          <a target="_blank">
           <img alt="" src="http://images.pcworld.com/images/article/2012/05/share1-11360178.jpg"/>
          </a>
         </figure>
         Click the
         <em>
          Sharing
         </em>
         button in the Web configuration screen's left-hand sidebar, and select
         <em>
          Add CIFS Share
         </em>
         . Give your share drive a name and select your volume by clicking the
         <em>
          Browse
         </em>
         button on the 'Path' line. Hint: The name you previously assigned to your volume should appear within the /mnt/ folder. Click it and click the
         <em>
          Close
         </em>
         button. Make sure that the box next to
         <em>
          Allow Guest Access
         </em>
         is checked, and then click
         <em>
          OK
         </em>
         .
        </p>
        <p>
         Finally, click the
         <em>
          Services
         </em>
         button on the left sidebar and select the
         <em>
          Control Services
         </em>
         option. Flip the switch on the main panel's CIFS setting to
         <em>
          on
         </em>
         . Voil! Type two bakslashes and then the FreeNAS IP address (for example,
         <strong>
          \\192.168.0.1
         </strong>
         ), into a Windows Explorer window and press
         <strong>
          Enter
         </strong>
         ; your shared drive should appear. Use Windows to map this network drive, and you'll never need to hunt it down with an IP address again.
        </p>
        <p>
         <figure class="image left small">
          <figcaption class="caption" href="http://images.pcworld.com/images/article/2012/05/share2-11360174.jpg" title="">
           Use Windows to map the share, so you don't need to remember its IP address.
          </figcaption>
          <a target="_blank">
           <img alt="" src="http://images.pcworld.com/images/article/2012/05/share2-11360174.jpg"/>
          </a>
         </figure>
         In covering this series of steps, we've glossed over some configuration options that could play a big role in allowing you to see your shared network drive--such as making sure that the CIFS settings (
         <em>
          Services
         </em>
         ,
         <em>
          CIFS
         </em>
         ) are using the correct workgroup for your network configuration. If you find that this basic setup guide isn't working for you, a network configuration issue is probably holding you back. To resolve the issue, make sure that your FreeNAS network settings basically match those of the other computers on your network.
        </p>
        <p>
         Our simple share drive is but the tip of the FreeNAS iceberg with regard to features. Options include locking drive access to users and groups, creating recurring snapshots for restoring older copies of files or deleted files entirely, and using the power combination of Dynamic DNS and FTP access to give your offsite workers easy access to the files they need. Now that you've mastered the basics, a world of advanced configurations and scenarios is yours to explore!
        </p>
        <h3 class="subhed">
         Backing Up
        </h3>
        <p>
         <figure class="image right small">
          <figcaption class="caption" href="http://images.pcworld.com/images/article/2012/05/backup1-11360121.jpg" title="">
           Windows 7 Professional and Ultimate allow you to back up and restore your PC over a network.
          </figcaption>
          <a target="_blank">
           <img alt="" src="http://images.pcworld.com/images/article/2012/05/backup1-11360121.jpg"/>
          </a>
         </figure>
         If you'd like to use your FreeNAS server as a host for automatic client
         <a href="/businesscenter/article/223354/choosing_cloud_backup_for_pcs.html">
          PC backups
         </a>
         , you can choose from any number of commercial and freeware applications to run on those clients. If you're running Windows 7 Professional or higher, you can use the built-in Windows Backup and Restore tool (lower versions won't allow you to back up over a network).
        </p>
        <p>
         Pull up the app within Windows 7 and click the
         <em>
          Set Up Backup
         </em>
         link. Select
         <em>
          Save on a Network
         </em>
         , and use the Browse button to pull up the shared FreeNAS folder you created. If you've already assigned yourself a user account on the FreeNAS server, type in your user name and password; if you haven't, use the default admin user name and password you set up earlier.
        </p>
        <p>
         <figure class="image left small">
          <figcaption class="caption" href="http://images.pcworld.com/images/article/2012/05/backup2-11360122.jpg" title="">
           Choose which files you want to back up, or let Windows 7 decide for you.
          </figcaption>
          <a target="_blank">
           <img alt="" src="http://images.pcworld.com/images/article/2012/05/backup2-11360122.jpg"/>
          </a>
         </figure>
         You can let Windows choose the files it thinks are the most important and should be backed up, or you can identify specific files and folders yourself. After that, you should be done with picking options. Your backup will start its first run. When it has finished, click the
         <em>
          Change Settings
         </em>
         link, and create an automated backup schedule.
        </p>
