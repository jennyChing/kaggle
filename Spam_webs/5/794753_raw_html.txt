       <h1 id="Sound_Output_Troubleshooting">
        Sound Output Troubleshooting
       </h1>
       <span class="anchor" id="line-6">
       </span>
       <p class="line867">
        <strong>
         Note:
        </strong>
        In some cases, the procedures listed below are not "simple fixes" and can cause problems later if the card needs to be replaced. It may be easier to simply replace or add a new card from the start rather than risk damaging the installation. Cards that work the best with Ubuntu are called "free software compatible."
        <a class="http" href="http://www.thinkpenguin.com/">
         ThinkPenguin.com
        </a>
        sells free software compatible audio cards that work with Ubuntu out of the box.
        <span class="anchor" id="line-7">
        </span>
        <span class="anchor" id="line-8">
        </span>
       </p>
       <p class="line867">
       </p>
       <h2 id="Basic_Troubleshooting_Steps">
        Basic Troubleshooting Steps
       </h2>
       <span class="anchor" id="line-9">
       </span>
       <p class="line874">
        Most of these steps apply equally to Ubuntu, Kubuntu and Xubuntu, since their core configuration is the same.
        <span class="anchor" id="line-10">
        </span>
        <span class="anchor" id="line-11">
        </span>
       </p>
       <p class="line867">
       </p>
       <h3 id="A1._Is_your_volume_turned_all_the_way_down.2C_or_is_your_speaker_muted.3F">
        1. Is your volume turned all the way down, or is your speaker muted?
       </h3>
       <span class="anchor" id="line-12">
       </span>
       <p class="line874">
        Click on the speaker icon in the upper right hand corner of the screen. This will launch a drop down menu which has a slider to control the volume. Make sure that the volume is not set too low.
        <span class="anchor" id="line-13">
        </span>
        <span class="anchor" id="line-14">
        </span>
       </p>
       <p class="line862">
        If volume is high you should also check if
        <strong>
         pacmd
        </strong>
        tool show that your sound is not mute:
        <span class="anchor" id="line-15">
        </span>
        <span class="anchor" id="line-16">
        </span>
        <span class="anchor" id="line-17">
        </span>
        <span class="anchor" id="line-18">
        </span>
        <span class="anchor" id="line-19">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1">
        </span>
        $ pacmd
        <span class="anchor" id="line-2">
        </span>
        Welcome to PulseAudio! Use "help" for usage information.
        <span class="anchor" id="line-3">
        </span>
        &gt;&gt;&gt; list-sinks
       </pre>
       <span class="anchor" id="line-20">
       </span>
       <span class="anchor" id="line-21">
       </span>
       <p class="line867">
       </p>
       <h3 id="A2._Can_you_play_a_sound_that_is_known_to_always_play_correctly.3F">
        2. Can you play a sound that is known to always play correctly?
       </h3>
       <span class="anchor" id="line-22">
       </span>
       <p class="line862">
        A standard test for speakers is to press Alt+&lt;F2&gt; and run
        <span class="anchor" id="line-23">
        </span>
        <span class="anchor" id="line-24">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-25">
        </span>
        <span class="anchor" id="line-26">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-1">
        </span>
        aplay /usr/share/sounds/alsa/Front_Center.wav
       </pre>
       <span class="anchor" id="line-27">
       </span>
       <span class="anchor" id="line-28">
       </span>
       <p class="line874">
        You should hear a voice saying "Front Center". If it doesn't play the first time you run that command, try running it again as root.
        <span class="anchor" id="line-29">
        </span>
        <span class="anchor" id="line-30">
        </span>
        <span class="anchor" id="line-31">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-2">
        </span>
        sudo aplay /usr/share/sounds/alsa/Front_Center.wav
       </pre>
       <span class="anchor" id="line-32">
       </span>
       <p class="line874">
        If it still doesn't play, that means there is a problem with your configuration.
        <span class="anchor" id="line-33">
        </span>
        If the sound plays as root, but not as a regular user, then you need to add the user to the "audio" group.
        <span class="anchor" id="line-34">
        </span>
        <span class="anchor" id="line-35">
        </span>
        <span class="anchor" id="line-36">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-3">
        </span>
        sudo addgroup &lt;username&gt; audio
       </pre>
       <span class="anchor" id="line-37">
       </span>
       <span class="anchor" id="line-38">
       </span>
       <p class="line867">
       </p>
       <h3 id="A3._Can_another_user_play_one_of_these_.22known-good.22_sounds.3F">
        3. Can another user play one of these "known-good" sounds?
       </h3>
       <span class="anchor" id="line-39">
       </span>
       <p class="line874">
        Log in with another user account. If there are no others, you should create one with default settings.
        <span class="anchor" id="line-40">
        </span>
        <span class="anchor" id="line-41">
        </span>
       </p>
       <p class="line874">
        Then follow the instructions in the previous section to play "Front_Center.wav."
        <span class="anchor" id="line-42">
        </span>
        <span class="anchor" id="line-43">
        </span>
       </p>
       <p class="line874">
        If sound is played in that user but not in yours, that means that something is wrong with your user configuration. This narrows down the problem somewhat. If you see any solutions that involve changing the whole system's configuration, you probably don't have to follow them.
        <span class="anchor" id="line-44">
        </span>
        <span class="anchor" id="line-45">
        </span>
       </p>
       <p class="line867">
       </p>
       <h3 id="A4._Is_the_system_recognizing_your_sound_card.3F">
        4. Is the system recognizing your sound card?
       </h3>
       <span class="anchor" id="line-46">
       </span>
       <p class="line862">
        Open a terminal (Applications -&gt; Accessories -&gt; Terminal) and type
        <span class="anchor" id="line-47">
        </span>
        <span class="anchor" id="line-48">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-49">
        </span>
        <span class="anchor" id="line-50">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-4">
        </span>
        sudo aplay -l
       </pre>
       <span class="anchor" id="line-51">
       </span>
       <span class="anchor" id="line-52">
       </span>
       <p class="line874">
        The output of that command should look something like this:
        <span class="anchor" id="line-53">
        </span>
        <span class="anchor" id="line-54">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-55">
        </span>
        <span class="anchor" id="line-56">
        </span>
        <span class="anchor" id="line-57">
        </span>
        <span class="anchor" id="line-58">
        </span>
        <span class="anchor" id="line-59">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-5">
        </span>
        **** List of PLAYBACK Hardware Devices ****
        <span class="anchor" id="line-2-1">
        </span>
        card 0: Intel [HDA Intel], device 0: ALC861VD Analog [ALC861VD Analog]
        <span class="anchor" id="line-3-1">
        </span>
        Subdevices: 0/1
        <span class="anchor" id="line-4">
        </span>
        Subdevice #0: subdevice #0
       </pre>
       <span class="anchor" id="line-60">
       </span>
       <span class="anchor" id="line-61">
       </span>
       <p class="line874">
        If you see this:
        <span class="anchor" id="line-62">
        </span>
        <span class="anchor" id="line-63">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-64">
        </span>
        <span class="anchor" id="line-65">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-6">
        </span>
        aplay: device_list:221: no soundcard found...
       </pre>
       <span class="anchor" id="line-66">
       </span>
       <span class="anchor" id="line-67">
       </span>
       <p class="line874">
        That means that Ubuntu is not recognizing your sound card. Check that you have the proper modules installed.
        <span class="anchor" id="line-68">
        </span>
        <span class="anchor" id="line-69">
        </span>
       </p>
       <p class="line867">
       </p>
       <h3 id="A5._Do_you_have_the_sound_modules_installed.3F">
        5. Do you have the sound modules installed?
       </h3>
       <span class="anchor" id="line-70">
       </span>
       <span class="anchor" id="line-71">
       </span>
       <p class="line874">
        Open a terminal and type
        <span class="anchor" id="line-72">
        </span>
        <span class="anchor" id="line-73">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-74">
        </span>
        <span class="anchor" id="line-75">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-7">
        </span>
        find /lib/modules/`uname -r` | grep snd
       </pre>
       <span class="anchor" id="line-76">
       </span>
       <span class="anchor" id="line-77">
       </span>
       <p class="line874">
        You should see a large list of items come up. If you don't, it means that the install process did not install the sound modules for you. To fix this, type in the terminal window:
        <span class="anchor" id="line-78">
        </span>
        <span class="anchor" id="line-79">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-80">
        </span>
        <span class="anchor" id="line-81">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-8">
        </span>
        sudo apt-get install linux-restricted-modules-`uname -r` linux-generic
       </pre>
       <span class="anchor" id="line-82">
       </span>
       <span class="anchor" id="line-83">
       </span>
       <p class="line874">
        After installing the modules, you will need to reboot for the changes to take effect.
        <span class="anchor" id="line-84">
        </span>
        <span class="anchor" id="line-85">
        </span>
       </p>
       <p class="line874">
        If the modules are already installed, proceed to the next step to check to see whether your hardware is recognizing the sound card as installed.
        <span class="anchor" id="line-86">
        </span>
        <span class="anchor" id="line-87">
        </span>
       </p>
       <p class="line867">
       </p>
       <h3 id="A6._Is_the_sound_card_physically_installed_and_recognized_by_your_hardware.3F">
        6. Is the sound card physically installed and recognized by your hardware?
       </h3>
       <span class="anchor" id="line-88">
       </span>
       <p class="line874">
        Open a terminal and type
        <span class="anchor" id="line-89">
        </span>
        <span class="anchor" id="line-90">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-91">
        </span>
        <span class="anchor" id="line-92">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-9">
        </span>
        lspci -v | grep -A7 -i "audio"
       </pre>
       <span class="anchor" id="line-93">
       </span>
       <span class="anchor" id="line-94">
       </span>
       <p class="line874">
        This should output some information about your audio hardware. An example is below.
        <span class="anchor" id="line-95">
        </span>
        <span class="anchor" id="line-96">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-97">
        </span>
        <span class="anchor" id="line-98">
        </span>
        <span class="anchor" id="line-99">
        </span>
        <span class="anchor" id="line-100">
        </span>
        <span class="anchor" id="line-101">
        </span>
        <span class="anchor" id="line-102">
        </span>
        <span class="anchor" id="line-103">
        </span>
        <span class="anchor" id="line-104">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-10">
        </span>
        00:1b.0 Audio device: Intel Corporation 82801G (ICH7 Family) High Definition Audio Controller (rev 02)
        <span class="anchor" id="line-2-2">
        </span>
        Subsystem: Toshiba America Info Systems Device ff01
        <span class="anchor" id="line-3-2">
        </span>
        Flags: bus master, fast devsel, latency 0, IRQ 22
        <span class="anchor" id="line-4-1">
        </span>
        Memory at dc440000 (64-bit, non-prefetchable) [size=16K]
        <span class="anchor" id="line-5">
        </span>
        Capabilities: &lt;access denied&gt;
        <span class="anchor" id="line-6">
        </span>
        Kernel driver in use: HDA Intel
        <span class="anchor" id="line-7">
        </span>
        Kernel modules: snd-hda-intel
       </pre>
       <span class="anchor" id="line-105">
       </span>
       <span class="anchor" id="line-106">
       </span>
       <p class="line862">
        Most people, especially laptop users, have a built-in sound card for their computer on the motherboard. If yours is built-in and isn't showing up in this step, then you probably need to enable it in your BIOS.  When you first reboot your computer, there's usually a key sequence listed telling you how to enter the BIOS configuration screen.  Since all BIOS's are differenct, you'll need to sort through your BIOS and enable the built in sound card. If you need help with this, you should ask at
        <a class="http" href="http://ubuntuforums.org">
         ubuntuforums.org
        </a>
        .
        <span class="anchor" id="line-107">
        </span>
        <span class="anchor" id="line-108">
        </span>
       </p>
       <p class="line874">
        If you are not using a built in sound card, and this step does not show your audio card, the card may be seated incorrectly on your motherboard, or may be bad or otherwise incompatible with your motherboard.  You will probably want to test the card in another computer to see if it works there.
        <span class="anchor" id="line-109">
        </span>
        <span class="anchor" id="line-110">
        </span>
       </p>
       <p class="line867">
       </p>
       <h3 id="A7._Is_my_sound_card_supported_by_the_Ubuntu_sound_system.3F">
        7. Is my sound card supported by the Ubuntu sound system?
       </h3>
       <span class="anchor" id="line-111">
       </span>
       <p class="line862">
        Ubuntu uses the Advanced Linux Sound Architecture (ALSA) for managing sound output.  There is a listing of all of the sound cards supported by ALSA at
        <a class="http" href="http://www.alsa-project.org/main/index.php/Matrix:Main">
         http://www.alsa-project.org/main/index.php/Matrix:Main
        </a>
        . You should look for the information that was listed by the command in the previous step.  If it happens that your sound card is not supported by ALSA, you have two options: obtain a new sound card that is supported by ALSA, or contact the ALSA developers and request support for your sound card.
        <span class="anchor" id="line-112">
        </span>
        <span class="anchor" id="line-113">
        </span>
       </p>
       <p class="line867">
       </p>
       <h2 id="Manual_Installation">
        Manual Installation
       </h2>
       <span class="anchor" id="line-114">
       </span>
       <p class="line874">
        If you've done the basic troubleshooting listed above and your sound still isn't working, you may have found a bug in Ubuntu. It could be something as simple as not detecting your hardware the first time or evidence of a deeper problem. Ubuntu developers need your feedback to see that your sound system is not working for you. If you've reached this point, you should file a bug report on launchpad.net under the Ubuntu distribution.  You should probably include output of the commands in the previous sections in the bug report.  Filing a bug report may seem like a worthless exercise, but it really helps in the development process so other people (and possibly even you) will not run
        <span class="anchor" id="line-115">
        </span>
        into the same problem.
        <span class="anchor" id="line-116">
        </span>
        <span class="anchor" id="line-117">
        </span>
       </p>
       <p class="line874">
        Once you've filed a report, you might want to follow these instructions to get your sound working in the meantime while the developers try to fix your bug.
        <span class="anchor" id="line-118">
        </span>
        <span class="anchor" id="line-119">
        </span>
       </p>
       <p class="line867">
       </p>
       <h3 id="Manually_starting_the_audio_driver">
        Manually starting the audio driver
       </h3>
       <span class="anchor" id="line-120">
       </span>
       <span class="anchor" id="line-121">
       </span>
       <p class="line874">
        Open a terminal and type
        <span class="anchor" id="line-122">
        </span>
        <span class="anchor" id="line-123">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-124">
        </span>
        <span class="anchor" id="line-125">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-11">
        </span>
        sudo modprobe snd-[NAME OF YOUR SOUNDCARD'S DRIVER]
       </pre>
       <span class="anchor" id="line-126">
       </span>
       <span class="anchor" id="line-127">
       </span>
       <p class="line874">
        For example, my driver is named "via82xx" so I would type
        <span class="anchor" id="line-128">
        </span>
        <span class="anchor" id="line-129">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-130">
        </span>
        <span class="anchor" id="line-131">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-12">
        </span>
        sudo modprobe snd-via82xx
       </pre>
       <span class="anchor" id="line-132">
       </span>
       <span class="anchor" id="line-133">
       </span>
       <p class="line874">
        After this, try playing sound. If you hear sound, it means that your installation isn't auto-detecting your sound card, but you have all of the right software.  This command temporarily loads the sound driver, but it will not be there on the next reboot.
        <span class="anchor" id="line-134">
        </span>
        <span class="anchor" id="line-135">
        </span>
       </p>
       <p class="line874">
        You can override the auto-detection and force your computer to load the driver on every reboot by adding the driver (i.e. snd-via82xx) to the /etc/modules file. Type
        <span class="anchor" id="line-136">
        </span>
        <span class="anchor" id="line-137">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-138">
        </span>
        <span class="anchor" id="line-139">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-13">
        </span>
        gksudo gedit /etc/modules
       </pre>
       <span class="anchor" id="line-140">
       </span>
       <span class="anchor" id="line-141">
       </span>
       <p class="line862">
        and add your sound driver as the last line of the file.  Save the file, and reboot to verify that the sound is working after a reboot.  Even if this step works, you should file a bug with the Ubuntu developers at
        <a class="http" href="http://launchpad.net">
         launchpad.net
        </a>
        if you haven't already so they can try to understand why your computer isn't auto-detecting the sound card.
        <span class="anchor" id="line-142">
        </span>
        <span class="anchor" id="line-143">
        </span>
       </p>

       <h3 id="Refreshing.2Freinstalling_the_drivers">
        Refreshing/reinstalling the drivers
       </h3>
       <span class="anchor" id="line-144">
       </span>
       <span class="anchor" id="line-145">
       </span>
       <p class="line874">
        Sometimes, sound might be configured correctly, but for some reason or another it stops working.  If this is the case, you can purge your custom changes, and reset your system to a clean base.  This may clear up your problem and restore your computer to a working state.
        <span class="anchor" id="line-146">
        </span>
        <span class="anchor" id="line-147">
        </span>
       </p>
       <p class="line874">
        Open a terminal and type
        <span class="anchor" id="line-148">
        </span>
        <span class="anchor" id="line-149">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-150">
        </span>
        <span class="anchor" id="line-151">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-14">
        </span>
        sudo aptitude --purge reinstall linux-sound-base alsa-base alsa-utils linux-image-`uname -r` linux-ubuntu-modules-`uname -r` libasound2
       </pre>
       <span class="anchor" id="line-152">
       </span>
       <span class="anchor" id="line-153">
       </span>
       <p class="line874">
        This will purge any custom configurations that you've made, and any hand-compiled modules that you've built, and restore your sound stack to the "Official" Ubuntu core.
        <span class="anchor" id="line-154">
        </span>
        <span class="anchor" id="line-155">
        </span>
       </p>

       </h2>
       <span class="anchor" id="line-156">
       </span>
       <span class="anchor" id="line-157">
       </span>
       <p class="line867">
        <img alt="Warning /!\" height="16" src="/moin_static193/light/img/icon_eek.png" title="Warning /!\" width="16"/>

         <a class="https" href="https://wiki.ubuntu.com/Audio/InstallingLinuxAlsaDriverModules">

         </span>

         <span class="anchor" id="line-161">
         </span>
        </li>
        <li>
         The other way, is getting the latest drivers from alsa-project.org. This page has the latest drivers available, which you might want to fix problems with. However, these have not been tested with Ubuntu and therefore should be used with caution.

       <h2 id="Getting_ALSA_to_work_after_suspend_.2F_hibernate">
        Getting ALSA to work after suspend / hibernate
       </h2>
       <span class="anchor" id="line-260">
       </span>
       <p class="line862">
        Some soundcards do not work after the computer has been suspended or hibernated. Files in
        <em>
         /etc/pm/sleep.d/
        </em>
        are read when the system is entering or leaving suspend mode. Create a file telling the system to restart alsa when the computer is being brought up from suspended mode to make the audio work again.
        <span class="anchor" id="line-261">
        </span>
        <span class="anchor" id="line-262">
        </span>
       </p>
       <p class="line862">
        The command
        <em>
         /sbin/alsa force-reload
        </em>
        will kill all running programs using the sound driver so the driver itself is able to be restarted.
        <span class="anchor" id="line-263">
        </span>
        <span class="anchor" id="line-264">
        </span>
       </p>
       <p class="line867">
        <span class="anchor" id="line-265">
        </span>
        <span class="anchor" id="line-266">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-21">
        </span>
        sudo nano /etc/pm/sleep.d/50alsa
       </pre>
       <span class="anchor" id="line-267">
       </span>
       <p class="line867">
        <span class="anchor" id="line-268">
        </span>
        <span class="anchor" id="line-269">
        </span>
        <span class="anchor" id="line-270">
        </span>
        <span class="anchor" id="line-271">
        </span>
        <span class="anchor" id="line-272">
        </span>
        <span class="anchor" id="line-273">
        </span>
        <span class="anchor" id="line-274">
        </span>
        <span class="anchor" id="line-275">
        </span>
        <span class="anchor" id="line-276">
        </span>
        <span class="anchor" id="line-277">
        </span>
        <span class="anchor" id="line-278">
        </span>
        <span class="anchor" id="line-279">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-22">
        </span>
        case "$1" in
        <span class="anchor" id="line-2-6">
        </span>
        hibernate|suspend)
        <span class="anchor" id="line-3-6">
        </span>
        # Stopping is not required
        <span class="anchor" id="line-4-5">
        </span>
        ;;
        <span class="anchor" id="line-5-4">
        </span>
        thaw|resume)
        <span class="anchor" id="line-6-4">
        </span>
        /sbin/alsa force-reload
        <span class="anchor" id="line-7-3">
        </span>
        ;;
        <span class="anchor" id="line-8-2">
        </span>
        *) exit $NA
        <span class="anchor" id="line-9-2">
        </span>
        ;;
        <span class="anchor" id="line-10-2">
        </span>
        esac
       </pre>
       <span class="anchor" id="line-280">
       </span>
       <span class="anchor" id="line-281">
       </span>
       <p class="line874">
        Make the newly created script to be executable with the following command:
        <span class="anchor" id="line-282">
        </span>
        <span class="anchor" id="line-283">
        </span>
        <span class="anchor" id="line-284">
        </span>
       </p>
       <pre>
        <span class="anchor" id="line-1-23">
        </span>
        sudo chmod +x /etc/pm/sleep.d/50alsa
       </pre>

        Line Input/Microphone Troubleshooting

       <h2 id="Increasing_very_low_microphone_input">
        Increasing very low microphone input
       </h2>
       <span class="anchor" id="line-289">
       </span>
       <span class="anchor" id="line-290">
       </span>
       <p class="line862">
        General note: This solution was tested for a stereo microphone (notebook ASUS EeePC 1005 PE), wowever it will probably work with other configurations. It is possible that the problem is caused by a default configuration of ALSA that cannot correctly recognise type of the microphone (mono/stereo). Anyway, it was observed on Ubuntu, Mint and possibly Debian. Thanks to user
        <a class="http" href="http://forums.linuxmint.com/memberlist.php?mode=viewprofile&amp;u=59374&amp;sid=14da20fdf50d9ca442607a52063e8285">
         "wyrdoak"
        </a>
        for a solution
        <a class="http" href="http://forums.linuxmint.com/viewtopic.php?f=49&amp;t=74288#p435604">
         http://forums.linuxmint.com/viewtopic.php?f=49&amp;t=74288#p435604
        </a>
        <span class="anchor" id="line-291">
        </span>
        <span class="anchor" id="line-292">
        </span>
       </p>
       <p class="line867">
        <span class="u">
         Symptoms:
        </span>
        The microphone provides an extremely low input signal level in all applications (skype, arecord, gnome-volume-control, etc.). Microphone input signal gains in Gnome mixer and Alsa mixer are set to 100%.
        <span class="anchor" id="line-293">
        </span>
        <span class="anchor" id="line-294">
        </span>
       </p>
       <p class="line867">
        <span class="u">
         Solution:
        </span>
        <span class="anchor" id="line-295">
        </span>
       </p>
       <ul>
        <li>
         Double check whether the microphone is unmuted:
         <span class="anchor" id="line-296">
         </span>
         <ul>
          <li>
           <p class="line862">
            Open sound preferences by left-clicking on a speaker icon in Gnome panel. Or press Alt+&lt;F2&gt; and type
            <span class="anchor" id="line-297">
            </span>
            <span class="anchor" id="line-298">
            </span>
           </p>
           <pre>
            <span class="anchor" id="line-1-24">
            </span>
            gnome-volume-control
           </pre>
           You can also type this in terminal.
           <span class="anchor" id="line-299">
           </span>
          </li>
          <li>
           Switch to Input tab and ensure the microphone is unmuted and the input level is not zero (preferably maximum). Don't worry, this will not damage the hardware.
           <span class="anchor" id="line-300">
           </span>
          </li>
         </ul>
        </li>
        <li>
         <p class="line862">
          Ensure you have
          <tt class="backtick">
           pavucontrol
          </tt>
          (Pulse Audio Vol
          <tt class="backtick">
          </tt>
          Ume CONTROL) installed:
          <span class="anchor" id="line-301">
          </span>
         </p>
         <ul>
          <li>
           <p class="line862">
            Press Alt+&lt;F2&gt; and type
            <span class="anchor" id="line-302">
            </span>
            <span class="anchor" id="line-303">
            </span>
           </p>
           <pre>
            <span class="anchor" id="line-1-25">
            </span>
            sudo apt-get install pavucontrol
           </pre>
           You can also type this in terminal.
           <span class="anchor" id="line-304">
           </span>
          </li>
         </ul>
        </li>
        <li>
         <p class="line862">
          Open pavucontrol: press Alt+&lt;F2&gt; and type
          <span class="anchor" id="line-305">
          </span>
          <span class="anchor" id="line-306">
          </span>
         </p>
         <pre>
          <span class="anchor" id="line-1-26">
          </span>
          pavucontrol
         </pre>
         You can also type this in terminal.
         <span class="anchor" id="line-307">
         </span>
        </li>
        <li>
         Ensure you are using appropriate device and its profile:
         <span class="anchor" id="line-308">
         </span>
         <ul>
          <li>
           <p class="line862">
            Go to
            <tt class="backtick">
             Configuration
            </tt>
            tab
            <span class="anchor" id="line-309">
            </span>
           </p>
          </li>
          <li>
           <p class="line862">
            Select
            <tt class="backtick">
             AnalogStereoDuplex
            </tt>
            or just
            <tt class="backtick">
             AnalogDuplex
            </tt>
            from the
            <tt class="backtick">
             Profile
            </tt>
            drop-down list
            <span class="anchor" id="line-310">
            </span>
           </p>
          </li>
          <li>
           <p class="line862">
            Note: this troubleshooting solution was tested on a laptop with only one audio card, that appeared in
            <tt class="backtick">
             Configuration
            </tt>
            tab as
            <tt class="backtick">
             InternalAudio
            </tt>
            .
            <span class="anchor" id="line-311">
            </span>
           </p>
          </li>
         </ul>
        </li>
        <li>
         Switch one of the input channels off:
         <span class="anchor" id="line-312">
         </span>
         <ul>
          <li>
           <p class="line862">
            Go to
            <tt class="backtick">
             InputDevices
            </tt>
            tab and ensure you have the
            <tt class="backtick">
             InternalAudioAnalogStereo
            </tt>
            or just
            <tt class="backtick">
             InternalAudioAnalog
            </tt>
            displayed by selecting
            <tt class="backtick">
             Allinputdevices
            </tt>
            from the
            <tt class="backtick">
             Display
            </tt>
            drop-down list.
            <span class="anchor" id="line-313">
            </span>
           </p>
          </li>
          <li>
           <p class="line862">
            Find the appropriate input device (microphone) and ensure its not muted - by unticking
            <tt class="backtick">
             muteaudio
            </tt>
            icon.
            <span class="anchor" id="line-314">
            </span>
           </p>
          </li>
          <li>
           Unlock channels so that they are not linked together - by pressing a padlock icon or chainlink icon.
           <span class="anchor" id="line-315">
           </span>
          </li>
          <li>
           <p class="line862">
            Slide the slider of
            <tt class="backtick">
             FrontRight
            </tt>
            all the way down to zero.
            <span class="anchor" id="line-316">
            </span>
           </p>
          </li>
          <li>
           <p class="line862">
            Talk to the microphone observing input level on the bar just below the two sliders or at the
            <tt class="backtick">
             Input
            </tt>
            tab of Gnome-volume-control.
            <span class="anchor" id="line-317">
            </span>
           </p>
          </li>
          <li>
           This should work when either channel is muted, though it is suggested to start with Front Right one.
           <span class="anchor" id="line-318">
           </span>
          </li>
         </ul>
        </li>
       </ul>
       <ul>
        <li>
         <p class="line862">
          Optionally you might wish to add a shortcut to pavucontrol into Gnome Menu (Mint Menu) or Gnome Panel - see for example here
          <a class="https" href="https://duckduckgo.com/?q=how+to+add+item+to+gnome+panel">
           https://duckduckgo.com/?q=how+to+add+item+to+gnome+panel
          </a>
          <span class="anchor" id="line-319">
          </span>
          <span class="anchor" id="line-320">
          </span>
         </p>
        </li>
       </ul>
       <p class="line867">
       </p>
       <h2 id="Changing_default_subdevice_configuration">
        Changing default subdevice configuration
       </h2>
       <span class="anchor" id="line-321">
       </span>
       <span class="anchor" id="line-322">
       </span>
       <ul>
        <li>
         Some sound devices have multiple capture subdevices (eg. hda-intel), but pulseaudio only seems to detect the parent device, and only listens to subdevice #0. ALSA may not be configured to use your preferred device as subdevice #0 by default.
         <span class="anchor" id="line-323">
         </span>
         <ul>
          <li>
           <p class="line862">
            Open the pulseaudio record meter (
            <em>
             pavumeter --record
            </em>
            )
            <span class="anchor" id="line-324">
            </span>
           </p>
          </li>
          <li>
           Talk into your chosen device while you carry out the process and watch the meter.
           <span class="anchor" id="line-325">
           </span>
           <ul>
            <li>
             Some channels are quite noisy even with no input (especially on integrated chipsets), so we are watching for movements that correspond to our voice.
             <span class="anchor" id="line-326">
             </span>
            </li>
           </ul>
          </li>
          <li>
           <p class="line862">
            Use
            <em>
             alsamixer -c n
            </em>
            (where n is your sound card number, probably 0)
            <span class="anchor" id="line-327">
            </span>
           </p>
           <ul>
            <li>
             Press F4 to get to the capture console
             <span class="anchor" id="line-328">
             </span>
            </li>
            <li>
             Make sure the first capture channel is enabled ("CAPTUR" in red, press space to toggle), and the volume turned up.
             <span class="anchor" id="line-329">
             </span>
            </li>
            <li>
             Check the first input source is switched to your chosen plug (Mic, in my case)
             <span class="anchor" id="line-330">
             </span>
            </li>
           </ul>
          </li>
          <li>
           At this point, you should see the record meter bouncing in response to your voice.
           <span class="anchor" id="line-331">
           </span>
          </li>
          <li>
           I found that I had to repeat this procedure each time I wanted to use the Mic, even though the settings "stuck" in the mixer, something in the soundcard wasn't initialized until it was repeated.
           <span class="anchor" id="line-332">
           </span>
          </li>
          <li>
           This also works if you use the "Options" tab in the standard mixer app to do the same thing.
           <span class="anchor" id="line-333">
           </span>
           <span class="anchor" id="line-334">
           </span>
          </li>
         </ul>
        </li>
       </ul>
       <p class="line867">

       <h1 id="MIDI_Troubleshooting_-_.2AEXPERIMENTAL.2A">
        MIDI Troubleshooting - *EXPERIMENTAL*
       </h1>
       <span class="anchor" id="line-335">
       </span>
       <span class="anchor" id="line-336">
       </span>
       <p class="line874">
        This section assumes you can successfully hear sound from your soundcard.
        <span class="anchor" id="line-337">
        </span>
        First of all, make sure that you actually have a MIDI port on your soundcard. Most onboard soundcards do not have a MIDI port.
        <span class="anchor" id="line-338">
        </span>
        <span class="anchor" id="line-339">
        </span>
       </p>
       <p class="line874">
        Next, open up this file:
        <span class="anchor" id="line-340">
        </span>
        <span class="anchor" id="line-341">
        </span>
       </p>
       <p class="line867">
        <tt>
         sudonano/etc/modprobe.d/alsa-base
        </tt>
        <span class="anchor" id="line-342">
        </span>
        Then add this options line
        <span class="anchor" id="line-343">
        </span>
        <tt>
         options&lt;sndmodulenameherei.e.snd-via82xx&gt;mpu_port=0x330
        </tt>
        <span class="anchor" id="line-344">
        </span>
        OR if you already have a options line for this soundcard add
        <tt>
         mpu_port=0x330
        </tt>
        to the line.
        <span class="anchor" id="line-345">
        </span>
        <span class="anchor" id="line-346">
        </span>
       </p>
       <p class="line874">
        The default MIDI port is 330. You should verify this number in your BIOS if you are not sure. If the number is not listed, it is most likely that the number is 330 (add the 0x for the file).
        <span class="anchor" id="line-347">
        </span>
        <span class="anchor" id="line-348">
        </span>
       </p>
       <p class="line874">
        If you get no errors, you have successfully installed your MIDI port. At the moment, I do not know if any further configuration is necessary.
        <span class="anchor" id="line-349">
        </span>
        <span class="anchor" id="line-350">
        </span>
       </p>
