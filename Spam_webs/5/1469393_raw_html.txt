        <p>
         <code>
          resource fopen(string path, string mode[, bool include [, resource context ]] )
         </code>
         <br/>
         <strong>

         </strong>
         opens the file specified by path and returns a file resource handle to the open file.  If path begins with

         </code>

        <h3>
         file_put_contents()
        </h3>
        <p>
         <code>
          int file_put_contents ( string $filename , mixed $data [, int $flags = 0 [, resource $context ]] )
         </code>
         <br/>
         A close cousin of the
         <code>
          file_get_contents()
         </code>
         function and an alternative to
         <code>
          fwrite()
         </code>
         is the
         <code>
          file_put_contents()
         </code>
         function.
         <strong>
          file_put_contents
         </strong>
         makes your life easy and writes the string contained in data to the file named in filename without any need for an
         <code>
          fopen()
         </code>
         (or
         <code>
          fclose()
         </code>
         ) function call!  Returns the number of bytes written to the file, or 1 on error. The flags argument is a bitfield with two possible values:
         <br/>
         <strong>
          FILE_USE_INCLUDE_PATH
         </strong>
         If specified, the include path is searched for the file and the file is written at the first location where the file already exists.
         <br/>
         <strong>
          FILE_APPEND
         </strong>
         If specified and if the file specified by path already exists, string is appended to the existing contents of the file.
         <br/>
         <strong>
          LOCK_EX
         </strong>
         Exclusively lock the file before writing to it.  Useful for preventing race conditions during file writing.
        </p>

          readfile()
         </code>
         except that instead of echoing the file to standard output, it turns it into an array.  Each key of the array contains one line of the file.  Flags can be one or more of the following constants when using the
         <strong>
          file
         </strong>
         function:
         <br/>
         <strong>
          FILE_USE_INCLUDE_PATH
         </strong>
         Search for the file in the include path as set in the php.ini file.
         <br/>
         <strong>
          FILE_IGNORE_NEW_LINES
         </strong>
         Do not add a newline at the end of the array elements.
         <br/>
         <strong>
          FILE_SKIP_EMPTY_LINES
         </strong>
         Skip any empty lines.
        </p>

         <strong>

        <h3>
         glob()
        </h3>
        <p>
         <code>
          array glob ( string $pattern [, int $flags = 0 ] )
         </code>
         <br/>
         <strong>
          glob
         </strong>
         returns a list of filenames matching the shell wildcard pattern given in pattern. The following characters and sequences make matches:
         <br/>
         <code>
          *
         </code>
         Matches any number of any character (equivalent to the regex pattern
         <code>
          .*
         </code>
         )
         <br/>
         <code>
          ?
         </code>
         Matches any one character (equivalent to the regex pattern
         <code>
          .
         </code>
         )

          GLOB_MARK
         </strong>
         Adds a slash to each item returned.
         <br/>
         <strong>

         </strong>

         <br/>

        <h3>
         mkdir()
        </h3>
        <p>
         <code>
          bool mkdir(string path[, int mode [, bool recursive [, resource context ]]])
         </code>
         <br/>
         <strong>
          mkdir
         </strong>
         creates the directory path with mode permissions. The mode is expected to be an octal number such as 0755. An integer value such as 755 or a string value such as u+x will not work as expected. Returns
         <code>
          true
         </code>
         if the operation was successful and
         <code>
          false
         </code>
         if not. If recursive is used, it allows for the creation of nested directories.
        </p>
        <hr/>
        <h3>
         opendir()
        </h3>
        <p>
         <code>
          resource opendir(string path[, resource context])
         </code>
         <br/>
         <strong>
          opendir
         </strong>
         opens the directory path and returns a directory handle for the path that is suitable for use in subsequent
         <code>
          readdir()
         </code>
         ,
         <code>
          rewinddir()
         </code>
         , and
         <code>
          closedir()
         </code>
         calls. If path is not a valid directory, if permissions do not allow the PHP process to read the directory, or if any other error occurs,
         <code>
          false
         </code>
         is returned.  Its use is similar to the use of
         <code>
          fopen()
         </code>
         for reading from files. Instead of passing it a filename, you should pass it a
         <em>
          directory
         </em>
         name.  The function returns a directory handle, again in much the same way as
         <code>
          fopen()
         </code>
         returns a file handle.  When the directory is open, you can read a filename from it by calling
         <strong>
          readdir($dir)
         </strong>
         , as shown in the example.  This function returns
         <code>
          false
         </code>
         when there are  no more files to be read.  Note that it will also return
         <code>
          false

        <h2>
         Create A File Class
        </h2>
        <p>
         Now that we have an overview of many of the most used file functions in PHP, we can create a file class!  Some of the file functions may seem a bit cryptic in their syntax, but by creating a file class, we can give our methods more meaningful names which can lead to more elegant and readable code.  Well  create a class that uses very simple static methods to wrap existing PHP file functions to make them easier to use.  Place this code into a file called
         <strong>
          fileclass.php

        <div class="crayon-syntax crayon-theme-github crayon-font-monaco crayon-os-pc print-yes notranslate" data-settings=" minimize scroll-always" id="crayon-559c38f30413d225549178" style=" margin-top: 12px; margin-bottom: 12px; font-size: 14px !important; line-height: 18px !important;">

           class File {
    public static function put($file, $data, $append = false) {
    	if ( $append ) {
		return file_put_contents($file, $data, FILE_APPEND | LOCK_EX);
		}
		return file_put_contents($file, $data, LOCK_EX); 
	}    

    public static function exists($file) {
    	return file_exists($file);
	}
    
    public static function read($file) {
        return readfile($file);
	}
	
	public static function size($file) {
	 	return filesize($file);
	}
	
	public static function name($file) {
	 	return pathinfo($file, PATHINFO_FILENAME);
	}

	public static function extension($file) {
	 	return pathinfo($file, PATHINFO_EXTENSION);
	}	
		
    public static function last_updated($file) {
		return filemtime($file);
	}	
	
    public static function get($file) {
		return static::exists($file)
		?  file_get_contents($file)
		:  false;
	}
    		
    public static function append($file, $data) {
		return static::put($file, $data, true);
	}
	
	public static function clear($file) {
		if ( static::exists($file) ) {
			$fp = fopen($file, 'w');
			fclose($fp);
		}
	}
    
    public static function delete($file) {
    	return unlink($file);
	}
    
}
