           <div>
            <h2 class="title" style="clear: both">
             <a name="inst-macosx">
             </a>
             Apple Mac OS X
            </h2>
           </div>
          </div>
         </div>
         <a class="indexterm" name="inst-macosx-indexterm">
         </a>
         <a class="indexterm" name="idm214695875408">
         </a>
         <p>
          Nmap has supported Mac OS X since 2001, and our support has only improved over time.  While Mac users can compile Nmap themselves, we also offer an executable installer.  Nmap
is also available through systems such as MacPorts and Fink which
package Unix software for Mac OS X.
         </p>
         <div class="sect2">
          <div class="titlepage">
           <div>
            <div>
             <h3 class="title">
              <a name="inst-macosx-installer">
              </a>
              Executable Installer
             </h3>
            </div>
           </div>
          </div>
          <a class="indexterm" name="idm214695872688">
          </a>
          <p>
           The easiest way to install Nmap and Zenmap on Mac OS X is to use
our installer. The
           <a class="ulink" href="http://nmap.org/download.html#macosx" target="_top">
            Mac OS X section of
the Nmap download page
           </a>
           provides a file named
           <code class="filename">
            nmap-
            <em class="replaceable">
             <code>
              &lt;version&gt;
             </code>
            </em>
            .dmg
           </code>
           , where
           <em class="replaceable">
            <code>
             &lt;version&gt;
            </code>
           </em>
           is the version number of the most
recent release. The
           <code class="filename">
            .dmg
           </code>
           <a class="indexterm" name="idm214695868256">
           </a>
           <a class="indexterm" name="idm214695866672">
           </a>
           file is known as a
           <span class="quote">
            
            <span class="quote">
             disk image
            </span>
            
           </span>
           . Installation instructions follow:
          </p>
          <div class="orderedlist">
           <ol class="orderedlist" type="1">
            <li class="listitem">
             <p>
              Download the file
              <code class="filename">
               nmap-
               <em class="replaceable">
                <code>
                 &lt;version&gt;
                </code>
               </em>
               .dmg
              </code>
              .
Double-click the icon to open it. (Depending on how you downloaded the
file, it may be opened automatically.)
             </p>
            </li>
            <li class="listitem">
             <p>
              The contents of the disk image will be displayed. One of
the files will be a Mac meta-package file named
              <code class="filename">
               nmap-
               <em class="replaceable">
                <code>
                 &lt;version&gt;
                </code>
               </em>
               .mpkg
              </code>
              .
Open it to start the installer.
             </p>
             <p>
              <a class="indexterm" name="idm214695860096">
              </a>
              <a class="indexterm" name="idm214695858992">
              </a>
              On OS X 10.8 and later, you may see a dialog like
              <a class="xref" href="inst-macosx.html#fig-mac-gatekeeper-blocked" title="Figure2.2.Apple Gatekeeper block screen">
               Figure2.2
              </a>
              .
             </p>
             <div class="figure">
              <a name="fig-mac-gatekeeper-blocked">
              </a>
              <p class="title">
               <b>
                Figure2.2.Apple Gatekeeper block screen
               </b>
              </p>
              <div class="figure-contents">
               <div class="mediaobject">
                <img alt="Apple Gatekeeper block screen" src="images/apple-gatekeeper-blocked.png"/>
               </div>
              </div>
             </div>
             <p>
              <br class="figure-break"/>
              If this happens, it is necessary to right-click or control-click on the
              <code class="filename">
               .mpkg
              </code>
              and select
              <span class="quote">
               
               <span class="quote">
                Open
               </span>
               
              </span>
              ,
as shown in
              <a class="xref" href="inst-macosx.html#fig-mac-gatekeeper-open" title="Figure2.3.Apple Gatekeeper Open menu">
               Figure2.3
              </a>
              .
             </p>
             <div class="figure">
              <a name="fig-mac-gatekeeper-open">
              </a>
              <p class="title">
               <b>
                Figure2.3.Apple Gatekeeper Open menu
               </b>
              </p>
              <div class="figure-contents">
               <div class="mediaobject">
                <img alt="Apple Gatekeeper Open menu" src="images/apple-gatekeeper-open.png"/>
               </div>
              </div>
             </div>
             <p>
              <br class="figure-break"/>
              A dialog similar to the first will appear, this time having an
              <span class="quote">
               
               <span class="quote">
                Open
               </span>
               
              </span>
              button (shown in
              <a class="xref" href="inst-macosx.html#fig-mac-gatekeeper-allowed" title="Figure2.4.Apple Gatekeeper Open screen">
               Figure2.4
              </a>
              ).
Click the button to continue.
             </p>
             <div class="figure">
              <a name="fig-mac-gatekeeper-allowed">
              </a>
              <p class="title">
               <b>
                Figure2.4.Apple Gatekeeper Open screen
               </b>
              </p>
              <div class="figure-contents">
               <div class="mediaobject">
                <img alt="Apple Gatekeeper Open screen" src="images/apple-gatekeeper-allowed.png"/>
               </div>
              </div>
             </div>
             <p>
              <br class="figure-break"/>
             </p>
            </li>
            <li class="listitem">
             <p>
              Follow the instructions in the
installer. You will be asked for your password since Nmap installs in a system directory.
             </p>
            </li>
            <li class="listitem">
             <p>
              Once the installer is finished, eject the disk image by
control-clicking on its icon and selecting
              <span class="quote">
               
               <span class="quote">
                Eject
               </span>
               
              </span>
              . The disk image may now be placed in
the trash.
             </p>
            </li>
           </ol>
          </div>
          <p>
           See the instructions in
           <a class="xref" href="inst-macosx.html#inst-macosx-exec" title="Executing Nmap on Mac OS X">
            the section called Executing Nmap on Mac OS X
           </a>
           for
help on running Nmap and Zenmap after they are installed.
          </p>
          <p>
           The programs installed by the installer will run on Intel Mac OS X 10.5
(Leopard) or later. Users of earlier versions will have to compile from
source or use a third-party package.  Instructions for PowerPC (PPC) Mac systems (which Apple ceased selling in 2006) are
           <a class="ulink" href="https://secwiki.org/w/Nmap/Mac_OSX_PPC" target="_top">
            available on our wiki
           </a>
           .
          </p>
         </div>
         <div class="sect2">
          <div class="titlepage">
           <div>
            <div>
             <h3 class="title">
              <a name="inst-macosx-source">
              </a>
              Compile from Source Code
             </h3>
            </div>
           </div>
          </div>
          <a class="indexterm" name="idm214695841136">
          </a>
          <p>
           Compiling Nmap from source on Mac OS X is no more difficult than
on other platforms once a proper build environment is in place.
          </p>
          <div class="sect3">
           <div class="titlepage">
            <div>
             <div>
              <h4 class="title">
               <a name="idm214695839072">
               </a>
               Compile Nmap from source code
              </h4>
             </div>
            </div>
           </div>
           <p>
            Compiling Nmap on Mac OS X requires
            <a class="ulink" href="https://developer.apple.com/xcode/" target="_top">
             Xcode
            </a>
            ,
            <a class="indexterm" name="idm214695837376">
            </a>
            Apple's developer tools that include GCC and the rest of the usual build
system. Xcode is not installed by default, but can be downloaded free of
charge from the
            <a class="ulink" href="https://developer.apple.com/xcode/" target="_top">
             Mac App
Store
            </a>
            . After installing Xcode, open
            <span class="quote">
             
             <span class="quote">
              Preferences
             </span>
             
            </span>
            , select the
            <span class="quote">
             
             <span class="quote">
              Downloads
             </span>
             
            </span>
            tab, and click the
            <span class="quote">
             
             <span class="quote">
              Install
             </span>
             
            </span>
            next to
            <span class="quote">
             
             <span class="quote">
              Command Line
Tools
             </span>
             
            </span>
            .
           </p>
           <p>
            Xcode installations don't always include the command line tools.  You can install them by opening Xcode from the Applications folder, opening Preferencechoosing the
            <code class="literal">
             Download
            </code>
            header icon and clicking the
            <code class="literal">
             Install
            </code>
            button next to
            <span class="quote">
             
             <span class="quote">
              Command Line Tools
             </span>
             
            </span>
            .
           </p>
           <p>
            Once you have installed Xcode and the command-line tools, follow the compilation instructions found in
            <a class="xref" href="inst-source.html" title="Linux/Unix Compilation and Installation from Source Code">
             the section called Linux/Unix Compilation and Installation from Source Code
            </a>
            .  Note that on some older versions of Mac OS X, you may have to replace the command
            <span class="command">
             <strong>
              ./configure
             </strong>
            </span>
            with
            <span class="command">
             <strong>
              ./configure CPP=/usr/bin/cpp
             </strong>
            </span>
            .
           </p>
          </div>
          <div class="sect3">
           <div class="titlepage">
            <div>
             <div>
              <h4 class="title">
               <a name="idm214695828640">
               </a>
               Compile Zenmap from source code
              </h4>
             </div>
            </div>
           </div>
           <a class="indexterm" name="idm214695827952">
           </a>
           <p>
            Zenmap depends on some external libraries that do not come with
Mac OS X, including GTK+ and PyGTK. These libraries have many dependencies
of their own. A convenient way to install all of them is to use a
third-party packaging system as described in
            <a class="xref" href="inst-macosx.html#inst-macosx-third-party" title="Third-party Packages">
             Section
            </a>
            . Once the dependencies are
installed, follow the instructions in
            <a class="xref" href="inst-source.html" title="Linux/Unix Compilation and Installation from Source Code">
             the section called Linux/Unix Compilation and Installation from Source Code
            </a>
            to
install Zenmap as usual.
           </p>
          </div>
         </div>
         <div class="sect2">
          <div class="titlepage">
           <div>
            <div>
             <h3 class="title">
              <a name="inst-macosx-third-party">
              </a>
              Third-party Packages
             </h3>
            </div>
           </div>
          </div>
          <a class="indexterm" name="idm214695823952">
          </a>
          <p>
           Another option for installing Nmap is to use a system
which packages Unix software for Mac OS X. The two discussed here are
           <a class="ulink" href="http://www.finkproject.org" target="_top">
            Fink
           </a>
           and
           <a class="ulink" href="http://www.macports.org" target="_top">
            MacPorts
           </a>
           . See the
respective projects' web sites for how to install the package
managers.
          </p>
          <a class="indexterm" name="idm214695820368">
          </a>
          <p>
           To install using Fink, run the command
           <span class="command">
            <strong>
             fink install
nmap
            </strong>
           </span>
           . Nmap will be installed as
           <code class="filename">
            /sw/bin/nmap
           </code>
           . To uninstall use the command
           <span class="command">
            <strong>
             fink remove nmap
            </strong>
           </span>
           .
          </p>
          <a class="indexterm" name="idm214695816960">
          </a>
          <p>
           To install using MacPorts, run
           <span class="command">
            <strong>
             sudo port
install nmap
            </strong>
           </span>
           . Nmap will be installed as
           <code class="filename">
            /opt/local/bin/nmap
           </code>
           . To uninstall, run
           <span class="command">
            <strong>
             sudo port uninstall nmap
            </strong>
           </span>
           .
          </p>
          <p>
           These systems install the
           <code class="filename">
            nmap
           </code>
           executable outside the global
           <code class="envar">
            PATH
           </code>
           . To enable Zenmap to
find it, set the
           <code class="varname">
            nmap_command_path
           </code>
           variable in
           <code class="filename">
            zenmap.conf
           </code>
           to
           <code class="filename">
            /sw/bin/nmap
           </code>
           or
           <code class="filename">
            /opt/local/bin/nmap
           </code>
           as described in
           <a class="xref" href="zenmap-files.html#zenmap-executable" title="The nmap Executable">
            the section called The
            <code class="filename">
             nmap
            </code>
            Executable
           </a>
           .
          </p>
         </div>
         <div class="sect2">
          <div class="titlepage">
           <div>
            <div>
             <h3 class="title">
              <a name="inst-macosx-exec">
              </a>
              Executing Nmap on Mac OS X
             </h3>
            </div>
           </div>
          </div>
          <a class="indexterm" name="idm214695808128">
          </a>
          <p>
           The terminal emulator in Mac OS X is called
           <span class="application">
            Terminal
           </span>
           , and is located in the directory
           <code class="filename">
            /Applications/Utilities
           </code>
           . Open it and a
terminal window appears. This is where you will type your commands.
          </p>
          <p>
           By default the root user is disabled on Mac OS X. To run a scan with
root privileges prefix the command name with
           <span class="application">
            sudo
           </span>
           ,
           <a class="indexterm" name="idm214695803552">
           </a>
           as
in
           <span class="command">
            <strong>
             sudo nmap -sS
             <em class="replaceable">
              <code>
               &lt;target&gt;
              </code>
             </em>
            </strong>
           </span>
           .
You will be asked for a password, which is just your normal login
password. Only users with administrator privileges can do this.
          </p>
          <p>
           Zenmap requires the
           <span class="application">
            X11
           </span>
           application to
be installed. If it was not installed by default it may be available as
an optional install on the Mac OS X installation discs.
          </p>
          <p>
           When Zenmap is started, a dialog is displayed requesting that you
type your password. Users with
administrator privileges
           <a class="indexterm" name="idm214695799424">
           </a>
           may enter their
password to allow Zenmap to run as the root user and run more advanced
scans. To run Zenmap in unprivileged mode, select the
           <span class="quote">
            
            <span class="quote">
             Cancel
            </span>
            
           </span>
           button on this authentication dialog.
          </p>
         </div>
