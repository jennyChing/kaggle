     <div class="row">
      <div class="col-sm-9 col-xs-12">
       <h2 class="row blogTitle" style="margin:0 -15px;">
        Bulk Update Ad Group Labels with an AdWords Script
       </h2>
      </div>
      <div class="col-sm-3 col-xs-12 single-post-social">
       <div class="row sharedFigure">
        51
       </div>
       <div class="row sharedUnit">
        SHARES
       </div>
      </div>
      <div class="col-xs-12">
       <p class="row meta">
        By
        <a href="http://white.net/blog/author/tamsin-mehew/">
         Tamsin Mehew
        </a>
        / June 17, 2014
       </p>
      </div>
     </div>
     <div class="row">
      <p style="text-align: left">
       AdWords Labels! Theyre so useful! At least they are once youve got them applied.
      </p>
      <p style="text-align: left">
       You can use them to segment ad groups by product types, or tag recently changed bids, or see how a multi-ad-group ad test is going with ease but for some reason, two years after their announcement, you still cant bulk edit labels. You cant even see them in Editor. Youve got to manually add them in the online interface.
      </p>
      <p style="text-align: left">
       You could
       <a href="http://www.passiondigital.co.uk/blog/2014/03/06/creating-labels-adwords-editor/">
        use comments in Editor instead of labels
       </a>
       , but its hard to make this work if youre in a team: comments arent shared automatically. Also you cant see totals, and there isnt an easy way to get a time series.
      </p>
      <p style="text-align: left">
       There is, however, an alternative: AdWords Scripts can read data from Google Doc spreadsheets, and they can create and apply labels  so all you need to do is have a bulk sheet in a Google Doc and let a Script do the hard work.
      </p>
      <p style="text-align: left">
       Heres how to do that to apply labels to ad groups.
      </p>
      <h2 style="text-align: left">
       The Google Doc
      </h2>
      <p style="text-align: left">
       Just go to Google Drive, create a spreadsheet, and fill it in a bit like this:
       <br/>
       <a href="http://d23ptq4fzmxqnk.cloudfront.net/blog/wp-content/uploads/2014/06/Labels-Google-Doc-1.png">
        <img alt="A spreadsheet with columns 'Campaign', 'Ad Group', 'Label' and 'Notes'" class="aligncenter wp-image-16653 size-full" height="217" src="http://d23ptq4fzmxqnk.cloudfront.net/blog/wp-content/uploads/2014/06/Labels-Google-Doc-1.png" width="616"/>
       </a>
      </p>
      <p style="text-align: left">
       Leave the Notes column blank  it isnt for your notes, its where the Script will say if its applied the label, or if theres been a problem.
      </p>
      <h2 style="text-align: left">
       The AdWords Script
      </h2>
      <p style="text-align: left">
       In your AdWords account go to Bulk operations, then Scripts, then click the red + SCRIPT button. Name it something sensible (like Ad Group Labeller) and copy and paste in this code:
      </p>
      <p>
       <code>
        function main() {
        <br/>
        // Applies labels to specified ad groups, based on a Google Doc Spreadsheet.
        <br/>
        // From http://white.net/blog/bulk-update-ad-group-labels-adwords-script/
        <br/>
        //Change this to your spreadsheet's URL!
        <br/>
        var spreadsheetUrl = "https://docs.google.com/spreadsheets/d/YOUR-URL-HERE";
        <br/>
        var inputSheet = SpreadsheetApp.openByUrl(spreadsheetUrl).getActiveSheet(); //The sheet in the Google Doc
        <br/>
        var i = 2; //Integer for the loop
        <br/>
        var numberOfLabels = inputSheet.getLastRow() //Number of rows in the sheet, which is the number of ad groups that want to be labelled
        <br/>
        while (i &lt;= numberOfLabels) //This loops through all the rows
        <br/>
        {
        <br/>
        if (inputSheet.getRange("D"+i).getValue() == "") //If there are no notes for the current row
        <br/>
        {
        <br/>
        var labelCampaignName = inputSheet.getRange("A" + i).getValue();
        <br/>
        var labelAdGroupName = inputSheet.getRange("B" + i).getValue();
        <br/>
        var labelText =inputSheet.getRange("C" + i).getValue();
        <br/>
        var printToSpreadsheet = "-"
        <br/>
        //This is a variable that records what will be noted in the spreadsheet for each ad group (whether the label has been added, or there was an error)
        <br/>
        var adGroupIterator = AdWordsApp.adGroups() //Finds the ad groups with the specified name and campaign name
        <br/>
        .withCondition("CampaignName = '" + labelCampaignName + "'")
        <br/>
        .withCondition("Name = '" + labelAdGroupName +"'")
        <br/>
        .get();
        <br/>
        if (adGroupIterator.hasNext()) //If there is an ad group then the label will be applied
        <br/>
        {
        <br/>
        var adGroup = adGroupIterator.next();
        <br/>
        AdWordsApp.createLabel(labelText); //Creates the label - if the label already exists then there may be an error, but the Script will continue
        <br/>
        adGroup.applyLabel(labelText); //Applies the label to the group
        <br/>
        printToSpreadsheet = "Done";
        <br/>
        }
        <br/>
        else
        <br/>
        {
        <br/>
        //This means the ad group iterator was empty, so there wasn't an ad group of the right name (or a campaign of the right name).
        <br/>
        printToSpreadsheet = "Ad group not found"; //An error message will be recorded in the spreadsheet.
        <br/>
        }
        <br/>
        inputSheet.getRange("D" + i).setValue(printToSpreadsheet); //Writes into the Notes column in the spreadsheet
        <br/>
        Logger.log("Label " + labelText + " applied to '" + labelAdGroupName + "' in '"+ labelCampaignName +"':" + printToSpreadsheet); //Also writes into the Log
        <br/>
        }//End 'if there are notes'
        <br/>
        i = i+1;
        <br/>
        }//End while loop
        <br/>
        }
       </code>
      </p>
      <p style="text-align: left">
       Remember to change the spreadsheet URL, so that it looks at the right spreadsheet!
      </p>
      <p style="text-align: left">
       You need to click the Authorise now button before the Script can run (dont worry  it wont make changes aside from adding labels). Then you can click Run script now and see if it works.
      </p>
      <h2 style="text-align: left">
       The Results
      </h2>
      <p style="text-align: left">
       Check your ad groups  they should be labelled!
      </p>
      <p style="text-align: left">
       Also check the Notes column in your Google Doc:
       <br/>
       <a href="http://d23ptq4fzmxqnk.cloudfront.net/blog/wp-content/uploads/2014/06/Labels-Google-Doc-2.png">
        <img alt="The spreadsheet, now with the Notes filled in" class="aligncenter wp-image-16654 size-full" height="214" src="http://d23ptq4fzmxqnk.cloudfront.net/blog/wp-content/uploads/2014/06/Labels-Google-Doc-2.png" width="635"/>
       </a>
      </p>
      <p style="text-align: left">
       Here you see theres been a problem with the last ad group  it looks like Ive mistyped the ad group name. So I can either add a row to the bottom of the spreadsheet with the correct ad group name, or I could apply the label manually.
      </p>
      <p style="text-align: left">
       If the Script timed out then there would be rows without anything in the Notes column: in that case you can just try running the Script again.
      </p>
      <p style="text-align: left">
       And next time you want to label a bunch of groups, you can replace the contents of the Doc with the new set of groups to label and run the Script again  without any more messing with code.
      </p>
      <h2 style="text-align: left">
       Note  Youre Likely to See Errors in the Summary
      </h2>
      <p style="text-align: left">
       When you run this Script theres likely to be errors saying Youre already using this label name. Thats because AdWords needs you to create a label before applying it to something, and I havent worked out an efficient way to check if a label exists: so the Script just tries to create a new label every time, and if it already exists then theres an error message which is just ignored. If anyone has a better idea how to handle this, please comment!
      </p>
      <p style="text-align: left">
       On the other hand, you
       <em>
        wont
       </em>
       see errors in the summary if ad groups havent been found: you need to check the Google Doc for that.
      </p>
      <h2 style="text-align: left">
       Alternatives
      </h2>
      <p style="text-align: left">
       <a href="http://white.net/services/pay-per-click/">
        PPC
       </a>
       Hero also have
       <a href="http://www.ppchero.com/labels-in-the-adwords-editor-is-it-but-a-dream/">
        a Script for applying labels to keywords
       </a>
       , but it applies the same label to a list of keywords (and the label text is hard-coded, so youd need to change the code manually to make different labels), and it doesnt have a way of coping if the Script times out. Also it goes through all keywords and then checks if they match one in the list  Im not sure if this is more efficient in general than my grab the data lots of times approach, but I suspect its not as good if you only want to label a few keywords in a large account.
      </p>
      <h2 style="text-align: left">
       Update: Here are

        the scripts for labelling ads and keywords
       </a>
      </h2>
      <p style="text-align: left">
       So, does this Script work for you? Are there other similar Scripts I havent found yet? Have you found a more efficient way to work with labels? If youve got any comments then write in below
      </p>
