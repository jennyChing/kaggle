           <p class="newslettertitle">
            <font color="red">
             Design Tip:
            </font>
            <br/>
            Web Site Usability Checklist
           </p>
           <p>
            by
            <a href="#" onclick="window.open('/news/email/mailto.htm','Mailto','width=440,height=540,scrollbars=no,resizable=yes'); return true;">
             Larisa Thomason
            </a>
            ,
            <br/>
            Senior Web Analyst,
            <br/>
            NetMechanic, Inc.
           </p>
           <table align="right" border="0" cellpadding="0" cellspacing="0" width="154">
            <tr>
             <td>
             </td>
             <td>
              <table bgcolor="#003399" border="0" cellpadding="2" cellspacing="0" width="144">
               <tr>
                <td>
                 <table align="center" border="0" cellpadding="5" cellspacing="0" width="138">
                  <tr>
                   <td align="center" bgcolor="#003399" class="newslettercurrent">
                    <b>
                     February 2004 (Part 2)
                     <br/>
                     Vol. 7, No. 4
                    </b>
                   </td>
                  </tr>
                  <tr>
                   <td bgcolor="#a3c2ff" class="newsletterpastissues">
                    
                    <a href="/news/vol7/promo_no4.htm">
                     Promotion Tip
                    </a>
                    <br/>
                    Design Tip
                    <br/>
                    
                    <a href="/news/vol7/beginner_no4.htm">
                     Beginner Tip
                    </a>
                    <br/>
                   </td>
                  </tr>
                 </table>
                </td>

              </table>
             </td>
             <td>
             </td>
            </tr>
           </table>
           <p align="justify">
            Search engine promotion is important: most visitors use a search engine to find your site.  But once they arrive, they have to be able to actually use the site and understand the content.  Otherwise, your site is a waste of their time.
           </p>
           <p align="justify">
            Our usability checklist highlights five important components of a usable Web site.  This month's Webmaster Tips issue also contains handy, annotated checklists that address other important areas:
           </p>
           <table>
            <tr>
             <td>
              <li>
               <a href="http://www.netmechanic.com/news/vol7/beginner_no4.htm">
                Prelaunch Checklist
               </a>
               <br/>
              </li>
              <li>
               <a href="http://www.netmechanic.com/news/vol7/promo_no4.htm">
                Promotion Checklist
               </a>
              </li>
             </td>
            </tr>
           </table>
           <br/>
           <table>
            <tr>
             <td>
              <ol>
               <li>
                <b>
                 Design a clear and simple navigation system.
                </b>
                According to Web usability expert, Jakob Nielsen, a good navigation system should answer three questions:
                <br/>
                <br/>
                <ol>
                 <li>
                  Where am I?
                 </li>
                 <li>
                  Where have I been?
                 </li>
                 <li>
                  Where can I go?
                 </li>
                </ol>
                <br/>
                <br/>
                Your site's navigation system will answer all three questions if you're careful to include these basic elements:
                <br/>
                <br/>
                <ul type="square">
                 <li>
                  <b>
                   Keep it consistent.
                  </b>
                  The navigation system should be in the same place on every page and have the same format.  Visitors will get confused and frustrated if links appear and disappear unpredictably.  Consider using
                  <a href="http://www.netmechanic.com/news/vol2/server_no8.htm">
                   Server Side Includes
                  </a>
                  for your main navigation system to make certain the navigation system stays consistent.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Use appropriate text inside links.
                  </b>
                  Don't make your visitors guess where a link is going to take them.  Visitors should be able to anticipate a link's destination by reading the text in the link or on the navigation button.  This isn't the time to be cute or obscure - visitors don't have the time or patience for it.
                  <br/>
                  <br/>
                  If there's any question about a link's destination, clarify the issue with a
                  <a href="http://www.netmechanic.com/news/vol5/usability_no7.htm">
                   TITLE attribute
                  </a>
                  that explains exactly where the link goes.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Use CSS to emphasize text links.
                  </b>
                  Some designers dislike underlined text links inside page content - although visitors expect to be able to click on underlined text.  If you decide to remove this important visual navigation clue,
                  <a href="http://www.netmechanic.com/news/vol4/css_no22.htm">
                   style your links with CSS
                  </a>
                  to replace underlining with another, consistent visual technique like a background color, different font, or text color that indicates a hyperlink.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Always include text links.
                  </b>
                  You can create some great looking menus using JavaScript or other scripting language, but never rely completely on a dynamic menu system.  Some users may have problems using a mouse to navigate through the menu and others may be listening to the page using a screen reader.  Every page should have basic text links that link to all major sections of the site.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Add a text-based site map.
                  </b>
                  Large or complex sites should always have a
                  <a href="http://www.netmechanic.com/news/vol5/usability_no20.htm">
                   text-based site map
                  </a>
                  in addition to text links.  Every page should contain a text link to the site map.  Lost visitors will use it to find their way, while search engines spiders will have reliable access to all your pages.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Include a home page link inside your main navigation system.
                  </b>
                  Visitors may enter your site via an internal page, but hopefully they'll want to head for the home page next.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Site logo links to home page.
                  </b>
                  Most sites include their logo somewhere at the top of every page - generally in the top, left-hand corner.  Visitors expect this logo to be a
                  <a href="http://www.netmechanic.com/news/vol2/design_no9.htm">
                   link to your site's home page
                  </a>
                  .  They'll often go there before looking for the home link in the navigation system.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Include a site search box.
                  </b>
                  A robust
                  <a href="http://www.netmechanic.com/news/vol5/design_no16.htm">
                   site search feature
                  </a>
                  helps visitors quickly locate the information they want.  Make the search box prominent and be sure that it searches all of your site - and only your site.  We've run across far too many Web sites that include a "Search the Web" search box on their home page.  The result?  Visitors hardly get to the site before the search function sends them to another site!
                 </li>
                </ul>
                <br/>
                <br/>
               </li>
               <li>
                <b>
                 Keep the content clear and simple.
                </b>
                You may attract visitors with an eye-catching design, but content is what keeps them at the site and encourages them to return.  Content is also the best way to boost your site in search engine rankings.
                <br/>
                <br/>
                Always keep search engines in mind when you write content, but remember that your ultimate audience is human visitors.  Present your content with humans in mind.
                <br/>
                <br/>
                <ul type="square">
                 <li>
                  <b>
                   Don't save the best for last.
                  </b>
                  Place your most important content high on the page.  Think of a newspaper: the top story is always prominently displayed above the fold.  Check your page display at in a number of different screen resolutions to make sure that your most important content is visible when the page loads.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Make page content easy to scan.
                  </b>
                  You'll spend hours - maybe days - writing your page content and it's really annoying to think that visitors may read less than half of it.  Format your content so that it's easy to scan.  Emphasize important points (or product characteristics) with a combination of header tags, bold type, color, or lists.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Avoid using text inside images whenever possible.
                  </b>
                  Text in images is invisible to search engine spiders and to visitors who may have images turned off in their browsers or who use assistive technologies like screen readers.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Add ALT and TITLE attributes to all images.
                  </b>
                  Each image should have a
                  <a href="http://www.netmechanic.com/news/vol4/promo_no7.htm">
                   descriptive ALT attribute
                  </a>
                  and
                  <a href="http://www.netmechanic.com/news/vol6/html_no1.htm">
                   TITLE attribute
                  </a>
                  associated with it - particularly images that are also links to other pages.  That way, they can quickly jump to the page they're interested in without having to wait for the entire page to load.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Contrast, contrast, contrast!
                  </b>
                  Be careful with
                  <a href="http://www.netmechanic.com/news/vol3/beginner_no11.htm">
                   background images and colors
                  </a>
                  because they can obscure the text content on the page.  Make sure you have a good reason to deviate from the successful dark text on a light background model.  Visitors can't buy your products if they can't read the content.
                 </li>
                </ul>
                <br/>
                <br/>
               </li>
               <li>
                <b>
                 Support your brand.
                </b>
                A good brand creates or reinforces a user's impression of the site.  When your site is strongly branded, that means that visitors will think of you first when they go shopping for your product or service.
                <br/>
                <br/>
                Branding on a Web site takes time, effort, and close attention to page design and layout.
                <br/>
                <br/>
                <ul type="square">
                 <li>
                  <b>
                   Keep colors and typefaces consistent.
                  </b>
                  Visitors should never click on an internal link in your site and wonder if they've left your Web site.
                  <a href="http://www.netmechanic.com/news/vol2/design_no7.htm">
                   Choose your colors
                  </a>
                  and fonts carefully and use them consistently throughout the site.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Keep page layout consistent.
                  </b>
                  Use a
                  <a href="http://www.netmechanic.com/news/vol7/design_no2.htm">
                   Web site template
                  </a>
                  to enforce a uniform page structure.  Visitors should be able to predict the location of important page elements after visiting just one page in your site.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Custom error page.
                  </b>
                  Create a useful custom error page that helps visitors if they should click on a broken internal link or type a URL incorrectly.  The
                  <a href="http://www.netmechanic.com/news/vol2/server_no6.htm">
                   custom error page
                  </a>
                  should reflect the site's overall color, type, and layout structure as much as possible and provide useful links to help visitors find what they're looking for.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Create a good tagline and use it on every page.
                  </b>
                  A
                  <a href="http://www.netmechanic.com/news/vol5/promo_no15.htm">
                   good tagline
                  </a>
                  clearly and concisely explains your "value proposition" or what makes your site stand out from competing sites.  It should be memorable and reinforce your brand in one quick phrase.
                 </li>
                </ul>
                <br/>
                <br/>
               </li>
               <li>
                <b>
                 Provide for visitor feedback.
                </b>
                Forms are critical to the success of ecommerce sites.  Without forms, you can't have a shopping cart.  But any site usually needs at least one form to allow for user feedback.  A form helps you
                <a href="http://www.netmechanic.com/news/vol4/design_no21.htm">
                 hide from email spiders
                </a>
                and also helps you control how user feedback is formatted and sent.
                <br/>
                <br/>
                <ul type="square">
                 <li>
                  <b>
                   Keep feedback forms short and clearly note which information is required to successfully submit the form.
                  </b>
                  Take care to
                  <a href="http://www.netmechanic.com/news/vol5/accessibility_no19.htm">
                   design accessible forms
                  </a>
                  that all visitors can use.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Remember your international users and don't require information they may not have - like area codes or ZIP codes.
                  </b>
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Present complete contact information including your business phone number and postal address.
                  </b>
                  A street address is preferred, but you may want to use a PO box if yours is a home-based business.  Visitors will probably prefer to contact you using email or a form, but they feel more comfortable with a site that allows other contact methods.
                 </li>
                </ul>
                <br/>
                <br/>
               </li>
               <li>
                <b>
                 Test the site on real users.
                </b>
                Remember that you're the designer so of course you effortlessly use the navigation system, love the content, and understand the value proposition.  But now it's time to get user feedback - before your online users start sending it in.
                <br/>
                <br/>
                <a href="http://www.netmechanic.com/news/vol5/usability_no14.htm">
                 Usability testing
                </a>
                <b>
                 helps you replicate the experience of the average Web site user and correct problems before online visitors find them.
                </b>
                It also gives you valuable answers to other questions:
                <br/>
                <br/>
                <ul type="square">
                 <li>
                  <b>
                   Do visitors enjoy using the site?
                  </b>
                  If so, they'll stay longer and read more content.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Do they understand the purpose of the site?
                  </b>
                  If not, there's no compelling reason to return.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Is there any incentive to return after the first visit?
                  </b>
                  Your site should try to be the ultimate authority on the Web for your topic.  A site with depth and breadth encourages visitors to bookmark it and refer friends interested in the same topic.
                  <br/>
                  <br/>
                 </li>
                 <li>
                  <b>
                   Can they recover from errors?
                  </b>
                  Usability testing is the best way to test how well your site search, site map, forms, and custom error pages function.  They should all work together to guide a visitor through the site and help him get where he's going.  Frustrated visitors aren't likely to return - ever.
                 </li>
                </ul>
               </li>
              </ol>
             </td>
            </tr>
           </table>
           <p align="justify">
            Web usability means designing for your visitors instead of for yourself or your client.  A site that
            <a href="http://www.netmechanic.com/news/vol6/design_no1.htm">
             conforms to user expectations
            </a>
            makes visitors more comfortable and more apt to visit again and recommend the site to their friends.  Good usability is critical to your site's success.
           </p>

			myMeta=myDocumentElements.item(i-1);
			if (myMeta.name){
				if (myMeta.name.indexOf('WT.')==0){
					WT[myMeta.name.substring(3)]=myMeta.content;
				}
				else if (myMeta.name.indexOf('DCSext.')==0){
					DCSext[myMeta.name.substring(7)]=myMeta.content;
				}
				else if (myMeta.name.indexOf('DCS.')==0){
					DCS[myMeta.name.substring(4)]=myMeta.content;
				}
			}
		}
	}
}

}
