   <h1 class="firstHeading" id="firstHeading" lang="en">
    Any key
   </h1>
   <div class="mw-body-content" id="bodyContent">
    <div id="siteSub">
     From Wikipedia, the free encyclopedia

     <p>
      Computer programmers historically used "Press
      <b>
       any key
      </b>
      to continue" (or a similar text) as a prompt to the user when it was necessary to pause processing. The system would resume after the user pressed any
      <a class="mw-redirect" href="/wiki/Keyboard_(computing)" title="Keyboard (computing)">
       keyboard
      </a>
      button.
     </p>
     <h2>
      <span class="mw-headline" id="History">
       History
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Any_key&amp;action=edit&amp;section=1" title="Edit section: History">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Early computers were typically operated using mechanical
      <a href="/wiki/Teleprinter" title="Teleprinter">
       teleprinters
      </a>
      , which provided a continuous printed record of their output. However, during the 1970s, these became obsolete and were replaced with
      <a href="/wiki/Computer_terminal" title="Computer terminal">
       visual display units
      </a>
      , and text was lost once it scrolled off the top of the screen. To compensate, programs typically paused operation after displaying one screen of data, so that the user could observe the results and then press a key to move to the next screen.
     </p>
     <p>
      A similar pause was also required when some hardware action was required from the user, such as inserting a
      <a href="/wiki/Floppy_disk" title="Floppy disk">
       floppy disk
      </a>
      or loading a
      <a href="/wiki/Printer_(computing)" title="Printer (computing)">
       printer
      </a>
      with paper.
     </p>
     <p>
      These prompts were commonplace on text-based
      <a href="/wiki/Operating_system" title="Operating system">
       operating systems
      </a>
      prior to the development of
      <a href="/wiki/Graphical_user_interface" title="Graphical user interface">
       graphical user interfaces
      </a>
      , which typically included
      <a href="/wiki/Scrollbar" title="Scrollbar">
       scrollbars
      </a>
      to enable the user to view more than one screen/window of data. They are therefore no longer required as a means of paginating output, but the graphical equivalent (such as "Click OK to continue") is still used for hardware interactions.
     </p>
     <p>
      The prompt is not strictly accurate in that, for the vast majority of computer systems,
      <a href="/wiki/Modifier_key" title="Modifier key">
       modifier keys
      </a>
      or
      <a href="/wiki/Lock_key" title="Lock key">
       lock keys
      </a>
      would not cause processing to resume, as they do not produce an actual character that the program could detect.
     </p>
     <p>
      Some Samsung remote controls for DVD players, as is the case of DVD-R130, have included an "anykey" to their interface. It is used to view the status of the DVD being watched.
      <sup class="reference" id="cite_ref-1">
       <a href="#cite_note-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <h2>
      <span class="mw-headline" id="Cultural_significance">
       Cultural significance
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Any_key&amp;action=edit&amp;section=2" title="Edit section: Cultural significance">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      A 1982
      <a class="mw-redirect" href="/wiki/Apple_Computer" title="Apple Computer">
       Apple Computer
      </a>
      manual for developers warned:
      <sup class="reference" id="cite_ref-apple1982_2-0">
       <a href="#cite_note-apple1982-2">
        <span>
         [
        </span>
        2
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <blockquote class="templatequote">
      <p>
       Do not tell the user to "press any key." ... On the Apple II series computers, you cannot read every key by itself:
       <code>
        RESET
       </code>
       ,
       <code>
        SHIFT
       </code>
       ,
       <code>
        CONTROL
       </code>
       . We have also found in testing that new users, in particular, panic when asked to press any key. Over 80% of them will turn around and say, "but what key should I press?" In questioning them about this response, we discovered that they are quite convinced that even though the prompt implied that all keys were OK to press, some could be dangerous. Of course, they were quite right.
      </p>
     </blockquote>
     <p>
      There are reports from as early as 1988 that some users have searched for such a key labelled "any", and called technical support when they have been unable to find it.
      <sup class="reference" id="cite_ref-cgw198811_3-0">
       <a href="#cite_note-cgw198811-3">
        <span>
         [
        </span>
        3
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-4">
       <a href="#cite_note-4">
        <span>
         [
        </span>
        4
        <span>
         ]
        </span>
       </a>
      </sup>
      The computer company
      <a href="/wiki/Compaq" title="Compaq">
       Compaq
      </a>
      <sup class="reference" id="cite_ref-5">
       <a href="#cite_note-5">
        <span>
         [
        </span>
        5
        <span>
         ]
        </span>
       </a>
      </sup>
      even edited their
      <a href="/wiki/FAQ" title="FAQ">
       FAQ
      </a>
      to explain that the "any" key does not exist,
      <sup class="reference" id="cite_ref-Inquirer_6-0">
       <a href="#cite_note-Inquirer-6">
        <span>
         [
        </span>
        6
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-Register_7-0">
       <a href="#cite_note-Register-7">
        <span>
         [
        </span>
        7
        <span>
         ]
        </span>
       </a>
      </sup>
      and at one point considered replacing the command "Press any key" with "Press
      <a class="mw-redirect" href="/wiki/Return_key" title="Return key">
       return key
      </a>
      ".
      <sup class="reference" id="cite_ref-8">
       <a href="#cite_note-8">
        <span>
         [
        </span>
        8
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      The concept of the "any key" has become a popular piece of computer-related humor,
      <sup class="reference" id="cite_ref-9">
       <a href="#cite_note-9">
        <span>
         [
        </span>
        9
        <span>
         ]
        </span>
       </a>
      </sup>
      in part because of
      <a class="mw-redirect" href="/wiki/King_Size_Homer" title="King Size Homer">
       an episode
      </a>
      of
      <i>
       <a href="/wiki/The_Simpsons" title="The Simpsons">
        The Simpsons
       </a>
      </i>
      in which main character
      <a href="/wiki/Homer_Simpson" title="Homer Simpson">
       Homer Simpson
      </a>
      asks where the "any key" is when confronted with the "press any key" prompt.
     </p>
     <p>
      Plastic "any keys" with adhesive backings are available as
      <a href="/wiki/Novelty_item" title="Novelty item">
       novelty gifts
      </a>
      .
      <sup class="reference" id="cite_ref-10">
       <a href="#cite_note-10">
        <span>
         [
        </span>
        10
        <span>
         ]
        </span>
       </a>
      </sup>
      In most cases the Any key is the space bar.
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (August 2014)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
     </p>
