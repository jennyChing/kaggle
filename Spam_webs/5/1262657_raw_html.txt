     <div class="md-12">
      <h1 class="page-header entry-title">
       How I monitor my VPS for free!
      </h1>
     </div>
     <article class="md-12 post">
      <p class="tldr entry-summary">
       Home made server monitoring with SMS alerts.
      </p>
      <div class="entry-select">
       <p>
        <strong>
         As I have moved over to a VPS server in the past 6 months I have run up agains the paranoia of is my server still running?
        </strong>
       </p>
       <p>
        I had been using
        <a href="http://pingdom.com">
         pingdom.com
        </a>
        ,a great service that pings a site every couple of minutes and notifies you of an outage. This was great when I was on a shared server and could more or less rely on things booting back up. Since I moved to a VPS where everything is my own problem I learned that it wont help if there is a DNS issue or even a Database issue. The site has to return a 404.
       </p>
       <p>
        I started looking into other solutions such as
        <a href="http://statuspage.io">
         statuspage.io
        </a>
        ( back when they had a free account, they have since removed that plan ) but it required some javascript on your error pages. This meant that your visitors would trigger the necessary outage message. Something I didnt want to have happen and probably would not have let me know of the issue until it was a problem.
       </p>
       <p>
        <strong>
         So I set out to find a solution.
        </strong>
       </p>
       <h2>
        The Goal
       </h2>
       <ul>
        <li>
         Monitor any DNS issue and make sure a status of 200 is returned
        </li>
        <li>
         Make sure that a connection to MySQL can be made
        </li>
        <li>
         Make sure that a domain returns a status of 200is returned
        </li>
       </ul>
       <h2>
        The Tools
       </h2>
       <p>
        Since P
        <span style="line-height: 1.6em;">
         ingdom already does an excellent job pinging my site I knew that I could rely on it to keep up the good work.
        </span>
       </p>
       <p>
        <span style="line-height: 1.6em;">
         I get a ping every 5 min using the free plan.
        </span>
       </p>
       <p>
        <span style="line-height: 1.6em;">
         Pingdom will send you an email when your site is down but again they wont help for any Apache or MySQL issues.
        </span>
       </p>
       <p>
        <span style="line-height: 1.6em;">
         So I chose to use
        </span>
        <a href="http://www.twilio.com" style="line-height: 1.6em;">
         Twillio
        </a>
        <span style="line-height: 1.6em;">
         , as their free account would send text messages ( you should be paying for this ).
        </span>
       </p>
       <p>
        <span style="line-height: 1.6em;">
         I hosted the script on my VPS because I dont allow outside MySQL connections and I had no reason to open one up. Remember, Pingdom will email me if everything goes pear shaped.
        </span>
       </p>
       <h2>
        Execution
       </h2>
       <p>
        <strong>
         Lets start by checking the DNS:
        </strong>
        using
        <a href="http://www.php.net/manual/en/function.dns-get-record.php" target="_blank">
         dns_get_record()
        </a>
        , It is important to note that the domain cant not contain the protocol or the subomain ( www ).
       </p>
       <pre data-pbcklang="php" data-pbcktabsize="4">
        <code class="language-php ">
         $url = 'tentaclecms.com';
$dns = dns_get_record($url);

if ($dns == null) {

    $data['dns']['success'] = 'false';
    $data['dns']['message'] = 'Problem connecting to the DNS. ';
    $data['dns']['status'] = 'red';
}
else {
    $data['dns']['success'] = 'true';
    $data['dns']['message'] = 'The DNS is good. ';
    $data['dns']['status'] = 'green';
}
        </code>
       </pre>
       <p>
        <strong>
         Next, lets make sure we can make a connection to the database:
        </strong>
       </p>
       <pre data-pbcklang="php" data-pbcktabsize="4">
        <code class="language-php ">
         $host = 'localhost';
$username = 'root';
$password = '';
$driver = 'mysql';
$table = 'table_name';

try {
    $pdo = new pdo( $driver.':host='.$host.';dbname='.$table.'',
        $username,
        $password,
        array(PDO::ATTR_ERRMODE =&gt; PDO::ERRMODE_EXCEPTION));

    $data['mysql']['success'] = 'true';
	$data['mysql']['message'] = 'The database connection is good. ';
	$data['mysql']['status'] = 'green';
}
catch(PDOException $ex){
    $data['mysql']['success'] = 'false';
	$data['mysql']['message'] = 'Problem connecting to the database. ';
	$data['mysql']['status'] = 'red';
}
        </code>
       </pre>
       <p>
        <strong>
         Checking the response headers:
        </strong>
       </p>
       <pre data-pbcklang="php" data-pbcktabsize="4">
        <code class="language-php ">
         function get_contents($url) {
    $http_response_header = '';
    @file_get_contents($url);

    return $http_response_header;
}

$server = get_contents('http://www.'.$url);

if (is_array($server)) {
    if (strpos($server[0], '200'))
    {
        $data['server']['success'] = 'true';
        $data['server']['message'] = 'OK, The page loaded correctly. ';
        $data['server']['status'] = 'green';
    } elseif (strpos($server[0], '404'))
    {
        $data['server']['success'] = 'true';
        $data['server']['message'] = 'The page is missing. ';
        $data['server']['status'] = 'yellow';
    } elseif (strpos($server[0], '500'))
    {
        $data['server']['success'] = 'false';
        $data['server']['message'] = 'There is an internal server error. ';
        $data['server']['status'] = 'red';
    } 
    
} else {
    $data['server']['success'] = 'false';
    $data['server']['message'] = 'Not sure whats wrong, the URL is missing. ';
    $data['server']['status'] = 'red';
}
        </code>
       </pre>
       <p>
        <strong>
         Lets do something with our Array.
        </strong>
       </p>
       <p>
        From this point I can now loop through the $data array to see if anything failed.
       </p>
       <p>
        Have a look at the
        <a href="http://www.twilio.com/docs/howto/sms-notifications-and-alerts">
         Twillio example
        </a>
        , Download and include the
        <strong>
         Services/Twilio.php.
        </strong>
        When we come across an error with our site we can trigger a SMS message to notify us of the problem. If the site its self is totally unreachable Pingdom will let us know.
       </p>
       <p>
        Make a function called
        <strong>
         sms
        </strong>
        <a href="http://www.twilio.com/docs/quickstart/php/sms/sending-via-rest" target="_blank">
         http://www.twilio.com/docs/quickstart/php/sms/sending-via-rest
        </a>
       </p>
       <pre data-pbcklang="php" data-pbcktabsize="4">
        <code class="language-php ">
         function sms ( $message ) {
	require "Services/Twilio.php";

	$AccountSid = "";
	$AuthToken = "";

	$client = new Services_Twilio($AccountSid, $AuthToken);

	$people = array( "+yournumber" =&gt; "Your Name" );

	foreach ($people as $number =&gt; $name) {
		$sms = $client-&gt;account-&gt;sms_messages-&gt;create(
			"twillionumber", 
			$number,
            $message
		);
	}	
}
        </code>
       </pre>
       <p>
        When we hit a problem we can pass the
        <strong>
         $data['server']['message']
        </strong>
        to sms().
       </p>
       <p>
        If SMS is not your thing then send an email.
       </p>
       <pre data-pbcklang="php" data-pbcktabsize="4">
        <code class="language-php ">
         function alert( $message ) {
    mail('to@gmail.com', 'Subject', $message);
}
        </code>
       </pre>
       <h2>
        Done!
       </h2>
