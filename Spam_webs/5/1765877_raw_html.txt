  <h1>
   Quadrature on sparse grids:
  </h1>
  <h2>
   Code to generate and readily evaluated nodes and weights
  </h2>
  To accompany the paper "Likelihood Approximation by Numerical Integration on Sparse Grids"
  <br/>
  by
  <a href="#Contact">
   Florian Heiss &amp; Viktor Winschel
  </a>
  <br/>
  <br/>
  This website offers three options to use quadrature on sparse grids:
  <ul>
   <li>
    <a href="#Matlab">
     Matlab code
    </a>
   </li>
   <li>
    <a href="#Stata">
     Stata code
    </a>
   </li>
   <li>
    <a href="#Nodes">
     Readily evaluated nodes and weights
    </a>
   </li>
  </ul>

   Updates
  </h2>

  </a>

  <h2>
   General remarks
  </h2>
  <p>
   To approximate a
   <i>
    D
   </i>
   -variate integral of the form
   <br/>
   
   <sub>
    
   </sub>
   <i>
    g(x)f(x)
   </i>
   d
   <i>
    x
   </i>
   <br/>
   where 
   <b>
    R
   </b>
   <sup>
    <i>
     D
    </i>
   </sup>
   and
   <i>
    f(x)
   </i>
   is a weight function, a quadrature rule
prescribes a
   <i>
    R
   </i>
   x
   <i>
    D
   </i>
   matrix of nodes
   <i>
    X
   </i>
   and a
   <i>
    R
   </i>
   x
   <i>
    1
   </i>
   vector of weights
   <i>
    w
   </i>
   depending on  and
   <i>
    f
   </i>
   . The approximation is calculated as
   <br/>
   <i>
    g(X)'w
   </i>
   .
  </p>
  <p>
   In all three versions provided here, four quadrature rules for
   <i>
    D
   </i>
   -dimensional numerical integration 
are implemented:
  </p>
  <ul>
   <li>
    Rules for the unweighted integration: =[0,1]
    <sup>
     <i>
      D
     </i>
    </sup>
    ,
    <i>
     f(x)
    </i>
    =1
   </li>
   <ul>
    <li>
     <strong>
      GQU:
     </strong>
     Univariate Gaussian quadrature rules as basis
    </li>
    <li>
     <strong>
      KPU:
     </strong>
     Univariate nested quadrature rules as basis - delayed Kronrod-Patterson rules, 
see Knut Petras (2003): "Smolyak cubature of given polynomial degree with few nodes for
increasing dimension."
     <i>
      Numerische Mathematik
     </i>
     <b>
      93
     </b>
     , 729-753.
    </li>
   </ul>
   <li>
    Rules for the integration over  =
    <i>
     <b>
      R
     </b>
     <sup>
      D
     </sup>
    </i>
    with Gaussian weight
    <i>
     f(x)
    </i>
    =exp(
    <i>
     -x'x
    </i>
    /2)/sqrt(2)
   </li>
   <ul>
    <li>
     <strong>
      GQN:
     </strong>
     Univariate Gaussian quadrature rules as basis
    </li>
    <li>
     <strong>
      KPN:
     </strong>
     Univariate nested quadrature rules as basis, see A. Genz and B. D. Keister (1996):
 "Fully symmetric interpolatory rules for multiple integrals over infinite regions with Gaussian weight."
     <i>
      Journal of Computational and Applied Mathematics
     </i>
     <b>
      71
     </b>
     , 299-309.
    </li>
   </ul>
  </ul>
  <p>
   As a general rule, the versions with the nested univariate rules are more efficient in the sense 
that they require fewer nodes to achieve the same level of polynomial exactness. The
   <a href="#Matlab">
    Matlab code
   </a>
   and the
   <a href="#Stata">
    Stata code
   </a>
   allow the user to use own one-dimensional quadrature rules.
  </p>
  <p>
   The accuracy level in each of the implementations is denoted as
   <i>
    k
   </i>
   in the paper. 
Each rule with accuracy level
   <i>
    k
   </i>
   integrates complete polynomials of total order
   <i>
    2k-1
   </i>
   exactly.
  </p>
  <h2>
   <a name="Matlab">
    Matlab code
   </a>
  </h2>
  <p>
   The Matlab code can be used to extend any univariate quadrature rule to multiple dimensions. The four rules
discussed above are included in the code.
  </p>
  <p>
   To
   <strong>
    install
   </strong>
   the package, perform the following steps:
  </p>
  <ul>
   <li>
    Download
    <a href="matlab/nwspgr.m">
     nwspgr.m
    </a>
    or download and unpack
    <a href="matlab/matlab.zip">
     matlab.zip
    </a>
    .
   </li>
   <li>
    Make sure Matlab will find the file: Put it into the current working directory, or somewhere into the Matlab 
search path. You can view and modify it in the menu, see
    <code>
     File
    </code>
    
    <code>
     Set Path...
    </code>
   </li>
  </ul>
  <p>
   The
   <strong>
    syntax
   </strong>
   is the following:
   <br/>
   <code>
    [n,w]=nwspgr(type,dim,acc,sym)
   </code>
   <br/>
  </p>
  <ul>
   <li>
    <code>
     type
    </code>
    is a string choosing the underlying univariate quadrature rule. It is either
    <code>
     'GQU'
    </code>
    ,
    <code>
     'KPU'
    </code>
    ,
    <code>
     'GQN'
    </code>
    , or
    <code>
     'KPN'
    </code>
    for the built-in
		rules, or the name of a function for user-supplied univariate rules. In the latter case, the user must
		make sure that this function is known to Matlab and that it
    <ul>
     <li>
      accepts one scalar argument as the accuracy level
     </li>
     <li>
      returns a column vector of nodes and a column vector of weights.
     </li>
    </ul>
    for example this function could be defined as
    <code>
     function [n w] = myquadrule(acc)
    </code>
    . In this case, 
			the argument
    <code>
     type
    </code>
    would be the string
    <code>
     'myquadrule'
    </code>
    .
   </li>
   <li>
    <code>
     dim
    </code>
    is the number of dimensions.
   </li>
   <li>
    <code>
     acc
    </code>
    is the accuracy level.
   </li>
   <li>
    <code>
     sym
    </code>
    is optional and only interpreted if a user-supplied univariate quadrature rule is used. 
		In this case, if
    <code>
     sym
    </code>
    is used and a value other than 0 is supplied, this has two effects: 
		(1) the code runs much faster with high
    <code>
     dim
    </code>
    and/or
    <code>
     acc
    </code>
    and (2) the result will 
		only be valid if the underlying univariate rule is "symmetric". This is the case if
    <ul>
     <li>
      it uses one node for accuracy level 1, call it
      <i>
       m
      </i>
     </li>
     <li>
      in higher accuracy levels, the nodes cosist of
      <i>
       m
      </i>
      and/or a set of other nodes. 
			For each of these other nodes
      <i>
       m+d
      </i>
      , there is also a node
      <i>
       m-d
      </i>
      with the same weight.
     </li>
    </ul>
   </li>
   <li>
    The results will be returned as a (
    <i>
     R
    </i>
    x
    <code>
     dim
    </code>
    ) matrix of nodes
    <code>
     n
    </code>
    and a 
		(
    <i>
     R
    </i>
    x1) vector of corresponding weights
    <code>
     w
    </code>
    , where
    <i>
     R
    </i>
    depends on
    <code>
     dim
    </code>
    and
    <code>
     acc
    </code>
    ,
		see the examples
    <a href="#Nodes">
     below
    </a>
    .
   </li>
  </ul>
  <p>
   An
   <strong>
    example
   </strong>
   of the program in action is in
   <a href="matlab/nwSpGr_demo.m">
    nwSpGr_demo.m
   </a>
   .
  </p>
  <h2>
   <a name="Stata">
    Stata code
   </a>
  </h2>
  <p>
   The Stata code will run in Stata versions 9 and above. There are three ways the nodes and weights can be obtained:
  </p>
  <ul>
   <li>
    As a Mata matrix
   </li>
   <li>
    As a Stata matrix
   </li>
   <li>
    As variables
   </li>
  </ul>
  <p>
   To
   <strong>
    install
   </strong>
   the package, perform the following steps:
  </p>
  <ul>
   <li>
    Download
    <a href="stata/stata.zip">
     stata.zip
    </a>
    and unpack its contents to a place where 
Stata will find the files. A natural place is the personal ado directory. If you don't know where it is, Stata's 
command
    <code>
     sysdir
    </code>
    will tell you.
   </li>
   <li>
    In order to re-index the installed functions, either close and reopen Stata or use the command
    <code>
     mata: mata mlib index
    </code>
    .
   </li>
  </ul>
  <p>
   The
   <strong>
    syntax
   </strong>
   is the following:
  </p>
  <ul>
   <li>
    For generating nodes and weights as Stata matrices and/or variables, use the command
    <br/>
    <code>
     nwspgr, type(string)
     <u>
      dim
     </u>
     ensions(integer)
     <u>
      acc
     </u>
     uracy(integer) 
	 [
     <u>
      matn
     </u>
     odes(string)
     <u>
      matw
     </u>
     eights(string)
     <u>
      varn
     </u>
     odes(string)
     <u>
      varw
     </u>
     eights(string) ]
    </code>
    <br/>
    Example:
    <code>
     nwspgr, type(KPN) dim(2) acc(4) varn(x) varw(w)
    </code>
    <br/>
    The first three options determine what is calculated:
    <ul>
     <li>
      <code>
       type(string)
      </code>
      chooses the underlying univariate quadrature rule, where
      <code>
       string
      </code>
      is either
      <code>
       GQU
      </code>
      ,
      <code>
       KPU
      </code>
      ,
      <code>
       GQN
      </code>
      , or
      <code>
       KPN
      </code>
      .
     </li>
     <li>
      <code>
       dim(integer)
      </code>
      determines the number of dimensions.
     </li>
     <li>
      <code>
       acc(integer)
      </code>
      determines the accuracy level.
     </li>
    </ul>
    The other options determine how the results are returned:
    <ul>
     <li>
      <code>
       matn(string)
      </code>
      requests that the matrix of nodes is returned as a matrix named
      <code>
       string
      </code>
      .
     </li>
     <li>
      <code>
       matw(string)
      </code>
      requests that the vector of weights is returned as a matrix named
      <code>
       string
      </code>
      .
     </li>
     <li>
      <code>
       varn(string)
      </code>
      requests that the matrix of nodes is returned as variables named
      <code>
       string1
      </code>
      ,...,
      <code>
       stringD
      </code>
      .
     </li>
     <li>
      <code>
       varw(string)
      </code>
      requests that the vector of weights is returned as a variable named
      <code>
       string
      </code>
      .
     </li>
    </ul>
    Not specifying any of those options is equivalent to specifying
    <code>
     matn(nodes) matw(weights)
    </code>
    .
   </li>
   <li>
    For generating a matrix of nodes and weights from within Stata's matrix language Mata, use the Mata function
    <br/>
    <code>
     nwspgr(type,dim,acc,sym)
    </code>
    <br/>
    Example:
    <code>
     mata: nw = nwspgr("KPN",2,4)
    </code>
    <ul>
     <li>
      <code>
       type
      </code>
      chooses the underlying univariate quadrature rule. It can be one of the strings
      <code>
       "GQU"
      </code>
      ,
      <code>
       "KPU"
      </code>
      ,
      <code>
       "GQN"
      </code>
      , or
      <code>
       "KPN"
      </code>
      if the built-in 
		rules are used. If the user wants to use an own underlying univariate quadrature rule,
      <code>
       type
      </code>
      has to be a pointer to a function. This function
      <ul>
       <li>
        accepts one scalar argument as the accuracy level
       </li>
       <li>
        returns a matrix with two columns, where the first column represents nodes and 
		the second column represents weights.
       </li>
      </ul>
      for example this function could be declared as
      <code>
       real matrix myquadrule(real scalar acc)
      </code>
      . 
		In this case, the argument
      <code>
       type
      </code>
      would be the pointer
      <code>
       &amp;myquadrule()
      </code>
      .
     </li>
     <li>
      <code>
       dim
      </code>
      is an integer determining the number of dimensions.
     </li>
     <li>
      <code>
       acc
      </code>
      is an integer determining the accuracy level.
     </li>
     <li>
      <code>
       sym
      </code>
      is optional and only interpreted if a user-supplied univariate quadrature rule is used. 
		In this case, if
      <code>
       sym
      </code>
      is used and a value other than 0 is supplied, this has two effects: 
		(1) the code runs much faster with high
      <code>
       dim
      </code>
      and/or
      <code>
       acc
      </code>
      and (2) the result will 
		only be valid if the underlying univariate rule is "symmetric". This is the case if
      <ul>
       <li>
        it uses one node for accuracy level 1, call it
        <i>
         m
        </i>
       </li>
       <li>
        in higher accuracy levels, the nodes cosist of
        <i>
         m
        </i>
        and/or a set of other nodes. 
			For each of these other nodes
        <i>
         m+d
        </i>
        , there is also a node
        <i>
         m-d
        </i>
        with the same weight.
       </li>
      </ul>
     </li>
    </ul>
    The result is a
    <i>
     R
    </i>
    x
    <code>
     dim+1
    </code>
    matrix where the last column represents the weights.
   </li>
  </ul>
  <p>
   One
   <strong>
    example
   </strong>
   for each of the types of use is here:
  </p>
  <ul>
   <li>
    Using Mata:
    <a href="stata/demo_mata.do">
     demo_mata.do
    </a>
   </li>
   <li>
    Using Stata variables:
    <a href="stata/demo_var.do">
     demo_var.do
    </a>
   </li>
   <li>
    Using Stata Matrices:
    <a href="stata/demo_matrix.do">
     demo_matrix.do
    </a>
   </li>
  </ul>
  <p>
   The
   <strong>
    source code
   </strong>
   of the compiled Mata library
   <code>
    lnwspgr.mlib
   </code>
   contained in the installation 
ZIP file is not necessary to use the program. For the interest users, here it is:
   <a href="stata/build_nwspgr.do">
    build_nwspgr.do
   </a>
  </p>
  <h2>
   <a name="Nodes">
    Readily evaluated nodes and weights
   </a>
  </h2>
  <p>
   For a variety of combinations of the number of dimensions and the accuracy level, readily generated nodes 
and weights can be downloaded as comma-seperated ASCII files. The file names are
   <br/>
   <i>
    TYPE
   </i>
   _d
   <i>
    DIM
   </i>
   _l
   <i>
    LEV
   </i>
   .asc
   <br/>
   where
   <i>
    TYPE
   </i>
   denotes the quadrature rule and is GQU, KPU, GQN, or KPN,
   <i>
    DIM
   </i>
   is the number of dimensions
with 1
   <i>
    DIM
   </i>
   20, and
   <i>
    LEV
   </i>
   is the accuracy level with 1
   <i>
    LEV
   </i>
   25. The ASCII files for
   <i>
    DIM
   </i>
   dimensions have
   <i>
    DIM
   </i>
   +1 columns, with the last column representing the 
integration weight.
  </p>
  <p>
   The following tables show the number of nodes required by the combinations of
   <i>
    TYPE
   </i>
   ,
   <i>
    DIM
   </i>
   , and
   <i>
    LEV
   </i>
   . Note that since the number of nodes increases with the number of dimensions, rules for higher dimensions are only 
implemented for lower accuracy levels. The files are provides as ZIPped archives - one for each
   <i>
    TYPE
   </i>
   and for each combination of
   <i>
    TYPE
   </i>
   and
   <i>
    DIM
   </i>
   .
  </p>
