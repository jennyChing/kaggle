      <h1 class="firstHeading">
       Creating accounts in Thunderbird for popular email providers
      </h1>
      <div id="bodyContent">
       <h3 id="siteSub">
        From MozillaZine Knowledge Base
       </h3>
       <div id="contentSub">
       </div>
       <!-- start content -->
       <p>
        <i>
         This article provided instructions for email providers that users frequently asked for help on. Due to the remote database used by the new account wizard in Thunderbird 3 its no longer being updated."
        </i>
       </p>
       <p>
        <br/>
        Thunderbird supports using POP and
        <a href="/IMAP" title="IMAP">
         IMAP
        </a>
        mail servers for fetching/reading new messages, and SMTP servers for sending messages. It does not support
        <a href="/Using_webmail_with_your_email_client" title="Using webmail with your email client">
         webmail
        </a>
        . If you want to use webmail (or WebDAV or HTTPMail) with Thunderbird you need to use an add-on that essentially makes it emulate a POP or IMAP account.
       </p>
       <p>
        Your email provider (frequently it's your ISP) creates an account (that you can also access via webmail), gives you permission to access their mail servers, assigns you a email address, username, password etc. Thunderbird doesn't do that, it's just an email client (like Outlook Express, Eudora, etc.) The account you create in Thunderbird is basically just a way to tell Thunderbird how to access your email providers account/mail servers.
       </p>
       <p>
        This
        <a class="external text" href="http://opensourcearticles.com/articles/thunderbird_15/english/part_01" rel="nofollow" title="http://opensourcearticles.com/articles/thunderbird_15/english/part_01">
         tutorial
        </a>
        explains how to create a account in Thunderbird, but it doesn't have any of the information you need to configure it (such as your email address, username, password, mail server etc.). If your email provider isn't listed and the support page on their web site doesn't have instructions for Thunderbird, it's easiest to follow instructions for Netscape or Mozilla as the main difference is the menu command used to display the account settings window.
       </p>
       <p>
        Thunderbird doesn't leave a copy on the POP server when it downloads a new message. Check "
        <a href="/Menu_differences_in_Windows%2C_Linux%2C_and_Mac" title="Menu differences in Windows, Linux, and Mac">
         Tools -&gt; Account Settings
        </a>
        -&gt; Server Settings -&gt; Leave messages on server" if you want to be able to access your mail using either Thunderbird or a browser. You may want to set the number of days it keeps a copy to avoid filling up your webmail mailbox.
       </p>
       <p>
        A few providers may state that you need to make a connection using SSL, while actually requiring TLS.
The two protocols are related but they are not interchangeable - you need to specify the correct protocol. If you can't make a connection using SSL, try either TLS or "TLS if available".
       </p>

       <h2>
        ISP Database
       </h2>
       <p>
        Mozilla Messaging has a list of email provider configuration information at the
        <a class="external text" href="http://ispdb.mozillamessaging.com/" rel="nofollow" title="http://ispdb.mozillamessaging.com/">
         Mozilla ISP database
        </a>
        that is used by the new account wizard added in Thunderbird 3.0. If the account wizard doesn't know about your email provider but it's listed in that database, that information was probably added too late for the current version of Thunderbird. In that case it's not as convenient, but you can still use that information to manually add an account.
       </p>
       <p>
        If your email provider isn't listed please tell them about that database and suggest they add their configuration information.
       </p>

         <a class="external text" href="http://www.comcast.com/Customers/FAQ/FaqCategory.ashx?CatId=264" rel="nofollow" title="http://www.comcast.com/Customers/FAQ/FaqCategory.ashx?CatId=264">

       <dl>
        <dd>
         Comcast's instructions state you should leave both "use secure authentication" and "use secure connection (SSL)" unchecked when configuring the POP account. Some
         <a class="external text" href="http://forums.mozillazine.org/viewtopic.php?f=39&amp;t=769835" rel="nofollow" title="http://forums.mozillazine.org/viewtopic.php?f=39&amp;t=769835">
          users
         </a>
         have enabled "secure authentication" and gotten it to sporadicly work. The SSL support seems to be reliable. It's recommended you use a secure connection (SSL) rather than secure authentication if you don't want to send the password in the clear when logging into the POP server.
        </dd>
       </dl>
       <dl>
        <dd>
         Comcast documents two SMTP configurations, a unsecure connection using port 25 and a secure connection using "TLS if available" and port 587. If you get a error message that the SMTP server may be unavailable or refusing SMTP connections there is a undocumented configuration that several users have gotten to work.  Use port 465 , set "use secure connection" to SSL, check the authentication required box and provide your full Comcast address as the username.
        </dd>
       </dl>

        <dd>
         The Comcast spam filters limit messages to Comcast addresses to six recipients per second. This may occasionally cause new mail to you to be bounced, or prevent you from sending a message. When this problem occurs the sender will get a 4xx error message (temporary failure) and a link for instructions on how to solve the problem.

        <b>
         Hughes.net
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://kb.hughesnet.com/al/12/2/1079.htm#Configuring_Email_in_Mozilla" rel="nofollow" title="http://kb.hughesnet.com/al/12/2/1079.htm#Configuring_Email_in_Mozilla">
          Hughes's instructions for Mozilla
         </a>
        </dd>
        <dd>
         Common problems are not authenticating your hughes.net email account (by logging into their webmail) before using Thunderbird, using .com instead of .net in the mail server, not using your email address as the username and configuring secure connections or authentication. If you were a DIRECWAY customer you can't use that username in the outgoing server settings, you have to setup a hughes.net account in www.myhughesnet.com .
        </dd>
       </dl>
       <dl>
        <dd>
         See
         <a class="external text" href="http://forums.mozillazine.org/viewtopic.php?t=404589" rel="nofollow" title="http://forums.mozillazine.org/viewtopic.php?t=404589">
          this forum thread
         </a>
         or
         <a class="external text" href="http://forums.mozillazine.org/viewtopic.php?t=399622&amp;highlight=hughes+net" rel="nofollow" title="http://forums.mozillazine.org/viewtopic.php?t=399622&amp;highlight=hughes+net">
          this forum thread
         </a>
         for more information.
        </dd>
       </dl>
       <p>
        <b>
         Inbox
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://www.inbox.com/faq_email.aspx?id=1389" rel="nofollow" title="http://www.inbox.com/faq_email.aspx?id=1389">
          Inbox's instructions for Outlook Express 6
         </a>
        </dd>
        <dd>
         <a class="external text" href="http://www.inbox.com/faq_email.aspx?id=1388" rel="nofollow" title="http://www.inbox.com/faq_email.aspx?id=1388">
          Inbox's instructions on how to enable POP/SMTP access to your account
         </a>
        </dd>
       </dl>
       <p>
        <b>
         Lavabit
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://lavabit.com/thunderbird.html" rel="nofollow" title="http://lavabit.com/thunderbird.html">
          Lavabit's instructions for Thunderbird
         </a>
        </dd>
       </dl>
       <p>
        <b>
         Lycos
        </b>
       </p>
       <dl>
        <dd>
         Lycos basic only supports webmail, using httpmail. See
         <a href="/Using_webmail_with_your_email_client" title="Using webmail with your email client">
          Using webmail with Thunderbird
         </a>
         .
        </dd>
        <dd>
         <a class="external text" href="http://help.lycos.com/faq.php?do=article&amp;articleid=186" rel="nofollow" title="http://help.lycos.com/faq.php?do=article&amp;articleid=186">
          Lycos's instructions for Lycos plus
         </a>
        </dd>
        <dd>
         It's unclear whether you have to use your ISPs SMTP server or can use smtp.mail.lycos.com on port 25 for Lycos plus.
        </dd>
       </dl>
       <p>
        <b>
         .Mac
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://docs.info.apple.com/article.html?artnum=306118#thunderbird" rel="nofollow" title="http://docs.info.apple.com/article.html?artnum=306118#thunderbird">
          .Mac's instructions for Thunderbird
         </a>
        </dd>
       </dl>
       <p>
        <b>
         MobileMe
        </b>
       </p>
       <dl>
        <dd>
         Use the setup data for Outlook Express from the
         <a class="external text" href="http://www.apple.com/mobileme/setup/" rel="nofollow" title="http://www.apple.com/mobileme/setup/">
          MobileMe Setup
         </a>
         page. The main difference is that in Thunderbird you create an account by pressing the Add Account button in Tools -&gt; Account Settings. Don't select "use secure authentication".
        </dd>
       </dl>
       <dl>
        <dd>
         <a class="external text" href="http://support.apple.com/kb/HT1625" rel="nofollow" title="http://support.apple.com/kb/HT1625">
          MobileMe: Configuring third-party email applications
         </a>
         has separate instructions for creating a Thunderbird account using me.com and mac.com addresses.
        </dd>
       </dl>
       <dl>
        <dd>
         In Thunderbird 3, a MobileMe account can be set up automatically.  Once it is set, simply go to the SMTP settings to adjust it.  For Connection Security to STARTTLS.  By doing so, the defaul port changes automatically to 25.  You will be able to send and receive emails with that minor change.
        </dd>
       </dl>
       <p>
        <br/>
        <b>
         MSN Premium
        </b>
       </p>
       <dl>
        <dd>
         Several ISPs offer MSN Premium accounts to their users instead of their own service (e.g. Qwest).  Most only provide instructions for setting up these accounts in Microsoft products, but they will work in Thunderbird as well.
        </dd>
       </dl>
       <dl>
        <dd>
         <a class="external text" href="http://support.microsoft.com/kb/930008" rel="nofollow" title="http://support.microsoft.com/kb/930008">
          Microsoft's instructions for Outlook.
         </a>
         The instructions in method 2 give the information that is needed to configure Thunderbird.
        </dd>
       </dl>
       <dl>
        <dd>
         <b>
          Note: For MSN Premium accounts to work in Thunderbird 1.5 and above you must change a pref in the
          <a class="external text" href="http://kb.mozillazine.org/Editing_configuration" rel="nofollow" title="http://kb.mozillazine.org/Editing_configuration">
           Config Editor.
          </a>
         </b>
         network.ntlm.send-lm-response must be set to true -
         <a class="external text" href="https://bugzilla.mozilla.org/show_bug.cgi?id=312344#c13" rel="nofollow" title="https://bugzilla.mozilla.org/show_bug.cgi?id=312344#c13">
          Bug 312344
         </a>
        </dd>
       </dl>
       <p>
        <b>
         MyRealBox
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://www.myrealbox.com/a?.BQ.DBE.Z5QQ.d" rel="nofollow" title="http://www.myrealbox.com/a?.BQ.DBE.Z5QQ.d">
          MyRealBox's instructions for Outlook Express
         </a>
        </dd>
        <dd>
         Since you're running beta versions of Novell Netmail if the Thunderbird account suddenly stops working you may have run into a bug that they will fix within a day or two.
        </dd>
       </dl>
       <p>
        <b>
         NTLWorld
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://www.ntlworld.com/helpsupport/support/solution.php?package=broadband&amp;curTitle=email&amp;lastLevel=&amp;curURL=/helpsupport/support/adding_an_account_to_oe.html" rel="nofollow" title="http://www.ntlworld.com/helpsupport/support/solution.php?package=broadband&amp;curTitle=email&amp;lastLevel=&amp;curURL=/helpsupport/support/adding_an_account_to_oe.html">
          NTLWorld's instructions for Outlook Express
         </a>
        </dd>
        <dd>
         You use a email address that looks like username@ntlworld.com , pop.ntlworld.com as the POP3 server, and smtp.ntlworld.com as the SMTP server.
        </dd>
       </dl>

        <b>
         Outlook Web Access (OWA)

       <dl>
        <dd>
         <a href="/Outlook_Web_Access" title="Outlook Web Access">
          Add-ons that support OWA
         </a>
        </dd>
        <dd>

        </dd>
       </dl>

       <dl>
        <dd>
         <a class="external text" href="http://home.peoplepc.com/helpinfo/faq/default.asp#pop3smtpoutlook2" rel="nofollow" title="http://home.peoplepc.com/helpinfo/faq/default.asp#pop3smtpoutlook2">
          PeoplePC's settings for Outlook Express
         </a>
        </dd>
       </dl>

        <b>
         Qwest
        </b>
       </p>
       <dl>
        <dd>
         See the instructions under MSN Premium.
        </dd>
       </dl>
       <p>
        <b>
         Road Runner
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://help.rr.com/HMSLogic/email_help_topic.aspx?topic=E-mail" rel="nofollow" title="http://help.rr.com/HMSLogic/email_help_topic.aspx?topic=E-mail">
          Road Runner's Email help page
         </a>
        </dd>
        <dd>
         Road Runner provides mail servers based on their customer's location.  If you try to log in to the wrong server it will reject your password as if you had entered it incorrectly.  Road Runner's email help page includes an "E-mail Quick Reference" box which shows users the correct servers based on that user's provider and location.
        </dd>
       </dl>
       <p>
        <b>
         Rogers
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://help.yahoo.com/rogers/mail/pop/" rel="nofollow" title="http://help.yahoo.com/rogers/mail/pop/">
          Rogers' instructions
         </a>
        </dd>
        <dd>
         Rogers has partnered with Yahoo, providing access to your rogers account via Yahoo's webmail but retaining their own pop and smtp servers. Common problems appear to be not entering your email address as the username, not checking the "Use name and password" option for the SMTP server or checking the secure connection or authentication checkbox.
        </dd>
       </dl>
       <p>
        <b>
         Sky.com
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://www.sky.com/portal/site/skycom/support/broadband/email?contentid=324110" rel="nofollow" title="http://www.sky.com/portal/site/skycom/support/broadband/email?contentid=324110">
          Sky's instructions for Outlook Express
         </a>
        </dd>
        <dd>
         Don't check the checkbox for secure connection or authentication.
        </dd>
       </dl>
       <p>
        <b>
         Sympatico.ca
        </b>
       </p>
       <dl>
        <dd>
         Thier web site requires you to enter a valid email address in order to view email help pages.
        </dd>
        <dd>
         You can figure out some of the information you need from this
         <a class="external text" href="http://forums.mozillazine.org/viewtopic.php?t=404135" rel="nofollow" title="http://forums.mozillazine.org/viewtopic.php?t=404135">
          forum thread
         </a>
         . Try "TLS if available" and port 25 if you want to make a secure connection to the SMTP server.
        </dd>
       </dl>
       <p>
        <b>
         Telus
        </b>
       </p>
       <dl>
        <dd>
         <a class="external text" href="http://www.mytelus.com/internet/email/help/index.vm" rel="nofollow" title="http://www.mytelus.com/internet/email/help/index.vm">
          Telus's instructions
         </a>
        </dd>
        <dd>
         Don't check the checkbox for secure connection or authentication. The troubleshooting section of the instructions mentions that you can get a "550 relaying mail is not allowed" error if you try to send a message using their SMTP server without being connected to the Internet using Telus. So it appears that you do not want to check "use name and password" in the SMTP settings. It also mentions a "The POP3 Server Responded: POP Lock Busy, Is Another Session Active" error , followed by a error about a bad password that can occur if there are two simultaneous attempts to access the mailbox. The workaround is to wait ten minutes.
        </dd>
       </dl>
