    <div id="wikitext">
     <p>
      <a class="wikilink" href="http://playground.arduino.cc/Code/LCD">
       Arduino LCD playground
      </a>
      |
      <span class="wikiword">
       KS0108
      </span>
      Graphics LCD library
     </p>
     <h1>
      <span class="wikiword">
       KS0108
      </span>
      Graphics LCD library
     </h1>
     <p>
      glcd-arduino (
      <span class="wikiword">
       GLCDv3
      </span>
      ) is an unofficial Arduino library that supports Graphic
      <span class="wikiword">
       LCDs
      </span>
      (GLCD) that use the
      <span class="wikiword">
       KS0108
      </span>
      (or equivalent) chip.
     </p>
     <p class="vspace">
     </p>
     <p>
      <span class="wikiword">
       GLCDv3
      </span>
      development has been idle since ca. 2012, the primary author has moved on to
      <a class="urllink" href="https://bitbucket.org/bperrybap/openglcd" rel="nofollow">
       openGLCD
      </a>
      .
      <span class="wikiword">
       OpenGLCD
      </span>
      provides greater functionality and wider hardware support, has a
      <span class="wikiword">
       GLCDv3
      </span>
      compatibility mode, and uses
      <span class="wikiword">
       GLCDv3
      </span>
      library wiring.
     </p>
     <p class="vspace">
     </p>
     <p>
      The current version of the glcd-arduino library (v3) can be found here:
      <a class="urllink" href="http://code.google.com/p/glcd-arduino/downloads/list" rel="nofollow">
       glcd library downloads
      </a>
      . The library includes example
sketches and a readme.txt file to explain the basics of adding a library as well as extensive documentation.
      <br/>
      For the latest and most complete documentation refer to the html documentation included in the library download.
(click on glcd/
      <span class="wikiword">
       doc/GLCDref
      </span>
      .htm to bring up HTML documentation)
     </p>
     <p class="vspace">
     </p>
     <p>
      <a class="urllink" href="http://code.google.com/p/glcd-arduino/downloads/detail?name=glcd-v3-20111205.zip" rel="nofollow">
       Version 3
      </a>
      (Officially released June 2012)
Source code and related downloads can be found on the Google code project:
      <a class="urllink" href="http://code.google.com/p/glcd-arduino/" rel="nofollow">
       glcd-arduino
      </a>
     </p>
     <p class="vspace">
     </p>
     <p>
      glcd-arduino has the following features:
     </p>
     <ul>
      <li>
       Support for Arduino 1.x (still works with pre 1.x
       <span class="wikiword">
        IDEs
       </span>
       )
      </li>
      <li>
       Support for Mega 2560, Sanguino and Teensy boards
      </li>
      <li>
       Easier pin assignment. Any pin can be assigned to any glcd function.
      </li>
      <li>
       Accurate/Consistent font rendering of any font size on any pixel location.
      </li>
      <li>
       User definable text areas
      </li>
      <li>
       Up/Down text scrolling
      </li>
      <li>
       Circle functions
      </li>
      <li>
       Additional example and demo sketches including a diagnostic sketch
      </li>
      <li>
       Fully search-able documentation in HTML format
      </li>
      <li>
       Pins can optionally be assigned by AVR port and bit instead of Arduino pin number.
      </li>
      <li>
       Supports fixed width fonts that provides up to 8 x 21 character display with the supplied system font.
      </li>
      <li>
       <span class="wikiword">
        CursorTo
       </span>
       method to position the cursor to a given font location (fixed width fonts only)
      </li>
      <li>
       <span class="wikiword">
        DrawBitmap
       </span>
       function to display bitmap images stored in Flash memory. A Processing utility is provided in the download to convert a monochrome bitmap file to a header file that stores the image in flash for use by the
       <span class="wikiword">
        DrwBitmap
       </span>
       method.
      </li>
     </ul>
     <p class="vspace">
     </p>
     <div>
      <img alt="" src="http://playground.arduino.cc/uploads/Code/ks0108-small.jpg" title=""/>
     </div>
     <p>
      Picture of a test sketch running on a max232 Freeduino board with a low cost GLCD panel
     </p>
     <p class="vspace">
     </p>
     <p>
      The library is based on the excellent ks0108 graphics routines written and copyright by Fabian Maximilian Thiele. The site link in his code does not respond but you can obtain a copy of his original work in the download section at the end of this article.
     </p>
     <p class="vspace">
     </p>
     <p>
      The code here has been converted to an Arduino library, has more flexibility in port addressing and improvements in I/O speed. The interface has been made more Arduino friendly and some convenience functions added. The method naming is mostly unchanged to facilitate porting of code written for the original version. Some methods now have default arguments to make them easer to use.
     </p>
     <p class="vspace">
     </p>
     <p>
      The test sketch included in the download demonstrates many of the capabilities of the library and if you start with this and use the default Arduino pin assignments, it is a good way to make sure that everything is working before you customize your configuration. Here is a simplified version of the example sketch in the download:
     </p>
     <p class="vspace">
     </p>
     <h5>
      Example GLCD sketch
     </h5>
     <pre>
      #include &lt;ks0108.h&gt;  // library header
  #include &lt;Arial14.h&gt;  // font definition for 14 point Arial font.
  #include "
      <span class="wikiword">
       SystemFont5x7
      </span>
      .h"   // system font
  #include "
      <span class="wikiword">
       ArduinoIcon
      </span>
      .h"     // bitmap
     </pre>
     <p class="vspace">
     </p>
     <pre>
      unsigned long startMillis;
  unsigned int iter = 0;
     </pre>
     <p class="vspace">
     </p>
     <p>
      void setup(){
     </p>
     <pre>
      GLCD.Init(NON_INVERTED);   // initialise the library
      <span class="wikiword">
       GLCD.ClearScreen
      </span>
      ();
      <span class="wikiword">
       GLCD.DrawBitmap
      </span>
      (
      <span class="wikiword">
       ArduinoIcon
      </span>
      , 32,0, BLACK); //draw the bitmap at the given x,y position
  delay(3000);
      <span class="wikiword">
       GLCD.ClearScreen
      </span>
      ();
      <span class="wikiword">
       GLCD.SelectFont
      </span>
      (System5x7);       // select fixed width system font
     </pre>
     <p>
      }
     </p>
     <p class="vspace">
     </p>
     <p>
      void  loop(){   // run over and over again
     </p>
     <pre>
      startMillis = millis();
  while( millis() - startMillis &lt; 1000){ // loop for one second
      <span class="wikiword">
       GLCD.DrawRect
      </span>
      (0, 0, 64, 61, BLACK); // rectangle in left side of screen
      <span class="wikiword">
       GLCD.DrawRoundRect
      </span>
      (68, 0, 58, 61, 5, BLACK);  // rounded rectangle around text area   
    for(int i=0; i &lt; 62; i += 4)
      <span class="wikiword">
       GLCD.DrawLine
      </span>
      (1,1,63,i, BLACK);  // draw lines from upper left down right side of rectangle
      <span class="wikiword">
       GLCD.DrawCircle
      </span>
      (32,31,30,BLACK);   // draw circle centered in the left side of screen
      <span class="wikiword">
       GLCD.FillRect
      </span>
      (92,40,16,16, WHITE); // clear previous spinner position
      <span class="wikiword">
       GLCD.CursorTo
      </span>
      (5,5);               // locate curser for printing text
      <span class="wikiword">

     <h3>
      Wiring and Configuration:
     </h3>
     <p>
      In v3 there are 3 types of configuration:
     </p>
     <ul>
      <li>
       Library
      </li>
      <li>
       Panel
      </li>
      <li>
       Pin
      </li>
     </ul>
     <p class="vspace">
     </p>
     <p>
      Each type of configuration is handled by separate configuration files.
     </p>
     <p class="vspace">
     </p>
     <p>
      The library configuration is handled by glcd/glcd_Config.h
The library configuration determines things like panel type, as well as a few other options. It is the master configuration file.
     </p>
     <p class="vspace">
     </p>
     <p>
      The panel configuration is used to configure things that are specific to a particular glcd panel like the geometry of the display, the chip select lines, and the low level timing for the panel. The glcd_Config.h library configuration file specifies which panel configuration file to use.
     </p>
     <p class="vspace">
     </p>
     <p>
      The pin configuration is used to assign pins.
A given panel configuration will automatically determine which pin configuration file to use based on which board type is being used in the IDE.
     </p>
     <p class="vspace">
     </p>
     <p>
      Pin assignments are contained in the appropriate pin configuration header file in glcd/config for the supported controller chips. These controller types are supported in the current version:
     </p>
     <ul>
      <li>
       ks0108_arduino.h  &lt;- this is for
       <span class="wikiword">
        ATmega168
       </span>
       and
       <span class="wikiword">
        ATmega328
       </span>
       boards
      </li>
      <li>
       ks0108_mega.h     &lt;- for the Arduino Mega (
       <span class="wikiword">
        ATmega1280
       </span>
       /2560)
      </li>
      <li>
       ks0108_Sanguino.h &lt;- for Sanguino (
       <span class="wikiword">
        ATmega644
       </span>
       )
      </li>
      <li>
       ks0108_Teensy.h   &lt;- for Teensy/Teensy++
      </li>
     </ul>
     <p class="vspace">
     </p>
     <p>
      See the readme file, glcd/glcd_Config.h and included HTML documentation supplied with the download for more details on configuration.
     </p>
     <p class="vspace">
     </p>
     <p>
      It is suggested that you wire up the panel using the default pin assignments. This diagram shows how panels should be connected using the default pin assignments in the distributed library.
     </p>
     <p class="vspace">
     </p>
     <h3>
      IMPORTANT NOTE:
     </h3>
     <p>
      The connections to the glcd module must be soldered.
While tempting to push wires or a 20 pin header into
the holes without soldering, that will not create proper electrical connections.
     </p>

     <h5>
      Wiring up the external components:
     </h5>
     <p>
      Most GLCD panels require an external preset pot to set the LCD working voltage (contrast) and a fixed resistor to limit the current in the backlight. The datasheet for your panel should provide specific information on the wiring and choice of components. A suggestion for wiring these up is to use a small piece of strip-board with header pins for 5V, ground and Reset providing connection to the Arduino. See the diagram above for layout.
      <br/>
      <strong>
       Contrast Pot NOTE:
      </strong>
      <br/>
      Play close attention for how to wire up the contrast pot. ks0108 modules do not wire up their contrast pot the same way as a typical hd44780 lcd. A hd44780 typically hooks its contrast pot legs to +5v and GND.
On a ks0108, the pot, which is typically between 10-20k, is used to create a varying negative voltage from Vee up to GND that is used to feed to the Vo input signal.
In order to do this, one leg of the pot needs to hook to ground, one leg needs to hook to the Vee negative voltage output pin and then the wiper (middle pin of the pot) will have the variable voltage output that can be fed to the Vo contrast control input pin.
     </p>
     <p class="vspace">
     </p>
     <h5>
      Changing the Arduino pin assignments:
     </h5>
     <p>
      The
      <span class="wikiword">
       KS0108
      </span>
      chip needs lots of pins. 8 data pins and 5 command pins are required in addition to the power connections. 
Ideally the command pins should all be on one port and all the data pins together on another. In practice this is not easy to do on a standard arduino.
     </p>
     <p class="vspace">
     </p>
     <p>
      Starting with glcd v3, pin assignment is much more flexible as any glcd function or data pin can be assigned to any Arduino pin. If you split data pins across ports the code will run slightly slower, but for all but the most speed critical graphic applications its not significant.
To change pin assignments you must modify the appropriate pin configuration file in the glcd/config directory (ks0108.h when using a m328 based arduino). Find the section in the file that begins:
     </p>
     <pre>
      /********************************************************/
 /* Configuration for assigning LCD bits to Arduino Pins */
 /********************************************************/
     </pre>
     <p>
      You will see the defines for the data pins and the five control pins with their default pin assignments:
     </p>
     <pre>
      <strong>
       Name    Arduino pin number
      </strong>
      <span style="font-size:83%">
       #define glcdData0Pin        8
      </span>
      <span style="font-size:83%">
       #define glcdData1Pin        9
      </span>
      <span style="font-size:83%">
       #define glcdData2Pin        10
      </span>
      <span style="font-size:83%">
       #define glcdData3Pin        11
      </span>
      <span style="font-size:83%">
       #define glcdData4Pin        4
      </span>
      <span style="font-size:83%">
       #define glcdData5Pin        5
      </span>
      <span style="font-size:83%">
       #define glcdData6Pin        6
      </span>
      <span style="font-size:83%">
       #define glcdData7Pin        7
      </span>
      <span style="font-size:83%">
       #define glcdCSEL1           14 (Analog pin 0)
      </span>
      <span style="font-size:83%">
       #define glcdCSEL2           15 (Analog pin 1)
      </span>
      <span style="font-size:83%">
       #define glcdRW              16 (Analog pin 2)
      </span>
      <span style="font-size:83%">
       #define glcdDI              17 (Analog pin 3)
      </span>
      <span style="font-size:83%">
       #define glcdEN              18 (Analog pin 4)
      </span>
     </pre>
     <p class="vspace">
     </p>
     <p>
      Any of these data pins and glcd functions can be assigned to any pin (as long as its not used by something else)
Here are the pin/port mappings for the Arduino m328 processor:
     </p>
     <ul>
      <li>
       Port B  8-13
      </li>
      <li>
       Port C  14-19 (the analog pins)
      </li>
      <li>
       Port D  0-7 (note 0 and 1 are used by hardware serial)
      </li>
     </ul>
     <p class="vspace">
     </p>
     <p>
      While any Arduino pin can can be used for any glcd data pin,
using Arduino pins that are all in the same port and that are in consecutive bit order will give slightly higher performance. The default pin assignments for the glcd data pins are assigned to take advantage of this optimization.
     </p>
     <p class="vspace">
     </p>
     <p>
      An example of remapping a pin might be to change glcdEN to use some other pin rather than 18 to allow using i2c on a m328 based board.
     </p>
     <p class="vspace">
     </p>
     <h5>
      Troubleshooting
     </h5>
     <p>
      No pixels visible on the display
     </p>
     <ul>
      <li>
       Check +5v and Gnd connections between Arduino and GLCD panel
      </li>
      <li>
       Check that all data and command pins are wired correctly
      </li>
      <li>
       Check contrast voltage (typically between -3 and -4 volts) on contrast-in pin of LCD panel. While the sketch is operating, try gradually adjusting the pot through its range. Some displays are very sensitive to this setting.
      </li>
      <li>
       Check sketch has compiled ok and downloaded to the arduino.
      </li>
     </ul>
     <p>
      Left and right side of image reversed
     </p>
     <ul>
      <li>
       swap
       <span class="wikiword">
        CSEL1
       </span>
       and
       <span class="wikiword">
        CSEL2
       </span>
       wires (or swap pin assignments in header file)
      </li>
     </ul>
     <p>
      Display garbled
     </p>
     <ul>
      <li>
       check all data and command pins are wired correctly and that these match the setting in the header file.
      </li>
     </ul>
     <p>
      Sketch upload not working
     </p>
     <ul>
      <li>
       Try the glcd with nothing connected to its reset line
      </li>
      <li>
       Try connecting the glcd reset line to vcc
      </li>
      <li>
       enable the library software reset functionality
      </li>
     </ul>
     <p>
      <br/>
      <strong>
       GLCD reset pin NOTE:
      </strong>
      <br/>
      The Arduino autoreset circuit is quit fragile. Depending on the particular Arduino board and glcd module, connecting the glcd module to the Arduino board reset line may interfere with the Arduino board's autoreset circuit. Most glcds will reset on power up so the glcd reset line can be left unconnected. Most of the remaining others will work when their reset line is connected to vcc. A small fraction of the glcds out there will need a reset pulse. For those that need a reset pulse and connecting the glcd to the arduino reset line causes the autoreset on the aruduino board to fail, the library must be used to reset the glcd which requires another Arduino pin. In order to enable this functionality edit the pin configuration file and uncomment the line that looks like this:
     </p>
     <p class="vspace">
     </p>
     <p>
      // Reset Bit  - uncomment the next line if reset is connected to an output pin
//#define glcdRES          19    // Reset Bit
     </p>
     <p class="vspace">
     </p>
     <p>
      And then connect the Arduino pin specified by glcdRES to the glcd module's reset pin.
     </p>
