             <h3>
              Performing the Crack
             </h3>
             <p>
              The Wi-Fi Alliance was wise to implement an eight character minimum for WPA-PSK. Making the key that long essentially renders  brute force methods useless. This is because the number of possible typeable character combinations for keys of an eight character length is just above
              <i>
               six quadrillion
              </i>
              (that's 94
              <sup>
               8
              </sup>
              or about 6 x 10
              <sup>
               15
              </sup>
              ).
             </p>
             <p>
              My poor little laptop can only crunch about 35 hashes a second, so it would take me about
              <i>
               five-and-a-half million years
              </i>
              (I'm not kidding here either, I did the math!) to create a hash table for an eight character hash table or to test all possible combinations when brute-forcing a key.
             </p>
             <p>
              And what's more, since the hash is salted with the SSID of the AP, that hash table I just spent five million years creating, would  be good only against APs with that exact SSID. So, clearly we're not going to be brute-forcing any WPA keys anytime soon.
             </p>
             <p>
              What we can do, however, is limit the list of possible passphrases by making educated guesses, compute the hashes of those guesses and check them against our captured key. This technique is referred to as a
              <b>
               dictionary attack
              </b>
              .
             </p>
             <p>
              BackTrack v2 comes bundled with a good offering of simple wordlists, as well as four lists of  passwords common in the '90s, reverse-sorted by occurrence (more common passwords are at the top, less common passwords are at the bottom). The lists seem to be missing from Backtrack v3, but there are
              <a href="http://www.openwall.com/passwords/wordlists/" target="_blank">
               plenty
              </a>
              of wordlists around the 'net.
             </p>
             <p>
              Using the wordlists in Backtrack version 2, we can mount a dictionary attack on our captured WPA handshake using either
              <b>
               aircrack-ng
              </b>
              or
              <b>
               <a href="http://wirelessdefence.org/Contents/coWPAttyMain.htm" target="_blank">
                coWPAtty
               </a>
              </b>
              . Aircrack-ng runs much faster on my attacking system (testing 3740 keys took 35 seconds), and has native optimization for multiple processors. coWPAtty, on the other hand, runs much slower (testing the same 3740 keys took almost 2 minutes) and can accept hash files precomputed by
              <b>
               genpmk
              </b>
              .
             </p>
             <div style="background-color:#FFFFCC; border:solid 1px black; padding:5px 30px 5px 5px; margin-left:18px;">
              <img alt="NOTE!" src="/images/stories/wireless/wpa_cracking/attentionsml.gif"/>
              <b>
               Note:
              </b>
              <br/>
              Some of the commands below have been formatted into multiple lines to fit our page. All commands should be entered on
              <b>
               one line
              </b>
              .
             </div>
             <div style="float:right;padding-left:5px;padding-top:5px;padding-bottom:5px;">
              <div id="OA_posid_4">
              </div>
              <noscript>
               <a href="http://vma.tgdaily.net/www/delivery/ck.php?n=a13b031" target="_blank">
                <img alt="" border="0" src="http://vma.tgdaily.net/www/delivery/avw.php?zoneid=281&amp;n=a13b031"/>
               </a>
              </noscript>
              <script type="text/javascript">
               OA_show_banner(4);
              </script>
             </div>
             <p>
              <b>
               aircrack-ng attack
              </b>
             </p>
             <p>
              Start a dictionary attack against a WPA key with the following:
             </p>
             <p>
              General Form:
             </p>
             <pre>
              aircrack-ng -e
              <b>
               <i>
                AP_SID
               </i>
              </b>
              -w
              <b>
               <i>
                dictionary_file
               </i>
              </b>
              <b>
               <i>
                capture_file
               </i>
              </b>
             </pre>
             <p>
              Example (BackTrack v3):
             </p>
             <pre>
              aircrack-ng -e snb -w /pentest/wireless
              <br/>
              /cowpatty-4.0/dict Kismet-Jan-15-2008-1.dump
             </pre>
             <p>
              Aircrack-ng shows the hex hashes of the keys as it tries them, which is nice since some attacks can take a long time. Figure 8 shows that Aircrack-ng took
              <b>
               35 seconds
              </b>
              to find the test key "dictionary".
             </p>
             <p align="center">
              <a href="/images/stories/wireless/wpa_cracking/aircrack_key_found_big.png" rel="thumbnail" style="cursor: url('http://www.smallnetbuilder.com//plugins/content/multithumb/magnify.cur'), auto;" target="_blank" title="Aircrack-ng, Key Found!">
               <img alt="Aircrack-ng, Key Found!" class="multithumb" height="314" src="http://www.smallnetbuilder.com/cache/multithumb_thumbs/b_550_0_16777215_00__images_stories_wireless_wpa_cracking_aircrack_key_found_big.png" title="Aircrack-ng, Key Found!" width="550"/>
              </a>
             </p>
             <h6 align="center">
              Figure 8: Aircrack-ng, Key Found!
             </h6>
             <p>
              <b>
               coWPAtty
              </b>
             </p>
             <p>
              First move into the cowpatty directory, either by selecting it from the menu or by changing to
              <i>
               /pentest/wireless/cowpatty-4.0
              </i>
              . Then run:
             </p>
             <p>
              General Form:
             </p>
             <pre>
              ./cowpatty -s
              <b>
               <i>
                AP_SID
               </i>
              </b>
              -f
              <b>
               <i>
                dictionary_file
               </i>
              </b>
              -r
              <b>
               <i>
                capture_file
               </i>
              </b>
             </pre>
             <p>
              Example:
             </p>
             <pre>
              ./cowpatty -s snb -f dict -r Kismet-Jan-15-2008-1.dump
             </pre>
             <p>
              coWPAtty doesn't say much about its run-time status, but  prints updates every thousand keys. Figure 9 shows that coWPAtty took a little over
              <b>
               two minutes
              </b>
              to recover the   test key  "dictionary".
             </p>
             <p align="center">
              <a href="/images/stories/wireless/wpa_cracking/cowpatty_key_found_big.png" rel="thumbnail" style="cursor: url('http://www.smallnetbuilder.com//plugins/content/multithumb/magnify.cur'), auto;" target="_blank" title="coWPAtty, Key Found!">
               <img alt="coWPAtty, Key Found!" class="multithumb" height="168" src="http://www.smallnetbuilder.com/cache/multithumb_thumbs/b_550_0_16777215_00__images_stories_wireless_wpa_cracking_cowpatty_key_found_big.png" title="coWPAtty, Key Found!" width="550"/>
              </a>
             </p>
             <h6 align="center">
              Figure 9: coWPAtty, Key Found!
             </h6>
             <p>
              Alternately, coWPAtty can use a precomputed hash file to attack a WPA key. Precomputed hash files use a technique similar to
              <a href="http://en.wikipedia.org/wiki/Rainbow_table" target="_blank">
               Rainbow Tables
              </a>
              allowing you to trade the amount of time required to crack a given key for hash file size (and precomputation time).
             </p>
             <p>
              Hashes are paired with their plain text precursor allowing the engine to simply look up the captured WPA key hash and read off its corresponding plain text key. Since WPA keys are salted, this technique only works against AP's with the same SSID used to compute the table.
             </p>
             <p>
              Hash tables can be very effective but require  disk space to store the tables that can get rather large, quickly. Even with these limitations, the
              <a href="http://www.renderlab.net/projects/WPA-tables/" target="_blank">
               Church of WiFi
              </a>
              has computed hash tables for the 1000 most common SSID's against one million common passphrases.
             </p>
             <p>
              You can generate a hash table from within the cowpatty directory with coWPAtty's
              <b>
               genpmk
              </b>
              :
             </p>
             <p>
              General Form:
             </p>
             <pre>
              ./genpmk -s
              <b>
               <i>
                AP_SID
               </i>
              </b>
              -f
              <b>
               <i>
                dictionary_file
               </i>
              </b>
              -d
              <b>
               <i>
                hash_output_file
               </i>
              </b>
             </pre>
             <p>
              Example:
             </p>
             <pre>
              ./genpmk -s snb -f dict -d dict_hash
             </pre>
             <p align="center">
              <a href="/images/stories/wireless/wpa_cracking/genpmk_hash_generation_big.png" rel="thumbnail" style="cursor: url('http://www.smallnetbuilder.com//plugins/content/multithumb/magnify.cur'), auto;" target="_blank" title="genpmk Hash Table Generation">
               <img alt="genpmk Hash Table Generation" class="multithumb" height="132" src="http://www.smallnetbuilder.com/cache/multithumb_thumbs/b_550_0_16777215_00__images_stories_wireless_wpa_cracking_genpmk_hash_generation_big.png" title="genpmk Hash Table Generation" width="550"/>
              </a>
             </p>
             <h6 align="center">
              Figure 10: genpmk Hash Table Generation
             </h6>
             <p>
              Now, using the newly created hash table, the crack takes only a fraction of a second (
              <b>
               0.11
              </b>
              to be precise). This is just shy of
              <b>
               1/1100th
              </b>
              the time it took when not using a hash table.
             </p>
             <p>
              General Form:
             </p>
             <pre>
              ./cowpatty -s
              <b>
               <i>
                AP_SID
               </i>
              </b>
              -d
              <b>
               <i>
                hash_output_file
               </i>
              </b>
              -r
              <b>
               <i>
                capture_file
               </i>
              </b>
             </pre>
             <p>
              Example:
             </p>
             <pre>
              ./cowpatty -s snb -d dict_hash -r Kismet-Jan-15-2008-1.dump
             </pre>
             <p align="center">
              <a href="/images/stories/wireless/wpa_cracking/cowpatty_hash_attack_big.png" rel="thumbnail" style="cursor: url('http://www.smallnetbuilder.com//plugins/content/multithumb/magnify.cur'), auto;" target="_blank" title="coWPAtty Hash Table Attack">
               <img alt="coWPAtty Hash Table Attack" class="multithumb" height="124" src="http://www.smallnetbuilder.com/cache/multithumb_thumbs/b_550_0_16777215_00__images_stories_wireless_wpa_cracking_cowpatty_hash_attack_big.png" title="coWPAtty Hash Table Attack" width="550"/>
              </a>
             </p>

              Figure 11: coWPAtty Hash Table Attack
