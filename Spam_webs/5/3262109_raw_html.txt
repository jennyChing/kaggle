     <p>
      <b>
       MPEG-4 Part 14
      </b>
      or
      <b>
       MP4
      </b>
      is a digital
      <a href="/wiki/Multimedia" title="Multimedia">
       multimedia
      </a>
      format most commonly used to store
      <a href="/wiki/Digital_video" title="Digital video">
       video
      </a>
      and
      <a href="/wiki/Digital_audio" title="Digital audio">
       audio
      </a>
      , but can also be used to store other data such as
      <a class="mw-redirect" href="/wiki/Subtitles" title="Subtitles">
       subtitles
      </a>
      and still images.
      <sup class="reference" id="cite_ref-2">
       <a href="#cite_note-2">
        <span>
         [
        </span>
        2
        <span>
         ]
        </span>
       </a>
      </sup>
      Like most modern
      <a class="mw-redirect" href="/wiki/Container_format_(digital)" title="Container format (digital)">
       container formats
      </a>
      , it allows
      <a href="/wiki/Streaming_media" title="Streaming media">
       streaming
      </a>
      over the
      <a href="/wiki/Internet" title="Internet">
       Internet
      </a>
      . The only official
      <a href="/wiki/Filename_extension" title="Filename extension">
       filename extension
      </a>
      for MPEG-4 Part 14 files is
      <b>
       .mp4
      </b>
      , but many have other extensions, most commonly
      <b>
       .m4a
      </b>
      and
      <b>
       .m4p
      </b>
      . M4A (audio only) is often compressed using
      <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">
       AAC
      </a>
      encoding (
      <a class="mw-redirect" href="/wiki/Lossy" title="Lossy">
       lossy
      </a>
      ), but can also be in
      <a href="/wiki/Apple_Lossless" title="Apple Lossless">
       Apple Lossless
      </a>
      format. M4P is a protected format which employs
      <a href="/wiki/Digital_rights_management" title="Digital rights management">
       DRM
      </a>
      technology to restrict copying. MPEG-4 Part 14 (formally
      <b>
       <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">
        ISO
       </a>
       /
       <a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">
        IEC
       </a>
       14496-14:2003
      </b>
      ) is a standard specified as a part of
      <a href="/wiki/MPEG-4" title="MPEG-4">
       MPEG-4
      </a>
      .
     </p>
     <p>
      Some devices advertised as "
      <a class="mw-redirect" href="/wiki/MP4_Player" title="MP4 Player">
       MP4 Players
      </a>
      " are simply
      <a class="mw-redirect" href="/wiki/MP3_Player" title="MP3 Player">
       MP3 Players
      </a>
      that also play
      <a href="/wiki/AMV_video_format" title="AMV video format">
       AMV video
      </a>
      or some other video format, and do not necessarily play the MPEG-4 Part 14 format.
     </p>

     <h2>
      <span class="mw-headline" id="History_of_MP4">
       History of MP4
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=MPEG-4_Part_14&amp;action=edit&amp;section=1" title="Edit section: History of MP4">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      MPEG-4 Part 14 is an instance of the more general
      <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">
       ISO
      </a>
      /
      <a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">
       IEC
      </a>
      14496-12:2004 (MPEG-4 Part 12:
      <a href="/wiki/ISO_base_media_file_format" title="ISO base media file format">
       ISO base media file format
      </a>
      ) which is directly based upon the
      <a href="/wiki/QuickTime_File_Format" title="QuickTime File Format">
       QuickTime File Format
      </a>
      .
      <sup class="reference" id="cite_ref-mp4ra-org-references_3-0">
       <a href="#cite_note-mp4ra-org-references-3">
        <span>
         [
        </span>
        3
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-mpeg4part12_4-0">
       <a href="#cite_note-mpeg4part12-4">
        <span>
         [
        </span>
        4
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-mpeg4part12--old_5-0">
       <a href="#cite_note-mpeg4part12--old-5">
        <span>
         [
        </span>
        5
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-mpeg4part12--new_6-0">
       <a href="#cite_note-mpeg4part12--new-6">
        <span>
         [
        </span>
        6
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-7">
       <a href="#cite_note-7">
        <span>
         [
        </span>
        7
        <span>
         ]
        </span>
       </a>
      </sup>
      MPEG-4 Part 14 is essentially identical to the QuickTime file format, but formally specifies support for Initial Object Descriptors (IOD) and other MPEG features.
      <sup class="reference" id="cite_ref-8">
       <a href="#cite_note-8">
        <span>
         [
        </span>
        8
        <span>
         ]
        </span>
       </a>
      </sup>
      MPEG-4 Part 14 revises and completely replaces Clause 13 of ISO/IEC 14496-1 (MPEG-4 Part 1: Systems), in which the file format for MPEG-4 content was previously specified.
      <sup class="reference" id="cite_ref-ISO-14496-14_9-0">
       <a href="#cite_note-ISO-14496-14-9">
        <span>
         [
        </span>
        9
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      The MPEG-4 file format specification was based on the QuickTime format specification published in 2001.
      <sup class="reference" id="cite_ref-quicktime-format-2001_10-0">
       <a href="#cite_note-quicktime-format-2001-10">
        <span>
         [
        </span>
        10
        <span>
         ]
        </span>
       </a>
      </sup>
      The MPEG-4 file format, version 1 was published in 2001 as ISO/IEC 14496-1:2001, which is a revision of the MPEG-4 Part 1: Systems specification published in 1999 (ISO/IEC 14496-1:1999).
      <sup class="reference" id="cite_ref-digitalpreservation-ISO-14496-1-2001_11-0">
       <a href="#cite_note-digitalpreservation-ISO-14496-1-2001-11">
        <span>
         [
        </span>
        11
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-rfc4337_12-0">
       <a href="#cite_note-rfc4337-12">
        <span>
         [
        </span>
        12
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-ISO-14496-1_13-0">
       <a href="#cite_note-ISO-14496-1-13">
        <span>
         [
        </span>
        13
        <span>
         ]
        </span>
       </a>
      </sup>
      In 2003, the first version of the MP4 file format was revised and replaced by MPEG-4 Part 14: MP4 file format (ISO/IEC 14496-14:2003), commonly named as MPEG-4 file format version 2.
      <sup class="reference" id="cite_ref-digitalpreservation-ISO-14496-14-2003_14-0">
       <a href="#cite_note-digitalpreservation-ISO-14496-14-2003-14">
        <span>
         [
        </span>
        14
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-15">
       <a href="#cite_note-15">
        <span>
         [
        </span>
        15
        <span>
         ]
        </span>
       </a>
      </sup>
      The MP4 file format was generalized into the ISO Base Media File format ISO/IEC 14496-12:2004, which defines a general structure for time-based media files. It in turn is used as the basis for other file formats in the family (for example MP4,
      <a class="mw-redirect" href="/wiki/3GP" title="3GP">
       3GP
      </a>
      ,
      <a class="mw-redirect" href="/wiki/MJ2" title="MJ2">
       Motion JPEG 2000
      </a>
      ).
      <sup class="reference" id="cite_ref-mp4ra-org-references_3-1">
       <a href="#cite_note-mp4ra-org-references-3">
        <span>
         [
        </span>
        3
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-mpeg4part12-mp4_16-0">
       <a href="#cite_note-mpeg4part12-mp4-16">
        <span>
         [
        </span>
        16
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-ISO-14496-12_17-0">
       <a href="#cite_note-ISO-14496-12-17">
        <span>
         [
        </span>
        17
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>

        ISO/IEC 14496-1:2001
       </td>
       <td>
        MPEG-4 Part 1 (Systems), First edition
       </td>
      </tr>
      <tr>
       <td>
        MP4 file format version 2
       </td>
       <td>
        2003
       </td>
       <td>
        ISO/IEC 14496-14:2003
       </td>
       <td>
        MPEG-4 Part 14 (MP4 file format), Second edition
       </td>
      </tr>
     </table>
     <p>
      The MP4 file format defined some extensions over the ISO Base Media File Format to support MPEG-4 visual/audio codecs and various MPEG-4 Systems features such as object descriptors and scene descriptions. Some of these extensions are also used by other formats based on ISO base media file format (e.g. 3GP).
      <sup class="reference" id="cite_ref-3gpp2-vs-3gpp_1-1">
       <a href="#cite_note-3gpp2-vs-3gpp-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
      A list of all registered extensions for ISO Base Media File Format is published on the official registration authority website
      <a class="external text" href="http://www.mp4ra.org" rel="nofollow">
       www.mp4ra.org
      </a>
      . The registration authority for code-points (identifier values) in "MP4 Family" files is Apple Inc. and it is named in Annex D (informative) in MPEG-4 Part 12.
      <sup class="reference" id="cite_ref-mpeg4part12-mp4_16-1">
       <a href="#cite_note-mpeg4part12-mp4-16">
        <span>
         [
        </span>
        16
        <span>
         ]
        </span>
       </a>
      </sup>
      Codec designers should register the codes they invent, but the registration is not mandatory
      <sup class="reference" id="cite_ref-ftyp-mp4ra_18-0">
       <a href="#cite_note-ftyp-mp4ra-18">
        <span>
         [
        </span>
        18
        <span>
         ]
        </span>
       </a>
      </sup>
      and some invented and used code-points are not registered.
      <sup class="reference" id="cite_ref-ftyp_19-0">
       <a href="#cite_note-ftyp-19">
        <span>
         [
        </span>
        19
        <span>
         ]
        </span>
       </a>
      </sup>
      When someone is creating a new specification derived from the ISO Base Media File Format, all the existing specifications should be used both as examples and a source of definitions and technology. If an existing specification already covers how a particular media type is stored in the file format (e.g. MPEG-4 audio or video in MP4), that definition should be used and a new one should not be invented.
      <sup class="reference" id="cite_ref-mpeg4part12-mp4_16-2">
       <a href="#cite_note-mpeg4part12-mp4-16">
        <span>
         [
        </span>
        16
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <h2>
      <span class="mw-headline" id="Filename_extensions">
       Filename extensions
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=MPEG-4_Part_14&amp;action=edit&amp;section=2" title="Edit section: Filename extensions">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      While the only official
      <a href="/wiki/Filename_extension" title="Filename extension">
       filename extension
      </a>
      defined by the standard is
      <tt>
       .mp4
      </tt>
      , various filename extensions are commonly used to indicate intended content:
     </p>
     <ul>
      <li>
       MPEG-4 files with audio and video generally use the standard
       <tt>
        .mp4
       </tt>
       extension.
      </li>
      <li>
       Audio-only MPEG-4 files generally have a
       <tt>
        .m4a
       </tt>
       extension. This is especially true of non-protected content.
       <ul>
        <li>
         MPEG-4 files with audio streams encrypted by
         <a href="/wiki/FairPlay" title="FairPlay">
          FairPlay
         </a>
         <a class="mw-redirect" href="/wiki/Digital_Rights_Management" title="Digital Rights Management">
          Digital Rights Management
         </a>
         as were sold through the
         <a href="/wiki/ITunes_Store" title="ITunes Store">
          iTunes Store
         </a>
         use the
         <tt>
          .m4p
         </tt>
         extension.
         <a class="mw-redirect" href="/wiki/ITunes_Plus" title="ITunes Plus">
          iTunes Plus
         </a>
         tracks, that the iTunes Store currently sells, are unencrypted and use
         <tt>
          .m4a
         </tt>
         accordingly.
        </li>
        <li>
         <a href="/wiki/Audiobook" title="Audiobook">
          Audiobook
         </a>
         and
         <a href="/wiki/Podcast" title="Podcast">
          podcast
         </a>
         files, which also contain metadata including chapter markers, images, and hyperlinks, can use the extension
         <tt>
          .m4a
         </tt>
         , but more commonly use the
         <tt>
          .m4b
         </tt>
         extension. An
         <tt>
          .m4a
         </tt>
         audio file cannot "bookmark" (remember the last listening spot), whereas
         <tt>
          .m4b
         </tt>
         extension files can.
         <sup class="reference" id="cite_ref-20">
          <a href="#cite_note-20">
           <span>
            [
           </span>
           20
           <span>
            ]
           </span>
          </a>
         </sup>
        </li>
        <li>
         The Apple
         <a href="/wiki/IPhone" title="IPhone">
          iPhone
         </a>
         uses MPEG-4 audio for its ringtones but uses the
         <tt>
          .m4r
         </tt>
         extension rather than the
         <tt>
          .m4a
         </tt>
         extension.
        </li>
       </ul>
      </li>
      <li>
       Raw
       <a class="mw-redirect" href="/wiki/MPEG-4_Visual" title="MPEG-4 Visual">
        MPEG-4 Visual
       </a>
       bitstreams are named
       <tt>
        .m4v
       </tt>
       but this extension is also sometimes used for video in MP4 container format.
       <sup class="reference" id="cite_ref-21">
        <a href="#cite_note-21">
         <span>
          [
         </span>
         21
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       <a href="/wiki/Mobile_phone" title="Mobile phone">
        Mobile phones
       </a>
       use
       <a class="mw-redirect" href="/wiki/3GP" title="3GP">
        3GP
       </a>
       , an implementation of MPEG-4 Part 12 (a.k.a. MPEG-4/JPEG2000 ISO Base Media file format), similar to MP4. It uses
       <tt>
        .3gp
       </tt>
       and
       <tt>
        .3g2
       </tt>
       extensions. These files also store non-MPEG-4 data (
       <a href="/wiki/H.263" title="H.263">
        H.263
       </a>
       , AMR, TX3G).
      </li>
     </ul>
     <h3>
      <span class="mw-headline" id=".MP4_versus_.M4A">
       .MP4 versus .M4A
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=MPEG-4_Part_14&amp;action=edit&amp;section=3" title="Edit section: .MP4 versus .M4A">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      M4A stands for MPEG 4 Audio and is a filename extension used to represent audio files.
     </p>
     <p>
      The existence of two different filename extensions, .MP4 and .M4A, for naming audio-only MP4 files has been a source of confusion among users and multimedia playback software. Some
      <a href="/wiki/File_manager" title="File manager">
       file managers
      </a>
      , such as
      <a class="mw-redirect" href="/wiki/Windows_Explorer" title="Windows Explorer">
       Windows Explorer
      </a>
      , look up the media type and associated applications of a file based on its filename extension. But since MPEG-4 Part 14 is a container format, MPEG-4 files may contain any number of audio, video, and even subtitle streams, making it impossible to determine the type of streams in an MPEG-4 file based on its filename extension alone. In response, Apple Inc. started using and popularizing the
      <tt>
       .m4a
      </tt>
      filename extension, which is used for MP4 containers with audio data in the
      <a href="/wiki/Lossy_compression" title="Lossy compression">
       lossy
      </a>
      <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">
       Advanced Audio Coding
      </a>
      (AAC) or its own
      <a href="/wiki/Apple_Lossless" title="Apple Lossless">
       Apple Lossless
      </a>
      (ALAC) formats. Software capable of audio/video playback should recognize files with either
      <tt>
       .m4a
      </tt>
      or
      <tt>
       .mp4
      </tt>
      filename extensions, as would be expected, since there are no file format differences between the two. Most software capable of creating MPEG-4 audio will allow the user to choose the filename extension of the created MPEG-4 files.
     </p>
