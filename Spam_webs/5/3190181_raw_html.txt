        <h1>
         Weblocks
        </h1>
        <p class="first">
         This page contains basic and probably outdated information about
         <em>
          Weblocks
         </em>
         . For more details please visit the
         <a href="http://weblocks-framework.info/">
          new
	  web page
         </a>
         .
        </p>
        <p>
         You can also check out our
         <a href="http://weblocks.viridian-project.de/weblocks-demo">
          demo application
         </a>
         to get a first impression.
        </p>
        <a href="weblocks-shot.png">
         <img alt="Screenshot of Weblocks." src="weblocks-shot-small.png" style="float: right; margin: 1em 0 1em 1em; border: none;"/>
        </a>
        <p class="question">
         What is
         <em>
          Weblocks
         </em>
         ?
        </p>
        <p class="answer">
         <em>
          Weblocks
         </em>
         is a continuations-based
	  web framework written in Common Lisp.
        </p>
        <p class="question">
         Why another web framework?
        </p>
        <p class="answer">
         <em>
          Weblocks
         </em>
         uses powerful Lisp
	  features like multiple dispatch, metaobject protocol,
	  lexical closures, keyword arguments, and macros to build
	  abstractions that make web development easy, intuitive, and
	  free of boilerplate. In addition, control flow is easily
	  expressed using continuations provided by
         <a href="http://common-lisp.net/project/cl-cont">
          cl-cont
         </a>
         library. Things that are hard or mundane in other frameworks
	  become easy and fun in
         <em>
          Weblocks
         </em>
         . Take a look at the
         <a href="http://trac.common-lisp.net/cl-weblocks/wiki/UserManual">
          user
	  manual
         </a>
         for more details.
        </p>
        <p class="question">
         What license is
         <em>
          Weblocks
         </em>
         released under?
        </p>
        <p class="answer">
         <em>
          Weblocks
         </em>
         is released under
         <a href="http://www.cliki.net/LLGPL">
          LLGPL
         </a>
         (LGPL with
	  Franz's
         <a href="http://opensource.franz.com/preamble.html">
          preamble
         </a>
         that resolves some Lisp related ambiguities). In human
	  terms, you can use
         <em>
          Weblocks
         </em>
         for anything you want
	  (including proprietary software), but you must release
	  changes made to
         <em>
          Weblocks
         </em>
         itself for the world to
	  enjoy.
        </p>
        <p class="question">
         Is
         <em>
          Weblocks
         </em>
         platform
	  independent?
        </p>
        <p class="answer">
         <em>
          Weblocks
         </em>
         is fully supported on
	  SBCL, CMUCL, Lispworks, AllegroCL, and OpenMCL on platforms
	  where multithreading is available. It is periodically tested
	  on Linux and OS X, and has been reported to run on
	  Win32.
        </p>
        <p class="question">
         What is the current status of
         <em>
          Weblocks
         </em>
         ?
        </p>
        <p class="answer">
         <em>
          Weblocks
         </em>
         is currently very
	  functional and has stood up to reasonably heavy load without
	  any stability issues. However, it is a very young piece of
	  software - it needs polish and API changes are not guaranteed
	  to be backwards compatible. If you feel adventurous, do give
         <em>
          Weblocks
         </em>
         a try!
        </p>
        <p class="question">
         Where can I find documentation?
        </p>
        <p class="answer">
         You can find the latest documentation
	  (including a user manual and reference documentation) on the
         <a href="http://trac.common-lisp.net/cl-weblocks">
          Trac
	  page
         </a>
         .
        </p>
        <p class="question">
         How do I get
         <em>
          Weblocks
         </em>
         ?
        </p>
        <p class="answer">
         Please visit
         <a href="http://trac.common-lisp.net/cl-weblocks/wiki/ObtainingAndInstalling">
          Obtaining
	  And Installing
         </a>
         page.
        </p>
        <p class="question">
         How do I get started writing
         <em>
          Weblocks
         </em>
         applications?
        </p>
        <p class="answer">
         Please visit
         <a href="http://trac.common-lisp.net/cl-weblocks/wiki/UserManual#BreakingtheIce">
          Breaking
	  The Ice
         </a>
         section of the user manual.
        </p>
        <p class="question">
         How do I get support?
        </p>
        <p class="answer">
         For support questions and bug reports

         <a href="http://groups.google.com/group/weblocks/topics">

           Weblocks

          google group
         </a>

        </p>
