      <h1 itemprop="headline">
       Build Your Own Web or Mobile App In Minutes With These Cloud Based Tools
      </h1>
      <!-- Make individual group author appear -->
      <div class="mninst">
      </div>
      <div class="comment_links">
       <a class="scroll_to post_your_comment" href="#comment_reply">

      <div class="body">
       <div class="body_inner">
        <div class="zemanta-img">
         <div class="wp-caption alignright" style="width: 235px">
          <img alt="A man waits in line outside the Apple store on..." class="zemanta-img-configured" height="300" src="http://b-i.forbesimg.com/reuvencohen/files/2013/09/225x30011.jpg" width="225"/>
          <p class="wp-caption-text">
          </p>
         </div>
        </div>
        <p>
         Its been awhile since Ive had to sit down and actually build something for the web. When I started developing websites way back in 1995 there really were not a lot of choices other than a text editor and some good old fashion trial and error. I still remember the day I received my first copy ofFrontPage, not the
         <a href="/companies/microsoft/">
          Microsoft
         </a>
         <span class="quotecard_hook" data-exchange="NASDAQ" data-naturalid="fred/company/2854" data-quotes-closing="45.53" data-quotes-now="45.62" data-ticker="MSFT" data-type="organization">
          <span class="wrapper">
           <a href="/companies/microsoft/">
            Microsoft
           </a>
          </span>
         </span>
         version but theVermeer Technologies version pre-acquisition. Later, as my company grew I began to do less and less in terms of actual development work until finally the company was
         <a href="http://gigaom.com/2011/12/15/virtustream-buys-cloud-pioneer-enomaly/">
          acquired
         </a>
         . Now in my new role, Im finally getting the opportunity to explore my developer roots and explore some of the latest approaches to creating web and mobile applications.
        </p>
        <p>
         A lot has changed since 1995, literally anyone with almost notechnicalcapabilities can crank out a web or mobile app with ease. So I thought Id take theopportunityto share some of the more interesting App development platforms Ive discoveredrecently.
        </p>
        <p>
         <a href="http://www.Knackhq.com">
          <strong>
           Knack -Create online databases. All by yourself.
          </strong>
         </a>
        </p>
        <p>
         Knack is an interesting platform. Where many companies in the data space focus on so called Big Data solutions, Knack has created what can be best described as a cross between
         <a href="http://www.filemaker.com/products/filemaker-pro/">
          Filemaker Pro
         </a>
         and
         <a href="http://www.caspio.com/">
          Caspio
         </a>
         for little data. They describe themselves as a way tobuild simple web apps. The DIY tool is focused on building web apps that work with your data. The platform makes including functionality like search, custom forms and data display a breeze. The cloud service can be easily integrated in your website via a Javascript or web API.
        </p>
        <p>
         Although the company is very new, it has already managed to attract a dedicated customer base including a variety of up-starts like
         <a href="http://milelogic.com">
          transportationmanagement platform
         </a>
         <a href="http://MileLogic.com">
          MileLogic.com
         </a>
         who are using the plaform to manage &amp; track fleet assets, truck mileage, jobs dispatching, and taxes. Another firm using the plaform is
         <a href="http://Prontomarketing.com">
          Prontomarketing.com
         </a>
         who uses it forcreating custom reports and pivot table searches. The company says that this sort of thing just isnt possible by using
         <a href="/companies/google/">
          Google
         </a>
         <span class="quotecard_hook" data-exchange="NASDAQ" data-naturalid="fred/company/1821" data-quotes-closing="546.5011" data-quotes-now="561.37" data-ticker="GOOG" data-type="organization">
          <span class="wrapper">
           <a href="/companies/google/">
            Google
           </a>
          </span>
         </span>
         Spreadsheet formulas. With Knack they are finding all sorts of new ways to explore and share our data.
        </p>
        <p>
         <strong>
          <a href="http://www.Appery.io">
           Appery.io-Cloud-based Mobile App Builder
          </a>
         </strong>
        </p>
        <p>
         Another very cool platform is
         <a href="http://www.Appery.io">
          Appery.io
         </a>
         formerly called
         <a href="http://tiggzi.com/features">
          Tiggzi
         </a>
         . The platfrom is described as a cloud-based HTML5, jQuery Mobile and Hybrid Mobile App Builder. Its very easy to use and builds real cross platform HTML5 applications using jQuery Mobile. It supports Android, iOS and Windows Phone using integrated
         <a href="/companies/apache/">
          Apache
         </a>
         <span class="quotecard_hook" data-exchange="NYSE" data-naturalid="fred/company/273" data-quotes-closing="53.26" data-quotes-now="54.39" data-ticker="APA" data-type="organization">
          <span class="wrapper">
           <a href="/companies/apache/">
            Apache
           </a>
          </span>
         </span>
         Cordova/PhoneGap output. As the builder is running in the cloud, there is nothing to install or download so its very easy to build your app using a powerful drag and drop visual builder tool. The company says amobile UI is not very useful without data so Appery.ioconnects to any REST API that can be easily used in your app. You can even test the REST API before using it on the page and automatically define the service response structure. This makes it very fast to try any new service.
        </p>
        <p>
         <strong>
          <a href="https://codiqa.com/">
           Codiqa -Pure HTML5 with jQuery Mobile
          </a>
         </strong>
        </p>
        <p>
         If you are less programmer and moredesignerthen youll loveCodiqa. Similar to Tiggzi, but less developer focused,Codiqasdrag-and-drop interface allows for fast and easy mobile prototype building, with a Preview mode to test your prototype. The cloud based platform builds your App with 100% HTML5 components. Their pitch is you can take your idea, prototype it, and output real, usable code. No need to code all over again after making your prototype.No coding necessary.
        </p>
        <p>
         <strong>
          <a href="http://mobile.conduit.com/Features.aspx">
           Conduit  Make Free Mobile Apps
          </a>
         </strong>
        </p>
        <p>
         By far the simplest mobile application builder Ive found is from Conduit. Their pitch is pretty simple, build stunning app designs quick and easy  no coding, one click and its ready! Ive tried the application for a
         <a href="http://virtustream.conduitapps.com">
          Virtustream Mobile App
         </a>
         and I was literally able to develop an App for both iOS and
         <a href="https://play.google.com/store/apps/details?id=com.conduit.app_10a6b86031e74de584dcd1b64c26ea2e.app">
          Android
         </a>
         in a matter of minutes. The tool includes a variety of components such as social feed, RSS, events, e-commerce,notificationsand more. They will even submit your App to the various App Stores for you. If youre non-technical. This is the solution for you.
        </p>
        <p>
         <strong>
          <a href="http://www.kinvey.com/">
           Kinvey -Bring store-ready mobile apps to market faster
          </a>
         </strong>
        </p>
        <p>
         Kinvey is probably the most technical of the various cloud based app development platforms I looked at. But if you dont mind a little programming, its probably the most powerful too. The company describes itself as taking the hassle out of building and operating mobile backends and like the others listed it comes through on its promise. Kinveys value proposition is toeliminate your risk by reducing backend development time from weeks to hours, allowing you to prototype faster and with less effort. No servers to spin up and manage, no API spaghetti to detangle, no tedious backend code (e.g., database, user / file management) to write. And you never have to pay unless your app is successful. Major companies like
         <a href="/companies/johnson-johnson/">
          Johnson &amp; Johnson
         </a>
         <span class="quotecard_hook" data-exchange="NYSE" data-naturalid="fred/company/2359" data-quotes-closing="100.27" data-quotes-now="99.78" data-ticker="JNJ" data-type="organization">
          <span class="wrapper">
           <a href="/companies/johnson-johnson/">
            Johnson &amp; Johnson
           </a>
          </span>
         </span>
         and startups like
         <a href="http://www.fetchnotes.com/">
          Fetchnotes
         </a>
         have built apps on Kinvey. Its a very nice platform and worth a closer look.
        </p>
        <p>
         <strong>
          <a href="http://Keen.io">
           Keen.io -The API for Custom Analytics
          </a>
         </strong>
        </p>
