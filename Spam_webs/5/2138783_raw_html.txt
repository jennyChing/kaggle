  <h3>
   About
  </h3>
  <p>
   Ultra Monkey is a project to create load balanced and highly available
  network services. For example a cluster of web servers that appear as a
  single web server to end-users. The service may be for end-users across
  the world connected via the internet, or for enterprise users connected
  via an intranet.
  </p>
  <p>
   Ultra Monkey makes use of the Linux operating system to provide a
  flexible solution that can be tailored to a wide range of needs. From
  small clusters of only two nodes to large systems serving thousands of
  connections per second.
  </p>

        <a name="2006080114">
         <b>
          heartbeat update (1.2.3.cvs.20050927-1bpo3 / 1.2.3.cvs.20050927-1.rh.el.um.3)
         </b>
         <p>
          New heartbeat packages have been made available for 
UltraMonkey3 on both
          <a href="http://www.redhat.com">
           RedHatEnterpriseLinux3[offsite]
          </a>
          and
          <a href="http://www.debian.org&gt;">
           DebianSarge[offsite]
          </a>
          in order to address the security flaw, CVE-2006-3121.
         </p>
         <p>
          <a href="http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2006-3121">
           CVE-2006-3121
          </a>
          is a remotely exploitable denail of service resulting from
incorrect parsing of data read from the network. The
result is that anyone who can send a packet to heartbeat
can cause it to segmentation fault.
         </p>
         <p>
          All users whose
heartbeat machines are not protected from hostile users by
a firewall are recommended to upgrade.
         </p>
         <p>
          The updated packages are available in the
          <a href="/download/3/">
           download directory
          </a>

      <h3>
       Key Features
      </h3>
      <ul>
       <li>
        Fast Load Balancing using
        <a href="3/lvs.html">
         The Linux Virtual Server
        </a>
       </li>
       <li>
        Flexible High Availability provided by the
        <a href="3/linux-ha.html">
         Linux-HA framework
        </a>
       </li>
       <li>
        Service level monitoring using ldirectord
        <a href="3/ldirectord.html">
         ldirectord
        </a>
       </li>
       <li>
        Supports Highly Available and/or Load Balanced 
    topologies with worked
        <a href="3/topologies">
         configuration examples
        </a>
       </li>
       <li>
        Easily expandable to a large number of IP based virtual services
    using
        <a href="3/topologies/ha-lb-eg-fwmark.html">
         fwmarks
        </a>
       </li>
       <li>
        Pre-built packages for
        <a href="http://www.debian.org">
         DebianSarge[offsite]
        </a>
        and
        <a href="http://www.redhat.com">
         RedHatEnterpriseLinux3[offsite]
        </a>
        .
       </li>
       <li>
        All Code is Open Source
       </li>
      </ul>
      <h3>
       New in UltraMonkey3
      </h3>
      <ul>
       <li>
        <a href="3/lvs.html">
         Connection Syncronisation
        </a>
        allows connections to continue
    even if the active linux-director fails and the standb-by
    is brought online
       </li>
       <li>
        Native health checking of MySQL servers using
        <a href="3/ldirectord.html">
         ldirectord
        </a>
       </li>
       <li>
        Improved handling of ARP packets in
        <a href="3/topologies/hc-ha-lb-overview.html">
         High Capacity Highly Availability and Load Balancing
        </a>
        and
        <a href="3/topologies/hc-ha-lb-overview.html">
         Streamline Highly Availability and Load Balancing
        </a>
        topologies.
       </li>
       <li>
        Custom kernel is no longer required as the kernels provided
    by supported distributions include all the features required
    for an UltraMonkey system.
       </li>
      </ul>
