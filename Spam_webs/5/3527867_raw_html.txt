         Google Font API: Fonts Viewer  Preview all available Fonts in Google Directory
        </h1>
       </header>
       <div class="content clearfix" itemprop="articleBody">
        <p>
         Continue from
         <a href="/2010/06/google-font-api-font-viewer-1.html">
          Google Font API: Why it exists and How it is useful!

         <br/>
         <strong>
          Google API Font Viewer Application:
         </strong>
         <br/>
         <a href="/2010/02/googles-secret-plan-be-a-data-center-of-whole-internet.html">
          Google
         </a>
         Font Viewer Application will display text using all fonts available as of today according to style you provide below. I checked the application only in Firefox. 'Show me' button will display your text in various fonts and 'Clear Display' button will clear the display so that you can try afresh.
        </p>
        <p>
         Web fonts is stored at different places other than clients system. When I was reading the
         <a href="http://code.google.com/apis/webfonts/docs/getting_started.html#Quick_Start" rel="external">
          Google Documentation for Font API
         </a>
         , I got that for italics or bold you need to specify in request itself, so that kind of glyph can be send in response.
        </p>

        <p>
         <b>
          Update
         </b>
         <small>
          (30-07-2010)
         </small>
         : Not all fonts have every variants. Example is
         <a href="http://code.google.com/webfonts/family?family=Tangerine#variants">
          Tangerine
         </a>
         . This does not have italics variant as said in Google webfonts.
         <br/>
         One thing to investigate is when I use here italics for the font then it give me any other font and does not look like Tangerine and it works when I use bold. According to Google WebFonts (as of now), bold is available for the font. But the same things is working in Google Font previewer even if they say that italics is not available! At least their they do not server different font.
         <br/>
         So, better try fonts without any variants first. Then see if your variants is available for the font or not!
        </p>
