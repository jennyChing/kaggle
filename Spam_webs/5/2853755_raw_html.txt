     <p>
      <b>
       Mega
      </b>
      (stylized in uppercase as
      <b>
       MEGA
      </b>
      ) is a
      <a class="mw-redirect" href="/wiki/Cloud_storage_service" title="Cloud storage service">
       cloud storage
      </a>
      and
      <a href="/wiki/File_hosting_service" title="File hosting service">
       file hosting service
      </a>
      produced by Mega Limited. The New Zealand-based website was launched on 19 January 2013 by
      <a href="/wiki/Kim_Dotcom" title="Kim Dotcom">
       Kim Dotcom
      </a>
      . Mega
      <a href="/wiki/Mobile_app" title="Mobile app">
       mobile apps
      </a>
      are available for
      <a href="/wiki/Windows_Phone" title="Windows Phone">
       Windows Phone
      </a>
      ,
      <a href="/wiki/Android_(operating_system)" title="Android (operating system)">
       Android
      </a>
      ,
      <a href="/wiki/BlackBerry_OS" title="BlackBerry OS">
       BlackBerry OS
      </a>
      and
      <a href="/wiki/IOS" title="IOS">
       iOS
      </a>
      .
     </p>
     <p>
      Mega most notably advertises its feature that all files are encrypted locally before they are uploaded
      <sup class="reference" id="cite_ref-3">
       <a href="#cite_note-3">
        <span>
         [
        </span>
        3
        <span>
         ]
        </span>
       </a>
      </sup>
      and 50
      <a href="/wiki/Gigabyte" title="Gigabyte">
       GB
      </a>
      of storage space are available for free
      <sup class="reference" id="cite_ref-free_space_4-0">
       <a href="#cite_note-free_space-4">
        <span>
         [
        </span>
        4
        <span>
         ]
        </span>
       </a>
      </sup>
      and up to 4
      <a href="/wiki/Terabyte" title="Terabyte">
       TB
      </a>
      for paid accounts.
      <sup class="reference" id="cite_ref-Mega_storage_5-0">
       <a href="#cite_note-Mega_storage-5">
        <span>
         [
        </span>
        5
        <span>
         ]
        </span>
       </a>
      </sup>
      As of November 2014, according to Dotcom, Mega has 15 million registered users, and utilizes 350
      <a href="/wiki/Gigabit" title="Gigabit">
       gigabits
      </a>
      of bandwidth on average.
      <sup class="reference" id="cite_ref-unBound_6-0">
       <a href="#cite_note-unBound-6">
        <span>
         [
        </span>
        6
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>

     <h2>
      <span class="mw-headline" id="History">
       History
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Mega_(service)&amp;action=edit&amp;section=1" title="Edit section: History">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Kim Dotcom's first file-hosting site
      <a class="mw-redirect" href="/wiki/Megaupload.com" title="Megaupload.com">
       Megaupload
      </a>
      was seized and shut down on 19 January 2012 by the United States Department of Justice, which began criminal cases against its owners. After
      <a href="/wiki/Gabon" title="Gabon">
       Gabon
      </a>
      denied the new company domain name me.ga,
      <sup class="reference" id="cite_ref-3news_gabon_no_7-0">
       <a href="#cite_note-3news_gabon_no-7">
        <span>
         [
        </span>
        7
        <span>
         ]
        </span>
       </a>
      </sup>
      <a href="/wiki/Kim_Dotcom" title="Kim Dotcom">
       Kim Dotcom
      </a>
      announced it would instead be registered in his home country of New Zealand under the domain name mega.co.nz.

     <h2>
      <span class="mw-headline" id="Data_encryption">
       Data encryption
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Mega_(service)&amp;action=edit&amp;section=2" title="Edit section: Data encryption">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <div class="thumb tright">
      <div class="thumbinner" style="width:222px;">
       <a class="image" href="/wiki/File:Mega.co.nz_file_uploader.PNG">
        <img alt="" class="thumbimage" data-file-height="250" data-file-width="400" height="138" src="//upload.wikimedia.org/wikipedia/en/thumb/b/b9/Mega.co.nz_file_uploader.PNG/220px-Mega.co.nz_file_uploader.PNG" srcset="//upload.wikimedia.org/wikipedia/en/thumb/b/b9/Mega.co.nz_file_uploader.PNG/330px-Mega.co.nz_file_uploader.PNG 1.5x, //upload.wikimedia.org/wikipedia/en/b/b9/Mega.co.nz_file_uploader.PNG 2x" width="220"/>
       </a>
       <div class="thumbcaption">
        <div class="magnify">
         <a class="internal" href="/wiki/File:Mega.co.nz_file_uploader.PNG" title="Enlarge">
         </a>
        </div>
        The Mega.co.nz
        <a href="/wiki/HTML5" title="HTML5">
         HTML5
        </a>
        file manager
       </div>
      </div>
     </div>
     <p>
      Dotcom has said that data on the Mega service will be encrypted
      <a href="/wiki/Client-side_encryption" title="Client-side encryption">
       client-side
      </a>
      using the
      <a href="/wiki/Advanced_Encryption_Standard" title="Advanced Encryption Standard">
       AES
      </a>
      algorithm. Since Mega does not know the encryption keys to uploaded files, they cannot decrypt and view the content. Therefore, they cannot be responsible for the contents of uploaded files.
      <sup class="reference" id="cite_ref-31">
       <a href="#cite_note-31">
        <span>
         [
        </span>
        31
        <span>
         ]
        </span>
       </a>
      </sup>
      Dotcom stated that encrypting files allows them to work with a larger number of data hosting companies around the world, decreasing the likelihood of a Megaupload-style seizure of servers by one government. He mentioned in an interview with
      <a href="/wiki/Ars_Technica" title="Ars Technica">
       Ars Technica
      </a>
      that "Each file will be kept with at least two different hosters [sic], [in] at least two different locations," and "Thats a great added benefit for us because you can work with the smallest, most unreliable [hosting] companies. It doesnt matter because they cant do anything with that data."
      <sup class="reference" id="cite_ref-ars_interview_32-0">
       <a href="#cite_note-ars_interview-32">
        <span>
         [
        </span>
        32
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      In the first few weeks after the Mega launch, various security problems were found that researchers said an attacker could use to gain access to a logged-in user's files.
      <sup class="reference" id="cite_ref-SO_attackers_33-0">
       <a href="#cite_note-SO_attackers-33">
        <span>
         [
        </span>
        33
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-mega_blog_3_34-0">
       <a href="#cite_note-mega_blog_3-34">
        <span>
         [
        </span>
        34
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-mega_blog_4_35-0">
       <a href="#cite_note-mega_blog_4-35">
        <span>
         [
        </span>
        35
        <span>
         ]
        </span>
       </a>
      </sup>
      In response, Mega started a vulnerability reward program which offers a reward of up to 10,000 for reporting security problems to Mega.
      <sup class="reference" id="cite_ref-mega_blog_6_36-0">
       <a href="#cite_note-mega_blog_6-36">
        <span>
         [
        </span>
        36
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      The Mega team indicated that some companies, such as film studios, will have direct access to remove files if they discover the encryption keys online and determine that the content infringes their copyright. Dotcom added that if such companies want to use that tool they would have to agree, prior to receiving access, not to sue Mega or hold the site accountable for the actions of its users.
      <sup class="reference" id="cite_ref-cnet_rises_37-0">
       <a href="#cite_note-cnet_rises-37">
        <span>
         [
        </span>
        37
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
