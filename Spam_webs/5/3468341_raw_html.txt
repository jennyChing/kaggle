		                &lt;p class="blog-post-meta"&gt;&lt;span class="uppercase-text"&gt;April 16, 2015 &middot; 
		                
		                &lt;a href='http://www.htmlcenter.com/blog/category/angular-js/'&gt;Angular.js&lt;/a&gt; &middot; &lt;a href='http://www.htmlcenter.com/blog/category/html-tutorials/'&gt;HTML Tutorials&lt;/a&gt; &middot; &lt;a href='http://www.htmlcenter.com/blog/category/html5-2/'&gt;HTML5&lt;/a&gt; &middot; 						&lt;/span&gt;
		                &lt;p&gt;&lt;em&gt;We have recently created &lt;a title="HTMLCenter Angular Starter app" href="https://github.com/PopularOwl/angular-starter"&gt;example Angular.js web app&lt;/a&gt; with the minimal generic functionality. Small enough to be called framework but with built-in test cases, controllers and several views with user forms. The goal is to have a reusable foundation for Angular projects. This blog post is about how we designed Angular Starter structure.&lt;/em&gt;&lt;/p&gt;
&lt;h3&gt;Why Angular Starter&lt;/h3&gt;
&lt;p&gt;There are many tutorials on how to start application development with Angular.js. As well as a good few projects on GitHub with the Angular app skeletons. Like the &lt;a title="Angular seed project" href="https://github.com/angular/angular-seed"&gt;angular-seed project&lt;/a&gt; created by Angular.js development team itself.&lt;/p&gt;
&lt;p&gt;What we needed was a minimal project, with pre-built basic functionality and structure of communication with web services and APIs. And of course automated test cases.&lt;/p&gt;
&lt;p&gt;We wanted test cases to get some special attention as we are big fans of TDD (test-driven development). Having the majority of test cases created even before any app functionality is coded is a good help to keep those bugs away.&lt;/p&gt;
&lt;p&gt;I haven&#8217;t found the open source project which completely match my requirements and Angular Starter was born. It&#8217;s an open source project, &lt;a title="Angular.js Starter application" href="https://github.com/PopularOwl/angular-starter"&gt;hosted on GitHub&lt;/a&gt; and I welcome all suggestions and comments.&lt;/p&gt;

&lt;h3&gt;Creating Parse User Database&lt;/h3&gt;
&lt;p&gt;Nowadaysyou have many options once it comes to cloud-based storage options for mobile web apps. We did a good &lt;a title="HTML BaaS" href="http://www.htmlcenter.com/blog/what-baas-for-html5-application/" target="_blank"&gt;summary on HTMLCenter&lt;/a&gt; a few posts ago.&lt;/p&gt;

They have free tier available for starting apps &#8211; it allows for up to 30 requests per second throughput and has 2TB data transfer limit per month. I&#8217;m sure we will not need this capacity in our tutorial!&lt;/p&gt;

		                &lt;/div&gt;
		              &lt;/div&gt;

		              &lt;div class="blog-post-content"&gt;
		                &lt;p class="blog-post-meta"&gt;&lt;span class="uppercase-text"&gt;March 31, 2015 &middot; 
		                
		                &lt;a href='http://www.htmlcenter.com/blog/category/blog-2/'&gt;Blog&lt;/a&gt; &middot; &lt;a href='http://www.htmlcenter.com/blog/category/html5-2/'&gt;HTML5&lt;/a&gt; &middot; &lt;a href='http://www.htmlcenter.com/blog/category/html5-tips/'&gt;HTML5 Tips&lt;/a&gt; &middot; 						&lt;/span&gt;
		                &lt;p&gt;&lt;em&gt;Many developers name 2 way binding as the main advantage of &lt;a title="Angular.js home" href="https://angularjs.org/" target="_blank"&gt;Angular.js framework&lt;/a&gt;. So what 2 way binding actually is and how it helpsAngular.js framework to deliver better experience?&lt;/em&gt;&lt;/p&gt;
&lt;h3&gt;1 way data binding&lt;/h3&gt;
&lt;p&gt;The below images explain the difference quite well.&lt;/p&gt;
&lt;p&gt;In &lt;a title="Model view controller" href="http://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93controller" target="_blank"&gt;MVC&lt;/a&gt; (model, view &amp; controller) world 1 way data binding happens when controller generates view bymerging model and view template. This process happens once and generated view is then sent to the client application. Once client application receives rendered view there is no easy way to automatically update thisview if model data changes.&lt;/p&gt;
&lt;p&gt;View is only regenerated once user refreshes the the app screen / page with current view. Or navigates to another application page.&lt;/p&gt;

		                &lt;/div&gt;
		              &lt;/div&gt;

		              &lt;div class="blog-post-content"&gt;
		                &lt;p class="blog-post-meta"&gt;&lt;span class="uppercase-text"&gt;March 18, 2015 &middot; 
		                
		                &lt;a href='http://www.htmlcenter.com/blog/category/blog-2/'&gt;Blog&lt;/a&gt; &middot; &lt;a href='http://www.htmlcenter.com/blog/category/twitter-bootstrap/'&gt;Bootstrap&lt;/a&gt; &middot; &lt;a href='http://www.htmlcenter.com/blog/category/community-2/'&gt;Community&lt;/a&gt; &middot; 						&lt;/span&gt;
		                &lt;p&gt;&lt;em&gt;Released back in 2011, Twitter&#8217;s &lt;a href="http://getbootstrap.com/"&gt;Bootstrap&lt;/a&gt;framework is one of the most popular &#8211; if not the most popular &#8211; front-end website development framework out there, at least in 2015.&lt;/em&gt;&lt;/p&gt;
&lt;p&gt;And not without a reason. Bootstrap provides a really easy to grasp way of crafting your site designs and thus allows you to create great looking and mobile-optimized builds in hours, instead of days.&lt;/p&gt;
&lt;p&gt;&lt;img class="aligncenter size-full wp-image-4015" src="http://www.htmlcenter.com/wp-content/uploads/2015/03/Bootstrap-should-you-use-this-framework.png" alt="Bootstrap should you use this framework" width="500" height="257" /&gt;&lt;/p&gt;
&lt;p&gt;That being said, there&#8217;s a lot of conflicting opinions around Bootstrap with almost equal number of people on both sides of the barricade. Some experienced developers (like the guys over at ThemeShaper, for example) have addressed various issues with the framework and pointed out why it might not be the perfect web development solution after all.&lt;/p&gt;
&lt;p&gt;So today, let&#8217;s look into some of the framework&#8217;s known traits, both in terms of the positives as well as the negatives. However, I&#8217;ll try not to sway your opinion in any direction by telling you what I consider the right solution here &#8211; in terms of using Bootstrap or not &#8211; but instead, I just want to provide enough insight for you to be able to make your own decision.&lt;/p&gt;
