       <div class="shareaholic-canvas" data-app="share_buttons" data-app-id="15186113" data-link="http://insights.dice.com/2014/09/04/key-skills-needed-cloud-engineers/" data-summary="" data-title="The Key Skills Needed by Cloud Engineers">
       </div>
       <p>
        <a href="http://insights.dice.com/wp-content/uploads/2014/09/CloudSkills.jpg">
         <img alt="Cloud Skills" class="aligncenter wp-image-122716" height="430" src="http://insights.dice.com/wp-content/uploads/2014/09/CloudSkills.jpg" width="585"/>
        </a>
       </p>
       <p>
        To outsiders,
        <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Cloud+Computing&amp;x=all&amp;p=/?icid=dicenewsEJ">
         cloud computing
        </a>
        sounds like something lightweight, fluffy and full of rainbows. But cloud computing is actually very deep and technical, and requires engineers to be versed in the cutting edge of technology.
       </p>
       <p>
        If youre an expert engineer in another discipline, you may already have many of the skills you need to succeed in the cloud. And since the cloud is still fairly new, you can set yourself apart by being able to apply what you know to learn fast on the job.
       </p>
       <p>
        <em>
         <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Cloud+Engineer&amp;x=all&amp;p=/?icid=dicenewsCH">
          Click here to find cloud engineer jobs.
         </a>
        </em>
       </p>
       <h3>
        A Mix of Operations, Software and Architecture
       </h3>
       <p>
        As a cloud engineer, you need to understand the ins and outs of building and running software in the cloud. Although this role typically requires programming and scripting experience, the specific language requirements tend to be a bit more relaxed than in traditional engineering jobs.
       </p>
       <p>
        What you need:
       </p>
       <ul>
        <li>
         <strong>
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=AWS&amp;x=all&amp;p=/?icid=dicenewsEJ">
           AWS
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Azure&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Azure
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=OpenStack&amp;x=all&amp;p=/?icid=dicenewsEJ">
           OpenStack
          </a>
         </strong>
         . You should be familiar with at least one of these. If you are well-versed in one stack, that knowledge will translate fairly easily to designing software for the others. Of course, that will involve a bit of learning on the job to do it well.
        </li>
        <li>
         <strong>
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Web+Services&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Web Services
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=API&amp;x=all&amp;p=/?icid=dicenewsEJ">
           API
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=REST&amp;x=all&amp;p=/?icid=dicenewsEJ">
           REST
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=RPC&amp;x=all&amp;p=/?icid=dicenewsEJ">
           RPC
          </a>
          .
         </strong>
         The underlying foundation of cloud architecture is based on APIs and Web Services. You probably already have experience with these types of service patterns and protocols from working on websites, and that knowledge will give you a head start on mastering cloud fundamentals.
        </li>
        <li>
         <strong>
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Virtualization&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Virtualization
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Storage&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Storage
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Networking&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Networking
          </a>
          .
         </strong>
         In the world of cloud computing, these skills can be very useful for designing and operating applications. If youve got these skills, thats a very good thing! At the very least, you should have a strong general programming background, because without it the learning curve may be a bit too steep.
        </li>
        <li>
         <strong>
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Disaster+Recovery&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Disaster Recovery
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=High+Availability&amp;x=all&amp;p=/?icid=dicenewsEJ">
           High Availability
          </a>
          ,
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Fail+Over&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Fail Over
          </a>
          and
          <a href="http://www.dice.com/job/results?caller=searchagain&amp;q=Redundancy&amp;x=all&amp;p=/?icid=dicenewsEJ">
           Redundancy
          </a>
         </strong>
         . These are methodologies that are central to operating software in the cloud, and are skills you typically get in an operations role. What if you havent done operations before? As long as youre familiar with these concepts, the pieces related to cloud architecture can be learned with a little bit of experience and training on the job.
        </li>
       </ul>
       <p>
        The best candidates for a cloud engineers role will possess strong technical skills, the ability to think through business use cases (does this system need to scale to accommodate increased traffic?) and an intellectual curiosity to learn new tools and technology.
       </p>
       <p>
        Is that you? Then you may be ready to reach for the sky and start engineering the cloud.
       </p>
