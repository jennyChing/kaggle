      <tr>
       <td colspan="2">
        <h1>
         How to use NOINDEX, NOFOLLOW and NOODP
        </h1>
       </td>
      </tr>
      <tr bgcolor="#eeeeee">
       <td class="smalltext">
        Posted On 09 Oct, 2007
       </td>
       <td align="right" class="smalltext">
        Views : 5621
        <a href="/articles/search-engine-optimization/23/robots-and-robotstxt-file.htm">
         Previous
        </a>
        | Next
       </td>
      </tr>
      <tr>
       <td colspan="2">
        <table align="right">
         <tr>
          <td>
          </td>
         </tr>
        </table>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <font size="3">
          <font face="Times New Roman">
           <strong style="mso-bidi-font-weight: normal">
            Use
           </strong>
           <st1:place>
            <strong style="mso-bidi-font-weight: normal">
             Meta
            </strong>
           </st1:place>
           <strong style="mso-bidi-font-weight: normal">
            Tags
            <o:p>
            </o:p>
           </strong>
          </font>
         </font>
        </p>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <font size="3">
          <font face="Times New Roman">
           Different meta tags have different instructions to follow, commonly used meta tags are:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          NOINDEX
         </span>
         <st1:place>
          <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
           Meta
          </span>
         </st1:place>
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Tag
          <o:p>
          </o:p>
         </span>
        </h2>
        <p>
         <font size="3">
          <font face="Times New Roman">
           If you include a tag like:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;
          </span>
         </strong>
         <st1:place>
          <strong style="mso-bidi-font-weight: normal">
           <span style='font-size: 12pt; font-family: "Times New Roman"'>
            META
           </span>
          </strong>
         </st1:place>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           NAME="ROBOTS" CONTENT="NOINDEX"&gt;
           <o:p>
           </o:p>
          </span>
         </strong>
        </pre>
        <p class="MsoNormal" style="margin: 0in 0in 0pt">
         <o:p>
          <font face="Times New Roman" size="3">
          </font>
         </o:p>
        </p>
        <p class="MsoNormal" style="margin: 0in 0in 0pt">
         <font size="3">
          <font face="Times New Roman">
           in your HTML document, that document won't be indexed.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Keep in mind that search engines will still spider these pages on a regular basis. They continue to crawl "noindex" pages in order to check the current status of a page's robots meta tag. There is no need to use an index tag; index is the default option. Using a default tag just adds
           <span style="mso-spacerun: yes">
           </span>
           bloat to your web pages. The only time you might use them is to override a global setting:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="robots" content="noindex" /&gt;
           <o:p>
           </o:p>
          </span>
         </code>
        </pre>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="googlebot" content="index" /&gt;
           <o:p>
           </o:p>
          </span>
         </code>
        </pre>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           <o:p>
           </o:p>
          </span>
         </code>
        </pre>
        <pre>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           Advantages
           <o:p>
           </o:p>
          </span>
         </strong>
        </pre>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l7 level1 lfo1">
          <font size="3">
           <font face="Times New Roman">
            Allows page level granularity of robots commands.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <strong style="mso-bidi-font-weight: normal">
          <font size="3">
           <font face="Times New Roman">
            Disadvantages
            <o:p>
            </o:p>
           </font>
          </font>
         </strong>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l4 level1 lfo2">
          <font size="3">
           <font face="Times New Roman">
            The use of a
            <em>
             <span style="font-style: normal; mso-bidi-font-style: italic">
              noindex
             </span>
            </em>
            meta tag is only possible with html pages (which includes dynamic pages such as php, jsp, asp). It is not possible to exclude other file types such as PDF, DOC, ODT which don't support html meta tags.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l4 level1 lfo2">
          <font size="3">
           <font face="Times New Roman">
            Pages will still be spidered by search engines to check the current robots meta tag settings. This additional traffic is avoided when using robots.txt file settings.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Nofollow
         </span>
         <st1:place>
          <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
           Meta
          </span>
         </st1:place>
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Tag
          <o:p>
          </o:p>
         </span>
        </h2>
        <p>
         <font size="3">
          <font face="Times New Roman">
           This is another method to tell search engine spiders to ignore one or more links on a page.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">
           If you do:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;
          </span>
         </strong>
         <st1:place>
          <strong style="mso-bidi-font-weight: normal">
           <span style='font-size: 12pt; font-family: "Times New Roman"'>
            META
           </span>
          </strong>
         </st1:place>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           NAME="ROBOTS" CONTENT="NOFOLLOW"&gt;
           <o:p>
           </o:p>
          </span>
         </strong>
        </pre>
        <p class="MsoNormal" style="margin: 0in 0in 0pt">
         <o:p>
          <font face="Times New Roman" size="3">
          </font>
         </o:p>
        </p>
        <p class="MsoNormal" style="margin: 0in 0in 0pt">
         <font size="3">
          <font face="Times New Roman">
           the links in that document will not be parsed by the robot.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">
           To specify
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             nofollow
            </span>
           </em>
           at the link level, add the attribute rel with the value nofollow to the link:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;a href="mypage.html" rel="nofollow" /&gt;
           <o:p>
           </o:p>
          </span>
         </code>
        </pre>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           <o:p>
           </o:p>
          </span>
         </code>
        </pre>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <strong style="mso-bidi-font-weight: normal">
          <font size="3">
           <font face="Times New Roman">
            Disadvantages
            <o:p>
            </o:p>
           </font>
          </font>
         </strong>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l3 level1 lfo5">
          <font size="3">
           <font face="Times New Roman">
            Tests show that some search engines do crawl and index nofollow links. The nofollow tag will probably diminish the ranking value a link will provide but it cannot be reliably used to stop search engines from following a link.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <h3 style="margin: auto 0in">
         <font face="Times New Roman">
          <span style="font-size: 12pt">
           Noarchive
          </span>
          <st1:place>
           <span style="font-size: 12pt">
            Meta
           </span>
          </st1:place>
          <span style="font-size: 12pt">
           Tag
           <o:p>
           </o:p>
          </span>
         </font>
        </h3>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Most search engines allow a user to view a copy of the web page that was actually indexed by the search engine. This snapshot of a page in time is called the cache copy. Internet visitors can find this functionality to be really useful if the link is no longer available or the site is down.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">
           There are several reasons to consider disabling the cache view feature for a page or an entire website.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l5 level1 lfo9">
          <font size="3">
           <font face="Times New Roman">
            Web site owners may not want visitors viewing data, such as price lists, which are not necessarily up to date.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l5 level1 lfo9">
          <font size="3">
           <font face="Times New Roman">
            Web pages viewed in a search engine cache may not display properly if embedded images are unavailable and/or browser code such as CSS and JS does not properly execute.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l5 level1 lfo9">
          <font size="3">
           <font face="Times New Roman">
            Cached page views will not show up in web log based web analytics systems. Reporting in tagged based solutions may be incorrect as well as the cached view is on a third party domain, not yours.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <p>
         <font size="3">
          <font face="Times New Roman">
           If you want a search engine to index your page without allowing a user to view a cached copy, use the
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             noarchive
            </span>
           </em>
           attribute which is officially supported by Google, Yahoo!, Windows Live and Ask:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="robots" content="noarchive" /&gt;
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Microsoft documents the
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             nocache
            </span>
           </em>
           attribute, which is equivalent to
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             noarchive
            </span>
           </em>
           , also supported by Microsoft; there is no reason to use it.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <h3 style="margin: auto 0in">
         <font face="Times New Roman">
          <span style="font-size: 12pt">
           nosnippet
          </span>
          <st1:place>
           <span style="font-size: 12pt">
            Meta
           </span>
          </st1:place>
          <span style="font-size: 12pt">
           Tag
           <o:p>
           </o:p>
          </span>
         </font>
        </h3>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Google offers an option to suppress the generation of page abstracts, called
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             snippets
            </span>
           </em>
           , in the search results. Use the following meta tag in your pages:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="googlebot" content="nosnippet" /&gt;
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <p>
         <font size="3">
          <font face="Times New Roman">
           They note that this also sets the
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             noarchive
            </span>
           </em>
           option. We would suggest you set it explicitly if that is what you want.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <h3 style="margin: auto 0in">
         <font face="Times New Roman">
          <span style="font-size: 12pt">
           noodp
          </span>
          <st1:place>
           <span style="font-size: 12pt">
            Meta
           </span>
          </st1:place>
          <span style="font-size: 12pt">
           Tag
           <o:p>
           </o:p>
          </span>
         </font>
        </h3>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Search engines generally use a page's html title when creating a search result title, the link a user clicks on to arrive at a website. In some cases, search engines may use an alternative title taken from a directory such as dmoz, the open directory, or the
           <strong>
            Yahoo! directory
           </strong>
           . Historically, many sites have had poor titles  i.e. just the company name, or worse, "default page title". Use of a human edited title from a well known directory was often a good solution. As webmasters improve the usability of their sites, page titles have become much more meaningful  and often better choices than the
           <strong>
            open directory
           </strong>
           title. The noodp metatag, supported by Microsoft, Google and Yahoo, allows a webmaster to indicate that a page's title should be used rather than the
           <strong>
            dmoz
           </strong>
           title.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="robots" content="noodp" /&gt;
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Similarly, Yahoo! offers a "
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             noydir
            </span>
           </em>
           " option to keep Yahoo! from using Yahoo! Directory titles in search results for a site's pages:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="slurp" content="noydir"&gt;
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <h3 style="margin: auto 0in">
         <font face="Times New Roman">
          <span style="font-size: 12pt">
           Expires After with unavailable_after
          </span>
          <st1:place>
           <span style="font-size: 12pt">
            Meta
           </span>
          </st1:place>
          <span style="font-size: 12pt">
           Tags
           <o:p>
           </o:p>
          </span>
         </font>
        </h3>
        <p>
         <font size="3">
          <font face="Times New Roman">
           One problem with search engines is the delay which occurs from when content is removed from a website and when that content actually disappears from search engine results. Typical time dependent content includes event information and marketing campaigns.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Pages removed from a website which still appear in search engine results generally result in a frustrating user experience  the Internet user clicks through to the website only to find themselves landing on a "Page not found" error page.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">
           In July 2007, Google introduced the "unavailable_after" tag which allows a website to specify in advance when a page should be removed from search engine results, i.e. when it will expire. This tag can be specified as a html meta tag attribute value:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="robots" content="unavailable_after:
           <u>
            dd-yy-mm
           </u>
           <u>
            time
           </u>
           CET" /&gt;
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <p>
         <font size="3">
          <font face="Times New Roman">
           or in an X-robots http header:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           X-Robots-Tag: unavailable_after:
           <u>
            dd-yy-mm
           </u>
           <u>
            time
           </u>
           GMT
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Google says the date format should be one of those specified by the ambiguous and obsolete RFC 850. We hope Google clarifies what date formats their parser can read by referering to a current date standard, such as IETF Internet standard RFC 3339. We'd also like to see detailed page crawl information in Google's Webmaster Tools. Not only could Google show when a page was last crawled, they could add expiration information, confirming proper use of the
           <strong>
            unavailable_after
           </strong>
           tag. At one point, Google did show an approximation of the number of pages crawled relative to the number specified in a sitemap, but that feature was removed. This is one case where Google should follow Yahoo's example.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <strong style="mso-bidi-font-weight: normal">
          <font size="3">
           <font face="Times New Roman">
            Advantages
            <o:p>
            </o:p>
           </font>
          </font>
         </strong>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l0 level1 lfo10">
          <font size="3">
           <font face="Times New Roman">
            A nice way to ensure search engine results are syncronized with current website content.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <strong style="mso-bidi-font-weight: normal">
          <font size="3">
           <font face="Times New Roman">
            Disadvantages
            <o:p>
            </o:p>
           </font>
          </font>
         </strong>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l2 level1 lfo11">
          <font size="3">
           <font face="Times New Roman">
            Old date specification RFC 850 is too ambiguous, thus subject to error.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l2 level1 lfo11">
          <font size="3">
           <font face="Times New Roman">
            <strong>
             unavailable_after
            </strong>
            support is currently limited to Google. We do hope the other major search engines embrace this approach as well.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <h3 style="margin: auto 0in">
         <span style="font-size: 12pt">
          <font face="Times New Roman">
           Site Preview
           <o:p>
           </o:p>
          </font>
         </span>
        </h3>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Microsoft's
           <strong>
            Live Search
           </strong>
           may offer a thumbnail view of the first six search results in some geographies. (
           <strong>
            Ask
           </strong>
           offers a similar feature called binoculars). The thumbnail preview can be disabled by blocking the
           <em>
            <span style="font-style: normal; mso-bidi-font-style: italic">
             searchpreview
            </span>
           </em>
           robot in the robots.txt file,
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           User-agent: searchpreview
           <o:p>
           </o:p>
          </span>
         </code>
        </pre>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           Disallow: /
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <p>
         <font size="3">
          <font face="Times New Roman">
           or by using a meta tag containing "noimageindex,nomediaindex":
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           &lt;meta name="robots" content="noimageindex,nomediaindex" /&gt;
          </span>
         </code>
         <span style='font-size: 12pt; font-family: "Times New Roman"'>
          <o:p>
          </o:p>
         </span>
        </pre>
        <p>
         <font size="3">
          <font face="Times New Roman">
           This meta tag was used by AltaVista at one point; it is not known to be used by any of the other major search engines.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">

        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Some other Methods
          <o:p>
          </o:p>
         </span>
        </h2>
        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Use X-Robots-Tag in your http headers
          <o:p>
          </o:p>
         </span>
        </h2>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Use of both Robot.text and meta tags are not error free because site may be exposed in robot.text file and meta tags works only for html documents, so there is no specific indexing instructions for PDF, odt, doc and other non-html files. In July 2007, Google officially introduced a solution to this problem: the ability to deliver indexing instructions in the http header information which is sent by the web server along with an object.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p>
         <font size="3">
          <font face="Times New Roman">
           The web server simply needs to add
           <strong>
            X-Robots-Tag
           </strong>
           and any of the Google supported meta tag values to the http header for an object:
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <code>
          <strong style="mso-bidi-font-weight: normal">
           <span style='font-size: 12pt; font-family: "Times New Roman"'>
            X-Robots-Tag: noindex
           </span>
          </strong>
         </code>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           <o:p>
           </o:p>
          </span>
         </strong>
        </pre>
        <pre>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           <o:p>
           </o:p>
          </span>
         </strong>
        </pre>
        <pre>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           Advantages
           <o:p>
           </o:p>
          </span>
         </strong>
        </pre>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l6 level1 lfo7">
          <font size="3">
           <font face="Times New Roman">
            An elegant way to specify
            <em>
             <span style="font-style: normal; mso-bidi-font-style: italic">
              search engine crawling instructions
             </span>
            </em>
            for non-html files without having to use robots.txt.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l6 level1 lfo7">
          <font size="3">
           <font face="Times New Roman">
            Easy to configure using the
            <u>
             <span class="Hyperlink6">
              <span style="color: windowtext">
               Apache
              </span>
             </span>
             <em>
              <span style="font-style: normal; mso-bidi-font-style: italic">
               Header append
              </span>
             </em>
            </u>
            <span class="Hyperlink6">
             <span style="color: windowtext">
              <u>
               syntax
              </u>
             </span>
            </span>
            .
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <strong style="mso-bidi-font-weight: normal">
          <font size="3">
           <font face="Times New Roman">
            Disadvantages
            <o:p>
            </o:p>
           </font>
          </font>
         </strong>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l9 level1 lfo8">
          <font size="3">
           <font face="Times New Roman">
            Most webmasters are probably not confortable setting http headers.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l9 level1 lfo8">
          <font size="3">
           <font face="Times New Roman">
            Microsoft IIS support for adding http headers has traditionally been very limited.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l9 level1 lfo8">
          <font size="3">
           <font face="Times New Roman">
            X-Robots-Tag support is currently limited to Google. We do hope the other major search engines embrace this approach as well.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Password protect sensitive content
          <o:p>
          </o:p>
         </span>
        </h2>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Sensitive content is usually protected by requiring visitors to enter a username and password. Such secure content won't be crawled by search engines. Passwords can be set at the web server level or at the application level. For server level logon setup, consult the Apache Authentication Documentation or the Microsoft IIS documentation.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <pre>
         <strong style="mso-bidi-font-weight: normal">
          <span style='font-size: 12pt; font-family: "Times New Roman"'>
           Advantages
           <o:p>
           </o:p>
          </span>
         </strong>
        </pre>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l10 level1 lfo3">
          <font size="3">
           <font face="Times New Roman">
            An effective way to keep search engines, other robots, and the general public away from content destined for a limited audience.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <strong style="mso-bidi-font-weight: normal">
          <font size="3">
           <font face="Times New Roman">
            Disadvantages
            <o:p>
            </o:p>
           </font>
          </font>
         </strong>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l1 level1 lfo4">
          <font size="3">
           <font face="Times New Roman">
            Visitors will only make an effort to access protected website areas if they have a strong motivation to view that content.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Don't link to pages you want to keep out of search engines
          <o:p>
          </o:p>
         </span>
        </h2>
        <p>
         <font size="3">
          <font face="Times New Roman">
           Search engines won't index content unless they know about it. Thus, if no one links to pages nor submits them to a search engine, a search engine won't find them. At least this is the theory. In reality, the web is so large, one can assume that sooner or later a search engine will find a page  someone will link to it.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <p class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto">
         <strong style="mso-bidi-font-weight: normal">
          <font size="3">
           <font face="Times New Roman">
            Disadvantages
            <o:p>
            </o:p>
           </font>
          </font>
         </strong>
        </p>
        <ul type="square">
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l8 level1 lfo6">
          <font size="3">
           <font face="Times New Roman">
            Anyone can link to your pages at any time.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
         <li class="MsoNormal" style="margin: 0in 0in 0pt; mso-margin-top-alt: auto; mso-margin-bottom-alt: auto; tab-stops: list .5in; mso-list: l8 level1 lfo6">
          <font size="3">
           <font face="Times New Roman">
            Some search engines can monitor pages visitors view through installed toolbars. They may use this information in the future as a means to discover and index new content.
            <o:p>
            </o:p>
           </font>
          </font>
         </li>
        </ul>
        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Partially Stop Page Content from appearing in Search Engines
          <o:p>
          </o:p>
         </span>
        </h2>
        <p>
         <font size="3">
          <font face="Times New Roman">
           There are times where only a section of a page should be kept out of a search engine. Yahoo supports a
           <strong>
            class="robots-nocontent"
           </strong>
           html tag attribute for this purpose.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
        <h2 style="margin: 12pt 0in 3pt">
         <span style='font-size: 12pt; font-style: normal; font-family: "Times New Roman"; mso-bidi-font-style: italic'>
          Removing pages which have already been indexed.
          <o:p>
          </o:p>
         </span>
        </h2>
        <p>
         <font size="3">
          <font face="Times New Roman">
           The best approach is to use one of the above methods. Over time search engines will update their indexes with regular crawling. If you want to remove content immediately, Google offers a tool specifically for this purpose. Pages will be removed for at least six months. This process is not without risk: improperly specify your URL and you may find your entire site removed.
           <o:p>
           </o:p>
          </font>
         </font>
        </p>
