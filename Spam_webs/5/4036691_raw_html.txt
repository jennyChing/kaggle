     <p>
      A
      <b>
       live
       <a class="mw-redirect" href="/wiki/CD" title="CD">
        CD
       </a>
      </b>
      ,
      <b>
       live
       <a href="/wiki/DVD" title="DVD">
        DVD
       </a>
      </b>
      , or
      <b>
       live disc
      </b>
      is a complete
      <a href="/wiki/Booting" title="Booting">
       bootable
      </a>
      computer installation including
      <a href="/wiki/Operating_system" title="Operating system">
       operating system
      </a>
      which runs in a computer's memory, rather than loading from a hard disk drive; the CD itself is
      <a href="/wiki/Read-only_memory" title="Read-only memory">
       read-only
      </a>
      . It allows users to run an
      <a class="mw-redirect" href="/wiki/Computer_operating_system" title="Computer operating system">
       operating system
      </a>
      for any purpose without installing it or making any changes to the computer's configuration. Live CDs can run on a computer without
      <a class="mw-redirect" href="/wiki/Computer_storage" title="Computer storage">
       secondary storage
      </a>
      , such as a hard disk drive, or with a corrupted hard disk drive or file system, allowing
      <a href="/wiki/Data_recovery" title="Data recovery">
       data recovery
      </a>
      . A live ISO is an
      <a href="/wiki/ISO_image" title="ISO image">
       ISO image
      </a>
      of a Live CD which can be used in virtual machine environments, mounted as if it were a CD/DVD and used as the virtual machine's boot CD. Live CDs, ISOs, and images usually include an operating system available without charge or restrictive license such as Linux, rather than a commercial one such as Microsoft Windows, for legal rather than technical reasons.
     </p>
     <p>
      The functionality of a live CD is also available with a bootable
      <a href="/wiki/Live_USB" title="Live USB">
       live USB
      </a>
      <a href="/wiki/USB_flash_drive" title="USB flash drive">
       flash drive
      </a>
      , or even an
      <a class="mw-redirect" href="/wiki/External_USB_drive" title="External USB drive">
       external USB drive
      </a>
      . These may have the added functionality of writing changes on the bootable medium.
      <a class="mw-redirect" href="/wiki/Sd_card#Optional_write-protect_tab" title="Sd card">
       Write-locked
      </a>
      <a class="new" href="/w/index.php?title=Live_SD&amp;action=edit&amp;redlink=1" title="Live SD (page does not exist)">
       Live SD
      </a>
      <a class="mw-redirect" href="/wiki/Write_Once_Read_Many" title="Write Once Read Many">
       WORM
      </a>
      systems are the direct
      <a href="/wiki/Solid-state_drive" title="Solid-state drive">
       solid-state
      </a>
      counterpart to live CDs, and can be booted natively in a media card slot or by using a USB adapter. Write-locked Live SD systems avoid excessive write cycles or corruption by ill-conditioned software, such as
      <a href="/wiki/Malware" title="Malware">
       malware
      </a>
      .
     </p>
     <p>
      While a live CD typically does not alter any operating system or files already installed on a computer's secondary storage (such as hard disk drives), many live CDs include software mechanisms and utilities for altering the host computer's data stores, including installation of an operating system. This is important for the system management aspect of live CDs, which can be useful for removing
      <a href="/wiki/Malware" title="Malware">
       malware
      </a>
      , for
      <a href="/wiki/Disk_image" title="Disk image">
       disk imaging
      </a>
      , and for
      <a href="/wiki/Recovery_disc" title="Recovery disc">
       system recovery
      </a>
      . Unless such software is used, at the end of a live CD session the computer remains as it was before. The live system is able to run without permanent installation by placing the files that normally would be stored on a hard drive into
      <a class="mw-redirect" href="/wiki/Random_access_memory" title="Random access memory">
       RAM
      </a>
      , typically in a
      <a class="mw-redirect" href="/wiki/RAM_disk" title="RAM disk">
       RAM disk
      </a>
      . The computer must have sufficient RAM both to store these files and maintain normal operation.
     </p>

     <h2>
      <span class="mw-headline" id="History">
       History
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=1" title="Edit section: History">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      All except the earliest digital computers are built with some form of minimal built-in loader, which loads a program or succession of programs from a storage medium, which then operate the computer. Initially a read-only medium such as punched tape or punched cards was used for initial program load. With the introduction of inexpensive read-write storage, read-write
      <a href="/wiki/Floppy_disk" title="Floppy disk">
       floppy disks
      </a>
      and hard disks were used as
      <a href="/wiki/Boot_disk" title="Boot disk">
       boot media
      </a>
      .
     </p>
     <p>
      After the introduction of the
      <a href="/wiki/Sound_recording_and_reproduction" title="Sound recording and reproduction">
       audio
      </a>
      <a href="/wiki/Compact_disc" title="Compact disc">
       compact disc
      </a>
      , it was adapted for use as a medium for storing and distributing large amounts of computer data. This data may also include application and operating-system software, sometimes packaged and archived in compressed formats. Later, it was seen to be convenient and useful to boot the computer directly from compact disc, often with a minimal working system to install a full system onto a hard drive. While there are read-write optical discs, either mass-produced read-only discs or write-once discs were used for this purpose.
     </p>
     <p>
      The first Compact Disc drives on personal computers were generally much too slow to run complex operating systems; computers were not designed to boot from an
      <a href="/wiki/Optical_disc" title="Optical disc">
       optical disc
      </a>
      . When operating systems came to be distributed on compact discs, either a
      <a class="mw-redirect" href="/wiki/Boot_floppy" title="Boot floppy">
       boot floppy
      </a>
      or the CD itself would boot specifically, and only, to install onto a hard drive. The first live CD was
      <a class="mw-redirect" href="/wiki/Towns_OS" title="Towns OS">
       FM Towns OS
      </a>
      first released in 1989.
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (November 2009)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
     </p>
     <h3>
      <span class="mw-headline" id="Origin_of_Linux_live">
       Origin of Linux live
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=2" title="Edit section: Origin of Linux live">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      Although early developers and users of distributions built on top of the
      <a href="/wiki/Linux_kernel" title="Linux kernel">
       Linux kernel
      </a>
      so it could take advantage of cheap optical disks and rapidly declining prices of CD drives for personal computers, the
      <a href="/wiki/Linux_distribution" title="Linux distribution">
       Linux distribution
      </a>
      CDs or "distros" were generally treated as a collection of installation packages that must first be permanently installed to hard disks on the target machine.
     </p>
     <p>
      However, in the case of these distributions built on top of the Linux kernel, the
      <a href="/wiki/Free_software" title="Free software">
       free
      </a>
      operating system was meeting resistance in the consumer market because of the perceived difficulty, effort, and risk involved in installing an additional partition on the hard disk, in parallel with an existing operating system installation.
     </p>
     <p>
      The term "live CD" was coined because, after typical PC RAM was large enough and 52x speed CD drives and CD burners were widespread among PC owners, it finally became convenient and practical to boot the kernel and run
      <a class="mw-redirect" href="/wiki/X11" title="X11">
       X11
      </a>
      , a window manager and GUI applications directly from a CD without disturbing the OS on the hard disk.
     </p>
     <p>
      This was a new and different situation for Linux than other operating systems, because the updates/upgrades were being released so quickly, different distributions and versions were being offered online, and especially because users were burning their own CDs.
     </p>
     <p>
      The first
      <a href="/wiki/Linux" title="Linux">
       Linux
      </a>
      -based 'Live CD' was
      <a class="mw-redirect" href="/wiki/Yggdrasil_Linux" title="Yggdrasil Linux">
       Yggdrasil Linux
      </a>
      first released in beta form 1992~1993 (ceased production in 1995), though in practice its functionality was hampered due to the low throughput of contemporary CD-ROM drives.
      <a href="/wiki/DemoLinux" title="DemoLinux">
       DemoLinux
      </a>
      , released in 1998, was the first Linux distribution specially designed as a live CD. The
      <a href="/wiki/Linuxcare" title="Linuxcare">
       Linuxcare
      </a>
      <a href="/wiki/Bootable_business_card" title="Bootable business card">
       bootable business card
      </a>
      , first released in 1999, was the first Live CD to focus on system administration, and the first to be distributed in the bootable business card form factor. As of 2010
      <sup class="plainlinks noprint asof-tag update" style="display:none;">
       <a class="external text" href="//en.wikipedia.org/w/index.php?title=Live_CD&amp;action=edit">
        [update]
       </a>
      </sup>
      ,
      <a href="/wiki/Finnix" title="Finnix">
       Finnix
      </a>
      (first released in 2000) is the oldest Live CD still in production.
      <a href="/wiki/Knoppix" title="Knoppix">
       Knoppix
      </a>
      , a
      <a href="/wiki/Debian" title="Debian">
       Debian
      </a>
      -derived Linux distribution, was released in 2003, and found popularity as both a
      <a href="/wiki/Data_recovery" title="Data recovery">
       rescue disk
      </a>
      system and as a primary distribution in its own right.
     </p>
     <p>
      Since 2003, the popularity of live CDs has increased substantially, partly due to
      <a class="external text" href="http://www.linux-live.org/" rel="nofollow">
       Linux Live scripts
      </a>
      and
      <a href="/wiki/Remastersys" title="Remastersys">
       remastersys
      </a>
      , which made it very easy to build customized live systems. Most of the popular
      <a href="/wiki/Linux_distribution" title="Linux distribution">
       Linux distributions
      </a>
      now include a live CD variant, which in some cases is also the preferred installation medium.
     </p>
     <h2>
      <span class="mw-headline" id="Uses">
       Uses
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=3" title="Edit section: Uses">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Live CDs made for many different uses. Some are designed to demonstrate or "test drive" a particular operating system (usually Linux or another
      <a href="/wiki/Free_software" title="Free software">
       free
      </a>
      or
      <a href="/wiki/Open_source" title="Open source">
       open source
      </a>
      operating system). Software can be tested, or run for a particular single use, without interfering with system setup. Data on a system which is not functioning normally due to operating system and software issues can be made available; for example, data can be recovered from a machine with an active virus infection without the virus process being active and causing more damage, and the virus can be removed with its defences against removal bypassed.
     </p>
     <p>
      Although some live CDs can load into memory to free the optical drive for other uses, loading the data from a CD-ROM is still slower than a typical hard drive boot, so this is rarely the default with large live CD images, but for smaller live CD images loading the filesystem directly into RAM can provide a significant performance boost, as RAM is much faster than a hard drive, and uses less power.
      <sup class="reference" id="cite_ref-Brickner000_1-0">
       <a href="#cite_note-Brickner000-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
      Experienced users of the operating system may also use a live CD to determine whether and to what extent a particular operating system or version is compatible with a particular hardware configuration and certain peripherals, or as a way to know beforehand which computer or peripheral will work before buying.
      <sup class="reference" id="cite_ref-Brickner000_1-1">
       <a href="#cite_note-Brickner000-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
      A live CD can be used to troubleshoot hardware, especially when a hard drive fails, and more generally as a
      <a href="/wiki/Recovery_disc" title="Recovery disc">
       recovery disc
      </a>
      in case of problems. Some live CDs can save user-created files in a Windows partition, a USB drive, a network drive, or other accessible media. Live backup CDs can create an image of drives, and back up files, without problems due to open files and inconsistent sets.
     </p>
     <p>
      A few additional uses include:
     </p>
     <ul>
      <li>
       installing a
       <a href="/wiki/Linux_distribution" title="Linux distribution">
        Linux distribution
       </a>
       to a hard drive
      </li>
      <li>
       testing new versions of software
      </li>
      <li>
       listing &amp; testing hardware
       <sup class="reference" id="cite_ref-2">
        <a href="#cite_note-2">
         <span>
          [
         </span>
         2
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       system repair and
       <a href="/wiki/Data_recovery" title="Data recovery">
        restoration
       </a>
      </li>
      <li>
       high security/non-invasive environment for a guest
      </li>
      <li>
       cracking, stealing, and changing passwords
      </li>
      <li>
       network security testing
      </li>
      <li>
       being the primary or backup operating system for any computer
      </li>
      <li>
       quick and simple clustering of computers
       <sup class="reference" id="cite_ref-3">
        <a href="#cite_note-3">
         <span>
          [
         </span>
         3
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       <a href="/wiki/Computer_forensics" title="Computer forensics">
        computer forensics
       </a>
      </li>
      <li>
       playing video games or running applications that require a different operating system
      </li>
      <li>
       providing a secure server platform where crucial files cannot be permanently altered
      </li>
      <li>
       providing a secure, reliable platform for the performance of high-vulnerability tasks such as
       <a class="mw-redirect" href="/wiki/Internet_banking" title="Internet banking">
        internet banking
       </a>
       ;
      </li>
      <li>
       <a class="mw-redirect" href="/wiki/Internet_kiosk" title="Internet kiosk">
        Internet kiosks
       </a>
       and public computers, which can be brought back to their original state by a reboot
      </li>
      <li>
       Bypassing
       <a href="/wiki/Parental_controls" title="Parental controls">
        parental controls
       </a>
       and other restrictions
      </li>
     </ul>
     <h3>
      <span class="mw-headline" id="Thematic_Live_CDs">
       Thematic Live CDs
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=4" title="Edit section: Thematic Live CDs">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      Several live CDs are dedicated to specific type of applications according to the requirements of thematic user communities. These CDs are tailored to the needs of the applications in subject including general knowledge, tutorial, specifications and trial data too.
     </p>
     <p>
      Some of these topics covers sub topics, e.g. IT administration breaks down to firewall, rescue, security, etc. type of live CDs. In some cases a particular LiveDVD covers more than one topic.
     </p>
     <h3>
      <span class="mw-headline" id="Live_CD_software_appliances">
       Live CD software appliances
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=5" title="Edit section: Live CD software appliances">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      Packaging a
      <a href="/wiki/Software_appliance" title="Software appliance">
       software appliance
      </a>
      as an installable live CD, or live ISO, can often be beneficial as a single image can run on both real hardware and on most types of virtual machines.
     </p>
     <p>
      This allows developers to avoid the complexities involved in supporting multiple incompatible
      <a href="/wiki/Virtual_machine" title="Virtual machine">
       virtual machine
      </a>
      images formats and focus on the lowest common denominator instead.
     </p>
     <p>
      Typically after booting the machine from the live CD, the appliance either runs in non-persistent demo mode or installs itself, at the user's request, to an available storage device.
     </p>
     <h3>
      <span class="mw-headline" id="Mounting_without_burning">
       Mounting without burning
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=6" title="Edit section: Mounting without burning">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      The files on a live CD
      <a href="/wiki/ISO_image" title="ISO image">
       ISO image
      </a>
      can be accessed in
      <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">
       Microsoft Windows
      </a>
      with a
      <a class="mw-redirect" href="/wiki/Disk_image_emulator" title="Disk image emulator">
       disk image emulator
      </a>
      such as
      <a href="/wiki/Daemon_Tools" title="Daemon Tools">
       Daemon Tools
      </a>
      , or in
      <a href="/wiki/Unix" title="Unix">
       Unix
      </a>
      variants by mounting a
      <a href="/wiki/Loop_device" title="Loop device">
       loop device
      </a>
      . Later versions of Windows (i.e. Windows 8 and later), and software available for earlier versions, allow an ISO to be mounted as a volume.
     </p>
     <p>
      After mounting the live CD's filesystem, software on the live CD can be run directly, without booting it, by
      <a href="/wiki/Chroot" title="Chroot">
       chrooting
      </a>
      into the mounted filesystem.
     </p>
     <p>
      A live CD ISO image can also be mounted by
      <a class="mw-redirect" href="/wiki/Virtual_Machine" title="Virtual Machine">
       Virtual Machine
      </a>
      software such as
      <a href="/wiki/VirtualBox" title="VirtualBox">
       VirtualBox
      </a>
      and
      <a href="/wiki/VMware_Workstation" title="VMware Workstation">
       VMware Workstation
      </a>
      or can be converted to a
      <a href="/wiki/Live_USB" title="Live USB">
       Live USB
      </a>
      using
      <a href="/wiki/SYSLINUX" title="SYSLINUX">
       SYSLINUX
      </a>
      . Tools such as
      <a href="/wiki/UNetbootin" title="UNetbootin">
       UNetbootin
      </a>
      can automate this process.
      <sup class="reference" id="cite_ref-4">
       <a href="#cite_note-4">
        <span>
         [
        </span>
        4
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <h2>
      <span class="mw-headline" id="Common_traits">
       Common traits
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=7" title="Edit section: Common traits">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Some live CDs come with an installation utility launchable from a desktop icon that can optionally install the system on a
      <a class="mw-redirect" href="/wiki/Hard_drive" title="Hard drive">
       hard drive
      </a>
      or
      <a href="/wiki/USB_flash_drive" title="USB flash drive">
       USB flash drive
      </a>
      . Most live CDs can access the information on internal and/or external hard drives,
      <a class="mw-redirect" href="/wiki/Diskette" title="Diskette">
       diskettes
      </a>
      and USB flash drives.
     </p>
     <p>
      Live CDs are usually distributed on
      <a href="/wiki/Read-only_memory" title="Read-only memory">
       read-only
      </a>
      media, requiring either copying to rewriteable media (i.e. a hard drive or CD writer) or complete
      <a class="mw-redirect" href="/wiki/Remaster_(software)" title="Remaster (software)">
       remastering
      </a>
      to install additional software; however, there are exceptions such as
      <a href="/wiki/Puppy_Linux" title="Puppy Linux">
       Puppy Linux
      </a>
      which has the ability to save files to the live CD itself or other
      <a class="mw-redirect" href="/wiki/Multisession" title="Multisession">
       multisession
      </a>
      media, allowing data, programs and customized settings to be written.
      <sup class="noprint Inline-Template" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify">
         <span title="This OR paragraph confuses read-only ''media'' with an image designed or not to ''allow'' configuration. Puppy Linux could be run from read-only CD, or another live CD could be run from rewritable CD, which would work only if designed to write appropriately (October 2013)">
          clarification needed
         </span>
        </a>
       </i>
       ]
      </sup>
     </p>
     <p>
      The first live CDs used Linux as their operating system
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (August 2014)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
      , available without charge or restrictive licence. The term came to be used for any CD containing operating system and software which could be run without installation on the host computer; example include
      <a href="/wiki/OpenSolaris" title="OpenSolaris">
       OpenSolaris
      </a>
      ,
      <a href="/wiki/BeleniX" title="BeleniX">
       BeleniX
      </a>
      and others based on
      <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">
       Solaris
      </a>
      . Other operating systems which can be used live include
      <a href="/wiki/AmigaOS_4" title="AmigaOS 4">
       AmigaOS 4
      </a>
      ,
      <a class="new" href="/w/index.php?title=Amithlon&amp;action=edit&amp;redlink=1" title="Amithlon (page does not exist)">
       Amithlon
      </a>
      ,
      <a class="mw-redirect" href="/wiki/AROS" title="AROS">
       AROS
      </a>
      ,
      <a href="/wiki/FreeBSD" title="FreeBSD">
       FreeBSD
      </a>
      ,
      <a href="/wiki/FreeDOS" title="FreeDOS">
       FreeDOS
      </a>
      ,
      <a href="/wiki/Mac_OS" title="Mac OS">
       Mac OS
      </a>
      ,
      <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">
       Microsoft Windows
      </a>
      installation and repair discs,
      <a href="/wiki/OS/2" title="OS/2">
       OS/2
      </a>
      ,
      <a href="/wiki/ReactOS" title="ReactOS">
       ReactOS
      </a>
      ,
      <a href="/wiki/NetBSD" title="NetBSD">
       NetBSD
      </a>
      ,
      <a href="/wiki/OpenBSD" title="OpenBSD">
       OpenBSD
      </a>
      ,
      <a href="/wiki/MINIX_3" title="MINIX 3">
       MINIX 3
      </a>
      ,
      <a href="/wiki/Plan_9_from_Bell_Labs" title="Plan 9 from Bell Labs">
       Plan 9 from Bell Labs
      </a>
      , and
      <a href="/wiki/MorphOS" title="MorphOS">
       MorphOS
      </a>
      . There are maintenance versions of Microsoft Windows bootable from CD such as
      <a href="/wiki/BartPE" title="BartPE">
       BartPE
      </a>
      ,
      <a href="/wiki/Windows_Preinstallation_Environment" title="Windows Preinstallation Environment">
       Windows PE
      </a>
      , and
      <a class="mw-redirect" href="/wiki/Microsoft_Diagnostics_and_Recovery_Toolset" title="Microsoft Diagnostics and Recovery Toolset">
       Microsoft Diagnostics and Recovery Toolset
      </a>
      (DaRT), previously known as Emergency Repair Disk Commander (ERD Commander).
     </p>
     <p>
      The first personal computer operating system on a CD to support "live" operations might have been the
      <a href="/wiki/AmigaOS" title="AmigaOS">
       AmigaOS
      </a>
      , which could be booted from CD on an
      <a class="mw-redirect" href="/wiki/Amiga_CDTV" title="Amiga CDTV">
       Amiga CDTV
      </a>
      in 1990.
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (November 2007)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
      . Earlier examples of live OS are of course the operating systems used from floppy, and most widely spread is
      <a href="/wiki/DOS" title="DOS">
       DOS
      </a>
      .
     </p>
     <p>
      Unlike previous operating systems on optical media, though, Linux and OS/2 "live CDs" were specifically designed to run without installation onto other media like a hard disk drive. The live CD concept was meant to promote Linux and showcase the abilities of the free,
      <a href="/wiki/Open_source" title="Open source">
       open source
      </a>
      operating system on conventional personal computers with Microsoft Windows already installed.
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (November 2008)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
     </p>
     <p>
      On a
      <a href="/wiki/Personal_computer" title="Personal computer">
       PC
      </a>
      , a bootable Compact Disc generally conforms to the
      <a href="/wiki/El_Torito_(CD-ROM_standard)" title="El Torito (CD-ROM standard)">
       El Torito
      </a>
      specification. Many Linux based live CDs use a compressed filesystem image, often with the
      <a href="/wiki/Cloop" title="Cloop">
       cloop
      </a>
      compressed loopback driver, or
      <a class="mw-redirect" href="/wiki/Squashfs" title="Squashfs">
       squashfs
      </a>
      compressed filesystem, generally doubling effective storage capacity, although slowing application start up
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (November 2009)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
      .
     </p>
     <p>
      The resulting environment can be quite rich: typical
      <a href="/wiki/Knoppix" title="Knoppix">
       Knoppix
      </a>
      systems include around 1,200 separate software packages. Live CDs have a reputation for supporting advanced
      <a href="/wiki/Auto-configuration" title="Auto-configuration">
       auto-configuration
      </a>
      and
      <a class="mw-redirect" href="/wiki/Plug-and-play" title="Plug-and-play">
       plug-and-play
      </a>
      functionality. This came out of necessity to avoid requiring the user to configure the system each time it boots and to make it easily usable by those who are new to the operating system.
     </p>
     <h2>
      <span class="mw-headline" id="Technique">
       Technique
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=8" title="Edit section: Technique">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      A read-only
      <a href="/wiki/File_system" title="File system">
       file system
      </a>
      , such as on a CD-ROM has the drawback of being unable to save any current working data. For this reason, a read-only file system is often merged with a temporary writable file system in the form of a
      <a class="mw-redirect" href="/wiki/RAM_disk" title="RAM disk">
       RAM disk
      </a>
      . Often the default Linux
      <a class="mw-redirect" href="/wiki/File_directory" title="File directory">
       directories
      </a>
      "
      <code>
       /home
      </code>
      " (containing users' personal files and
      <a href="/wiki/Configuration_file" title="Configuration file">
       configuration files
      </a>
      ) and "
      <code>
       /var
      </code>
      " (containing variable
      <a href="/wiki/Data_(computing)" title="Data (computing)">
       data
      </a>
      ) are kept in ramdisk, because the system updates them frequently. Puppy Linux and some other live CDs allow a configuration and added files to be written and used in later sessions.
     </p>
     <p>
      In modern live CDs, a read-only file system is merged with a
      <a href="/wiki/RAM_drive" title="RAM drive">
       RAM drive
      </a>
      using
      <a class="mw-redirect" href="/wiki/Transparency_(human-computer_interaction)" title="Transparency (human-computer interaction)">
       transparent
      </a>
      techniques such as
      <a href="/wiki/UnionFS" title="UnionFS">
       UnionFS
      </a>
      ,
      <a class="mw-redirect" href="/wiki/AuFS" title="AuFS">
       AuFS
      </a>
      or
      <a href="/wiki/Enhanced_Write_Filter" title="Enhanced Write Filter">
       EWF
      </a>
      . Boot loaders like syslinux can boot ISO files from USB memory devices.
     </p>
     <p>
      Live CDs have to be able to detect and use a wide variety of hardware (including
      <a class="mw-redirect" href="/wiki/Network_card" title="Network card">
       network cards
      </a>
      ,
      <a class="mw-redirect" href="/wiki/Graphic_card" title="Graphic card">
       graphic cards
      </a>
      etc.) in realtime, often using facilities such as
      <a href="/wiki/Udev" title="Udev">
       udev
      </a>
      ,
      <a class="new" href="/w/index.php?title=Hotplug_(software)&amp;action=edit&amp;redlink=1" title="Hotplug (software) (page does not exist)">
       hotplug
      </a>
      , hal, udisk etc.. which are a common part of all distributions based on
      <a href="/wiki/Linux_kernel" title="Linux kernel">
       Linux kernel
      </a>
      2.6.
     </p>
     <h3>
      <span class="mw-headline" id="Boot_code">
       Boot code
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Live_CD&amp;action=edit&amp;section=9" title="Edit section: Boot code">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      During live CD initialization, a user typically may resort to using one or more boot codes to change the booting behavior. These vary from distribution to distribution but can most often be accessed upon first boot screen by one of the
      <a href="/wiki/Function_key" title="Function key">
       function keys
      </a>
      .
     </p>
