               <div class="field-item even">
                <p>
                 Javascript Tools provides a common set of methods extending those available in Drupal core (drupal.js) for Javascript and AJAX module development in Drupal.
                </p>
                <p>
                 <em>
                  Note: you should not install this module for Drupal 6 unless it is required by another module.
                 </em>
                </p>
                <p>
                 <em>
                  Note: As of Oct. 20, 2009, the 5.x version is no longer supported.
                 </em>
                </p>
                <p>
                 Prior to Drupal 6, Javascript Tools included an integrated set of Javascript and AJAX modules. For Drupal 6, modules previously part of Javascript Tools now have the following separate projects:
                </p>
                <p>
                 Active menus,
                 <a href="http://drupal.org/project/activemenu">
                  http://drupal.org/project/activemenu
                 </a>
                 (requires Javascript Tools 6)
                 <br/>
                 Collapsiblock,
                 <a href="http://drupal.org/project/collapsiblock">
                  http://drupal.org/project/collapsiblock
                 </a>
                 (requires Javascript Tools 6)
                 <br/>
                 Tabs:
                 <a href="http://drupal.org/project/tabs">
                  http://drupal.org/project/tabs
                 </a>
                 <br/>
                 Active edit,
                 <a href="http://drupal.org/project/activeedit">
                  http://drupal.org/project/activeedit
                 </a>
                 (prototype only as yet, not usable)
                 <br/>
                 Ajax submit,
                 <a href="http://drupal.org/project/ajaxsubmit">
                  http://drupal.org/project/ajaxsubmit
                 </a>
                </p>
                <p>
                 Install Javascript Tools 6 only if you use Active menus or Collapsiblock or another module that requires it.
                </p>
                <!--break-->
                <p>
                 There are no current plans to update the following to Drupal 6:
                </p>
                <p>
                 Active search: the approach is awkward and hard to maintain and there are now other AJAX searching approaches.
                 <br/>
                 Dynamic load: See the
                 <a href="http://drupal.org/project/asynchronous" rel="nofollow">
                  Asynchronous
                 </a>
                 module for Drupal 6.
                 <br/>
                 Jscalendar: The JS library it integrates is obsolete and unmaintained; there are jQuery alternatives available. Look at the jQuery popup calendar that comes with the Date Popups submodule of the
                 <a href="//drupal.org/project/date" rel="nofollow">
                  Date
                 </a>
                 package.
                </p>
                <h3>
                 Currently available in the 5.x stable release
                </h3>
                <ul>
                 <li>
                  <strong>
                   Activeedit
                  </strong>
                  <br/>
                  Provides three types of AJAX editing: in place (click on a button to make e.g. text editable in place); links (click on a "add new comment" link to bring up an in-place form for adding a comment); and autocomplete (when there are no matches found, load a form in place to add a new record). All methods are implemented through hooks so that other modules can readily implement them.
                 </li>
                 <li>
                  <strong>
                   Activemenu
                  </strong>
                  <br/>
                  Activemenus makes the standard Drupal navigation  menu expandable. Clickable, highlighted regions are added to the menu. When a collapsed menu item is expanded, the child menu items are loaded dynamically through AJAX.
                 </li>
                 <li>
                  <strong>
                   Active Search
                  </strong>
                  <br/>
                  Adds AJAX to core Drupal search. Known bug in Safari not yet resolved.
                 </li>
                 <li>
                  <strong>
                   Ajaxsubmit
                  </strong>
                  <br/>
                  Enables form submitting through AJAX.
                 </li>
                 <li>
                  <strong>
                   Collapsiblock
                  </strong>
                  <br/>
                  Makes all Drupal blocks collapsible.
                 </li>
                 <li>
                  <strong>
                   Tabs
                  </strong>
                  <br/>
                  The tabs module provides developer tools to produce client-side tabbed pages using the using the jQuery tabs plugin by Klaus Hartl,
                  <a href="http://stilbuero.de/index.php?s=tabs&amp;x=0&amp;y=0" rel="nofollow">
                   stilbuero.de/tabs
                  </a>
                  . The Tabs module is co-maintained by
                  <a href="/user/99777" rel="nofollow" title="View user profile.">
                   Wim Leers
                  </a>
                  .
                 </li>
                 <li>
                  <strong>
                   Jscalendar
                  </strong>
                  <br/>
                  Popup calendar library using the
                  <a href="http://www.dynarch.com/projects/calendar/" rel="nofollow">
                   Jscalendar
                  </a>
                  library. To use, simply add a 'jscalendar' class to a textfield. As the Jscalendar library is no longer maintained, this module is deprecated past Drupal 5. There are jQuery-based alternatives.
                 </li>
                </ul>
                <!--break-->
                <p>
                 5.0 compatible, but not stable, and therefore left out of the stable release:
                </p>
                <ul>
                 <li>
                  <strong>
                   Dynamicload
                  </strong>
                  <br/>
                  The Dynamicload module provides methods for dynamically loading selected page content without page refreshes. Includes dynamic loading of blocks.
                 </li>
                </ul>
                <p>
                 These modules were in the 4.7 version but were not updated:
                </p>
                <ul>
                 <li>
                  <strong>
                   Columns
                  </strong>
                  <br/>
                  The
                  <em>
                   experimental
                  </em>
                  Columns module formats content into columns according to the CCS3 spec, using a
                  <a href="http://www.csscripting.com" rel="nofollow">
                   CSS3 Multi Column library
                  </a>
                  by Cdric Savarese. To use, enable the module and enclose node content in div elements with a class of "two-col", "three-col" or "four-col" for two, three, or four column layout.
                 </li>
                 <li>
                  <strong>
                   Formcheck
                  </strong>
                  <br/>
                  Based on code by
                  <a href="/user/12932" rel="nofollow">
                   m3avrck
                  </a>
                  (see
                  <a href="/node/30220" rel="nofollow">
                   this issue
                  </a>
                  ), Formcheck warns users navigating away from forms about unsaved data. Admins can select which forms to attach the behaviour to.
                 </li>
                 <li>
                  <strong>
                   TOC
                  </strong>
                  <br/>
                  The TOC module generates a clickable table of contents for nodes, based on all h2 tags in the node body.
                 </li>
                 <li>
                  <strong>
                   wForms
                  </strong>
                  <br/>
                  Cdric Savarese's
                  <a href="http://www.formassembly.com/wForms/" rel="nofollow">
                   wForms
                  </a>
                  is "an open-source, unobtrusive javascript library that adds commonly needed behaviors to traditional web forms without the need for any programming skill." The wforms module can be used for client-side multi-page forms, client-side form validation, and more.
                 </li>
                </ul>
               </div>
              </div>
