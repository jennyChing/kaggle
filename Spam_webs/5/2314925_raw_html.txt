       <h3>
        Wesnoth 1.12.4, 1.13.1, and Security Advisory
       </h3>
       <p class="date">
        Monday, June 29 2015
       </p>
       <p>
        <b>
         Wesnoth 1.12.4
        </b>
         a maintenance release for the stable 1.12.x series  and
        <b>
         Wesnoth 1.13.1
        </b>
         the second 1.13.x development release  are now available. Both include various fixes and improvements made since the previous releases, as well as a fix for an important security vulnerability which allows a malicious user to steal add-on upload credentials.
        <b style="color:#F00">
         We urge content authors using
         <u>
          any previous version
         </u>
         to upgrade immediately.
        </b>
        <br/>
        Check the respective forum threads for these releases for a list of the most notable changes in both versions:
        <br/>
        <a href="http://forums.wesnoth.org/viewtopic.php?t=42775">
         Wesnoth 1.12.4
        </a>
         maintenance release
        <br/>
        <a href="http://forums.wesnoth.org/viewtopic.php?t=42774">
         Wesnoth 1.13.1
        </a>
         development release
        <br/>
        See also our
        <a href="http://forums.wesnoth.org/viewtopic.php?t=42776">
         security advisory
        </a>
        for previous versions.
        <br/>
        The source code and the Windows installer files for both versions are already available on the
        <a href="http://wiki.wesnoth.org/Download#Stable_.281.12_branch.29">
         downloads page
        </a>
        . You may also find packages for other platforms there as they become available.
       </p>
      </div>
      <div class="news">
       <h3>
        About Add-on Passphrase Security
       </h3>
       <p class="date">
        Thursday, June 11 2015
       </p>
       <p>
        Content creators who have published user-made add-ons to the Wesnoth add-ons server are surely aware that we currently use a very primitive authentication mechanism that works on a per-add-on basis. An uploader-defined passphrase is provided in the add-ons
        <a href="http://wiki.wesnoth.org/PblWML">
         <code>
          .pbl
         </code>
         file
        </a>
        and this is matched against the add-ons servers records.
        <br/>
        <br/>
        What is not necessarily obvious is that the passphrase is stored in clear text form not only on the clients side, but also on the server. This means that any person with access to the server configuration can see every add-ons passphrase in a human-readable format that makes it trivial for it to be stolen. Furthermore, it is also possible for add-ons to obtain add-on passphrases from the client and transmit them over the network. Because of this, we advise content uploaders to use unique passphrases for their content and
        <strong>
         never reuse an existing password
        </strong>
        that could grant a malicious party access to their systems or other sites. Also, in order to prevent vandalism, we suggest either using hard-to-guess passphrases, or leaving the passphrase field blank or omitting it altogether when first uploading an add-on so that the add-ons client will generate and save a random one instead.
        <br/>
        <br/>
        People who suspect they may be using insecure passphrases for their add-ons should
        <a href="http://forums.wesnoth.org/ucp.php?i=pm&amp;mode=compose&amp;g=6641">
         send a private message
        </a>
        to the Forum Administrators group to request changing passphrases; or use the command-line add-ons client with the following parameters if possible, substituting the text within brackets and replacing
        <code>
         1.12.x
        </code>
        with
        <code>
         1.13.x
        </code>
        or
        <code>
         1.10.x
        </code>
        if applicable:
        <br/>
        <br/>
        <code>
         wesnoth_addon_manager -p 1.12.x --change-passphrase &lt;Addon_Folder_Name&gt; &lt;old passphrase&gt; &lt;new passphrase&gt;
        </code>
       </p>
