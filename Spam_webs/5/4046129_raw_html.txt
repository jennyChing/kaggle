       <div class="entry-content">
        <div itemscope="" itemtype="http://schema.org/BlogPosting">
         <p>
          A new year is just beginning and you have to be more cautious in analyzing a website than that of any other time from this 2015. Because Google is continuously improving its
          <a href="http://devsteam.com/all-google-algorithm-updates-of-2013-updated/" target="_blank">
           <strong>
            search algorithms
           </strong>
          </a>
          to produce to best possible results in 1
          <sup>
           st
          </sup>
          page of SERP. So the effectiveness of tradition old school link building is waning.
         </p>
         <p>
          <a href="http://i0.wp.com/devsteam.com/wp-content/uploads/2015/01/technical-seo-audit.jpg">
           <img alt="technical seo audit" class=" wp-image-2547 aligncenter" data-recalc-dims="1" src="http://i1.wp.com/devsteam.com/wp-content/uploads/2015/01/technical-seo-audit.jpg?resize=597%2C300"/>
          </a>
         </p>
         <p>
          So in such circumstances, in order to ensure your online business success, whether you are working on a blog or e-commerce website, having clear and 100% concept of the overall condition of your website is a must. For your convenient, today I am going to discuss about some critical issues that you must carefully consider in order to perform a comprehensive technical site audit for your website.
         </p>
         <h2>
          <strong>
           Check Indexed Pages:
          </strong>
         </h2>
         <p>
          Your site audit should start with searching the domain name in
          <a href="http://devsteam.com/google-ranking-algorithm/" target="_blank">
           <strong>
            Google
           </strong>
          </a>
          . Do a Site:Domain.Com search to check how many pages are returned in search results. Also check whether the home page is showed in 1
          <sup>
           st
          </sup>
          position of SERP. If the home page isnt showing up at the first position, then there might be issues like poor internal linking/ site structure affecting the site.
         </p>
         <h2>
          <strong>
           Robots.txt:
          </strong>
         </h2>
         <p>
          Another crucial issue of site audit is to be sure whether a website is perfectly accessible to search engines. Any mistake here and the search engines wont able to crawl the site which eventually leads to getting no ranking position in organic search results. There is a file stored on the web server named
          <strong>
           robots.txt
          </strong>
          with which you can tell search engine bots which contents to crawl and which not. If anyone accidentally blocks a crucial page or directory of your website, your website will dramatically loose traffic and ranking on from search results. However, any problem with this robots.txt file is quite easy to identify and fix. To find out whether any crucial page has been block or not, you can use a good quality Robots.txt Checker Tool.
         </p>
         <h2>
          <strong>
           HTTP Status Codes:
          </strong>
         </h2>
         <p>
          When a search engine crawler requests a page to web server, the web server returns with a HTTP status code along with the response. Your web server should always returns with a HTTP 200 code, which means everything is OK. But when a web server returns with a HTTP status code of 4xx or 5xx, this often refers to an error for which search engines cant access to crawl a web page.
         </p>
         <p>
          Its always great to use a tool like
          <a href="http://www.screamingfrog.co.uk/seo-spider/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://www.screamingfrog.co.uk/seo-spider/', 'Screaming Frog']);" target="_blank">
           <strong>
            Screaming Frog
           </strong>
          </a>
          or Xenu Link Sleuth to check such HTTP errors. And in case you have the GWT access, you can check such crawling error in the easiest way. Just login to
          <a href="https://www.google.com/webmasters/tools/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://www.google.com/webmasters/tools/', 'Google Webmaster Tools']);" target="_blank">
           <strong>
            Google Webmaster Tools
           </strong>
          </a>
          and then go to the Crawl &gt; Crawl Errors to check the sites health. It may look like this:
          <a href="http://i2.wp.com/devsteam.com/wp-content/uploads/2015/01/error-code.png">
           <img alt="Error message in Google Webmaster Tools" class=" size-full wp-image-2546 aligncenter" data-recalc-dims="1" src="http://i1.wp.com/devsteam.com/wp-content/uploads/2015/01/error-code.png?resize=451%2C141"/>
          </a>
         </p>
         <h2>
          <strong>
           Canonicalization:
          </strong>
         </h2>
         <p>
          Canonicalization occurs when either the WWW or non-WWW version of domain doesnt redirect to the other one. This is perhaps one of the most common problems that most of the websites often suffer from. Because of this canonicalization problem, most websites are actually splitting their domain authority between their www and non-www version of domains. If a website has this canonicalization problem and doesnt fix this, search engines will crawl and index both copies of your site which ultimately make your website content duplicate. You can easily be sure about whether your website has this problem by searching in Google for:
         </p>
         <blockquote>
          <p>
           <em>
            Site:DomaiName.com www
           </em>
          </p>
         </blockquote>
         <p>
          If the result doesnt match with any document, than its fine. Otherwise, you need to modify your htaccess file from cPanel to redirect one version of domain to another.
         </p>
         <h2>
          <strong>
           XML Sitemap:
          </strong>
         </h2>
         <p>
          Having a XML sitemap within website can ensure better crawling for webpages informing search engines about any updates within that website. So here is couple of considerations that you should check while analyzing your websites sitemap:
         </p>
         <ul>
          <li>
           Is the sitemap a well-formed XML sitemap?
          </li>
          <li>
           Is your sitemap following proper XML protocol?
          </li>
          <li>
           Is your XML sitemap structured to show indexation problem?
          </li>
          <li>
           Has the sitemap been submitted to Google Webmaster Tools?
          </li>
          <li>
           Are there any pages in the site crawl, which do not appear in sitemap?
          </li>
          <li>
           Are there any pages in sitemap which arent showing in site crawl?
          </li>
         </ul>
         <h2>
          <strong>
           Site Loading Time:
          </strong>
         </h2>
         <p>
          A website with longer loading time always faces higher traffic drop out and eventually increased bounce rate and lost conversion as well. Similarly, search engines bots allocate very limited time to visit each website on the internet. Hence a quickly loading website is crawled more consistently and thoroughly than a slower one.
         </p>
         <p>
          <a href="http://i0.wp.com/devsteam.com/wp-content/uploads/2015/01/100TB-Page-Speed.png">
           <img alt="Site Loading Time" class=" size-full wp-image-2548 aligncenter" data-recalc-dims="1" src="http://i1.wp.com/devsteam.com/wp-content/uploads/2015/01/100TB-Page-Speed.png?resize=680%2C300"/>
          </a>
         </p>
         <p>
          In most case, a home page should have weight around 130KB and other inner pages may have weight of 180KB. There are plenty of online tools with which you can evaluate your site loading time like
          <a href="http://tools.pingdom.com/fpt/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://tools.pingdom.com/fpt/', 'Pingdom']);" target="_blank">
           <strong>
            Pingdom
           </strong>
          </a>
          or
          <a href="https://developers.google.com/speed/pagespeed/insights/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://developers.google.com/speed/pagespeed/insights/', 'Google Page Speed tool']);" target="_blank">
           <strong>
            Google Page Speed tool
           </strong>
          </a>
          . These tools really work great in finding issues that are serving as bottlenecks for your website.
         </p>
         <h2>
          <strong>
           Broken Links:
          </strong>
         </h2>
         <p>
          As you know Google and all other search engines crawl web links to link, having too many broken links can result in serious SEO problem. When Google face too many broken links on a website, it may assume that the site has a poor UX, which often result in decreased crawl rate as well as both indexing and ranking problems. Unfortunately broken links can also happen for other webmasters linking to your website incorrectly. So having broken links is very common for a website and you can solve all your broken links with 301 redirection. You can identify broken links of your website using tools like Google Webmaster Tools or
          <a href="http://home.snafu.de/tilman/xenulink.html" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://home.snafu.de/tilman/xenulink.html', 'Xenu Link Sleuth']);" target="_blank">
           <strong>
            Xenu Link Sleuth
           </strong>
          </a>
          tool and then fix all those broken links with a 301(permanent) redirect.
         </p>
         <p>
          Conducting a comprehensive technical SEO audit can be exhausting depending on the complexity and size of a website. But once you are done with this technical audit, you will surely able to improve your sites overall visibility and ranking in SERP.
         </p>
