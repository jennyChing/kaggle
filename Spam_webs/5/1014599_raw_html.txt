     <p>
      An
      <b>
       Internet protocol camera
      </b>
      , or IP camera, is a type of digital video camera commonly employed for
      <a href="/wiki/Surveillance" title="Surveillance">
       surveillance
      </a>
      , and which, unlike analog
      <a class="mw-redirect" href="/wiki/Closed_circuit_television" title="Closed circuit television">
       closed circuit television
      </a>
      (CCTV) cameras, can send and receive data via a computer network and the Internet. Although most cameras that do this are
      <a href="/wiki/Webcam" title="Webcam">
       webcams
      </a>
      , the term "IP camera" or "netcam" is usually applied only to those used for surveillance. The first centralized IP camera was Axis Neteye 200, released in 1996 by Axis Communications.
     </p>
     <p>
      There are two kinds of IP cameras:
     </p>
     <ul>
      <li>
       Centralized IP cameras, which require a central
       <a href="/wiki/Network_Video_Recorder" title="Network Video Recorder">
        Network Video Recorder
       </a>
       (NVR) to handle the recording, video and alarm management.
      </li>
      <li>
       Decentralized IP cameras, which do not require a central
       <a href="/wiki/Network_Video_Recorder" title="Network Video Recorder">
        Network Video Recorder
       </a>
       (NVR), as the cameras have recording function built-in and can thus record directly to any standard storage media, such as SD cards, NAS (network attached storage) or a PC/Server.
      </li>
     </ul>

     <h2>
      <span class="mw-headline" id="History">
       History
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=IP_camera&amp;action=edit&amp;section=1" title="Edit section: History">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      The first centralized IP camera was released in 1996 by
      <a href="/wiki/Axis_Communications" title="Axis Communications">
       Axis Communications
      </a>
      .
      <sup class="reference" id="cite_ref-1">
       <a href="#cite_note-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
      It was called the Axis Neteye 200 and was developed by the team of Martin Gren and Carl-Axel Alm. It used a custom
      <a href="/wiki/Web_server" title="Web server">
       web server
      </a>
      internal to the camera. In late 1999, the company started using embedded
      <a href="/wiki/Linux" title="Linux">
       Linux
      </a>
      to operate its cameras. Axis also released documentation for its low-level
      <a href="/wiki/Application_programming_interface" title="Application programming interface">
       API
      </a>
      called "VAPIX", which builds on the open standards of
      <a class="mw-redirect" href="/wiki/HTTP" title="HTTP">
       HTTP
      </a>
      and
      <a class="mw-redirect" href="/wiki/Real_time_streaming_protocol" title="Real time streaming protocol">
       real time streaming protocol
      </a>
      (RTSP). This open architecture was intended to encourage third-party software manufacturers to develop compatible management and recording software.
     </p>
     <p>
      The first decentralized IP camera was released in 1999 by Mobotix.
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (February 2013)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
      The camera's
      <a href="/wiki/Linux" title="Linux">
       Linux
      </a>
      system contained video, alarm, and recording management function, thus the camera system did not require licensed video management software to manage the recording event, or video management.
     </p>
     <p>
      The first IP camera with onboard video content analytics (
      <a class="mw-redirect" href="/wiki/Video_Content_Analysis" title="Video Content Analysis">
       VCA
      </a>
      ) was released in 2005 by
      <a class="new" href="/w/index.php?title=Intellio&amp;action=edit&amp;redlink=1" title="Intellio (page does not exist)">
       Intellio
      </a>
      .
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (February 2013)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
      This camera was able to detect a number of different events, such as if an object was stolen, a human crossed a line, a human entered a predefined zone, or if a car moved in the wrong direction.
     </p>
     <p>
      IP cameras are available at resolutions from 0.3 (
      <a class="mw-redirect" href="/wiki/VGA" title="VGA">
       VGA
      </a>
      resolution) to 29
      <a href="/wiki/Pixel#Megapixel" title="Pixel">
       megapixels
      </a>
      .
      <sup class="reference" id="cite_ref-2">
       <a href="#cite_note-2">
        <span>
         [
        </span>
        2
        <span>
         ]
        </span>
       </a>
      </sup>
      As in the consumer TV business, in the early 21st century, there has been a shift towards
      <a href="/wiki/High-definition_video" title="High-definition video">
       high-definition video
      </a>
      resolutions, e.g.
      <a href="/wiki/720p" title="720p">
       720p
      </a>
      or
      <a href="/wiki/1080i" title="1080i">
       1080i
      </a>
      and
      <a href="/wiki/16:9" title="16:9">
       16:9
      </a>
      widescreen format.
     </p>

     <h2>
      <span class="mw-headline" id="Standards">
       Standards
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=IP_camera&amp;action=edit&amp;section=2" title="Edit section: Standards">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Analog
      <a class="mw-redirect" href="/wiki/Closed_circuit_television" title="Closed circuit television">
       closed circuit television
      </a>
      uses established broadcast television formats (e.g.
      <a href="/wiki/Common_Intermediate_Format" title="Common Intermediate Format">
       Common Intermediate Format
      </a>
      (CIF),
      <a href="/wiki/NTSC" title="NTSC">
       NTSC
      </a>
      ,
      <a href="/wiki/PAL" title="PAL">
       PAL
      </a>
      , and
      <a href="/wiki/SECAM" title="SECAM">
       SECAM
      </a>
      ). Generally speaking, each make of IP camera will differ in its features and functions, video encoding (compression) schemes, available network protocols, and the API to be used by video management software.
     </p>
     <p>
      In order to address issues of standardization of IP video surveillance, two industry groups were formed in 2008: the Open Network Video Interface Forum (
      <a href="/wiki/ONVIF" title="ONVIF">
       ONVIF
      </a>
      ) and the Physical Security Interoperability Alliance (
      <a class="mw-disambig" href="/wiki/PSIA" title="PSIA">
       PSIA
      </a>
      ). While the PSIA was founded by 20 member companies including
      <a href="/wiki/Honeywell" title="Honeywell">
       Honeywell
      </a>
      ,
      <a href="/wiki/GE_Security" title="GE Security">
       GE Security
      </a>
      and
      <a class="mw-redirect" href="/wiki/Cisco" title="Cisco">
       Cisco
      </a>
      , and ONVIF was founded by
      <a href="/wiki/Axis_Communications" title="Axis Communications">
       Axis Communications
      </a>
      ,
      <a href="/wiki/Robert_Bosch_GmbH" title="Robert Bosch GmbH">
       Bosch
      </a>
      and
      <a href="/wiki/Sony" title="Sony">
       Sony
      </a>
      , each group now has numerous members. As of January 2009, each group had released version 1.0 of their specification.
     </p>
     <h2>
      <span class="mw-headline" id="Potential_advantages">
       Potential advantages
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=IP_camera&amp;action=edit&amp;section=3" title="Edit section: Potential advantages">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <ul>
      <li>
       Two-way audio via a single network cable allows users to communicate with what they are seeing (e.g.
       <a class="mw-redirect" href="/wiki/Gas_station" title="Gas station">
        gas station
       </a>
       clerk assisting a customer on how to use the pay pumps)
      </li>
      <li>
       Flexibility: IP cameras can be moved around anywhere on a network (wireless).
      </li>
      <li>
       <a href="/wiki/Distributed_artificial_intelligence" title="Distributed artificial intelligence">
        Distributed intelligence
       </a>
       : with IP cameras, video analytics can be placed in the camera itself allowing ability in analytics solutions.
       <sup class="reference" id="cite_ref-3">
        <a href="#cite_note-3">
         <span>
          [
         </span>
         3
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       Transmission of commands for PTZ (
       <a href="/wiki/Panning_(camera)" title="Panning (camera)">
        pan
       </a>
       ,
       <a href="/wiki/Tilt_(camera)" title="Tilt (camera)">
        tilt
       </a>
       ,
       <a href="/wiki/Zoom_lens" title="Zoom lens">
        zoom
       </a>
       ) cameras via a single network.
      </li>
      <li>
       Encryption &amp; authentication: IP cameras offer secure data transmission through encryption and authentication methods such as
       <a href="/wiki/Wi-Fi_Protected_Access" title="Wi-Fi Protected Access">
        WPA
       </a>
       ,
       <a class="mw-redirect" href="/wiki/WPA2" title="WPA2">
        WPA2
       </a>
       ,
       <a href="/wiki/Temporal_Key_Integrity_Protocol" title="Temporal Key Integrity Protocol">
        TKIP
       </a>
       ,
       <a href="/wiki/Advanced_Encryption_Standard" title="Advanced Encryption Standard">
        AES
       </a>
       .
      </li>
      <li>
       Remote accessibility:
       <sup class="reference" id="cite_ref-4">
        <a href="#cite_note-4">
         <span>
          [
         </span>
         4
         <span>
          ]
         </span>
        </a>
       </sup>
       live video from selected cameras can be viewed from any computer, anywhere, and also from many mobile
       <a href="/wiki/Smartphone" title="Smartphone">
        smartphones
       </a>
       and other devices.
      </li>
      <li>
       IP cameras are able to function on a
       <a href="/wiki/Wireless_network" title="Wireless network">
        wireless network
       </a>
       .
       <sup class="reference" id="cite_ref-5">
        <a href="#cite_note-5">
         <span>
          [
         </span>
         5
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       PoE -
       <a href="/wiki/Power_over_Ethernet" title="Power over Ethernet">
        Power over Ethernet
       </a>
       : Modern IP cameras have the ability to operate without a power supply. They can work with the PoE-protocol.
      </li>
      <li>
       IP camera communication signals are not just electronic voltage, it is numerically decoded as bits and bytes with security features and TCP/IP protocol.
      </li>
     </ul>
     <h2>
      <span class="mw-headline" id="Potential_disadvantages">
       Potential disadvantages
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=IP_camera&amp;action=edit&amp;section=4" title="Edit section: Potential disadvantages">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <ul>
      <li>
       Higher initial cost per camera,
       <sup class="reference" id="cite_ref-6">
        <a href="#cite_note-6">
         <span>
          [
         </span>
         6
         <span>
          ]
         </span>
        </a>
       </sup>
       <sup class="reference" id="cite_ref-7">
        <a href="#cite_note-7">
         <span>
          [
         </span>
         7
         <span>
          ]
         </span>
        </a>
       </sup>
       except where cheaper
       <a href="/wiki/Webcam" title="Webcam">
        webcams
       </a>
       are used.
       <sup class="reference" id="cite_ref-8">
        <a href="#cite_note-8">
         <span>
          [
         </span>
         8
         <span>
          ]
         </span>
        </a>
       </sup>
       <sup class="reference" id="cite_ref-9">
        <a href="#cite_note-9">
         <span>
          [
         </span>
         9
         <span>
          ]
         </span>
        </a>
       </sup>
       <sup class="reference" id="cite_ref-10">
        <a href="#cite_note-10">
         <span>
          [
         </span>
         10
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       High network bandwidth requirements: a typical CCTV camera with resolution of 640x480 pixels and 10 frames per second (10 frame/s) in MPEG mode requires about 3 Mbit/s.
       <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
        [
        <i>
         <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
          <span title="This claim needs references to reliable sources. (February 2013)">
           citation needed
          </span>
         </a>
        </i>
        ]
       </sup>
      </li>
      <li>
       As with a CCTV/DVR system, if the video is transmitted over the public Internet rather than a private IP LAN, the system becomes open to a wider audience of hackers and hoaxers. Criminals can hack into a CCTV system to
       <a href="/wiki/Countersurveillance" title="Countersurveillance">
        observe security measures
       </a>
       and personnel, thereby facilitating criminal acts and rendering the surveillance counterproductive.
      </li>
      <li>
       public internet connection video is rather complicated to set up and requires a dynamic DNS. Recently, some producers of IP cameras (D-Link, Axis) have made their own software available with built-in dynamic DNS.
       <sup class="reference" id="cite_ref-11">
        <a href="#cite_note-11">
         <span>
          [
         </span>
         11
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
     </ul>
