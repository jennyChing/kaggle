      <a name="smoke">

      <h1>
       Smoke (DRAFT)
      </h1>
      <h2>
       Project Leads: Michael Lentine and Andrew Selle
      </h2>
      <table>
       <td align="center" valign="top">
        <p>
         <a href="movies/smoke.avi">
          <img src="images/smoke-small.png" width="200"/>
         </a>
        </p>
       </td>
       <td valign="top">
        <p>
         The goal of this project is create a simple smoke solver.
		This project leverages the
         <a href="http://physbam.stanford.edu/">
          PhysBAM
         </a>
         code base. This simple project
		depends on the PhysBAM code for Advection and Projection. This project also relies heavily on parallelization code include
		both MPI and threading and therefore the necessary libraries need to be linked to this project for compiliation. These libraries
		can be found on the
         <a href="http://physbam.stanford.edu/">
          PhysBAM
         </a>
         website.
        </p>
        <p>
         This project runs a smoke solver with a constant velocity source at the bottom of the domain.
         <a href="http://physbam.stanford.edu/~mlentine/smoke.tar.gz">
          Download the source
         </a>
         . To install and run it you will first need to download
		the
         <a href="http://physbam.stanford.edu/">
          Tools library
         </a>
         . Then follow the instructions given in
		the tarball to install and run.
        </p>
        <p>
         Note that although parallelism is supported in this version of the release threading is still in the experimental stages
		and may not give the desired performance. New versions will be released with updated parallization in the future.
        </p>
        <p>
         For this simple smoke solver we have created two subprojects which breaks up the necessary parts of this simple solver into
		an advection and a projection project. All of these projects can be run independently and viewed with either the ray tracer
		or the opengl viewer found on the physbam website.
        </p>
       </td>
      </table>
     </div>

      <h1>
       Subprojects
      </h1>
      <h2>
       Advection
      </h2>
      <table>
       <td align="center" valign="top">
        <p>
         <a href="images/advection.png">
          <img src="images/advection.png" width="200"/>
         </a>
        </p>
       </td>
       <td valign="top">
        <p>
         A simple advection project which passively advects a given density field through a constant velocity field. This projects takes
		a velocity field V at position x specified by V(x)=1 and advects a passive density field D(x)=0.5*sin(4*pi*(x-0.25)-0.5*pi)+1 for x
		between 0.25 and 0.75 and 0 elsewhere. The output will be dumped into a directory named output.
        </p>
        <p>
         Dowload the source
         <a href="http://physbam.stanford.edu/~mlentine/advection.tar.gz">
          here

		simply follow the instructions to run this project.
        </p>

      <a name="water">
      </a>
      <h1>
       Water (DRAFT)
      </h1>
      <h2>
       Project Leads: Michael Lentine and Andrew Selle
      </h2>
      <table>
       <td align="center" valign="top">
        <p>
         <a href="movies/pls.avi">
          <img src="images/pls-small.png" width="200"/>
         </a>
        </p>
       </td>
       <td valign="top">
        <p>
         The goal of this project is create a generalized fluid solver that can be used for a wide variety of applications including special effects
		and video games.
		This project leverages the
         <a href="http://physbam.stanford.edu/">
          PhysBAM
         </a>
         code base. The base of the
		standard fluid solver uses the partilce levelset method. This project depends on the PhysBAM code for Incompressible Flow, Level Sets, and
		Particle Level Sets. This project also relies heavily on parallelization code include both MPI and threading and therefore the necessary
		libraries need to be linked to this project for compiliation.
        </p>
        <p>
         This project runs a levelset based water simulator with a constant velocity source at the left of the domain.
         <a href="http://physbam.stanford.edu/~mlentine/water.tar.gz">
          Download the source
         </a>
         . To install and run it you will first need to download
		the
         <a href="http://physbam.stanford.edu/">
          Geometry library
         </a>
         . Then follow the instructions given in
		the tarball to install and run.
        </p>
        <p>
         Note that although parallelism is supported in this version of the release threading is still in the experimental stages
		and may not give the desired performance. New versions will be released with updated parallization in the future.
        </p>
        <p>
         In addition to the standard Fluid solver there are a number of related projects discussed below that we will provide to achieve
	    faster and/or different types of fluid simulations. This project is intented to primarily simulation water and does not implement
		any solid/fluids coupling. For the purposes of this project all solids are one-way coupled to the water flow.
        </p>
       </td>
      </table>
     </div>
