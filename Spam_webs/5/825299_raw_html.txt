				&lt;div class="post-content"&gt;
					&lt;p&gt;Here are 20 of the best free tools that will help you conduct a digital forensic investigation. Whether its for an internal human resources case, an investigation into unauthorized access to a server, or if you just want to learn a new skill, these suites and utilities will help you conduct memory forensic analysis, hard drive forensic analysis, forensic image exploration, forensic imaging and mobile forensics. As such, they all provide the ability to bring back in-depth information about whats under the hood of a system.&lt;/p&gt;
&lt;p&gt;This is by no means an extensive list and may not cover everything you need for your investigation. You might also need additional utilities such a file viewers, hash generators, and text editors  checkout &lt;b&gt;&lt;a href="http://www.gfi.com/blog/101-free-admin-tools/"&gt;101 Free Admin Tools&lt;/a&gt;&lt;/b&gt; for some of these. My articles on &lt;b&gt;&lt;a href="http://www.gfi.com/blog/top-10-free-system-troubleshooting-tools-for-sysadmins/"&gt;Top 10 Free Troubleshooting Tools for SysAdmins&lt;/a&gt;&lt;/b&gt;, &lt;b&gt;&lt;a href="http://www.gfi.com/blog/the-top-20-free-network-monitoring-and-analysis-tools-for-sys-admins/"&gt;Top 20 Free Network Monitoring and Analysis Tools for Sys Admins&lt;/a&gt; &lt;/b&gt;and &lt;b&gt;&lt;a href="http://www.gfi.com/blog/the-top-20-free-file-management-tools-for-sys-admins/"&gt;Top 20 Free File Management Tools for Sys Admins&lt;/a&gt; &lt;/b&gt;might also come in handy since they contain a bunch of tools that can be used for Digital Forensic Investigations (e.g. BackTrack and the SysInternals Suite or the NirSoft Suite of tools).&lt;/p&gt;
&lt;p&gt;Even if you may have heard of some of these tools before, Im confident that youll find a gem or two amongst this list.&lt;b&gt;&lt;/b&gt;&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://computer-forensics.sans.org/community/downloads" target="_blank"&gt;01 SANS SIFT&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;The SANS Investigative Forensic Toolkit (SIFT) is an Ubuntu based Live CD which includes all the tools you need to conduct an in-depth forensic or incident response investigation. It supports analysis of Expert Witness Format (E01), Advanced Forensic Format (AFF), and RAW (dd) evidence formats. SIFT includes tools such as log2timeline for generating a timeline from system logs, Scalpel for data file carving, Rifiuti for examining the recycle bin, and lots more.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/01-SANS-SIFT.jpg"&gt;&lt;img class="aligncenter  wp-image-10822" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/01-SANS-SIFT.jpg" alt="01 SANS SIFT" width="402" height="221" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you first boot into the SIFT environment, I suggest you explore the documentation on the desktop to help you become accustomed to what tools are available and how to use them. There is also a good explanation of where to find evidence on a system. Use the top menu bar to open a tool, or launch it manually from a terminal window.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.techpathways.com/desktopdefault.aspx?tabindex=8&amp;tabid=14" target="_blank"&gt;02 ProDiscover Basic&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;ProDiscover Basic is a simple digital forensic investigation tool that allows you to image, analyse and report on evidence found on a drive. Once you add a forensic image you can view the data by content or by looking at the clusters that hold the data. You can also search for data using the Search node based on the criteria you specify.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/02-ProDiscover-Basic.jpg"&gt;&lt;img class="aligncenter  wp-image-10823" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/02-ProDiscover-Basic.jpg" alt="02 ProDiscover Basic" width="472" height="251" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch ProDiscover Basic you first need to create or load a project and add evidence from the Add node. You can then use the Content View or Cluster View nodes to analyse the data and the Tools menu to perform actions against the data. Click the Report node to view important information about the project.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="https://www.volatilesystems.com/default/volatility" target="_blank"&gt;03 Volatility&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;Volatility is a memory forensics framework for incident response and malware analysis that allows you to extract digital artefacts from volatile memory (RAM) dumps. Using Volatility you can extract information about running processes, open network sockets and network connections, DLLs loaded for each process, cached registry hives, process IDs, and more.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/03-Volatility.jpg"&gt;&lt;img class="aligncenter  wp-image-10824" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/03-Volatility.jpg" alt="03 Volatility" width="473" height="113" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;If you are using the standalone Windows executable version of Volatility, simply place volatility-2.1.standalone.exe into a folder and open a command prompt window. From the command prompt, navigate to the location of the executable file and type volatility-2.1.standalone.exe f &lt;FILENAME&gt; &#8211;profile=&lt;PROFILENAME&gt; &lt;PLUGINNAME&gt; without quotes  FILENAME would be the name of the memory dump file you wish to analyse, PROFILENAME would be the machine the memory dump was taken on and PLUGINNAME would be the name of the plugin you wish to use to extract information.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Note:&lt;/b&gt; In the example above I am using the connscan plugin to search the physical memory dump for TCP connection information.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.sleuthkit.org/" target="_blank"&gt;04 The Sleuth Kit (+Autopsy)&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;The Sleuth Kit is an open source digital forensics toolkit that can be used to perform in-depth analysis of various file systems. Autopsy is essentially a GUI that sits on top of The Sleuth Kit. It comes with features like Timeline Analysis, Hash Filtering, File System Analysis and Keyword Searching out of the box, with the ability to add other modules for extended functionality.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Note: &lt;/b&gt;You can use The Sleuth Kit if you are running a Linux box and Autopsy if you are running a Windows box.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/1.jpg"&gt;&lt;img class="aligncenter  wp-image-10827" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/1.jpg" alt="1" width="506" height="311" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch Autopsy, you can choose to create a new case or load an existing one. If you choose to create a new case you will need to load a forensic image or a local disk to start your analysis. Once the analysis process is complete, use the nodes on the left hand pane to choose which results to view.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.accessdata.com/support/product-downloads" target="_blank"&gt;05 FTK Imager&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;FTK Imager is a data preview and imaging tool that allows you to examine files and folders on local hard drives, network drives, CDs/DVDs, and review the content of forensic images or memory dumps. Using FTK Imager you can also create SHA1 or MD5 hashes of files, export files and folders from forensic images to disk, review and recover files that were deleted from the Recycle Bin (providing that their data blocks havent been overwritten), and mount a forensic image to view its contents in Windows Explorer.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Note:&lt;/b&gt; There is a portable version of FTK Imager that will allow you to run it from a USB disk.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/2.jpg"&gt;&lt;img class="aligncenter  wp-image-10829" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/2.jpg" alt="2" width="507" height="460" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch FTK Imager, go to File &gt; Add Evidence Item to load a piece of evidence for review. To create a forensic image, go to File &gt; Create Disk Image and choose which source you wish to forensically image.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;06 Linux dd&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;dd comes by default on the majority of Linux distributions available today (e.g. Ubuntu, Fedora). This tool can be used for various digital forensic tasks such as forensically wiping a drive (zero-ing out a drive) and creating a raw image of a drive.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Note:&lt;/b&gt; dd is a very powerful tool that can have devastating effects if not used with care. It is recommended that you experiment in a safe environment before using this tool in the real world.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Tip:&lt;/b&gt; A modified version of dd is available from &lt;a href="http://sourceforge.net/projects/dc3dd/" target="_blank"&gt;http://sourceforge.net/projects/dc3dd/&lt;/a&gt; &#8211; dc3dd includes additional features that were added specifically for digital forensic acquisition tasks.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/3.jpg"&gt;&lt;img class="aligncenter  wp-image-10831" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/3.jpg" alt="3" width="490" height="313" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;To use dd, simply open a terminal window and type dd followed by a set of command parameters (which command parameters will obviously depend on what you want to do). The basic dd syntax for forensically wiping a drive is:&lt;/p&gt;
&lt;p&gt;dd if=/dev/zero of=/dev/sdb1 bs=1024&lt;br /&gt;
&lt;b&gt;&lt;br /&gt;
&lt;/b&gt;where if = input file, of = output file, bs = byte size&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Note:&lt;/b&gt; Replace /dev/sdb1 with the drive name of the drive you want to forensically wipe and 1024 with the size of the byte blocks you want to write out.&lt;/p&gt;
&lt;p&gt;The basic dd syntax for creating a forensic image of a drive is:&lt;/p&gt;
&lt;p&gt;dd if=/dev/sdb1 of=/home/andrew/newimage.dd bs=512 conv=noerror,sync&lt;/p&gt;
&lt;p&gt;where if = input file (or in this case drive), of = output file, bs = byte size, conv = conversion options&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Tip:&lt;/b&gt; For additional usage info, from a terminal window, type man dd without quotes to bring up the help manual for the dd command.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.caine-live.net/page5/page5.html" target="_blank"&gt;07 CAINE&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;CAINE (Computer Aided INvestigative Environment) is Linux Live CD that contains a wealth of digital forensic tools. Features include a user-friendly GUI, semi-automated report creation and tools for Mobile Forensics, Network Forensics, Data Recovery and more.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/4.jpg"&gt;&lt;img class="aligncenter  wp-image-10832" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/4.jpg" alt="4" width="508" height="382" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you boot into the CAINE Linux environment, you can launch the digital forensic tools from the CAINE interface (shortcut on the desktop) or from each tools shortcut in the Forensic Tools folder on the applications menu bar.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.oxygen-forensic.com/en/download/freeware" target="_blank"&gt;08 Oxygen Forensic Suite 2013 Standard&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;If you are investigating a case that requires you to gather evidence from a mobile phone to support your case, Oxygen Forensics Suite (Standard Edition) is a tool that will help you achieve this. Features include the ability to gather Device Information (Manufacturer, OS Platform, IMEI, Serial Number, etc.), Contacts, Messages (Emails, SMS, MMS, etc.) and recovery of deleted messages, Call Logs, and Calendar and Task information. It also comes with a file browser which allows you to access and analyse user photos, videos, documents and device databases.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/5.jpg"&gt;&lt;img class="aligncenter  wp-image-10833" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/5.jpg" alt="5" width="507" height="354" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch Oxygen Forensic Suite, hit the Connect new device button on the top menu bar to launch the Oxygen Forensic Extractor wizard that guides you through selecting the device and type of information you wish to extract.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.hhdsoftware.com/free-hex-editor" target="_blank"&gt;09 Free Hex Editor Neo&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;Free Hex Editor Neo is a basic hex editor that was designed to handle very large files. While a lot of the additional features are found in the commercial versions of Hex Editor Neo, I find this tool useful for loading large files (e.g. database files or forensic images) and performing actions such as manual data carving, low-level file editing, information gathering, or searching for hidden data.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/9.jpg"&gt;&lt;img class="aligncenter  wp-image-10834" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/9.jpg" alt="9" width="508" height="393" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;Use File &gt; Open to load a file into Hex Editor Neo. The data will appear in the middle window where you can begin to navigate through the hex manually or press CTRL + F to run a search.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.forensicswiki.org/wiki/Bulk_extractor" target="_blank"&gt;10 Bulk Extractor&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;bulk_extractor is a computer forensics tool that scans a disk image, file, or directory of files and extracts information such as credit card numbers, domains, e-mail addresses, URLs, and ZIP files. The extracted information is output to a series of text files (which can be reviewed manually or analysed using other forensics tools or scripts).&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Tip:&lt;/b&gt; Within the output text files you will find entries for data that resemble a credit card number, e-mail address, domain name, etc. You will also see a decimal value in the first column of the text file that, when converted to hex, can be used as the pointer on disk where the entry was found (i.e. if you were analysing the disk manually using a hex editor for example, you would jump to this hexadecimal value to view the data).&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/10.jpg"&gt;&lt;img class="aligncenter  wp-image-10835" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/10.jpg" alt="10" width="508" height="401" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;Bulk_extractor comes as a command-line tool or a GUI tool. In the example above I set the bulk extractor tool to extract information from a forensics image I took earlier and output the results to a folder called BE_Output. The results can then be viewed in the Bulk Extractor Viewer and the output text files mentioned above.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.deftlinux.net/download/" target="_blank"&gt;11 DEFT&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;DEFT is another Linux Live CD which bundles some of the most popular free and open source computer forensic tools available. It aims to help with Incident Response, Cyber Intelligence and Computer Forensics scenarios. Amongst others, it contains tools for Mobile Forensics, Network Forensics, Data Recovery, and Hashing.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/11.jpg"&gt;&lt;img class="aligncenter  wp-image-10836" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/11.jpg" alt="11" width="508" height="384" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you boot using DEFT, you are asked whether you wish to load the live environment or install DEFT to disk. If you load the live environment you can use the shortcuts on the application menu bar to launch the required tools.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.xplico.org/download" target="_blank"&gt;12 Xplico&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;Xplico is an open source Network Forensic Analysis Tool (NFAT) that aims to extract applications data from internet traffic (e.g. Xplico can extract an e-mail message from POP, IMAP or SMTP traffic). Features include support for a multitude of protocols (e.g. HTTP, SIP, IMAP, TCP, UDP), TCP reassembly, and the ability to output data to a MySQL or SQLite database, amongst others.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/12.jpg"&gt;&lt;img class="aligncenter  wp-image-10837" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/12.jpg" alt="12" width="508" height="259" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;Once youve installed Xplico, access the web interface by navigating to http://&lt;IPADDRESS&gt;:9876 and logging in with a normal user account. The first thing you need to do is create a case and add a new session. When you create a new session you can either load a PCAP file (acquired from Wireshark for example) or start a live capture. Once the session has finished decoding, use the navigation menu on the left hand side to view the results.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.nirsoft.net/utils/computer_activity_view.html" target="_blank"&gt;13 LastActivityView&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;I briefly touched on LastActivityView when pointing out the NirSoft suite of tools in my &lt;a href="http://www.gfi.com/blog/top-10-free-system-troubleshooting-tools-for-sysadmins/" target="_blank"&gt;&lt;b&gt;Top 10 Free System Troubleshooting Tools for SysAdmins &lt;/b&gt;&lt;/a&gt;article. LastActivityView allows you to view what actions were taken by a user and what events occurred on the machine. Any activities such as running an executable file, opening a file/folder from Explorer, an application or system crash or a user performing a software installation will be logged. The information can be exported to a CSV / XML / HTML file. This tool is useful when you need to prove that a user (or account) performed an action he or she said they didnt.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/13.jpg"&gt;&lt;img class="aligncenter  wp-image-10838" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/13.jpg" alt="13" width="507" height="293" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch LastActivityView, it will immediately start displaying a list of actions taken on the machine it is being run on. Sort by action time or use the search button to start investigating what actions were taken on the machine.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.digital-forensic.org/" target="_blank"&gt;14 Digital Forensic Framework&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;The Digital Forensics Framework (DFF) is a digital forensic investigation tool and a development platform that allows you to collect, preserve and reveal digital evidence. Amongst others, DFF&#8217;s features include the ability to read RAW, EWF and AFF forensic file formats, access local and remote devices, analyse registry, mailbox and file system data and recover hidden and deleted files.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/14.jpg"&gt;&lt;img class="aligncenter  wp-image-10839" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/14.jpg" alt="14" width="507" height="282" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch DFF, you first need to load an evidence file (i.e. a forensic image you acquired previously) or open a device ready for analysis. You can then process the evidence file or device against one of the in-built modules to begin analysing data.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="https://www.mandiant.com/resources/download/redline" target="_blank"&gt;15 Mandiant RedLine&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;RedLine offers the ability to perform memory and file analysis of a specific host. It collects information about running processes and drivers from memory, and gathers file system metadata, registry data, event logs, network information, services, tasks, and Internet history to help build an overall threat assessment profile.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/15.jpg"&gt;&lt;img class="aligncenter  wp-image-10840" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/15.jpg" alt="15" width="508" height="312" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch RedLine, you will be given a choice to Collect Data or Analyze Data. Unless you already have a memory dump file available, youll need to create a collector to gather data from the machine and let that process run through to completion. Once you have a memory dump file to hand you can begin your analysis.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.plainsight.info/index.html" target="_blank"&gt;16 PlainSight&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;PlainSight is a Live CD based on Knoppix (a Linux distribution) that allows you to perform digital forensic tasks such as viewing internet histories, data carving, USB device usage information gathering, examining physical memory dumps, extracting password hashes, and more.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/16.jpg"&gt;&lt;img class="aligncenter  wp-image-10841" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/16.jpg" alt="16" width="509" height="383" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you boot into PlainSight, a window pops up asking you to select whether you want to perform a scan, load a file or run the wizard. Enter a selection to begin the data extraction and analysis process.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://mh-nexus.de/en/hxd/" target="_blank"&gt;17 HxD&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;HxD is one of my personal favourites. It is a user-friendly hex editor that allows you to perform low-level editing and modifying of a raw disk or main memory (RAM). HxD was designed with easy-of-use and performance in mind and can handle large files without issue. Features include searching and replacing, exporting, checksums/digests, an in-built file shredder, concatenation or splitting of files, generation of statistics and more.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/17.jpg"&gt;&lt;img class="aligncenter  wp-image-10842" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/17.jpg" alt="17" width="473" height="451" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;From the HxD interface start your analysis by opening a file from File &gt; Open, loading a disk from Extras &gt; Open disk or loading a RAM process from Extras &gt; Open RAM&#8230;.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://www.e-fense.com/products.php" target="_blank"&gt;18 HELIX3 Free&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;HELIX3 is a Live CD based on Linux that was built to be used in Incident Response, Computer Forensics and E-Discovery scenarios. It is packed with a bunch of open source tools ranging from hex editors to data carving software to password cracking utilities, and more.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Note:&lt;/b&gt; The HELIX3 version you need is 2009R1. This version was the last free version available before HELIX was taken over by a commercial vendor. HELIX3 2009R1 is still valid today and makes for a useful addition to your digital forensics toolkit.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/18.jpg"&gt;&lt;img class="aligncenter  wp-image-10843" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/18.jpg" alt="18" width="475" height="359" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you boot using HELIX3, you are asked whether you want to load the GUI environment or install HELIX3 to disk. If you choose to load the GUI environment directly (recommended), a Linux-based screen will appear giving you the option to run the graphical version of the bundled tools.&lt;/p&gt;
&lt;h2&gt;&lt;b&gt;&lt;a href="http://netgrab.co.uk/netsleuth/download-netsleuth/" target="_blank"&gt;19 NetSleuth&lt;/a&gt;&lt;/b&gt;&lt;/h2&gt;
&lt;p&gt;NetSleuth is a network forensics analysis tool that identifies devices on your network. It operates in &#8216;live&#8217; mode (where it will actively capture network packets and interpret device information) or in &#8216;offline&#8217; mode where it will process a PCAP file that you import.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Note: &lt;/b&gt;At the time of writing, NetSleuth is in BETA. It is not recommended that you run this in a production environment. It made this list because it promises to be a handy addition to your forensic toolkit. The author of this tool is currently asking for feedback from the community so now is your chance to contribute!&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/19.jpg"&gt;&lt;img class="aligncenter  wp-image-10844" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/19.jpg" alt="19" width="475" height="295" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch NetSleuth, you can either initiate a live analysis from the Live Capture tab, or load a PCAP file from the Offline Analysis tab. Once NetSleuth has identified at least one device, you can double click on it to open the Device Information window.&lt;/p&gt;
&lt;h2&gt;&lt;strong&gt;&lt;b&gt;&lt;a href="http://www.paraben.com/p2-explorer.html" target="_blank"&gt;20 P2 eXplorer Free&lt;/a&gt;&lt;/b&gt;&lt;/strong&gt;&lt;/h2&gt;
&lt;p&gt;P2 eXplorer is a forensic image mounting tool that allows you to mount a forensic image as a physical disk and view the contents of that image in Windows Explorer or load it into an external forensic analysis tool. P2 eXplorer supports images in RAW, DD, IMG, EX01, SMART and SafeBack format, amongst others.&lt;/p&gt;
&lt;p style="text-align: center;"&gt;&lt;a href="http://www.gfi.com/blog/wp-content/uploads/2013/09/20.jpg"&gt;&lt;img class="aligncenter  wp-image-10845" src="http://www.gfi.com/blog/wp-content/uploads/2013/09/20.jpg" alt="20" width="475" height="295" /&gt;&lt;/a&gt;&lt;/p&gt;
&lt;p&gt;When you launch P2 eXplorer, choose an available drive letter to mount the image to and click File &gt; Mount Image to choose the image to mount. Once the image has been mounted, double click on the associated drive letter to view the contents of that image in Windows Explorer.&lt;/p&gt;
&lt;p&gt;&lt;b&gt;Tip:&lt;/b&gt; In &lt;b&gt;&lt;a href="http://www.gfi.com/blog/top-20-free-disk-tools-for-sysadmins/"&gt;Top 20 Free Disk Tools for SysAdmins&lt;/a&gt;&lt;/b&gt; I mentioned another image mounting tool called OSFMount. OSFMount is very similar to P2 eXplorer but also supports the mounting of VMWare files and the creation of RAM disks. Part of the OSFMount family is a digital forensics suite called OSForensics  the freeware version of this application is available for personal, educational or home use to allow you to experiment and become acquainted with digital forensics concepts.&lt;/p&gt;

One thing to note is that most software-based commercial mobile forensics tools have evaluation editions available for you to try and these tend to be more feature-rich than the open source / freeware alternatives.&lt;/p&gt;
&lt;p&gt;Kalimuthu &#8211; Thanks. Glad you found it useful! To answer your question, it really depends how the user accessed these applications. It won&#8217;t show that the internet history was cleared or that the MRU list was deleted but it might show that the user logged on, and that regedit.exe and iexplore.exe were executed for example.&lt;/p&gt;
&lt;p&gt;Quoting from the LastActivityView website, &#8220;the following actions and events are currently supported by LastActivityView:&lt;/p&gt;
&lt;p&gt;Run .EXE file: .EXE file run directly by the user, or by another software/service running in the background.&lt;/p&gt;
&lt;p&gt;Select file in open/save dialog-box: The user selected the specified filename from the standard Save/Open dialog-box of Windows.&lt;/p&gt;
&lt;p&gt;Open file or folder: The user opened the specified filename from Windows Explorer or from another software.&lt;/p&gt;
&lt;p&gt;View Folder in Explorer: The user viewed the specified folder in Windows Explorer.&lt;/p&gt;
&lt;p&gt;Software Installation: The specified software has been installed or updated.&lt;/p&gt;
&lt;p&gt;System Started: The computer has been started.&lt;/p&gt;
&lt;p&gt;System Shutdown: The system has been shut down, directly by the user, or by a software that initiated a reboot.&lt;/p&gt;
&lt;p&gt;Resumed from sleep: The computer has been resumed from sleep mode.&lt;/p&gt;
&lt;p&gt;Network Connected: Network connected, after previously disconnected.&lt;/p&gt;
&lt;p&gt;Network Disconnected: Network has been disconnected&lt;/p&gt;
&lt;p&gt;Software Crash: The specified software has been crashed.&lt;/p&gt;
&lt;p&gt;Software stopped responding (hang): The specified software stopped responding.&lt;/p&gt;
&lt;p&gt;Blue Screen: Blue screen event has been occurred on the system.&lt;/p&gt;
&lt;p&gt;User Logon: The user logged on to the system.&lt;/p&gt;
&lt;p&gt;User Logoff: The user logged off from the system. This even might caused by a software that initiated a reboot.&lt;/p&gt;
&lt;p&gt;Restore Point Created: Restore point has been created by Windows operating system.&lt;/p&gt;
&lt;p&gt;Windows Installer Started&lt;/p&gt;
&lt;p&gt;Windows Installer Ended&#8221;&lt;/p&gt;

       &lt;p&gt;Jerri Corbett &#8211; Thanks for your comment. As I mentioned in a previous reply to Masoud Al Tawqi, unfortunately there arent many open source / freeware forensic tools specifically for mobile devices. One that comes to mind is iPhone Analyzer (available from: &lt;a href="http://sourceforge.net/projects/iphoneanalyzer/" rel="nofollow"&gt;http://sourceforge.net/projects/iphoneanalyzer/&lt;/a&gt;) or you could look at the list available in the DEFT Linux Live CD. One thing to note is that most software-based commercial mobile forensics tools have evaluation editions available for you to try and these tend to be more feature-rich than the open source / freeware alternatives.&lt;/p&gt;
&lt;p&gt;Dee Brown &#8211; Thanks for your feedback! I am not aware of any forensic software that specifically allows you to find concealed data in BIOS chips. If you wanted to analyze a BIOS chip for hidden data you could use a tool to dump the contents of the BIOS to disk/USB and then use a Hex Editor to view the data (some BIOS flash tools allow you to take a backup of the BIOS to RAM/disk before updating). It would be pretty difficult to determine what data is actually &#8216;hidden&#8217; but a good method of analysis would be to do a side-by-side comparison of a chip you think has hidden data with one of the same type that you know doesn&#8217;t have hidden data.&lt;/p&gt;
