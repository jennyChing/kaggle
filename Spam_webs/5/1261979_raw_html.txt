        <h1 class="entry-title">
         RDFa Markup for Videos
        </h1>
        <!--				&lt;div class="comments-link"&gt; 
					&lt;a href="http://www.algohunters.com/rdfa-markup-videos/#respond" title="Comment on RDFa Markup for Videos"&gt;&lt;span class="dsq-postid" data-dsqidentifier="233 http://www.algohunters.com/?p=233"&gt;&lt;span class="leave-reply"&gt;Leave a reply&lt;/span&gt;&lt;/span&gt;&lt;/a&gt;				&lt;/div&gt; -->

       <div class="entry-content">
        <div class="kindleWidget kindleLight">
         <img src="http://www.algohunters.com/wp-content/plugins/send-to-kindle/media/black-15.png"/>
         <span>
          Send to Kindle
         </span>
        </div>
        <div class="pf-content">
         <p>
          With the recent loss of keyword data in Google Analytics and the launch of the new Hummingbird search algorithm, SEOs are presented with some new challenges and opportunities. One of the opportunities which is more pronounced now than ever before is the use of structured data markup.
         </p>
         <p>
          One area that has been largely ignored by many is the ability
          <a href="http://www.seoworkers.com/seo-articles-tutorials/video-rdfa-rich-snippets.html" itemprop="significantLink" target="_blank">
           to mark up videos
          </a>
          , in order to convey as much information as possible to the search engines. With the growing popularity of videos, particularly via Google Hangouts and YouTube, this is an opportunity thats too good to miss.
          <span id="more-233">
          </span>
         </p>
         <h2>
          Why Markup a YouTube Video for Structured Data?
         </h2>
         <p>
          You probably already know what a standard YouTube URL looks like:
         </p>
         <p>
          <em>
           http://www.youtube.com/watch?v=86GHCVRReJs
          </em>
          .
         </p>
         <p>
          The last portion,
          <em>
           86GHCVRReJs
          </em>
          , is the video ID. But there can be variations to this URL.
         </p>
         <p>
          If you want to share that video as a link, for instance, YouTube provides a share code, like this:
          <em>
           http://youtu.be/86GHCVRReJs
          </em>
          . If you want to embed it on your website, they offer an embed code, like this:
         </p>
         <p>
          <code>
           &lt;iframe width="420" height="315" src="//www.youtube.com/embed/86GHCVRReJs" frameborder="0" allowfullscreen&gt;&lt;/iframe&gt;
          </code>
         </p>
         <p>
          The embed code identifies and locates the video, but it also sizes it, determines the border and allows for full screen viewing, all within an iframe. But iframes and semantic markup dont always play well together and theres really no need to use iframes they way we prefer to do it.
         </p>
         <p>
          There is a wealth of information that can be communicated to the search engines without cluttering up the users experience. For a few reasons, we combine our accessibility solutions with our markup, to keep things tidy.
         </p>
         <h2>
          How to Markup a Video with RDFa
         </h2>
         <p>
          The RDFa markup in the example (at the bottom of the page) addresses a number of different areas. It provides the ability to be viewed by all users, whether theyve enabled JavaScript or not, as well as bearing a nocookie attribute. It can also incorporate several other features, some of which dont apply to this particular example.
         </p>
         <p>
          By using
          <strong>
           RDFa (Yahoo! SearchMonkey)
          </strong>
          , youre assured of your markup being supported by Google, Bing and Yahoo! The same is true of
          <strong>
           schema.org
          </strong>
          , but RDFa is much more extensible. It also has the ability to enable markup for multiple videos on a page, unlike Share and OpenGraph.
         </p>
         <p>
          When using
          <strong>
           Yahoo! SearchMonkey RDFa
          </strong>
          , the following properties are supported:
         </p>
         <ul>
          <li>
           Property Description
          </li>
          <li>
           <strong>
            media:video
           </strong>
           (Required.) A URL to the video to play when the user clicks the play button.
          </li>
          <li>
           <strong>
            xmlns:media
           </strong>
           (Required.) Must consist of the following URL: http://search.yahoo.com/searchmonkey/media/.
          </li>
          <li>
           <strong>
            media:thumbnail
           </strong>
           (Required.) A URL pointing to a preview thumbnail, which must be a GIF, PNG, or JPG image. The preview thumbnail must be hosted on the same domain as the video.
          </li>
          <li>
           <strong>
            xmlns:dc
           </strong>
           A valid URL to the Dublin Core namespace. The only acceptable value is http://purl.org/dc/terms/. Required only if you are using Dublin Core metadata such as dc:description.
          </li>
          <li>
           <strong>
            dc:license
           </strong>
           Indicates the license for this content. You may use
           <strong>
            dc:license
           </strong>
           or
           <strong>
            cc:license
           </strong>
           .
          </li>
          <li>
           <strong>
            dc:description
           </strong>
           A short (up to 200 characters) description of the video.
          </li>
          <li>
           <strong>
            media:title
           </strong>
           Specifies the title of a video. Up to 60 characters.
          </li>
          <li>
           <strong>
            media:width
           </strong>
           The videos width in pixels, including any chrome provided by the third-party player.
          </li>
          <li>
           <strong>
            media:height
           </strong>
           The videos height in pixels, including any chrome provided by the third-party player.
          </li>
          <li>
           <strong>
            media:type
           </strong>
           The videos MIME-type. Currently, the only acceptable value is application/x-shockwave-flash.
          </li>
          <li>
           <strong>
            media:region
           </strong>
           An international region where the video may be played. The default is * (play in all regions).
          </li>
          <li>
           <strong>
            media:duration
           </strong>
           The duration of the video, in seconds.
          </li>
         </ul>
         <p>
          (Note that there are some properties that are required, and a couple that, at present, have only one acceptable value.)
         </p>
         <p>
          <strong>
           RDFa markup
          </strong>
          really isnt as complex as it may seem at first glance. As is the case with any coding effort, of course, spaces and operators can be critical to proper processing. Fortunately, there are on-line tools for checking your markup for configuration and function. Googles Rich Snippet Testing Tool can be helpful in ensuring your markup is correct.
         </p>
         <p>
          Rather than walking you through each line of code to explain what the markup does, Im going to suggest you read it. That way, youll learn to understand it and youll retain the process better. Every video you decide to markup with RDFa will contain the same elements, just perhaps with different values.
         </p>
         <p>
          Pick a video and mark it up, then put in online and run it through Googles testing tool. You can watch this 1-1/2 minute video first, for a broad understanding, and if you need more specific guidance, you can also visit this
          <a href="https://developers.google.com/webmasters/videosearch/markups">
           Google Webmaster Tools Help page
          </a>
          on the topic.
         </p>
         <p>
          If youd like still more information, visit the W3C website  theres a wealth of information there on semantic markup. Here are a few good starting points, for understanding the procedure:
         </p>
         <ul>
          <li>
           <a href="http://www.w3.org/2006/07/SWD/RDFa/primer/">
            /http://www.w3.org/2006/07/SWD/RDFa/primer/
           </a>
          </li>
          <li>
           <a href="http://www.w3.org/TR/2011/WD-rdfa-primer-20110419/">
            http://www.w3.org/TR/2011/WD-rdfa-primer-20110419/
           </a>
          </li>
          <li>
           <a href="http://www.w3.org/TR/xhtml-rdfa-primer/">
            http://www.w3.org/TR/xhtml-rdfa-primer/
           </a>
          </li>
         </ul>

         <h2>
          Example: Markup for the above video:
         </h2>
         <p>
          <code>
           &lt;div&gt;&lt;object type="application/x-shockwave-flash" width="560" height="315" data="http://www.youtube.com/v/86GHCVRReJs?version=3&amp;hl=en_US"
           <br/>
           xmlns:media="http://search.yahoo.com/searchmonkey/media/" xmlns:dc="http://purl.org/dc/terms/" rel="media:video" property="og:video" content="http://www.youtube-nocookie.com/v/86GHCVRReJs?version=3&amp;hl=en_US" resource="http://www.youtube-nocookie.com/v/86GHCVRReJs version=3&amp;hl=en_US"&gt;&lt;param name="movie" value="http://www.youtube.com/v/86GHCVRReJs?version=3&amp;hl=en_US"/&gt;
           <br/>
           &lt;param name="allowFullScreen" value="true"/&gt;
           <br/>
           &lt;param name="allowscriptaccess" value="always"/&gt;
           <br/>
           &lt;param name="movie" value="http://www.youtube-nocookie.com/v/86GHCVRReJs?version=3&amp;hl=en_US"/&gt;
           <br/>
           &lt;param name="wmode" value="transparent"/&gt;&lt;/object&gt;&lt;a rel="media:thumbnail" href="http://i1.ytimg.com/vi/86GHCVRReJs/default.jpg"&gt;&lt;/a&gt;
           <br/>
           &lt;a rel="dc:license" href="http://www.youtube.com/yt/copyright/creative-commons.html"&gt;&lt;/a&gt;
           <br/>
           &lt;span property="dc:description" content="Does it make sense to use rel="nofollow" for internal links? Like, for example, to link to your login page? Does it really make a difference?"&gt;&lt;/span&gt;
           <br/>
           &lt;span property="media:title" content="Should I use rel="nofollow" on internal links to a login page?"&gt;&lt;/span&gt;
           <br/>
           &lt;span property="media:width" content="420"&gt;&lt;/span&gt;
           <br/>
           &lt;span property="media:height" content="315"&gt;&lt;/span&gt;
           <br/>
           &lt;span property="media:duration" content="148"&gt;&lt;/span&gt;
           <br/>
           &lt;span property="media:type" content="application/x-shockwave-flash"&gt;&lt;/span&gt;
           <br/>
           &lt;span property="media:expirationDate" content="*"&gt;&lt;/span&gt;
           <br/>
           &lt;span property="media:region" content="*"&gt;&lt;/span&gt;

          </code>

          by Google.
         </p>
