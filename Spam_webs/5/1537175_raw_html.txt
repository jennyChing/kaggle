        Manual outlook configuration with Exchange 2013
       </a>
      </h2>
      <div class="post-meta">
       <span class="meta-prep meta-prep-author posted">
        Posted on
       </span>
       <a href="http://www.zerohoursleep.com/2013/02/manual-outlook-configuration-with-exchange-2013/" rel="bookmark" title="Manual outlook configuration with Exchange 2013">

       </a>

       </span>
       <span class="author vcard">

      </div>
      <!-- end of .post-meta -->
      <div class="post-entry">
       <h4>
        Introduction
       </h4>
       <p>
        When I first heard that Exchange 2013 has killed MAPI and that now even internal clients connect to exchange using RPC over HTTP (or outlook anywhere), the first question that popped to my mind was What should we enter in the server name then?
       </p>
       <p>
        Well it wasnt too long before I got the answer to that question, as you can see in the image below the servername is no longer the server FQDN, instead outlook uses a new connection point MailboxGUID @ UPNsuffix e.g.: 271d1d80-72c1-440f-b34c-f748dba61332@zerohoursleep.com this change has for goal to remove the infamous Your administrator has made a change to your mailbox. Please restart.
       </p>
       <p>
        <img alt="" src="https://lh6.googleusercontent.com/-Qpy-3KPNHSQ/UUGTRB1goZI/AAAAAAAAA8A/QtCDX71nPBI/s640/OutlookManualConfig00.PNG"/>
       </p>
       <p>
        Yes youve read it right, it is the MailboxGUID so it will change for every user! You wouldnt care a lot about this if you are using outlook auto-configuration, however if you want to configure Outlook manually keep on reading.
       </p>
       <h4>
        Getting the Server Name
       </h4>
       <p>
        The only way I found to retrieve this GUID is through the Exchange Management Shell by running
        <em>
         Get-mailbox username | ft name,ExchangeGUID
        </em>
        , the GUID in question is that big funny string.
        <br/>
        <a class="more-link" href="http://www.zerohoursleep.com/2013/02/manual-outlook-configuration-with-exchange-2013/#more-1640">
         Read more 
        </a>
       </p>
      </div>

        KB2756920 causes IIS to crash
       </a>
      </h2>
      <div class="post-meta">
       <span class="meta-prep meta-prep-author posted">
        Posted on
       </span>
       <a href="http://www.zerohoursleep.com/2013/01/kb2756920-causes-iis-to-crash/" rel="bookmark" title="KB2756920 causes IIS to crash">
        <time class="timestamp updated" datetime="2013-01-20T14:40:06+00:00">
         January 20, 2013
        </time>
       </a>
       <span class="byline">
        by
       </span>
       <span class="author vcard">
        <a class="url fn n" href="http://www.zerohoursleep.com/author/admin/" title="View all posts by Antoine Khater">
         Antoine Khater
        </a>
       </span>
       <span class="comments-link">
        <span class="mdash">
         
        </span>
        <a href="http://www.zerohoursleep.com/2013/01/kb2756920-causes-iis-to-crash/#respond" title="Comment on KB2756920 causes IIS to crash">
         No Comments 
        </a>
       </span>
      </div>
      <!-- end of .post-meta -->
      <div class="post-entry">
       <p>
        I have faced a recent issue on OWA of Exchange 2010 installed on windows 2008 R2 on various server.
       </p>
       <p>
        The issues caused OWA  Blackberry  Out of office and autodiscover to totally crash and the application log was filled with errors similar to the below
       </p>
       <blockquote>
        <p>
         EventID: 3
         <br/>
         Source: System.ServiceModel 3.0.0.0
        </p>
       </blockquote>
       <blockquote>
        <p>
         WebHost failed to process a request.
         <br/>
         Sender Information: System.ServiceModel.ServiceHostingEnvironment+HostingManager/30542218
         <br/>
         Exception: System.ServiceModel.ServiceActivationException: The service /EWS/exchange.asmx cannot be activated due to an exception during compilation.  The exception message is: Method not found: System.String System.ServiceModel.Activation.Iis7Helper.ExtendedProtectionDotlessSpnNotEnabledThrowHelper(System.Object).. &gt; System.MissingMethodException: Method not found: System.String System.ServiceModel.Activation.Iis7Helper.ExtendedProtectionDotlessSpnNotEnabledThrowHelper(System.Object).
         <br/>
         at System.ServiceModel.WasHosting.MetabaseSettingsIis7V2.WebConfigurationManagerWrapper.BuildExtendedProtectionPolicy(ExtendedProtectionTokenChecking tokenChecking, ExtendedProtectionFlags flags, List`1 spnList)
        </p>
       </blockquote>
       <p>
        After some troubleshooting removing
        <a href="http://support.microsoft.com/kb/2756920">
         KB2756920
        </a>
        solved the issue.
       </p>
       <p>
        Although I only faced the issue with Microsoft Exchange, this is rather an IIS issue and I guess it could affect any other web-based application.
       </p>
