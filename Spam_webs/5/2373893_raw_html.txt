     <h2>
      openSUSE Announces First Public Release of openQA
     </h2>
     <small>
      October 11th, 2011 by
      <a href="https://news.opensuse.org/author/jospoortvliet/" rel="author" title="Posts by Jos Poortvliet">
       Jos Poortvliet
      </a>
     </small>
     <div class="share-links">
      <div class="share-link">
       <a class="twitter-share-button" data-count="horizontal" data-hashtags="openSUSE" data-text="openSUSE News: openSUSE Announces First Public Release of openQA" data-url="https://news.opensuse.org/2011/10/11/opensuse-announces-first-public-release-of-openqa/" data-via="openSUSE" href="http://twitter.com/share">
        Tweet

     <div class="entry">
      <p>
       <a href="//news.opensuse.org/2011/10/11/opensuse-announces-first-public-release-of-openqa/factory-tested/" rel="attachment wp-att-11196">
        <img alt="" class="alignright size-full wp-image-11196" height="232" src="//news.opensuse.org/wp-content/uploads/2011/10/factory-tested.png" title="factory-tested" width="217"/>
       </a>
      </p>
      <p>
       The openSUSE Project announces the 1.0 release of the unique cross-distribution-capable, fully automated testing framework
       <strong>
        <a href="http://openqa.opensuse.org">
         openQA
        </a>
       </strong>
       . openQA is the only comprehensive testing tool which can run tests on every level of the OS, from core functionality like the bootloader and booting the kernel up to testing applications like Firefox and LibreOffice. It shows the results in a convenient web interface and allows testers to see screenshots and even
       <em>
        videos
       </em>
       of the issues found. openQA is used to run nightly tests of the Factory development repository for the upcoming
       <a href="http://en.opensuse.org/Portal:12.1">
        openSUSE 12.1 release
       </a>
       . openQA is available under the GPL version 2 or later.
       <br/>
       <span id="more-11194">
       </span>
      </p>
      <h2>
       Software Testing is Hard
      </h2>
      <p>
       Testing complex software is a crucial part of development. However, in-depth, frequent testing is difficult, time-consuming and boring. To ensure that software is tested regularly and predictably, automation is crucial. Many in-house tools which can test specific applications have been developed but there was no comprehensive testing tool for a full operating system until openQA. openQA does its work entirely hands-off and offers a convenient web interface with an overview of the results. With openQA developers can catch and fix issues in the development process much faster and thus ensure a more reliable product.
      </p>
      <div class="wp-caption alignright" id="attachment_11205" style="width: 160px">
       <a href="//news.opensuse.org/2011/10/11/opensuse-announces-first-public-release-of-openqa/openqa-test-overview/" rel="attachment wp-att-11205">
        <img alt="openQA test overview" class="size-thumbnail wp-image-11205" height="150" src="//news.opensuse.org/wp-content/uploads/2011/10/openQA-test-overview-150x150.png" title="openQA test overview" width="150"/>
       </a>
       <p class="wp-caption-text">
        openQA test overview
       </p>
      </div>
      <h2>
       openQA Makes it Easy
      </h2>
      <p>
       openQA can test any operating system which is able to run in a virtual machine. By taking screenshots of the process and comparing these to reference images it determines if tests are passed or not. The OS-autoinst tool, part of openQA, controls the testing process by sending virtual keyboard and mouse events to the operating system being installed and run. It is able to respond to what is shown on the screen so it can handle a variety of issues while not having to stop the test.
      </p>
      <p>
       openQA determines if the installation of the operating system worked correctly and whether a number of applications started properly. At the end of the test, all images are encoded into a video for consultation by a human tester in case this is needed. A status overview of the tests can be shown
       <a href="http://openqa.opensuse.org/results/">
        in a web interface
       </a>
       , currently available for the openSUSE Factory development repository on
       <a href="openqa.opensuse.org">
        openqa.opensuse.org
       </a>
       .
      </p>
      <div class="wp-caption alignright" id="attachment_11207" style="width: 160px">
       <a href="//news.opensuse.org/2011/10/11/opensuse-announces-first-public-release-of-openqa/openqa-screenshot/" rel="attachment wp-att-11207">
        <img alt="openQA screenshot" class="size-thumbnail wp-image-11207" height="150" src="//news.opensuse.org/wp-content/uploads/2011/10/openQA-screenshot-150x150.png" title="openQA screenshot" width="150"/>
       </a>
       <p class="wp-caption-text">
        openQA screenshot
       </p>
      </div>
      <p>
       Coolo, openSUSE release manager noted:
       <br/>
      </p>
      <blockquote>
       Im thrilled to see the 1.0 release of openQA. openQAs automated testing has been very beneficial to the stability of openSUSE Factory and Im confident it can do the same for other Linux distributions
      </blockquote>
      <h2>
       openQA is Cross-platform Capable
      </h2>
      <p>
       openQA is developed as two software pieces:
       <a href="http://www.os-autoinst.org/">
        OS-autoinst
       </a>
       and
       <a href="http://openqa.opensuse.org">
        the web interface
       </a>
       . OS-autoinst right now supports Fedora, Ubuntu, Debian, FreeBSD and even OpenIndiana. It can be used to test even MS DOS and Windows systems. A number of videos on how openQA works can be found at
       <a href="http://video.os-autoinst.zq1.de/opensuse/video/">
        this site
       </a>
      </p>
      <p>
       The
       <a href="http://openqa.opensuse.org">
        openQA web interface
       </a>
       is openSUSE specific right now. It gives an overview of the status of all tests and allows the tester to look at specific screenshots of issues as well as videos of the entire testing process.
      </p>
      <div class="wp-caption alignright" id="attachment_11206" style="width: 160px">
       <a href="//news.opensuse.org/2011/10/11/opensuse-announces-first-public-release-of-openqa/openqa-test-overview1/" rel="attachment wp-att-11206">
        <img alt="openQA test details" class="size-thumbnail wp-image-11206" height="150" src="//news.opensuse.org/wp-content/uploads/2011/10/openQA-test-overview1-150x150.png" title="openQA test details" width="150"/>
       </a>
       <p class="wp-caption-text">
        openQA test details
       </p>
      </div>
      <h2>
       openQA is Free
      </h2>
      <p>
       OpenQA has been published under the General Public License version 2 or later. Installation instructions can be found on the wiki and the team is eager to hear feedback on openQA and use cases.
      </p>
      <p>
       Check out openQA in action on
       <a href="http://openqa.opensuse.org">
        openqa.opensuse.org
       </a>
       !
      </p>

       Both comments and pings are currently closed.
