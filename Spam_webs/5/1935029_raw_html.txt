          <div class="post entry-content " itemprop="commentText">
           I have nextel and discovered what Sprint uses to verify if your number is a landline and not a cell phone.
           <br/>
           <br/>
           A few months ago when I ordered their service I gave them one of my VoIP numbers as my home contact and they said that wasn't acceptable because it was a cell phone.  And I said no it's not.  It's my home number.  I then told them that it was actually a VoIP number.  She said no I had a sprint PCS number.  I was quite confused why my voip number showed up sprint PCS.
           <br/>
           <br/>
           My VoIP DID uses an exchange where certain blocks are owned by other carriers
           <br/>
           For example.  909-853-1XXX could be Verizon GTE and 909-853-2XXX could be some CLEC in california, and 909-853-4XXX could be AT&amp;T Wireless.
           <br/>
           <br/>
           Well appearently my number was lets say 8XXX which was the CLEC that my VoIP carrier used.  And a cell phone carrier owned a block on 3XXX.   And they said my number even though it wasn't on a cell phone block, they still said it was a cell phone number.  I asked what they used to look up their numbers and....
           <br/>
           <br/>
           get ready.......
           <br/>
           all sprint uses to see if a number is a landline or not....is.......
           <br/>
           SEARCHBUG!!!!!
           <br/>
           <br/>
           <br/>
           So here's an example.
           <br/>
           The exchange 310-208.
           <br/>
           If you look up on telcodata
           <br/>
           <a class="bbc_url" href="http://www.telcodata.us/telcodata/telco?npa=310&amp;exchange=208" rel="nofollow external" title="External link">
            http://www.telcodata...10&amp;exchange=208
           </a>
           <br/>
           <br/>
           You'll see that 310-208 is a verizon landline but 310-208-9XXX is a NEXTEL block.
           <br/>
           <br/>
           So we know that 310-208-1234 would landline number and not a cell phone.
           <br/>
           <br/>
           Well if you put 310-208-1234 in search bug...what do we get....
           <br/>
           <a class="bbc_url" href="http://www.searchbug.net/peoplefinder/landline-or-cellphone.aspx?PhoneNumber=3102081234" rel="nofollow external" title="External link">
            http://www.searchbug...mber=3102081234
           </a>
           <br/>
           <br/>
           A cell phone!
           <br/>
           <br/>
           <br/>
           So that's Sprint/Nextel's special fancy tool to see if a number is a landline or cell phone...search bug.
           <br/>
           <br/>
           So what i'm going to recommend is when Sprint/Nextel starts allowing you to call "home" for unlimited time.  Get a voip number that doesn't share its exchange with any cell phone carriers and tell them thats your home number and you should be fine.   They also would probably want to call the number to test it so have it go directly to a phone you can pick up at first.
