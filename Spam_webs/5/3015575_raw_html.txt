       Installing SNORT on Ubuntu 10.04
      </a>
     </h3>
     <div class="postcontent">
      <!-- Quick Adsense WordPress Plugin: http://quicksense.net/ -->
      <div style="float:none;margin:10px 0 10px 0;text-align:center;">

      <p>
       My company recently removed a Cisco IPS device (pile of crap) along with a Microsoft 2006 ISA Server (more of a pile of crap) from the network infrastructure.  It cost too much to maintain licensing, and it was starting to become a huge headache for company growth.
      </p>
      <p>
       However, the director of IT requested that we still have some sort of logging for potential breaches.  Every firewall block in/out is sent to us via email, and every firewall action is kept in a syslog server (and backed up to another syslog server and NAS system), but it would be nice to see the packets and payloads of every network transaction.
      </p>
      <p>
       Snort to the rescue!  Snort is free (well, if you dont mind having to setup/maintain/administer it without a ton of help and really dont mind having slightly older rules than paying customers), and doesnt really require a lot of hardware to get it to work.  Our production Snort is running on an HP DL320 1U with a 3.2GHz Pentium 4, 2GB RAM, and a mirror of 80GB SATA.  For testing I used VMWare Workstation 7, but Im sure you can use VMWare Server/ESX/ESXi or Xenapp etc.
      </p>
      <p>
       The settings I used for VMWare:
      </p>
      <blockquote>
       <p>
        I named it SNORT
        <br/>
        1 processor, 2 cores
        <br/>
        512MB RAM
        <br/>
        Bridged Networking single interface (eth0)
        <br/>
        SCSI LSI Logic I/O
        <br/>
        SCSI 20GB HD Partition Split into 2GB Files
        <br/>
        Removed the Floppy Disk
        <br/>
        <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/hardware.png">
         <img alt="" class="alignnone size-full wp-image-516" height="233" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/hardware.png" title="hardware" width="326"/>
        </a>
       </p>
      </blockquote>
      <p>
       Im also using Ubuntu Server 10.04 LTS 32bit.  Grab that ISO if you dont already have it.  Oh, and since some of my readers want me to put this: I grabbed the information used in this post from the ubuntuforums, from bil at play, and from Nick Moore.
      </p>
      <p>
       I assume that you already know how to load Ubuntu Server.  If not, then Snort will probably be a little bit over your head.  http://linux.insanelan.com.  Ubuntu was loaded with the following settings:
      </p>
      <blockquote>
       <p>
        Language  English
        <br/>
        Hostname  snortsvr
        <br/>
        Time Zone  CDT
        <br/>
        Partitioning method  Guided used entire disk
        <br/>
        Full name for new user  buddy
        <br/>
        Username for account  buddy
        <br/>
        Password  buddy (I know its weak)
        <br/>
        Encrypt home directory  no
        <br/>
        Automatic Updates  no
        <br/>
        <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/packages.png">
         <img alt="" class="alignnone size-full wp-image-517" height="166" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/packages.png" title="packages" width="229"/>
        </a>
       </p>
       <p>
        I always install LAMP, Mail, and OpenSSH.  Just a standard setup for me.
       </p>
       <p>
        MySQL Root Password  password
        <br/>
        Postfix configuration  internet site
        <br/>
        System mail name  snortsvr.local
        <br/>
        Load Grub to MBR  yes
        <br/>
        Continue, and then itll reboot your system.
       </p>
      </blockquote>
      <p>
       Login as buddy/buddy.  Now I dislike the whole sudo crap.  Its my server, I dont want to have to ask for permission to use it.  If this is a shared server  sure go ahead  otherwise do the following:
       <br/>
       <code>
        sudo passwd root
       </code>
       <br/>
       Enter your buddy password
       <br/>
       Enter your root password (I put in password)
       <br/>
       <code>
        su
       </code>
       <br/>
       Enter your root password
      </p>
      <p>
       This is optional but I find it a lot easier to SSH via putty to my system than to control it via the VMWare interface.
       <br/>
       <code>
        ifconfig eth0
       </code>
       <br/>
       Make note of your IP address
       <br/>
       Open Putty and SSH to your Server (in my case its 10.4.0.54)
       <br/>
       Login as root
      </p>
      <p>
       Youll notice that there will be several packages to update.  Update them.
       <br/>
       <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/update.png">
        <img alt="" class="alignnone size-full wp-image-519" height="259" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/update.png" title="update" width="635"/>
       </a>
       <br/>
       <code>
        apt-get update
       </code>
       <br/>
       <code>
        apt-get upgrade
       </code>
      </p>
      <p>
       Depending on the speed of your internet connection, you may have time to watch a movie  or maybe its already done and youre wondering why I typed this sentence.  Either way, keep reading:
      </p>
      <p>
       <strong>
        DOWNLOADS AND INSTALLATIONS
       </strong>
       <br/>
       Install the following on your Ubuntu system:
       <br/>
       <code>
        apt-get install libpcap0.8-dev libmysqlclient15-dev bison flex apache2 php5 libapache2-mod-php5 php5-gd php5-mysql libtool libpcre3-dev php-pear
       </code>
      </p>
      <p>
       Make a directory for you to download stuff:
       <br/>
       <code>
        mkdir ~/snortfiles
       </code>
       <br/>
       <code>
        cd ~/snortfiles
       </code>
      </p>
      <p>
       Youll need to download the following:
       <br/>
       Base 1.4.5
       <br/>
       <code>
        wget http://downloads.sourceforge.net/project/secureideas/BASE/base-1.4.5/base-1.4.5.tar.gz?use_mirror=voxel
       </code>
      </p>
      <p>
       ADOdb 4991
       <br/>
       <code>
        wget http://downloads.sourceforge.net/project/adodb/adodb-php-4-and-5/adodb-4991-for-php/adodb4991.tgz?use_mirror=voxel
       </code>
      </p>
      <p>
       Barnyard2 1.7
       <br/>
       <code>
        wget http://www.securixlive.com/download/barnyard2/barnyard2-1.7.tar.gz
       </code>
      </p>
      <p>
       <strong>
        Setup Snort
       </strong>
       <br/>
       <code>
        apt-get install snort-mysql
       </code>
       <br/>
       I used my local LAN (10.4.0.0/24)
       <br/>
       Click OK
       <br/>
       YES  you do want to setup a database for snort-mysql
       <br/>
       Click OK
       <br/>
       <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/snort-mysql.png">
        <img alt="" class="alignnone size-full wp-image-525" height="403" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/snort-mysql.png" title="snort-mysql" width="646"/>
       </a>
      </p>
      <p>
       <code>
        mysql -u root -p
       </code>
       <br/>
       My password is still password
      </p>
      <p>
       <code>
        create database snort;
       </code>
       <br/>
       <code>
        grant CREATE, INSERT, SELECT, DELETE, UPDATE on snort.* to snort@localhost;
       </code>
       <br/>
       <code>
        SET PASSWORD FOR snort@localhost=PASSWORD('yourpasswordhere');
       </code>
       <br/>
       <code>
        exit
       </code>
       <br/>
       I used password once again for the password.
      </p>
      <p>
       <code>
        cd /usr/share/doc/snort-mysql/
       </code>
       <br/>
       Import the snort DB schema
       <br/>
       <code>
        zcat create_mysql.gz | mysql -u root -p snort
       </code>
      </p>
      <p>
       Verify that the import went well
       <br/>
       <code>
        mysql -u root -ppassword
       </code>
       <br/>
       <code>
        SHOW DATABASES;
       </code>
       <br/>
       <code>
        use snort;
       </code>
       <br/>
       <code>
        SHOW TABLES;
       </code>
       <br/>
       You should see 16 rows in the set.  If not, then you did something wrong.
       <br/>
       <code>
        exit
       </code>
       <br/>
       <strong>
        <br/>
        Edit snort.conf
       </strong>
       <br/>
       <code>
        nano /etc/snort/snort.conf
       </code>
      </p>
      <blockquote>
       <p>
        Find var HOME_NET any, and change it to 
        <code>
         var HOME_NET $eth0_ADDRESS
        </code>
        
        <br/>
        Find Output log_tcpdump: tcpdump.log and change it to 
        <code>
         #Output log_tcpdump: tcpdump.log
        </code>
        
        <br/>
        Find output log_unified and insert 
        <code>
         output unified2: filename snort.log, limit 128
        </code>
         below it.
        <br/>
        save and exit
       </p>
      </blockquote>
      <p>
       Youre now done installing snort!
      </p>
      <p>
       <strong>
        Setting up BASE and ABOdb
       </strong>
       <br/>
       <code>
        pear install --alldeps Mail
       </code>
       <br/>
       <code>
        pear install --alldeps Mail_Mime
       </code>
       <br/>
       <code>
        pear install --alldeps Image_Canvas-0.3.2
       </code>
       <br/>
       <code>
        pear install --alldeps Image_Graph-0.7.2
       </code>
      </p>
      <p>
       Navigate to your snortfiles folder
       <br/>
       <code>
        cd
       </code>
       <br/>
       <code>
        cd snortfiles
       </code>
       <br/>
       <code>
        tar -zxvf adodb4991.tgz
       </code>
       <br/>
       <code>
        tar -zxvf base-1.4.5.tar.gz
       </code>
       <br/>
       <code>
        mv adodb /var/www
       </code>
       <br/>
       <code>
        mv base-1.4.5 /var/www
       </code>
      </p>
      <p>
       <code>
        nano /etc/php5/apache2/php.ini
       </code>
      </p>
      <blockquote>
       <p>
        Find Dynamic Extensions and add the following to the end of that section:
        <br/>
        <code>
         extension=mysql.so
        </code>
        <br/>
        <code>
         extension=gd.so
        </code>
        <br/>
        Find error_reporting = E_ALL &amp; ~E_DEPRECATED
        <br/>
        Replace with
        <code>
         error_reporting = E_ALL &amp; ~E_NOTICE
        </code>
        <br/>
        save and exit
       </p>
      </blockquote>
      <p>
       <code>
        nano /etc/apache2/apache2.conf
       </code>
       <br/>
       Insert the following at the very bottom of the file: 
       <code>
        servername snortsvr.local
       </code>
       
       <br/>
       save and exit
      </p>
      <p>
       <code>
        apache2ctl restart
       </code>
      </p>
      <p>
       <code>
        cd /var/www
       </code>
       <br/>
       <code>
        ln -s base-1.4.5 ./base
       </code>
       <br/>
       <code>
        chmod a+w base
       </code>
      </p>
      <p>
       Open up a web browser and navigate to http://IPADDRESSOFYOURSERVER/base
      </p>
      <blockquote>
       <p>
        1.) Set the path to adodb to /var/www/adodb
        <br/>
        2.) Database Name=snort
        <br/>
        Database Host=localhost
        <br/>
        Database User=snort,
        <br/>
        Database Password=yourpassword
        <br/>
        3.) check use authentication system
        <br/>
        Admin User Name=snort
        <br/>
        Password=yourpassword
        <br/>
        Full Name=snort
        <br/>
        4.) Click Create BASE AG and click on step 5
        <br/>
        5.) Test your login and password and verify everything is working OK
       </p>
      </blockquote>
      <p>
       <code>
        chmod og-w base
       </code>
      </p>
      <p>
       <strong>
        Setting up Barnyard2
       </strong>
      </p>
      <p>
       Find where you downloaded Barnyard2-1.7.tar.gz (Home directory anyone?  cd ~)
      </p>
      <p>
       <code>
        tar -zxvf barnyard2-1.7.tar.gz
       </code>
       <br/>
       <code>
        cd barnyard2-1.7
       </code>
       <br/>
       <code>
        ./configure --with-mysql &amp;&amp; make &amp;&amp; make install
       </code>
       <br/>
       <code>
        cp etc/barnyard2.conf /etc/snort
       </code>
       <br/>
       <code>
        mkdir /var/log/barnyard2
       </code>
      </p>
      <p>
       <code>
        nano /etc/snort/barnyard2.conf
       </code>
       <br/>
       Find #config hostname: thor and change it to 
       <code>
        config hostname: localhost
       </code>
       
       <br/>
       Find #config interface: eth0 and change it to 
       <code>
        config interface: eth0
       </code>
       
       <br/>
       Find output database and insert below that line 
       <code>
        output database: alert, mysql, user=snort password=password dbname=snort host=localhost
       </code>
       
       <br/>
       Save and exit
      </p>
      <p>
       <strong>
        Starting Snort/Barnyard
       </strong>
       <br/>
       <code>
        snort -c /etc/snort/snort.conf -i eth0
       </code>
       <br/>
       If you see Not Using PCAP_FRAMES youre doing OK!
      </p>
      <p>
       Open up another Putty session and SSH to your server as root.
       <br/>
       <code>
        ls -la /var/log/snort
       </code>
       <br/>
       Look for a 10 digit suffix on snort.log.  If you have more than one file with digits, look at the timestamp and grab the newest one!
       <br/>
       <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/tendigits.png">
        <img alt="" class="alignnone size-full wp-image-531" height="98" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/tendigits.png" title="tendigits" width="545"/>
       </a>
       <br/>
       <code>
        nano /var/log/snort/barnyard.waldo
       </code>
       <br/>
       Paste in the following:
      </p>
      <blockquote>
       <p>
        <code>
         /var/log/snort
        </code>
        <br/>
        <code>
         snort.log
        </code>
        <code>
         10DIGIT NUMBER FROM YOUR SNORT LOG STEP ABOVE
        </code>
        <br/>
        <code>
         0
        </code>
        <br/>
        <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/waldo.png">
         <img alt="" class="alignnone size-full wp-image-533" height="410" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/waldo.png" title="waldo" width="662"/>
        </a>
        <br/>
        save and exit
       </p>
      </blockquote>
      <p>
       Run this SINGLE LINE command:
       <br/>
       <code>
        /usr/local/bin/barnyard2 -c /etc/snort/barnyard2.conf -G /etc/snort/gen-msg.map -S /etc/snort/sid-msg.map -d /var/log/snort -f snort.log -w /var/log/snort/barnyard.waldo
       </code>
      </p>
      <p>
       Ping your server a few times.  Then open up your browser to http://YOURSERVERIPADDRESS/base and see if anything shows up.
      </p>
      <p>
       <strong>
        Auto Starting Snort
       </strong>
       <br/>
       Control + C out of either Snort or Barnyard.  Then reboot the server.
       <br/>
       <code>
        reboot
       </code>
      </p>
      <p>
       Log back into your server as root using SSH/Putty.
       <br/>
       <code>
        ps -A | grep snort
       </code>
       <br/>
       If nothing shows up (which nothing should), then continue on.
       <br/>
       <code>
        /etc/init.d/snort start
       </code>
       <br/>
       <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/grepsnort.png">
        <img alt="" class="alignnone size-full wp-image-536" height="129" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/grepsnort.png" title="grepsnort" width="502"/>
       </a>
      </p>
      <p>
       <code>
        mv /etc/snort/db-pending-config /etc/snort/db-pending-config.orig
       </code>
       <br/>
       <code>
        /etc/init.d/snort start
       </code>
       <br/>
       <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/initstart.png">
        <img alt="" class="alignnone size-full wp-image-537" height="64" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/initstart.png" title="initstart" width="666"/>
       </a>
      </p>
      <p>
       <code>
        nano /etc/init/barnyard2.conf
       </code>
      </p>
      <blockquote>
       <p>
        <code>
         # rc - System V runlevel compatibility
        </code>
        <br/>
        <code>
         # This task runs the old System V-style rc script when changing between
        </code>
        <br/>
        <code>
         # runlevels.
        </code>
        <br/>
        <code>
         description     "Barnyard2 for Snort support"
        </code>
        <br/>
        <code>
         author          "bil b@unc.edu"
        </code>
        <br/>
        <code>
         start on started networking
        </code>
        <br/>
        <code>
         #start on startup
        </code>
        <br/>
        <code>
         #start on (startup
        </code>
        <br/>
        <code>
         #          and filesystem
        </code>
        <br/>
        <code>
         #          and started udev)
        </code>
        <br/>
        <code>
         #stop on runlevel [!023456]
        </code>
        <br/>
        <code>
         respawn
        </code>
        <br/>
        <code>
         exec /usr/local/bin/barnyard2 \
        </code>
        <br/>
        <code>
         -c /etc/snort/barnyard2.conf \
        </code>
        <br/>
        <code>
         -G /etc/snort/gen-msg.map \
        </code>
        <br/>
        <code>
         -S /etc/snort/sid-msg.map \
        </code>
        <br/>
        <code>
         -d /var/log/snort -f snort.log \
        </code>
        <br/>
        <code>
         -w /var/log/snort/barnyard.waldo
        </code>
        <br/>
        save and exit
        <br/>
        <code>
         reboot
        </code>
       </p>
      </blockquote>
      <p>
       <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/barnyard.png">
        <img alt="" class="alignnone size-full wp-image-541" height="335" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/barnyard.png" title="barnyard" width="579"/>
       </a>
      </p>
      <p>
       <strong>
        Verify Everything Is Working
       </strong>
      </p>
      <p>
       After your server reboots, log back into it via SSH as root.
      </p>
      <p>
       <code>
        ps -A | grep snort
        <br/>
        <code>
         ps -A | grep barnyard
         <br/>
         As long as you see the process ID and the name, you should be OK.  You can check that barnyard will auto restart by killing the process ID and then checking to see if it's running.  It should have a new ID afterwards.
         <br/>
         <a href="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/killprocess.png">
          <img alt="" class="alignnone size-full wp-image-538" height="129" src="http://it.thelibrarie.com/weblog/wp-content/uploads/2010/06/killprocess.png" title="killprocess" width="329"/>
         </a>
        </code>
       </code>
      </p>
      <p>
       ***EDIT***
       <br/>
       I found a much easier solution - install Snorby.  Snorby is a pretty nice front-end for Snort.  Barnyard is still utilized to take the load off of Snort.
       <br/>
       You can grab the All In One solution (installs the OS, MySQL, Snort, Apache, Barnyard, Snorby, Etc) here:
       <br/>
       <a href="http://bailey.st/blog/snorby-spsa/">
        http://bailey.st/blog/snorby-spsa/
       </a>
       <br/>
       Supposedly it auto updates once a day with Oinkmaster, but I haven't really gone through to verify yet.  It'd be nice if some of the options were included in the GUI, but for free who am I to complain?
      </p>
