      </h2>
     </div>
    </div>

    <div class="row">
     <div class="small-12 medium-10 medium-centered large-8 columns leftPageContents" data-break="large">
      <div class="cms" itemprop="articleBody">
       <h4>
        If you're looking to get an upperhand and cut some corners with your PHP development, you may be interested in giving PHP 5.6 beta a spin.
       </h4>
       <p>
        <img alt="What's New in PHP 5.6 for Web Developers" src="http://static.madebymagnitude.com/blog/20381706895322053fd8bcf8.49462153_abdbe27c.jpg" style="width: 726px; height: 357px;"/>
       </p>
       <p>
        Although there is no official release date as of yet, there is already a swarm of great new features that are definitely not worth playing around with before the official release.
       </p>
       <h3>
        Farewell Raw Post Data
       </h3>
       <p>
        So far PHP 5.6 only see's two deprecated functions, one of them being the deprecation of the $
        <strong>
         HTTP_RAW_POST_DATA
        </strong>
        which is expected to be removed in a future release.
       </p>
       <h3>
        Changes to File Uploads
       </h3>
       <p>
        Prior to PHP 5.6, no uploads over 2GB were possible due to their poor handling and processing. However, the PHP team have now outlined that uploads of arbitrary size are now fully supported. In addition to this, the POST data memory usage has been shrunk by 2 - 3 times its predecessor. This reduction in memory footprint comes following the removal of the raw post data mentioned above.
       </p>
       <p>
        These changes that rely on accessing php://input to read the data mean that you're now able to reopen and reuse the input as many times as required, aiding the memory footprint reduction.
       </p>
       <h3>
        Variadic Functions
       </h3>
       <p>
        For those wondering what variadic functions are - they're those that accept a variable length of arguments on each call.
       </p>
       <pre class="language-php">
        <code>
         # Prior to PHP 5.6
function variadic($integer) {
	return $integer + array_sum(array_slice(func_get_args(), 1));
}# In PHP 5.6
function variadic($integer, ...$params) {
	return $integer + array_sum($params);
}# Example Calls
variadic(1); # Returns 1
variadic(1, 2); #Returns 3
variadic(1, 2, 3); # Returns 6
        </code>
       </pre>
       <p>
        Using the new syntax, PHP will assign the first variable $
        <strong>
         integer
        </strong>
        and then place any other parameters into the $
        <strong>
         params
        </strong>
        variable. Removing the need of calling
        <strong>
         func_get_params
        </strong>
        . This also allows us to conserve our memory footprint by passing the parameters over by reference with the ampersand syntax of
        <strong>
         &amp;...$params
        </strong>
        .
       </p>
       <h3>
        Argument Unpacking via ...
       </h3>
       <p>
        When you call a function using the new ... syntax you can unpack array and traversable objects into argument lists. For those that program in Ruby, this is known as the splat operator.
       </p>
       <p>
        Given the example from PHP.net, we create a function which accepts three parameters and then returns the sum of them.
       </p>
       <pre class="language-php">
        <code>
         function add($a, $b, $c) {
    return $a + $b + $c;
}# A very simple call:
add(1, 2, 6); # Returns 6
        </code>
       </pre>
       <p>
        As of PHP 5.6, we're able to pass over arrays or traversable objects:
       </p>
       <pre class="language-php">
        <code>
         # Using the new syntax of PHP 5.5
$numbers = [1, 2, 3];# The array is unpacked and passed over
add(...$numbers);
        </code>
       </pre>
       <p>
        Want to find out more? Read about
        <a href="https://wiki.php.net/rfc/argument_unpacking" rel="nofollow external" target="_blank">
         PHP argument unpacking
        </a>
        on their RFC.
       </p>
       <h3>
        Importing Namespaced Constants and Functions
       </h3>
       <p>
        With PHP 5.6 you're able to import namespaced constants and functions directly into other classes which expands on what is currently possible; being able to import namespaces and types such as classes, interfaces and traits.
       </p>
       <pre class="language-php">
        <code>
         namespace NamespaceTest {
	const SCORE = 1;	function getScore() {
		# ...
	}
}namespace {
	use NamespaceTest  as Test;
	var_dump(TestgetScore());
}
        </code>
       </pre>
       <p>
        From PHP 5.6 we're able to use the
        <strong>
         use function
        </strong>
        and
        <strong>
         use const
        </strong>
        statements to import standalone segments of code:
       </p>
       <pre class="language-php">
        <code>
         namespace {
	use function NamespaceTestgetScore;
	use const NamespaceTestSCORE;
	getScore();
}
        </code>
       </pre>
       <h3>
        Default Character Encoding
       </h3>
       <p>
        The default value of the character encoding has been changed to 'UTF-8' and can be controlled by accessing
        <strong>
         default_charset()
        </strong>
        . Updating the default value is reflected in the mbstring and iconv modules in addition to htmlentities, htmlspecialchars and html_entity_decode.
       </p>
       <h3>
        PHPDBG Interactive Debugging
       </h3>
       <p>
        The
        <a href="http://phpdbg.com/" rel="nofollow external" target="_blank">
         phpdbg debugger
        </a>
        is now bundled by default as SAPI module (Server API). You can access it from the command line and step through your code at runtime, inspecting breakpoints or Opcode cache generated.
       </p>
       <h3>
        Under the Bonnet
       </h3>
       <p>
        PHP 5.6 includes updates to OpenSSL for SSL/TLS improvements, Zip for on-the-fly password protecting archives and Crypto hashing updates.
       </p>
