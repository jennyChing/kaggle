      <h2 class="post-title" id="single-title">
       Android/Arduino RF Outlets Selector
      </h2>
      <p class="post-byline" id="single-byline">
       By
       <a href="http://www.33snowflakes.com/blog/author/stephen/" rel="author" title="Posts by stephen">
        stephen
       </a>
       on January 1st, 2013
      </p>
      <h2>
       Overview
      </h2>
      <p>
       Heres a great little project I implemented with Arduino and Android  switching outlets from my smart phone!
      </p>
      <p>
      </p>
      <p>
       <iframe allowfullscreen="" frameborder="0" height="281" src="http://www.youtube.com/embed/57lqu7YIad8?feature=oembed" width="500">
       </iframe>
      </p>
      <p>
      </p>
      <p>
       I started with off the shelf RF Wireless Outlets. At $6 per outlet, they are a complete steal. I got lucky and found all 4 channels of these wireless/RF outlets at a local Big Lots. Ive seen them at Home Depot too. And of course theyre available on Amazon. Compare that pricing to one unit of the
       <a href="http://www.amazon.com/gp/product/B00BB2MMNE/ref=as_li_ss_tl?ie=UTF8&amp;camp=1789&amp;creative=390957&amp;creativeASIN=B00BB2MMNE&amp;linkCode=as2&amp;tag=33snowfl-20">
        Belkin WeMo Switch
       </a>
       <img alt="" border="0" height="1" src="http://ir-na.amazon-adsystem.com/e/ir?t=33snowfl-20&amp;l=as2&amp;o=1&amp;a=B00BB2MMNE" style="border:none !important; margin:0px !important;" width="1"/>
       <br/>
       <img alt="Big Lots Outlets" src="http://www.33snowflakes.com/blog/wp-content/uploads/2013/01/IMG_20130101_202515-e1357136105326.jpg"/>
      </p>
      <h3>
       Diagram:
      </h3>
      <p>
       <a href="http://www.33snowflakes.com/blog/wp-content/uploads/2014/04/OutletDiagram.png">
        <img alt="OutletDiagram" class="alignnone size-medium wp-image-254" height="176" src="http://www.33snowflakes.com/blog/wp-content/uploads/2014/04/OutletDiagram-300x176.png" width="300"/>
       </a>
      </p>
      <h3>
       Parts List:
      </h3>
      <ul>
       <li>
        <b>
         315MHz Wireless Outlets 
        </b>
        <br/>
        <a href="http://www.amazon.com/dp/B003ZTWYXY/?tag=33snowfl-20">
         Woods 13569 Indoor Wireless Outlet (3-Pack) on Amazon
        </a>
       </li>
       <li>
        <b>
         315MHz Chip 
        </b>
        <br/>
        <a href="http://www.amazon.com/dp/B00MO0XBG4/?tag=33snowfl-20">
         SMAKN 315Mhz Rf Transmitter and Receiver Kit on Amazon
        </a>
       </li>
       <li>
        <b>
         Arduino 
        </b>
        <br/>
        <a href="http://www.amazon.com/dp/B006H06TVG/?tag=33snowfl-20">
         Arduino UNO R3 on Amazon
        </a>
       </li>
       <li>
        <b>
         Arduino Ethernet Shield 
        </b>
        <br/>
        <a href="http://www.amazon.com/dp/B00CRX7BI2/?tag=33snowfl-20">
         Arduino Ethernet Shield R3 on Amazon
        </a>
       </li>
      </ul>
      <p>
       All the code is on
       <a href="https://bitbucket.org/thebostik/arduino-projects" title="Bitbucket project repo">
        Bitbucket
       </a>
       . There is also a simple remote control Android / IPhone application available in the Google Play and ITunes stores. (Better looking than in the video, too!)
       <br/>
       <br/>
       <br/>
       In the last year or so, there may be better options than Arduino for the brain, but that code rewrite will have to wait until another day. This is what the circuit looks like very simple, just hook up the 4 pins of the transmitter to the correct place and you will be ready to load the code onto the Arduino!
       <br/>
       <img alt="Completed Circuit" src="http://www.33snowflakes.com/blog/wp-content/uploads/2013/01/IMG_20130102_090254.jpg"/>
       <br/>
       <br/>
       <br/>
       The Arduino can hook up to your router via the Ethernet shield, and when its turned on will be running a web server at the URL:
       <code>
        {IP address}/outlet?outlet={outlet}&amp;channel={channel}&amp;state={state}
       </code>
       <br/>
       <br/>
       <br/>
       And of course the Android and IPhone app comes with a UI added to those URLs, so you can click a toggle button for all the outlets, rename them, etc. See more about the App
       <a href="http://www.33snowflakes.com/blog/homeremote-v1-launch/" title="HomeRemote V1.0  Android and iPhone Apps Launched!">
        here
       </a>
       .
      </p>
      <p>
       I based the code on a cool outlet project that was pretty mature, called
       <a href="http://code.google.com/p/rc-switch/">
        RC Switch
       </a>
       , but in Europe the RF protocols and outlets are a bit different. There was also a reference from Instructables, but those RF on/off timings didnt match my outlets. If you dont have luck with my code as-is, modify the pulse length timings to match what is found
       <a href="http://www.instructables.com/id/Cheap-Home-Automation-using-Wireless-Outlet-Module/" title="Instructables project">
        here
       </a>
       . There may be more variants that look the same, as a lot of these outlet modules look the same but are rebranded.
       <br/>
       <br/>
      </p>
      <h2>
       Reverse Engineering / Troubleshooting
      </h2>
      <p>
       Hopefully during the project you can implement it in a straight-forward way, but if the same protocol does not control the relays, read on to how I troubleshot this project originally
       <br/>
       To figure out the problem above, I used the Arduino as a logic analyzer to figure out what exactly wasnt working, and that was probably the most fun part of the project. I soldered leads onto the stock wireless remote to spy on the signal that was being transmitted, and saw the digital signal graph on my TV screen. Amazing! As a programmer, I forget sometimes that there is electricity flowing through everything and changing direction every few nanoseconds.
       <br/>
       <br/>
       <br/>
       This
       <a href="https://github.com/gillham/logic_analyzer">
        Arduino logic analyzer
       </a>
       is a great open-source code project, and saved me big time here. It has 500 kHz max resolution, but since there is only 1kb of RAM, youre limited to 1024 samples. That resolution was actually so good, that the first few times I thought it wasnt working I was zoomed in too far and didnt see any transitions. As soon as I zoomed out to 1 kHz, I saw everything! And 20 kHz was finally the sweet spot to measure the timings accurately.
       <br/>
       <br/>
       <br/>
       This is how I instrumented the remote control for use with the logic analyzer. I soldered many jumpers onto the circuit  only 2 carried the signal I wanted. The orange wire, and the right-most white wire both had the correct waveform  though the white one was inverted. I also provided power from the Arduino, since the remote had a 12V battery, some traces might have been carrying too much juice for the Arduino!
       <br/>
       <img alt="Wireless Remote instrumented" src="http://www.33snowflakes.com/blog/wp-content/uploads/2013/01/IMG_20130102_090158.jpg"/>
       <br/>
       <br/>
       <br/>
       Feel free to provide critique or suggestions!
      </p>
