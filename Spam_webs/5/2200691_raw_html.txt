      <p>
       En servidores como en equipos de escritorio es importante tener correctamente configuradas la fecha y la hora del sistema y para hacer eso siempre es til hacerlo por medio de un
       <strong>
        servidor NTP
       </strong>
       (Network Protocol Time) y aqu te digo
       <strong>
        como instalar un servidor NTP en Centos 7.
       </strong>
      </p>
      <p>
       <img alt="centos ntp server" class="aligncenter size-full wp-image-2056" height="400" src="http://www.comoinstalarlinux.com/wp-content/uploads/centos-ntp-server.jpg" width="620"/>
      </p>
      <p>
       <span id="more-2055">
       </span>
      </p>
      <h2>
       Como instalar NTP en centos 7
      </h2>
      <p>
       Segn Wikipedia la definicin de NTP se puede leer como sigue:
      </p>
      <blockquote>
       <p>
        Network Time Protocol es un protocolo de Internet para sincronizar los relojes de los sistemas informticos a travs del enrutamiento de paquetes en redes con latencia variable.
       </p>
      </blockquote>
      <p>
       Y en
       <a href="http://www.comoinstalarlinux.com/tag/centos-7/" title="Centos 7">
        Centos 7
       </a>
       puedes
       <strong>
        instalar un servidor NTP
       </strong>
       que permita sincronizar la hora y fecha de los sistemas de otros equipos ya sean servidores o equipos de escritorio.
      </p>
      <p>
       Para instalar un servidor NTP en Centos 7 puedes ejecutar el siguiente comando:
      </p>
      <pre>
       # yuminstall ntp ntpdate
      </pre>
      <p>
       Una vez que se encuentre instalado el servidor puede configurar el reloj del sistema mediante el siguiente comando:
      </p>
      <pre>
       #ntpdate -u 0.pool.ntp.org
      </pre>
      <p>
       Luego se pueden definir las direcciones IP a las que se dar permiso para obtener los datos del horario. Para esto edita el archivo/etc/ntp.conf asi:
      </p>
      <pre>
       # vi /etc/ntp.conf
      </pre>
      <p>
       y agrega las direcciones que consideres adecuada para tu LAN o para las redes que consideres adecuadas.
      </p>
      <pre>
       # For more information about this file, see the man pages
# ntp.conf(5), ntp_acc(5), ntp_auth(5), ntp_clock(5), ntp_misc(5), ntp_mon(5).

driftfile /var/lib/ntp/drift

# Permit time synchronization with our time source, but do not
# permit the source to query or modify the service on this system.
restrict default nomodify notrap nopeer noquery

# Permit all access over the loopback interface.  This could
# be tightened as well, but to do so would effect some of
# the administrative functions.
restrict 127.0.0.1 
restrict ::1

# Hosts on local network are less restricted.
#restrict 192.168.1.0 mask 255.255.255.0 nomodify notrap
       <span style="color: #003366;">
        restrict 192.168.1.0 mask 255.255.255.0 nomodify notrap
       </span>
       <span style="color: #003366;">
        restrict 192.168.70.0 mask 255.255.255.128 nomodify notrap
       </span>
       <span style="color: #003366;">
        restrict 172.16.1.0 mask 255.255.255.240 nomodify notrap
       </span>
       <span style="color: #003366;">
        restrict 10.0.1.0 mask 255.255.255.248 nomodify notrap
       </span>
       # Use public servers from the pool.ntp.org project.
# Please consider joining the pool (http://www.pool.ntp.org/join.html).
server 0.centos.pool.ntp.org iburst
server 1.centos.pool.ntp.org iburst
server 2.centos.pool.ntp.org iburst
server 3.centos.pool.ntp.org iburst

#broadcast 192.168.1.255 autokey	# broadcast server
#broadcastclient			# broadcast client
#broadcast 224.0.1.1 autokey		# multicast server
#multicastclient 224.0.1.1		# multicast client
#manycastserver 239.255.254.254		# manycast server
#manycastclient 239.255.254.254 autokey # manycast client

# Enable public key cryptography.
#crypto

includefile /etc/ntp/crypto/pw

# Key file containing the keys and key identifiers used when operating
# with symmetric key cryptography. 
keys /etc/ntp/keys

# Specify the key identifiers which are trusted.
#trustedkey 4 8 42

# Specify the key identifier to use with the ntpdc utility.
#requestkey 8

# Specify the key identifier to use with the ntpq utility.
#controlkey 8

# Enable writing of statistics records.
#statistics clockstats cryptostats loopstats peerstats

# Disable the monitoring facility to prevent amplification attacks using ntpdc
# monlist command when default restrict does not include the noquery flag. See
# CVE-2013-5211 for more details.
# Note: Monitoring will not be disabled with the limited restriction flag.
disable monitor
      </pre>
      <p>
       Cuando hayas agregado las IPs o segmentos de IP adecuados graba el archivo y reinicia el servicio.
      </p>
      <pre>
       # systemctl enable ntpd
# systemctl start ntpd
      </pre>
      <p>
       Para configurar los servidores o equipos de escritorio puedes ejecutar el comando ntodate con la IP del servidor NTP con el nombre de host que pueda ser resuelto correctamente.
      </p>
      <pre>
       #ntpdate -u mi-servidorntp.local
# ntpdate -u 172.16.0.2
      </pre>
      <p>
       Espero que te sea de mucha utilidad y puedas configurar tu servicio NTP en Centos 7.
      </p>

       Tu Servidor y Amigo
