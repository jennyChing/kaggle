       Setting Up an Email Server
      </h1>

       <div class="content">
        <p>
         Within these pages I attempt to document what I am doing to setup an Linux email server that gets user information from a Windows NT server. The main reason for doing this was to get rid of Microsoft Exchange. Not because Exchange wasn't working, though I have encountered problems with it, but because we were not using all of it's features and, being a small company, we really cannot afford it. This does not mean we will not be making the appropriate donations once we settle on the packages we will be using.
        </p>
        <p>
         <i>
          (These pages cover now older versions of software who's configuration options may have changed. However I am leaving them up for reference. They do cover a setup I ran successfully for a year. I left the company after that, as it was purchased by another firm, to work for a much larger company running a Unix-based commercial package. I had no problems getting up to speed with it and found many similarities with what I had put together with open source software.)
         </i>
        </p>
        <p>
         If you are interested in building an email server without the NT part, just skip the samba stuff and also see the links at the end of this page.
        </p>
        <p>
         <i>
          These pages are currently a work in progress as I try different packages and configurations.
         </i>
        </p>
        <h3>
         Our requirements
        </h3>
        <ol>
         <li>
          No or very few extra steps when adding and removing users from the domain via NT.
         </li>
         <li>
          Ability to backup and restore individual user's email boxes without costly additional backup 'plug-ins'.
         </li>
         <li>
          Spam filtering. Preferably with RBL lookups, white lists, and individual user preferences.
         </li>
         <li>
          A public mailbox readable by the 'Domain Users' group and writable by a
          <i>
           PublicFolder
          </i>
          group.
         </li>
         <li>
          System wide address book(s).
         </li>
         <li>
          Access via the web for traveling folks.
         </li>
         <li>
          Virus scanning of all email as it arrives.
         </li>
         <li>
          An email client not as prone to vulnerabilities as Outlook is. (which is sort of a separate issue)
         </li>
        </ol>
        <p>
         With Exchange 5.5 you can do everything except #2 (that I've found doing
         <a href="http://www.microsoft.com/technet/treeview/default.asp?url=/technet/ittasks/support/sysrecov.asp">
          disaster recovery
         </a>
         ). Also, spam filtering requires expensive third party add-ons or the solution I came up with using
         <a href="/article/lrp-exim-setup">
          LRP and Exim
         </a>
         .
        </p>
        <h3>
         What I came up with
        </h3>
        <ul>
         <li>
          <b>
           OS =
           <a href="http://www.redhat.com/">
            Red Hat
           </a>
           8.0
          </b>
          <br/>
          Decided to use this mainly because I am the most familiar with it. No, I'm not switching to 9 and I haven't had time to play with Fedora yet.
         </li>
         <li>
          <a href="http://www.samba.org/">
           <b>
            Samba
           </b>
          </a>
          <br/>
          For communication with the NT server that we are keeping, for now.
         </li>
         <li>
          <b>
           MTA =
           <a href="http://www.exim.org/">
            Exim
           </a>
           4.x
          </b>
          <br/>
          Went with Exim after I got a feel for it on the LRP firewall. Most folks seem to use Sendmail.
         </li>
         <li>
          <b>
           IMAP =
           <a href="http://www.courier-mta.org/">
            Courier's imap server
           </a>
           <br/>
          </b>
          Another alternative is
          <a href="http://www.washington.edu/imap/">
           University of Washington's imap
          </a>
          server (uw-imap)
          <br/>
          This comes default with Red Hat, but I had to build my own to do something special.
         </li>
         <li>
          <b>
           Web Server =
           <a href="http://www.apache.org/">
            Apache
           </a>
          </b>
          <br/>
          I had long considered switching our IIS to Apache. It is a really nice web server.
         </li>
         <li>
          <b>
           Web Mail =
           <a href="http://www.squirrelmail.org/">
            Squirrelmail
           </a>
          </b>
          <br/>
          Another alternative is
          <a href="http://www.horde.org/imp/">
           Horde IMP
          </a>
          . I really like this package, but there are some things that irk me a little.
         </li>
         <li>
          <b>
           Anti-Spam =
           <a href="http://www.spamassassin.org/">
            Spamassassin
           </a>
          </b>
          <br/>
          I like it's configuration options. Exim can do RBL and white list as well.
         </li>
         <li>
          <b>
           Anti-Virus =
           <a href="http://www.clamav.net/">
            ClamAV
           </a>
          </b>
          <br/>
          <a href="http://www3.ca.com/Solutions/Product.asp?ID=3049">
           InoculateIT
          </a>
          is our AV software, but we need to upgrade to get Linux support. For now, ClamAV is working fine.
         </li>
         <li>
          <b>
           System-wide address books =
           <a href="http://www.openldap.org">
            openLDAP
           </a>
           with
           <a href="http://www.postgresql.org/">
            PostgreSQL
           </a>
           database as a backend
          </b>
          <br/>
          We needed to be able to use our contact information with MS Access and I wanted to eliminate keeping information up to date in two places. If we didn't need this I would only use LDAP.
          <a href="http://www.mysql.com/">
           MySQL
          </a>
          can also be used
         </li>
         <li>
          <b>
           Email client (aka MUA) =
           <a href="http://www.mozilla.org/products/thunderbird/">
            Thunderbird
           </a>
          </b>
          <br/>
          I tried/wanted to use
          <a href="http://www.pmail.com/">
           Pegasus
          </a>
          . It would have worked, but it needed to be a fairly user friendly system (read switching Outlook users). I feel that its imap integration is still a little rough around the edges from a basic user standpoint,	though I have heard that this part is soon to be rewritten. Thunderbird is working fine, so I doubt I'll switch once Pegasus is cleaned up.
         </li>
        </ul>
        <h3>
         Configuration Steps
        </h3>
        <p>
         In the following pages I have tried to outline what I have done. If a component did not meet my requirements but I had it working I will leave the steps for others. Most of this you can do with RPMs, but in a few cases you need to build the packages. If you have never messed with Linux, it is not as hard as you might think. Because my test server, an old workstation, only has a total of 1.3gig in hardrive space I have not installed X-Windows. Everything is done via the command line or through Webmin.
        </p>
        <p>
         Note: The order listed below is not necessarily the order you have to install the packages. For instance I list LDAP last, but if you need Exim with LDAP support then of course you will need LDAP installed first. Also there may be components you do not need, such as webmail access which would knock out Apache, PHP, and Squirrelmail or Imp. And, too, there are other components that do the same things such as using sendmail instead of Exim.
        </p>
        <ol>
         <li>
          <b>
           Install Linux
          </b>
          <br/>
          Download or buy the flavor of your choice.
          <a href="http://www.redhat.com/">
           Red Hat is here
          </a>
          .
          <br/>
          The list of packages I installed on the test box for Red Hat 8.0 is
          <a href="/Linux-RHInstallPkgs.html">
           here
          </a>
          .
         </li>
         <li>
          <b>
           <a href="/article/setting-samba">
            Setup Samba and Winbind
           </a>
          </b>
          <br/>
          This is only required if you want to get user information from a NT server. There are many other ways to store users that a mail server can make use of.
         </li>
         <li>
          <b>
           <a href="/article/setting-clamav">
            Install and configure antivirus
           </a>
          </b>
          <br/>
          (Originally this page had more than just ClamAV, but I think that is the only page that might be worth leaving up now.)
         </li>
         <li>
          <b>
           <a href="/article/setting-exim">
            Build and Setup Exim 4.x
           </a>
          </b>
          <span style="color: red">
           *New: Building RPMs*
          </span>
          <br/>
          For virus and/or spam detection:
          <br/>
          <a href="/article/setting-exiscan">
           Patch into Exim and Setup exiscan
          </a>
          (read before building Exim)
          <br/>
          or
          <br/>
          <a href="/article/setting-mailscanner-exim">
           Install and Setup MailScanner
          </a>
         </li>
         <li>
          <b>
           <a href="/article/setting-imap">
            Build and Setup UW or Courier's IMAP
           </a>
          </b>
          <span style="color: red">
           *New: Building RPMs*
          </span>
          <br/>
          You can use the Red Hat RPM for UW if you do not want the ability to auto-create user directories on login via pam.
         </li>
         <li>
          <b>
           Setup Apache and PHP4
          </b>
          <br/>
          Using Red Hat's RPMs for now.
          <br/>
          <a href="/article/setting-ssl-certificates-apache">
           Setup SSL Certificates on Apache
          </a>
          <br/>
          Some of my notes on setting up Apache (removed)
          <br/>
          I'll go more into the configuration at some point. For now see the respected package's web site:
          <br/>
          <a href="http://www.apache.org/">
           Apache
          </a>
          <br/>
          <a href="http://www.php.net/">
           PHP
          </a>
         </li>
         <li>
          <b>
           Setup a system-wide address book
          </b>
          <br/>
          <a href="/article/setting-postgresql-odbc">
           Setting up PostgreSQL &amp; ODBC
          </a>
          <span style="color: red">
           *New*
          </span>
          <br/>
          or
          <br/>
          <a href="/article/setting-mysql-iodbc">
           Setting up MySQL &amp; ODBC
          </a>
          <br/>
          then
          <br/>
          <a href="/article/setting-ldap-back-sql">
           Setting up openLDAP with back-sql
          </a>
          <span style="color: red">
           *New: Building RPMs, v2.1.x, PostgreSQL*
          </span>
         </li>
         <li>
          <b>
           Install and configure webmail
          </b>
          <br/>
          <a href="/article/setting-squirrelmail">
           Install and configure Squirrelmail
          </a>
          <br/>
          This comes with Red Hat 8.0, but the RPM expects sendmail (use the --nodep option). There are a few things in here that might help if you go with the RPM and it changes fairly
    quickly, so learning to install is not a bad idea.
          <br/>
          or
          <br/>
          <a href="/article/setting-horde-imp">
           Install and configure IMP
          </a>
         </li>
         <li>
          <b>
           Install and configure spam detection by keyword
          </b>
          <br/>
          <i>
           to come shortly
          </i>
         </li>
        </ol>
        <p>
         At this point I am still exploring backup options. We currently have a single server version of BackupExec for NT, though an older one. The main debate is do we keep the tape backup and it's software on the NT server or move it to the Linux server. If our email box works out for us over time we may consider dumping NT all together. Samba 3.0, along with Webmin, may be the key for us to switch.
        </p>
        <p>
         The main problem I have with this setup is if the NT server goes down then Exim cannot get user information and there is no way for the users to login. I've found a way to defer messages being delivered if that happens, but I still think it would be better if I could replicate the SAM to this box. Maybe Samba x.x will have the ability to be a BDC to an NT PDC. Or maybe
         <a href="http://www.samba-tng.org/">
          Samba-TNG
         </a>
         is the way to go. I've tried these rather hastily and could not get them to work 100%, so I will have to come back to them later.
        </p>
