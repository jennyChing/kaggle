        <h1 class="entry-title">
         For Lazy Admins Part I
        </h1>
       </header>
       <!-- .entry-header -->
       <div class="entry-content">
        <script>
         &lt;!--
dynamicgoogletags.update();
//--&gt;
        </script>
        <p>
         In this series,
         <strong>
          For Lazy Admins
         </strong>
         I decided to write the tips, which I follow, for speed up the server Administration Job.I think this series will help many Administrators to do more jobs in less time. All the tips here am going to give are simple. But trust me, all these will help you to do things faster.
         <a href="http://techtrunch.com/wp-content/uploads/2010/09/lazy.jpg">
          <img alt="" class="aligncenter size-full wp-image-483" height="330" src="http://techtrunch.com/wp-content/uploads/2010/09/lazy.jpg" title="lazy" width="373"/>
         </a>
        </p>
        <p>
         <span id="more-482">
         </span>
         <strong>
          Note : Remember, more speed in pro-Job will give you more time to do your own hobbies :).
         </strong>
        </p>
        <p>
         <strong>
          1. Creating php info page with a one liner.
         </strong>
        </p>
        <p>
         <span style="font-family: monospace;">
         </span>
        </p>
        <pre class="brush:shell">
         echo -e "&lt;?php \nphpinfo(); \n?&gt;" &gt; info.php
        </pre>
        <p>
         Afterrunning this just change the owner ship of
         <code>
          info.php
         </code>
         using
         <code>
          chown
         </code>
         command.
        </p>
        <p>
         <strong>
          2. Creating a php test page
         </strong>
        </p>
        <p>
         <span style="font-family: monospace;">
         </span>
        </p>
        <pre class="brush:shell">
         echo -e "&lt;?php \necho \"testpage\"  \n?&gt;" &gt; test.php
        </pre>
        <p>
         Afterrunning this just change the owner ship of
         <code>
          test.php
         </code>
         using
         <code>
          chown
         </code>
         command.
        </p>
        <p>
         <strong>
          3. Clearing
          <code>
           session
          </code>
          and
          <code>
           dat
          </code>
          files from
          <code>
           /tmp
          </code>
          , deleting all frozen mails from
          <code>
           exim
          </code>
          queue and finally run
          <code>
           top
          </code>
          command.
         </strong>
        </p>
        <p>
         <strong>
         </strong>
        </p>
        <p>
         <span style="font-family: monospace;">
         </span>
        </p>
        <pre class="brush:shell">
         rm -vf /tmp/sess_[a-d]* ; rm -vf /tmp/sess_[a-o]* ; rm -vf /tmp/sess_[a-z]* ; rm -vf /tmp/sess_[0-4]* ; rm -vf /tmp/sess_[0-9]* ; rm -vf /tmp/sess_* ; rm -vf /tmp/cache_* ; rm -vf /tmp/*.dat ; exim -bp | grep frozen | awk '{ print $3 }' | xargs exim -Mrm ; sleep 3 ; top
        </pre>
        <p>
         <strong>
          4. Removing blocked IPs from firewalls [csf] blocked list
         </strong>
        </p>
        <p>
         <strong>
         </strong>
        </p>
        <pre class="brush:plain">
         root@server10 [~]# grep xx.90.233.56 /etc/csf/* -irl
/etc/csf/csf.deny
/etc/csf/csf.tempip
root@server10 [~]# sed -n '/xx.90.233.56/p' /etc/csf/csf.deny
xx.90.233.56 # lfd: 5 (cpanel) login failures from xx.90.233.56 (IN/India/-) in the last 300 secs - Wed Sep  8 10:38:24 2010
root@server10 [~]# sed -i '/xx.90.233.56/d' /etc/csf/csf.deny
root@server10 [~]# sed -i '/xx.90.233.56/d' /etc/csf/csf.tempip
root@server10 [~]# grep xx.90.233.56 /etc/csf/* -irl
root@server10 [~]# /etc/init.d/csf restart
        </pre>
        <p>
         <strong>
          5. Counting number of proccesors on a Linux server
         </strong>
        </p>
        <p>
         <strong>
         </strong>
        </p>
        <pre class="brush:plain">
         root@server3 [~]# grep -c processor /proc/cpuinfo
        </pre>
        <p>
         <strong>
          6. Changing recursively the permissions of files/directories
         </strong>
        </p>
        <p>
         <strong>
         </strong>
        </p>
        <pre class="brush:plain">
         find . -type f -exec chmod 644 {} \; ## set 644 permission for files recursively
find . -type d -exec chmod 755 {} \; ## set 755 permission for directories recursively
        </pre>
        <p>
         <strong>
          7. Finding out total IPs in a server.
         </strong>
         <br/>
         On viewing this one liner you may think this is veryunnecessary. But I wrote this one liner for a purpose. One day one of my company client asked me to give him a report on the total IPs in all his servers. There are almost 37 linux servers. Collecting IPs is pretty much easier using simple
         <code>
          ifconfig grep
         </code>
         combination. But a well formatted report requires some dirty manual jobs. Thus I decided to write a one liner and here it is
         <img alt=":)" class="wp-smiley" src="http://www.techtrunch.com/wp-includes/images/smilies/simple-smile.png" style="height: 1em; max-height: 1em;"/>
        </p>
        <p>
         <span style="font-family: monospace;">
         </span>
        </p>
        <pre class="brush:shell">
         ifconfig | grep -E "inet addr:" | grep -v 127.0.0.1 | awk -F":" 'BEGIN{ print "'"$HOSTNAME"'" } { print NR ")"$2 }' | awk '{ print $1 }'
        </pre>
        <p>
         <strong>
          8. Viewing important php security related settings with a one liner.
         </strong>
        </p>
        <p>
         During serversecurityaudits a Server administrator needs to make sure about certain php values/settings. But its really boring to open
         <code>
          php.ini
         </code>
         file of each server to check important values. For me, I cant think about it. But as a Server Administrator its a must to ensure about the security of the server. So I prepared a one liner to check certain set of php values.
        </p>
        <pre class="brush:shell">
         grep -E "display_errors =|register_globals =|^safe_mode =|^disable_functions =" `php -i | grep -E "Loaded Configuration File" | awk '{ print $5 }'`
        </pre>
        <p>
         <strong>
          Note :
         </strong>
         You can add your own values to this one line script. Its really very simple. :). If any one have more suggestions share with me.
        </p>
        <p>
         <strong>
          9. One liner for collecting hardware details including memory slots, processor information and hard disk.
         </strong>
        </p>
        <p>
         <strong>
         </strong>
        </p>
        <pre class="brush:shell">
         echo -e $HOSTNAME"\n===================\n\nIP_DETAILS\n==================="; ifconfig | grep -E "inet addr:" | grep -v 127.0.0.1 | awk -F":" '{ print NR ")"$2 }' | awk '{ print $1 }';echo -e "\nTOTAL NUMBER OF MEMORY SLOTS\n===================\n`dmidecode -t 17 | grep -c Size:`\n"; echo -e "MEMORY SLOT DETAILS\n===================\n `dmidecode -t 17 | grep Size:`"; free -m | grep Mem: | awk '{ print "\nTOTAL MEMORY\n===================\n"$2" MB" }'; echo -e "\nPROCESSOR DETAILS\n===================\n`cat /proc/cpuinfo | grep -E 'model name|cache size'`"; echo -e "\nTOTAL NUMBER OF HDDs\n===================\n`fdisk -l | grep -c "Disk /dev/"`"; echo -e "\nHDD DETAILS\n===================\n`fdisk -l | grep "Disk /dev/" | awk '{ print $2 " " $3" "$4 }' | sed 's/,$//g'`"
        </pre>
        <p>
         This one liner saved me from a .
         <img alt=":)" class="wp-smiley" src="http://www.techtrunch.com/wp-includes/images/smilies/simple-smile.png" style="height: 1em; max-height: 1em;"/>
         <br/>
         <em>
          Output:
         </em>
        </p>
        <div style="width: 100%; white-space: nowrap; overflow-x: scroll;">
         <a href="http://techtrunch.com/wp-content/uploads/2010/09/server_details.png">
          <img alt="" class="aligncenter size-full wp-image-555" height="586" src="http://techtrunch.com/wp-content/uploads/2010/09/server_details.png" title="server_details" width="640"/>
         </a>
        </div>
        <p>
         <strong>
          10. Find the IP address to which a domain is resolving [cPanel server]
         </strong>
        </p>
        <p>
         You can make use of this script for finding out the IPs to which all domains in a cPanel server is resolving. This script is very simple to understand. In this script domain names are grepping out from
         <code>
          /etc/trueuserdomains
         </code>
         file.
        </p>
        <pre class="brush:shell">
         #!/bin/bash
awk '{ print $1 }' /etc/trueuserdomains | cut -d : -f1 &gt; domains
awk 'BEGIN{FS=" ";printf("%-60s%-50s\n","DOMAIN","IP")}'
for i in `cat domains`
do
host $i | grep $i | grep address | awk '{printf("%-60s%-50s\n",$1,$4)}'
done
        </pre>
        <p>
         <strong>
          One liner for the above script
         </strong>
        </p>
        <p>
         <span style="font-family: monospace;">
         </span>
        </p>
        <pre class="brush:shell">
         awk {'print $1'} /etc/trueuserdomains | cut -d : -f1 &gt; domains; awk 'BEGIN{FS=" ";printf("%-60s%-50s\n","DOMAIN","IP")}'; for i in `cat domains`; do host $i | grep $i | grep address | awk '{printf("%-60s%-50s\n",$1,$4)}'; done
        </pre>
        <p>
        </p>
