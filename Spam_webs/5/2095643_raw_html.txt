       <h1 class="entry-title">
        Which Firefox is right for you -- 9, 10, 11, 12 or UX?
       </h1>

        <p>
         <img alt="" class="alignnone size-full wp-image-51407" height="432" src="http://betanews.com/wp-content/uploads/2011/12/Firefox-12-Nightly-e1324653964739.jpg" title="Firefox 12 Nightly" width="600"/>
        </p>
        <p>
         The Firefox development merry-go-round has moved on again, with
         <a href="http://www.downloadcrew.com/article/25727-firefox_beta" target="_blank">
          Firefox 10 Beta
         </a>
         and
         <a href="http://www.downloadcrew.com/article/21612-firefox_aurora" target="_blank">
          Firefox 11 Aurora
         </a>
         builds being joined by two separate versions of Firefox 12:
         <a href="http://www.downloadcrew.com/article/22815-firefox_nightly_32-bit" target="_blank">
          Firefox 12 Nightly
         </a>
         and
         <a href="http://www.downloadcrew.com/article/25064-firefox_ux" target="_blank">
          Firefox 12 UX
         </a>
         , the latter build representing the efforts of a team focussed on delivering improvements to the Firefox user interface.
        </p>
        <p>
         Despite the lack of any major revamps, theres enough new features and interface tweaks filtering through the various builds to make the next three planned versions worth looking out for. Get a heads up on whats coming and discover which build is best for your current needs with our updated guide to what the future holds in store for Firefox.
        </p>
        <div class="articleBoxad">
         <script type="text/javascript">
          //&lt;![CDATA[

        <p>
         <strong>
          <a href="http://www.downloadcrew.com/article/24333-firefox_9" target="_blank">
           Firefox 9.0.1 FINAL
           <br/>
          </a>
         </strong>
        </p>
        <p>
         This is the latest stable release of Firefox, and as such is recommended for most users, although the number of interesting new features appearing in later builds coupled with the lack of new features in this latest release make it something more experienced and impatient users might be tempted to skip.
        </p>
        <p>
         Firefox 9 contains only one change of note, and its an under-the-hood tweak at that. Nevertheless, support for type inference means web pages containing Javascript will load noticeably quicker than before. In addition, OS X Lion users will see another two features -- support for two-finger swiping when navigating, and better integration of Firefox themes -- worth checking out. Ultimately, though, version 9 wont go down in the annals of major Firefox releases.
        </p>
        <p>
         <strong>
          <a href="http://www.downloadcrew.com/article/25727-firefox_beta" target="_blank">
           Firefox 10 Beta
           <br/>
          </a>
         </strong>
        </p>
        <p>
         The most significant change Firefox 10 will bring to most users is its new assumption that most add-ons are compatible with an updated build unless the developer specifically says otherwise. This should lead to a more seamless update to new versions of Firefox, and full details of how the new system works can be read at this
         <a href="https://wiki.mozilla.org/Features/Add-ons/Add-ons_Default_to_Compatible" target="_blank">
          Mozilla Wiki article
         </a>
         .
        </p>
        <p>
         Theres also a change to the Launch bar, with the browser forward button disappearing from view.
        </p>
        <p>
         Web developers will be particularly pleased with Firefox 10, thanks to a number of major new tools, all accessible from the Firefox &gt; Web Developer menu. The most notable of these is an Inspect option that allows developers to visually inspect pages with the help of a Highlighter tool that makes checking a pages CSS and HTML code that much easier. This feature was originally planned for Firefox 4, but since its postponement has been significantly overhauled.
        </p>
        <p>
         <strong>
          <a href="http://www.downloadcrew.com/article/21612-firefox_aurora" target="_blank">
           Firefox 11.0a2 Aurora
          </a>
         </strong>
        </p>
        <p>
         <strong>
         </strong>
         Aurora is an alpha build of Firefox, which means its undergone minimum testing only. As such its not suitable for everyday use, which is why Firefox Aurora is installed as a separate build alongside the stable or beta build, allowing you to test its features without affecting your day-to-day browsing. Settings are shared between Firefox Aurora and your other build, however, so again caution should be exercised before installing it.
        </p>
        <p>
         Version 11 doesnt exhibit much in the way of major features -- a number slated for inclusion in Firefox 11 according to the
         <a href="https://wiki.mozilla.org/Features/Release_Tracking#Firefox_11" target="_blank">
          Features/Release Tracking page
         </a>
         arent present in this first alpha build. Those that are present include support for synchronizing Add-Ons in Firefox Sync, partial support for importing data (bookmarks, history and cookies at this early stage) from Google Chrome and two new tools for developers: a Style Editor for tweaking CSS settings on the fly, and a new 3D view (Tilt) for Page Inspector.
        </p>
        <p>
         <strong>
          <a href="http://www.downloadcrew.com/article/22815-firefox_nightly" target="_blank">
           Firefox 12.0a1 Nightly
           <br/>
          </a>
         </strong>
        </p>
        <p>
         Firefoxs Nightly channel gives users access to code hot off the press, but while youre looking at the latest bleeding-edge version of Firefox, youre also venturing into uncharted waters because much of this new code has had no testing at all. Nightly builds update regularly, so once installed youll find your build updating on a much more frequent basis than other unstable releases.
        </p>
        <p>
         At the time of writing, Firefox Nightly exhibits no new features above and beyond that of Aurora - but there are some interesting developments in the pipeline, as revealed on the
         <a href="https://wiki.mozilla.org/Features/Release_Tracking#Firefox_12" target="_blank">
          Features/Release Tracking page
         </a>
         of the Mozilla Wiki.
        </p>
        <p>
         These include a number of features ported across from the UX build (see below), including an integrated download manager instead of a separate pop-up window, plus full integration of Web Apps into the desktop. Also slated is renewed support for inline autocomplete, a feature ditched from version 4.
        </p>
        <p>
         Nightly is also available as a
         <a href="http://www.downloadcrew.com/article/25739-firefox_nightly" target="_blank">
          64-bit build
         </a>
         for compatible versions of Windows and Linux.
        </p>
        <p>
         Wed recommend all but developers and serious, knowledgeable enthusiasts avoid the Nightly builds of Firefox.
        </p>
        <p>
         <strong>
          <a href="http://www.downloadcrew.com/article/25064-firefox_ux" target="_blank">
           Firefox 12.0a1 UX
          </a>
         </strong>
        </p>
        <p>
         Introduced in our last update, Firefox UX (also available as a dedicated
         <a href="http://www.downloadcrew.com/article/25743-firefox_ux" target="_blank">
          64-bit build
         </a>
         for Windows and Linux) provides a parallel Nightly build of Firefox that concentrates on developing a new interface for Firefox. Some of the features previewed here have finally made the transition to the Aurora channel, meaning they will soon appear in the version of Firefox most people have installed on their machines.
        </p>
        <p>
         Other changes remain exclusive to UX for now, including the integrated download manager widget that appears in the top right-hand corner of the window, a rather rudimentary looking New Tab page, and some other more minor tweaks like the removal of Address and Search bars when running a Web App such as twitter.com via its own button.
        </p>
        <p>
         For an idea of what else UX may see appearing in the near future, check out this
         <a href="http://people.mozilla.com/~shorlander/ux-presentation/ux-presentation.html" target="_blank">
          Mozilla UX presentation
         </a>
         from August.
        </p>
        <p>
         <strong>
          In Summary
          <br/>
         </strong>
        </p>
        <p>
         Which version of Firefox should you try? Stick to the most stable version you feel comfortable with, although there are a number of new features filtering through versions 10, 11 and 12 to tempt even the most casual user into taking a sneak peek into the future and install a less stable build. And despite the lack of new features in version 12 of Firefox UX, if youve not yet previewed some of the changes to the Firefox user interface being developed, it still represents a tantalising glimpse into the future.
        </p>
        <p>
         Back up if you plan to try out Beta or Aurora builds of Firefox. And If you do decide to give the Nightly or UX builds a try, consider using a non-critical machine or virtual setup (try
         <a href="http://www.downloadcrew.com/article/17-virtualbox" target="_blank">
          VirtualBox
         </a>
         ) instead of your main computer, just in case
        </p>
