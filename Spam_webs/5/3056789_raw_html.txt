            <p class="first-text" data-textannotation-id="5006438a43d1344e5c3329bfd0b06b65">
             To a lot of non-developers, learning to code seems like an impossibly daunting task. However, thanks to a number of great resources that have recently been put online for free teaching yourself to code has never been easier. I started learning to code earlier this year and can say from experience that
             <strong>
             </strong>
             learning enough to build your own prototype is not as hard as it seems. In fact, if you want to have a functioning prototype within two months
             <em>
              without
             </em>
             taking a day off work, its completely doable.
            </p>
            <p data-textannotation-id="e01c682abfc5fa9c14076f4e4d0c4402">
             Below, Ive outlined a simple path from knowing nothing about software development to having a working prototype in eight weekends that roughly mirrors the steps I took.
            </p>
            <h3 data-textannotation-id="800f935f25ebf64968e13eda01734967">
             Introduce Yourself to the Web Stack (10 Minutes)
            </h3>
            <p data-textannotation-id="e33e426e2c2c17b05049f503454b95fc">
             The presence of unfamiliar terminology makes any subject seem more confusing than it actually is. Yipit founder/CEO
             <a href="http://viniciusvacanti.com/2010/11/01/6-things-you-need-to-learn-to-build-your-own-prototype/" target="_blank">
              Vin Vacanti has a great overview
             </a>
             of some of the key terms youll want to be familiar with in language youll understand.
            </p>
            <div class="ad-unit js_ad-unit hide-contents ad-mobile js_ad-mobile">
             <div class="ad-mobile-inner">
              <p class="ad-label proxima">
               <small class=" proxima">
                Advertisement
               </small>
              </p>
              <div class="ad-container js_ad-container" data-zone-type="MOBILE" id="ad-container-28208865">
              </div>
              <p class="ad-label-bottom">
              </p>
             </div>
            </div>
            <h3 data-textannotation-id="c9b391bfd41b13f60d428115c77215c2">
             Get an Introductory Grasp of Python and General Programming Techniques (1 Weekend)
            </h3>
            <ul>
             <li data-textannotation-id="20d19a18f7f4829c4c9fd082636d7d1c">
              <p data-textannotation-id="045ecfeae4230ad326f178619da563cb">
               <strong>
                <a href="http://learnpythonthehardway.org/book/" target="_blank">
                 Learn Python the hard way
                </a>
               </strong>
               : Despite the title, the straightforward format makes learning basic concepts really easy and most lessons take less than 10 minutes. However, I found that the format didnt work as well for some of the more advanced topics, so Id recommend stopping after lesson 42 and moving on.
              </p>
             </li>
             <li data-textannotation-id="f5814fb838118c5fe23d81416976fe39">
              <p data-textannotation-id="5e868524b41fb638f4c60c1980a09e21">
               <strong>
                <a href="http://code.google.com/edu/languages/google-python-class/introduction.html" target="_blank">
                 Googles Python class
                </a>
               </strong>
               : Read the notes and / or watch the videos and do all of the associated exercises until you get them right
               <em>
                without
               </em>
               looking at the answers. Struggling through the exercises I kept getting wrong was the best learning experience. I would have learned far less had I just looked at the answers and tried to convince myself that I understood the concepts.
              </p>
             </li>
            </ul>
            <p data-textannotation-id="a435613e7ad48c3996d8db8aa705999a">
             These two resources are somewhat substitutable and complementary. I recommend doing the first few lessons from both to see which you like better. Once youve finished one, skim through the other looking for concepts you arent fully comfortable with as a way to get some extra practice.
            </p>
            <h3 data-textannotation-id="041bcb10b336c47151342bdfcc574724">
             Get an Introductory Understanding of Django (1 Weekend)
            </h3>
            <ul>
             <li data-textannotation-id="093510a4cfb20cdf90deb1d7b6cc7ad4">
              <p data-textannotation-id="1817054fb8a8172d211cf4a3d9e952f8">
               Work through the
               <a href="https://docs.djangoproject.com/en/1.4/intro/tutorial01/" target="_blank">
                Django tutorial
               </a>
               .
              </p>
             </li>
             <li data-textannotation-id="e83b718434da50b042ee7b58c4f37ac4">
              <p data-textannotation-id="c714ce18439fe1f0b84f687dcceddb05">
               Delete all your code.
              </p>
             </li>
             <li data-textannotation-id="b114355e51e8db9ce3afb516592e9c52">
              <p data-textannotation-id="6c164e89d16c11508b9b2a79c565be98">
               Work through the tutorial again, from scratch.
              </p>
             </li>
            </ul>
            <p data-textannotation-id="5f5e8cdaf5cf10cc76d49d29da0f42e6">
             The first time I went through the tutorial I inevitably ended up just following the instructions step-by-step without really understanding what each step did since everything felt so new.
            </p>
            <p data-textannotation-id="09ae42663dee439946b4734f303bb264">
             The second time through I wasnt as focused on the newness of the concepts was better able to focus on understanding how all the parts work together.
            </p>
            <h3 data-textannotation-id="5be3a7287b41ff729d2bac9c5b54b268">
             Get a Deeper Understanding of Python/General Programming Concepts (2-4 Weekends)
            </h3>
            <ul>
             <li data-textannotation-id="c26e531e9c3c4703df7e62a893433b6e">
              <p data-textannotation-id="11420eacafc010f427c5fb8271c5c966">
               <strong>
                <a href="http://www.udacity.com/view#Course/cs101/CourseRev/apr2012/Unit/671001/Nugget/675002" target="_blank">
                 Udacitys intro CS class
                </a>
                :
               </strong>
               Udacitys courses are generally 7 session classes (2-3 hours per session) that you can at your own pace. (Im a huge fan of Udacitys pedagogy and recommend the
               <a href="http://www.udacity.com/view#Course/cs212/CourseRev/apr2012/Unit/3001/Nugget/3002" target="_blank">
                intermediate programming
               </a>
               class or the
               <a href="http://www.udacity.com/view#Course/cs253/CourseRev/apr2012/Unit/4001/Nugget/5002" target="_blank">
                web development
               </a>
               class as follow-ups to this two-month curriculum.)
              </p>
             </li>
             <li data-textannotation-id="6fd9dc008ff59ad2977aa4d6fb3539b7">
              <p data-textannotation-id="d0803bc7533ea70ff653f909a24645a1">
               <strong>
                <a href="http://ocw.mit.edu/courses/electrical-engineering-and-computer-science/6-00sc-introduction-to-computer-science-and-programming-spring-2011/" target="_blank">
                 Unit 1 of MITs intro CS course
                </a>
               </strong>
               : Really well taught and surprisingly approachable.
              </p>
             </li>
            </ul>
            <p data-textannotation-id="9e153a8fb70cb7ccbfc9beba235c53dd">
             Again, I would sample each and see which you like the best. I ended up doing both but that was probably overkill.
            </p>
            <h3 data-textannotation-id="ee7925c2d5db3330b9d3efc0b495e2f6">
             Practice Building Simple Web Applications (1 Weekend)
            </h3>
            <ul>
             <li data-textannotation-id="6f8b0f76042aaebbe029e1fde08fba41">
              Work through a few of the exercises in
              <a href="http://lightbird.net/dbe/" target="_blank">
               Django by example
              </a>
              . These exercises dont hold your hand quite as much as the Django tutorial but they still provide a fair bit of guidance so I found it to be a nice way to start taking the training wheels off.
             </li>
            </ul>
            <h3 data-textannotation-id="7b5f2cb191f66a86a25d6117c44bc75e">
             Build Your Prototype (1 Weekend)
            </h3>
            <ul>
             <li data-textannotation-id="6b6d2c8a5ffbc3978cd398e7387020a3">
              <a href="http://viniciusvacanti.com/2010/12/20/the-shortcut-we-took-to-build-yipit-in-three-days/" target="_blank">
               Build a prototype in just one weekend?
              </a>
              Yes, youll be embarrassed by what it looks like (I sure was) but
              <a href="http://startupquote.com/post/855482768" target="_blank">
               thats the whole point
              </a>
              .
             </li>
            </ul>
            <p data-textannotation-id="a9e70137876914168f2a62bca4db01a1">
             Thats it. Eight weekends (or less) and youve gone from zero to a functioning prototype. Not so daunting after all is it?
            </p>
            <p data-textannotation-id="cf647351c2344cfc12398b749d32458b">
             <em>
              Authors Note: It goes without saying that theres a huge difference between the relatively cursory amount of knowledge needed to build a simple prototype (the focus of this post) and the depth of knowledge and experience needed to be a truly qualified software engineer. If you want to learn all that it takes to build modern web applications at scale, getting professional web development experience at a fast-growing startup like
              <a href="http://yipit.com/" target="_blank">
               Yipit
              </a>
              is a great next step. If youre smart, hard-working, and passionate about creating amazing consumer web experiences drop us a line at
              <a href="https://mail.google.com/mail/?view=cm&amp;fs=1&amp;tf=1&amp;to=jobs@yipit.com" target="_blank">
               jobs@yipit.com
              </a>
              were always looking for great people to join our team.
             </em>
            </p>
