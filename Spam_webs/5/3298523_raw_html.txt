       <h1 class="entry-title">
        How to fix 400 Bad Request in Gmail after Offline Mode
       </h1>
       <div class="entry-meta">
        <span class="meta-prep meta-prep-author">
         Posted on
        </span>
        <a href="http://www.sampletheweb.com/2009/02/18/how-to-fix-400-bad-request-in-gmail-after-offline-mode/" rel="bookmark" title="7:12 pm">
         <span class="entry-date">
          2/18/2009
         </span>
        </a>
        <span class="meta-sep">
         by
        </span>
        <span class="author vcard">
         <a class="url fn n" href="http://www.sampletheweb.com/author/ck/" title="View all posts by C.K. Sample III">
          C.K. Sample III
         </a>
        </span>
       </div>
       <!-- .entry-meta -->
       <div class="entry-content">
        <p>
         <span class="drop_cap">
          G
         </span>
         mail Offline mode failed me for the second time today as I was testing it. For some reason, it keeps thinking that Im not online when I am. So I disabled it, but made the mistake of keeping all the offline files associated with it, thinking that I might turn it back on again later. I saved the settings and reloaded Gmail. It still didnt think I was connected and thought that Google Offline was still enabled. So I emptied my cache and restarted Safari.
        </p>
        <p>
         When I tried to launch Gmail again, I received a BAD REQUEST ERROR 400 message in large bold letters. When I loaded Gmail in Firefox it worked fine. So I emptied my cache and deleted my cookies and restarted Safari. Same error. Then I went to
         <em>
          Safari&gt;Google Gears Settings
         </em>
         and removed all the Google Gears information related with my Gmail account. I then flushed the cache and restarted Safari again. Same error. I then tried resetting Safari. Same error.
        </p>
        <p>
         If you encounter this error, heres how to fix it:
        </p>
        <p>
         <strong>
          Step 1:
         </strong>
         Locate the Google Gears database and permissions files on your computer. I found them via the information provided
         <a href="http://www.paulspoerry.com/2009/01/29/offline-gmail-is-it-secure/">
          in this very helpful blog post that talks about how Google Offline isnt really a secure option
         </a>
         . If you are using Safari on OS X, the files are located in this folder:
         <br/>
         <code>
          ~/Library/Application Support/Google/Google Gears for Safari
         </code>
         <br/>
         <strong>
          Step 2:
         </strong>
         Grab localserver.db and permissions.db inside this folder. Move them to the Trash.
         <br/>
         <strong>
          Step 3:
         </strong>
         Restart Safari (or whatever browser in which you were having this problem). When you try to login again it should work fine.
        </p>
        <p>
         So, evidently, the
         <em>
          Google Gears Settings
         </em>
         function doesnt really work at all for truly removing corrupt Google Gears databases. Fun! And none of Googles help documents explain any of this. Huzzah!
        </p>
        <p>
         Hope this helps!
        </p>
        <p>
         <strong>
          UPDATE:
         </strong>
         Alternate solution / version of the problem. So in addition to the above problem, this also appears to happen from time to time simply because of the cookies associated with Gmail and it is
         <a href="http://mail.google.com/support/bin/static.py?page=known_issues.cs">
          a known issue
         </a>
         that Google is currently ignoring, from the sounds of this:
        </p>
        <blockquote>
         <p>
          <strong>
           Error 400 Bad Request on Safari
          </strong>
          <br/>
          Users with Mac OSX 10.5.6 may see this error when accessing Gmail on Safari. Please reset Safari or use a different browser to access Gmail while the problem continues.
         </p>
        </blockquote>
        <p>
         Google advises resetting Safari, which is overkill. To fix it, simply open up your Preferences:
         <em>
          Safari&gt;Preferences
         </em>
         and then select
         <em>
          Security
         </em>
         and then choose
         <em>
          Show Cookies
         </em>
         . Select all the cookies beginning with mail.google.com (SHIFT+CLICK to select multiple) and then choose Remove. Reloading Gmail should work now, however, its not a permanent fix, so get used to repeating these steps several times a week until Google fixes this problem.
        </p>
        <p>
         <strong>
          Another Update:
         </strong>
         If you want to fully uninstall Google Gears from Safari, simply navigate to your Applications folder, then your Utilities folder and launch the Terminal application. At the command line type in:
         <br/>
         <code>
          /Library/Internet\ Plug-Ins/Gears.plugin/Contents/Resources/uninstall.command
         </code>
         <br/>
         Hit return and you will be prompted for your password. Enter your password and it will be removed. Restart Safari and youre good to go.
         <br/>
         <a href="http://i0.wp.com/media.sampletheweb.com/wp-content/uploads/byegooglegears.jpg">
          <img alt="bye google gears" class="aligncenter size-full wp-image-3955" data-recalc-dims="1" src="http://i0.wp.com/media.sampletheweb.com/wp-content/uploads/byegooglegears.jpg?resize=585%2C447" title="bye google gears"/>
         </a>
         <br/>
         <a href="http://gears.google.com/support/bin/answer.py?answer=79848&amp;topic=13200">
          Full Google Gears uninstallation instructions for Windows, OS X, Linux, and even Windows Mobile can be found here
         </a>
         .
        </p>
