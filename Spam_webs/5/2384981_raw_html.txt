     <h2>
      <span class="mw-headline" id="D.C3.A9veloppement">
       Dveloppement
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=1" title="Modifier la section : Dveloppement">

        modifier le code
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <ul>
      <li>
       Andree Buschmann initia en 1997 le dveloppement de Musepack. Le dveloppement sera repris plus tard par Franck Klemm.
       <span style="border-bottom: 1px solid blue;cursor:help;" title="Ce passage devrait prciser au minimum la dcennie ou l'anne.">
        Actuellement
       </span>
       <sup class="need_ref_tag" style="padding-left:2px; cursor:help;" title="Ce passage devrait prciser au minimum la dcennie ou l'anne.">
        <a href="/wiki/Aide:Quand" title="Aide:Quand">
         [Quand?]
        </a>
       </sup>
       , la Musepack Development Team (MDT) assure le dveloppement, avec l'assistance de Frank Klemm.
      </li>
      <li>
       Musepack est
       <a href="/wiki/Logiciel_libre" title="Logiciel libre">
        libre
       </a>
       , ainsi, le dveloppement est ouvert  tous.
      </li>
     </ul>

     <h3>
      <span class="mw-headline" id="Qualit.C3.A9.2C_rapidit.C3.A9">
       Qualit, rapidit
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=3" title="Modifier la section : Qualit, rapidit">
        modifier
       </a>
       <span class="mw-editsection-divider">
        |
       </span>
       <a href="/w/index.php?title=Musepack&amp;action=edit&amp;section=3" title="Modifier la section : Qualit, rapidit">
        modifier le code
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <ul>
      <li>
       Musepack est conu pour produire un encodage dit transparent (c'est--dire indiffrenciable du fichier original)  des dbits compris entre 160 et 180 kilobits par seconde.
      </li>
      <li>
       La compression et la dcompression du Musepack s'effectuent trs rapidement. Les excutables sont particulirement vloces sur des architectures
       <a href="/wiki/X86" title="X86">
        x86
       </a>
       ,
       <a href="/wiki/PowerPC" title="PowerPC">
        PowerPC
       </a>
       ,
       <a href="/wiki/Architecture_ARM" title="Architecture ARM">
        ARM
       </a>
       ou encore
       <a href="/wiki/Architecture_SPARC" title="Architecture SPARC">
        SPARC
       </a>
       , notamment la partie dcodage  travers la bibliothque libmpcdec (sous licence BSD). Nanmoins, sur des plates-formes plus exotiques, telles que celles supportes par
       <a href="/wiki/Rockbox" title="Rockbox">
        Rockbox
       </a>
       , cet avantage est moindre  cause de l'absence d'optimisations
       <a href="/wiki/Assembleur" title="Assembleur">
        assembleur
       </a>
       .
      </li>
     </ul>

       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=5" title="Modifier la section : Portabilit">

       <a href="/wiki/Portabilit%C3%A9_(informatique)" title="Portabilit (informatique)">

       <a href="/wiki/Foobar2000" title="Foobar2000">

       </a>
       ,
       <a href="/wiki/Winamp" title="Winamp">

       </a>

      </li>
     </ul>

      <span class="mw-headline" id="Format_des_m.C3.A9tadonn.C3.A9es_.28ou_.22tags.22.29">

      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=6" title='Modifier la section : Format des mtadonnes (ou "tags")'>

      <a href="/wiki/ID3" title="ID3">
       ID3v1
      </a>
      sont aussi supports, mais dconseills; les tags ID3v2 sont  proscrire (provoquent de nombreux problmes)
     </p>
     <h3>
      <span class="mw-headline" id="Extension_de_fichier">
       Extension de fichier
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=7" title="Modifier la section : Extension de fichier">
        modifier
       </a>

        ]
       </span>
      </span>
     </h3>
     <p>
      Les fichiers audio obtenus avec Musepack portent l'extension "
      <i>
       .mpc
      </i>
      ". On peut aussi rencontrer les extensions "
      <i>
       .mpp
      </i>
      " ou "
      <i>
       .mp+
      </i>
      ", mais cela est trs rare. En effet, Musepack tait auparavant appel MPEGplus, MPEG+ ou bien encore MP+.
     </p>

      </a>
      , l'encodage en Musepack s'effectue avec mppenc.exe, un utilitaire en ligne de commande. Celui-ci peut tre utilis de 3 manires diffrentes:
     </p>
     <ul>
      <li>
       tel quel, en ligne de commande
      </li>
      <li>
       depuis un
       <a class="mw-redirect" href="/wiki/Environnement_graphique" title="Environnement graphique">
        GUI
       </a>
       spcifique, par exemple MuseDrop
      </li>
      <li>
       appel par un logiciel externe: Exact Audio Copy, CDex...
      </li>
     </ul>
     <p>
      Le fonctionnement est similaire sur les autres
      <a href="/wiki/Syst%C3%A8me_d%27exploitation" title="Systme d'exploitation">
       systmes d'exploitation
      </a>
      .
     </p>
     <p>
      Sous
      <a href="/wiki/GNU/Linux" title="GNU/Linux">
       GNU/Linux
      </a>
      , Il est possible galement d'encoder/dcoder sous ce format en utilisant par exemple
      <a class="new" href="/w/index.php?title=DBPowerAmp&amp;action=edit&amp;redlink=1" title="DBPowerAmp (page inexistante)">
       dBPowerAmp
      </a>
      , par le biais de
      <a href="/wiki/Wine" title="Wine">
       Wine
      </a>
      . dBPowerAmp est conu pour Windows, mais est parfaitement fonctionnel sous Linux en passant par Wine, et s'utilise de faon pratiquement transparente, et en lui adjoignant les codecs disponibles sur le site d'
      <a class="new" href="/w/index.php?title=Illustrate&amp;action=edit&amp;redlink=1" title="Illustrate (page inexistante)">
       illustrate
      </a>
      , il obtient les mmes fonctionnalits, et cre des fichiers *.mpc de qualit identique  ceux crs sous Windows.
     </p>
     <h3>
      <span class="mw-headline" id="Param.C3.A8tres">
       Paramtres
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=10" title="Modifier la section : Paramtres">

      </span>
     </h3>
     <h4>
      <span class="mw-headline" id="G.C3.A9n.C3.A9ralit.C3.A9s">
       Gnralits
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=11" title="Modifier la section : Gnralits">
        modifier
       </a>
       <span class="mw-editsection-divider">
        |
       </span>
       <a href="/w/index.php?title=Musepack&amp;action=edit&amp;section=11" title="Modifier la section : Gnralits">
        modifier le code
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h4>
     <p>
      Un encodage s'effectue avec la commande suivante:
     </p>
     <pre>
      mppenc.exe [--quality n] input.wav [output.mpc]
     </pre>
     <p>
      "n" est le facteur de qualit (voir ci-dessous). Si l'option "--quality n" n'est pas prcise, le paramtre par dfaut, soit "--quality 5", est utilis.
     </p>
     <p>
      "output.mpc" est le nom du fichier obtenu. Si cette option n'est pas prcise, c'est le nom du fichier d'entre qui est utilis, avec l'extension "
      <i>
       .mpc
      </i>
      " (ici, nous obtiendrions "input.mpc").
     </p>
     <h4>
      <span class="mw-headline" id="R.C3.A9glage_de_la_qualit.C3.A9">
       Rglage de la qualit
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=12" title="Modifier la section : Rglage de la qualit">
        modifier
       </a>
       <span class="mw-editsection-divider">
        |
       </span>
       <a href="/w/index.php?title=Musepack&amp;action=edit&amp;section=12" title="Modifier la section : Rglage de la qualit">
        modifier le code
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h4>
     <p>
      Il est conseill de choisir comme facteur de qualit un entier compris entre 5 et 7. Soit:
     </p>
     <ul>
      <li>
       --quality 5 (auparavant --standard): bitrate d'environ 170 kb/s, variant entre 142 et 184 kb/s
      </li>
      <li>
       --quality 6 (auparavant --xtreme): bitrate d'environ 200 kb/s, variant entre 168 et 212 kb/s
      </li>
      <li>
       --quality 7 (auparavant --insane): bitrate d'environ 220 kb/s
      </li>
     </ul>
     <p>
      Dans la quasi-totalit des cas, le profil "standard" ne provoque que peu d'altrations audibles du signal audio. Les audiophiles exigeants peuvent utiliser les profils "xtreme" et "insane". De tels profils peuvent galement s'avrer utiles si le fichier est rencod par la suite.
     </p>
     <p>
      Musepack n'est pas conu pour les profils infrieurs  "standard" (par exemple "--quality 4", soit "--radio"). Cependant, ses performances avec de tels profils demeurent plutt bonnes.
     </p>
     <h4>
      <span class="mw-headline" id="Pr.C3.A9vention_du_.22clipping.22">
       Prvention du "clipping"
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=13" title='Modifier la section : Prvention du "clipping"'>
        modifier
       </a>
       <span class="mw-editsection-divider">
        |
       </span>
       <a href="/w/index.php?title=Musepack&amp;action=edit&amp;section=13" title='Modifier la section : Prvention du "clipping"'>
        modifier le code
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h4>
     <p>
      L'option "--xlevel" est une option permettant d'viter le "clipping" (saturation du signal sonore). Depuis la version 1.15s de mppenc, cette option est active par dfaut. Il n'y a donc plus lieu de l'ajouter  la ligne de commande.
     </p>
     <h4>
      <span class="mw-headline" id="Exemple">
       Exemple
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a class="mw-editsection-visualeditor" href="/w/index.php?title=Musepack&amp;veaction=edit&amp;vesection=14" title="Modifier la section : Exemple">
        modifier
       </a>
       <span class="mw-editsection-divider">
        |
       </span>
       <a href="/w/index.php?title=Musepack&amp;action=edit&amp;section=14" title="Modifier la section : Exemple">
        modifier le code
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h4>
     <p>
      Ces 2 lignes produisent le mme rsultat:
     </p>
     <pre>
      mppenc.exe --quality 7 musique.wav musique.mpc
     </pre>
     <pre>
      mppenc.exe --insane musique.wav
     </pre>
