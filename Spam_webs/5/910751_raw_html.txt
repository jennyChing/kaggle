          <a href="https://www.faludi.com/examples/xbee-api-library-for-processing/" rel="bookmark" title="Permanent Link to XBee API Library for Processing">
           XBee API Library for Processing
          </a>
         </h1>
        </div>
        <div class="entry-content">
         <ul>
          <li>
           <span style="color: #ff0000;">
            NOTE:
           </span>
           <em>
            In my
           </em>
           <a href="http://oreilly.com/catalog/9780596807740">
            <em>
             new book
            </em>
           </a>
           <em>
            Ive focused on using the more complete
           </em>
           <a href="http://code.google.com/p/xbee-api/">
            <em>
             XBee-API libraries
            </em>
           </a>
           <em>
            for Java created by Andrew Rapp. They cover both the Series 1 and Series 2 radios, providing a full suite of API interactions. Code examples that use these libraries are available on the
           </em>
           <a href="/bwsn">
            <em>
             Building Wireless Sensor Networks page
            </em>
           </a>
           <em>
            .
           </em>
          </li>
         </ul>
         <p>
          <a href="http://shiffman.net/">
           Dan Shiffman
          </a>
          and I developed a
          <a href="http://processing.org">
           Processing
          </a>
          library for Digis
          <a href="http://www.digi.com/xbee">
           XBee
          </a>
          radios.
          <a href="http://tigoe.net">
           Tom Igoe
          </a>
          also contributed code. The library facilitates receiving multiple sample I/O packets in API mode (ATAP1) from both the 802.15.4 and Series 2 XBee radios, and returns an object that contains the analog values, digital values, senders address and RSSI values. We started with I/O frames because the only way to receive this information is via API mode. The ability to send remote AT commands is now included. The library has been tested on Mac OS X and Windows platforms.
         </p>
         <p>
          <strong>
           DOWNLOAD
          </strong>
         </p>
         <ul>
          <li>
           <a href="http://dam.mellis.org/xbee-api-library-for-processing-1.5.zip">
            XBee API Library 1.5
           </a>
           for Processing
           <br/>
           includes
           <a href="http://dam.mellis.org/2009/11/updated_xbee_library_for_processing/">
            modifications
           </a>
           by David Mellis
          </li>
          <li>
           <a href="http://faludi.com/downloads/xbee/XBee_Processing_API_v1.4/xbee.zip?c07118">
            XBee API Library 1.4
           </a>
           for Processing
           <br/>
           <em>
            including Example and new File Display Example
           </em>
           <br/>
           (posted January 2009)
          </li>
          <li>
           <a href="http://www.faludi.com/downloads/xbee/XBee_Processing_API_v1.3/xbee.zip?c07118">
            XBee API LIbrary 1.3
           </a>
           for Processing
          </li>
          <p>
           (posted November 2008)
          </p>
          <li>
           <a href="http://www.faludi.com/downloads/xbee/XBee_Processing_API_v1.3/XbeeApiExample13.zip?c07118">
            XbeeApiExample13.zip

          </li>
         </ul>

            XBee API 1.2 Example

         </ul>
         <p>
          <strong>
           DOCUMENTATION
          </strong>
         </p>
         <p>
          <strong>
           XBeeReader
          </strong>
          class
         </p>
         <p>
          XBeeReader constructor takes the parent PApplet this and a reference to a made serial port
         </p>
         <pre>
          port = new Serial(this, Serial.list()[0], 9600);
         </pre>
         <pre>
          xbee = new XBeeReader(this,port);
         </pre>
         <p>
          start()  takes in the string of your setup commands and returns a string of responses. The first command in the string must begin with AT. Any commands that follow must be separated by commas and do not use AT. There is no comma after the last command.
         </p>
         <pre>
          println("Setting up Xbee"); String response = xbee.start("ATRE,ID3333,MY89,DH0,DL0");
         </pre>
         <pre>
          println("Setup response: " + response);
         </pre>
         <p>
          getXbeeReading()  returns the XBeeDataFrame object
         </p>
         <pre>
          XBeeDataFrame data = xbee.getXBeeReading();
         </pre>
         <p>
          <strong>
           XBeeDataFrame
          </strong>
          class
         </p>
         <p>
          Contains the information delivered in an XBee data frame. IN THIS INITIAL EXAMPLE IT IS ONLY I/O DATA. The available methods are:
         </p>
         <p>
          getAddress16()  returns the 16-bit transmitter address as an integer
          <br/>
          getAddress64()  returns the 64-bit transmitter address as an long
          <br/>
          getRSSI()  returns the RSSI reading in dBm as an integer (XBee Series 1 only)
          <br/>
          getTotalSamples()  returns the total number of samples contained in the data frame
          <br/>
          getDigital()  returns an array of integers that represent the current state of each digital channel with -1 indicating that the channel is not configured for digital. Use this when there is only one sample per frame.
          <br/>
          getDigital(int
          <em>
           n
          </em>
          )  returns the
          <em>
           n
          </em>
          th sample of digital data as an array of integers with -1 indicating that the channel is not configured for digital.
         </p>
         <p>
          getAnalog()  returns an array of integers that represents the current state of each analog channel with -1 indicating that the channel is not configured for analog. Use this when there is only one sample per frame.
          <br/>
          getAnalog(int
          <em>
           n
          </em>
          )  returns the
          <em>
           n
          </em>
          th sample of analog data as an array of integers with -1 indicating that the channel is not configured for analog.
         </p>
         <pre>
          int addr = data.getAddress16();
         </pre>
         <pre>
          int rssi = data.getRSSI();
         </pre>
         <pre>
          int[] digital = data.getDigital();
         </pre>
         <pre>
          int[] analog = data.getAnalog();
         </pre>
         <p>
          <strong>
           XBeeEvent
          </strong>
          function
         </p>
         <p>
          This function is similar to SerialEvent. The XBeeEvent is called for you when there is data available from your XBee radio.
         </p>
         <pre>
          public void xBeeEvent(XBeeReader xbee) {
         </pre>
         <pre>
          XBeeDataFrame data = xbee.getXBeeReading();
         </pre>
         <pre>
          }
         </pre>
