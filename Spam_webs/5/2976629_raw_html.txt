     <h1>
      HTML &lt;div&gt; Tag
     </h1>
     <div class="ad">
      <script async="" src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js">
      </script>
      <!-- Q Responsive Top -->
      <ins class="adsbygoogle" data-ad-client="ca-pub-6331358926293806" data-ad-format="auto" data-ad-slot="2095624553" style="display:block">

     </div>
     <p>
      The HTML
      <code>
       &lt;div&gt;
      </code>
      tag represents a generic section within an HTML document. The
      <code>
       &lt;div&gt;
      </code>
      tag enables you to group sections of HTML elements together and format them with CSS.
     </p>
     <p>
      The
      <code>
       &lt;div&gt;
      </code>
      element is similar to the
      <a href="/html/tags/html_span_tag.cfm">
       <code>
        &lt;span&gt;
       </code>
      </a>
      element, however the
      <a href="/html/tags/html_span_tag.cfm">
       <code>
        &lt;span&gt;
       </code>
      </a>
      element is used with inline elements, whereas the
      <code>
       &lt;div&gt;
      </code>
      element is used with block-level elements.
     </p>
     <p>
      The
      <code>
       &lt;div&gt;
      </code>
      element accepts "flow content", which refers to most elements that can appear inside a document's body.
     </p>
     <p>
      The
      <code>
       &lt;div&gt;
      </code>
      should only be used as a last resort when there isn't another suitable HTML element to use. HTML5 has introduced a number of new elements that can (and should) be used in place of the
      <code>
       &lt;div&gt;
      </code>
      . These include
      <a href="/html/tags/html_article_tag.cfm">
       <code>
        &lt;article&gt;
       </code>
      </a>
      ,
      <a href="/html/tags/html_aside_tag.cfm">
       <code>
        &lt;aside&gt;
       </code>
      </a>
      ,
      <a href="/html/tags/html_header_tag.cfm">
       <code>
        &lt;header&gt;
       </code>
      </a>
      , and
      <a href="/html/tags/html_footer_tag.cfm">
       <code>
        &lt;footer&gt;
       </code>
      </a>
      and others. See below under "Differences Between HTML 4 &amp; HTML 5" for more information on this.
     </p>
     <h2>
      Syntax
     </h2>
     <p>
      The
      <code>
       &lt;div&gt;
      </code>
      tag is written as
      <code>
       &lt;div&gt;
      </code>
      <code>
       &lt;/div&gt;
      </code>
      with the generic content inserted between the start and end tags.
     </p>
     <p>
      Like this:
     </p>

      Examples

      Basic tag usage
     </h3>
     <p>
      As you can see, simply placing content inside a
      <code>
       &lt;div&gt;
      </code>
      tag doesn't really do anything or change the content's appearance. It simply acts as a container that holds the enclosed content.
     </p>

     <h3>
      Applying Styles
     </h3>
     <p>
      One of the most common reasons for using the
      <code>
       &lt;div&gt;
      </code>
      element is to apply styles. You can apply styles to the element itself, or to any element enclosed within it.
     </p>
     <h4>
      Inline Styles
     </h4>
     <p>
      You can style an element using
      <a href="/css/inline_style_sheets.cfm">
       inline styles
      </a>
      , where you apply the styles via the
      <code>
       style
      </code>
      attribute.
     </p>

     <h4>
      Embedded &amp; External Styles
     </h4>
     <p>
      You can also style an element using either
      <a href="/css/embedded_style_sheets.cfm">
       embedded styles
      </a>
      or an
      <a href="/css/external_style_sheets.cfm">
       external style sheet
      </a>
      .
     </p>
     <p>
      Here's an example of using embedded styles and applying them via the
      <code>
       class
      </code>
      attribute of the
      <code>
       &lt;div&gt;
      </code>
      tag:
     </p>

     <h2>
      Differences Between HTML 4 &amp; HTML 5
     </h2>
     <p>
      In HTML 4.01 documents, the
      <code>
       &lt;div&gt;
      </code>
      tag was often used for specifying the various navigational sections of the HTML document (such as the header, footer, content area, side bars, etc).
     </p>
     <p>
      The HTML 5 specification has introduced a number of new elements that can (and should) be used instead of the
      <code>
       &lt;div&gt;
      </code>
      element. Examples of these new elements include
      <a href="/html/tags/html_article_tag.cfm">
       <code>
        &lt;article&gt;
       </code>
      </a>
      ,
      <a href="/html/tags/html_aside_tag.cfm">
       <code>
        &lt;aside&gt;
       </code>
      </a>
      ,
      <a href="/html/tags/html_header_tag.cfm">
       <code>
        &lt;header&gt;
       </code>
      </a>
      , and
      <a href="/html/tags/html_footer_tag.cfm">
       <code>
        &lt;footer&gt;
       </code>
      </a>
      , as well as others.
     </p>
     <p>
      Therefore, the
      <code>
       &lt;div&gt;
      </code>
      element should generally be used as an extension mechanism which is used only if there isn't another suitable HTML element to use.
     </p>
     <p>
      To see more detail on the two versions see
      <a href="http://www.quackit.com/html_5/tags/html_div_tag.cfm" title="HTML5 version of this element">
       HTML5
       <code>
        &lt;div&gt;
       </code>
       Tag
      </a>
      and
      <a href="http://www.quackit.com/html/html_4/tags/html_div_tag.cfm" title="HTML4 version of this element">
       HTML4
       <code>
        &lt;div&gt;
       </code>
       Tag
      </a>
      . Also check out the links to the official specifications below.
     </p>

       &lt;div&gt;
      </code>
      element.
     </p>
     <ul>

      </li>

       (W3C)
      </li>

     <h3>
      What's the Difference?
     </h3>
     <p>
      W3C creates "snapshot" specifications that don't change once defined. So the HTML5 specification won't change once it becomes an official recommendation. WHATWG on the other hand, develops a "living standard" that is updated on a regular basis. In general, you will probably find that the HTML living standard will be more closely aligned to the current W3C draft than to the HTML5 specification.
     </p>
