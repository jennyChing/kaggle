       <h1 class="headline" itemprop="name">
        HowTo: Extract an RPM Package Files Without Installing It
       </h1>
       <div class="byline small">
        <span class="post_author_intro">
         by
        </span>
        <span class="post_author" itemprop="author">
         Vivek Gite
        </span>
        <meta content="2006-10-18" itemprop="datePublished"/>

      <div class="post_content" itemprop="articleBody">
       <div style="float:right;margin-top:0px;margin-left:5px;">
        <a href="http://www.cyberciti.biz/tips/category/redhatfedora-linux" title="See all Redhat/CentOS/Fedora Core related tips/articles">

       <p>
        <span class="drop_cap">
         A
        </span>
        s most of you may know to how extract a
        <a href="http://www.cyberciti.biz/faq/how-do-i-compress-a-whole-linux-or-unix-directory/">
         tarballs
        </a>
        and/or a
        <a href="http://www.cyberciti.biz/tips/how-can-i-zipping-and-unzipping-files-under-linux.html">
         zip files
        </a>
        . Someone, recently PM me with a question:
       </p>
       <p>
        <em>
         Dear nixCraft,
        </em>
       </p>
       <p>
        How do I extract an RPM package file without installing it on my Fedora Linux or CentOS or RHEL (Red Hat Enterprise Linux) Suse Linux?
       </p>
       <p>
        Sincerely,
       </p>
       <p>
        CentOS user.
        <br/>
        <br/>
        <span id="more-541">
        </span>
       </p>
       <h2>
        Extracting rpm file command
       </h2>
       <p>
        To be frank there is no direct option available for rpm command to extract an RPM file. But there is a small nifty utility available called rpm2cpio. It rxtract cpio archive from RPM Package Manager (RPM) package. With the following hack you will be able to extract an RPM file.
       </p>
       <p>
        First you use
        <kbd>
         rpm2cpio
        </kbd>
        to convert  the .rpm file into a cpio archive on standard out. If a
        <kbd>
         -
        </kbd>
        argument is given, an rpm stream is read from standard in. The basic syntax is as follows:
       </p>
       <pre class="bash">
        pm2cpio myrpmfile.rpm
rpm2cpio - &lt; myrpmfile.rpm
rpm2cpio myrpmfile.rpm |
        <span style="color: #c20cb9; font-weight: bold;">
         cpio
        </span>
        -idmv
       </pre>
       <h3>
        Examples - Extract files from rpm
       </h3>
       <p>
        Download an RPM file:
        <br/>
        <code>
         $ mkdir test
         <br/>
         $ cd test
         <br/>
         $ wget http://www.cyberciti.biz/files/lighttpd/rhel4-php5-fastcgi/php-5.1.4-1.esp1.x86_64.rpm
         <br/>
        </code>
        <br/>
        To extract RPM file using rpm2cpio and
        <a href="http://www.cyberciti.biz/faq/how-do-i-use-cpio-command-under-linux/">
         cpio command
        </a>
        , type:
        <br/>
        <code>
         $ rpm2cpio php-5.1.4-1.esp1.x86_64.rpm | cpio -idmv
        </code>
        <br/>
        Sample outputs:
       </p>
       <pre>
        /etc/httpd/conf.d/php.conf
./etc/php.d
./etc/php.ini
./usr/bin/php
./usr/bin/php-cgi
./usr/lib64/httpd/modules/libphp5.so
./usr/lib64/php
./usr/lib64/php/modules
....
.....
..
./var/lib/php/session
./var/www/icons/php.gif
19188 blocks
       </pre>
       <p>
        In this example, output of rpm2cpio command piped to
        <a href="http://www.cyberciti.biz/faq/how-do-i-use-cpio-command-under-linux/">
         cpio command
        </a>
        with following options:
       </p>
       <ul>
        <li>
         <kbd>
          i
         </kbd>
         : Restore archive
        </li>
        <li>
         <kbd>
          d
         </kbd>
         : Create leading directories where needed
        </li>
        <li>
         <kbd>
          m
         </kbd>
         : Retain previous file modification times when creating files
        </li>
        <li>
         <kbd>
          v
         </kbd>
         : Verbose i.e. display progress
        </li>
       </ul>
       <p>
        Verify that you have extracted an RPM file in the current directory:
        <br/>
        <code>
         $ ls
        </code>
        <br/>
        Sample outputs:
       </p>
       <pre>
        etc  php-5.1.4-1.esp1.x86_64.rpm  usr  var
       </pre>
       <h2>
        Say hello to Midnight  Commander
       </h2>
       <p>
        GNU Midnight Commander (mc) is  a  directory  browser/file  manager  for Unix-like operating systems. Install mc using the following
        <a href="http://www.cyberciti.biz/faq/rhel-centos-fedora-linux-yum-command-howto/" title="See Linux/Unix yum command examples for more info">
         yum command
        </a>
        :
        <br/>
        <code>
         # yum install mc
        </code>
       </p>
       <h3>
        Opening an RPM file using Midnight Commander (mc)
       </h3>
       <p>
        You can use mc command as follows to browse or extract rpm files:
        <br/>
        <code>
         # mc
        </code>
        <br/>
        Sample outputs:
        <br/>
       </p>
       <div class="wp-caption aligncenter" style="width: 608px">
        <a href="http://s0.cyberciti.org/images/tips/2013/12/mc-1.png">
         <img alt="Fig.01: Browsing / viewing an rpm file with GNU Midnight Commander " class="" height="358" src="http://s0.cyberciti.org/images/tips/2013/12/mc-1.png" width="598"/>
        </a>
        <p class="wp-caption-text">
         Fig.01: Browsing / viewing an rpm file with GNU Midnight Commander
        </p>
       </div>
       <p>
        Next select an RPM file (such as php-5.3*.rpm) by highlighting the package name and press Enter key. You need to select
        <kbd>
         CONTENTS.cpio
        </kbd>
        file:
        <br/>
       </p>
       <div class="wp-caption aligncenter" style="width: 608px">
        <a href="http://s0.cyberciti.org/images/tips/2013/12/mc-2.png">
         <img alt="Fig.02: The actual php rpm files are contained in the CONTENTS.cpio file" class="" height="362" src="http://s0.cyberciti.org/images/tips/2013/12/mc-2.png" width="598"/>
        </a>
        <p class="wp-caption-text">
         Fig.02: The actual php rpm files are contained in the CONTENTS.cpio file
        </p>
       </div>
       <br/>
       To view/edit/extract files click or press on the special function keys. In this example, I pressed
       <kbd>
        F3
       </kbd>
       function key to view php.conf file:
       <br/>
       <div class="wp-caption aligncenter" style="width: 608px">
        <a href="http://s0.cyberciti.org/images/tips/2013/12/mc-3.png">
         <img alt="Fig.03: Viewing php.conf file" class="" height="359" src="http://s0.cyberciti.org/images/tips/2013/12/mc-3.png" width="598"/>
        </a>
        <p class="wp-caption-text">
         Fig.03: Viewing php.conf file
        </p>
       </div>
       <h2>
        Conclusion
       </h2>
       <p>
        I hope you will find these tips useful to extract configuration file or other file without installing an RPM file.

          postinstall scriptlet (using /bin/sh):
#
# upgrade rsnapshot config file
#
RSNAPSHOT_CONFIG_VERSION=`/usr/bin/rsnapshot check-config-version`
if test $? != 0; then
        echo "Error upgrading /etc/rsnapshot.conf"
fi
if test "$RSNAPSHOT_CONFIG_VERSION" = "1.2"; then
        # already latest version
        exit 0
fi
if test "$RSNAPSHOT_CONFIG_VERSION" = "unknown"; then
        /usr/bin/rsnapshot upgrade-config-file
        RETVAL=$?
        exit $RETVAL
fi
