        <h3>
         10 Simple steps to transfer WordPress to new Server or Host
        </h3>
        <p>
         <br/>
        </p>
        <h5>
         Step #1 Prepare the new host space
        </h5>
        <p>
         Create a directory on your new hosting space where your blog has to be moved to. If your new host supports multiple domain hosting, you might want to
         <strong>
          add an addon domain
         </strong>
         using the cPanel. As much as possible, try to
         <strong>
          use the same directory name
         </strong>
         as your existing blog installation directory in order to simplify the moving process.
        </p>
        <h5>
         Step #2 Do a fresh WordPress installation
        </h5>
        <p>
         Now use your new hosts
         <strong>
          cPanel Fantastico tool
         </strong>
         (or the
         <a href="http://codex.wordpress.org/Installing_WordPress#Famous_5-Minute_Install">
          famous 5 minute install
         </a>
         ) to do a fresh WordPress installation.
        </p>
        <p>
         You have to make sure that:
        </p>
        <ul>
         <li>
          The new WordPress
          <strong>
           release version is the same or higher
          </strong>
          than your  current WP release
         </li>
         <li>
          Use the
          <strong>
           same admin panel login &amp; password
          </strong>
          as your current installation, to simplify things
         </li>
         <li>
          Use the
          <strong>
           temporary WordPress installation URL
          </strong>
          to fill the
          <strong>
           WordPress address (URL)
          </strong>
          parameter in your
          <strong>
           <a href="http://www.dollarshower.com/hotdeal/managewp" rel="nofollow" style="" target="_blank">
            WordPress Admin
           </a>
           =&gt;Settings=&gt;General
          </strong>
          page. Also, change the
          <strong>
           Blog address (URL)
          </strong>
          accordingly (e.g. something like http://YourHostServerIPAddress/YourAccount/YourDomain.com/)
         </li>
        </ul>
        <p>
         Visit your new blog installation using the Blog address (URL) set above and make sure that the new WordPress installation (without your posts and theme) works fine. In order to avoid any accidental overwriting, you may want to
         <strong>
          keep a backup of your new wp-config.php
         </strong>
         in a safe place.
        </p>
        <h5>
         Step #3 Upgrade your current WordPress installation, if needed
        </h5>
        <p>
         From Step #2, if you installed a newer release of WordPress on your new host, make sure that your
         <strong>
          current WordPress installation is upgraded to that release
         </strong>
         via the standard upgrade process. It is better to complete the upgrade activities on your current playing field than on a new server and host. After the upgrade
         <strong>
          make sure that your upgraded
          <a href="http://www.dollarshower.com/hotdeal/wordpresswizard" rel="nofollow" style="" target="_blank">
           WordPress blog
          </a>
          and all your plugins work fine
         </strong>
        </p>
        <p>
         (This step can be avoided if you decide to keep the target (new) WordPress release the same as your current one from Step #2. But this may not be possible when you use)
        </p>
        <h5>
         Step #4 Backup your current wp-content directory
        </h5>
        <p>
         Use your favorite FTP program (like the free
         <a href="http://filezilla-project.org/" rel="nofollow">
          FileZilla
         </a>
         FTP tool) to backup the whole
         <strong>
          wp-content directory
         </strong>
         . In addition, you have to backup your
         <strong>
          robot.txt, wp-config.php and .htaccess files
         </strong>
         . Backup any other folders (e.g. download, temp, junk etc) and files that you might have created as well.
        </p>
        <h5>
         Step #5 Backup your current WordPress database
        </h5>
        <p>
         Use a tool such as
         <strong>
          phpMyAdmin
         </strong>
         (mostly available in your cPanel) to
         <strong>
          export your current WordPress database
         </strong>
         .
         <strong>
          Do not use the export feature in your
          <a href="http://www.dollarshower.com/hotdeal/cmsc" rel="nofollow" style="" target="_blank">
           WordPress admin
          </a>
          panel
         </strong>
         as this would not take all your current settings but only the content (basically posts, pages, comments etc).
        </p>
        <p>
         While using phpMyAdmin export, please make sure that you select:
        </p>
        <ul>
         <li>
          All your WordPress tables and
          <strong>
           SQL
          </strong>
          as the Export option
         </li>
         <li>
          Add DROP TABLE/VIEW/PROCEDURE/FUNCTION option
         </li>
         <li>
          Save as File
         </li>
        </ul>
        <p>
         Now, when you hit Go you will be prompted to
         <strong>
          save the database backup file on to your disk
         </strong>
         .
        </p>
