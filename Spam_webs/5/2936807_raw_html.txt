            <div class="tile-content">
             <strong>
              ScrollMagic
             </strong>
             is a
             <a href="http://jquery.com/">
              jQuery
             </a>
             plugin which lets you use the scrollbar like a playback scrub control. Using this, you can build some
             <strong>
              extremely beautiful
             </strong>
             landing pages and websites. Normally, we wouldnt do a tutorial on using a single jQuery plugin, but scrollMagic does
             <strong>
              a lot
             </strong>
             and has quickly become one of my favorite plugins. In this article, Ill cover my general opinion on scroll plugins, how to get started with ScrollMagic, and some basic and over-the-top creative demos. Finally, some starter templates for you to repurpose are in works to be released very soon in part II of this tutorial.

              A Note About a Users Scroll

             <p>
              Im

              </strong>
              of hijacking a users scroll period. I personally believe its way to easy to ruin a users experience and it makes it difficult to quickly navigate to specific content. It takes a lot for me to consider using a jQuery plugin that heavily affects normal scroll behavior. ScrollMagic doesnt really hijack a users scroll despite its name alluding to the idea that it would. It simply just triggers a bunch of events during a users scroll. For example, compare these two sites:

             <h2>
              The ScrollMagic Controller
             </h2>
             <p>
              Typically when you initiate a jQuery plugin you just pass a bunch of options and call it a day. Sometimes a plugin will have advanced features like a
              <a href="http://kenwheeler.github.io/slick/#settings">
               callbacks API
              </a>
              or the ability to
              <a href="http://panels.scotch.io/#dev-way">
               return the entire plugin as an object
              </a>
              with some public functions so you can get real custom with it.
             </p>
             <p>
              ScrollMagic is a little bit different than this. Were going to initiate a ScrollMagic Controller, create a bunch of animation objects, create some Scene (where the animation happens) objects, combine the animation and scene objects, then pass it all back to the main ScrollMagic Controller. So our general steps will be:
             </p>
             <ol>
              <li>
               <strong>
                Create the ScrollMagic Controller
               </strong>
               (and select general options)
              </li>
              <li>
               <strong>
                Create an Animation Object
               </strong>
               (and select animation options)
              </li>
              <li>
               <strong>
                Create a Scene Object
               </strong>
               (and select scene options)
              </li>
              <li>
               <strong>
                Add our Animation Object to the Scene Object
               </strong>
              </li>
              <li>
               <strong>
                Add the Scene Object to the ScrollMagic Controller
               </strong>
              </li>
             </ol>
             <p>
              Its nothing too crazy as its your typical JavaScript stuff, but understanding the underlying structure of how it all plays together will help you move forward with it. Its a little bit more involved than your standard jQuery plug and chug plugins.
             </p>
             <p>
              Now, all that being said, this is how to initiate the ScrollMagic Controller:
             </p>
             <pre>
              <code class="language-javascript">
               $(function() {
    // Init Controller
    var scrollMagicController = new ScrollMagic();
});
              </code>
             </pre>
             <p>
             </p>

             <h2>
              Basic Example
             </h2>
             <p>
              Now lets create the two most basic examples that ScrollMagic does for us.
             </p>
             <h3>
              Animation Trigger Example
             </h3>
             <p>
              All this does is trigger an animation. Well do two things to get this working. First, create the Animation on the element we want to animate. Then, second, well create the Scene which is going to trigger the animation when it is scrolled into view. So, lets go ahead and create that first animation (well cover these more in depth further in the article):
             </p>
             <pre>
              <code class="language-javascript">
               // Create Animation for 0.5s
var tween = TweenMax.to('#animation', 0.5, {
    backgroundColor: 'rgb(255, 39, 46)',
    scale: 7,
    rotation: 360
});
              </code>
             </pre>
             <p>
              Pretty simple! This will add those CSS properties to the element with the ID of
              <code>
               #animation
              </code>
              . However, we need to control when those animations happen. ScrollMagic will make it easy to bind the animation to certain scroll events by creating Scenes. Heres the next piece of code:
             </p>
             <pre>
              <code class="language-javascript">
               // Create the Scene and trigger when visible with ScrollMagic
var scene = new ScrollScene({
    triggerElement: '#scene',
    offset: 150 /* offset the trigger 150px below #scene's top */
})
.setTween(tween)
.addTo(scrollMagicController);
              </code>
             </pre>
             <p>
              We create the Scene as an object to be triggered later, then we pass which animations we want to that Scene, and, finally, we pass it all back to the
              <code>
               ScrollMagicController
              </code>
              to be handled. Heres a stripped and naked example to help explain:
             </p>

              Animations Binded to Scroll
             </h3>
             <p>
              The last example only triggers the animation at the specified Scene trigger point. ScrollMagic can bind your animation to the scroll event. This acts as a rewind and fast-forward scrubber for your animation. Heres the code for doing that:
             </p>
             <pre>
              <code class="language-javascript">
               // Duration ignored / replaced by scene duration now
var tween = TweenMax.to('#animation', 0.5, {
    backgroundColor: 'rgb(255, 39, 46)',
    scale: 5,
    rotation: 360
});

var scene = new ScrollScene({
    triggerElement: '#scene',
    duration: 300 /* How many pixels to scroll / animate */
})
.setTween(tween)
.addTo(scrollMagicController);
              </code>
             </pre>

              You should immediately see that the only difference that matters between the two examples is that the

             <h2>
              Tween Animations in Depth
             </h2>
             <p>
              Theres a ton of options for doing animations. Ill cover some of the more various ones, but, first lets do the most common one  tweening using the
              <a href="greensock.com/docs/#/HTML5/GSAP/">
               GreenSock Animation Platform
              </a>
              .
             </p>
             <p>
              Tweening is what the GSAP calls their animations. Were specifically using their
              <a href="greensock.com/docs/#/HTML5/GSAP/TweenMax/">
               TweenMax library
              </a>
              . TweenMax is awesome because it encompasses all their various plugins and additions into one. This gives us some cross-browser stuff, makes the browser use CSS3 animations first, is extremely performant, and it lets you create complex animations and key frames with ease. Alternatively, you can work piecemeal and pick exactly which components you want with
              <a href="http://greensock.com/docs/#/HTML5/GSAP/TweenLite">
               TweenLite
              </a>
              and its
              <a href="https://greensock.com/plugins/">
               plugins
              </a>
              .
             </p>
             <h3>
              TweenMax.to()
             </h3>
             <p>
              This lets us create our most standard animations. For example if you want an elements background color to go from its default
              <strong>
               to
              </strong>
              red. Heres an example:
             </p>
             <pre>
              <code class="language-javascript">
               // TweenMax.to(selectorOfElementYouWantToAnimate, DurationOfAnimation, AnimationProperties);
var tween = TweenMax.to('#first-animation', 0.5, {backgroundColor: 'red'});
              </code>
             </pre>
             <p>
              You can get as infinitely creative as you want with this. For example, the following tween will make the background red, it grow 5 times in size, and do a full spin rotation using CSS3 Transforms.
             </p>
             <pre>
              <code class="language-javascript">
               var tween = TweenMax.to('#first-animation', 0.5, {
    backgroundColor: 'red',
    scale: 5,
    rotation: 360
});
              </code>
             </pre>
             <p>
              You can do pretty much anything you would be able to do with CSS3 animations  colors, transforms, etc. Heres the
              <a href="http://greensock.com/docs/#/HTML5/GSAP/Plugins/CSSPlugin/">
               official resource
              </a>
              so you can reference for syntax. You can view the examples directly above to see the
              <code>
               TweenMax.to()
              </code>
              function in action.
             </p>
             <h3>
              TweenMax.from()
             </h3>
             <p>
              This works exactly the opposite of
              <code>
               TweenMax.to()
              </code>
              . It will animate to the default styles
              <strong>
               from
              </strong>
              the specified animation options. Heres some example code:
             </p>
             <pre>
              <code class="language-javascript">
               var tween = TweenMax.from('#animation', 0.5, {
    backgroundColor: 'rgb(255, 39, 46)',
    scale: 5,
    rotation: 360
});
              </code>
             </pre>
             <p>
              Heres an example from one of the basic demos using the
              <code>
               from()
              </code>
              function instead:
             </p>
             <p class="codepen" data-default-tab="result" data-height="600" data-slug-hash="pvJoyx" data-theme-id="0" data-user="ncerminara">

             <h3>
              TweenMax.fromTo()
             </h3>
             <p>
              This function is exactly what it sounds like. Youll specify two animation properties for it to animate
              <strong>
               from
              </strong>
              one and then
              <strong>
               to
              </strong>
              the other. Hopefully youre wondering why you cant just use the
              <code>
               to()
              </code>
              function and set the start styles with CSS. Well, you can and thats totally okay. The function

               fromTo
              </code>

              </code>

              <code>

        left: -400
    },

        scale: 1,
        left: 400,
        rotation: 360,

             <h2>
              Custom Containers and Mobile Support
             </h2>
             <p>
              I bunched custom containers and mobile support together because theyre really one and the same. Typically on a mobile touch device, the scroll event isnt detected until the scroll has stopped. This is unfortunate for what were doing. Fortunately that only occurs when youre scrolling on the entire body. If say your scrolling in an element that is set to
              <code>
               overflow: scroll
              </code>
              each moment of the scroll is detected.
             </p>
             <p>
              ScrollMagic lets you specify any container you want for your scenes by a simple option. Heres a code example:
             </p>
             <pre>
              <code class="language-javascript">
               var scrollMagicController = new ScrollMagic({container: '#my-container'});
              </code>
             </pre>
             <p>
              You can do cool things with this like putting your animations and scroll inside of a div or section of your website. Mobile support takes this exact same concept and creates a container or wrapper around the whole site and just binds ScrollMagic to that container.
             </p>
             <p>
              Truth be told its a
              <em>
               hackish
              </em>
              workaround. The major downside is this kills off support for
              <a href="http://css-tricks.com/snippets/css/momentum-scrolling-on-ios-overflow-elements/">
               momentum scrolling
              </a>
              . ScrollMagics official answer to this is to use something like
              <a href="http://iscrolljs.com/">
               iScroll.js
              </a>
              to bring scroll momentum to touch devices. You may also be able to just add this CSS3 property to the container
              <code>
               -webkit-overflow-scrolling: touch;
              </code>
              , but that obviously wont work in all browsers. Its entirely up to you to support this or not, but if you get it right it can really provide a seamless experience on mobile.
             </p>
             <p>
              Lastly, if you want to just disable ScrollMagic on mobile or touch devices, you can do it easily with Modernizr:
             </p>
             <pre>
              <code class="language-javascript">
               if (!Modernizr.touch) {
    // Start ScrollMagic code
}
              </code>
             </pre>
             <p>
              If youre not a fan of using another library like Modernizr just to detect touch, you can use this function:
             </p>
             <pre>
              <code class="language-javascript">
               if (!is_touch_device()) {
    // Start ScrollMagic code
}
function is_touch_device() {
  return 'ontouchstart' in window // works on most browsers 
      || 'onmsgesturechange' in window; // works on ie10
};
              </code>
             </pre>
             <p>
              I grabbed that function from
              <a href="http://stackoverflow.com/questions/4817029/whats-the-best-way-to-detect-a-touch-screen-device-using-javascript">
               this
              </a>
              StackOverflow post. It seems to update a lot in case you want to check to see if thats still up to date.
             </p>
             <p>
             </p>
            </div>
           </div>
           <div class="tile">
            <div class="tile-content">
             <h2>
              Conclusion
             </h2>
             <p>
              ScrollMagics
              <a href="http://janpaepke.github.io/ScrollMagic/docs/">
               official documentation
              </a>
              and
              <a href="http://janpaepke.github.io/ScrollMagic/examples/">
               examples
              </a>
              are amazing. I definitely recommend heading over there and checking out all the other things ScrollMagic can do. Some of those things include:
             </p>
             <ul>
              <li>
               <a href="http://janpaepke.github.io/ScrollMagic/examples/basic/simple_pinning.html">
                Pinning
               </a>
              </li>
              <li>
               <a href="http://janpaepke.github.io/ScrollMagic/examples/basic/going_horizontal.html">
                Horizontal Scrolling
               </a>
              </li>
              <li>
               <a href="http://janpaepke.github.io/ScrollMagic/examples/basic/scene_manipulation.html">
                Scene Manipulation
               </a>
              </li>
              <li>
               <a href="http://janpaepke.github.io/ScrollMagic/examples/advanced/parallax_scrolling.html">
                Parallax
               </a>
              </li>
              <li>
               <a href="http://janpaepke.github.io/ScrollMagic/examples/advanced/svg_drawing.html">
                SVG Drawing
               </a>
              </li>
             </ul>
             <p>
              Theres definitely a lot you can do with ScrollMagic. I think its generally smart to use this with caution in risk of ruining a user experience with some bad animations. Complex ScrollMagic websites are probably best saved for landing pages and using this subtly is best with content based websites. Its all about finding a balance. That being said, I plan on releasing some starter templates soon that would be good for portfolios, product launches, and stuff like that. Stay tuned!
             </p>
             <p>
             </p>
