        <p style="text-align: justify;">
         <a href="http://itsfoss.com/?attachment_id=489" rel="attachment wp-att-489">
          <img alt="Scan Wireless Network Linux" class="alignright size-full wp-image-489" height="150" src="http://itsfoss.itsfoss.netdna-cdn.com/wp-content/uploads/2012/09/wifi-scanner.jpg" title="wifi-scanner" width="200"/>
         </a>
         Wireless Network are the most easy as well as desirable target for wannabe hackers. Ever wondered that someone might be leeching of your hard paid wifi network? You may opt out for an expensive way to keep the hackers away by buying
         <a href="http://www.computerandyou.net/2012/07/secure-wifi-wallpaper/" target="_blank">
          hacker-proof wallpapers
         </a>
         for your home but that really is far fetched. An easy and free way is to scan your network for the device connected with it. This way you can find out if some unknown or unwanted device is connected to it.
        </p>
        <p style="text-align: justify;">
         We will use terminal for finding out what devices are connected to your network in Linux. The process is very simple and easy to use even for beginners. Here we go:
        </p>
        <h2 style="text-align: justify;">
         Get nmap:
        </h2>
        <p style="text-align: justify;">
         nmap is one of the most popular network scanning tool in Linux. Use the following command to install nmap in Ubuntu:
        </p>
        <p style="text-align: justify;">
         sudo apt-get install nmap
        </p>
        <h2 style="text-align: justify;">
         Get IP range of the network:
        </h2>
        <p>
         Now we need to know the IP address range of the network. Use the
         <em>
          <strong>
           ifconfig
          </strong>
         </em>
         command to find it out. Look for
         <strong>
          wlan0
         </strong>
         if you are using wifi or
         <strong>
          eth0
         </strong>
         if you are usingEthernet.
        </p>
        <blockquote>
         <p>
          user@user-notebook:~$ ifconfig
         </p>
         <p>
          wlan0 Link encap:Ethernet HWaddr 70:f1:a1:c2:f2:e9
          <br/>
          <strong>
           inet addr:192.168.1.91
          </strong>
          Bcast:192.168.1.255
          <strong>
           Mask:255.255.255.0
          </strong>
          <br/>
          inet6 addr: fe80::73f1:a1ef:fec2:f2e8/64 Scope:Link
          <br/>
          UP BROADCAST RUNNING MULTICAST MTU:1500 Metric:1
          <br/>
          RX packets:2135051 errors:0 dropped:0 overruns:0 frame:0
          <br/>
          TX packets:2013773 errors:0 dropped:0 overruns:0 carrier:0
          <br/>
          collisions:0 txqueuelen:1000
          <br/>
          RX bytes:1434994913 (1.4 GB) TX bytes:636207445 (636.2 MB)
         </p>
        </blockquote>
        <p style="text-align: justify;">
         The important things are highlighted in bold. As you see my IP is 192.168.1.91 and the
         <a href="http://en.wikipedia.org/wiki/Subnetwork" target="_blank">
          subnet mask
         </a>
         is 255.255.255.0 which means that the ip address range on my network varies from 192.168.1.0 to 192.168.1.255.
        </p>
        <h2 style="text-align: justify;">
         Scan the network:
        </h2>
        <p style="text-align: justify;">
         It is advisable to use rootprivileges while scanning the network for more accurate information. Use the nmapcommandin following way:
        </p>
        <blockquote>
         <p>
          user@user-notebook:~$sudo nmap -sP 192.168.1.0/24
          <br/>
          Starting Nmap 5.21 ( http://nmap.org ) at 2012-09-01 21:59 CEST
         </p>
         <p>
          Nmap scan report for neufbox (192.168.1.1)
          <br/>
          Host is up (0.012s latency).
          <br/>
          MAC Address: E0:A1:D5:72:5A:5C (Unknown)
          <br/>
          Nmap scan report for takshak-bambi (192.168.1.91)
          <br/>
          Host is up.
          <br/>
          Nmap scan report for android-95b23f67te05e1c8 (192.168.1.93)
          <br/>
          Host is up (0.36s latency).
         </p>
        </blockquote>
        <p>
         As you can see there three devices connected to my network. The router itself, my laptop and my Galaxy S2. What is connected to yournetwork?
         <span style="text-align: justify;">
         </span>
        </p>
