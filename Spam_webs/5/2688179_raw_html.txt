         <span class="octicon octicon-book">
         </span>
         README.md
        </h3>
        <article class="markdown-body entry-content" itemprop="mainContentOfPage">
         <h1>
          <a aria-hidden="true" class="anchor" href="#iframe-resizer" id="user-content-iframe-resizer">
           <span class="octicon octicon-link">
           </span>
          </a>
          iFrame Resizer
         </h1>
         <p>
          <a href="http://badge.fury.io/bo/iframe-resizer">
           <img alt="Bower version" data-canonical-src="https://badge.fury.io/bo/iframe-resizer.svg" src="https://camo.githubusercontent.com/1cb540043a3b05f869967ba8689d629bda951dff/68747470733a2f2f62616467652e667572792e696f2f626f2f696672616d652d726573697a65722e737667" style="max-width:100%;"/>
          </a>
          <a href="http://badge.fury.io/js/iframe-resizer">
           <img alt="NPM version" data-canonical-src="https://badge.fury.io/js/iframe-resizer.svg" src="https://camo.githubusercontent.com/0a813e9d8b61094108cd177e6ee2b8a9515afe09/68747470733a2f2f62616467652e667572792e696f2f6a732f696672616d652d726573697a65722e737667" style="max-width:100%;"/>
          </a>
          <a href="https://travis-ci.org/davidjbradshaw/iframe-resizer">
           <img alt="Build Status" data-canonical-src="https://travis-ci.org/davidjbradshaw/iframe-resizer.png?branch=master" src="https://camo.githubusercontent.com/a0afce5932401b93f3c0c6b1d8d9cfab6db2fffe/68747470733a2f2f7472617669732d63692e6f72672f64617669646a62726164736861772f696672616d652d726573697a65722e706e673f6272616e63683d6d6173746572" style="max-width:100%;"/>
          </a>
          <a href="/davidjbradshaw/iframe-resizer/blob/master/LICENSE">
           <img alt="license" data-canonical-src="http://img.shields.io/badge/license-MIT-brightgreen.svg?style=flat" src="https://camo.githubusercontent.com/4440d5deb3a53c4f8661ee765378e6071e7878e8/687474703a2f2f696d672e736869656c64732e696f2f62616467652f6c6963656e73652d4d49542d627269676874677265656e2e7376673f7374796c653d666c6174" style="max-width:100%;"/>
          </a>
         </p>
         <p>
          This library enables the automatic resizing of the height and width of both same and cross domain iFrames to fit the contained content. It provides a range of features to address the most common issues with using iFrames, these include:
         </p>
         <ul>
          <li>
           Height and width resizing of the iFrame to content size.
          </li>
          <li>
           Works with multiple and nested iFrames.
          </li>
          <li>
           Automatic domain authentication for cross domain iFrames.
          </li>
          <li>
           Provides a range of page size calculation methods to support complex CSS layouts.
          </li>
          <li>
           Detects changes to the DOM that can cause the page to resize using
           <a href="https://developer.mozilla.org/en/docs/Web/API/MutationObserver">
            MutationObserver
           </a>
           .
          </li>
          <li>
           Detects events that can cause the page to resize (Window Resize, CSS Animation and Transition, Device Orientation and Mouse events).
          </li>
          <li>
           Simplified messaging from iFrame to host page via
           <a href="https://developer.mozilla.org/en-US/docs/Web/API/window.postMessage">
            postMessage
           </a>
           .
          </li>
          <li>
           Fixes in page links in iFrame and supports links from the iFrame to the parent page.
          </li>
          <li>
           Provides custom sizing and scrolling methods.
          </li>
          <li>
           Works with
           <a href="http://viewerjs.org/">
            ViewerJS
           </a>
           to support PDF and ODF documents.
          </li>
          <li>
           Fallback support down to IE8.
          </li>
         </ul>
         <h3>
          <a aria-hidden="true" class="anchor" href="#getting-started" id="user-content-getting-started">
           <span class="octicon octicon-link">
           </span>
          </a>
          Getting started
         </h3>
         <p>
          The package contains two minified JavaScript files in the
          <a href="/davidjbradshaw/iframe-resizer/blob/master/js">
           js
          </a>
          folder. The first (
          <a href="https://raw.githubusercontent.com/davidjbradshaw/iframe-resizer/master/js/iframeResizer.min.js">
           iframeResizer.min.js
          </a>
          ) is for the page hosting the iFrames. It can be called with
          <strong>
           native
          </strong>
          JavaScript;
         </p>
         <div class="highlight highlight-js">
          <pre>
           iFrameResize( [{options}], [css selector]
           <span class="pl-k">
            ||
           </span>
           [iframe] );
          </pre>
         </div>
         <p>
          or via
          <strong>
           jQuery
          </strong>
          . (See
          <a href="#browser-compatibility">
           notes
          </a>
          below for using native version with IE8).
         </p>
         <div class="highlight highlight-js">
          <pre>
           $(
           <span class="pl-s">
            <span class="pl-pds">
             '
            </span>
            iframe
            <span class="pl-pds">
             '
            </span>
           </span>
           ).iFrameResize( [{options}] );
          </pre>
         </div>
         <p>
          The second file (
          <a href="https://raw.github.com/davidjbradshaw/iframe-resizer/master/js/iframeResizer.contentWindow.min.js">
           iframeResizer.contentWindow.min.js
          </a>
          ) is a
          <strong>
           native
          </strong>
          JavaScript file that needs placing in the page contained within your iFrame.
          <i>
           This file is designed to be a guest on someone else's system, so has no dependencies and won't do anything until it's activated by a message from the containing page
          </i>
          .
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#typical-setup" id="user-content-typical-setup">
           <span class="octicon octicon-link">
           </span>
          </a>
          Typical setup
         </h3>
         <p>
          The normal configuration is to have the iFrame resize when the browser window changes size or the content of the iFrame changes. To set this up you need to configure one of the dimensions of the iFrame to a percentage and tell the library only to update the other dimension. Normally you would set the width to 100% and have the height scale to fit the content.
         </p>
         <div class="highlight highlight-html">
          <pre>
           &lt;
           <span class="pl-ent">
            iframe
           </span>
           <span class="pl-e">
            src
           </span>
           =
           <span class="pl-s">
            <span class="pl-pds">
             "
            </span>
            http://anotherdomain.com/iframe.html
            <span class="pl-pds">
             "
            </span>
           </span>
           <span class="pl-e">
            width
           </span>
           =
           <span class="pl-s">
            <span class="pl-pds">
             "
            </span>
            100%
            <span class="pl-pds">
             "
            </span>
           </span>
           <span class="pl-e">
            scrolling
           </span>
           =
           <span class="pl-s">
            <span class="pl-pds">
             "
            </span>
            no
            <span class="pl-pds">
             "
            </span>
           </span>
           &gt;&lt;/
           <span class="pl-ent">
            iframe
           </span>
           &gt;
           <span class="pl-s1">
            &lt;
            <span class="pl-ent">
             script
            </span>
            &gt;iFrameResize({log
            <span class="pl-k">
             :
            </span>
            <span class="pl-c1">
             true
            </span>
            })&lt;/
            <span class="pl-ent">
             script
            </span>
            &gt;
           </span>
          </pre>
         </div>
         <p>
          Note that scrolling is set to 'no', as older versions of IE don't allow this to be turned off in code and can just slightly add a bit of extra space to the bottom of the content that it doesn't report when it returns the height. If you have problems, check the
          <a href="#troubleshooting">
           troubleshooting
          </a>
          section below.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#example" id="user-content-example">
           <span class="octicon octicon-link">
           </span>
          </a>
          Example
         </h3>
         <p>
          To see this working take a look at this
          <a href="http://davidjbradshaw.com/iframe-resizer/example/">
           example
          </a>
          and watch the
          <a href="https://developer.mozilla.org/en-US/docs/Tools/Web_Console">
           console
          </a>
          .
         </p>
         <h2>
          <a aria-hidden="true" class="anchor" href="#options" id="user-content-options">
           <span class="octicon octicon-link">
           </span>
          </a>
          Options
         </h2>
         <h3>
          <a aria-hidden="true" class="anchor" href="#log" id="user-content-log">
           <span class="octicon octicon-link">
           </span>
          </a>
          log
         </h3>
         <pre>
          <code>
           default: false
type:    boolean
          </code>
         </pre>
         <p>
          Setting the
          <code>
           log
          </code>
          option to true will make the scripts in both the host page and the iFrame output everything they do to the JavaScript console so you can see the communication between the two scripts.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#autoresize" id="user-content-autoresize">
           <span class="octicon octicon-link">
           </span>
          </a>
          autoResize
         </h3>
         <pre>
          <code>
           default: true
type:    boolean
          </code>
         </pre>
         <p>
          When enabled changes to the Window size or the DOM will cause the iFrame to resize to the new content size. Disable if using size method with custom dimensions.
         </p>
         <p>
          <i>
           Note: When set to false the iFrame will still inititally size to the contained content, only additional resizing events are disabled.
          </i>
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#bodybackground" id="user-content-bodybackground">
           <span class="octicon octicon-link">
           </span>
          </a>
          bodyBackground
         </h3>
         <pre>
          <code>
           default: null
type:    string
          </code>
         </pre>
         <p>
          Override the body background style in the iFrame.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#bodymargin" id="user-content-bodymargin">
           <span class="octicon octicon-link">
           </span>
          </a>
          bodyMargin
         </h3>
         <pre>
          <code>
           default: null
type:    string || number
          </code>
         </pre>
         <p>
          Override the default body margin style in the iFrame. A string can be any valid value for the CSS margin attribute, for example '8px 3em'. A number value is converted into px.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#checkorigin" id="user-content-checkorigin">
           <span class="octicon octicon-link">
           </span>
          </a>
          checkOrigin
         </h3>
         <pre>
          <code>
           default: true
type:    boolean || array
          </code>
         </pre>
         <p>
          When set to true, only allow incoming messages from the domain listed in the
          <code>
           src
          </code>
          property of the iFrame tag. If your iFrame navigates between different domains, ports or protocols; then you will need to provide an array of domains or disable this option.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#enableinpagelinks" id="user-content-enableinpagelinks">
           <span class="octicon octicon-link">
           </span>
          </a>
          enableInPageLinks
         </h3>
         <pre>
          <code>
           default: false
type:    boolean
          </code>
         </pre>
         <p>
          When enabled in page linking inside the iFrame and from the iFrame to the parent page will be enabled.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#enablepublicmethods" id="user-content-enablepublicmethods">
           <span class="octicon octicon-link">
           </span>
          </a>
          enablePublicMethods
         </h3>
         <pre>
          <code>
           default: false
type:    boolean
          </code>
         </pre>
         <p>
          If enabled, a
          <code>
           window.parentIFrame
          </code>
          object is created in the iFrame that contains methods outlined
          <a href="#iframe-methods">
           below
          </a>
          .
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#interval" id="user-content-interval">
           <span class="octicon octicon-link">
           </span>
          </a>
          interval
         </h3>
         <pre>
          <code>
           default: 32  (in ms)
type:    number
          </code>
         </pre>
         <p>
          In browsers that don't support
          <a href="https://developer.mozilla.org/en/docs/Web/API/MutationObserver">
           mutationObserver
          </a>
          , such as IE10, the library falls back to using setInterval, to check for changes to the page size. The default value is equal to two frame refreshes at 60Hz, setting this to a higher value will make screen redraws noticeable to the user.
         </p>
         <p>
          Setting this property to a negative number will force the interval check to run instead of
          <a href="https://developer.mozilla.org/en/docs/Web/API/MutationObserver">
           mutationObserver
          </a>
          .
         </p>
         <p>
          Set to zero to disable.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#heightcalculationmethod" id="user-content-heightcalculationmethod">
           <span class="octicon octicon-link">
           </span>
          </a>
          heightCalculationMethod
         </h3>
         <pre>
          <code>
           default: 'bodyOffset'
values:  'bodyOffset' | 'bodyScroll' | 'documentElementOffset' | 'documentElementScroll' |
         'max' | 'min' | 'grow' | 'lowestElement'
          </code>
         </pre>
         <p>
          By default the height of the iFrame is calculated by converting the margin of the
          <code>
           body
          </code>
          to
          <i>
           px
          </i>
          and then adding the top and bottom figures to the offsetHeight of the
          <code>
           body
          </code>
          tag.
         </p>
         <p>
          In cases where CSS styles causes the content to flow outside the
          <code>
           body
          </code>
          you may need to change this setting to one of the following options. Each can give different values depending on how CSS is used in the page and each has varying side-effects. You will need to experiment to see which is best for any particular circumstance.
         </p>
         <ul>
          <li>
           <strong>
            bodyScroll
           </strong>
           uses
           <code>
            document.body.scrollHeight
           </code>
           <sup>
            *
           </sup>
          </li>
          <li>
           <strong>
            documentElementOffset
           </strong>
           uses
           <code>
            document.documentElement.offsetHeight
           </code>
          </li>
          <li>
           <strong>
            documentElementScroll
           </strong>
           uses
           <code>
            document.documentElement.scrollHeight
           </code>
           <sup>
            *
           </sup>
          </li>
          <li>
           <strong>
            max
           </strong>
           takes the largest value of the main four options
           <sup>
            *
           </sup>
          </li>
          <li>
           <strong>
            min
           </strong>
           takes the smallest value of the main four options
           <sup>
            *
           </sup>
          </li>
          <li>
           <strong>
            grow
           </strong>
           same as
           <strong>
            max
           </strong>
           but disables the double resize that is used to workout if the iFrame needs to shrink. This provides much better performance if your iFrame will only ever increase in size
          </li>
          <li>
           <strong>
            lowestElement
           </strong>
           Loops though every element in the the DOM and finds the lowest bottom point
           <sup>
            
           </sup>
          </li>
         </ul>
         <p>
          <i>
           Notes:
          </i>
         </p>
         <p>
          <i>
           <strong>
            If the default option doesn't work then the best solution is often to use
           </strong>
           lowestElement
           <strong>
            in modern browsers and
           </strong>
           max
           <strong>
            in IE10 downwards.
           </strong>
          </i>
         </p>
         <div class="highlight highlight-js">
          <pre>
           <span class="pl-k">
            var
           </span>
           isOldIE
           <span class="pl-k">
            =
           </span>
           (
           <span class="pl-c1">
            navigator
           </span>
           .
           <span class="pl-c1">
            userAgent
           </span>
           .
           <span class="pl-c1">
            indexOf
           </span>
           (
           <span class="pl-s">
            <span class="pl-pds">
             "
            </span>
            MSIE
            <span class="pl-pds">
             "
            </span>
           </span>
           )
           <span class="pl-k">
            !==
           </span>
           <span class="pl-k">
            -
           </span>
           <span class="pl-c1">
            1
           </span>
           );
           <span class="pl-c">
            // Detect IE10 and below
           </span>
           iFrameResize( {
    heightCalculationMethod
           <span class="pl-k">
            :
           </span>
           isOldIE
           <span class="pl-k">
            ?
           </span>
           <span class="pl-s">
            <span class="pl-pds">
             '
            </span>
            max
            <span class="pl-pds">
             '
            </span>
           </span>
           <span class="pl-k">
            :
           </span>
           <span class="pl-s">
            <span class="pl-pds">
             '
            </span>
            lowestElement
            <span class="pl-pds">
             '
            </span>
           </span>
           });
          </pre>
         </div>
         <p>
          <sup>
           
          </sup>
          <i>
           The
           <strong>
            lowestElement
           </strong>
           option is the most reliable way of determining the page height. However, it does have a performance impact in older versions of IE. In one screen refresh (16ms) Chrome can calculate the position of around 10,000 html nodes, whereas IE 8 can calculate approximately 50. It is recommend to fallback to
           <strong>
            max
           </strong>
           or
           <strong>
            grow
           </strong>
           in IE10 and below.
          </i>
         </p>
         <p>
          <sup>
           *
          </sup>
          <i>
           The
           <strong>
            bodyScroll
           </strong>
           ,
           <strong>
            documentElementScroll
           </strong>
           ,
           <strong>
            max
           </strong>
           and
           <strong>
            min
           </strong>
           options can cause screen flicker and will prevent the
           <a href="#interval">
            interval
           </a>
           trigger downsizing the iFrame when the content shrinks. This is mainly an issue in IE 10 and below, where the
           <a href="https://developer.mozilla.org/en/docs/Web/API/MutationObserver">
            mutationObserver
           </a>
           event is not supported. To overcome this you need to manually trigger a page resize by calling the
           <a href="#size-customheight-customwidth">
            parentIFrame.size()
           </a>

          <code>

          <a href="https://github.com/davidjbradshaw/iframe-resizer/blob/master/CONTRIBUTING.md">

          </a>

         <h3>
          <a aria-hidden="true" class="anchor" href="#iframe-not-sizing-correctly" id="user-content-iframe-not-sizing-correctly">
           <span class="octicon octicon-link">
           </span>
          </a>
          IFrame not sizing correctly
         </h3>
         <p>
          If a larger element of content is removed from the normal document flow, through the use of absolute positioning, it can prevent the browser working out the correct size of the page. In such cases you can change the
          <a href="#heightcalculationmethod">
           heightCalculationMethod
          </a>
          to uses one of the other sizing methods, normally you will be best off selecting the
          <strong>
           max
          </strong>
          or
          <strong>
           lowestElement
          </strong>
          options to avoid cross browser differences.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#iframe-not-downsizing" id="user-content-iframe-not-downsizing">
           <span class="octicon octicon-link">
           </span>
          </a>
          IFrame not downsizing
         </h3>
         <p>
          The most likely cause of this problem is having set the height of an element to be 100% of the page somewhere in your CSS. This is normally on the
          <code>
           html
          </code>
          or
          <code>
           body
          </code>
          elements, but it could be on any element in the page.
         </p>
         <p>
          Not having a valid
          <a href="http://en.wikipedia.org/wiki/Document_type_declaration">
           HTML document type
          </a>
          in the iFrame can also sometimes prevent downsizing. At it's most simplest this can be the following.
         </p>
         <div class="highlight highlight-html">
          <pre>
           &lt;!DOCTYPE html&gt;
          </pre>
         </div>
         <h3>
          <a aria-hidden="true" class="anchor" href="#iframe-not-resizing" id="user-content-iframe-not-resizing">
           <span class="octicon octicon-link">
           </span>
          </a>
          IFrame not resizing
         </h3>
         <p>
          The most common cause of this is not placing the
          <a href="https://raw.github.com/davidjbradshaw/iframe-resizer/master/js/iframeResizer.contentWindow.min.js">
           iframeResizer.contentWindow.min.js
          </a>
          script inside the iFramed page. If the other page is on a domain outside your control and you can not add JavaScript to that page, then now is the time to give up all hope of ever getting the iFrame to size to the content. As it is impossible to work out the size of the contained page, without using JavaScript on both the parent and child pages.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#iframe-not-detecting-css-hover-events" id="user-content-iframe-not-detecting-css-hover-events">

         <h3>
          <a aria-hidden="true" class="anchor" href="#iframe-not-detecting-textarea-resizes" id="user-content-iframe-not-detecting-textarea-resizes">
           <span class="octicon octicon-link">
           </span>
          </a>
          IFrame not detecting textarea resizes
         </h3>
         <p>
          Both FireFox and the WebKit based browsers allow the user to resize
          <code>
           textarea
          </code>
          input boxes. Unfortunately the WebKit browsers don't trigger the mutation event when this happens. This can be worked around to some extent with the following code.
         </p>
         <pre>
          <code>
           $(document).ready(function(){
    function resize(){
        if ('parentIFrame' in window) parentIFrame.size();
    }

    var $textareas = $('textarea');

    // store init (default) state
    $textareas.data('x', $textareas.outerWidth()).data('y', $textareas.outerHeight());

    $textareas.on('mouseover mouseup mouseout',function(){

        var $this = $(this);

        if (  $this.outerWidth()  != $this.data('x') || $this.outerHeight() != $this.data('y') ){
            resize();
        }

        // store new height/width
        $this.data('x', $this.outerWidth()).data('y', $this.outerHeight());
    });

    $(window).on('mouseout',resize);
});
          </code>
         </pre>

         </h3>
         <p>
          Some of the alternate
          <a href="#heightcalculationmethod">
           height calculation methods
          </a>
          , such as
          <strong>
           max
          </strong>

          <a href="#minheight--minwidth">
           minSize
          </a>
          value, so that the iFrame is not reset to zero height before regrowing.
         </p>
         <p>
          In modern browsers, if the default
          <a href="#heightcalculationmethod">
           height calculation method
          </a>
          does not work, then it is normally best to use
          <strong>
           lowestElement
          </strong>
          , which is flicker free, and then provide a fallback to
          <strong>
           max
          </strong>
          in IE10 downwards.

           <span class="octicon octicon-link">
           </span>
          </a>
          PDF and OpenDocument files
         </h3>
         <p>
          It is not possible to add the required JavaScript to PDF and ODF files. However, you can get around this limitation by using
          <a href="http://viewerjs.org/">
           ViewerJS
          </a>
          to render these files inside a HTML page, that also contains the iFrame JavaScript file (
          <a href="https://raw.github.com/davidjbradshaw/iframe-resizer/master/js/iframeResizer.contentWindow.min.js">
           iframeResizer.contentWindow.min.js
          </a>
          ).
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#unexpected-message-received-error" id="user-content-unexpected-message-received-error">
           <span class="octicon octicon-link">
           </span>
          </a>
          Unexpected message received error
         </h3>
         <p>
          By default the origin of incoming messages is checked against the
          <code>
           src
          </code>
          attribute of the iFrame. If they don't match an error is thrown. This behaviour can be disabled by setting the
          <a href="#checkorigin">
           checkOrigin
          </a>
          option to
          <strong>
           false
          </strong>
          .
         </p>
         <h2>
          <a aria-hidden="true" class="anchor" href="#browser-compatibility" id="user-content-browser-compatibility">
           <span class="octicon octicon-link">
           </span>

          Browser compatibility
         </h2>
         <h3>
          <a aria-hidden="true" class="anchor" href="#jquery-version" id="user-content-jquery-version">
           <span class="octicon octicon-link">

          jQuery version
         </h3>
         <p>
          Works with all browsers which support
          <a href="http://caniuse.com/#feat=x-doc-messaging">
           window.postMessage
          </a>
          (IE8+).
