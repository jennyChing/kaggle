         <h1 class="title single-title">
          How to Add  Video to Your Web Pages with HTML5
         </h1>
         <div class="post-info">
          <span class="theauthor">
           <a href="http://www.natebal.com/blog/author/nbalcom/" rel="nofollow" title="Posts by Nate Balcom">

         <p>
          The advent of
          <a href="http://www.natebal.com/blog/?s=html5">
           Html5
          </a>
          has radically changed how modern day web pages are coded. In many ways
          <a href="http://www.natebal.com">
           HTML5 code
          </a>
          is less with more. A more condensed code base with more options than previous versions of html. One new aspect of HTML5 is the video tag. The new tag comes complete with a library of pre-defined audio and video controls built in that help condense code and frankly make it easier to look at and understand. That being said, I wanted to give a little explanation of new file formats and code configuration.
         </p>
         <video controls="" width="665px">
          <source src="http://www.natebal.com/blog/video/simpsons-comeback.mp4" type="video/mp4">
           <source src="http://www.natebal.com/blog/video/simpsons-comeback.webm" type="video/webm">

             <a href="http://www.natebal.com/blog/video/simpsons-comeback.avi">
              download this video here
             </a>
             .http://www.w3schools.com/
            </source>
           </source>
          </source>
         </video>
         <p>
          Theres been a lot written about the newish HTML5
          <a href="http://www.natebal.com/blog/?s=video">
           video
          </a>
          tag. This posts purpose is to give more information about what the new file formats are and how to implement them on your own web site.  As it stands there are 3 supported video formats one of which is a augmented version of the previous mpeg files of the past. The other 2 are developed by organizations interested in tools for all users.
         </p>
         <h2>
          Supported Html5 Video Formats Defined
         </h2>
         <p>
          The 3 supported file formats are
          <a href="http://en.wikipedia.org/wiki/MPEG-4_Part_14">
           MP4
          </a>
          ,
          <a href="http://en.wikipedia.org/wiki/WebM">
           WebM
          </a>
          , and
          <a href="http://en.wikipedia.org/wiki/Ogg">
           Ogg
          </a>
          and are defined by
          <a href="http://www.w3schools.com/">
           W3Schools.com
          </a>
          below.
         </p>
         <p>
          <u>
           MP4:
          </u>
          <em>
           MPEG-4 Part 14 or MP4 is a digital multimedia format most commonly used to store video and audio.
          </em>
         </p>
         <p>
          <u>
           WebM:
          </u>
          <em>
           WebM is a video file format intended primarily for royalty-free use in the HTML5 video tag. The WebM Project releases WebM-related software under a BSD license and all users are granted a worldwide, non-exclusive, no-charge, royalty-free patent license.
          </em>
         </p>
         <p>
          <u>
           OGG:
          </u>
          <em>
           Ogg is a free, open container format maintained by the
           <a href="http://xiph.org/">
            Xiph.Org Foundation
           </a>
           . The Xiph.Org Foundation is a non-profit corporation dedicated to protecting the foundations of Internet multimedia from control by private interests. The creators of the Ogg format state that it is unrestricted by software patents and is designed to provide for efficient streaming and manipulation of high quality digital
           <a href="http://www.natebal.com/blog/how-to-add-video-to-your-web-pages-with-html5/">
            multimedia
           </a>
           . Thanks Google.
          </em>
         </p>
         <h3>
          HTML5 Video Tag Implementation
         </h3>
         <p>
          Take a look at the code below. Its very readable and very straight forward. The video tag uses javascript to provide video and audio control options that are assigned in the container tag surrounding file format types. You can use one of the file formats listed here or all of them. The reason for more than one is to cover your bases in case one format isnt supported or is more popular than another. If by chance someone is using an antiquated browser you can add a link to a more traditionally supported video format. In this case I linked to an avi.
         </p>
         <p>
          <strong>
           THE CODE
          </strong>
         </p>
         <blockquote>
          <p>
           <br/>
           <br/>
           <br/>
           <br/>
           Your browser does not support the video tag. You can
           <a href="simpsons-comeback.avi">
            download this video here
           </a>
           .
           <br/>
          </p>
         </blockquote>
         <h4>
          HTML5 Video Control Methods &amp; Properties
         </h4>
         <p>
          There is a whole gallery of attributes that can be added to your video tag that can be seen
          <a href="http://www.w3schools.com/tags/ref_av_dom.asp" target="_blank">
           here
          </a>
          . The following are commonly used Html5 video methods and properties.
         </p>
         <p>
          <strong>
           pause()
          </strong>
           Pauses the currently playing audio or video
          <br/>
          <strong>
           autoplay
          </strong>
           Sets video to play as soon as its loaded.
          <br/>
          <strong>
           controls
          </strong>
           Adds controls to your videos.
          <br/>
          <strong>
           loop
          </strong>
           Control sets the video to loop.
          <br/>
          <strong>
           muted
          </strong>
           Mutes video
          <br/>
          <strong>
           preload
          </strong>
           Tells the web page to load video while page is loading.
         </p>
