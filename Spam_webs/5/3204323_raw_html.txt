      <p>
       Using data- attributes for indexation, and a dynamic stylesheet with a CSS3 selector for search, it is straightforward to implement a client-side full-text search in CSS rather than JavaScript. Here is an example.
      </p>
      <h2>
       The Searchable List
      </h2>
      <script src="https://gist.github.com/fzaninotto/a37929724abdcebd935a.js">
      </script>
      <h2>
       The Search Code
      </h2>
      <p>
       The search is very straightforward: it relies on two well-supported CSS3 selectors (
       <code>
        :not()
       </code>
       and
       <code>
        [attr*=]
       </code>
       ) and the rewriting of a style rule each time the search input is modified:
      </p>
      <script src="https://gist.github.com/fzaninotto/2da116a146457aa7e78c.js">
      </script>
      <p>
       The advantage of using CSS selectors rather than JavaScript
       <code>
        indexOf()
       </code>
       for search is speed: you only change one element at each keystroke (the
       <code>
        &lt;style&gt;
       </code>
       tag) instead of changing all the elements matching the query. Using the
       <code>
        :not()
       </code>
       selector, this implementation works on IE9+, but it could easily be made compatible with IE8+ by using two rules instead of one.
      </p>
