    function asyncGetData(feedUrl) {
      return $.ajax({ url: feedUrl, dataType: "jsonp", jsonp: "jsonp" });
    }


    function refresh(feed) {

		var feedUrl = feed.url + feed.pageIndex;
		var feedContainer = $("#" + feed.containerId);
		var pdata = asyncGetData(feedUrl);
		pdata.done(function(data) { 
			var container = feedContainer;
			container.empty();
			feed.totalCount=data.TotalItems;
			for(var i = 0; i &lt; pageSize &amp;&amp; i &lt; data.Items.length; i++) {
			  container.append($(bindTemplate(data.Items[i])));
			}
		});
    }
    
    function bindTemplate(item) {
      return template.replace(/\{([^}]+)\}/gi, function(m) { 
				with(item){ 
                   var placeholder = m.substr(1, m.length - 2);
                   var value;
				   if(placeholder == 'AuthorEncoded'){
					  value = encodeURIComponent(Author);
				   }else {
					  value =  eval(placeholder);
				   }
                   if(placeholder === 'Content') {
						var node = $("&lt;div&gt;" + value + "&lt;/div&gt;");
						var text = node.text();
						var max = 150;
						if(text.length &gt; max) {
							text = text.substring(0, max);
							text = text.replace(/\w+$/, '');
							value =  text + "...";
						}
						else{
							value = text;
						}
                   }
                    return value; 	
				}});
    }
    
    $(function() {
    	for(var i = 0; i &lt; feeds.length; i++){
    		refresh(feeds[i]); 
		}
    });
  })();

            <h3 class="post-name">
             Just My Code for JavaScript (Pierson Lee)
            </h3>
            <div class="post-rating">
             Rate This
             <br/>

            <div class="post-content user-defined-markup">
             <p>
              With the new
              <a href="http://aka.ms/officedevtoolsforvs2013">
               Office Developer Tools for Visual Studio 2013  March 2014 Update
              </a>
              comes a feature called
              <i>
               Just My Code for JavaScript
              </i>
              , which can speed up your debugging experience by allowing you to focus on debugging your own code, and not library or framework code. In this post, Ill walk through how this works.
             </p>
             <h3>
              What is Just My Code for JavaScript?
             </h3>
             <p>
              Just My Code for JavaScript will give you a similar debugging experience that you may be used to with managed code (VB/C#). While debugging, if Just My Code is enabled, you will avoid stepping into library and framework code. Just My Code categorizes code into three categories:
             </p>
             <ol>
              <li>
               <b>
                User Code:
               </b>
               Code that is written and controlled by you.
               <b>
               </b>
              </li>
              <li>
               <b>
                Library Code:
               </b>
               Code that is part of libraries such as WinJS, DataJS, jQuery, etc.
               <b>
               </b>
              </li>
              <li>
               <b>
                Unrelated:
               </b>
               Code that is running in your application, but is neither library code nor code that you own, and is not necessary in order for your application to function properly. For example, JavaScript code from the SharePoint server would be labeled
               <i>
                unrelated
               </i>
               .
               <b>
               </b>
              </li>
             </ol>
             <p>
              By default, the following is considered Library code in LightSwitch and Cloud Business App projects:
             </p>
             <ul>
              <li>
               Any file that is a framework reference. This includes MSLS, jQuery, jQueryMobile, WinJS, and DataJS code.
              </li>
              <li>
               Any file that has been minified, such that the file name ends in .min.js.
              </li>
              <li>
               Any file that is located in output directorys Scripts\Generated folder. This is code generated for you by the project.
              </li>
              <li>
               Any file that matches the RegEx patterns located in %ProgramFiles%\Microsoft Visual Studio 12.0\Common7\IDE\Extensions\Microsoft\Web Tools\Page Inspector\JScriptLibraries.txt.
              </li>
             </ul>
             <p>
              By enabling Just My Code, it will provide an easier debugging experience by allowing you to focus only on the code that you wrote and reduce the noise that may be generated by library and framework code.
             </p>
             <h3>
              How do I Enable Just My Code?
             </h3>
             <p>
              To enable Just My Code, find the Options menu item under the Tools menu. Then under the Debugging category, locate the Just My Code check box and select it.
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/3580.clip_5F00_image002_5F00_02ECEBBA.jpg">
               <img alt="clip_image002" border="0" height="337" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/7774.clip_5F00_image002_5F00_thumb_5F00_4C7ECA68.jpg" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image002" width="579"/>
              </a>
             </p>
             <p>
              Note that because of the integration with Internet Explorer, in order for Just My Code for Javascript to work, it requires that
              <a href="http://windows.microsoft.com/en-us/internet-explorer/download-ie">
               Internet Explorer 11
              </a>
              is installed.
             </p>
             <h3>
              Lets Try It Out!
             </h3>
             <p>
              For this example, well use Matt Sampsons
              <a href="http://blogs.msdn.com/b/rmattsampson/archive/2013/10/29/html-upload-control-for-lightswitch-sharepoint-apps.aspx">
               HTML Upload Control for LightSwitch SharePoint Apps
              </a>
              example. Following his instructions for the project, well take a look at debugging the upload feature.
             </p>
             <p>
              In the completed example, open the AddEditBooks.lsml.js file which has the two upload methods. The file should look similar to this:
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/8863.clip_5F00_image003_5F00_455F8DF0.png">
               <img alt="clip_image003" border="0" height="372" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/6710.clip_5F00_image003_5F00_thumb_5F00_1A1AF6E9.png" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image003" width="709"/>
              </a>
             </p>
             <p>
              We would like to see what the myUpload variable contains without stepping into code that we dont own. To do this, set a breakpoint on the following line:
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/4666.clip_5F00_image004_5F00_79FFEA2B.png">
               <img alt="clip_image004" border="0" height="373" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/6131.clip_5F00_image004_5F00_thumb_5F00_59E4DD6E.png" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image004" width="711"/>
              </a>
             </p>
             <p>
              Now we will debug it by hitting F5 and adding a new book. Our first breakpoint will be hit like so:
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/4571.clip_5F00_image005_5F00_32AA9439.png">
               <img alt="clip_image005" border="0" height="506" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/7217.clip_5F00_image005_5F00_thumb_5F00_5C21662A.png" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image005" width="719"/>
              </a>
             </p>
             <p>
              With Just My Code enabled, choose to Step Into (F11) the SingleUpload method. At this point youll step into code provided by Matt Sampson because Visual Studio assumes that this is also code we wrote (i.e. its not a library provided by the client framework). However, we can customize the Just My Code feature so that it will also step over this code when debugging. Lets see how we can do that.
             </p>
             <h3>
              How do I customize Just My Code?
             </h3>
             <p>
              To customize Just My Code functionality, you can add a file called mycode.json to the root of the HTMLClient project within your solution. This file will change the default behavior. To do this, on the AllMyBooks.HTMLClient, right click and choose Add -&gt; New Item. Select JavaScript File and set the name to
              <b>
               mycode.json
              </b>
              .
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/5164.clip_5F00_image006_5F00_02EF7C6B.png">
               <img alt="clip_image006" border="0" height="348" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/4442.clip_5F00_image006_5F00_thumb_5F00_3240F1F5.png" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image006" width="292"/>
              </a>
             </p>
             <p>
              The file needs to have the following format:
             </p>
             <pre>
              {
    "Eval" : "MyCode",
    "Function" : "Library",
    "ScriptBlock" : "Unrelated",
    "MyCode" : [
    ],
    "Libraries" : [
    ],
    "Unrelated" : [
    ]
}
             </pre>
             <p>
              This file does not override default behavior of Just My Code. Rather, it enhances it. This file will be used to first to resolve all JavaScript files and the default behavior will be the fall back in case this file does not have the JavaScript file explicitly listed.
             </p>
             <p>
              We will want to have the below block in the file.
             </p>
             <pre>
              {
    "Eval" : "MyCode",
    "Function" : "Library",
    "ScriptBlock" : "Unrelated",
    "MyCode" : [
    ],
    "Libraries" : [
		'*sharepoint.js',
		'*SingleUpload.js'
    ],
    "Unrelated" : [
    ]
}
             </pre>
             <p>
              Notice that wildcard characters are necessary at this time. Now both sharepoint.js and SingleUpload.js will be considered library code. If you restart the debugging and step into the above breakpoint, youll notice that it will step over the code, allowing you to inspect the myUpload object and see the values within it.
             </p>
             <h3>
              Debugging Behavior Details
             </h3>
             <p>
              <b>
               Debugging Commands
              </b>
             </p>
             <p>
              During debugging the following behavior changes occur with Just My Code enabled:
             </p>
             <ul>
              <li>
               While in user code, a Step Into command will be treated as a Step Over command if an attempt is made to step into non-user code.
              </li>
              <li>
               Breakpoints set in non-user code will still be hit. When this breakpoint is hit, stepping at this point will behave the same as if Just My Code was not enabled until you step back into user code. At that point, it will behave like Just My Code is enabled.
              </li>
              <li>
               A Step Out will stop on the next line of user code. If no user code is encountered, then this is the same as a Continue command.
              </li>
             </ul>
             <p>
              <b>
               Call Stack Window
              </b>
             </p>
             <p>
              Another change is what is seen in the call stack window. When Just My Code is enabled, the call stack window will show any non-user code as External Code. In our previous example, when the breakpoint is hit, this is what the Call Stack window looks like.
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/7288.clip_5F00_image007_5F00_590F0835.png">
               <img alt="clip_image007" border="0" height="255" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/4035.clip_5F00_image007_5F00_thumb_5F00_38F3FB78.png" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image007" width="371"/>
              </a>
             </p>
             <p>
              To view the full call stack with Just My Code enabled, you can access the Show External Code option within the context menu of the Call Stack window.
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/3377.clip_5F00_image008_5F00_5FC211B8.png">
               <img alt="clip_image008" border="0" height="510" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/1234.clip_5F00_image008_5F00_thumb_5F00_3FA704FB.png" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image008" width="372"/>
              </a>
             </p>
             <p>
              After this has been enabled, you will see the full call stack including external code.
             </p>
             <p>
              <a href="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/8171.clip_5F00_image009_5F00_347D7AB1.png">
               <img alt="clip_image009" border="0" height="351" src="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-39-73-metablogapi/0652.clip_5F00_image009_5F00_thumb_5F00_2D5E3E39.png" style="border: 0px currentcolor; display: inline; background-image: none;" title="clip_image009" width="372"/>
              </a>
             </p>
             <h3>
              Summary
             </h3>
             <p>
              Using
              <i>
               Just My Code for JavaScript
              </i>
              within your LightSwitch or Cloud Business App project will give you an enhanced debugging experience by allowing you to focus on debugging
              <b>
               your
              </b>
              code and avoid the noise of framework and library code. If you have any questions or comments, please let us know through comments below or by posting in
              <a href="http://social.msdn.microsoft.com/Forums/vstudio/en-US/home?forum=lightswitch">
               our forum
              </a>
              .
             </p>
             <p>
              - Pierson Lee, Software Development Engineer, Cloud Platform Tools
             </p>
