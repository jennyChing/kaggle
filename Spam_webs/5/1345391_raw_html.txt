       <h2>
        andLinux: a new breed of Linux distribution
       </h2>
       <div class="postmeta">
       </div>
       <div class="entry">
        <p>
         When people are asked why they havent tried out Linux yet, quite often you hear arguments like: I dont want to install another disk drive, I dont want to mess with my Windows partition, etc. To a certain extent those are valid arguments, especially if you are scared messing with your computers hardware or some of the more geeky aspects of the software environment.
        </p>
        <p>
         While there was always the possibility to get your feet wet using virtualization software like
         <a href="http://www.vmware.com/" onclick="javascript:pageTracker._trackPageview('/outgoing/www.vmware.com/');">
          VMWare
         </a>
         , where the software simulates a virtual machine that is completely independent from the host operating system (think computer inside a computer), now theres an even easier method.
        </p>
        <p>
         Let me introduce
         <a href="http://www.andlinux.org/" onclick="javascript:pageTracker._trackPageview('/outgoing/www.andlinux.org/');">
          andLinux
         </a>
         . andLinux is based on the work that went into the
         <a href="http://www.colinux.org/" onclick="javascript:pageTracker._trackPageview('/outgoing/www.colinux.org/');">
          Cooperative Linux
         </a>
         project. Instead of relying on virtualization technologies, coLinux managed to compile the Linux kernel in a way that allows it to run side-by-side with the host OS. No virtualization required, no messing with your disk layout, no additional software required. andLinux installs just like any other Windows application. All you need is a nice chunk of disk space. Unlike coLinux, andLinux deals with some of the scary parts of the coLinux configuration. It will automatically configure network connections correctly and even share your windows filesystem with the Linux portion. The later allows you to access all your windows files from Linux.
        </p>
        <p>
         I tried the
         <a href="http://www.andlinux.org/downloads.php" onclick="javascript:pageTracker._trackPageview('/outgoing/www.andlinux.org/downloads.php');">
          KDE-version
         </a>
         of andLinux the other day and just wanted to write a bit about my experience.
        </p>
        <p>
         After downloading some 665MB I ended up with a single installer executable.
        </p>
        <p>
         The installation process was painless and I quickly answered the few questions that came up during the installation: you tell it where to install it, you tell it how much memory you want to dedicate to your Linux computer, if and how you want to share the windows filesystem (I picked the easier coFS option) and off you go.
         <br/>
         During the installation a new network driver was copied to the system (
         <a href="http://colinux.wikia.com/wiki/TAP-Win32_driver" onclick="javascript:pageTracker._trackPageview('/outgoing/colinux.wikia.com/wiki/TAP-Win32_driver');">
          TAP-Win32
         </a>
         ), which required a restart once the installation completed. The network driver allows communication between the Windows OS and Linux OS as if those systems resided on a different network and were connected via Ethernet-cables.
        </p>
        <p>
        </p>
        <center>
         <span class="frame-outer small ">
          <span>
           <span>
            <span>
             <span>
              <img alt="desktop/taskbar icons" height="73" src="http://www.kahunaburger.com/wp-content/images/2008-02-22-and-shortcuts.png" title="desktop/taskbar icons" width="415"/>
             </span>
            </span>
           </span>
          </span>
         </span>
        </center>
        <p>
         <span class="frame-outer " style="float: right;">
          <span>
           <span>
            <span>
             <span>
              <img alt="systray" height="586" hspace="10" src="http://www.kahunaburger.com/wp-content/images/2008-02-22-and-tray.png" title="systray" width="164"/>
             </span>
            </span>
           </span>
          </span>
         </span>
         The installer left three desktop shortcuts on the desktop and two quickstart icons in the taskbar (pictured above, I dragged the two items from the taskbar [right-most] on the desktop).
        </p>
        <p>
         After the restart I also found a new item in my system tray (pictured on the right). Those items in the systray-menu allow you to execute a bunch of the most often used applications directly from there. Once andLinux is running on your system, you right-click on the tray-icon, select the application and it will be run under Linux.
        </p>
        <p>
         So I finally went ahead and started andLinux via the Start andLinux shortcut on the desktop. A new console window (labelled andServer (CoLinux)) opened and I saw messages scrolling by that documented the startup process (youll see that console window further down in this post).
        </p>
        <p>
         From the startup messages you can see that I opted to reserve up to 384MB of my memory for andLinux. The documentation suggests that you should not use less than 256MB, however under certain circumstances you might even get away with 128MB (expect the performance to become sluggish at that value).
        </p>
        <p>
         Linux root file system is in a single file called base.drv. Extra swap space is created in a separate file called swap.drv. During the startup process andLinux will attach to those files and treat the single file like a disk.
        </p>
        <p>
         As you can see from the startup messages, andLinux also configures two network adapters (eth0 and eth1). The first one is used to bridge Linux networking with your real network adapter on Windows  this allows Linux to reach the Internet. The second one is used as a communication device between Windows and Linux and thats the place where the above mentioned TAP-Win32 driver comes into play.
        </p>
        <p>
        </p>
        <center>
         <span class="frame-outer ">
          <span>
           <span>
            <span>
             <span>
              <img alt="andLinux during startup" height="193" src="http://www.kahunaburger.com/wp-content/images/2008-02-22-and-server.png" title="andLinux during startup" width="415"/>
             </span>
            </span>
           </span>
          </span>
         </span>
        </center>
        <p>
         With the andLinux server running I clicked on the KHomeFolder shortcut and within a second I had KDEs Konqueror window with roots home folder on my screen.
        </p>
        <p>
        </p>
        <center>
         <span class="frame-outer ">
          <span>
           <span>
            <span>
             <span>
              <img alt="KDE and Windows at the same time" height="329" src="http://www.kahunaburger.com/wp-content/images/2008-02-22-and-desktop.png" title="KDE and Windows at the same time" width="415"/>
             </span>
            </span>
           </span>
          </span>
         </span>
        </center>
        <p>
         I tried a number of other applications and the performance was very nice. Overall the applications felt very responsive and I did not get the impression that those were not native Windows applications.
        </p>
        <p>
         Behind the scenes a special version of the X11-server,
         <a href="http://freedesktop.org/wiki/Xming" onclick="javascript:pageTracker._trackPageview('/outgoing/freedesktop.org/wiki/Xming');">
          Xming
         </a>
         is being used to accomplish that. Xming is launched when the andLinux server starts up. It creates an X11-screen that coexists with the Windows desktop. Theres no switching between virtual screens, everything looks like its part of your standard Windows installation.
        </p>
        <p>
         After starting and closing a number of applications, I was curious what Linux memory consumption looked like. To my surprise I found the following:
        </p>
        <p>
        </p>
        <center>
         <span class="frame-outer ">
          <span>
           <span>
            <span>
             <span>
              <img alt="andLinux - memory" height="391" src="http://www.kahunaburger.com/wp-content/images/2008-02-22-and-memory.png" title="andLinux - memory" width="415"/>
             </span>
            </span>
           </span>
          </span>
         </span>
        </center>
        <p>
         I still had 280MB (from the total of 384MB) physical memory available  nice!
        </p>
        <p>
         Network connectivity worked right after the start. Where I previously had to tweak/configure settings in coLinux, andLinux made that portion a snap and configured everything for you.
        </p>
        <p>
        </p>
        <center>
         <span class="frame-outer small ">
          <span>
           <span>
            <span>
             <span>
              <img alt="andLinux - ifconfig" height="100" src="http://www.kahunaburger.com/wp-content/images/2008-02-22-and-network.png" title="andLinux - ifconfig" width="415"/>
             </span>
            </span>
           </span>
          </span>
         </span>
        </center>
        <p>
         Internet access from Linux applications worked as expected and my Windows file-system was automatically mounted (you can see it above in the Konquerer screenshot).
        </p>
        <p>
         All in all a more than pleasant experience.
        </p>
        <p>
         I guess theres no more excuse now not to give Linux a try. The only thing that makes you scratch your head: why would you want to run a more stable operating system under a less stable one? But, I guess, thatll be subject of another post.
        </p>
