    <h1>
     About PostGIS
    </h1>
    <p>
     <span itemscope="" itemtype="http://www.schema.org/SoftwareApplication">
      <span itemprop="about">
       <span itemprop="name">
        PostGIS
       </span>
       is a
       <span itemprop="applicationSubCategory">
        spatial
       </span>
       <span itemprop="applicationCategory">
        database
       </span>
       extender for
       <a href="http://postgresql.org">
        <span itemprop="requirements">
         PostgreSQL
        </span>
       </a>
       object-relational database.  It adds support for geographic objects allowing location queries to be run in SQL.
      </span>
      <pre class="sql" style="font-family:monospace;">
       <span style="color: #993333; font-weight: bold;">
        SELECT
       </span>
       superhero
       <span style="color: #66cc66;">
        .
       </span>
       name
       <span style="color: #993333; font-weight: bold;">
        FROM
       </span>
       city
       <span style="color: #66cc66;">
        ,
       </span>
       superhero
       <span style="color: #993333; font-weight: bold;">
        WHERE
       </span>
       ST_Contains
       <span style="color: #66cc66;">
        (
       </span>
       city
       <span style="color: #66cc66;">
        .
       </span>
       geom
       <span style="color: #66cc66;">
        ,
       </span>
       superhero
       <span style="color: #66cc66;">
        .
       </span>
       geom
       <span style="color: #66cc66;">
        )
       </span>
       <span style="color: #993333; font-weight: bold;">
        AND
       </span>
       city
       <span style="color: #66cc66;">
        .
       </span>
       name
       <span style="color: #66cc66;">
        =
       </span>
       <span style="color: #ff0000;">
        'Gotham'
       </span>
       ;
      </pre>
     </span>
    </p>
    <p>
     In addition to basic location awareness, PostGIS offers many features rarely found in other competing spatial databases
such as Oracle Locator/Spatial and SQL Server. Refer to
     <a href="http://www.postgis.net/features">
      PostGIS Feature List
     </a>
     for more details.
    </p>

    <h3>
     <a href="http://www.postgis.net/2014/12/19/postgis_fdw">
      Foreign Data Wrappers for PostGIS
     </a>
    </h3>
    <p>
     The last couple weeks have seen two interesting updates in the world of PostgreSQL  
     <a href="https://wiki.postgresql.org/wiki/Foreign_data_wrappers">
      foreign data wrappers
     </a>
      (FDW). Foreign data wrappers allow you to access remote data inside your database, exactly like other tables. PostgreSQL ships with two example implementations, one for accessing text files, and the other for accessing remote PostgreSQL servers.
    </p>
    <p>
     The two updates of interest to PostGIS users are:
    </p>
    <ul>
     <li>
      The
      <a href="http://laurenz.github.io/oracle_fdw/">
       Oracle FDW
      </a>
      implementation was recently enhanced to support spatial columns, so an Oracle table with SDO_GEOMETRY columns can be exposed in PostgreSQL as a table with PostGIS geometry columns.
     </li>
     <li>
      A new
      <a href="https://github.com/pramsey/pgsql-ogr-fdw">
       OGR FDW
      </a>
      implementation
      <a href="http://boundlessgeo.com/2014/12/ogr-fdw-ftw/">
       was released
      </a>
      that supports exposing any OGR data source as a table with PostGIS geometry columns.
     </li>
    </ul>
    <p>
     Now you can access your PostGIS data without even going to the trouble of importing it first!
    </p>

The last iteration went something like this:
    </p>
    <p>
     <em>
      I cant use
      <code>
       ST_AsPNG
      </code>
      when doing something like
     </em>
    </p>

     <span style="color: #993333; font-weight: bold;">
      SELECT
     </span>
     ST_AsPNG
     <span style="color: #66cc66;">
      (
     </span>
     rast
     <span style="color: #66cc66;">
      )
     </span>
     <span style="color: #993333; font-weight: bold;">
      FROM
     </span>
     sometable;
