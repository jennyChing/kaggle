         <h2>
          <a aria-hidden="true" class="anchor" href="#latest-release" id="user-content-latest-release">
           <span class="octicon octicon-link">
           </span>
          </a>
          Latest Release
         </h2>
         <p>
          The latest version of the module is v3.0.5. Refer the
          <a href="https://github.com/kartik-v/yii2-grid/blob/master/CHANGE.md">
           CHANGE LOG
          </a>
          for details.
         </p>
         <p>
          New features with release 2.7.0.
         </p>
         <ol>
          <li>
           A brand new column
           <code>
            ExpandRowColumn
           </code>
           has been added which allows one to expand grid rows, show details, or load content via ajax. Check the
           <a href="http://demos.krajee.com/grid#expand-row-column">
            ExpandRowColumn documentation
           </a>
           for further details. The features available with this column are:
           <ul>
            <li>
             Ability to expand grid rows and show a detail content in a new row below it like a master detail record.
            </li>
            <li>
             Allows configuring the column like any grid DataColumn. The value of the column determines if the row is to be expanded or collapsed by default.
            </li>
            <li>
             Allows you to configure/customize the expand and collapse indicators.
            </li>
            <li>
             Ability to configure only specific rows to have expand/collapse functionality.
            </li>
            <li>
             Ability to disable the expand / collapse behavior and indicators for selective rows.
            </li>
            <li>
             Allows you to configure the detail content markup directly in the column configuration (using
             <code>
              detail
             </code>
             property). This can be set as a HTML markup directly or via Closure callback using column parameters.
            </li>
            <li>
             Allows you to load the detail content markup via ajax. Set the
             <code>
              detailUrl
             </code>
             property directly or via a Closure callback using column parameters.
            </li>
            <li>
             Automatically caches the content loaded via ajax so that the content is rendered from local on toggling the expand/collapse indicators, until the grid state is changed via filtering, sorting, or pagination.
            </li>
            <li>
             Ability to batch expand or batch collapse grid rows from the header. If content is loaded via ajax, the batch expand and collapse will fire the ajax requests to load and use intelligently from cache where possible.
            </li>
           </ul>
          </li>
          <li>
           Included
           <code>
            prepend
           </code>
           and
           <code>
            append
           </code>
           settings within
           <code>
            pageSummaryOptions
           </code>
           to prepend/append content to page summary.
          </li>
          <li>
           All asset (JS &amp; CSS) files have been carefully isolated to only load them if the specific grid feature has been selected.
          </li>
          <li>
           Enhancements for JS confirmation popups being hidden by browser's hide dialog settings.
          </li>
          <li>
           Recursively replace/merge PDF export configuration correctly.
          </li>
          <li>
           Include demo messages for auto generating via config.
          </li>
          <li>
           Allows grouping grid column data, including master detail groups and generating group summaries (since v3.0.5).
          </li>
         </ol>
         <blockquote>
          <p>
           NOTE: This extension depends on other yii2 extensions based on the functionality chosen by you. It will not install such dependent packages by default, but will prompt through an exception, if accessed.
For example, if you choose to enable PDF export, then the
           <a href="http://demos.krajee.com/mpdf">
            yii2-mpdf
           </a>
           will be mandatory and exception will be raised if
           <code>
            yii2-mpdf

           is not installed.
Check the
           <a href="https://github.com/kartik-v/yii2-grid/blob/master/composer.json">
            composer.json
           </a>
           for other extension dependencies.
          </p>

         <h2>
          <a aria-hidden="true" class="anchor" href="#gridview" id="user-content-gridview">
           <span class="octicon octicon-link">
           </span>
          </a>
          GridView
         </h2>
         <h3>
          <a aria-hidden="true" class="anchor" href="#kartikgridgridview" id="user-content-kartikgridgridview">
           <span class="octicon octicon-link">
           </span>
          </a>
          \kartik\grid\GridView
         </h3>
         <p>
          The following functionalities have been added/enhanced:
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#table-styling-enhanced" id="user-content-table-styling-enhanced">
           <span class="octicon octicon-link">
           </span>
          </a>
          Table Styling (Enhanced)
         </h3>
         <p>
          Control various options to style your grid table. Added
          <code>
           containerOptions
          </code>
          to customize your grid table container. Enhancements for grid and columns to work with yii\widgets\Pjax.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#grid-grouping-new" id="user-content-grid-grouping-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Grid Grouping (New)
         </h3>
         <p>
          With release v3.0.5, the module allows grouping of GridView data by setting various
          <code>
           group
          </code>
          related properties at the
          <code>
           kartik\grid\DataColumn
          </code>
          level. The following functionalities are supported:
         </p>
         <ul>
          <li>
           Ability to group and merge similar data for each column.
          </li>
          <li>
           Allow multi level/complex grouping and making a sub group dependent on a parent group.
          </li>
          <li>
           Allow displaying grouped data as a separate grouped row above the child rows.
          </li>
          <li>
           Allow configuring and displaying of group level summary rows.
          </li>
          <li>
           Summaries can be setup as a group footer OR a group header.
          </li>
          <li>
           Summaries intelligently embed between sub-groups and parent groups.
          </li>
          <li>
           Summaries can include auto calculated values (for numbers) at runtime based on previous child column data.
          </li>
          <li>
           Summaries can include advanced calculations using a javascript callback configuration.
          </li>
          <li>
           Ability to merge columns in the summary group header or footer.
          </li>
          <li>
           Complex configurations of groups will allow -  group properties to be set dynamically using Closure.
          </li>
          <li>
           Allow you to style your group cells in various ways including setting odd and even row CSS properties.
          </li>
         </ul>
         <h3>
          <a aria-hidden="true" class="anchor" href="#pjax-settings-new" id="user-content-pjax-settings-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Pjax Settings (New)
         </h3>
         <p>
          Inbuilt support for Pjax. Enhancements for grid and columns to work with
          <code>
           yii\widgets\Pjax
          </code>
          . Auto-reinitializes embedded javascript plugins when GridView is refreshed via Pjax. Added
          <code>
           pjax
          </code>
          property to enable pjax and
          <code>
           pjaxSettings
          </code>
          to customize the pjax behavior.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#custom-header--footer-new" id="user-content-custom-header--footer-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Custom Header &amp; Footer (New)
         </h3>
         <p>
          Add custom header or footer rows, above / below your default grid header and footer.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#resizing-columns-new" id="user-content-resizing-columns-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Resizing Columns (New)
         </h3>
         <p>
          Allows resizing of the columns just like a spreadsheet (since v3.0.0). Uses the
          <a href="https://github.com/dobtco/jquery-resizable-columns">
           JQuery ResizableColumns plugin
          </a>
          for resize and
          <a href="https://github.com/marcuswestin/store.js/">
           store.js
          </a>
          for localStorage persistence.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#floating-header-new" id="user-content-floating-header-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Floating Header (New)
         </h3>
         <p>
          Allows the grid table to have a floating table header. Uses the
          <a href="http://mkoryak.github.io/floatThead">
           JQuery Float THead plugin
          </a>
          to display a seamless floating table header.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#panel-new" id="user-content-panel-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Panel (New)
         </h3>
         <p>
          Allows configuration of GridView to be enclosed in a panel that can be styled as per  Bootstrap 3.x. The panel will enable configuration of  various
sections to embed content/buttons, before and after header, and before and after footer.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#toolbar-new" id="user-content-toolbar-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Toolbar (New)
         </h3>
         <p>
          The grid offers ability to configure toolbar for adding various actions. The default templates place the toolbar in the
          <code>
           before
          </code>
          section of the
          <code>
           panel
          </code>
          . The toolbar is by default styled using Bootstrap button groups. Some of the default actions like the
          <code>
           export
          </code>
          button is by default appended to the toolbar. 
With version v2.1.0, if you are using the
          <code>
           yii2-dynagrid
          </code>
          extension it automatically displays the
          <strong>
           personalize
          </strong>
          ,
          <strong>
           sort
          </strong>
          , and
          <strong>
           filter
          </strong>
          buttons in the toolbar. The toolbar can be configured as a simple array. Refer the
          <a href="http://demos.krajee.com/grid">
           docs and demos
          </a>
          for details.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#grid-plugins-new" id="user-content-grid-plugins-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Grid Plugins (New)
         </h3>
         <p>
          The grid now offers ability to plugin dynamic content to your grid at runtime. A new property
          <code>
           replaceTags
          </code>
          has been added with v2.3.0. This allows you to specify tags which will be replaced dynamically at grid rendering time and wherever you set these tags in any of the grid layout templates.
         </p>
         <h3>
          <a aria-hidden="true" class="anchor" href="#page-summary-new" id="user-content-page-summary-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Page Summary (New)
         </h3>
         <p>
          This is a new feature added to the GridView widget. The page summary is an additional row above the footer - for displaying the
summary/totals for the current GridView page. The following parameters are applicable to control this behavior:
         </p>
         <ul>
          <li>
           <code>
            showPageSummary
           </code>
           :
           <em>
            boolean
           </em>
           whether to display the page summary row for the grid view. Defaults to
           <code>
            false
           </code>
           .
          </li>
          <li>
           <code>
            pageSummaryRowOptions
           </code>
           :
           <em>
            array
           </em>
           , HTML attributes for the page summary row. Defaults to
           <code>
            ['class' =&gt; 'kv-page-summary warning']
           </code>
           .
          </li>
         </ul>
         <h3>
          <a aria-hidden="true" class="anchor" href="#export-grid-data-new" id="user-content-export-grid-data-new">
           <span class="octicon octicon-link">
           </span>
          </a>
          Export Grid Data (New)
         </h3>
         <p>
          This is a new feature added to the GridView widget. It allows you to export the displayed grid content as HTML, CSV, TEXT, EXCEL, PDF, &amp; JSON. It uses the rendered grid data on client to convert to one of the format specified using JQuery. 
This is supported across all browsers. The PDF rendering is achieved through a separate extension
          <a href="http://demos.krajee.com/mpdf">
           yii2-mpdf
          </a>
          .
         </p>
         <p>
          Features offered by yii2-grid export:
         </p>
         <ul>
          <li>
           Ability to preprocess and convert column data to your desired value before exporting. There is a new property
           <code>
            exportConversions
           </code>
           that can be setup in GridView. 
For example, this currently is set as a default to convert the HTML formatted icons for BooleanColumn to user friendly text like
           <code>
            Active
           </code>
           or
           <code>
            Inactive
           </code>
           after export.
          </li>
          <li>
           Hide any row or column in the grid by adding one or more of the following CSS classes:
           <ul>
            <li>
             <code>
              skip-export
             </code>
             : Will skip this element during export for all formats (
             <code>
              html
             </code>
             ,
             <code>
              csv
             </code>
             ,
             <code>
              txt
             </code>
             ,
             <code>
              xls
             </code>
             ,
             <code>
              pdf
             </code>
             ,
             <code>
              json
             </code>
             ).
            </li>
            <li>
             <code>
              skip-export-html
             </code>
             : Will skip this element during export only for
             <code>
              html
             </code>
             export format.
            </li>
            <li>
             <code>
              skip-export-csv
             </code>
             : Will skip this element during export only for
             <code>
              csv
             </code>
             export format.
            </li>
            <li>
             <code>
              skip-export-txt
             </code>
             : Will skip this element during export only for
             <code>
              txt
             </code>
             export format.
            </li>
            <li>
             <code>
              skip-export-xls
             </code>
             : Will skip this element during export only for
             <code>
              xls
             </code>
             (excel) export format.
            </li>
            <li>
             <code>
              skip-export-pdf
             </code>
             : Will skip this element during export only for
             <code>
              pdf
             </code>
             export format.
            </li>
            <li>
             <code>
              skip-export-json
             </code>
             : Will skip this element during export only for
             <code>
              json
             </code>
             export format.
These CSS can be set virtually anywhere. For example
             <code>
              headerOptions
             </code>
             ,
             <code>
              contentOptions
             </code>
             ,
             <code>
              beforeHeader
             </code>
             etc.
            </li>
           </ul>
          </li>
          <li>
           With release v2.1.0, you can now merge additional action items to the export button dropdown.
          </li>
          <li>
           With release v2.3.0 the export functionality includes these additional features:
           <ul>
            <li>
             A separate export popup progress window is now shown for download.
            </li>
            <li>
             Asynchronous export process on the separate window - and avoid any grid refresh
            </li>
            <li>
             Set export mime types to be configurable
            </li>
            <li>
             Includes support for exporting new file types:
             <ul>
              <li>
               JSON export
              </li>
              <li>
               PDF export (using
               <code>
                yii2-mpdf
               </code>
               extension)
              </li>
             </ul>
            </li>
            <li>
             Adds functionality for full data export
            </li>
            <li>
             Enhance icons formatting for export file types (and beautify optionally using font awesome)
            </li>
            <li>
             Ability to hide entire column from export using
             <code>
              hiddenFromExport
             </code>
             property, but show them in normal on screen display.
            </li>
            <li>
             Ability to do reverse of above. Hide column in display but show on export using
             <code>
              hidden
             </code>
             property.
            </li>
           </ul>
          </li>
          <li>
           Adds ability to integrate a separate extension for full data export i.e.
           <a href="https://github.com/kartik-v/yii2-export">
            yii2-export
           </a>
           .
          </li>
         </ul>

          \kartik\grid\FormulaColumn
         </h3>
         <p>
          This is a new grid column class that extends the \kartik\grid\DataColumn class. It allows calculating formulae just like in spreadsheets - based on
values of other columns in the grid. The formula calculation is done at grid rendering runtime and does not need to query the database. Hence you can use formula columns

           documentation
          </a>
          for details.
