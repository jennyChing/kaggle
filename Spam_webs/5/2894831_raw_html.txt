        <h1 class="entry-title" itemprop="headline">
         Whats the best way to produce an ebook?
        </h1>
        <p class="entry-meta">
         By
         <span class="entry-author" itemprop="author" itemscope="itemscope" itemtype="http://schema.org/Person">
          <a class="entry-author-link" href="http://rogerpacker.com/author/admin/" itemprop="url" rel="author">
           <span class="entry-author-name" itemprop="name">
            Roger Packer
           </span>
          </a>
         </span>
        </p>
       </header>
       <div class="entry-content" itemprop="text">
        <p>
         <img alt="chimney1722x1024" class="alignright wp-image-1764" data-recalc-dims="1" src="http://i2.wp.com/rogerpacker.com/wp-content/uploads/2014/01/chimney1722x1024.jpg?resize=346%2C491"/>
         A lot of people ask me whats the best way to produce ebooks so I thought I would set out here a few ways of getting the job done.
        </p>

        <h2>
         <span id="Using_a_word_processor_and_uploading_for_conversion">
          Using a word processor and uploading for conversion
         </span>
        </h2>
        <p>
         The most basic way of doing it is to use Word or Open Office to produce a doc or docx file and upload that to
         <a href="https://kdp.amazon.com/" target="_blank">
          Kindle Direct Publishing
         </a>
         or a distributor such as
         <a href="http://www.smashwords.com" target="_blank">
          Smashwords
         </a>
         or
         <a href="http://www.draft2digital.com" target="_blank">
          Draft2digital
         </a>
         which will do the conversion for you.
        </p>
        <p>
         Ive never used this way of conversion as I prefer to build books myself so Im sure theyre going to look like I want them to look but a lot of people do just upload their doc files and seem happy with the conversion results.
         <br/>
         <span id="more-1757">
         </span>
        </p>
        <h2>
         <span id="Jutoh">
          Jutoh
         </span>
        </h2>
        <p>
         <strong>
          <a href="http://www.jutoh.com" target="_blank">
           <img alt="jutoh2" class="alignright size-medium wp-image-1758" data-recalc-dims="1" src="http://i2.wp.com/rogerpacker.com/wp-content/uploads/2014/01/jutoh2.png?resize=300%2C211"/>
          </a>
          The main program I use is
          <a href="http://www.jutoh.com" target="_blank">
           Jutoh
          </a>
          , which is brilliant software produced by an independent company, run and programmed largely by one man, Dr Julian Smart.
         </strong>
        </p>
        <p>
         Its very reasonably priced at 24 and has versions for Windows, Mac and Linux. Theres a free book, called Creating Great Books With Jutoh which is a very good and complete guide to using the software.
        </p>
        <p>
         Jutoh isnt the easiest program to use initially as it doesnt have the familiarity of some software but once youve used it a few times youll get the hang of it.
        </p>
        <p>
         Ebooks are basically HTML (or XHTML) files and Jutoh could be thought of as being a powerful HTML editor with CSS capabilities and word processing thrown in. I often use it now to build a book from the beginning, writing the book within Jutoh, although this really isnt what its designed for.
        </p>
        <blockquote>
         <p>
          <strong>
           The general way of doing things is to write your book using a word processor such as Word or Open Office and then import that file into Jutoh. Although Jutoh imports a lot of file formats, it doesnt import doc files but its preferred import format is docx, so you can use Word. It originally specialised in odt files imported from Open Office.
          </strong>
         </p>
        </blockquote>
        <p>
         Jutohs way of working, which is similar to most ebook production software, is to split a book file into sections or chapters, according to whatever criteria you set, so youll have a series of chapters which will be compiled into an ebook.
        </p>
        <p>
         One of the great things about Jutoh is you can compile and output for a wide range of ebook formats, including mobi (used by Amazon) and epub (used by most everyone else). Most ebook production programs offer only output to the epub format and do not cover mobi.
        </p>
        <h2>
         <span id="Adobe_InDesign">
          Adobe InDesign
         </span>
        </h2>
        <p>
         <strong>
          <a href="http://www.adobe.com/uk/products/indesign.html" target="_blank">
           <img alt="indesign1" class="alignright size-medium wp-image-1759" data-recalc-dims="1" src="http://i1.wp.com/rogerpacker.com/wp-content/uploads/2014/01/indesign1.png?resize=300%2C243"/>
          </a>
          Other ebook production software includes
          <a href="http://www.adobe.com/uk/products/indesign.html">
           Adobe InDesign
          </a>
          . If youve ever used this in producing print projects, where its the market leader, then youll find it fairly easy to use for ebooks and it really is a powerhouse in terms of its range of functions.
         </strong>
        </p>
        <p>
         A lot of people swear by it and produce all their ebooks with it. I do use it on occasions but although I think its a great program for print, I find it has some drawbacks for ebooks, including the fact that you cant output to mobi format direct from the program (at least with the Creative Cloud version) and compiling a table of contents can be a tedious task.
        </p>
        <p>
         InDesign was built as a print layout program and has been adapted for ebooks so I feel there are some uneasy compromises while a program like Jutoh was designed to produce ebooks.
        </p>
        <blockquote>
         <p>
          <strong>
           Theres also the drawback of cost. InDesign used to cost about 1,000 to buy outright but Adobe have given up on that and now only offer it as part of their Creative Cloud rental which costs around 25 a month if you only want the InDesign program and 40 a month or so if you want Photoshop, etc.
          </strong>
         </p>
        </blockquote>
        <p>
         I suppose you could say if you only want it for a month then its reasonable but it can be annoying if you want to make a few changes to a book that youd have to go and reactivate the program and pay another 25.
        </p>
        <h2>
         <span id="Scrivener">
          Scrivener
         </span>
        </h2>
        <p>
         <strong>
          <a href="http://www.literatureandlatte.com" target="_blank">
           <img alt="scrivener2" class="alignright size-medium wp-image-1761" data-recalc-dims="1" src="http://i0.wp.com/rogerpacker.com/wp-content/uploads/2014/01/scrivener2.png?resize=300%2C195"/>
          </a>
          <strong>
           <a href="http://literatureandlatte.com/scrivener.php" target="_blank">
            Scrivener
           </a>
          </strong>
          is a great word processor program that also excels at ebook production. Its a bargain at about 30 and is produced by an independent company.
         </strong>
        </p>
        <p>
         Its very popular with a lot of writers and has become a cult program. Its fast to compile and export ebooks, including both mobi and epub formats, and also has lots of bells and whistles to help writers plan books.
        </p>
        <p>
         If you havent used Scrivener before then you would need to get familiar with it as there are a lot of differences between it and word processors such as Word.
        </p>
        <blockquote>
         <p>
          <strong>
           Scrivener, like Jutoh, operates using separate items such as chapters or sections within an overall file. I found this confusing initially but once I got used to it I realised how much better it is and now I find Word to be rather limiting.
          </strong>
         </p>
        </blockquote>
        <p>
         Scrivener offers a PDF manual which you can download which covers how to use the program and there are several third-party guides, including Scrivener for Dummies, available as print or ebooks.
        </p>
        <h2>
         <span id="The_importance_of_setting_styles">
          The importance of setting styles
         </span>
        </h2>
        <p>
         <strong>
          With all these programs, including Word or Open Office, its important that you set and name consistent styles, including paragraph and character styles, as that is the key to producing good ebooks.
         </strong>
        </p>
        <p>
         Everything in an ebook should be linked to a set style. Even if you want, say, a single word made bold in the middle of a sentence you should apply a set character style to do this rather than simply selecting the text and clicking on the bold icon.
        </p>
        <h2>
         <span id="Go_with_the_flow_of_ebooks">
          Go with the flow of ebooks
         </span>
        </h2>
        <p>
         <strong>
          An ebook is very different from a print book. Ive found that some writers get confused and think they can place items precisely as they would in a print book.
         </strong>
        </p>
        <p>
         Its best to think of an ebook as a flow of words that can change course and appearance depending on what device is being used for viewing. Remember, you dont know what device your reader will be using, it could be anything, including a desktop computer monitor, a full-size iPad, an iPad mini, a Kindle Fire, a Kindle Paperwhite or any make of smartphone. Thats one of the great beauties of ebooks, you can read them on any device with the right app.
        </p>
        <p>
         You can produce so-called fixed-layout ebooks but there are so many disadvantages theyre not worth considering unless youre publishing picture books.
        </p>
        <h2>
         <span id="Vellum">
          Vellum
         </span>
        </h2>
        <p>
         There is another ebook program which has been recently released called
         <a href="http://180g.co/vellum/" target="_blank">
          Vellum
         </a>
         which Im intending to review soon. It looks promising at first glance but I was dismayed to learn that although the program itself is free, it costs $50 per book to generate an ebook.
        </p>
        <p>
         Thats disappointing as it isnt far off the price you would pay for a third-party ebook service to convert your book for you and with Vellum youre still doing the work yourself. There are discounts, so you can pay $150 for 10 books or $300 for unlimited use.
        </p>
        <h2>
         <span id="Sigil">
          Sigil
         </span>
        </h2>
        <p>
         I should also mention
         <a href="http://code.google.com/p/sigil/downloads/detail?name=Sigil-0.7.4-Mac-Package.dmg" target="_blank">
          Sigil
         </a>
         , which is a free open source program that can produce epub files. This was promising when I tried it out some time ago but does have the drawback of only dealing with epubs. There were some problems over whether the program would continue last year but a new update appeared in October so it could be worth a try.
        </p>

           </a>
           covers more general issues of ebook production which you might find of use.
