          Cortana
         </a>
         announcements, putting our sultry temptress in the hands of all consumers whether they use a phone, desktop or tablet.
        </p>
        <p>
         Cortana gives us the best personal assistant money can buy, but the system also offers a very extensive speech system that is available to us humble developers.
        </p>
        <p>
         So what can Cortana do for our apps and games?
        </p>
        <ul>
         <li>
          Provide a full text to speech system, complete with custom grammar.

         </li>

        <p>
         Creating a VCD file is very easy. First, create a new XML document in your solution (
         <span class="italic">
          Right click -&gt; Add new Item -&gt; XML File

         ) and then adding basic VCD content to the new XML file, I created a

         &lt;?xml version="1.0" encoding="utf-8"?&gt;
        &lt;VoiceCommands xmlns="http://schemas.microsoft.com/voicecommands/1.1"&gt;

          &lt;CommandSet xml:lang="en-gb"&gt;
            &lt;CommandPrefix&gt;Unity Battle&lt;/CommandPrefix&gt;

            &lt;Command Name="startWithText"&gt;
              &lt;Example&gt;Say the message you want to appear in the game&lt;/Example&gt;
              &lt;ListenFor&gt; {*} &lt;/ListenFor&gt;
              &lt;Feedback&gt; Starting Game... &lt;/Feedback&gt;

            &lt;/Command&gt;

          &lt;/CommandSet&gt;

        &lt;/VoiceCommands&gt;

         In this XML we first define the
         <span class="bold">
          CommandSet.

         <span class="bold">
          Key Word / CommandPrefix

         the player will use to identify your application through voice, in this case "
         <span class="bold">

         <span class="bold">
          App.xaml.CS
         </span>

         <span class="bold">
          <span class="italic">

         protected override async void OnLaunched(LaunchActivatedEventArgs args)
        {
          //Install VCD command file
          Uri voiceCommandUri = new Uri("ms-appx:///VoiceCommandDefinition.xml");
          var storageFile = await Windows.Storage.StorageFile.GetFileFromApplicationUriAsync(voiceCommandUri);
          await Windows.Media.SpeechRecognition.VoiceCommandManager.InstallCommandSetsFromStorageFileAsync(storageFile);
        </pre>
        <p>
         This simply locates your VCD xml file, loads it into memory and then installs it using the
         <span class="bold">
          VoiceCommandManager
         </span>
         . This happens each and every time the app is launched to ensure it is always using the latest version.
        </p>
        <p>
         <span class="bold">
          Setup the Command Handler
         </span>
        </p>
        <p>
         Next we'll create a new C# script called
         <span class="bold">
          SpeechHelper
         </span>
         in the
         <span class="bold">
          Windows Phone
         </span>
         project to contain all the code for the speech integration. Once you have created the script, replace its contents with the following:
        </p>
        <pre class="c#" id="" name="code" style="">
         using System;
        using Windows.ApplicationModel.Activation;

        namespace CortanaUnity
        {
          public class SpeechHelper
          {
            public static void HandleSpeechCommand(IActivatedEventArgs args)
            {
              var commandArgs = args as Windows.ApplicationModel.Activation.VoiceCommandActivatedEventArgs;
              Windows.Media.SpeechRecognition.SpeechRecognitionResult speechRecognitionResult = commandArgs.Result;

              string textSpoken = speechRecognitionResult.Text;

              CortanaInterop.CortanaText = textSpoken;
            }
          }
        }
        </pre>
        <p>
         There is a fair amount to digest here, but putting it simply we declare a new instance of the
         <span class="bold">
          VoiceCommandActivatedEventArgs
         </span>
         which interrogates the launch conditions of the device, which we then gather the result of in the
         <span class="bold">
          speechRecognitionResult
         </span>
         parameter.
        </p>
        <p>
         We then take the spoken text result from this and pass it to our static
         <span class="bold">
          CortanaText
         </span>
         property in the
         <span class="bold">
          Unity Interop
         </span>
         class, thus making the text available to the Unity application when it finally starts.
        </p>
        <p>
         <span class="bold">
          Register the call handler on start-up
         </span>
        </p>

         As the final piece of the puzzle, we simply need to tell our application that when it is starting it should use our speech helper class to check for and validate any speech input commands.
        </p>
        <p>
         To do this we return to our
         <span class="bold">
          App.xaml.cs
         </span>
         file and locate the
         <span class="bold">
          <span class="italic">
           OnActivated
          </span>
         </span>
         method. In this method Unity already checks to see if a Unity app was launched via a custom URL (or protocol). To this we need to add a case for our
         <span class="bold">
          VoiceCommand
         </span>
         start method, as follows:
        </p>
        <pre class="c#" id="" name="code" style="">
         protected override void OnActivated(IActivatedEventArgs args)
        {
          string appArgs = "";
          Windows.ApplicationModel.Activation.SplashScreen splashScreen = null;
          switch (args.Kind)
          {
            case ActivationKind.Protocol:
              ProtocolActivatedEventArgs eventArgs = args as ProtocolActivatedEventArgs;
              splashScreen = eventArgs.SplashScreen;
              appArgs += string.Format("Uri={0}", eventArgs.Uri.AbsoluteUri);
            break;
            
            //Add VoiceCommand start detection and use the SpeechHelper handler
            case ActivationKind.VoiceCommand:
              SpeechHelper.HandleSpeechCommand(args);
            break;
          }
          InitializeUnity(appArgs, splashScreen);
        }
        </pre>

         In the above code, you can see by the commented line that when a voice launch is detected, it calls our new handler method we created in our
         <span class="bold">
          SpeechHelper
         </span>
         class.

        <p>
         <span class="bold">
          Dictation. I did spell that right, didn't I?
         </span>
        </p>
        <p>
         We have used a voice command to get text into our game when it wasn't running. Fairly useful, especially if you want some shortcuts for players to use when launching your game. However, we can also use these fancy speech recognition features while the game is running as well. This could be as simple as a dictation mode the player can use when submitting feedback (we wouldn't want their fingers to get tired now would we?) or it could be another way of the player interacting with the game, like commands to open doors, cast spells, etc. (I'll let your creative imagination go wild on that one).
        </p>
        <p>
         It doesn't take much to enable this mode, so long as you stick to the Interop design patterns. For this we need to have an event / trigger in our Interop class to tell the device we want some speech recognised, then in the Windows Phone project we simply hook to this event and kick off the speech recognition before then feeding the translated result back to Unity.
        </p>
        <p>
         <span class="bold">
          Creating the Unity hook
         </span>
        </p>
        <p>
         First, open up Unity and edit the
         <span class="bold">
          CortanaInterop
         </span>
         class. Once open we need to add the following static event code, which is the interface action for our speech recognition:
        </p>
        <p>
         (We'll be reusing the same text property from the previous example to feed the recognised speech back, but you can create another if you wish)
        </p>
        <pre class="c#" id="" name="code" style="">
         public static bool SpeechInProgress;
 
        public static event EventHandler SpeechRequested;
        public static void GetMeSomeVoice()
        {
          if (SpeechRequested != null &amp;&amp; !SpeechInProgress)
          {
            SpeechRequested(null, null);
          }
        }
        </pre>
        <p>
         This simply creates an event hook, so that when some code in Unity calls the
         <span class="bold">
          <span class="italic">
           GetMeSomeVoice
          </span>
         </span>
         method, it passes that request down to anyone listening for the event (which we'll do in the "Stitching everything together" section a bit further down).  If no one is listening we do nothing, because if you call an event without a subscriber, you get a nasty error.
        </p>
        <p>
         We also have an additional property called
         <span class="bold">
          <span class="italic">
           SpeechInProgress
          </span>
         </span>
         . This is there because the speech recognition is an asynchronous process on the device and can cause issues if you try to request multiple operations. To ensure you only ask once and cannot ask again until the last operation was complete, we have a flag.
        </p>
        <p>
         Now build and export your project again before continuing.
        </p>
        <p>
         <span class="bold">
          Initialising speech recognition
         </span>
        </p>
        <p>
         Switching over to the Windows Phone project, the first thing you need to do is
         <span class="bold">
          BUILD IT
         </span>
         . This is simply because the Windows Phone solution needs to compile the Interop code before it is made available in the OS project. If you know your code by heart, you can start coding straight away. Then when the project next builds, everything should line up (however, I like IntelliSense too much and don't trust my spellings :S).
        </p>
        <p>
         The first thing we need is a method to initialise the speech recognition system for the current language of the user's device.
        </p>
        <p class="textCenter borderTop1 borderBottom1 block padding20 margin10 cyanDark">
         <span class="italic">
          If you have read the official MS docs, they put all the code together for initialising and running the speech recognition. In my experience, however, this causes lag and other issues with Unity projects. It's best to split them up.
         </span>
        </p>
        <p>
         Open up the
         <span class="bold">
          SpeechHandler
         </span>
         class and add the following property:
        </p>
        <pre class="c#" id="" name="code" style="">
         private static Windows.Media.SpeechRecognition.SpeechRecognizer speechRecognizer;
        </pre>
        <p>
         This is our reference to the speech engine, which we'll keep alive while our game is running. Next add the following
         <span class="bold">
          <span class="italic">
           InitialiseSpeechRecognition
          </span>
         </span>
         method:
        </p>
        <pre class="c#" id="" name="code" style="">
         public static async System.Threading.Tasks.Task InitialiseSpeechRecognition()
        {
          // Create an instance of SpeechRecognizer.
          speechRecognizer = new Windows.Media.SpeechRecognition.SpeechRecognizer();

          // Add a web search grammar to the recognizer.
          var dictationGrammar = new Windows.Media.SpeechRecognition.SpeechRecognitionTopicConstraint(Windows.Media.SpeechRecognition.SpeechRecognitionScenario.Dictation, "dication");

          speechRecognizer.UIOptions.AudiblePrompt = "Please give us your feedback...";
          speechRecognizer.UIOptions.ExampleText = @"This game is awesome, rate me 5 stars";
          speechRecognizer.UIOptions.IsReadBackEnabled = false;
          speechRecognizer.Constraints.Add(dictationGrammar);

          // Compile the dictation grammar by default.
          await speechRecognizer.CompileConstraintsAsync();
        }
        </pre>
        <p>
         Here we create a new instance of the
         <span class="bold">
          <span class="italic">
           SpeechRecognizer
          </span>
         </span>
         and then define our default grammar (the operation mode for the speech recognition). We then add any additional options we want to apply, including the text to appear on screen and what you want to say to the user. I also disabled the reading back of the captured speech because I found it too annoying. The final setup is of course up to you and what fits your title.
        </p>
        <p class="textCenter borderTop1 borderBottom1 block padding20 margin10 cyanDark">
         <span class="italic">
          For more information on the
          <span class="bold">
           SpeechRecognizer
          </span>
          , see this quick start guide from MSDN:
          <a class="hoverOrange" href="https://msdn.microsoft.com/en-us/library/windows/apps/xaml/dn630427.aspx" id="" onclick="dcsSetVar('WT.z_ea_name','Making your game stand out with Cortana - Body - SpeechRecognizer - Local','WT.z_ea_actionoffer','Vue-Sit','WT.z_ea_msproduct','Unknown','WT.z_ea_group','Articles','WT.z_ea_funnel','Awareness','WT.z_ea_targetcampaign','000XXX00000');MscomCustomEvent('ms.ea_offer','sit','ms.ea_action','vue','ms.ea_name','Making your game stand out with Cortana - Body - SpeechRecognizer - Unknown - Awareness - Local','ms.cpgnguid','000XXX00000','ms.lang','en','ms.loc','gb');  " target="_blank" title="SpeechRecognizer">
           https://msdn.microsoft.com/en-us/library/windows/apps/xaml/dn630427.aspx
          </a>
         </span>
        </p>
        <p>
         <span class="bold">
          Letting the player get a word in
         </span>
        </p>
        <p>
         The next method is what will be used by the Unity Interop to perform speech collection and recognition, before sending the result back to the Unity Interop. This new
         <span class="bold">
          <span class="italic">
           StartRecognising
          </span>
         </span>
         method is as follows:
        </p>
        <pre class="c#" id="" name="code" style="">
         public static async void StartRecognizing(object sender, EventArgs e)
        {
          CortanaInterop.SpeechInProgress = true;

          // Start recognition.
          Windows.Media.SpeechRecognition.SpeechRecognitionResult speechRecognitionResult = await speechRecognizer.RecognizeWithUIAsync();

          // Do something with the recognition result.
          CortanaInterop.CortanaText = speechRecognitionResult.Text;
          CortanaInterop.SpeechInProgress = false;
        }
        </pre>
        <p>
         Here we set the Interop
         <span class="bold">
          <span class="italic">
           SpeechInProgress
          </span>
         </span>
         flag to stop Unity from sending multiple requests, and then kick off Cortana in all her glory.
        </p>
        <p class="textCenter borderTop1 borderBottom1 block padding20 margin10 cyanDark">
         <span class="italic">
          You will also notice the
          <span class="bold">
           async
          </span>
          and
          <span class="bold">
           await
          </span>
          keywords, which are a sign of the new
          <span class="bold">
           Async programming model
          </span>
          introduced in Windows Phone 8. This basically allows for more efficient background processing of multiple tasks without overloading the frontend UI.
         </span>
        </p>
        <p>
         Once Cortana is finished speaking with the user, we grab the result, pass it back to Unity and allow further requests.
        </p>
        <p>
         <span class="bold">
          Stitching everything together
         </span>
        </p>
        <p>
         Now that we have methods to do all the work, we just need to hook everything up. For the speech initialisation, we just need to add that to the
         <span class="bold">
          App.xaml.cs
          <span class="italic">
           OnLaunched
          </span>
         </span>
         method, just after the code to install the VCD file for the voice commands using:
        </p>
        <pre class="c#" id="" name="code" style="">
         //Initialise Speech recognition
        await SpeechHelper.InitialiseSpeechRecognition();
        </pre>
        <p>
         Next we need to bind our Unity Interop event to our new
         <span class="bold">
          <span class="italic">
           StartRecognizing
          </span>
         </span>
         method in the
         <span class="bold">
          SpeechHelper
         </span>
         class. To do this, we need to add it to the code for the XAML page that the Unity game actually runs on, and by default this is the page called
         <span class="bold">
          MainPage.xaml.cs
         </span>
         . Open this file and locate the constructor for the page which looks like this:
        </p>
        <pre class="c#" id="" name="code" style="">
         public MainPage(SplashScreen splashScreen)
        {
        </pre>
        <p>
         To this method we need to add the following code to register our speech function to the Unity Interop event:
        </p>
        <pre class="c#" id="" name="code" style="">
         CortanaInterop.SpeechRequested += SpeechHelper.StartRecognizing;
        </pre>
        <p>
         With this, whenever a script in Unity calls the
         <span class="bold">
          <span class="italic">
           GetMeSomeVoice
          </span>
         </span>
         method of the
         <span class="bold">
          CortanaInterop
         </span>
         class, it launches the event. This is picked up by the Windows Phone platform, which in turn calls the
         <span class="bold">
          <span class="italic">
           StartRecognizing
          </span>
         </span>
         function and kicks off speech. Now breathe!
        </p>
        <p>
         It's just a simple change of events to jump from Unity to the device platform and back again.
        </p>
        <p>
         In the example, I bound a new UI button to the
         <span class="bold">
          GetMeSomeVoice
         </span>
         method, so that when you run the app on a device (and it has to be a device because the emulator just won't cut it), the speech prompt appears and then my text gets back to Unity.
        </p>
        <p class="center maxWidth500">
         <img alt="The speech prompt appears" class="" src="/en-gb/developers/images/articles/content/562/07.png?v=0.0" style=""/>
        </p>
        <hr/>
        <p>
         <span class="bold">
          Giving the player some Jip
         </span>
        </p>
        <p>
         So far we've covered launching with voice and letting the user interact with voice while the game is running. However, Cortana also has a voice of her own which we can use to badger the player. Normally in games when we want characters to speak to the player we record a media file for speech and then play it back to the user at the appropriate time. This pattern has one unique flaw; it is bound to the language you recorded it in. If you want to support multiple languages, you need to re-record the audio in all the languages you want to support and ship ALL of them with your title, unless you use regional asset bundles which requires Unity Pro.
        </p>
        <p>
         Cortana, however, has as many languages as Windows Phone supports. All you need to do is pass her the text you want spoken (granted this must be natural language text as she won't translate for you) and she will voice it, even in regional dialects in some cases. The only drawback is that there is only one voice and the user selects which voice that is.
        </p>
        <p>
         Setting up the speech synthesis side is very similar to the voice input of the previous example, so let's whip through it quickly.
        </p>
        <p>
         <span class="bold">
          Yet more Interop
         </span>
        </p>
        <p>
         Another command/action, another Interop event to kick it off. Similar to the previous approach, add the following to the
         <span class="bold">
          CortanaInterop
         </span>
         class in your
         <span class="bold">
          Unity
         </span>
         project:
        </p>
        <pre class="c#" id="" name="code" style="">
         public static event EventHandler SpeechSynthRequested;
        public static void YellAtPlayer(string textToSpeak)
        {
          if (SpeechSynthRequested != null &amp;&amp; !SpeechInProgress)
          {
            SpeechSynthRequested(textToSpeak, null);
          }
        }
        </pre>
        <p>
         I have added a new
         <span class="bold">
          <span class="italic">
           YellAtPlayer
          </span>
         </span>
         method, to which I've also added a string parameter. This parameter specifies the text that you want Cortana to read out. When you call this method from Unity, it fires the
         <span class="bold">
          <span class="italic">
           SpeechSynthRequested
          </span>
         </span>
         event to tell the device to process the message, sending the text along with the request.
        </p>
        <p>
         <span class="bold">
          Initialise Speech Synthesis
         </span>
        </p>
        <p>
         Like with the speech recognition, it's a good idea to initialise the speech synthesis when the app starts, so that it's ready when required. For this we'll add an
         <span class="bold">
          <span class="italic">
           InitialiseSpeechSynthesis
          </span>
         </span>
         method and some new properties to our
         <span class="bold">
          SpeechHelper
         </span>
         class:
        </p>
        <pre class="c#" id="" name="code" style="">
         private static Windows.Media.SpeechSynthesis.SpeechSynthesizer synth;
        private static MediaElement mediaElement;

        public static void InitialiseSpeechSynthesis(MediaElement MediaElement)
        {
          // The media object for controlling and playing audio.
          mediaElement = MediaElement;

          // The object for controlling the speech synthesis engine (voice).
          synth = new Windows.Media.SpeechSynthesis.SpeechSynthesizer();
        }
        </pre>
        <p>
         Here we simply initialise all the relevant components, ready for use.
        </p>
        <p class="textCenter borderTop1 borderBottom1 block padding20 margin10 cyanDark">
         <span class="italic">
          As part of this example, I have used the audio output from the phone to play the output of the synthesis. If you wish you can bypass this and pass the bytestream that the speech synthesis generates back to unity to play. The choice is up to you.
         </span>
        </p>
        <p>
         <span class="bold">
          Speech Synth Action
         </span>
        </p>
        <p>
         The other method we need in our
         <span class="bold">
          SpeechHelper
         </span>
         class will do the actual conversion of text in to voice and play it, so add the following method to the
         <span class="bold">
          SpeechHelper
         </span>
         class as well:
        </p>
        <pre class="c#" id="" name="code" style="">
         public static async void StartSpeaking(object sender, EventArgs e)
        {
          var textToSpeech = (string)sender;
          if (mediaElement != null &amp;&amp; !string.IsNullOrEmpty(textToSpeech))
          {
            // Generate the audio stream from plain text.
            SpeechSynthesisStream stream = await synth.SynthesizeTextToStreamAsync(textToSpeech);
            App.appCallbacks.InvokeOnUIThread(() =&gt; {
            // Send the stream to the media object.
            mediaElement.SetSource(stream, stream.ContentType);
            mediaElement.Play();
            }, true);
          }
        }
        </pre>
        <p>
         Here we check if we have actually been sent any text (as there's no point in converting white noise) and use the asynchronous
         <span class="bold">
          <span class="italic">
           SynthesizeTextToStreamAsync
          </span>
         </span>
         operation to convert the text to audio. The audio stream that this method generates is then played back through a XAML media element on the device (though as mentioned previously, you could just return this stream to Unity to play if you wished).
        </p>
        <p class="textCenter borderTop1 borderBottom1 block padding20 margin10 cyanDark">
         <span class="italic">
          Because asynchronous operations are performed on a background thread, they are returned on a background thread. If you then want to play them, you need to send them to the UI thread to be actioned. Not doing so will crash and burn your app in a horrible "access violation" error.
         </span>
        </p>
        <p>
         The last thing we need to do to make this script run is to update one of the auto-generated pieces of code that Unity provides. This enables us to access the
         <span class="bold">
          <span class="italic">
           appCallbacks
          </span>
         </span>
         function of the Unity Engine. Open the
         <span class="bold">
          App.xaml.cs
         </span>
         class and update the line at the top of the file to:
        </p>
        <pre class="c#" id="" name="code" style="">
         internal static AppCallbacks appCallbacks;
        </pre>
        <p>
         This makes the property available for just the internals of our application to use.
        </p>
        <p>
         <span class="bold">
          The last of the wiring
         </span>
        </p>
        <p>
         To finish off this little puzzle, we need to hook everything up in the same way as before. This time we only need to do this from the
         <span class="bold">
          MainPage
         </span>
         , so open
         <span class="bold">
          MainPage.xaml.cs
         </span>
         and add the following to the constructor as before:
        </p>
        <pre class="c#" id="" name="code" style="">
         SpeechHelper.InitialiseSpeechSynthesis(this.media);
        CortanaInterop.SpeechSynthRequested += SpeechHelper.StartSpeaking;
        </pre>
        <p>
         As this example is using the phone's audio, we also need to add a
         <span class="bold">
          MediaElement
         </span>
         to the
         <span class="bold">
          XAML
         </span>
         of the page else it won't work. If you have chosen to handle the audio in Unity, be sure to change this here as well. Open
         <span class="bold">
          MainPage.xaml
         </span>
         and add the following line after the
         <span class="bold">
          <span class="italic">
           &lt;/grid&gt;
          </span>
         </span>
         element for the
         <span class="bold">
          splashscreen
         </span>
         and before the
         <span class="bold">
          <span class="italic">
           &lt;/SwapChainPanel&gt;
          </span>
         </span>
         element as shown here:
        </p>
        <pre class="xml" id="" name="code" style="">
         &lt;MediaElement x:Name="media" /&gt;
        &lt;/SwapChainPanel&gt;
        </pre>
        <p>
         All done! If you now add something in Unity to call the
         <span class="bold">
          <span class="italic">
           YellAtPlayer
          </span>
         </span>
         method of the
         <span class="bold">
          CortanaInterop
         </span>
         class, like a button, then when you build and run the project the phone will start speaking to you.
        </p>
        <p>
         In the example I provide, I also mixed it up and added a script with an array of text. Each hit of the button randomly selects a line of text to insult the player with.
        </p>
