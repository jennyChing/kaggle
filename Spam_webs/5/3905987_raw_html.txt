        <h1 class="p-name">
         Easily Password Protect a Website or Subdirectory
        </h1>
        <p class="time">
         <a href="https://css-tricks.com/author/chriscoyier/">
          <img alt="" class="avatar avatar-200 photo" height="200" src="https://2.gravatar.com/avatar/e6e3b0dc1a26476da301047fb70784e5?s=200&amp;d=mm&amp;r=pg" srcset="https://2.gravatar.com/avatar/e6e3b0dc1a26476da301047fb70784e5?s=400&amp;d=mm&amp;r=pg 2x" width="200"/>
          <span class="p-author">
           Chris Coyier
          </span>
         </a>
         //
         <time class="dt-published" datetime="2008-07-23">
          July 23, 2008
         </time>
        </p>
       </header>
       <div class="module e-content">
        <div class="google-ad">
         <div class="bsarocks bsap_3469a2a501a9e18091036aa0c89f9dcb" id="bsap_1279518">
         </div>
        </div>
        <p>
         Working on a website that you need others to see, but not the whole world? Password protecting a website (or a sub directory within a website) is actually a pretty easy thing to do.
        </p>
        <p>
         <img alt="" height="207" src="//css-tricks.com/wp-content/csstricks-uploads/protected-directory.png" title="" width="452"/>
        </p>
        <p>
         <span id="more-825">
         </span>
        </p>
        <h3>
         .htaccess file
        </h3>
        <pre>
         <code>
          AuthType Basic  
AuthName "restricted area"  
AuthUserFile /path/to/the/directory/you/are/protecting/.htpasswd  
require valid-user
         </code>
        </pre>
        <p>
         The exact path to the file is extremely important here. If you have it wrong, it will still ask for a password but nothing will seem to work. On Media Temple, my path looks like this:
        </p>
        <p>
         /home/21410/domains/css-tricks.com/html/examples/PasswordProtected/.htpasswd
        </p>
        <p>
         You should be able to to echo out a phpinfo(); on a page in that directory to scope this directory out.
        </p>
        <p>
        </p>
        <h3>
         .htpasswd file
        </h3>
        <pre>
         <code>
          css-tricks:csmBH6tTLNZBE
         </code>
        </pre>
        <p>
         That is what the contents of the .htpasswd file should look like. One username and password per line, separated by a colon. Notice the password is encrypted though. You will need to use a special tool to encrypt your password in this way (MD5). David Walsh has a
         <a href="http://davidwalsh.name/web-development-tools">
          tool just for this
         </a>
         .
        </p>
        <p>
         In fact, I obviously first learned this from David as pretty much this
         <a href="http://davidwalsh.name/password-protect-directory-using-htaccess">
          exact same tip
         </a>
         is on his site. Still, I think it's worthy of re-posting because this is an extremely useful tool to have in your toolbox.
        </p>
        <h3>
         See it in action
        </h3>
        <p>
         <a href="http://css-tricks.com/examples/PasswordProtected/">
          Live example here
         </a>
         . Login/Password is css-tricks/css-tricks
        </p>

           <div class="comment-content">
            <p>
             To Robert Augustin: MD5 is a one-way cryptographic hash function. Therefore it is impossible to decrypt a hashed password to get its original value. The way this authentication method works (I assume) is that the user enters a password, which is hashed and compared to the stored password hash for that username. If the two match, then authentication is successful. This negates the need for storing plain-text or decryptable passwords. MD5 does have its problems, however, and it is possible to find a text string which generates the same hash value as a stored password. To prevent this, a salt value should be used when hashing..
            </p>

           <div class="comment-content">
            <p>
             A few points Id like to add:
            </p>
            <p>
             The hash algorithm used in the example .htpasswd in the article doesnt look like MD5 at all, looks more like crypt() which is the default (except in Windows).
             <br/>
             MD5 is not encryption, its a one way hash/digest algorithm. Despite the name, crypt() is not an encryption either.
             <br/>
             Putting .htpasswd in the same directory as .htaccess, increases the chance of exposing the user and password hashes publicly. Its probably wiser to put it outside of Apaches DocumentRoot completely. Recent installations of Apache httpd probably has a rule that blocks direct access to .htaccess and .htpasswd files by default, but its not always the case.
            </p>
