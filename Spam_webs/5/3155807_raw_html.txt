        <h2 id="vboxheadless-running-virtual-machines-with-virtualbox-on-a-headless-ubuntu-server">
         VBoxHeadless - Running Virtual Machines With VirtualBox 3.1 On A Headless Ubuntu 9.10 Server
        </h2>
        <p>
         Version 1.0
         <br/>
         Author: Falko Timme
         <br/>
         <a href="http://twitter.com/howtoforgecom" mce_real_href="http://twitter.com/howtoforgecom" target="_blank">
         </a>
         <a class="fancybox" href="https://www.howtoforge.com/images/socialmedia/big/twitter.png" id="img-twitter">
          <img alt="" border="0" height="16px" mce_real_src="https://www.howtoforge.com/images/socialmedia/twitter.png" src="https://www.howtoforge.com/images/socialmedia/twitter.png" width="16px"/>
         </a>
         <a href="http://twitter.com/howtoforgecom" mce_real_href="http://twitter.com/howtoforgecom" target="_blank">
          Follow me on Twitter
         </a>
         <br/>
         Last edited 01/25/2010
        </p>
        <div style="width:336px;float:left; margin: 10px 15px 10px 0px;background-color:white;">
         <script type="text/javascript">
          GA_googleFillSlot('howtoforge_com_article_rectangle_a_300x250');
         </script>
        </div>
        <p>
         This guide explains how you can run virtual machines with
         <a href="http://www.virtualbox.org/" mce_real_href="http://www.virtualbox.org/" target="_blank">
          Sun VirtualBox 3.1
         </a>
         on a headless Ubuntu 9.10 server. Normally you use the VirtualBox GUI to manage your virtual machines, but a server does not have a desktop environment. Fortunately, VirtualBox comes with a tool called VBoxHeadless that allows you to connect to the virtual machines over a remote desktop connection, so there's no need for the VirtualBox GUI.
        </p>
        <p>
         I do not issue any guarantee that this will work for you!
        </p>
        <p>
        </p>
        <h3 id="-preliminary-note">
         1 Preliminary Note
        </h3>
        <p>
         I have tested this on an Ubuntu 9.10 server (host system) with the IP address
         <span class="system">
          192.168.0.100
         </span>
         where I'm logged in as a normal user (user name
         <span class="system">
          administrator
         </span>
         in this example) instead of as
         <span class="system">
          root
         </span>
         .
        </p>
        <p>
        </p>
        <h3 id="-installing-virtualbox">
         2 Installing VirtualBox
        </h3>
        <p>
         To install VirtualBox 3.1 on our Ubuntu 9.10 server, we open
         <span class="system">
          /etc/apt/sources.list
         </span>
         ...
        </p>
        <p class="command">
         sudo vi /etc/apt/sources.list
        </p>
        <p>
         ... and add the following line to it:
        </p>
        <table align="center" bgcolor="#cccccc" border="1" bordercolor="#000000" cellpadding="2" cellspacing="0" class="" width="90%">
         <tbody>
          <tr>
           <td class="">
            <pre>
             [...]
deb http://download.virtualbox.org/virtualbox/debian karmic non-free
            </pre>
           </td>
          </tr>
         </tbody>
        </table>
        <p>
         Then we download Sun's public key...
        </p>
        <p class="command">
         wget -q http://download.virtualbox.org/virtualbox/debian/sun_vbox.asc -O- | sudo apt-key add -
        </p>
        <p>
         ... and update our package database:
        </p>
        <p class="command">
         sudo aptitude update
        </p>
        <p>
         Afterwards, we install VirtualBox 3.1 as follows:
        </p>
        <p class="command">
         sudo aptitude install linux-headers-$(uname -r) build-essential virtualbox-3.1 dkms
        </p>
        <p>
         (The
         <span class="system">
          dkms
         </span>
         package ensures that the VirtualBox host kernel modules are properly updated if the Linux kernel version changes.)
        </p>
        <p>
         <span class="system">
          Creating group 'vboxusers'
          <br/>
          Users of VirtualBox must be members of that group. Host network interfaces will be assigned to that group.
         </span>
         <span class="highlight">
          &lt;-- Ok
         </span>
        </p>
        <p>
         <span class="system">
          Should the vboxdrv kernel module be compiled now?
         </span>
         <span class="highlight">
          &lt;-- Yes
         </span>
        </p>
        <p>
         Now we must add the user that will run VirtualBox (
         <span class="system">
          administrator
         </span>
         in this example) to the
         <span class="system">
          vboxusers
         </span>
         group:
        </p>
        <p class="command">
         sudo adduser administrator vboxusers
        </p>
        <p>
         VirtualBox is now installed and ready to be used.
        </p>
        <p>
        </p>
        <h3 id="-using-virtualbox-on-the-command-line">
         3 Using VirtualBox On The Command Line
        </h3>
        <h4 id="-creating-a-vm">
         3.1 Creating A VM
        </h4>
        <p>
         To create a VM on the command line, we can use the
         <span class="system">
          VBoxManage
         </span>
         command. See
        </p>
        <p class="command">
         VBoxManage --help
        </p>
        <p>
         for a list of available switches and (highly recommended!) take a look at
         <a href="http://www.virtualbox.org/manual/UserManual.html#vboxmanage" mce_real_href="http://www.virtualbox.org/manual/UserManual.html#vboxmanage" target="_blank">
          http://www.virtualbox.org/manual/UserManual.html#vboxmanage
         </a>
         .
        </p>
        <p>
         I will now create an Ubuntu 9.10 Server VM with 256MB memory and a 10GB hard drive from the Ubuntu 9.10 Server iso image (which I have stored in
         <span class="system">
          /home/ubuntu-9.10-server-amd64.iso
         </span>
         ):
        </p>
        <p class="command">
         VBoxManage createvm --name "Ubuntu 9.10 Server" --register
         <br/>
         VBoxManage modifyvm "Ubuntu 9.10 Server" --memory 256 --acpi on --boot1 dvd --nic1 bridged --bridgeadapter1 eth0
         <br/>
         VBoxManage createhd --filename Ubuntu_9_10_Server.vdi --size 10000 --register
         <br/>
         VBoxManage storagectl "Ubuntu 9.10 Server" --name "IDE Controller" --add ide
         <br/>
         VBoxManage storageattach "Ubuntu 9.10 Server" --storagectl "IDE Controller" --port 0 --device 0 --type hdd --medium Ubuntu_9_10_Server.vdi
         <br/>
         VBoxManage storageattach "Ubuntu 9.10 Server" --storagectl "IDE Controller" --port 1 --device 0 --type dvddrive --medium /home/ubuntu-9.10-server-amd64.iso
        </p>
        <div style="width:336px;float:left; margin: 10px 15px 10px 0px;background-color:white;">
         <script type="text/javascript">
          GA_googleFillSlot('howtoforge_com_article_rectangle_b_300x250');
         </script>
        </div>
        <p>
        </p>
        <h4 id="-importing-an-existing-vm">
         3.2 Importing An Existing VM
        </h4>
        <p>
         Let's assume you have a VM called
         <span class="system">
          examplevm
         </span>
         that you want to reuse on this host. On the old host, you should have a directory
         <span class="system">
          Machines/examplevm
         </span>
         in the VirtualBox directory;
         <span class="system">
          Machines/examplevm
         </span>
         should contain the
         <span class="system">
          examplevm.xml
         </span>
         file. Copy the
         <span class="system">
          examplevm
         </span>
         directory (including the
         <span class="system">
          examplevm.xml
         </span>
         file) to your new
         <span class="system">
          Machines
         </span>
         directory (if your user name is administrator, this is
         <span class="system">
          /home/administrator/.VirtualBox/Machines
         </span>
         - the result should be
         <span class="system">
          /home/administrator/.VirtualBox/Machines/examplevm/examplevm.xml
         </span>
         ).
        </p>
        <p>
         In addition to that copy the
         <span class="system">
          examplevm.vdi
         </span>
         file from the old
         <span class="system">
          VDI
         </span>
         directory to the new one (e.g.
         <span class="system">
          /home/administrator/.VirtualBox/VDI/examplevm.vdi
         </span>
         ).
        </p>
        <p>
         Afterwards, you must register the imported VM:
        </p>
        <p class="command">
         VBoxManage registervm Machines/examplevm/examplevm.xml
        </p>
        <p>
        </p>
        <h4 id="-starting-a-vm-with-vboxheadless">
         3.3 Starting A VM With VBoxHeadless
        </h4>
        <p>
         Regardless of if you create a new VM or import and old one, you can start it with the command:
        </p>
        <p class="command">
         VBoxHeadless --startvm "Ubuntu 9.10 Server"
        </p>
        <p>
         (Replace
         <span class="system">
          Ubuntu 9.10 Server
         </span>
         with the name of your VM.)
        </p>
        <p>
         <span class="system">
          VBoxHeadless
         </span>
         will start the VM and a VRDP (
         <span class="system">
          VirtualBox Remote Desktop Protocol
         </span>
         ) server which allows you to see the VM's output remotely on another machine.
        </p>
        <p>
         To stop a VM, run
        </p>
        <p class="command">
         VBoxManage controlvm "Ubuntu 9.10 Server" poweroff
        </p>
        <p>
         To pause a VM, run
        </p>
        <p class="command">
         VBoxManage controlvm "Ubuntu 9.10 Server" pause
        </p>
        <p>
         To reset a VM, run
        </p>
        <p class="command">
         VBoxManage controlvm "Ubuntu 9.10 Server" reset
        </p>
        <p>
         To learn more about
         <span class="system">
          VBoxHeadless
         </span>
         , take a look at
        </p>
        <p class="command">
         VBoxHeadless --help
        </p>
        <p>
         and at
         <a href="http://www.virtualbox.org/manual/UserManual.html#id2515738" mce_real_href="http://www.virtualbox.org/manual/UserManual.html#id2515738" target="_blank">
          http://www.virtualbox.org/manual/UserManual.html#id2515738
         </a>
         .
        </p>
