          <p>
           <firstimage>
            <img alt="vim text editor" class="align-right" src="http://cdn.makeuseof.com/wp-content/uploads/2011/05/vim-editor_logo-300x300.png?9e15ff"/>
            If you do any sort of development work on your computer, youre going to need a text editor every now and then (or all the time). And by development I dont just mean programming; it can be Web development too, or even maintaining and tweaking an existing website.
           </firstimage>
          </p>
          <p>
           For years, Ive tried one text editor after another. You name it, I tried it  jEdit, Notepad++, SciTE, PSPad, Komodo Edit Im not just name-dropping here. I used each and every one of these editors for over two months as my primary day-to-day editor. I also had a brief stint with UltraEdit, a venerable (but non-free) editor. Somehow, I couldnt find an editor that just worked right, and was customizable enough to fit my needs.
          </p>
          <p>
           That all changed a couple of months ago, when I bit the bullet and started using
           <a href="http://www.vim.org">
            Vim
           </a>
           text editor full-time. Heres 6 reasons why you should consider giving it a go.
          </p>
          <h2>
           Reason 1: Vim Is Old
          </h2>
          <p>
           <img alt="vim text editor" class="aligncenter" height="350" src="http://cdn.makeuseof.com/wp-content/uploads/2011/05/image-30.jpg?9e15ff" width="580"/>
          </p>
          <p>
           In software, we often cultivate the mindset that says newer is better. Were always after the latest browser, the latest OS, the latest game. But theres a lot to be said for ancient pieces of software that are still actively used by lots of people.
          </p>
          <p>
           Did you know MS Office was first released in 1990? Thats 21 years ago, and look how far it has come. Thats about the time Vim was first released, too (1991). Only Vim is based on a much older editor, Vi, that got its start in 1976.
          </p>
          <p>
           That alone should make you wonder; what makes such an
           <em>
            ancient
           </em>
           piece of software so compelling, that thousands of new users discover it year by year?
          </p>
          <div>
           <div class="ad-unit-post" id="adsense2">
            <small>
             <a href="https://support.google.com/adsense/troubleshooter/1631343?url=http%3A%2F%2Fwww.makeuseof.com%2Ftag%2Ftop-7-reasons-to-give-the-vim-text-editor-a-chance%2F&amp;client=ca-pub-9894449314507324" style="color:LightGray">

          <h2>
           Reason 2: Vim Is Free &amp; Has A Vibrant Community
          </h2>
          <p>
           Vim is actually defined as charityware, and is free to use and open-source. It does ask you to donate for orphans in Uganda, but there are no nags.
          </p>
          <p>
           If you go to
           <a href="http://www.vim.org/">
            the Vim webpage
           </a>
           , you will find lots of recent news items. I counted nine script updates in one day, for example. The site is constantly updated, and Vims main developer, Bram Moolenaar, personally posted an update on April 28.
          </p>
          <h2>
           Reason 3: Vim Is Very Customizable &amp; Extensible
          </h2>
          <p>
           <img alt="vim editor" class="aligncenter" height="103" src="http://cdn.makeuseof.com/wp-content/uploads/2011/05/image-34.jpg?9e15ff" width="580"/>
          </p>
          <p>
           Those scripts I just mentioned? Theyre the key to much of Vims power. There are scripts for just about anything. In the screenshot above you can see
           <a href="http://www.vim.org/scripts/script.php?script_id=1984">
            FuzzyFinder
           </a>
           , one of my all-time favorites. Its a script that lets you type partial names of files, commands, help entries, etc. and searches for them incrementally, on-the-fly. You dont have to type from the beginning of the name, and its got a lot of options  and thats just one script!
          </p>
          <p>
           Other notable scripts include
           <a href="http://www.vim.org/scripts/script.php?script_id=1658">
            NERD Tree
           </a>
           that implements a file-system tree,
           <a href="http://www.vim.org/scripts/script.php?script_id=1643">
            SuperTab Continued
           </a>
           for powerful tab-completion, and
           <a href="http://www.vim.org/scripts/script.php?script_id=2540">
            snipMate
           </a>
           for replicating TextMates snippet functionality. There are lots, lots
           <a href="http://www.vim.org/scripts/script_search_results.php?order_by=downloads">
            more
           </a>
           .
          </p>
          <h2>
           Reason 4: Vim Works Over Telnet &amp; SSH Connections
          </h2>
          <p>
           <img alt="vim editor" class="aligncenter" height="338" src="http://cdn.makeuseof.com/wp-content/uploads/2011/05/vimterminal.png?9e15ff" width="580"/>
          </p>
          <p>
           I sometimes need to manually edit webpages that reside on remote servers. I used to use WinSCP to simulate local editing, but it broke every now and then. A few of the editors mentioned above also have built-in FTP clients, but navigating complex folder trees was always cumbersome and time-consuming.
          </p>
          <p>
           With Vim Text Editor, I can just open an
           <a href="http://www.makeuseof.com/tag/ssh-ftp-technology-explained/">
            SSH connection
           </a>
           to my Web server and run a remote instance of Vim right on the server. Vim is
           <em>
            designed
           </em>
           to work with slow terminals, so it actually feels fast. As you can see in the screenshot above, colors work just fine over SSH. Being able to work directly on the server has made a huge difference in my Web development work. Not only is it faster, but I can now search through files right on the server, and use Vim to jump between the results quickly and find exactly what I need to modify with pinpoint precision. Simply awesome.
          </p>
          <h2>
           Reason 5: Vims Configuration Is Portable
          </h2>
          <p>
           <img alt="vim editor" class="aligncenter" height="350" src="http://cdn.makeuseof.com/wp-content/uploads/2011/05/image-31.jpg?9e15ff" width="580"/>
          </p>
          <p>
           As mentioned above, Vim is very customizable. That means I had to work quite a bit until I got it just so on my Windows system. Having to do all that on my remote server would have been a bit of a drag. Luckily, I simply needed to copy a few directories and one all-important master configuration file, called
           <em>
            .vimrc
           </em>
           .
          </p>
          <p>
           This basically copied my entire configuration from my Windows computer to the remote Linux server. I then just edited my configuration file and modified a couple of things for the server, and I was basically done. Of course I couldnt keep myself from tweaking things further, but thats just me.
          </p>
          <h2>
           Reason 6: Vim Is Thoroughly Documented
          </h2>
          <p>
           Note that nowhere on this list does it say Vim is intuitive. Thats because its not, really. There
           <em>
            is
           </em>
           a learning curve when you first start using Vim, and it does require a bit of a commitment. But there are quite a few things that can help. Here are just three examples:
          </p>
          <p>
           <img alt="vim text editor" class="aligncenter" height="350" src="http://cdn.makeuseof.com/wp-content/uploads/2011/05/image-32.jpg?9e15ff" width="580"/>
          </p>
          <ul>
           <li>
            PeepCodes
            <a href="http://peepcode.com/products/smash-into-vim-i">
             Smash into Vim
            </a>
            screencasts (shown in the screenshot above): These are actually not free; they cost $12 each (there are two), but are a very worthwhile investment. When I decided I wanted to give Vim a serious spin, I bought the first one and watched it. It was enough to get me started with confidence. It wasnt boring, either.
           </li>
           <li>
            Vims own
            <strong>
             *:help*
            </strong>
            command leads into a treasure trove of carefully-written documentation. Every script has its own documentations; default key bindings are carefully documented, all of the commands are explained, etc.
           </li>
           <li>
            Vim also has a
            <a href="http://vim.wikia.com/wiki/Vim_Tips_Wiki">
             wiki
            </a>
            with oodles of tips, tricks and helpful code snippets for you to copy and try.
           </li>
          </ul>
          <h2>
           Reason 7: Its Way Better Than Emacs
          </h2>
          <p>
           OK, so maybe Im not entirely serious about this last one. As you may have noticed in the introduction, Ive never actually given Emacs much of a spin (not for two months, at least). According to the scientifically-minded
           <span class="removed_link" title="http://www.tarunz.org/~vassilii/srom/">
            $EDITORs Sucks-Rules-O-Meter
           </span>
           , Vim seems to be in the lead in the Editor War, at least for now. But Im willing to be convinced otherwise. Youre welcome to plead Emacs case in the comments.
          </p>
          <p>
           Speaking of comments: if there are any
           <a href="http://www.vim.org">
            Vim
           </a>
           users in the audience whod like to share tips or ask questions, I am all ears. I know Ive barely scratched the surface on Vim in this post, so if youd like to see more detailed posts about Vim, do share your ideas and wishes.
          </p>

                 Creams approach is completely different. Cream attempts to make vim more like other desktop software with a set of key reamapings  like Ctrl+c for copy and a bunch of menus. Id argue at that point, why use vim at all?If you learn cream, youll be completely unable to use stock vim, with Spf13-vim youll feel right at home.

                 Spf13-vims approach is to stick to vims approach to things. Spf13-vim provides a richer set of features without changing the way vim works. For instance it provides snippet functionality and a rich set of code snippets for a ton of languages.. but this doesnt alter the way vim works. It provides syntax checking for php, python, c and ruby.

                <p>
                 If you like vim and the way vim works, but feel like features like autocomplete, snippets, git support and more would make vim even better, than checkout spf13-vim.
