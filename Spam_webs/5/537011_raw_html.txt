         <h2>
          <span class="mw-headline" id="Common_questions">
           Common questions
          </span>
         </h2>
         <div class="linebox" style="border: 1px solid #404040; color: white; background-color: #222; -moz-border-radius:1em; -webkit-border-radius:1em; padding: 1em;">
          <h4>
           <span class="mw-headline" id="Can_I_use_NEWD_or_NEWD2_builds.3F">
            Can I use NEWD or NEWD2 builds?
           </span>
          </h4>
          <ol>
           <li>
            In general it has been recommended to stay with NEWD builds for most users as of Feb 2010.
           </li>
           <li>
            Both builds can support 802.11n on the routers.
           </li>
           <li>
            If you need to use both radios (wl0 and wl1), you should stay with NEWD builds.
           </li>
           <li>
            NEWD2 has newer wireless drivers for Broadcom based routers that support 11n and posts on the forum indicate a performance advantage when using N. However, it appears you lose the ability to use the wl1 radio on these builds:
            <ul>
             <li>
              reference wiki entry:
              <a class="external text exitstitial" href="http://www.dd-wrt.com/wiki/index.php/NEWD2_Support_on_some_models" rel="nofollow">
               NEWD2_Support_on_some_models
              </a>
              (if this changes, someone should update this item)
             </li>
             <li>
              be sure to research if it is still appropriate to use NEWD2 releases as DD-WRT is constantly evolving with updates
             </li>
            </ul>
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="There_are_a_gazillion_different_build_versions._Which_ones_can_I_use.3F">
            There are a gazillion different build versions. Which ones can I use?
           </span>
          </h4>
          <ol>
           <li>
            Remember this router has a 4mbit flash chip. You can use most of the "std" (standard/generic) or "mini" builds and most of their sub variants from brainslayer or eko. *NEVER* use any of the micro or mega builds on this router - you will either neuter or brick it.
            <ol>
             <li>
              It is now being reported that .bin files must be smaller than approximately 3648kb (3735552 bytes) in size, otherwise they will fail to flash. This means that Brainslayer builds released since about Dec 2009 will likely be too large.
             </li>
            </ol>
           </li>
           <li>
            if you are still using the factory Netgear firmware, there really is only one initial firmware you can load first. After that you can load other certain versions. See the separate section on flashing the WNDR3300 further down in this wiki entry.
           </li>
           <li>
            <i>
             WARNING! Even though the builds listed are classified as known to work, some versions (old and new) may not load due to the actual firmware bin file size. Please see the FLASHING section of this entry for more information.
            </i>
           </li>
           <li>
            if you like the brainslayer builds (search the wiki about the specific differences)
            <ol>
             <li>
              dd-wrt.v24_std_generic.bin
              <ul>
               <li>
                suggested for most users as it has all the standard options included
               </li>
              </ul>
             </li>
             <li>
              dd-wrt.v24_nokaid_generic.bin
              <ul>
               <li>
                for those that do not need xbox/kaid support and to save a little space on the flash
               </li>
              </ul>
             </li>
            </ol>
           </li>
           <li>
            if you like eko builds, suggestions are:
            <ol>
             <li>
              NEWD_std-nokaid.bin
              <ul>
               <li>
                NEWD builds for users who do not need xbox/kaid but like to have all of the other standard options. This will also save some space on your flash if you want to use jffs
               </li>
              </ul>
             </li>
             <li>
              NEWD_std-nokaid_nohotspot_nostor.bin
              <ul>
               <li>
                NEWD builds for users who do not need xbox, hotspot or external/CIFS storage options to save space on your flash. This build leaves the most free flash space for JFFS in standard releases without having to step down to mini builds.
               </li>
              </ul>
             </li>
             <li>
              NEWD_openvpn_jffs_small.bin
              <ul>
               <li>
                similar to the std nokaid_nohotspot_nostor version but with JFFS and OpenVPN support
               </li>
              </ul>
             </li>
            </ol>
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="I_see_builds_for_K24_and_K26._What_are_the_differences.3F">
            I see builds for K24 and K26. What are the differences?
           </span>
          </h4>
          <ol>
           <li>
            Basically that means builds are using the Linux kernals v2.4 (aka K24) or v2.6 (K26).
           </li>
           <li>
            DD-WRT has been built on the K24 platform for many years and is proven quite stable. It is recommended that general users stay with these builds for the immediate future.
           </li>
           <li>
            There is a movement to get DD-WRT on the more modern and supported 2.6 kernal recently - starting around late 2009. This will take some to get it fully stable. Note that only certain recent routers will be able to support these versions.
            <ol>
             <li>
              If you have time and are will to help test these K26 versions, please do!
              <ul>
               <li>
                WARNING! Note that you should be an "advanced user" of DD-WRT if you plan to try K26 builds. Meaning, you know how to handle the router if you brick it.
               </li>
               <li>
                As of Feb 2010, the possibility exists that some K26 builds may brick your router and require serial console access to recover. If you do not know what that means, DO NOT TRY the K26 builds.
               </li>
              </ul>
             </li>
             <li>
              You can follow these threads for additional information:
              <ol>
               <li>
                <a class="external text exitstitial" href="http://www.dd-wrt.com/phpBB2/viewtopic.php?t=63757" rel="nofollow">
                 K26 Build Supported Router List
                </a>
               </li>
               <li>
                <a class="external text exitstitial" href="http://www.dd-wrt.com/phpBB2/viewtopic.php?t=63948" rel="nofollow">
                 WNDR3300 13575M NEWD-2 K2.6 Eko
                </a>
               </li>
              </ol>
             </li>
            </ol>
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="The_router_is_advertised_as_having_dual_radio_support._What_are_my_options_on_using_them.3F">
            The router is advertised as having dual radio support. What are my options on using them?
           </span>
          </h4>
          <ol>
           <li>
            the main wireless radio dubbed "wl0" in DD-WRT supports two wireless bands: 2.4GHz *or* 5GHz (or disable it). You can pick from the following configuration for the wl0 radio:
            <ol>
             <li>
              Disabled: radio is powered off
             </li>
             <li>
              Mixed: supporting 802.11b/g (11mpbs and 54mbps on the 2.4GHz band)
             </li>
             <li>
              B only: supporting 802.11b (11mbps on the 2.4GHz band)
             </li>
             <li>
              G only: supporting 802.11g (54mbps on the 2.4GHz band)
             </li>
             <li>
              N only-2.4: supporting 802.11n (up to 130mbps on the 2.4GHz band depending on the channel and width you pick)
             </li>
             <li>
              A only: supporting 802.11a (54mbps on the 5GHz band)
             </li>
             <li>
              NA-Mixed: supporting 802.11a and 802.11n (standard 54mbps *and* up to 270mbps on the 5GHz band depending on the channel and width you pick)
             </li>
             <li>
              N only-5: supporting 802.11n (up to 270mbps on the 5GHz band depending on the channel and width you pick)
             </li>
            </ol>
           </li>
           <li>
            secondary wireless radio dubbed "wl1" in DD-WRT supports only one band: 2.4GHz (or disable it)
            <ol>
             <li>
              Disabled: radio is powered off
             </li>
             <li>
              Mixed: supporting 802.11b/g (11mpbs and 54mbps on the 2.4GHz band)
             </li>
             <li>
              B only: supporting 802.11g (11mbps on the 2.4GHz band)
             </li>
             <li>
              G only: supporting 802.11g (54mbps on the 2.4GHz band)
             </li>
            </ol>
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="There_are_many_posts_about_stability._What_about_it.3F">
            There are many posts about stability. What about it?
           </span>
          </h4>
          <ol>
           <li>
            There are MANY users running this router in a variety of different environments and most have no or minimal issues. To be fair, some versions of DD-WRT have been better than others. However, the most recent firmware revisions have been stable in most typical setups. Exotic setups, like using WDS and client bridging, require additional care that may be discussed in other sections of this wiki entry.
           </li>
           <li>
            Many posts with problems can be traced to not fully completing the flashing procedures - specifically the reset handling.
           </li>
           <li>
            If your WAN port is dropping connections - this is not normal and would require troubleshooting that may generally apply to any router running DD-WRT
           </li>
           <li>
            If the wireless connections are not stable (eg, regularly drops connections), there are any number of reasons for this that MAY NOT BE related to this router or DD-WRT itself.
           </li>
           <li>
            I've tried all sorts of things and it still is not "stable"! This does not mean the refurb unit you bought from a link on a deal site is bad. Please review potential sections like power supply information, spend time and read through the various forum posts as someone may have had your issue before asking for help.
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="What_kind_of_performance_can_I_get_from_N_on_this_router.3F">
            What kind of performance can I get from N on this router?
           </span>
          </h4>
          <ol>
           <li>
            Some users have performed iperf testing. Results indicate N over 5GHz looks very reasonable depending on the configuration and surrounding environment. Watch for future separate section in this wiki entry for benchmark information.
           </li>
           <li>
            Remember that this router, by default, has internal antennas and that affects performance greatly for either band, but especially for N over 5GHz.
           </li>
          </ol>
         </div>
         <h2>
          <span class="mw-headline" id="Flashing">
           Flashing
          </span>
         </h2>
         <div class="linebox" style="border: 1px solid #404040; color: white; background-color: #222; -moz-border-radius:1em; -webkit-border-radius:1em; padding: 1em;">
          <h4>
           <span class="mw-headline" id="IMPORTANT_FLASHING_PREREQUISITES">
            IMPORTANT FLASHING PREREQUISITES
           </span>
          </h4>
          <ol>
           <li>
            DO YOUR RESEARCH BEFORE BLINDLY FLASHING WITH ANY VERSION OF DD-WRT. THIS WIKI ENTRY IS JUST A STARTING POINT.
           </li>
           <li>
            STOP! If you have not read the
            <a class="external text exitstitial" href="http://www.dd-wrt.com/phpBB2/viewtopic.php?t=51486" rel="nofollow">
             Peacock Thread
            </a>
            , do so now before reading the rest of the wiki entry or asking for help in the forums.
           </li>
           <li>
            WARNING! Confirm the file size of the *.bin file you are planning to use.
            <ul>
             <li>
              Although router has 4mb of flash, but no one has exactly listed what the approx threshold was for the firmware size you can use. Based on the OpenWRT wiki flash ROM partition sizing for this router, the max size bin firmware you can load should be:
             </li>
             <li>
              3735552 bytes or about 3648kb (the available partition size for linux and rootfs combined which equal what the firmware is)
             </li>
             <li>
              If you try to load a *.bin file larger than this, the flash will fail and the router will remain in TFTP recovery mode. You can then load another smaller bin file.
              <ul>
               <li>
                eg, recent std-generic.bin files from Dec 2009 to Feb 2010 are over 30k too large to load
               </li>
              </ul>
             </li>
             <li>
              Some routers may have larger flashable space for firmware, however, this router has slightly reduced space then other 4mbit Broadcom units due to the flash partitioning the manufacturer is using
             </li>
            </ul>
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="If_you_are_running_the_factory_Netgear_firmware">
            If you are running the factory Netgear firmware
           </span>
          </h4>
          <ol>
           <li>
            you need to download a special dd-wrt "trailed" firmware package for this router: the *.chk file.
            <ul>
             <li>
              DO NOT USE THE 12-28-09 r13525 brainslayer chk build. This has been found by a user to have a bad checksum and will leave your router in recovery/tftp mode - see page 71 in the main support thread. Note that the regular (std/mini/generic/etc) *.bin files for this release appear to be fine though.
             </li>
             <li>
              A known confirmed flashable version as of Feb 2010:
              <a class="external text exitstitial" href="http://www.dd-wrt.com/dd-wrtv2/downloads/others/eko/BrainSlayer-V24-preSP2/01-16-10-r13637/broadcom/dd-wrt.v24_std-wndr3300.chk" rel="nofollow">
               01-16-10-r13637/broadcom/dd-wrt.v24_std-wndr3300.chk
              </a>
             </li>
            </ul>
           </li>
           <li>
            it is recommended you perform a reset to factory defaults (eg, 30/30/30 reset)
           </li>
           <li>
            be sure to perform all firmware updates from a wired ethernet connection. Do not perform a firmware upgrade over the wireless.
           </li>
           <li>
            hook up the router from one of the four available ethernet ports (do NOT use the WAN port) and connect the ethernet cable to a NIC (fast ethernet or gigabit) port on your computer.
           </li>
           <li>
            if you have reset the Netgear firmware to factory settings, it is recommended you use a static IP address on the NIC:
            <ul>
             <li>
              IP address: 192.168.1.2
             </li>
             <li>
              subnet mask: 255.255.255.0
             </li>
             <li>
              gateway: 192.168.1.1
             </li>
            </ul>
           </li>
           <li>
            log in to the Netgear Web GUI. If you are running a Microsoft Windows OS, it is recommended you use Internet Explorer to perform the initial firmware upgrade to DD-WRT. You can use any other browser there after. Once you log in...
            <ul>
             <li>
              Click on "Router Upgrade" in the LEFT navigation side bar
             </li>
             <li>
              Click on "Browse..." and select the .chk file firmware you wish to load or manually type in the filename
             </li>
             <li>
              Click on "Upload"
             </li>
             <li>
              You will get a prompt to "Continue? All existing Internet connections will be terminated"
             </li>
             <li>
              Click on "OK" to proceed.
             </li>
             <li>
              You will see the following screen appear:
              <br/>
              -----------------------
              <br/>
              Router Upgrade
              <br/>
              Warning! You are trying to upload a world wide firmware which is different from the NA firmware you had.
              <br/>
              Do you still want to continue?
              <br/>
              Current Version V1.0.XX_1.0.XXNA
              <br/>
              Uploaded Version V1.0.14_1.0.14
              <br/>
              -----------------------
             </li>
             <li>
              Where "XX" is a version number. Typically 29 for most refurbs or 45 for fresh retail models.
             </li>
             <li>
              Ignore the Uploaded Version "V1.0.14_1.0.14" string, this is just the version the original DD-WRT chk file uses so the Netgear firmware will accept the file.
             </li>
             <li>
              Click on "Yes" to initiate the update.
             </li>
             <li>
              You will see the following message:
              <br/>
              Note:It will take about 1.5 minutes for firmware upgrade.
              <br/>
              Please don't turn off the power or press reset button.
             </li>
            </ul>
           </li>
           <li>
            WAIT until the Power LED stops flashing and allow the router to reboot. BE SURE YOU ALLOW SUFFICIENT TIME FOR THE PROCESS TO COMPLETE. Follow appropriate recommendations in the Peacock thread.
           </li>
           <li>
            If the Power LED flashes for more than 10-15 minutes, something is probably wrong. Check on how to revert back to factory firmware and try and other .chk file.
           </li>
           <li>
            Check for recommended builds
            <a href="/wiki/Build_Links#Recommended_Builds" title="Build Links">
             here
            </a>
            .
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="If_you_are_running_dd-wrt">
            If you are running dd-wrt
           </span>
          </h4>
          <ul>
           <li>
            See
            <a href="#Upgrading">
             Upgrading
            </a>
           </li>
          </ul>
          <h4>
           <span class="mw-headline" id="After_flashing">
            After flashing
           </span>
          </h4>
          <ol>
           <li>
            <b>
             ABSOLUTELY RECOMMENDED:
            </b>
            perform the 30/30/30 reset per the Peacock thread
           </li>
           <li>
            <b>
             HIGHLY SUGGESTED:
            </b>
            to ALSO do a full power cycle after the 30/30/30 reset
           </li>
           <li>
            these steps are important to help clear old settings and provide some level of stability
           </li>
           <li>
            if you just upgraded from the Netgear original firmware, you can now re-flash with any specific DD-WRT version/variant you require
           </li>
          </ol>

         <h2>
          <span class="mw-headline" id="Upgrading">
           Upgrading
          </span>
         </h2>
         <div class="linebox" style="border: 1px solid #404040; color: white; background-color: #222; -moz-border-radius:1em; -webkit-border-radius:1em; padding: 1em;">
          <p>
           If dd-wrt is already on the router follow these instructions. If stock firmware is on the router follow the
           <a href="#Flashing">
            flashing
           </a>
           instructions.
          </p>
          <ol>
           <li>
            Check for recommended builds
            <a href="/wiki/Build_Links#Recommended_Builds" title="Build Links">
             here
            </a>
            first.
           </li>
           <li>
            Set your computer to a static IP of 192.168.1.7. (or to whatever subnet the router is on) Disable all firewalls and security. Disable wireless on your computer and only have the router connected to the flashing computer by the ethernet cable between the two.
           </li>
           <li>
            <a href="/wiki/Hard_reset_or_30/30/30" title="Hard reset or 30/30/30">
             Hard reset or 30/30/30
            </a>
            (If the router supports it, if not, reset to defults in the GUI) prior to flashing. Wait. Check for password page on re-login and change password.
           </li>
           <li>
            Flash firmware. You can use the webgui except if you have a belkin router. (For belkin use tftp.exe to flash)
           </li>
           <li>
            Wait...at least three minutes. Lights should return to normal. See important2, below. Failing to wait is how most people brick their routers.
           </li>
           <li>
            Do a power cycle of the router. (Unplug the cord, count to 30 and plug it back in.)
           </li>
           <li>
            Wait for the lights to return to normal usually about 2 minutes.
           </li>
           <li>
            <a href="/wiki/Hard_reset_or_30/30/30" title="Hard reset or 30/30/30">
             Hard reset or 30/30/30
            </a>
            again (If the router supports it, if not, reset to defults in the GUI). Wait. Check for the password page and re-login to change the password. Then you can reconfigure your settings manually.
           </li>
           <li>
            Once configured set your computer back to autoIP and autoDNS.
            <br/>
           </li>
          </ol>
          <p>
           Important1: This
           <a href="/wiki/Hard_reset_or_30/30/30" title="Hard reset or 30/30/30">
            Hard reset or 30/30/30
           </a>
           works fine for Asus router, but you do have to power cycle after the reset.
          </p>
          <p>
           Important2:
           <b>
            After you flash the firmware, and before you do the hard reset, the router will be building some nvram settings. YOU MUST WAIT FOR THIS TO FINISH PRIOR TO DOING ANYTHING WITH THE ROUTER INCLUDING A HARD RESET. Usually, you can tell when this process is completed by the WAN light coming on, but it does take several minutes. Go have a beer. There are starting to be more and more people who BRICK their routers by not waiting until the nvram is rebuilt, PRIOR to doing a hard reset. YOU NEED TO WAIT!
           </b>
          </p>
         </div>
         <h2>
          <span class="mw-headline" id="Reverting">
           Reverting
          </span>
         </h2>
         <div class="linebox" style="border: 1px solid #404040; color: white; background-color: #222; -moz-border-radius:1em; -webkit-border-radius:1em; padding: 1em;">
          <ul>
           <li>
            <a class="external text exitstitial" href="http://www.dd-wrt.com/phpBB2/viewtopic.php?p=458510#458510" rel="nofollow">
             Reported by buddee
            </a>
            : The reversion guide in the wiki will apply if you are running CFE 1.3, if they sent you a 1.5, the wiki method will not work, or at least for some, giving problems due to an improper board_id.
           </li>
          </ul>
          <p>
           <br/>
           If you have DD-WRT installed and need to re-install the Netgear firmware for some reason, or just put the router in to tftp acceptance mode...
          </p>
          <ol>
           <li>
            ssh or telnet to the router
           </li>
           <li>
            enter the following command at the shell prompt (this effectively deletes the OS causing the router to go in to recovery mode on the next reboot/power cycle):
            <ul>
             <li>
              "erase linux"
             </li>
            </ul>
           </li>
           <li>
            you should see a message similar to:
            <ul>
             <li>
              Unlocking linux ...
              <br/>
              Erasing linux ...
             </li>
            </ul>
           </li>
           <li>
            wait until you are returned back to the shell prompt
           </li>
           <li>
            power cycle the router
           </li>
           <li>
            allow the router to finish booting (the Power LED will be lit solid for 10-15 seconds)
           </li>
           <li>
            there after the Power LED should be flashing
           </li>
           <li>
            you should be able to ping the router IP, generally 192.168.1.1
           </li>
           <li>
            if you get a ping response the router is in tftp recovery mode
           </li>
           <li>
            you should now be able to tftp an appropriate *.chk file to the router
            <ul>
             <li>
              you can upload either an original Netgear firmware OR a known working DD-WRT chk file for this router
             </li>
            </ul>
           </li>
          </ol>

           <br/>
          </p>

         <h2>
          <span class="mw-headline" id="Power_supply_information">
           Power supply information
          </span>
         </h2>
         <div class="linebox" style="border: 1px solid #404040; color: white; background-color: #222; -moz-border-radius:1em; -webkit-border-radius:1em; padding: 1em;">
          <p>
           Initial refurbs sold in 2008 and early 2009 had potential issues with their power supplies. This caused stability issues with the routers.
          </p>
          <p>
           Most recent models (new and refurbs from mid 2009 on wards) come with a switching power supply:
          </p>
          <ul>
           <li>
            voltage: 12 vDC
           </li>
           <li>
            amp: 1500mA
           </li>
           <li>
            plug size:
            <ul>
             <li>
              2.1mm ID (confirmed with digital caliper)
             </li>
             <li>
              5.5mm OD (confirmed with digital caliper)
             </li>
             <li>
              Center=Positive (center pin)
             </li>
             <li>
              Shield=Negative
             </li>
            </ul>
           </li>
          </ul>
          <ul>
           <li>
            kill-a-watt reported power usage:
            <ul>
             <li>
              3 watts on initial boot up or with all radios disabled (idle)
             </li>
             <li>
              5 watts with just one radio (with default TX power) - idle
             </li>
             <li>
              6 watts with both radios (with default TX power on each radio) - idle
             </li>
            </ul>
           </li>
          </ul>
          <p>
           Power supplies for this router are manufactured by two vendors (according to FCC information and based on purchases from 2009 and 2010):
          </p>
          <ul>
           <li>
            DVE - units from this manufacturer have been known to be bad or inconsistent from the early batches of routers sold in 2008 to mid 2009. Please read the main support thread for more information about this.
           </li>
           <li>
            ITE
           </li>
          </ul>
          <p>
           It has been recommended to go with a non-switching transformer for those wanting to increase the power to the radios beyond the default mW per radio. However, some have found it difficult to find such power supplies (transformer based above 1500mA). A suggestion is to use a *quality* regulated 12vdc 2000mA (or higher amperage) switching power supply. If you believe to have stability issues that a 30/30/30 reset + full power cycle may not be not solving... try another power supply if you have one available.
          </p>
          <p>
           Be sure to review the additional discussions links in the Peacock and main support thread regarding power supplies and bad capacitors. There have been no known reports regarding the router itself suffering from bad capacitors.
          </p>

         <h3>
          <span class="mw-headline" id="5GHz_.28and_N_over_5GHz.29_information">
           5GHz (and N over 5GHz) information
          </span>
         </h3>
         <div class="linebox" style="border: 1px solid #404040; color: white; background-color: #222; -moz-border-radius:1em; -webkit-border-radius:1em; padding: 1em;">
          <p>
           (work in progress)
          </p>
          <p>
           The primary advantage some users of this router believe is the ability to use 802.11n over the less crowded 5ghz band. However, there are many considerations. We will try to list some of those here over time.
          </p>
          <ol>
           <li>
            from a forum post, read this article about
            <a class="external text exitstitial" href="http://db.tidbits.com/article/10849?print_version=1" rel="nofollow">
             Improve Wi-Fi Performance in the 5 GHz Band
            </a>
           </li>
           <li>
            as noted several times in this wiki entry, the internal antennas reduce the potential range of using the 5Ghz band on this router. It is still usable for many depending on router placement and possible usage requirements.
           </li>
          </ol>
          <p>
           How do I get "270 mpbs" 802.11n?
          </p>
          <ol>
           <li>
            you can only obtain this advertised connection speed on the wl0 radio through the 5GHz band (N only-5). The 2.4ghz band on this specific router only supports 20mhz wide for a maximum of "130 mbps" connection speeds.
           </li>
           <li>
            you need to set your channel width to 40mhz. If you set to 20mhz you'll get the 130 mpbs rated connection speed.
           </li>
           <li>
            it is recommended to NOT set the channel and/or channel width to AUTO as it may not guarantee that the router will select a channel or width your wireless NIC or WDS/bridge may support.
           </li>
           <li>
            note that although the router and/or your wireless NIC may say connected at 270 mbps (for 40mhz channel width users) or 130 mpbs (for 20mhz users on either the 2.4GHz or 5GHz band)... THAT DOES NOT MEAN YOU WILL GET THOSE ACTUAL TRANSFER SPEEDS IN REAL WORLD USAGE. If you doubt that, read the article above about 5GHz performance and use google to learn why.
           </li>
          </ol>
         </div>

           <li>
            What do the front LEDs mean and how do they indicate status?
            <ol>
             <li>
              reference
              <a class="external text exitstitial" href="http://forum1.netgear.com/showthread.php?t=45097" rel="nofollow">
               WNDR3300 one activity LED flashes infrequently
              </a>
              (credit for partial paste for below)
              <ul>
               <li>
                Power (not directly labeled) - The power light blinks when it is starting up or the restore factory settings button is pressed.
               </li>
               <li>
                2.4GHz Mode - This light will blink if there are data transmitted wirelessly in this mode
               </li>
               <li>
                5 GHz Mode - This light will blink if there are data transmitted wirelessly in this mode
               </li>
               <li>
                Internet (not directly labeled, looks like an lower case "i")- This light will blink if an IP address has been received and data is being transmitted and received
               </li>
               <li>
                LAN 1-4 (labeled with actual numbers "1" "2" "3" and "4") - This will blink if there is traffic or data passing on this port.
               </li>
              </ul>
             </li>
            </ol>
           </li>
          </ol>
          <h4>
           <span class="mw-headline" id="Dome_LEDs">
            Dome LEDs
           </span>
          </h4>
          <ol>
           <li>
            How are the dome LEDs handled?
            <ol>
             <li>
              the original Netgear firmware flashed the dome lights based on various activities on the wireless radios, etc.
             </li>
             <li>
              DD-WRT automatically turns off the dome lights on boot up with code changes since builds around Nov 2008
              <ol>
               <li>
                reference
                <a class="external text exitstitial" href="http://svn.dd-wrt.com:8000/dd-wrt/changeset/10994" rel="nofollow">
                 Changeset 10994
                </a>
               </li>
              </ol>
             </li>
            </ol>
           </li>
           <li>
            Can I control the dome LEDs?
            <ol>
             <li>
              it does not appear that access to the dome lights are easily exposed to end users
             </li>
             <li>
              reference thread:
              <a class="external text exitstitial" href="http://www.dd-wrt.com/phpBB2/viewtopic.php?t=51442" rel="nofollow">
               WNDR3300 Dome Lights
              </a>
             </li>
            </ol>
           </li>
           <li>
            What if the dome LEDs come back on after I've flashed with a good known version of DD-WRT?
            <ul>
             <li>
              back up your settings using the hardware independent backup script (search the wiki and forums)
             </li>
             <li>
              try performing a full 30/30/30 reset per Peacock, and do a full power cycle
             </li>
             <li>
              if the dome does not turn off, either your unit is going bad or you can physically unplug the ribbon cable inside the unit to permanently disable it
             </li>
            </ul>
           </li>
          </ol>
