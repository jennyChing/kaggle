         <h1>
          Free* SSH file transfer, terminal and tunneling!
         </h1>

         </a>
         <p>
          Our SSH and SFTP client for Windows incorporates:
         </p>
         <ul>
          <li>
           One of the most advanced
           <a href="#gui-sftp">
            graphical SFTP
           </a>
           clients.
          </li>
          <li>
           <b>
            Single-click Remote Desktop forwarding.
           </b>
          </li>
          <li>
           State-of-the-art
           <a href="#term">
            terminal emulation
           </a>
           with support for the bvterm, xterm, and vt100 protocols.
          </li>
          <li>
           Support for corporation-wide
           <a href="#sso">
            single sign-on
           </a>
           using
           <b>
            SSPI (GSSAPI) Kerberos 5 and NTLM
           </b>
           user authentication, as well as
           <b>
            Kerberos 5 host authentication
           </b>
           .
          </li>
          <li>
           Support for ECDSA, RSA and DSA
           <b>
            public key authentication
           </b>
           with comprehensive user keypair management.
          </li>
          <li>
           <b>
            FIPS 140-2 validation
           </b>
           : Bitvise SSH Client 6.xx uses the FIPS-certified DLL version of Crypto++ 5.3.0, validated by NIST under
           <a href="http://csrc.nist.gov/cryptval/140-1/1401val2007.htm" onclick='window.open("http://csrc.nist.gov/cryptval/140-1/1401val2007.htm"); return false;'>
            certificate #819
           </a>
           .
          </li>
          <li>
           <b>
            Obfuscated SSH
           </b>
           with an optional keyword. When supported and enabled in both the client and server, obfuscation makes it more difficult for an observer to detect that the protocol being used is SSH.
          </li>
          <li>
           Powerful
           <a href="#port-forwarding">
            SSH port forwarding
           </a>
           capabilities, including dynamic forwarding through integrated SOCKS and HTTP CONNECT proxy.
          </li>
          <li>
           Powerful command-line parameters which make the SSH client
           <a href="#custom">
            highly customizable
           </a>
           and suitable for use in specific situations and controlled environments.
          </li>
          <li>
           An
           <a href="#ftp-bridge">
            FTP-to-SFTP bridge
           </a>
           allowing you to connect to an SFTP server using legacy FTP applications.
          </li>
          <li>
           An advanced, scriptable command-line SFTP client,
           <a href="#sftpc">
            sftpc
           </a>
           .
          </li>
          <li>
           A scriptable command-line remote execution client,
           <a href="#sexec">
            sexec

           , and a command-line terminal console client,

            stermc
           </b>
           .
          </li>
          <li>
           A scriptable command-line tunneling client,
           <a href="#stnlc">
            stnlc
           </a>
           , with support for static port forwarding rules, dynamic SOCKS-based tunneling, and FTP-to-SFTP bridge.

           Bitvise SSH Server
           <a href="#with-bvsshserver">
            remote administration
           </a>
           features.

         <a name="gui-sftp">
         </a>
         <h1>
          Graphical SFTP File Transfers
         </h1>
         <p>
          Bitvise SSH Client incorporates what we believe is one of the most advanced SFTP file transfer clients, with support for the following:
         </p>
         <ul>
          <li>
           automatic resuming, text file awareness, recursive subdirectory transfers;
          </li>
          <li>
           powerful, advanced
           <b>
            transfer list management
           </b>
           ;
          </li>
          <li>
           <b>
            high speed
           </b>
           - in the tens or hundreds of MB/s when connected to Bitvise SSH Server, network bandwidth and disk resources permitting;
          </li>
          <li>
           <b>
            fast responsiveness
           </b>
           to user input even when in the middle of multiple file transfers.
          </li>
         </ul>
         <p>
          Bitvise SSH Client also incorporates an advanced, scriptable
          <a href="#sftpc">
           command-line SFTP client
          </a>
          .
         </p>
         <a name="term">
         </a>
         <h1>
          Terminal Access - Remote Console
         </h1>
         <p>
          Bitvise SSH Client provides integrated access to the SSH server's console, either via
          <b>
           VT-100
          </b>
          and
          <b>
           xterm
          </b>
          protocols supported by most SSH servers on any platform, or via the enhanced
          <b>
           bvterm
          </b>
          protocol supported by Bitvise SSH Server:
         </p>
         <ul>
          <li>
           all terminal emulation is done in a regular Windows console, using standard
Windows fonts, supporting standard settings for Windows consoles;
          </li>
          <li>
           UTF support with VT-100 and xterm, Unicode support with bvterm (full Unicode support on NT-series Windows platforms);
          </li>
          <li>
           full color support with xterm and bvterm;
          </li>
          <li>
           support for large screen buffers, all key combinations including Alt-keys and
accurate console rendering with bvterm (fully accurate on NT-series
Windows platforms).
          </li>
         </ul>
         <p>
          Bitvise SSH Client also incorporates a command line terminal emulation client,
          <b>
           stermc
          </b>
          , as well as a
          <a href="#sexec">
           command-line remote execution client
          </a>
          (sexec).
         </p>
         <a name="sso">
         </a>
         <h1>
          Single sign-on
         </h1>
         <p>
          When using Bitvise SSH Client to connect to a GSSAPI-enabled SSH server in the same or a trusted Windows domain, you can let
          <b>
           Kerberos 5
          </b>
          (or on older platforms, NTLM) perform the server as well as user authentication for you. No manual host key verification; no management of user passwords and public keys. Just tell the SSH client which server in the domain to connect to, and if that server is Bitvise SSH Server (WinSSHD) or another server with compatible support for GSSAPI, the two programs will authenticate and establish a secure connection automatically.
         </p>
         <a name="port-forwarding">
         </a>
         <h1>
          Port Forwarding
         </h1>
         <p>
          Bitvise SSH Client is a powerful SSH2 port forwarding client with many features, including:
         </p>
         <ul>
          <li>
           <p>
            Dynamic tunneling via
            <b>
             integrated proxy
            </b>
            supporting SOCKS4, SOCKS4A, SOCKS5 and HTTP CONNECT proxy tunneling
protocols. Configure your applications to connect to target hosts
through the SSH Client's proxy, and no manual tunneling rules need be
configured.
           </p>
           <p>
            How to set up programs for dynamic tunneling:
            <a href="/files/socks-firefox.gif" onclick='window.open("/files/socks-firefox.gif"); return false;'>
             Firefox
            </a>
            ,
            <a href="/files/socks-ie.gif" onclick='window.open("/files/socks-ie.gif"); return false;'>
             IE
            </a>
            ; with
            <a href="/files/socks-tunnelier.gif" onclick='window.open("/files/socks-tunnelier.gif"); return false;'>
             these settings
            </a>
            in Bitvise SSH Client.
           </p>
          </li>
          <li>
           Port forwarding settings can be saved to and loaded from a
           <b>
            profile
           </b>
           . One can maintain multiple complex tunneling configurations without having to manually enter parameters before each session.
          </li>
          <li>
           Bitvise SSH Client minimizes its presence by displaying only a
           <b>
            system tray icon
           </b>
           when running in the background. If an error is encountered while the program is minimized, the icon reflects that.
          </li>
          <li>
           <b>
            Server-side forwarding
           </b>
           :
with Bitvise SSH Server and Client, a server and multiple clients can be set up
so that all port forwarding rules are configured centrally at the
server, without requiring any client-side setting updates. The SSH
clients only need to be configured once, and port forwarding rules can
easily be changed when necessary.
          </li>
          <li>
           Command-line parameters are supported. Using
           <b>
            command-line automation
           </b>
           ,
a port forwarding session can be started from a link in the Startup
menu without requiring any user interaction whatsoever. Help with the
command-line parameters can be found in the SSH client log area when it
is first started, or by executing 'BvSsh /?' from a command prompt.
          </li>
          <li>
           After an SSH session is established, any external application can be
           <b>
            launched
           </b>
           automatically.
          </li>
          <li>
           If an SSH session is interrupted, Bitvise SSH Client can
           <b>
            automatically reconnect
           </b>
           to the server.
          </li>
         </ul>
         <p>
          Bitvise SSH Client also incorporates an advanced, scriptable
          <a href="#stnlc">
           command-line tunneling client
          </a>
          .
         </p>
         <a name="custom">
         </a>
         <h1>
          Highly customizable
         </h1>
         <p>
          Run 'BvSsh -?' for a list of
          <a href="/files/tunnelier-params.txt">
           command-line parameters
          </a>
          providing flexibility to:
         </p>
         <ul>
          <li>
           make Bitvise SSH Client load a profile and connect immediately on startup and exit on disconnect using the -profile, -loginOnStartup and -exitOnLogout
parameters;
          </li>
          <li>
           make Bitvise SSH Client hide portions of its user interface (main window, authentication messages, access to SSH features) using the -hide and -menu parameters;
          </li>
          <li>
           run Bitvise SSH Client from removable media and prevent it from making any changes to the Windows registry - and yet use public key authentication and
verify host keys using the -noRegistry, -keypairFile and -hostKeyFile
parameters;
          </li>
         </ul>
         <p>
          and more.
         </p>
         <p>
          Additionally, when the graphical Bitvise SSH Client is running, you can use the command line utility
          <b>
           BvSshCtrl
          </b>
          to programmatically cause the client to log in, log out, or exit. Run 'BvSshCtrl' for help.
         </p>
         <a name="portable">
         </a>
         <h1>
          Portable
         </h1>
         <p>
          Our users have made use of Bitvise SSH Client's customizability to create
          <a href="tunnelier-portable">
           portable distributions
          </a>
          that can run from a USB key, in the U3 and PortableApps formats.
         </p>
         <a name="sftpc">
         </a>
         <h1>
          Command-Line SFTP Client
         </h1>
         <p>
          Bitvise SSH Client incorporates
          <b>
           sftpc
          </b>
          , an advanced command-line SFTP client which supports:
         </p>
         <ul>
          <li>
           queued
           <b>
            background transfers
           </b>
           so you can continue browsing directories while a file downloads ('get * -bg');
          </li>
          <li>
           powerful command-line parameters for automated
           <b>
            scripted transfers
           </b>
           launched from a batch file or an external program;
          </li>
          <li>
           automatic
           <b>
            text file conversion
           </b>
           when transferring files between platforms (the 'type' command);
          </li>
          <li>
           <b>
            wildcards
           </b>
           - the * and ? wildcard characters can match more than one file;
          </li>
          <li>
           <b>
            recursive transfers
           </b>
           - a simple 'get * -s' downloads all files and subdirectories;
          </li>
          <li>
           <b>
            resume support
           </b>
           - 'get file.txt -r' resumes a previously downloaded file if it already exists;
          </li>
          <li>
           <b>
            high speed
           </b>
           - in the tens of MB/s when connected to Bitvise SSH Server, network bandwidth and disk resources permitting.

         <a name="sexec">
         </a>
         <h1>
          Command-Line Remote Execution Client
         </h1>
         <p>
          Bitvise SSH Client incorporates
          <b>
           sexec
          </b>
          ,
a scriptable command-line remote execution client which can be used to
securely execute programs on remote machines from a batch file. The
batch file can perform actions based on the exit code reported from the
remote program, and can capture the remote program's output for
processing by simple redirection of sexec output.
         </p>
         <p>
          Another command-line program included with Bitvise SSH Client,
          <b>
           stermc
          </b>
          ,
conveniently opens an SSH terminal emulation session with xterm, VT-100
and bvterm support in the same Command Prompt window it is started
from. If you SSH from computer A to computer B running Bitvise SSH Server, and you also have Bitvise SSH Client installed on computer B, you can use stermc to
conveniently establish an onward SSH terminal session to computer C.
         </p>
         <p>
          All Bitvise command-line clients support all authentication methods
supported by Bitvise SSH Client, can load settings from a profile configured in the graphical client, work with the same user keypair and host key database as the graphical client, can connect through a SOCKS or HTTP CONNECT proxy, and support a common set of command-line parameters controlling most aspects of establishing an SSH connection.
         </p>
         <a name="stnlc">
         </a>
         <h1>
          Command-Line Tunneling Client
         </h1>
         <p>
          Bitvise SSH Client incorporates
          <b>
           stnlc
          </b>
          , a scriptable command line port forwarding client with support for static port forwarding rules, dynamic SOCKS-based tunneling, as well as the FTP-to-SFTP bridge.
         </p>
         <p>
          For more information about using this client, run 'stnlc -h'.
         </p>
         <a name="ftp-bridge">
         </a>
         <h1>
          FTP-to-SFTP Bridge
         </h1>
         <p>
          Bitvise SSH Client incorporates an FTP-to-SFTP bridge which performs translation between the FTP and SFTP protocols, allowing any FTP client application to connect securely to an SFTP server through Bitvise SSH Client. Both
          <b>
           active mode
          </b>
          and
          <b>
           passive mode
          </b>
          FTP data connections are supported, as well as restarted transfers,
directory listings and other features. All FTP clients should be able
to list directories and transfer files across Bitvise SSH Client's FTP-to-SFTP
bridge, including integrated clients in applications such as
DreamWeaver, GUI clients such as BulletProof FTP, and command line
clients such as the default ftp.exe client accompanying Windows. If an
FTP client you use fails to work with Bitvise SSH Client's FTP-to-SFTP bridge,
          <a href="https://fogbugz.bitvise.com/default.asp?pg=pgPublicEdit&amp;ixProject=3">
           let us know
          </a>
          .
         </p>
         <p>
          See
          <a href="/ftp-bridge">
           Accessing an SFTP Server with an FTP Client
          </a>
          for more information about the FTP bridge.
         </p>
         <a name="with-bvsshserver">
         </a>
         <h1>
          Using with Bitvise SSH Server
         </h1>
         <p>
          When connected to Bitvise SSH Server, our client can do all of the above, as well as the following:
         </p>
         <ul>
          <li>
           <b>
            bvterm console
           </b>
           :
when connected to Bitvise SSH Server, our client provides you with the best console
available on the Windows platform. Our bvterm protocol supports the
full spectrum of a Windows console's features: colors, Unicode
characters, all-key access and large scrollable buffers.
          </li>
          <li>
           <b>
            remote server configuration
           </b>
           :
using Bitvise SSH Client, all SSH server settings can be configured remotely as if you were sitting in front of the server itself.
          </li>
         </ul>
