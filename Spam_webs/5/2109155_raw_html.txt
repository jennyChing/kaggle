         I've been getting more and more interested in how folks extend their applications using plugins and things. In my new ongoing
         <a href="http://www.hanselman.com/blog/ReadingToBeABetterDeveloperTheCoding4FunDevKit.aspx">
          quest to
          <strong>
           read source code to be a better developer
          </strong>

         , Dear Reader, I present to you
         <em>
          twenty-eighth in a infinite
         </em>
         number of posts of "
         <a href="http://www.hanselman.com/blog/CategoryView.aspx?category=Source+Code">
          <strong>
           The Weekly Source Code

         </a>
         ."
         <p>
          The
          <a href="http://www.apple.com/iphone">
           iPhone 2.0
          </a>
          was announced today and even though I am trying to come up with a better solution to show it off
          <a href="http://www.hanselman.com/blog/CommunityCallToActionNOTNorthwind.aspx">
           than Northwind
          </a>
          , I was moved when I saw
          <a href="http://weblogs.asp.net/aaronlerch/archive/2008/06/08/rock-the-iphone-with-asp-net-mvc.aspx">
           Aaron's "Rock the iPhone with ASP.NET MVC" post
          </a>
          . I've had this same post in my "to blog" queue for at least two months, but now he's done it himself and beat us all to it. Kudos to Aaron.
         </p>
         <p>
          I'm primarily an ASP.NET developer and I browse on mobile devices all the time, and have been doing so for years. I made DasBlog and this website
          <a href="http://www.hanselman.com/blog/MakingDasBlogWorkOnMobileDevices.aspx">
           sniff and apply a mobile template almost two years ago
          </a>
          . If you visit hanselman.com on a BlackBerry or Windows Mobile or a number of other tiny devices, you'll get the same site with a mobile skin. I'll need to do something for the iPhone as well.
         </p>
         <p>
          <a href="http://alpascual.com/blog/al/archive/2007/12/20/add-support-for-the-iphone-to-your-asp-net-pages.aspx">
           Al Pascual added iPhone "support" to Community Server
          </a>
          late last year by redirecting iPhone users to his RSS Feed, giving those users a cleaner experience than they would have gotten ordinarily.
         </p>
         <p>
          <a href="http://dotnetslackers.com/articles/aspnet/DevelopingForTheiPhone.aspx">
           Steve Orr has a good article on iPhone Development
          </a>
          pointing out a number of important details like the viewport meta tags that give the iPhone clues on how to scale the page as well as his
          <a href="http://dotnetslackers.com/articles/aspnet/AdvancediPhoneDevelopment.aspx">
           Advanced iPhone Development
          </a>
          article with a smidge more detail (not really advanced).
         </p>
         <p>
          I was wondering how easy it'd be to make an iPhone application with ASP.NET MVC, and it turns out it's pretty easy with a combination of things. First, the
          <a href="http://code.google.com/p/iui/">
           IUI project at Google Code
          </a>
          is a series of Javascript and PNG assets to make a basic iPhone apps. It's really quite elegant, what the iui team has done, based on work from
          <a href="http://joehewitt.com/">
           Joe Hewitt
          </a>
          . Joe has a
          <a href="http://www.joehewitt.com/blog/introducing_iui.php">
           fine introductory blog post that explains some of the concepts
          </a>
          .
         </p>
         <p>
          You can use Safari for to
          <a href="http://developer.apple.com/iphone/program/">
           simulate an iPhone
          </a>
          , or
          <a href="http://paininthetech.com/2007/10/03/fake-iphone-user-agent/">
           Firefox with the User Agent Switcher and some customization
          </a>
          .
         </p>

         <p>
          Notice the little toggle switches? They are in a div with class="toggle" and a click event handler gets hooked up to those in Javascript:
         </p>
         <pre class="javascript" name="code">
          addEventListener("click", function(event)
          <br/>
          {
          <br/>
          var div = findParent(event.target, "div");
          <br/>
          if (div &amp;&amp; hasClass(div, "toggle"))
          <br/>
          {
          <br/>
          div.setAttribute("toggled", div.getAttribute("toggled") != "true");
          <br/>
          event.preventDefault();
          <br/>
          }
          <br/>
          }, true);
         </pre>
         <p>
          Based on the toggle state, the switch renders as appropriate:
         </p>
         <pre class="css" name="code">
          .toggle {

          <br/>
          background: #FFFFFF url(toggle.png) repeat-x;
          <br/>
          font-size: 19px;
          <br/>
          font-weight: bold;
          <br/>
          line-height: 30px;
          <br/>
          }
          <br/>
          <br/>
          .toggle[toggled="true"] {
          <br/>
          border: 1px solid #143fae;
          <br/>
          background: #194fdb url(toggleOn.png) repeat-x;
          <br/>
          }
         </pre>
         <p>
          The iPhone HTML page above has multiple forms, explicit control over divs - all things that MVC is better suited for than WebForms.
          <a href="http://weblogs.asp.net/aaronlerch/archive/2008/06/08/rock-the-iphone-with-asp-net-mvc.aspx">
           Aaron explains in his blog post
          </a>
          how he just brings IUI into his content folder, but the trick was that IUI expects HTML fragments to be returned from Ajax calls, not JSON.
         </p>
         <p>
          Aaron's Views are then Partial Views intended for consumption from built-in Ajax calls by IUI, so the product's list page looks just like this. Note the markup is all alone.
         </p>
         <pre class="html" name="code">
          &lt;ul id="products" title="Products"&gt;
          <br/>
          &lt;%  foreach (Product p in ViewData.Model) { %&gt;
          <br/>
          &lt;li&gt;&lt;%= Html.ActionLink(p.ProductName, "Index", "Products", new { id = p.ProductID }) %&gt;&lt;/li&gt;
          <br/>
          &lt;% } %&gt;
          <br/>
          &lt;/ul&gt;
         </pre>
         <p>
          Very cool and very easy.
         </p>
