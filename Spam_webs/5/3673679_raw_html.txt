            <span class="vote-accepted-on load-accepted-answer-date" title="loading when this answer was accepted...">
             accepted
            </span>
           </div>
          </td>
          <td class="answercell">
           <div class="post-text" itemprop="text">
            <p>
             In a POSIX shell, the syntax
             <code>
              ${t:-2}
             </code>
             means something different - it expands to the value of
             <code>
              t
             </code>
             if
             <code>
              t
             </code>
             is set and non null, and otherwise to the value
             <code>
              2
             </code>
             . To trim a single character by parameter expansion, the syntax you probably want is
             <code>
              ${t%?}
             </code>
            </p>
            <p>
             Note that in
             <code>
              bash
             </code>
             ,
             <code>
              ${t:(-2)}
             </code>
             <em>
              is
             </em>
             legal as a substring expansion but is probably not what you want, since it returns the substring starting at a position 2 characters in from the end (i.e. it removes the
             <em>
              first
             </em>
             character of the string).
            </p>
           </div>

           <div class="post-text" itemprop="text">
            <p>
             for removing the last
             <code>
              n
             </code>
             characters from a line that makes no use of
             <code>
              sed
             </code>
             OR
             <code>
              awk
             </code>
             :
            </p>
            <pre>
             <code>
              &gt; echo lkj | rev | cut -c (n+1)- | rev
             </code>
            </pre>
            <p>
             so for example you can delete the last character
             <code>
              one character
             </code>
             using this:
            </p>
            <pre>
             <code>
              &gt; echo lkj | rev | cut -c 2- | rev

&gt; lk
             </code>
            </pre>
            <p>
             from
             <code>
              rev
             </code>
             manpage:
            </p>
            <blockquote>
             <p>
              DESCRIPTION
              <br/>
              The rev utility copies the specified files to the standard output,
       reversing the order of characters in every line.  If no files are speci-
       fied, the standard input is read.
             </p>
            </blockquote>
            <p>
             UPDATE:
            </p>
            <p>
             if you don't know the length of the string, try:
            </p>
            <pre>
             <code>
              $ x="lkj"
$ echo "${x%?}"
lk
             </code>
            </pre>

             <code>
              t=lkj
echo ${t:0:${#t}-1}
             </code>
            </pre>
            <p>
             You get a substring from 0 to the string length -1. Note however that this substraction is  bash specific, and won't work on other shells.
            </p>

             <code>
              dash
             </code>
             isn't able to parse even
            </p>
            <pre>
             <code>
              echo ${t:0:$(expr ${#t} - 1)}
             </code>
            </pre>
            <p>
             For example, on Ubuntu,
             <code>
              /bin/sh
             </code>
             is
             <code>
              dash
             </code>

           <div class="post-text" itemprop="text">
            <p>
             You can also use
             <code>
              head
             </code>
             to print out all but the last character.
            </p>
            <pre>
             <code>
              $ s='i am a string'
$ news=$(echo -n $s | head -c -1)
$ echo $news
i am a strin
             </code>
            </pre>
            <p>
             But unfortunately some versions of
             <code>
              head
             </code>
             do not include the leading
             <code>
              -
             </code>
             option. This is the case for the
             <code>
              head
             </code>
             that comes with OS X.
            </p>
