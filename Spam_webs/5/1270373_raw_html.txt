   <h1 class="firstHeading" id="firstHeading" lang="en">
    Blue Pill (software)
   </h1>
   <div class="mw-body-content" id="bodyContent">

    </div>

    <div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en">
     <p>
      <b>
       Blue Pill
      </b>
      is the codename for a
      <a href="/wiki/Rootkit" title="Rootkit">
       rootkit
      </a>
      based on
      <a href="/wiki/X86_virtualization" title="X86 virtualization">
       x86 virtualization
      </a>
      . Blue Pill originally required
      <a class="mw-redirect" href="/wiki/AMD-V" title="AMD-V">
       AMD-V
      </a>
      (Pacifica) virtualization support, but was later ported to support
      <a class="mw-redirect" href="/wiki/Intel_VT-x" title="Intel VT-x">
       Intel VT-x
      </a>
      (Vanderpool) as well. It was designed by
      <a href="/wiki/Joanna_Rutkowska" title="Joanna Rutkowska">
       Joanna Rutkowska
      </a>
      and originally demonstrated at the
      <a href="/wiki/Black_Hat_Briefings" title="Black Hat Briefings">
       Black Hat Briefings
      </a>
      on August 3, 2006, with a reference implementation for the
      <a class="mw-redirect" href="/wiki/Microsoft_Windows_Vista" title="Microsoft Windows Vista">
       Microsoft Windows Vista
      </a>
      kernel.
     </p>
     <p>
      The name is a reference to the
      <a href="/wiki/Red_pill_and_blue_pill" title="Red pill and blue pill">
       red pill/blue pill
      </a>
      concept from the 1999 film
      <i>
       <a href="/wiki/The_Matrix" title="The Matrix">
        The Matrix
       </a>
      </i>
      .
     </p>
     <p>
     </p>

     <h2>
      <span class="mw-headline" id="Overview">
       Overview
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Blue_Pill_(software)&amp;action=edit&amp;section=1" title="Edit section: Overview">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      The Blue Pill concept is to trap a running instance of the operating system by starting a thin
      <a href="/wiki/Hypervisor" title="Hypervisor">
       hypervisor
      </a>
      and virtualizing the rest of the machine under it. The previous operating system would still maintain its existing references to all devices and files, but nearly anything, including hardware interrupts, requests for data and even the system time could be intercepted (and a fake response sent) by the hypervisor. The original concept of Blue Pill was published by another researcher at IEEE Oakland on May 2006, under the name VMBR (virtual-machine based rootkit).
      <sup class="reference" id="cite_ref-1">
       <a href="#cite_note-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      <a href="/wiki/Joanna_Rutkowska" title="Joanna Rutkowska">
       Joanna Rutkowska
      </a>
      claims that, since any detection program could be fooled by the hypervisor, such a system could be "100% undetectable". Since AMD virtualization is seamless by design, a virtualized guest is not supposed to be able to query whether it is a guest or not. Therefore, the only way Blue Pill could be detected is if the virtualization implementation were not functioning as specified.
      <sup class="reference" id="cite_ref-2">
       <a href="#cite_note-2">
        <span>
         [
        </span>
        2
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      This assessment, repeated in numerous press articles, is disputed: AMD issued a statement dismissing the claim of full undetectability.
      <sup class="reference" id="cite_ref-3">
       <a href="#cite_note-3">
        <span>
         [
        </span>
        3
        <span>
         ]
        </span>
       </a>
      </sup>
      Some other security researchers and journalists also dismissed the concept as implausible.
      <sup class="reference" id="cite_ref-4">
       <a href="#cite_note-4">
        <span>
         [
        </span>
        4
        <span>
         ]
        </span>
       </a>
      </sup>
      Virtualization could be detected by a
      <a href="/wiki/Timing_attack" title="Timing attack">
       timing attack
      </a>
      relying on external sources of time.
      <sup class="reference" id="cite_ref-5">
       <a href="#cite_note-5">
        <span>
         [
        </span>
        5
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      In 2007, a group of researchers led by
      <a class="new" href="/w/index.php?title=Thomas_Ptacek&amp;action=edit&amp;redlink=1" title="Thomas Ptacek (page does not exist)">
       Thomas Ptacek
      </a>
      of
      <a class="new" href="/w/index.php?title=Matasano_Security&amp;action=edit&amp;redlink=1" title="Matasano Security (page does not exist)">
       Matasano Security
      </a>
      challenged Rutkowska to put Blue Pill against their rootkit detector software at that year's Black Hat conference,
      <sup class="reference" id="cite_ref-6">
       <a href="#cite_note-6">
        <span>
         [
        </span>
        6
        <span>
         ]
        </span>
       </a>
      </sup>
      but the deal was deemed a no-go following Rutkowska's request for $384,000 in funding as a prerequisite for entering the competition.
      <sup class="reference" id="cite_ref-7">
       <a href="#cite_note-7">
        <span>
         [
        </span>
        7
        <span>
         ]
        </span>
       </a>
      </sup>
      Rutkowska and Alexander Tereshkin countered detractors' claims during a subsequent Black Hat speech, arguing that the proposed detection methods were inaccurate.
      <sup class="reference" id="cite_ref-8">
       <a href="#cite_note-8">
        <span>
         [
        </span>
        8
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      The source code for Blue Pill has since been made public,
      <sup class="reference" id="cite_ref-9">
       <a href="#cite_note-9">
        <span>
         [
        </span>
        9
        <span>
         ]
        </span>
       </a>
      </sup>
      <sup class="reference" id="cite_ref-10">
       <a href="#cite_note-10">
        <span>
         [
        </span>
        10
        <span>
         ]
        </span>
       </a>
      </sup>
      under the following license:
      <i>
       Any unauthorized use (including publishing and distribution) of this software requires a valid license from the copyright holder. This software has been provided for the educational use only during the Black Hat training and conference.
      </i>
      <sup class="reference" id="cite_ref-11">
       <a href="#cite_note-11">
        <span>
         [
        </span>
        11
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <h2>
      <span class="mw-headline" id="Red_Pill">
       Red Pill
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Blue_Pill_(software)&amp;action=edit&amp;section=2" title="Edit section: Red Pill">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Red Pill is a technique to detect the presence of a virtual machine also developed by
      <a href="/wiki/Joanna_Rutkowska" title="Joanna Rutkowska">
       Joanna Rutkowska
      </a>
      .
      <sup class="reference" id="cite_ref-12">
       <a href="#cite_note-12">
        <span>
         [
        </span>
        12
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
