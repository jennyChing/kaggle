      <h2>
       <span class="mw-headline" id="Description">
        Description
       </span>
      </h2>
      <p>
       This
       <a href="/Conditional_Tags" title="Conditional Tags">
        Conditional Tag
       </a>
       checks if
       <a href="/Pages" title="Pages">
        Pages
       </a>
       are being displayed. This is a boolean function, meaning it returns either TRUE or FALSE. This tag must be used BEFORE
       <a href="/The_Loop" title="The Loop">
        The Loop
       </a>
       and does not work inside The Loop (see
       <a href="#Notes">
        Notes
       </a>
       below).
      </p>
      <h2>
       <span class="mw-headline" id="Usage">
        Usage
       </span>
      </h2>
      <p>
       <code>
        <span style="color: #000000">
         <span style="color: #0000BB">
          &lt;?phpis_page
         </span>
         <span style="color: #007700">
          (
         </span>
         <span style="color: #0000BB">
          $page
         </span>
         <span style="color: #007700">
          );
         </span>
         <span style="color: #0000BB">
          ?&gt;
         </span>
        </span>
       </code>
      </p>
      <h2>
       <span class="mw-headline" id="Parameters">
        Parameters
       </span>
      </h2>
      <dl>
       <dt>
        <tt>
         $page
        </tt>
       </dt>
       <dd>
        (
        <a href="/How_to_Pass_Tag_Parameters#Mixed" title="How to Pass Tag Parameters">
         <i>
          mixed
         </i>
        </a>
        ) (
        <i>
         optional
        </i>
        ) Page ID, Page Title or Page Slug
        <dl>
         <dd>
          Default:
          <i>
           None
          </i>
         </dd>
        </dl>
       </dd>
      </dl>
      <h2>
       <span class="mw-headline" id="Return_Values">
        Return Values
       </span>
      </h2>
      <dl>
       <dt>
        <tt>
         (boolean)
        </tt>
       </dt>
       <dd>
        <tt>
         true
        </tt>
        on success,
        <tt>
         false
        </tt>
        on failure.
       </dd>
      </dl>
      <h2>
       <span class="mw-headline" id="Examples">
        Examples
       </span>
      </h2>
      <pre>
       is_page();
// When any single Page is being displayed.

is_page( 42 );
// When Page 42 (ID) is being displayed.

is_page( 'Contact' );
// When the Page with a
       <i>
        post_title
       </i>
       of "Contact" is being displayed.

is_page( 'about-me' );
// When the Page with a
       <i>
        post_name
       </i>
       (slug) of "about-me" is being displayed.

is_page( array( 42, 'about-me', 'Contact' ) );
// Returns true when the Pages displayed is either post ID 42, or
       <i>
        post_name
       </i>
       "about-me", or
       <i>
        post_title
       </i>
       "Contact".  Note: the array ability was added at
       <a href="/Version_2.5" title="Version 2.5">
        Version 2.5
       </a>
       .
      </pre>

      <h4>
       <span class="mw-headline" id="Testing_for_paginated_Pages">
        Testing for paginated Pages
       </span>
      </h4>
      <p>
       You can use this code to check whether you're on the nth page in a Post or PAGE Page that has been divided into pages using the
       <tt>
        &lt;!--nextpage--&gt;
       </tt>
       QuickTag. This can be useful, for example, if you wish to display meta-data only on the first page of a post divided into several pages.
      </p>
      <p>
       <b>
        Example 1
       </b>
      </p>
      <pre>
       &lt;?php         
$paged = $wp_query-&gt;get( 'paged' );

if (! $paged || $paged &lt; 2 ) 
{
    // This is not a paginated page (or it's simply the first page of a paginated page/post)

} 
else 
{
   // This is a paginated page.

}
?&gt;
      </pre>
      <p>
       <b>
        Example 2
       </b>
      </p>
      <pre>
       &lt;?php
$paged = get_query_var( 'paged' )? get_query_var( 'paged' ): false;
if ( $paged === false ) 
{
    // This is not a paginated page (or it's simply the first page of a paginated page/post)

} 
else 
{
   // This is a paginated page.

}
?&gt;
      </pre>
      <p>
       <b>
        Example 3
       </b>
      </p>
      <pre>
       &lt;?php
if ( 0 === get_query_var( 'page' ) ) {
	// This is not a paginated page (or it's simply the first page of a paginated page/post)
}
else {
	// This is a paginated page.
}
      </pre>
      <h4>
       <span class="mw-headline" id="Testing_for_sub-Pages">
        Testing for sub-Pages
       </span>
      </h4>
      <p>
       There is no
       <tt>
        is_subpage()
       </tt>
       function yet, but you can test this with a little code:
      </p>
      <p>
       <b>
        Snippet 1
       </b>
      </p>
      <pre>
       &lt;?php

global $post;     // if outside the loop

if ( is_page() &amp;&amp; $post-&gt;post_parent ) {
    // This is a subpage

} else {
    // This is not a subpage
}
?&gt;
      </pre>
      <p>
       You can create your own is_subpage() function using the code in Snippet 2. Add it to your
       <a class="external text" href="http://codex.wordpress.org/Theme_Development#Theme_Functions_File">
        functions.php
       </a>
       file. It tests for a parent page in the same way as Snippet 1, but will return the ID of the page parent if there is one, or
       <tt>
        false
       </tt>
       if there isn't.
      </p>
      <p>
       <b>
        Snippet 2
       </b>
      </p>
      <pre>
       function is_subpage() {
    global $post;                              // load details about this page

    if ( is_page() &amp;&amp; $post-&gt;post_parent ) {   // test to see if the page has a parent
        return $post-&gt;post_parent;             // return the ID of the parent post

    } else {                                   // there is no parent so ...
        return false;                          // ... the answer to the question is false
    }
}
      </pre>
      <p>
       It is advisable to use a function like that in Snippet 2, rather than using the simple test like Snippet 1, if you plan to test for sub pages frequently.
      </p>
      <p>
       To test if the parent of a page is a specific page, for instance "About" (page id
       <i>
        pid
       </i>
       2 by default), we can use the tests in Snippet 3. These tests check to see if we are looking at the page in question, as well as if we are looking at any child pages. This is useful for setting variables specific to different sections of a web site, so a different banner image, or a different heading.
      </p>
      <p>
       <b>
        Snippet 3
       </b>
      </p>
      <pre>
       &lt;?php

if ( is_page( 'about' ) || '2' == $post-&gt;post_parent ) {    
    // the page is "About", or the parent of the page is "About"
    $bannerimg = 'about.jpg';

} elseif ( is_page( 'learning' ) || '56' == $post-&gt;post_parent ) {	
    $bannerimg = 'teaching.jpg';

} elseif ( is_page( 'admissions' ) || '15' == $post-&gt;post_parent ) { 
    $bannerimg = 'admissions.jpg';

} else { 
    $bannerimg = 'home.jpg'; // just in case we are at an unclassified page, perhaps the home page
}	

?&gt;
      </pre>
      <p>
       Snippet 4 is a function that allows you to carry out the tests above more easily. This function will return true if we are looking at the page in question (so "About") or one of its sub pages (so a page with a parent with ID "2").
      </p>
      <p>
       <b>
        Snippet 4
       </b>
      </p>
      <pre>
       function is_tree( $pid ) {      // $pid = The ID of the page we're looking for pages underneath
    global $post;               // load details about this page

    if ( is_page($pid) )
        return true;            // we're at the page or at a sub page

    $anc = get_post_ancestors( $post-&gt;ID );
    foreach ( $anc as $ancestor ) {
        if( is_page() &amp;&amp; $ancestor == $pid ) {
            return true;
        }
    }

    return false;  // we arn't at the page, and the page is not an ancestor
}
      </pre>
      <p>
       Add Snippet 4 to your
       <a class="external text" href="http://codex.wordpress.org/Theme_Development#Theme_Functions_File">
        functions.php
       </a>
       file, and call
       <tt>
        is_tree( 'id' )
       </tt>
       to see if the current page is the page, or is a sub page of the page. In Snippet 3,
       <tt>
        is_tree( '2' )
       </tt>
       would replace "
       <tt>
        is_page( 'about' ) || '2' == $post-&gt;post_parent
       </tt>
       " inside the first
       <tt>
        if
       </tt>
       tag.
      </p>
      <p>
       Note that if you have more than one level of pages the parent page is the one directly above and not the one at the very top of the hierarchy.
      </p>
      <h4>
       <span class="mw-headline" id="Is_a_Page_Template">
        Is a Page Template
       </span>
      </h4>
      <p>
       Allows you to determine whether or not you are in a page template or if a specific page template is being used.
      </p>
      <dl>
       <dt>
        <tt>
         <a class="external text" href="http://codex.wordpress.org/Function_Reference/is_page_template">
          is_page_template()
         </a>
        </tt>
       </dt>
       <dd>
        Is a
        <a href="/Page_Templates" title="Page Templates">
         Page Template
        </a>
        being used?
       </dd>
       <dt>
        <tt>
         is_page_template( 'about.php' )
        </tt>
       </dt>
       <dd>
        Is
        <a href="/Page_Templates" title="Page Templates">
         Page Template
        </a>
        'about' being used? Note that unlike with other conditionals, if you want to specify a particular Page Template, you need to use the filename, such as about.php or my_page_template.php. Note: if the file is in a subdirectory you must include this as well. Meaning that this should be the filepath in relation to your theme as well as the filename, for example 'page-templates/about.php'.
       </dd>
      </dl>
      <h2>
       <span class="mw-headline" id="Notes">
        Notes
       </span>
      </h2>
      <h3>
       <span class="mw-headline" id="Passing_Empty_Value_Returns_TRUE">
        Passing Empty Value Returns TRUE
       </span>
      </h3>
      <p>
       Be very careful if there's a possibility of passing an empty value as a parameter to check for a specific page, since the following lines will return
       <tt>
        true
       </tt>
       :
      </p>
      <pre>
       is_page( '' )
is_page( 0 )
is_page( '0' )
is_page( null )
is_page( false )
is_page( array() )
      </pre>
      <ul>
       <li>
        See also:
        <a href="/Function_Reference/is_singular" title="Function Reference/is singular">
         is_singular()
        </a>
       </li>
      </ul>
      <h3>
       <span class="mw-headline" id="Cannot_Be_Used_Inside_The_Loop">
        Cannot Be Used Inside The Loop
       </span>
      </h3>
      <p>
       Due to certain global variables being overwritten during The Loop
       <code>
        is_page()
       </code>
       will not work. In order to use it after The Loop you must call
       <a href="/Function_Reference/wp_reset_query" title="Function Reference/wp reset query">
        wp_reset_query()
       </a>
       after The Loop.
      </p>
