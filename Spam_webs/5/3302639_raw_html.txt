  <h1>
   Emacs Shortcuts
  </h1>
  I prefer using the keyboard to the mouse, so I use several
key-combinations rather than the mouse/menu system that's available in
emacs/xemacs.
  <p>
   In the table and descriptions below the modifier
   <b>
    M
   </b>
   refers to the
escape key. The M is used for Meta, a modifier on the computer keyboards
used when emacs was first developed. The modifier
   <b>
    C
   </b>
   refers to the
control key. For example,
   <tt>
    M-x replace-string
   </tt>
   means "press
escape, then x" in the command mini-buffer the prompt
   <tt>
    M-x
   </tt>
   will
appear, type
   <em>
    replace-string
   </em>
   in the mini-buffer.  As another
example,
   <tt>
    C-x c
   </tt>
   means "press the control key and the x key at
the same time, then press the c key".
  </p>
  <p>
   Note that M, or escape, is pressed before another key whereas C, or
control, is pressed at the same time as a key.  You can press
the escape key directly, but
   <tt>
    C-[
   </tt>
   is an abbreviation for escape
(it doesn't require stretching your pinky a long way to get to
the escape key).
  </p>
  <p>
   At any point you can find all the bindings in effect by typing
  </p>
  <pre>
   C-h b
  </pre>
  (that's help + b as in bindings)
  <p>
  </p>
  <h1>
   Commands I use all the time
  </h1>
  Some of these commands use the emacs concept of
  <em>
   point
  </em>
  and
  <em>
   mark
  </em>
  .  Together these specify
a region. The point is where the cursor is.  You can set the mark
explicitly by typing
  <tt>
   M-space
  </tt>
  (that's escape followed by the
space-bar).  For example, to delete a region
  <ul>
   <li>
    Set the mark
   </li>
   <li>
    Move the cursor/point -- in xemacs a region is grayed out when you 
	do this
   </li>
   <li>
    Delete the current region (betwen point and mark) with
    <tt>
     C-w
    </tt>
   </li>
  </ul>
  <p>
  </p>

  <h2>
   Saving and Searching
  </h2>
  <table>
   <th width="30%">
    key combination
   </th>
   <th>
    functionality
   </th>
   <tr>
    <td>
     <tt>
      C-x s
     </tt>
    </td>
    <td>
     visit all buffers, prompt to save if changed
    </td>
   </tr>
   <tr>
    <td>
     <tt>
      C-x c
     </tt>
    </td>
    <td>
     compile current buffer. (this is potential trouble,
note that
     <tt>
      C-x C-c
     </tt>
     quits emacs).
    </td>
   </tr>
   <tr>
    <td>
     <tt>
      C-x C-c
     </tt>
    </td>
    <td>
     quit emacs (prompt to save modified buffers)
    </td>
   </tr>
   <tr>
    <td>
     <tt>
      C-s
     </tt>
    </td>
    <td>
     search for a string forwards (enter string
in mini-buffer)
    </td>
   </tr>
   <tr>
    <td>
     <tt>
      C-s C-s
     </tt>
    </td>
    <td>
     repeats the last search (no prompt)
    </td>
   </tr>
   <tr>
    <td>
     <tt>
      C-r
     </tt>
    </td>
    <td>
     search backwards in buffer (enter string
in mini-buffer)
    </td>
   </tr>
   <tr>
   </tr>
  </table>
