        <header class="entry-header">
         <h1 class="blogh1">
          What RAID Level is Best?
         </h1>
         <div class="entry-meta">
          <span class="sep">
           Posted on
          </span>
          <a href="http://www.ait.com/raid-level-best/" rel="bookmark" title="12:52 pm">
           <time class="entry-date" datetime="2015-02-18T12:52:19+00:00" pubdate="">
            February 18, 2015
           </time>
          </a>
          <span class="byline">
           <span class="sep">
            by
           </span>
           <span class="author vcard">
            <a class="url fn n" href="http://www.ait.com/author/mroberts/" rel="author" title="View all posts by Michael Roberts">
             Michael Roberts
            </a>
           </span>
          </span>
         </div>
         <script type="text/javascript">
          var addthis_config={pubid:"c7f1a591801626ae207"}
         </script>
         <a class="addthis_button" href="http://www.addthis.com/bookmark.php?v=250">
          <img alt="Share" border="0" height="16" src="http://s7.addthis.com/static/btn/v2/lg-share-en.gif" width="125"/>
         </a>
         <script src="http://s7.addthis.com/js/250/addthis_widget.js" type="text/javascript">
         </script>
        </header>
        <br/>
        <div class="entry-content">
         <p>
          I am asked this question from time to time, and the answer is always different.  You see, when someone asks this question, it tells me that they dont thoroughly understand RAID.  Thats because there is no best RAID level.  There is a good option for each type of scenario you may find.  RAID is a technology that is used to increase the performance and/or reliability of data storage. Thus, increasing the performance for a web server is not the same as increasing the performance for a database server, which is not the same as increasing the performance of a file server.   Lets talk about the general background for RAID before progressing.
          <span id="more-10681">
          </span>
         </p>
         <p>
          RAID stands for
          <strong>
           Redundant Array of Inexpensive (or Independent) Disks
          </strong>
          .  A RAID system consists of two or more drives working in parallel. These disks can be hard discs but
          <a href="http://www.ait.com/sas-hdd-vs-ssd-dedicated-servers-infographic/">
           we are seeing increasing use of SSD (solid state drives) for server environments
          </a>
          .  There are different RAID levels, each optimized for a specific situation.  These are not standardized by an industry group. This explains why there is a lot of confusion surrounding RAID and
          <strong>
           which RAID level is best
          </strong>
          because honestly, most folks do not know.
         </p>
         <h3 class="blue-text">
          Types of RAID
         </h3>
         <ul>
          <li>
           RAID 0  Striping
          </li>
          <li>
           RAID 1  Mirroring
          </li>
          <li>
           RAID 5  Striping with parity
          </li>
          <li>
           RAID 6  Striping with double parity
          </li>
          <li>
           RAID 10  Combining mirroring and striping
          </li>
          <li>
           Hardware RAID  Providing RAID via a hardware RAID controller
          </li>
          <li>
           Software RAID  Providing increased disk performance and/or fault tolerance via software, not a hardware controller
          </li>
         </ul>
         <h3 class="blue-text">
          RAID 0  Striping
         </h3>
         <p>
          In a RAID 0 system, data is split up in blocks that get written across all the drives in the array. By using multiple disks (at least 2) at the same time, this offers superior I/O performance. This performance can be enhanced further by using multiple controllers, ideally one controller per disk.
         </p>
         <div class="text-center">
          <img alt="Graphical representation of RAID 0" src="/images/2015/blog-raid0.png"/>
         </div>
         <h4>
          Advantages of RAID 0
         </h4>
         <ul>
          <li>
           RAID 0 offers the best performance, both in read and write operations. There is no overhead caused by parity controls.
          </li>
          <li>
           All storage capacity is used, there is no overhead.
          </li>
          <li>
           The technology is easy to implement.
          </li>
         </ul>
         <h4>
          Disadvantages of RAID 0
         </h4>
         <p>
          RAID 0 is not fault-tolerant.  If one drive fails, all data in the RAID 0 array is lost. It should not be used for mission-critical systems.
         </p>
         <h4>
          Ideal use of RAID 0
         </h4>
         <p>
          RAID 0 is ideal for non-critical storage of data that have to be read/written at a high speed, such as a graphic designer workstation or video editing workstation.  If you want to use RAID 0 purely to combine the storage capacity of two drives in a single volume, consider mounting one drive in the folder path of the other drive. This is supported in almost all distributions of Linux, Mac OS X, and Windows and has the advantage that a single drive failure has no impact on the data of the second disk or SSD drive.
         </p>
         <h3 class="blue-text">
          RAID 1  Mirroring
         </h3>
         <p>
          RAID 1 has several advantages over RAID 0.  In RAID 1, data is stored in two locations, or sets of data drives.  The first location is the live dataset.  The second is a mirror drive (or set of drives) . If a drive fails, the controller uses either the data drive or the mirror drive for data recovery and continues operation.  You need at least 2 drives for a RAID 1 array.
         </p>
         <div class="text-center">
          <img alt="Graphical representation of RAID 1" src="/images/2015/blog-raid1.png"/>
         </div>
         <h4>
          Advantages of RAID 1
         </h4>
         <ul>
          <li>
           RAID 1 offers excellent read speed and a write-speed that is comparable to that of a single drive.
          </li>
          <li>
           In case a drive fails, data doesnt have to be repaired.  Depending on the configuration, a drive replacement can rectify the problem and youre back online with minimal involvement.
          </li>
          <li>
           RAID 1 is a very simple technology.
          </li>
         </ul>
         <h4>
          Disadvantages of RAID 1
         </h4>
         <ul>
          <li>
           The main disadvantage is that the effective storage capacity is only half of the total drive capacity because all data gets written twice.
          </li>
          <li>
           Software RAID 1 solutions do not always allow a hot swap of a failed drive (meaning it cannot be replaced while the server keeps running). Ideally a hardware controller needs to be used in order for a replacement drive to be put online while the server is online.
          </li>
         </ul>
         <h4>
          Ideal use of RAID 1
         </h4>
         <p>
          RAID 1 is ideal for mission critical storage, for instance for web servers, or servers with small chassis where only 2 drives can be used.  This is especially valuable if data on these systems doesnt change often, so restoration of the data from a backup is not very complex.
         </p>
         <h3 class="blue-text">
          RAID 5  Parity with striping
         </h3>
         <p>
          RAID 5 is the most common secure RAID level. It requires at least 3 drives but can work with up to 16. Data blocks are striped across the drives and on one drive a parity check sum of all the block data is written. The parity data is not written to a fixed drive, they are spread across all drives, as the diagram shows. Using the parity data, the server can recalculate the data of one of the other data blocks, should that data no longer be available. That means a RAID 5 array can withstand a single drive failure without losing data.  Although RAID 5 can be achieved in software, a hardware controller is recommended.  Often extra cache memory is used on these controllers to improve the write performance.
         </p>
         <div class="text-center">
          <img alt="Graphical representation of RAID 5" src="/images/2015/blog-raid5.png"/>
         </div>
         <h4>
          Advantages of RAID 5
         </h4>
         <ul>
          <li>
           Read data transactions are very fast while write data transactions are somewhat slower (due to the parity that has to be calculated).
          </li>
          <li>
           If a drive fails, you still have access to all data, even while the failed drive is being replaced and the storage controller rebuilds the data on the new drive.
          </li>
         </ul>
         <h4>
          Disadvantages of RAID 5
         </h4>
         <ul>
          <li>
           Drive failures have an effect on throughput, although this is still acceptable.
          </li>
          <li>
           This is complex technology. If one of the disks in an array using 4TB disks fails and is replaced, restoring the data (the rebuild time) may take a day or longer, depending on the load on the array and the speed of the controller. If another disk goes bad during that time, data is lost forever.
          </li>
         </ul>
         <h4>
          Ideal use for RAID 5
         </h4>
         <p>
          RAID 5 is a good all-round system that combines efficient storage with excellent security and decent performance. It is ideal for file and application servers that have a limited number of data drives.
         </p>
         <h3 class="blue-text">
          RAID 6  Striping with double parity
         </h3>
         <p>
          RAID 6 is like RAID 5 but it writes the parity data to two drives. That means it requires
          <strong>
           at least 4 drives
          </strong>
          and can withstand 2 drives dying simultaneously. The chances that two drives break down at exactly the same moment are of course very small. However, if a drive in a RAID 5 systems dies and is replaced by a new drive, it takes hours to rebuild the swapped drive. If another drive dies during that time, you still lose all of your data. With RAID 6, the RAID array will even survive that second failure.
         </p>
         <div class="text-center">
          <img alt="Graphical representation of RAID 6" src="/images/2015/blog-raid6.png"/>
         </div>
         <h4>
          Advantages of RAID 6
         </h4>
         <ul>
          <li>
           Like with RAID 5, read data transactions are very fast.
          </li>
          <li>
           If two drives fail, you still have access to all data, and can replace the drives while the server is still online. So RAID 6 is more redundant than RAID 5.
          </li>
         </ul>
         <h4>
          Disadvantages of RAID 6
         </h4>
         <ul>
          <li>
           Generally more expensive because you are paying for at least 4 hard drives as opposed to 2 or 3 in other RAID configurations.
          </li>
          <li>
           Write data transactions are slowed down due to the parity that has to be calculated.
          </li>
          <li>
           Recovery time is longer than in most configurations.  Rebuilding an array in which one drive failed can take a long time to completely repair.
          </li>
         </ul>
         <h4>
          Ideal use of RAID 6
         </h4>
         <p>
          RAID 6 is a good all-round system that combines efficient storage with excellent security and decent performance.  It is preferable over RAID 5 in file and application servers that use many large drives for data storage.
         </p>
         <h3 class="blue-text">
          RAID 10  Combining RAID 1 &amp; RAID 0
         </h3>
         <p>
          It is possible to combine the advantages (and disadvantages) of RAID 0 and RAID 1 in one single system.  This is a nested or hybrid RAID configuration. It provides security by mirroring all data on secondary drives while using striping across each set of drives to speed up data transfers.
         </p>
         <div class="text-center">
          <img alt="Graphical representation of RAID 10" src="/images/2015/blog-raid10.png"/>
         </div>
         <h4>
          Advantages of RAID 10
         </h4>
         <ul>
          <li>
           If something goes wrong with one of the disks in a RAID 10 configuration, the rebuild time is very fast since all that is needed is copying all the data from the surviving mirror to a new drive. This can take as little as 30 minutes for drives of  1 TB.
          </li>
         </ul>
         <h4>
          Disadvantages of RAID 10
         </h4>
         <ul>
          <li>
           Half of the storage capacity goes to mirroring, so compared to large RAID 5  or RAID 6 arrays, this is an expensive way to have redundancy.
          </li>
         </ul>
         <h4>
          Ideal use of RAID 10
         </h4>
         <p>
          In a larger MS SQL server environment, RAID 10 is great for log data.  As Microsoft has stated over and over again, RAID 1+0 will provide better throughput for write-intensive applications. The amount of performance gained will vary based on the HW vendors RAID implementations. Most common alternative to RAID 1+0 is RAID 5. Generally, RAID 1+0 provides better write performance than any other RAID level providing data protection, including RAID 5.
         </p>
         <h3 class="blue-text">
          Other RAID Resources
         </h3>
         <ul>
          <li>

            RAID Calculator
           </a>
            Find out how much usable disk space youll have after a RAID configuration.
          </li>
         </ul>
