          <h3 class="post-title">
           <a href="http://orkutchamp.blogspot.com/2008/03/w32scrapkut-orkut-worm-spreading-to.html">
            W32.Scrapkut Orkut Worm spreading to millions of users
           </a>
          </h3>
          <div class="post-header-line-1">
          </div>
          <div class="post-body">
           <p>
            <span class="fullpost">
             Friends how many of you have fallen victim to a worm, which apparently looks like this in your scrapbook, but let me warn you its a deadly worm, recently renamed to W32.Scrapkut, uses active code injection as a vehicle to propagate to the Orkut friends of its unfortunate victim.
             <br/>
             <br/>
             Wondering how this thing happens, heres the whole story about it:
            </span>
            <br/>
            &gt; a malicious scrap is posted to the victims scrapbook, containing a link to what appears to be a YouTube video
            <a href="http://bp1.blogger.com/_gTKISBHBW4I/R82b9vng5MI/AAAAAAAAANE/fjJPZI5mfh8/s1600-h/image1.jpg" onblur="try {parent.deselectBloggerImageGracefully();} catch(e) {}">
             <img alt="" border="0" id="BLOGGER_PHOTO_ID_5173963031975093442" src="http://bp1.blogger.com/_gTKISBHBW4I/R82b9vng5MI/AAAAAAAAANE/fjJPZI5mfh8/s320/image1.jpg" style="margin: 0px auto 10px; display: block; text-align: center; cursor: pointer; width: 241px; height: 234px;"/>
            </a>
            <br/>
            &gt; When a victim clicks on the link, they are redirected to an external site which prompts them to download the file flashx_player_9.8.0.exe.
            <a href="http://bp3.blogger.com/_gTKISBHBW4I/R82b-Png5NI/AAAAAAAAANM/Q6tt_M9V61A/s1600-h/image2_sml.jpg" onblur="try {parent.deselectBloggerImageGracefully();} catch(e) {}">
             <img alt="" border="0" id="BLOGGER_PHOTO_ID_5173963040565028050" src="http://bp3.blogger.com/_gTKISBHBW4I/R82b-Png5NI/AAAAAAAAANM/Q6tt_M9V61A/s320/image2_sml.jpg" style="margin: 0px auto 10px; display: block; text-align: center; cursor: pointer;"/>
            </a>
            <br/>
            &gt; When executed, flashx_player_9.8.0.exe retrieves the files windosremote.exe, logservicess.exe and win32chekupdate.exe from http://[REMOVED].ifastnet.com. These files download additional files that perform a variety of malicious actions, but logservicess.exe is the main executable for further propagation. Logservicess.exe first copies itself as maindwxp.exe to four different locations on the system to ensure it is executed on startup.
            <a href="http://bp1.blogger.com/_gTKISBHBW4I/R82b-vng5OI/AAAAAAAAANU/-Gn7e8PMyCk/s1600-h/image3_sml.jpg" onblur="try {parent.deselectBloggerImageGracefully();} catch(e) {}">
             <img alt="" border="0" id="BLOGGER_PHOTO_ID_5173963049154962658" src="http://bp1.blogger.com/_gTKISBHBW4I/R82b-vng5OI/AAAAAAAAANU/-Gn7e8PMyCk/s320/image3_sml.jpg" style="margin: 0px auto 10px; display: block; text-align: center; cursor: pointer;"/>
            </a>
            <br/>
            &gt; Maindwxp.exe then checks in with the command and control server via a GET request with specific parameter values. Interestingly, the page returned simply contains the word Rastreados followed by a number. In Portuguese, rastreados means crawled - at last check the number was 13559.
            <br/>
            &gt; Maindwxp.exe then executes and begins checking for an active browser window, waiting for the victim to visit Orkut. Once the victim is in an authenticated Orkut session, maindwxp.exe injects Javascript code into the active Orkut web session.
            <a href="http://bp2.blogger.com/_gTKISBHBW4I/R82b-_ng5PI/AAAAAAAAANc/W9cjHRObQ5w/s1600-h/image4_sml.jpg" onblur="try {parent.deselectBloggerImageGracefully();} catch(e) {}">
             <img alt="" border="0" id="BLOGGER_PHOTO_ID_5173963053449929970" src="http://bp2.blogger.com/_gTKISBHBW4I/R82b-_ng5PI/AAAAAAAAANc/W9cjHRObQ5w/s320/image4_sml.jpg" style="margin: 0px auto 10px; display: block; text-align: center; cursor: pointer;"/>
            </a>
            <br/>
            &gt; This Javascript code which is actually based on a popular Greasemonkey script is then executed within the context of the Orkut domain and the users authenticated session, resulting in the malicious scrapbook entry being sent to all the victims friends, and the cycle begins again.
            <br/>
            <br/>
            This way the malicious worm is spreading virally on the orkut, So how you can be saved from this malicious worm now, well in past i have posted about
            <span style="font-weight: bold;">
             "
            </span>
            <a href="http://orkutchamp.blogspot.com/2008/01/browse-orkut-without-javascript-support.html" style="font-weight: bold;">
             how to surf orkut without javascript
            </a>
            <span style="font-weight: bold;">
             "
            </span>
            which would help in prevention from such viruses and always remember not to use any javascript that are posted in your scrap even by your trusted friends...
