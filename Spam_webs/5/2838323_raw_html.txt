        <article class="post-12064 post type-post status-publish format-standard hentry category-ios" id="post-12064">
         <hgroup>
          <aside class="avatar">
           <img alt="" class="avatar avatar-200 wp-user-avatar wp-user-avatar-200 photo avatar-default" height="200" src="https://secure.gravatar.com/avatar/eb7046aac742abfd3d160267b3353808?s=200&amp;d=https%3A%2F%2Fsecure.gravatar.com%2Favatar%2Fad516503a11cd5ca435acc9bb6523536%3Fs%3D200&amp;r=G" width="200"/>
          </aside>
          <span class="date">
           June 26, 2014
          </span>
          <h1>
           <a href="https://blog.xamarin.com/3d-in-ios-8-with-scene-kit/" rel="bookmark" title="Permalink to Lights, Camera, Action  3D in iOS 8 with Scene Kit">
            Lights, Camera, Action  3D in iOS 8 with Scene Kit
           </a>
          </h1>
          <span class="author">
           By
           <a href="https://blog.xamarin.com/" rel="author" title="Posts by Mike Bluestein">
            Mike Bluestein
           </a>
          </span>
         </hgroup>
         <div class="entry-content rich-content">
          <p>
           <strong>
            Scene Kit
           </strong>
           is a powerful 3D graphics API that makes working with 3D a snap. It was first introduced in OS X 10.8, and has now come to iOS 8. With Scene Kit, creating immersive 3D visualizations and casual 3D games no longer requires expertise in
           <a href="http://www.opengl.org/" target="_blank" title="opengl.org">
            OpenGL
           </a>
           . Building on common scene graph concepts, Scene Kit abstracts away the complexities of OpenGL and OpenGL ES, making it very easy to add 3D content to an application. However, if you are an OpenGL expert, Scene Kit has great support for tying in directly with OpenGL as well. It also includes numerous features that complement 3D graphics, such as physics, and integrates very well with several other Apple frameworks, such as Core Animation, Core Image and Sprite Kit.
          </p>
          <p>
           <img alt="3D earth" class="aligncenter size-full wp-image-12071" height="542" src="/wp-content/uploads/2014/06/earth.png" width="258"/>
          </p>
          <p>
           Scene Kit is extremely easy to work with. It is a declarative API that takes care of rendering. You simply set up a scene, add properties to it, and the magic of Scene Kit brings it all to life.
          </p>
          <p>
           To work with Scene Kit you create a scene graph using the
           <strong>
            SCNScene
           </strong>
           class. A scene contains a hierarchy of nodes, represented by instances of
           <strong>
            SCNNode
           </strong>
           , defining locations in 3D space. Each node has properties such as geometry, lighting and materials that affect its appearance.
          </p>
          <p>
           <img alt="node tree" class="aligncenter size-full wp-image-12073" height="324" src="/wp-content/uploads/2014/06/nodetree.png" width="432"/>
          </p>
          <p>
           To make a scene appear on screen, you add it to an
           <strong>
            SCNView
           </strong>
           by assigning it to the views
           <strong>
            Scene
           </strong>
           property. Additionally, if you make any changes to the scene,
           <strong>
            SCNView
           </strong>
           will update itself to display the changes.
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           scene = SCNScene.Create ();
sceneView = new SCNView (View.Frame);
sceneView.Scene = scene;
          </pre>
          <p>
           Scenes can be populated from files exported via a 3d modeling tool, or programmatically from geometric primitives. For example, this is how to create a sphere and add it to the scene:
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           sphere = SCNSphere.Create (10.0f);
sphereNode = SCNNode.FromGeometry (sphere);
sphereNode.Position = new SCNVector3 (0, 0, 0);
scene.RootNode.AddChildNode (sphereNode);
          </pre>
          <h2>
           Let There Be Light
          </h2>
          <p>
           At this point the sphere wont display anything because there is no light in the scene. Lights in Scene Kit are created by attaching
           <strong>
            SCNLight
           </strong>
           instances to nodes. There are several types of lights ranging from various forms of directional lighting to ambient lighting. For example the following code creates an omni directional light on the side of the sphere.
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           // omni-directional light
var light = SCNLight.Create ();
var lightNode = SCNNode.Create ();
light.LightType = SCNLightType.Omni;
light.Color = UIColor.Blue;
lightNode.Light = light;
lightNode.Position = new SCNVector3 (-40, 40, 60);
scene.RootNode.AddChildNode (lightNode);
          </pre>
          <p>
           Omni directional lighting produces a diffuse reflection resulting in an even lighting, sort of like shining a flashlight. Creating ambient light is similar, although it has no direction as it shines equally in all directions. Think of it like mood lighting
           <img alt=":)" class="wp-smiley" src="https://blog.xamarin.com/wp-includes/images/smilies/icon_smile.gif"/>
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           // ambient light
ambientLight = SCNLight.Create ();
ambientLightNode = SCNNode.Create ();
ambientLight.LightType = SCNLightType.Ambient;
ambientLight.Color = UIColor.Purple;
ambientLightNode.Light = ambientLight;
scene.RootNode.AddChildNode (ambientLightNode);
          </pre>
          <p>
           With the lights in place, the sphere is now visible in the scene.
          </p>
          <p>
           <img alt="sphere" class="aligncenter size-full wp-image-12075" height="542" src="/wp-content/uploads/2014/06/sphere1.png" width="258"/>
          </p>
          <h2>
           Adding a Camera
          </h2>
          <p>
           Adding a camera (
           <strong>
            SCNCamera
           </strong>
           ) to the scene changes the point of view. The pattern to add the camera is similar. Create the camera, attach it to a node and add the node to the scene.
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           // camera
camera = new SCNCamera {
	XFov = 80,
	YFov = 80
};
cameraNode = new SCNNode {
	Camera = camera,
	Position = new SCNVector3 (0, 0, 40)
};
scene.RootNode.AddChildNode (cameraNode);
          </pre>
          <p>
           As you can see from the code above, Scene Kit objects can be created using constructors or from the
           <strong>
            Create
           </strong>
           factory method. The former allows using C# initializer syntax, but which one to use is largely a matter of preference.
          </p>
          <p>
           With the camera in place, the entire sphere is visible to the user.
          </p>
          <p>
           <img alt="sphere" class="aligncenter size-full wp-image-12077" height="542" src="/wp-content/uploads/2014/06/sphere2.png" width="258"/>
          </p>
          <p>
           You can add additional lights to the scene as well. Here is what it looks like with a few more omni-directional lights:
          </p>
          <p>
           <img alt="sphere" class="aligncenter size-full wp-image-12079" height="542" src="/wp-content/uploads/2014/06/sphere3.png" width="258"/>
          </p>
          <p>
           Additionally, by setting
           <code>
            sceneView.AllowsCameraControl = true
           </code>
           , the user can change the point of view with a touch gesture.
          </p>
          <h2>
           Materials
          </h2>
          <p>
           Materials are created with the
           <strong>
            SCNMaterial
           </strong>
           class. For example to add an image onto the spheres surface, set the image to the materials diffuse contents.
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           material = SCNMaterial.Create ();
material.Diffuse.Contents = UIImage.FromFile ("monkey.png");
sphere.Materials = new SCNMaterial[] { material };
          </pre>
          <p>
           This layers the image onto the node as shown below.
          </p>
          <p>
           <img alt="sphere" class="aligncenter size-full wp-image-12081" height="542" src="/wp-content/uploads/2014/06/sphere4.png" width="258"/>
          </p>
          <p>
           A material can be set to respond to other types of lighting too. For example, the object can be made shiny and have a its specular contents set to display specular reflection, resulting in a bright spot on the surface.
          </p>
          <p>
           <img alt="sphere" class="aligncenter size-full wp-image-12083" height="542" src="/wp-content/uploads/2014/06/sphere5.png" width="258"/>
          </p>
          <p>
           Materials are very flexible, allowing you to achieve a lot with very little code. For example, instead of setting the image to the diffuse contents, set it to the reflective contents instead.
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           material.Reflective.Contents = UIImage.FromFile ("monkey.png");
          </pre>
          <p>
           Now the monkey sits visually within the sphere, independent of the point of view, creating a monkey in a psychedelic 1960s sphere so to speak.
          </p>
          <p>
           <img alt="spinning monkey" class="aligncenter size-full wp-image-12091" height="279" src="/wp-content/uploads/2014/06/psychmonkey.gif" width="155"/>
          </p>
          <h2>
           Animation
          </h2>
          <p>
           Scene Kit is designed to work well with Core Animation. You can create both implicit or explicit animations. When creating an implicit animation, Scene Kit provides its own transition class,
           <strong>
            SCNTransaction
           </strong>
           .
          </p>
          <p>
           Heres a example that rotates the sphere:
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           SCNTransaction.Begin ();
SCNTransaction.AnimationDuration = 2.0;
sphereNode.Rotation = new SCNVector4 (0, 1, 0, (float)Math.PI * 4);
SCNTransaction.Commit ();
          </pre>
          <p>
           <img alt="rotation" class="aligncenter size-full wp-image-12087" height="542" src="/wp-content/uploads/2014/06/rotation.png" width="536"/>
          </p>
          <p>
           You can animate much more than rotation though. Many properties of Scene Kit are animatable. For example, the following code animates the materials
           <strong>
            Shininess
           </strong>
           to increase the specular reflection.
          </p>
          <pre class="brush: csharp; title: ; notranslate" title="">
           SCNTransaction.Begin ();
SCNTransaction.AnimationDuration = 2.0;
material.Shininess = 0.1f;
SCNTransaction.Commit ();
          </pre>
          <p>
           <img alt="shininess" class="aligncenter size-full wp-image-12088" height="542" src="/wp-content/uploads/2014/06/shininess.png" width="536"/>
          </p>
          <p>
           As you can see, Scene Kit is incredibly easy to use. This post just scratches the surface though. There are a wealth of additional features including constraints, physics, declarative actions, 3D text, depth of field support, Sprite Kit integration and Core Image integration to name just a few. Its exciting to see this API come to iOS 8.
          </p>
          <p>
           The source code from this post is available
           <a href="https://gist.github.com/mikebluestein/8d8486c200dd6d0f8d06" target="_blank" title="Hello Scene Kit">
            here
           </a>
           .
          </p>
