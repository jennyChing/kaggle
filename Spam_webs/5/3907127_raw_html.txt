           The Mosslands School
           <br/>
           Unit 28 - Website Architecture
           <br/>
           Outline the web architecture and components which enable internet and web functionality.
           <br/>
           P1
           <br/>
           There are 4 main types of servers Normal servers, Web servers, Email servers and Proxy servers.
           <br/>
           Servers
           <br/>
           Normal servers
           <br/>
           A server Is a computer specially built with higher RAM
           <br/>
           and CPU power so that in can store a website and handle
           <br/>
           mass amounts of people connected to it as possible.
           <br/>
           This is a server which other users can access through the use of the internet.
           <br/>
           <br/>
           A web server holds a live copy of the web page which can be updated after editing the copy you hold offline.
           <br/>
           Web Servers
           <br/>
           These servers hold databases of all the users on the server and give them a virtual post box where other users of the server can send those messages. If the server is switched off then mail cannot be sent or received.
           <br/>
           Email servers
           <br/>
           Proxy Servers
           <br/>
           A proxy server is a stand in server which can deal with
           <br/>
           requests instead of using the main server, if a web page has
           <br/>
           been loaded on a users computer before the proxy server will
           <br/>
           instantly resend them to the page instead of the actual server
           <br/>
           re sending them the information to download. A proxy Filter
           <br/>
           can also be put on a network; these are most commonly used
           <br/>
           in schools and work places as they stop the users from accessing
           <br/>
           certain websites.
           <br/>
           When you type in the domain name of a website your computer sends a HTTP request to the server and receives the data for the website from the domain in the form of HTML ( hyper text markup language) and CSS (cascading style sheet) if a cache is present and you have visited the website before then the data will already be stored and would not need to be re downloaded. The internet browser then translates the data to be displayed on the GUI of the web browser on the screen and sends requests to all embedded objects such as images, videos and SWF files.
           <br/>
           Accessing a Server
           <br/>
           An IP address a list of numbers unique for every computer. An IP address is linked to a web page which is loaded when domain name is typed into a URL bar.
           <br/>
           What is an IP address?
           <br/>
           This is the web address which links a user to the IP address of the server, usually you have to pay to keep your domain up by outsourcing as it is easier to host your domain and more cost and time efficient.
           <br/>
           What is  Domain
           <br/>
           The world wide web is made up of websites hosted by servers which are all connected together through the internet.
           <br/>
           The world wide web
           <br/>
           Websites are groups of web pages which are all hosted on a server or a series of servers which are connected together. To allow a website to be seen across the internet it needs to be uploaded onto a web server, this process is called FTP (file transfer protocol), this can be achieved directly through a browser or you can download programs to uploaded it for you, when uploading a website you also have to uploaded all images and files embedded into the website, if the files are not embedded then they will not appear as they should but as a broken link.
           <br/>
           Websites
           <br/>
           Explain the security risks and protection mechanisms involved in website performance.
           <br/>
           P3
           <br/>
           Virus
           <br/>
           A virus is a code which, after entering a computer system, can perform various tasks; such as deleting all the files off a computer, or copying data such as passwords and credit card details - sending information back to the source of the virus to be hacked. They are called viruses as, just like a real virus, they can spread throughout the host console and can also spread from computer to computer.
           <br/>
           Hacking
           <br/>
           This is where hackers gain access to your personal files or accounts without consent. Often this is done by phishing. Phishing is where a file is sent in an email (pretending to be an official organisation - such as a bank) informing the victim that their account is about to be deactivated and encouraging them to enter some personal information, such as an account number or pin. This could lead victims to a fake website which appears identical to the organisation's but made to record personal information that can be used to hack accounts.
           <br/>
           ID Theft
           <br/>
           This is where someone assumes your identity or uses an account either online or banking without your permission, often done by hacking passwords or through phishing websites.
           <br/>
           Personal details are most commonly used to purchase items online or to transfer money to a hacker's account. Everyone should be wary of  suspicious emails and websites, and ensure they use a secure password. In attempts to prevent this, most responsible organsations will not ask you to send personal details such as PIN codes or account details in emails.
           <br/>
           Firewall
           <br/>
           A firewall is a program that checks every data packet for viruses and other harmful forms of data. Websites sometimes purchase cheap firewalls, these take a long time to check data packets and slow down websites significantly - especially when several users are trying to access the website at once. More expensive firewall programs can send, check and receive data in a matter of milliseconds, without a firewall program website can be sped up significantly but can suffer the risk of hacking or getting a virus.
           <br/>
           SSL
           <br/>
           SSL (secure socket layer) allows for secure communication between computers. It is used on most websites that have a user system. To use SSL, the company needs to have an SSL certificate - a small text file containing details of the website's information, such as server and domain name.
           <br/>
           A phishing website will not have an SSL certificate, this  makes them easier to identify as a pop up will appear after attempting to log in to let you know that the certificate is not signed and giving you an option to not continue.
           <br/>

           Explain the role of web architecture in website communications

           M1
           <br/>
           Explain the security risks and protection mechanisms involved in website performance.
           <br/>
           M2
           <br/>
           Web 2.0 Technology
           <br/>
           Web technology is the interaction the users have with a website. Web 1.0 allowed users to view a website but only the host of the website could make changes where as web 2.0 allows for full interactivity from a user to the website whether it is social networking or uploading videos or to make an account on a blog website where they can post articles for others to read.
           <br/>
           Social networking
           <br/>
           This enables accounts to be made on a website such as Facebook or Twitter and allows them to add other users as 'friends' so they can have instant access to their pages on the website. this allows to upload photos, share videos and send messages.
           <br/>
           YouTube
           <br/>
           YouTube is a website where people can share videos with the rest of the world. This can be used for many purposes such as song artists uploading music videos, television programs so they can be watched again or if they were missed they can be found online or simply for people to 'vlog' which is a video diary where people just talk about their life.
           <br/>
           'Wikis'
           <br/>
           These usually tend to be a source of information  based on one specific area although Wikipedia (WWW.Wikipedia.com) has pages for nearly everything. A wiki is editable by all users which have signed up to it but all posts are approved or taken down b admins.
           <br/>
           Blogs
           <br/>
           A blog is created so people can express there ideas
           <br/>
           on something or to relay information to others. You can host you own blog site or you can sign up to a current site which allows blogs to be created through them.
           <br/>
           Web 3.0
           <br/>
           Web 3.0 covers personality and is already implemented on the internet, from changing the theme of your web browser to eBay recommended items based on your previous views.
           <br/>
           web 3.0 will allow other devices to communicate over the internet such as televisions, fridges and printers. search engines will automatically display suggested searches and websites will allow you to choose how you view them and edit their GUI for your own personal use.

           <br/>
           Explain the techniques that can be used on web pages to aid user access to information

           D2
           <br/>
           By Johnathan Matthews
           <br/>

           <br/>
           Outline the web architecture and components which enable internet and web functionality.
           <br/>
           P3
           <br/>
           Explain the security risks and protection mechanisms involved in website performance.
           <br/>
           P4
           <br/>
           <strong>
            TCP/IP Protocols
           </strong>
           <br/>
           <strong>
            TCP/IP Protocols are a collection of protocols which are used to communicate that are based on TCP  and protocols.
           </strong>
           <br/>
           TCP Protocol
           <br/>
           The TPC (transmission control) protocol is similar to the IP protocol in the respect that it transfers data across network. The data is broken down into data packets which get sent from one computer to another and, the receiving computer then puts the packets of data back together to create the original file.
           <br/>
           IP Protocol
           <br/>
           The IP (internet) protocol is used to transfer data between two different computers over the internet.
           <br/>
           This uses the internet or LAN to send the data.
           <br/>
           SSL
           <br/>
           The SSL (secure socket layers) is active during the time that the web browser interacts with the web server, in encrypts all the data sent from and received by the web browser whilst it is being transferred so that if a third party was to intercept the data they would not be able to decrypt it.
           <br/>
           HTTP/S
           <br/>
           This is used by the web browser so it can transfer data with the web server the user wishes to access. The browser sends a request to the server asking for the web pages data, the server then sends the data in packets back to the web browser which then loads the packets and uses them to display the web page on the screen. If a website has HTTPS instead of the usual HTTP it means it has SSL enabled.
           <br/>
           SMTP
           <br/>
           SMTP (Simple Mail Transfer Protocol) is the first step in sending an email. When you click send on an E-mail the computer connects to the hosts SMTP. The computer will send it all the information that is in the message, including who it is from, where it is going, and it's content. The recipients name is then looked up on the servers DNS (domain name server) and if it is located the E-mail is sent to that person.
           <br/>
           POP
           <br/>
           POP(Post office Protocol) allows an E-mail server to receive E-mails, you can do this in a number of ways, retrieve the mail and choose do delete it or not, check to see if new mail has been received without actually reading it and also allows you to read the first few lines of an E-mail without actually opening it.
