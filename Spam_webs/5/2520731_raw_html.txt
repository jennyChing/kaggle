                     <div class="separator" style="clear: both; text-align: left;">
                      <span style="color: #444444; font-family: Arial, Helvetica, sans-serif;">

                       </a>
                       . After simple normalization and registration, I can get an aligned, animated GIF looping through the 20 contestants as shown below.
                      </span>

                     <div class="separator" style="clear: both; text-align: left;">
                      <span style="color: #444444; font-family: Arial, Helvetica, sans-serif;">
                       <br/>
                      </span>
                     </div>
                     <div class="separator" style="clear: both; text-align: center;">
                      <span style="font-family: Arial, Helvetica, sans-serif;">
                       <span style="color: #444444;">
                       </span>
                      </span>
                      <a href="http://1.bp.blogspot.com/-bx1vtqG74Sg/UXmvW0GeaKI/AAAAAAAAWo0/bFcFd04rM3w/s1600/align.gif" imageanchor="1" style="margin-left: 1em; margin-right: 1em; text-align: center;">

                     <div class="separator" style="clear: both; text-align: left;">
                      <span style="color: #444444; font-family: Arial, Helvetica, sans-serif;">

                       <a href="https://sites.google.com/site/jbhuang0604/courses" target="_blank">

                      </span>
                      <span style="color: #444444; font-family: Arial, Helvetica, sans-serif;">

                      </span>
                     </div>

                     <div class="separator" style="clear: both; text-align: left;">
                      <span style="font-family: Arial, Helvetica, sans-serif;">
                       Using the shape of the face images, I can generate a short movie of face morphing from contestant number 1 to 20. We can now see how one face is smoothly transformed into another.

                      <div style="text-align: left;">
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        First, we construct the
                        <a href="http://en.wikipedia.org/wiki/Eigenvalue,_eigenvector_and_eigenspace" target="_blank">
                         eigenspaces
                        </a>

                        <a href="http://en.wikipedia.org/wiki/Principal_component_analysis" target="_blank">

                        </a>

                        <span style="font-family: Arial, Helvetica, sans-serif;">

                       <a href="http://1.bp.blogspot.com/-oTRBFt9lY8E/UXrFyH-ZSfI/AAAAAAAAWrE/WI63oi6qdUU/s1600/coefAll.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;">
                        <span style="font-family: Arial, Helvetica, sans-serif;">
                        </span>
                       </a>
                       <br/>
                      </div>
                      <div style="text-align: left;">
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        Fourth, we can try to construct the affinity matrix which encodes the pairwise similarity. (blue indicates similar and red indicates dissimilar).
                       </span>
                      </div>
                      <span style="font-family: Arial, Helvetica, sans-serif;">
                       <br/>
                      </span>
                      <br/>
                      <div class="separator" style="clear: both; text-align: center;">
                       <a href="http://1.bp.blogspot.com/-q2Wh2WLxMjs/UXrF2kdqCVI/AAAAAAAAWrU/t_C1YxeH35g/s1600/affinityMat.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;">
                        <span style="font-family: Arial, Helvetica, sans-serif;">
                         <img border="0" height="481" src="http://1.bp.blogspot.com/-q2Wh2WLxMjs/UXrF2kdqCVI/AAAAAAAAWrU/t_C1YxeH35g/s640/affinityMat.png" width="640"/>
                        </span>
                       </a>
                      </div>
                      <div class="separator" style="clear: both; text-align: left;">
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        This affinity matrix facilitates many interesting applications. For example, b
                       </span>
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        y summing up over columns or rows, we get a measure of how a contestant is different from the rest. In the plot below, x-axis denotes the contestant index (from 1- 20). The y-axis indicates how distinct the contestant is. This measure can also beinterpretedas the salient object among the 20 contestants using the surround center information divergence.
                       </span>
                      </div>
                      <div class="separator" style="clear: both; text-align: left;">
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        (If you are interested in saliency, please see
                        <a href="https://dl.dropbox.com/u/2810224/Homepage/publications/2012/SaliencyDetection_ICPR_2012/Saliency_ICPR_2012.pdf" target="_blank">
                         our paper on ICPR
                        </a>
                        or
                        <a href="http://www.slideshare.net/jbhuang/saliency-detection-via-divergence-analysis-a-unified-perspective" target="_blank">
                         slide
                        </a>
                        .)
                       </span>
                      </div>
                      <div class="separator" style="clear: both; text-align: left;">
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        <a href="http://1.bp.blogspot.com/-H2msVxgv3Fw/UXrFyAz4TXI/AAAAAAAAWrA/hK_m5CRE1dA/s1600/distinctInd.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;">
                         <img border="0" height="481" src="http://1.bp.blogspot.com/-H2msVxgv3Fw/UXrFyAz4TXI/AAAAAAAAWrA/hK_m5CRE1dA/s640/distinctInd.png" width="640"/>
                        </a>
                       </span>
                      </div>
                      <div style="text-align: left;">
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        <br/>
                       </span>
                      </div>
                      <div style="text-align: left;">
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        From the plot above, we can see that contestant 1, 2 and 6 are more distinct (or more salient) than the rest of the contestants. On the other hand, contestant 7, 12, and 15 are more common (i.e., closed to the averaged face) than others.
                       </span>
                       <br/>
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        <br/>
                        Salient contestants (1, 2, 6)
                       </span>
                       <br/>
                       <div class="separator" style="clear: both; text-align: center;">

                        </span>
                       </div>
                       <br/>
                       <div class="separator" style="clear: both; text-align: left;">
                        <span style="font-family: Arial, Helvetica, sans-serif;">
                         <br/>
                        </span>
                       </div>
                       <div class="separator" style="clear: both; text-align: left;">
                        <span style="font-family: Arial, Helvetica, sans-serif;">
                         It is interesting that one actually used these most salient contestants (1, 2, 6) to support their opinions on how they are dissimilar [
                         <a href="http://www.youtube.com/watch?v=gE-arobPS0Q&amp;feature=youtu.be&amp;t=3m33s" target="_blank">
                          link
                         </a>
                         ]. (I am not sure whether they did the same eigen analysis or not.)
                        </span>
                       </div>

                        </span>
                       </div>
                       <div style="text-align: left;">
                        <span style="font-family: Arial, Helvetica, sans-serif;">
                         Another application of using the affinity matrix is to come up with a displaying order with maximized or minimized similarity from one to another. You may use one of them to support your opinion on whether they are similar or not.
                        </span>
                       </div>
                      </div>
                      <div style="text-align: left;">
                       <div class="separator" style="clear: both; text-align: center;">
                        <span style="font-family: Arial, Helvetica, sans-serif;">
                         <br/>
                        </span>
                       </div>
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        Consider a complete graph of 20 vertices with edges encode the pairwise similarity, the problem of finding a displaying order is the problem of finding a (weighted)
                        <a href="http://en.wikipedia.org/wiki/Hamiltonian_cycle" target="_blank">
                         Hamiltonian circuit
                        </a>
                        in an undirected graph [see
                        <a href="http://en.wikipedia.org/wiki/Hamiltonian_path_problem" target="_blank">
                         Hamiltonian path problem
                        </a>
                       </span>
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        ]. As the problem is known to be NP-hard, we could use a greedy heuristic to find approximated solutions.
                       </span>
                       <br/>
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        <br/>
                        The heuristic algorithm works as follows. Start from a random contestant, choose the next contestant (that has not been selected before) with maximum or minimum similarity with the previous contestant. Repeat the process until we find the ordering.
                       </span>
                       <br/>
                       <span style="font-family: Arial, Helvetica, sans-serif;">
                        <br/>
                        Here is the result of ordering with
                        <i>
                         maximized
                        </i>
                        pairwise similarity. Note that the transition between contestants are more smooth.
                       </span>
                       <br/>
                       <div class="separator" style="clear: both; text-align: center;">
                        <a href="http://2.bp.blogspot.com/--fejdrqSJPg/UX3OHqVM4II/AAAAAAAAWtA/Cv7p1LslcTQ/s1600/similar.gif" imageanchor="1" style="margin-left: 1em; margin-right: 1em; text-align: center;">

                         </span>

                        <i>
                         minimized
                        </i>
                        pairwise
                        <i>
                        </i>
                        similarity. They look more dissimilar in this displaying order.
                       </span>

                                  <p class="comment-content" id="bc_0_36MC">
                                   Hi Kevin,
                                   <br/>
                                   <br/>
                                   Nope, the raw data is not aligned.
                                   <br/>
                                   To align the images using eye positions.
                                   <br/>
                                   1. I manually labeled the eye positions for each faces.
                                   <br/>
                                   2. Compute mean locations of these eye positions.
                                   <br/>
                                   3. Compute similarity transformation from the eye positions in one image to the averaged position.
                                   <br/>
                                   4. Wrap each image according to the similarity transformation computed in (3)
                                   <br/>
                                   <br/>
                                   To compute the averaged face, I did something more. I first compute the mean shape. Then, wrap each image to the mean shape. Lastly, perform pixel-wise average.
                                   <br/>
                                   <br/>
                                   You may take a look of my code to see exactly how these steps are performed.
