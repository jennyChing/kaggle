       <h2>
        Getting to know Directory Opus
       </h2>
       <hr/>
       <h3>
        <img border="0" height="30" src="http://nudel.dopus.com/gtko_images/b_Opus.gif" width="30"/>
        <a name="opus9">
         This guide is old! :-(
        </a>
       </h3>
       <p>
        Directory Opus 10, a major new version, is now available.
       </p>
       <p>
        This guide was written for Opus 8, along with a
        <a href="../opus9/">
         supplemental guide Opus 9
        </a>
        , both of which are rather old now. Most of what's here is still relevant and useful and you'll still learn a lot about Opus from reading it, but some things have moved around or look different now, and there are obviously hundreds of new features which aren't covered.
       </p>
       <p>
        I have started work on an extended update and re-write of this guide, but along the way I got distracted by writing code instead. I would still like to finish the guide at some point, but at least at the moment it's more important to write code to improve the program itself. :)
       </p>
       <p>
        Here are some other resources you may find useful, all of which are more recently updated than this guide:
       </p>
       <ul>
        <li>
         <a href="http://gpsoft.com.au/help/opus10/default.htm">
          The official Opus manual
         </a>
         , which should not be overlooked. Browse through the Basic Concepts section and have a look at the other chapters to see what interests you.
        </li>
        <li>
         <a href="http://www.dearopus.com/">
          Dear Opus
         </a>
         , a guide by Andy, an Opus user, explaining his custom config and various tips and observations in depth.
        </li>
        <li>
         <a href="http://resource.dopus.com/">
          The Opus forums
         </a>
         , not just for tech support but also for shared tutorials, add-ons, ideas, etc.
        </li>
        <li>
         <a href="http://www.youtube.com/user/Nudelyn">
          My YouTube channel
         </a>
         , where you'll find lots of videos showing aspects of Opus. Some of them are structured tutorials. Most are guides to new versions, which also aim to teach you a thing or two while showing you new features (so they are unstructured but will still show you how to do things you probably didn't know).
        </li>
       </ul>
       <p>
        --Leo, 25th October 2013
       </p>

    <h3>
     <img border="0" height="30" src="http://nudel.dopus.com/gtko_images/b_Opus.gif" width="30"/>
     <a name="introduction">
      Introduction
     </a>
    </h3>
    <table class="shot">
     <tr>
      <td>
       <img height="219" src="http://nudel.dopus.com/gtko_images/s_simple.png" width="565"/>
      </td>

    <p>
     Directory Opus is a file manager, and then some. Explaining what's so special about it is difficult because Opus is the sum of many parts. A refined toolbox that can improve the way you use your computer. Flexible. Configurable. Powerful.
    </p>
    <p>
     But all that's a bit vague.
    </p>
    <p>
     This article aims to show you some real-world examples of what Opus can do, from the big features you may wonder how you lived without, to the neat little things that only save a couple of mouse-clicks but which you use many times a day.
    </p>
    <p>
     This is by no means a complete list, nor an objective one. I've omitted things which are killer features for some because this isn't supposed to be a list of everything Opus can do. The beauty of Opus is that you're given a big box of tricks to use as you like. Opus doesn't force you to work in a particular way; instead, you configure Opus to work the way you want and it will bend over backwards to suit you. Or forwards, if you prefer.
    </p>
    <p>
     I hope the next few pages give you a better idea of what Opus does and a desire to try it and explore it for yourself.
    </p>
    <hr/>
    <h3>
     <img border="0" height="30" src="http://nudel.dopus.com/gtko_images/b_Opus.gif" width="30"/>

      Configuration
     </a>
    </h3>
    <p>
     Let's start with a brief look at configuration in Directory Opus. I'm barely going to skim the surface because there's no need to duplicate the online help, 200-page manual and web forum which already exist to explain everything in detail.
    </p>
    <ul>
     <li>
      <p>
       Menus and toolbars are completely customisable. Adding new buttons and removing unwanted ones is easy. Drag &amp; drop buttons from a categorised list of over 180 pre-defined commands or create your own using the internal language, demonstrated later.

     <li>
      <p>
       Buttons can launch other programs, passing them details such as the selected files or the directory you're working from. This means you can use Opus to automate other applications. Batch process everything in the current directory.
       <a href="http://www.gpsoft.com.au/Support/NeroStart.html">

       using Nero.
       <a href="http://resource.dopus.com/viewtopic.php?t=15">

       via mIRC... You can do it directly from your file manager at the click of a button.
      </p>

       Multiple internal and external commands can be chained together on a single button to execute complex tasks.
      </p>

     <li>
      <p>
       You can assign hotkeys to any command, internal or external. You can set any or all hotkeys to be system-wide so that they still work when another program has the keyboard focus, even if there are no Opus windows open at the time. Hotkeys can be associated with toolbar buttons or defined independently as required.
      </p>

        </th>

     <li>
      <p>
       Opus has a vast Preferences dialog where you can tweak just about everything. For many, the first pleasure of using Opus is going through all the options and setting them exactly as they want. Others will use the default settings and only venture into the Preferences dialog when they need to change something. Whichever group you fall into, you'll find settings to suit you.
      </p>
      <p>
       Preferences range from the essential, such as which folder is displayed for new windows (I want the Desktop but maybe you want the last used folder), right down to the tweak level: Don't want to be asked for confirmation when overwriting a read-only file? No problem. Want the full path in titlebars or just the folder name? It's up to you. And yes, you can get the Opus icon in cornflower blue.
      </p>
     </li>
     <li>
      <p>
       How Opus looks is very much up to you. You can move most of the elements around or turn them off completely. Pick your own fonts, colours, icons and background graphics or go with the standard Windows look. Tabs at the top or bottom? Lots of small thumbnails or a few big, detailed ones? Whatever suits you best. Want a bar graph in the status bar showing how much of a blank CD the selected files will use up? Knock yourself out.
      </p>
