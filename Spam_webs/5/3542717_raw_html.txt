     <h1 class="heading">
      <a href="http://thestartuptoolkit.com/blog/2011/10/its-the-ceos-job-to-email-the-first-1000-signups/">
       Its the CEOs job to email the first 1000 signups
      </a>
     </h1>
     <div class="entry">
      <p>
       Until youve passed a thousand signups, the CEO should be personally emailing every new user. Im going to cover:
      </p>
      <ol>
       <li>
        Practicalities
       </li>
       <li>
        How to mess it up
       </li>
       <li>
        Common objections
       </li>
       <li>
        Goals, perks &amp; benefits
       </li>
      </ol>
      <h2>
       The signup thank you note
      </h2>
      <p>
       Its not a big message. Mine look like this:
      </p>
      <blockquote>
       <p>
        Hey Jackie,
       </p>
       <p>
        Thanks for taking the time to check out STK. Give me a shout if you have any questions or if theres anything else I can help with.
       </p>
       <p>
        Stay well,
        <br/>
        Rob
        <br/>
        Founder | +44 7940435340 | @robfitz
       </p>
      </blockquote>
      <p>
       The specifics (like the footer) are just my personal preference[1].
      </p>
      <p>
       What I hope youll extract from the example is the
       <strong>
        casual tone and brevity
       </strong>
       .
      </p>
      <h2>
       How to mess this up
      </h2>
      <p>
       There are only two ways to mess this up and theyre both easy to avoid.
      </p>
      <p>
       The first is to be demanding. I dont even like to ask any questions at this point. Your users owe you nothing.
      </p>
      <p>
       Youre just politely putting your hand up and saying:
      </p>
      <blockquote>
       <p>
        Hey, here I am. You arent alone.
       </p>
      </blockquote>
      <p>
       The second way to mess up is by forgetting common courtesy. They took the time to sign up, you can take the time to try to parse a first name out of their email address and send them a personal one line email.
      </p>
      <p>
       Also, make sure you do it every day. Set up your website to show you a list of new signups at the beginning or end of each day.
      </p>
      <p>
       Ive seen some intros come through with a survey, which I [personally] find to be fairly disrespectful toward your new users time (aka my time). If its mass-mailed and arriving a week later, then were really not starting out on the right foot.
      </p>
      <h2>
       Common objections
      </h2>
      <p>
       This is an easy list to make because I rattled it off to my investors practically every Friday for a year.
      </p>
      <p>
       The number one objection and/or excuse is that you dont want to annoy people. But theres seriously nothing to worry about if you arent being demanding, respect their time by keeping it brief, and reach out both promptly and personally.
      </p>
      <p>
       Also, since youre sending the emails personally and every day, the maximum number of people you can offend is just a days worth of sign ups. It will only take a week or two to find a voice youre comfortable with.
      </p>
      <h2>
       How this will help you
      </h2>
      <p>
       First,
       <strong>
        it ferrets out earlyvangelists
       </strong>
       . Theyll respond to your one line email with a book of suggestions and use cases. Treasure them.
      </p>
      <p>
       Second, a non-negligible percent of your otherwise
       <strong>
        silent cancellations will get in touch
       </strong>
       with dealbreaker feature requests and support crises.
      </p>
      <p>
       Third, your
       <strong>
        users with sales-potential will identify themselves
       </strong>
       by reaching out. If you email all your trial users, the ones who are seriously considering a purchase will jump at the chance to talk directly to the CEO or founder.
      </p>
      <p>
       More than once, Ive said hello to a user with an email like meredith@gmail.com and received a reply from meredith@mtv.com with a title like VP of digital entertainment in the footer.
      </p>
      <p>
       Fourth, and arguably most importantly,
       <strong>
        its just polite
       </strong>
       . Someone took the time to read about and try your startup: thats awesome!
      </p>
      <p>
       You would never think to host a dinner party and then refuse to say hello to anyone who walked through the door. Youve invited people into your website. Introduce yourself and make them feel welcome.
      </p>
      <p>
       <span class="footnote">
        [1] Very few people will actually make unannounced calls to phone numbers in a footer, but I happily answer when they do: free customer development meeting. Also, relegating the title of founder or ceo to the footer saves you from having to waste a sentence in the body on awkward self-aggrandisement.
       </span>
      </p>

      function rss_yay_init(tileset_url) {

            var distance = 150;

            var avatar = $(".rss_avatar");
            var icon = $(".rss_icon");

            avatar.attr("src", tileset_url);
            avatar.css("top", "100px");

            icon.hover( function ( event ) {
                   $(this).data("hover", true); 

                    // get real excited
                    avatar.css("left", "-200px").css("top", 0); 
                },
                function ( event ) {
                   $(this).data("hover", false); 
                });

            icon.mousedown( function ( event ) {
                // so dang excited you won't even believe it
                avatar.css("left", "-300px").css("top", 0); 
            });

            $(document).mousemove( function ( event ) {

                var icon = $(".rss_icon");

                if (icon.data("hover")) {
                    // if mouseover, then don't worry about moves
                    // since the mouseover state trumps it
                    return;
                }

                var dx = event.pageX - icon.offset().left;
                var dy = event.pageY - icon.offset().top; 

                if (dx &gt; 0) {
                    if (dx &lt; icon.outerWidth()) {
                        dx = 0;
                    }
                    else {
                        dx -= icon.outerWidth();
                    }
                }
                if (dy &gt; 0) {
                    if (dy &lt; icon.outerHeight()) {
                        dy = 0; 
                    }
                    else {
                        dy -= icon.outerHeight();
                    }
                }

                dx = Math.abs(dx);
                dy = Math.abs(dy);

                var d = distance + 1;

                if (dx &lt; distance || dy &lt; distance) {
                    d = Math.sqrt( dx * dx + dy * dy );
                }

                if ( d &lt; distance) {
                    if (d &lt; distance / 5) {
                        // getting excited 
                        avatar.css("left", "-100px");
                    }
                    else {
                        avatar.css("left", "0");
                    }

                    // how close to the target we're getting.
                    // this will change the height of the image
                    var percent = Math.floor(100 * d / distance);

                    // not excited, but interested in what's going
                    // on and gradually inching up
                    avatar.css("top", percent + "px"); 
                } 
                else {
                    avatar.css("left", "0").css("top", "100px");
                }
            }); 
        }
