           Unfortunately I think this is very much still a 'hidden' function - i.e. it can only be accessed by modifying the sysex dump from the BCR.  None of the BC editors (that I know about, anyway) even encode the 96 96 96 96 code at the moment and there certainly doesn't seem to be anything obvious when programming the unit in standalone mode (although I guess there's every chance there could be some obscure key combination that allows you to get at these parameters!).
           <br/>
           <br/>
           The only point to add to this is that using the 'acceleration' code does reduce the resolution of the encoders when you turn them quickly (to confirm this, just put in 768 768 768 768 so that you only get the 'super-fast' mode and all will become clear), so maybe that's why Behringer haven't made a big thing of it?  But if, like me, you don't mind this and
           <span style="font-style: italic">
            really
           </span>
           need to speed the encoders up then 96 192 384 768 is your friend!

           <br/>
           For mixing, this is so much better than the normal setup. I think I'll actually be able to ditch my kill switches and free them up for something else.
           <br/>
           <br/>
           Do the numbers refer to how far the encoder will jump for a given "speed" of movement? Do you have any more details on how this is determined? Presumably, you could reduce the speed by reducing the scaling of the numbers? Or you could change it to a flatter/sharper exponential/logarithm setup by varying the "curve" - this in particular would seem like a good idea for DJ'ing, as it would give you accuracy with sharp cuts. I'll have a tinker and post back if I have any further suggestions on this.
           <br/>
           <br/>
           Also, for mac os x users, I found two very simple utilities for making these changes - sysex librarian and hex fiend:
           <br/>
           <!-- m -->
           <a class="postlink" href="http://www.snoize.com/SysExLibrarian/" rel="nofollow">
            http://www.snoize.com/SysExLibrarian/
           </a>
           <!-- m -->
           <br/>
           <!-- m -->
           <a class="postlink" href="http://ridiculousfish.com/hexfiend/" rel="nofollow">
            http://ridiculousfish.com/hexfiend/
           </a>
           <!-- m -->
           <br/>
           <br/>
           It really wasn't hard to do at all:
           <br/>
           <br/>
           1. Start Sysex Library and select the port your bcr is on (probably port 1, if connect via usb.
           <br/>
           2. Click "Record Many" and press edit and preset &gt; on the bcr 2000.
           <br/>
           3. Start Hex Fiend and then click show file in Sysyex Library.
           <br/>
           4. Find the name of the encoder you want to change by holding edit and turning it until the bcr goes into edit mode. The display will read E #, where # is the encoder number.
           <br/>
           5. Press Apple-F in Hex Fiend and search for:
           <br/>
           encoder #
           <br/>
           <br/>
           Press Apple-F again and search for:
           <br/>
           resolution 96 96 96 96
           <br/>
           <br/>
           replace this with
           <br/>
           resolution 96 192 384 768
           <br/>
           <br/>
           (the encoders are listed in strict numerical order (enc 1, enc 2, enc 3 etc...), so if you want to change several in a batch you can just find and replace down the file)
           <br/>
           <br/>
           6. Having made all the changes, save the file (Apple-S).
           <br/>
           7. Go back to Sysex Library and play the file back to the BCR (ensuring the BCR is in it's normal mode). You'll see the display on the BCR cycling if all is going to plan.
           <br/>
           <br/>
           Enjoy!
           <br/>
           <br/>
           J
