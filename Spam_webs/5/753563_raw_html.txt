           <p>
            <strong>
             Dear Testdroiders,
            </strong>
           </p>
           <p>
            Testdroid providesa very powerful and useful API for its users to manage all aspects of mobile development and testing automatically. Were further improving our capability to serve your needs to instantly access our device farm, manage your projects, your test runs and results, plus many other things that will makeyour mobile app, game and web testing smoother, faster and less stressful on real Android and iOS devices.
           </p>
           <p>
            <span id="more-8718">
            </span>
           </p>
           <p>
            <a href="http://docs.testdroid.com/_pages/client-v2.html" target="_blank" title="Testdroid API">
             <img alt="Testdroid API" class="aligncenter size-full wp-image-8755" height="317" src="http://testdroid.com/wp-content/uploads/testdroid_api.jpg" width="650"/>
            </a>
            Testdroid API is an easy-to-use gateway for managing your development and testing effort on real Android and iOS devices. As an alternative for an official Testdroid Cloud site, the API provides all infrastructure through different clients (e.g. Java, Python, Ruby) following the RESTful architecture paradigm and returning JSON with appropriate HTTP response codes.
           </p>
           <p>
            This blog will show you the very basic examples of using the Testdroid API. Full API documentation can be found
            <a href="http://docs.testdroid.com/_pages/client-v2.html" target="_blank" title="Testdroid API">
             <strong>
              here
             </strong>
            </a>
            .
           </p>
           <h1>
            The Basics
           </h1>
           <p>
            In case you are just starting out with API or you dont have experience with basic HTTP requests and responses, you should
            <a href="http://code.tutsplus.com/tutorials/http-the-protocol-every-web-developer-must-know-part-1--net-31177" target="_blank" title="HTTP basics">
             <strong>
              start with this article
             </strong>
            </a>
            . In a nutshell, HTTP allows for communication between a variety of hosts and clients, and support a variety of network configurations.This makes HTTP a
            <strong>
             stateless
            </strong>
            protocol. The communication usually takes place over TCP/IP, but any reliable transport can be used.
           </p>
           <p>
            <a href="http://docs.testdroid.com/_pages/client-v2.html" target="_blank" title="Testdroid API">
             <img alt="HTTP Request - Response" class="aligncenter size-full wp-image-8760" height="409" src="http://testdroid.com/wp-content/uploads/reqres.png" width="650"/>
            </a>
            <a href="http://docs.testdroid.com/_pages/client-v2.html" target="_blank" title="API">
             <br/>
            </a>
            <span style="color: #4b4b4b;">
             Communication between a host and a client occurs, via a
            </span>
            <strong style="color: #4b4b4b;">
             request/response pair
            </strong>
            <span style="color: #4b4b4b;">
             . The client initiates an HTTP request message, which is serviced through a HTTP response message in return. We will look at this fundamental message-pair in the next section  with Authentication/Authorization example.
            </span>
           </p>
           <p>
            Were using
            <strong>
             <a href="http://curl.haxx.se/" target="_blank" title="cURL">
              cURL
             </a>
            </strong>
            in the following examples. In case you are not familiar with it, cURL is a software providing a library and command-line tool for transferring data using various protocols. You can download it from
            <a href="http://curl.haxx.se/download.html" target="_blank" title="cURL Download">
             <strong>
              here
             </strong>
            </a>
            .
           </p>
           <h1>
            <strong>

            </strong>
           </h1>

             Testdroid API uses

              OAuth 2.0
             </a>

              an open standard for authorization. The OAuth 2.0 focuses on client developer simplicity while providing specific authorization flows for web apps. As it is seamlessly used with Testdroid API, you can use it to authorize further API calls to our cloud back-end.
            </span>
           </p>

           <h1>

            </code>
           </p>
           <p>
            <strong>
             Get details of a specific project  defined by PROJECT_ID:
            </strong>
           </p>
           <p>
            <code>
             curl -H "Accept: application/json" -H "Authorization: Bearerabcdefgh-1234-ijkl-m5n6-opqrstuvxyxz" https://cloud.testdroid.com/api/v2/projects/PROJECT_ID
            </code>
           </p>
           <p>

            </code>

            <strong>
             Example with Ruby:
            </strong>
           </h1>
           <p>
            The Rubysample of Testdroid API can be found
            <a href="https://github.com/bitbar/testdroid-api-client-ruby" target="_blank" title="Ruby sample">
             <strong>
              here
             </strong>
            </a>
            .
           </p>
           <p>
            <strong>
             Installation:
            </strong>
            <br/>
            <code>
             <br/>
             gem "testdroid-api-client"
             <br/>
             bundle install
            </code>
           </p>
           <p>
            <strong>
             Sample client usage  Authentication:
            </strong>
            <br/>
            <code>
             <br/>
             require 'testdroid-api-client'
             <br/>
             client = TestdroidAPI::Client.new('admin@localhost', 'admin')
            </code>
           </p>
           <p>
            <strong>
             Sample usage  Get projects:
            </strong>
           </p>
           <p>
            @user = client.authorize
            <br/>
            projects = @user.projects.list
           </p>
           <p>
            <strong>
             Get project by id:
            </strong>
           </p>
           <p>
            project_id = 123
            <br/>
            project123 = @user.projects.get(project_id)
            <br/>
            # output project name
            <br/>
            p Project name #{project123.name}
           </p>
           <p>
            <strong>
             Start project:
            </strong>
           </p>
           <p>
            test_run = project123.run
           </p>
           <p>
            <strong>
             Check test run status:
            </strong>
           </p>
           <p>
            p Project state #{test_run.state}
           </p>
           <p>
            <strong>
             Download all logs from test run:
            </strong>
           </p>
           <p>
            test_run.device_runs.list({:params =&gt; {:limit =&gt; 100}}).each { |drun| drun.download_logs(#{drun.id}_log) }
           </p>
           <p>
            <strong>
             Using device labels:
            </strong>
           </p>
           <p>
            # Get label for android os version 2.1
            <br/>
            lg_android_version_2_1 = client.label_groups.list.detect {|lg| lg.display_name.casecmp(android version) == 0 }
           </p>
           <p>
            os_v2_1 = client.label_groups.get(lg_android_versions.id).labels.list.detect {|l| l.display_name.casecmp(2.1) == 0 }
           </p>
           <p>
            #get all devices with android os level 2.1
            <br/>
            devices = client.label_groups.get(lg_android_versions.id).labels.get(os_v2_1.id).devices
           </p>
           <p>
            # get speficic device from devices list
            <br/>
            lenovo_a820 = devices.list.detect {|d| d.display_name == Lenovo A820}
           </p>
           <p>
            <strong>
             Device Sessions:
            </strong>
           </p>
           <p>
            # To create a new device session:
            <br/>
            device_session = user.device_sessions.create({:params =&gt; {deviceModelId =&gt; 1}})
           </p>
           <p>
            # To release device session:
            <br/>
            device_session.release()
