       <p>
        From the cryptsetup project's
        <a class="external text" href="https://gitlab.com/cryptsetup/cryptsetup/wikis/DMCrypt" rel="nofollow">
         wiki
        </a>
        :
       </p>
       <dl>
        <dd>
         Device-mapper is infrastructure in the Linux 2.6 and 3.x kernel that provides a generic way to create virtual layers of block devices. Device-mapper crypt target provides transparent encryption of block devices using the kernel crypto API. The user can basically specify one of the symmetric ciphers, an encryption mode, a key (of any allowed size), an iv generation mode and then the user can create a new block device in /dev. Writes to this device will be encrypted and reads decrypted. You can mount your filesystem on it as usual or stack dm-crypt device with another device like RAID or LVM volume. Basic documentation of dm-crypt mapping table comes with kernel source and the latest version is available in git repository.
        </dd>
       </dl>

       <h2>
        <span class="mw-headline" id="Common_scenarios">
         Common scenarios
        </span>
       </h2>
       <p>
        This section introduces common scenarios to employ
        <i>
         dm-crypt
        </i>
        to encrypt a system or individual filesystem mount points. The scenarios cross-link to the other subpages where needed. It is meant as starting point to familiarize with different practical encryption procedures.
       </p>
       <p>
        See
        <a href="/index.php/Dm-crypt/Encrypting_a_non-root_file_system" title="Dm-crypt/Encrypting a non-root file system">
         Dm-crypt/Encrypting a non-root file system
        </a>
        if you need to encrypt a device that is not used for booting a system, like a
        <a href="/index.php/Dm-crypt/Encrypting_a_non-root_file_system#Partition" title="Dm-crypt/Encrypting a non-root file system">
         partition
        </a>
        or a
        <a href="/index.php/Dm-crypt/Encrypting_a_non-root_file_system#Loop_device" title="Dm-crypt/Encrypting a non-root file system">
         loop device
        </a>
        .
       </p>
       <p>
        See
        <a href="/index.php/Dm-crypt/Encrypting_an_entire_system" title="Dm-crypt/Encrypting an entire system">
         Dm-crypt/Encrypting an entire system
        </a>
        if you want to encrypt an entire system, in particular a root partition. Several scenarios are covered, including the use of
        <i>
         dm-crypt
        </i>
        with the
        <i>
         LUKS
        </i>
        extension,
        <i>
         plain
        </i>
        mode encryption and encryption and
        <i>
         LVM
        </i>
        .
       </p>

        <span class="mw-headline" id="Drive_preparation">

        </span>

        </a>
        and
        <i>
         dm-crypt
        </i>
        specific points for
        <a href="/index.php/Dm-crypt/Drive_preparation#Partitioning" title="Dm-crypt/Drive preparation">
         partitioning it
        </a>
        .
       </p>
       <p>
        See
        <a href="/index.php/Dm-crypt/Drive_preparation" title="Dm-crypt/Drive preparation">
         Dm-crypt/Drive preparation
        </a>
        .
       </p>
       <h2>
        <span class="mw-headline" id="Device_encryption">
         Device encryption
        </span>
       </h2>
       <p>
        This section covers how to manually utilize dm-crypt to encrypt a system through the
        <a href="/index.php/Dm-crypt/Device_encryption#Cryptsetup_usage" title="Dm-crypt/Device encryption">
         cryptsetup
        </a>
        command. It covers examples of the
        <a href="/index.php/Dm-crypt/Device_encryption#Encryption_options_with_dm-crypt" title="Dm-crypt/Device encryption">
         Encryption options with dm-crypt
        </a>
        , deals with the creation of
        <a href="/index.php/Dm-crypt/Device_encryption#Keyfiles" title="Dm-crypt/Device encryption">
         keyfiles
        </a>
        , LUKS specific commands for
        <a href="/index.php/Dm-crypt/Device_encryption#Key_management" title="Dm-crypt/Device encryption">
         key management
        </a>
        as well as for
        <a href="/index.php/Dm-crypt/Device_encryption#Backup_and_restore" title="Dm-crypt/Device encryption">
         Backup and restore
        </a>
        .
       </p>
       <p>
        See
        <a href="/index.php/Dm-crypt/Device_encryption" title="Dm-crypt/Device encryption">
         Dm-crypt/Device encryption
        </a>
        .
       </p>

       <h2>
        <span class="mw-headline" id="Swap_device_encryption">
         Swap device encryption
        </span>
       </h2>
       <p>
        A swap partition may be added to an encrypted system, if required. The swap partition must be encrypted as well to protect any data swapped out by the system. This part details methods
        <a class="mw-redirect" href="/index.php/Dm-crypt_with_LUKS/Swap_Encryption#Without_suspend-to-disk_support" title="Dm-crypt with LUKS/Swap Encryption">
         without
        </a>
        and
        <a class="mw-redirect" href="/index.php/Dm-crypt_with_LUKS/Swap_Encryption#With_suspend-to-disk_support" title="Dm-crypt with LUKS/Swap Encryption">
         with
        </a>
        suspend-to-disk support.
       </p>
       <p>
        See
        <a href="/index.php/Dm-crypt/Swap_encryption" title="Dm-crypt/Swap encryption">
         Dm-crypt/Swap encryption
        </a>
        .
       </p>

        <span class="mw-headline" id="Specialties">
