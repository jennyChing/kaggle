        <h1>
         HTML5 Offers Scoped CSS for Precision Styling
        </h1>
        <div class="entryAuthor">
         By
         <a href="http://www.webmonkey.com/author/luxagraf/" rel="author" title="Posts by Scott Gilbertson">
          Scott Gilbertson
         </a>
        </div>
       </div>
       <div class="entry">
        <p>
         <!-- wpautop enabled -->
        </p>
        <p>
         <img alt="" class="size-full wp-image-55943" data-lazy-src="http://www.webmonkey.com/wp-content/uploads/2012/04/htmlscoped4.jpg" height="195" src="http://archive.wired.com/images/1x1.trans.gif" title="htmlscoped4" width="280"/>
        </p>
        <noscript>
         <img alt="" class="size-full wp-image-55943" height="195" src="http://www.webmonkey.com/wp-content/uploads/2012/04/htmlscoped4.jpg" title="htmlscoped4" width="280"/>
        </noscript>
        HTML5s controversial scoped style attribute is now supported in the
        <a href="http://tools.google.com/dlpage/chromesxs">
         latest Canary builds of Googles Chrome web browser
        </a>
        and Mozilla may eventually
        <a href="https://bugzilla.mozilla.org/show_bug.cgi?id=508725">
         add support to Firefox
        </a>
        as well.
        <p>
         HTML5
         <a href="http://www.w3.org/TR/html5/the-style-element.html#attr-style-scoped">
          adds an attribute, scoped, to the style element
         </a>
         which limits the scope of the styles contained within the tag. Googles Alex Danilo has
         <a href="http://updates.html5rocks.com/2012/03/A-New-Experimental-Feature-style-scoped">
          a good introduction to scoped
         </a>
         over at HTML5Rocks. Essentially scoped allows you to nest styles within HTML and those styles will only apply to any child elements. The easiest way to understand scoped is to see it in action:
        </p>
        <p style="clear:both;">
        </p>
        <pre class="brush: html">
         &lt;html lang="en"&gt;
    &lt;head&gt;
        &lt;style&gt;
            p { color: white; }
        &lt;/style&gt;
    &lt;/head&gt;
    &lt;body&gt;
        &lt;p&gt;The text in this will be white&lt;/p&gt;
        &lt;div&gt;
            &lt;style scoped&gt;
                p { color: red; }
            &lt;/style&gt;
            &lt;p&gt;The text in this paragraph will be red&lt;/p&gt;
        &lt;/div&gt;
        &lt;p&gt;And we're back to white text&lt;/p&gt;
    &lt;/body&gt;
&lt;/html&gt;
        </pre>
        <p>
         In this example the first style declaration applies to all the p tags on the page while the scoped style declaration applies only to the p tags within that scope (in this case all the p tags within our div tag). The scope rule has a higher specificity so it overrides the other rule and makes the middle text red.
        </p>
        <p>
         It seems handy at first glance, but scoped can be a step backward if you overuse it. Its akin to using inline styles, a long-frowned-upon practice that means mingling content and presentation. Indeed, scoped should be used sparingly; it will cause you pain if you just start throwing it around.
        </p>
        <p>
         That said, there are some cases where scoped makes sense and makes developers lives easier. If youre pulling in content from some outside source and displaying it on your page, styles and all, scoped can help you avoid style conflicts. Similarly if youre building a widget designed to be embedded elsewhere scoped ensures your styles wont affect the rest of the page.
        </p>
        <p>
         Another use case thats worth mentioning is within content management systems. Developer Arley McBlain recently covered the
         <a href="http://css-tricks.com/saving-the-day-with-scoped-css/">
          advantages of the scoped attribute for handling CMS content
         </a>
         in some depth over at CSS Tricks. McBlain even has
         <a href="http://arleym.com/scopedcss/">
          a demo WordPress site
         </a>
         that uses scoped to style content on a per-post basis.
        </p>
        <p>
         The big catch to using scoped right now is that it doesnt work anywhere but Chrome 19+. Worse, because other browsers do understand the style element, theyll apply your styles, but theyll apply them globally. In other words, dont use scoped without a polyfill like the
         <a href="https://github.com/thingsinjars/jQuery-Scoped-CSS-plugin">
          JQuery Scoped CSS Plugin
         </a>
         .
        </p>
        <p>
         Also note that to test out Chromes scoped support youll need to be using Chrome 19 and then head to chrome://flags. Look for the Enable &lt;style scoped&gt; towards the end of the list and click Enable. Restart the browser and scoped should work.
        </p>
