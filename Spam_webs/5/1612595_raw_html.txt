  <p style="text-align: center;">
   Packages of
   <a href="http://www.go-mono.org/">
    Mono
   </a>
   , a Free Software
   <a href="http://msdn.microsoft.com/netframework/">
    .NET framework
   </a>
   implementation, are now available for
   <a href="http://www.debian.org/">
    Debian
   </a>
   GNU/Linux. It is hoped that feedback from users of these packages will facilitate the development of a framework for the seamless integration of Mono and the .NET environment into Debian GNU/Linux.
  </p>

     <h2>
      Download
     </h2>
     <p>
      The packages available from the local apt source have been tested on
      <tt>
       unstable
      </tt>
      Debian GNU/Linux systems.
     </p>
     <p>
      To get started, install
      <tt>
       mono
      </tt>
      , a metapackage which will provide a complete .NET development and testing evironment.
     </p>
     <p>
      To get the documentation, install the
      <tt>
       monodoc
      </tt>
      , a metapackage
which will install the manual and at least one viewer package.
     </p>
     <p>
      The
      <a href="http://www.monodevelop.com/">
       MonoDevelop
      </a>
      IDE can be
installed with the
      <tt>
       monodevelop
      </tt>
      package.
     </p>
     <h3>
      Release packages for Debian unstable (sid)
     </h3>
     The packages enter Sid when they are ready. Sometimes there are timing
problems when certain packages stay for too long in the NEW queue. If
you really need fresh packages, you can test some from our preview
repository.
     <h3>
      Unreleased preview packages for Debian unstable (sid)
     </h3>
     <p>
      <tt>
       /etc/apt/sources.list
      </tt>
      should contain:
     </p>
     <pre>
      deb http://debian.meebey.net/pkg-mono ./
     </pre>
     <h3>
      Release packages for Debian Testing
     </h3>
     <p>
      There are not explicite packages for Debian Testing since maintaining
them is sisyphus work. Please wait for Testing to adopt enough
packages from Unstable to fullfil the dependencies.
     </p>

     <h2>
      Various comments
     </h2>
     <h3>
      BINFMT_MISC
     </h3>
     <p>
      The packages integrate .NET executables into the Linux system using
      <tt>
       binfmt-support
      </tt>
      . This is an optional feature, the recommended
way is using the
      <tt>
       cli-wrapper
      </tt>
      or a wrapper script for installed
applications. Make sure that your kernel is compiled with
      <tt>
       <a href="binfmt_misc.txt">
        BINFMT_MISC
       </a>
      </tt>
      support.
This shouldn't be an issue if you're using the kernel provided by your vendor / Debian.
     </p>
