       <h2>
        <span class="mw-headline" id="Linux_webcam_support">
         Linux webcam support
        </span>
       </h2>
       <p>
        Most probably your webcam will work out of the box. In that case you may skip to section
        <a href="#Webcam_configuration">
         #Webcam configuration
        </a>
        if you want to configure color, brightness and other parameters. Otherwise follow the steps below.
       </p>
       <h2>
        <span class="mw-headline" id="Identify_your_webcam">
         Identify your webcam
        </span>
       </h2>
       <p>
        Identify the name of your webcam (using, for example,
        <code>
         lsusb
        </code>
        ) and find a proper driver.  Below is a list of webcams, and what drivers they work with. If you get your webcam to work, add the name of the webcam and the driver you used to the list!
       </p>
       <h3>
        <span class="mw-headline" id="pwc">
         pwc
        </span>
       </h3>
       <ul>
        <li>
         Creative Labs Webcam Pro Ex
        </li>
        <li>
         Logitech QuickCam Notebook Pro (only the "Pro" models)
        </li>
        <li>
         Logitech Quickcam Pro 4000
        </li>
        <li>
         Philips ToUCams (not confirmed at the moment, but it is using the pwc driver if I remember correctly)
        </li>
        <li>
         Philips SPC900NC
        </li>
       </ul>
       <h3>
        <span class="mw-headline" id="qc-usb">
         qc-usb
        </span>
       </h3>
       <ul>
        <li>
         Dexxa Webcam
        </li>
        <li>
         Labtec Webcam (old model)
        </li>
        <li>
         LegoCam
        </li>
        <li>
         Logitech Quickcam Express (old model)
        </li>
        <li>
         Logitech QuickCam Notebook (not the "Pro" models)
        </li>
        <li>
         Logitech Quickcam Web
        </li>
       </ul>
       <h3>
        <span class="mw-headline" id="qc-usb-messenger">
         qc-usb-messenger
        </span>
       </h3>
       <ul>
        <li>
         Logitech Quickcam Messenger
        </li>
        <li>
         Logitech Quickcam Communicate (for Communicate MP/S5500 or "for Business" see the linux-uvc section below)
        </li>
       </ul>
       <p>
        It is now in the community repo.
       </p>
       <div style="padding: 5px; margin: 0.50em 0; background-color: #DDDDFF; border: thin solid #BBBBDD; overflow: hidden;">
        <strong>
         Note:
        </strong>
        <ul>
         <li>
          If qc-usb-messenger does not work use the gspca module, by installing the gspcav1 package.
         </li>
         <li>
          Now this driver is a module included in kernel 2.6.27.
         </li>
        </ul>
       </div>
       <h3>
        <span class="mw-headline" id="zr364xx">
         zr364xx
        </span>
       </h3>
       <p>
        This driver can be used for many webcams like:
       </p>
       <ul>
        <li>
         Aiptek PocketDV 3300
        </li>
        <li>
         Creative PC-CAM 880
        </li>
        <li>
         Konica Revio 2
        </li>
        <li>
         Genius Digital Camera
        </li>
        <li>
         Maxell Maxcam PRO DV3
        </li>
       </ul>
       <p>
        You can find the full list of supported devices
        <a class="external text" href="http://royale.zerezo.com/zr364xx/" rel="nofollow">
         here
        </a>
        . You can find a PKGBUILD for this driver in the
        <a class="mw-redirect" href="/index.php/AUR" title="AUR">
         AUR
        </a>
        .
       </p>
       <h3>
        <span class="mw-headline" id="sn9c102">
         sn9c102
        </span>
       </h3>
       <ul>
        <li>
         Trust Spacecam series
        </li>
        <li>
         Maxell Smartcam (for notebooks): 352x288 max. resolution @ 3fps
        </li>
       </ul>
       <h3>
        <span class="mw-headline" id="gspca">
         gspca
        </span>
       </h3>
       <p>
        An extensive list of supported webcams is available
        <a class="external autonumber" href="http://git.kernel.org/?p=linux/kernel/git/torvalds/linux-2.6.git;a=blob;f=Documentation/video4linux/gspca.txt;hb=HEAD" rel="nofollow">
         [1]
        </a>
        .
       </p>
       <div style="padding: 5px; margin: 0.50em 0; background-color: #DDDDFF; border: thin solid #BBBBDD; overflow: hidden;">
        <strong>
         Note:
        </strong>
        This driver does not have V4L1 support.
       </div>
       <h3>
        <span class="mw-headline" id="stv680">
         stv680
        </span>
       </h3>
       <p>
        Many cheap no-name cameras that came out Asia in the last couple of years use the stv680 chipset. Most of these cameras were novelty items (i.e. Pencam, SpyC@m and LegoCam).
       </p>
       <ul>
        <li>
         Aiptek PenCam series
        </li>
        <li>
         Digitaldream series
        </li>
        <li>
         Dolphin Peripherals series
        </li>
        <li>
         Lego LegoCam
        </li>
        <li>
         Trust SpyC@m series
        </li>
        <li>
         Welback Coolcam
        </li>
       </ul>
       <p>
        A more-complete list of webcams that use the stv680 chipset is available
        <a class="external text" href="http://webcam-osx.sourceforge.net/cameras/index.php?orderBy=controller" rel="nofollow">
         here
        </a>
        .
       </p>
       <h3>
        <span class="mw-headline" id="linux-uvc">
         linux-uvc
        </span>
       </h3>
       <ul>
        <li>
         Genius iLook 1321
        </li>
        <li>
         Logitech   Webcam C210
        </li>
        <li>
         Logitech   Webcam C250
        </li>
        <li>
         Logitech   Webcam C270
        </li>
        <li>
         Logitech   Webcam C600
        </li>
        <li>
         Logitech HD Webcam C525
        </li>
        <li>
         Logitech HD Pro Webcam C920
        </li>
        <li>
         Logitech Quickcam Pro 5000
        </li>
        <li>
         Logitech Quickcam Pro 9000
        </li>
        <li>
         Logitech Quickcam Orbit AF
        </li>
        <li>
         Logitech Quickcam Orbit MP
        </li>
        <li>
         Logitech Quickcam S5500
        </li>
        <li>
         Microdia Pavilion Webcam (on MSI PR200)
        </li>
        <li>
         Logitech Quickcam Communicate MP/S5500 or "for Business"
        </li>
        <li>
         Chicony Electronics CNF7051
        </li>
       </ul>
       <p>
        You can find a full list of supported UVC devices
        <a class="external text" href="http://www.ideasonboard.org/uvc/" rel="nofollow">
         here
        </a>
        .
       </p>
       <p>
        As of kernel 2.6.26 linux-uvc is part of the kernel. Just load the uvcvideo module.
       </p>
       <div style="padding: 5px; margin: 0.50em 0; background-color: #DDDDFF; border: thin solid #BBBBDD; overflow: hidden;">
        <strong>
         Note:
        </strong>
        <ul>
         <li>
          This driver does not have V4L1 support.
         </li>
         <li>
          With WebCam SCB-0385N (usb ID 2232:1005), WebCam SC-0311139N (usb ID 2232:1020) and WebCam SC-03FFL11939N (usb ID 2232:1028), you might need to add some configuration to the module if the usage of the camera makes the system freeze:
         </li>
        </ul>
        <pre style="margin-bottom: 0; border-bottom:none; padding-bottom:0.8em;">
         /etc/modprobe.d/uvcvideo.conf
        </pre>
        <pre style="margin-top: 0; border-top-style:dashed; padding-top: 0.8em;">
         options uvcvideo nodrop=1
        </pre>
       </div>
       <h3>
        <span class="mw-headline" id="ov51x-jpeg">

       <h2>
        <span class="mw-headline" id="Make_sure_the_module_is_loaded_for_your_webcam">
         Make sure the module is loaded for your webcam
        </span>
       </h2>
       <p>
        Add your webcam's
        <a href="/index.php/Kernel_modules#Loading" title="Kernel modules">
         module
        </a>
        in
        <code>
         /etc/modules-load.d/webcam.conf
        </code>
        so it will be loaded into the kernel during init stage bootstrapping.
       </p>
       <p>
        If your webcam is USB, the kernel
        <i>
         should
        </i>
        automatically load the proper driver.  If this is the case, check dmesg after you plug your webcam in. You should see something like this:
       </p>
       <pre style="margin-bottom: 0; border-bottom:none; padding-bottom:0.8em;">
        $ dmesg|tail
       </pre>
       <pre style="margin-top: 0; border-top-style:dashed; padding-top: 0.8em;">
        sn9c102: V4L2 driver for SN9C10x PC Camera Controllers v1:1.24a
usb 1-1: SN9C10[12] PC Camera Controller detected (vid/pid 0x0C45/0x600D)
usb 1-1: PAS106B image sensor detected
usb 1-1: Initialization succeeded
usb 1-1: V4L2 device registered as /dev/video0
usb 1-1: Optional device control through 'sysfs' interface ready
usbcore: registered new driver sn9c102
       </pre>

        <span class="plainlinks archwiki-template-pkg">

       </p>
       <h2>
        <span class="mw-headline" id="Get_software_to_use_your_webcam">
         Get software to use your webcam
        </span>
       </h2>
       <p>
        Version 2.6.27 of the Linux kernel supports
        <a class="external text" href="http://mxhaard.free.fr/spca5xx.html" rel="nofollow">
         many new webcam drivers
        </a>
        .  Legacy Video4Linux API has been dropped, and these drivers now only support Video4Linux version 2.  Pixel format decoding has been pushed to user space, since Video4Linux version 2 does not support kernel space decoding.  The libv4l library provides userland applications with pixel decoding services and will be used by most programs.  Other compatibility layers are also available.
       </p>
       <p>
        <b>
         If your device is created but your image looks strange (mine was nearly completely green), you probably need this.
        </b>
       </p>
       <p>
        If the application has V4L2 support but no pixelformat support (eg: cheese) then use the following command:
       </p>
       <pre>
        LD_PRELOAD=/usr/lib/libv4l/v4l2convert.so cheese
       </pre>
       <p>
        If the application only supports the older version of V4L (Skype is the most popular of this kind of software) then use this command:
       </p>
       <pre>
        LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so skype
       </pre>
       <div style="padding: 5px; margin: 0.50em 0; background-color: #DDFFDD; border: thin solid #BBDDBB; overflow: hidden;">
        <strong>
         Tip:
        </strong>
        You also might want to put a line like the following into
        <code>
         /etc/profile
        </code>
        or
        <a href="/index.php/Xprofile" title="Xprofile">
         xprofile
        </a>
        so you do not have to type that long command all the time:
        <pre>
         export LD_PRELOAD=/usr/'$LIB'/libv4l/v4l2convert.so
        </pre>
        <p>
         or
        </p>
        <pre>
         export LD_PRELOAD=/usr/'$LIB'/libv4l/v4l1compat.so
        </pre>
       </div>
       <p>
        For 32-bit applications (e.g. Skype) within Arch64, install the
        <span class="plainlinks archwiki-template-pkg">
         <a class="external text" href="https://www.archlinux.org/packages/?name=lib32-v4l-utils" rel="nofollow">
          lib32-v4l-utils
         </a>
        </span>
        package.
       </p>
       <p>
        If the webcam works fine on guvcview, but it does not work in Skype, you may also need to set
       </p>
       <pre>
        export XLIB_SKIP_ARGB_VISUALS=1
       </pre>
       <p>
        before starting it.
       </p>

       <h3>
        <span class="mw-headline" id="VLC">
         VLC
        </span>
       </h3>
       <p>
        <a class="mw-redirect" href="/index.php/VLC" title="VLC">
         VLC
        </a>
        can also be used to view and record your webcam. In VLC's file menu, open the 'Capture Device...' dialog and enter the video and audio device files. Or from the command line, do:
       </p>
       <pre>
        $ vlc v4l://:v4l-vdev="/dev/video0":v4l-adev="/dev/audio2"
       </pre>
       <p>
        This will make VLC mirror your webcam. To take stills, simply choose 'Snapshot' in the 'Video' menu. To record the stream, you add a
        <code>
         --sout
        </code>
        argument, e.g.
       </p>
       <pre>
        $ vlc v4l://:v4l-vdev="/dev/video0":v4l-adev="/dev/audio2" \ 
  --sout "#transcode{vcodec=mp1v,vb=1024,scale=1,acodec=mpga,ab=192,channels=2}:duplicate{dst=std{access=file,mux=mpeg1,dst=/tmp/test.mpg}}"
       </pre>
       <p>
        (Obviously a bit overkill with regard to the bit rates but it is fine for testing purposes.) Notice that this will not produce a mirror on the display - in order to see what you are recording, you would need to add the display as a destination to the argument:
       </p>
       <pre>
        ...:duplicate{dst=display,dst=std{access= ....
       </pre>
       <p>
        (Though this can tax older hardware somewhat...)
       </p>
       <h3>
        <span class="mw-headline" id="MPlayer">
         MPlayer
        </span>
       </h3>
       <p>
        To use
        <a href="/index.php/MPlayer" title="MPlayer">
         MPlayer
        </a>
        to take snapshots from your webcam run this command from the terminal:
       </p>
       <pre>
        $ mplayer tv:// -tv driver=v4l2:width=640:height=480:device=/dev/video0 -fps 15 -vf screenshot
       </pre>
       <p>
        From here you have to press
        <code>
         s
        </code>
        to take the snapshot. The snapshot will be saved in your current folder as
        <b>
         shotXXXX.png
        </b>
        .
If you want to record continuous video:
       </p>
       <pre>
        $ mencoder tv:// -tv driver=v4l2:width=640:height=480:device=/dev/video0:forceaudio:adevice=/dev/dsp -ovc lavc -oac mp3lame -lameopts cbr:br=64:mode=3 -o
        <i>
         filename
        </i>
        .avi
       </pre>
       <p>
        Press
        <code>
         Ctrl+c
        </code>
        to end the recording.

       <h3>
        <span class="mw-headline" id="Motion">
         Motion
        </span>
       </h3>
       <dl>
        <dd>
         <i>
          Motion is a program that monitors the video signal from cameras. It is able to detect if a significant part of the picture has changed; in other words, it can detect motion.
         </i>
        </dd>
       </dl>
       <p>
        <span class="plainlinks archwiki-template-pkg">
         <a class="external text" href="https://www.archlinux.org/packages/?name=motion" rel="nofollow">
          motion
         </a>
        </span>
        can only handle v4l2 devices so if you need to use a camera that only has v4l1 drivers you need to preload v4l1compat.so as previously mentioned. Otherwise you will get loads of errors about motion not able to find a suitable palette.
       </p>

        </strong>

        <a href="/index.php/Udev#Writing_udev_rules" title="Udev">

       <h2>
        <span class="mw-headline" id="Troubleshooting">
         Troubleshooting
        </span>
       </h2>
       <h3>
        <span class="mw-headline" id="Microsoft_Lifecam_Studio.2FCinema">
         Microsoft Lifecam Studio/Cinema
        </span>
       </h3>
       <p>
        Under certain configurations, the Microsoft lifecam studio/cinema may request too much usb bandwidth and fail
        <a class="external text" href="http://www.ideasonboard.org/uvc/#footnote-13" rel="nofollow">
         see Uvcvideo FAQ
        </a>
        . In this case, change the buffering by loading the
        <code>
         uvcvideo
        </code>
        driver with
        <code>
         quirks=0x80
        </code>
        . Add it to
        <code>
         /etc/modprobe.d/uvcvideo.conf
        </code>
        :
       </p>
       <pre style="margin-bottom: 0; border-bottom:none; padding-bottom:0.8em;">
        /etc/modprobe.d/uvcvideo.conf
       </pre>
       <pre style="margin-top: 0; border-top-style:dashed; padding-top: 0.8em;">
        ## fix bandwidth issue for lifecam studio/cinema
options uvcvideo quirks=0x80
       </pre>
       <div style="padding: 5px; margin: 0.50em 0; background-color: #DDDDFF; border: thin solid #BBBBDD; overflow: hidden;">
        <strong>
         Note:
        </strong>
        If delays are visible in the logs, or the camera works periodically, this workaround should apply generally. Bigger values such as
        <code>
         quirks=0x100
        </code>
        are possible.
       </div>
