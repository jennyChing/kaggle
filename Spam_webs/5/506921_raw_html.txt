     <b>
      Concepts
     </b>
     <p>
      Several concepts are of particular importance to data warehousing.  They are discussed in detail in this section.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/dimensional.html">
         Dimensional Data Model
        </a>
       </b>
      </u>
      :  Dimensional data model is commonly used in data warehousing systems.  This section describes this modeling technique, and the two common schema types,
      <a href="http://www.1keydata.com/datawarehousing/star-schema.html">
       star schema
      </a>
      and
      <a href="http://www.1keydata.com/datawarehousing/snowflake-schema.html">
       snowflake schema
      </a>
      .
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/slowly-changing-dimensions.html">
         Slowly Changing Dimension
        </a>
       </b>
      </u>
      :  This is a common issue facing data warehousing
practioners.  This section explains the problem, and describes the three
ways of handling this problem with examples.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/conceptual-data-model.html">

      </u>

      </u>
      : What is a logical data model, its features, and an example of this type of data model.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/physical-data-model.html">
         Physical Data Model
        </a>
       </b>
      </u>
      : What is a physical data model, its features, and an example of this type of data model.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/data-modeling-levels.html">
         Conceptual, Logical, and Physical Data Model
        </a>
       </b>
      </u>
      : Different levels of abstraction for a data model.  This section compares and contrasts the three different types of data models.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/data-integrity.html">

      </u>
      :  What is data integrity and how it is enforced in data warehousing.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/what-is-olap.html">
         What is OLAP
        </a>
       </b>
      </u>
      :  Definition of OLAP.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/molap-rolap.html">
         MOLAP, ROLAP, and HOLAP
        </a>
       </b>
      </u>
      :  What are these different types of OLAP technology?  This section discusses how they are different from the other, and the advantages and disadvantages of each.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/inmon-kimball.html">
         Bill Inmon vs. Ralph Kimball
        </a>
       </b>
      </u>
      :  These two data warehousing heavyweights
have a different view of the role between data warehouse and data mart.
     </p>
     <p>
      <u>
       <b>
        <a href="http://www.1keydata.com/datawarehousing/factless-fact-table.html">
         Factless Fact Table
        </a>
       </b>
      </u>
      :  A fact table without any fact may sound silly, but there are real life instances when a factless fact table is useful in data warehousing.
     </p>
