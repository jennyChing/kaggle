          <div class="message-panel message-panel-info join-promotion">
           <h3>
            <strong>
             Join the Community!
            </strong>
            Creating your account only takes a few minutes.
           </h3>
           <a class="sui-bttn sui-bttn-orange" data-modal-name="join" data-target="#" data-toggle="modal" href="http://community.spiceworks.com/join" onclick="GoogleAnalytics.trackEvent('Join','JoinNowButton','IT-Pro Clicked')">
            Join Now
           </a>
          </div>
          <div class="sui-opt-in root-post post " data-author-user-name="Kelly Fine (Spiceworks)" data-deleted="false" data-ip="" data-post-id="3135752" data-sticky="false" id="root_post">
           <p>
            <img src="http://butlerway.com/wp-content/uploads/2013/12/XP-Grave.jpg" title="Image: http://butlerway.com/wp-content/uploads/2013/12/XP-Grave.jpg"/>
            <em>
             Photo via
             <a href="http://butlerway.com/2013/12/windows-xp-rip-august-2001-april-2014.html" rel="nofollow" target="_blank" title="Link: http://butlerway.com/2013/12/windows-xp-rip-august-2001-april-2014.html">
              Butler Way
             </a>
            </em>
           </p>
           <p>
            When Bill Gates announced Windows XP, then called Whistler, at his Consumer Electronic Show keynote in January of 2001, eyebrows were raised. Steve Guggenheimer promised "we'll bring you the dependability of our highest end corporate desktop, and total dependability, to the home. But, at the same time, we're starting to move it in the direction of making it very consumer-oriented. Making it very friendly for the home user to use."
           </p>
           <p>
            Thirteen years later and just weeks away from XP's official end of life, we'll look back at 3 predictions on the future of XP, and determine whether or not they panned out.
           </p>
           <p>
            <span>
             <b>
              "In the past six years, Microsoft has released four significant upgrades to the Windows system.
             </b>
            </span>
            <b>
             Weary users who have finally figured out how to prevent their Windows 98 SE systems from crashing every time they try to print Web pages may be reluctant to tinker with computers that are working at last." -
            </b>
            J.D. Biersdorfer,
            <a href="http://www.nytimes.com/2001/04/19/technology/if-microsoft-builds-it-will-they-upgrade.html?smid=pl-share" rel="nofollow" title="Link: http://www.nytimes.com/2001/04/19/technology/if-microsoft-builds-it-will-they-upgrade.html?smid=pl-share">
             The New York Times
            </a>
            , April 2001
           </p>
           <p>
            Like Biersdorfer warned, just ten percent of Windows users upgraded to XP within the first year. However, by 2003, 42 percent of PC users were running the OS, and at its March 2007 peak, more than two thirds of all PCs were running XP, according to
            <a href="http://www.w3schools.com/browsers/browsers_os.asp" rel="nofollow" title="Link: http://www.w3schools.com/browsers/browsers_os.asp">
             W3's statistics
            </a>
            .
           </p>
           <b>
            "Back in 2001, Kennedy and InfoWorld were bashing XP and recommending that their readers stay with Windows 2000. Today, theyre bashing Vista and hawking their 'save XP' campaign. But judging by the progression that XP made in six years, all that the Windows Vista architecture needs is time and a hardware replacement cycle or two.

And we'll be able to read all about in InfoWorld's "Save Vista" campaign."-
           </b>
           Ed Bott,
           <a href="http://www.zdnet.com/blog/bott/remembering-windows-xps-early-days/382" rel="nofollow" title="Link: http://www.zdnet.com/blog/bott/remembering-windows-xps-early-days/382">
            ZDNet
           </a>
           , February 2008
           <div>
            <br/>
            <p>
             In this article, Bott predicted that the XP "love fest" would eventually be replaced with a Windows Vista "love fest." While Vista adoption did continue to rise throughout 2009, it peaked at 18% usage and declined relatively sharply, holding just 1.5% now, while XP currently maintains an 11 percent share despite its impending death.
            </p>
            <p>
             <b>
              "Nonetheless, it will be several years before developers and administrators can put Windows XP behind them. Its support is due to end on April 8th, 2014, and while that date is still some years off, it's all but inevitable that there will be organizations still using the operating system right up to the cut-off. Beyond the support cut-off, companies will still be allowed to downgrade their volume licenses, but they won't receive any further security patches if they do."
             </b>
             - Peter Bright,
             <a href="http://arstechnica.com/information-technology/2011/10/ten-years-of-windows-xp-how-longevity-became-a-curse/" rel="nofollow" title="Link: http://arstechnica.com/information-technology/2011/10/ten-years-of-windows-xp-how-longevity-became-a-curse/">
              ArsTechnica
             </a>
             , October 2011
            </p>
           </div>
           <div>
            Well, Bright nailed it. Most major tech publications have a guide to help aid users hoping to continue using the "zombie OS."
            <span>
             Check out more content about these hangers-on, including
            </span>

      // Handle server errors, don't enable button and show error
      form_error.text("Well, that didn't work as expected. Contact support if this happens again.").addClass('error').show();
      submit_button.attr('disabled', true);
      return;
    }

    if (jqXHR.status == 400) {
      // Validation failed
      var response_errors = $.parseJSON(jqXHR.responseText).errors;

      // Custom error handling for attachment errors
      if ("post_attachment" in response_errors) {
        var wysi_editor = $('#post_text').data('wysihtml5');

        // There was a validation error with the attachment, show error in editor
        wysi_editor.showError(response_errors.post_attachment);
        TopicReply.showForm();
        setTimeout(function () {
          wysi_editor.hideError();
        }, 5000);
      }
    }

    // Otherwise enable submit button and let them attempt again
    submit_button.attr('disabled', false);
  });

  var expandReplyOnFocus = function () {
    TopicReply.showForm(null, null, 500, function () {
      replyForm.addClass('expanded');
      // Remove the event handler so it only fires this first time
      if(editor) {
        editor.stopObserving("focus", expandReplyOnFocus);
      }
    });
  }

  if(editor) {
    editor.on("focus", expandReplyOnFocus);
  }
});
