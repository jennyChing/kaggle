       <h1 class="entry-title">
        zsh-dwim: Transformations for the Find Command
       </h1>
       <p class="meta">
        <time data-updated="true" datetime="2013-04-26T09:00:00-05:00" pubdate="">
         Apr 26
         <span>
          th
         </span>
         , 2013
        </time>
        |
        <a href="#disqus_thread">
         Comments
        </a>
       </p>
      </header>
      <div class="entry-content">
       <p>
        Not long after I got
        <a href="https://github.com/oknowton/zsh-dwim" title="zsh-dwim at gitHub">
         zsh-dwim
        </a>
        up and running, I went mining in my old Zsh and Bash history files looking for potential transformation candidates.  I thought I already implemented them all, but I was doing some spring cleaning and came across some of my notes.  It turns out that I was wrong.  There were still quite a few ideas in there that I havent implemented yet.
       </p>
       <p>
        One of the more interesting ideas in there was related to the
        <code>
         find
        </code>
        command.  Each time I write anything more complicated than the simplest
        <code>
         find
        </code>
        command line, I usually follow the same careful workflow:
       </p>
       <ul>
        <li>
         Run the command with no
         <code>
          -exec
         </code>
         option to make sure the correct files are chosen
        </li>
        <li>
         Add the
         <code>
          -exec
         </code>
         option, but
         <code>
          echo
         </code>
         the commands first
        </li>
        <li>
         Remove the
         <code>
          echo
         </code>
         , and run it one last time
        </li>
       </ul>
       <div class="embed-video-container" id="wN3qpVo9cV4 ">
        <iframe src="http://www.youtube.com/embed/wN3qpVo9cV4 ">
        </iframe>
       </div>
       <p>
        As you can see in the video, thats precisely what this transformation does.  If it isnt already there, hitting
        <code>
         control-u
        </code>
        will add the
        <code>
         -exec echo {} \;
        </code>
        to the command and put the cursor in front of the curly bracket.  If the
        <code>
         -exec
        </code>
        is already followed by an
        <code>
         echo
        </code>
        , then the
        <code>
         echo
        </code>
        is removed.
       </p>
       <p>
        For good measure, one more press of
        <code>
         control-u
        </code>
        will convert the
        <code>
         -exec
        </code>
        to a
        <code>
         -print0 | xargs -0
        </code>
        .
       </p>
       <h2>
        There is a tiny new feature in
        <a href="https://github.com/oknowton/zsh-dwim" title="zsh-dwim at gitHub">
         zsh-dwim
        </a>
       </h2>
       <p>
        I wanted to be able to automatically place the cursor directly before the curly bracket, but I didnt have an existing mechanism to do that.  The function that places the cursor after a successful transformation now sets the cursor position to the value of the
        <code>
         _dwim_cursor
        </code>
        variable, if it is defined.
       </p>
