       <h2 class="singleh2">
        <a href="http://www.detector-pro.com/2008/10/how-to-boot-ubuntu-804-from-usb-and.html" rel="bookmark" title="Permanent Link to How to boot Ubuntu from USB and make persistent install for saving sessions">
         How to boot Ubuntu from USB and make persistent install for saving sessions
        </a>
       </h2>
       <div class="written_line">
        <span class="written">
         By Viktor Ustijanoski |
        </span>
        <span class="posted">
         21 October 2008
        </span>
       </div>
       <br/>
       <!-- Google code -->
       <script type="text/javascript">
        &lt;!--
google_ad_client = "pub-6722919530728564";
/* nov-det-in-post  468x60, created 6/20/10 */

       </script>
       <script src="http://pagead2.googlesyndication.com/pagead/show_ads.js" type="text/javascript">

       <!-- End Google Code -->
       <br/>
       <br/>
       <div class="entry">
        <p>
         You can try Ubuntu with downloading the Live CD and booting up from a CD or USB, but every time you get the same session without saving your work. Do you like the idea that you can always carry your personal OS in your pocket everywhere? And also saving your sessions? Checking mail with your personal OS at your friends computer? Here is tutorial (http://www.pendrivelinux.com) how to do that with Live CD:
        </p>
        <p>
         <a href="http://1.bp.blogspot.com/_Xy2g3N79xiI/SP3LLxwOb1I/AAAAAAAAAMM/MKqc8rrEDDY/s1600-h/UbuntuUSB.jpg" onblur="try {parent.deselectBloggerImageGracefully();} catch(e) {}" rel="nofollow">
          <img alt="" border="0" id="BLOGGER_PHOTO_ID_5259583343033216850" src="http://1.bp.blogspot.com/_Xy2g3N79xiI/SP3LLxwOb1I/AAAAAAAAAMM/MKqc8rrEDDY/s400/UbuntuUSB.jpg" style="margin: 0px auto 10px; display: block; text-align: center; cursor: pointer;"/>
         </a>
         <br/>
         You need Ubuntu Live CD and USB 1GB or more!
        </p>
        <ol>
         <li>
          First download Live CD from Ubuntu official site, or just click
          <a href="http://www.ubuntu.com/getubuntu/download" rel="nofollow">
           here
          </a>
          , and burn it to empty CD.
         </li>
         <li>
          Now boot up from a Ubuntu CD
         </li>
         <li>
          When Ubuntu is started insert the USB
         </li>
         <li>
          On main Ubuntu menu
          <span style="font-weight: bold;">
           click Applications/Accessories/Terminal
          </span>
          , and type
          <span style="font-weight: bold;">
           sudo su
          </span>
         </li>
         <li>
          Next type
          <span style="font-weight: bold;">
           fdisk -l
          </span>
          <span>
           to list available drives/partitions (note which device is your flash drive Example: /dev/sdb). Throughout this tutorial, replace all instances of
           <span style="color: rgb(255, 0, 0);">
            x
           </span>
           with your flash drive letter. For example, if your flash drive is sdb, replace
           <span style="color: rgb(255, 0, 0);">
            x
           </span>
           with
           <span>
            b
           </span>
           .
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            umount /dev/sd
           </span>
           <span style="color: rgb(255, 0, 0); font-weight: bold;">
            x
           </span>
           <span style="font-weight: bold;">
            1
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            fdisk /dev/sd
           </span>
           <span style="color: rgb(255, 0, 0); font-weight: bold;">
            x
           </span>
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            p
           </span>
           to show the existing partition
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            d
           </span>
           to delete it
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            p
           </span>
           again to show any remaining partitions
           <span>
            (
            <span style="font-weight: bold;">
             if partitions exist, repeat the previous step
            </span>
            )
           </span>
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            n
           </span>
           to make a new partition
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            p
           </span>
           for primary partition
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            1
           </span>
           to make this the first partition
          </span>
         </li>
         <li>
          <span>
           hit
           <span style="font-weight: bold;">
            enter
           </span>
           to use the default 1st cylinder
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            +750M
           </span>
           to set the partition size
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            a
           </span>
           to make this partition active
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            1
           </span>
           to select partition 1
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            t
           </span>
           to change the partition filesystem
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            6
           </span>
           to select the fat16 file system
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            n
           </span>
           to make another new partition
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            p
           </span>
           for primary partition
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            2
           </span>
           to make this the second partition
          </span>
         </li>
         <li>
          <span>
           hit
           <span>
            <span style="font-weight: bold;">
             enter
            </span>
           </span>
           to use the default cylinder
          </span>
         </li>
         <li>
          <span>
           hit
           <span style="font-weight: bold;">
            enter again
           </span>
           to use the default last cylinder
          </span>
         </li>
         <li>
          <span>
           type
           <span style="font-weight: bold;">
            w
           </span>
           to write the new partition table
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            umount /dev/sd
            <span style="color: rgb(255, 0, 0);">
             x
            </span>
            1
           </span>
           <span style="font-weight: bold;">
            to unmount the partition
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            mkfs.vfat -F 16 -n ubuntu8 /dev/sd
            <span style="color: rgb(255, 0, 0);">
             x
            </span>
            1
           </span>
           to format the first partition
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            umount /dev/sd
            <span style="color: rgb(255, 0, 0);">
             x
            </span>
            2
           </span>
           to ensure the partition is unmounted
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            mkfs.ext2 -b 4096 -L casper-rw /dev/sd
            <span style="color: rgb(255, 0, 0);">
             x
            </span>
            2
           </span>
           to format the second partition
          </span>
         </li>
         <li>
          <span>
           <span style="font-weight: bold;">
            Remove and re-insert your flash drive
           </span>
           (if prompted that a new medium has been detected,
           <span>
            select to open in a new window and click ok
           </span>
           )
          </span>
         </li>
         <li>
          <span>
           In terminal, type
           <span style="font-weight: bold;">
            sudo apt-get install syslinux mtools
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span>
            <span style="font-weight: bold;">
             syslinux -sf /dev/sd
            </span>
            <span style="color: rgb(255, 0, 0); font-weight: bold;">
             x
            </span>
            <span style="font-weight: bold;">
             1
            </span>
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span>
            <span style="font-weight: bold;">
             cd /cdrom
            </span>
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            cp -rfv casper dists install pics pool preseed .disk isolinux/* md5sum.txt README.diskdefines install/mt86plus /media/ubuntu8
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            cd /media/ubuntu8
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span style="font-weight: bold;">
            wget pendrivelinux.com/downloads/u8/syslinux.cfg
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span>
            <span style="font-weight: bold;">
             cd casper
            </span>
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span>
            <span style="font-weight: bold;">
             rm initrd.gz
            </span>
           </span>
          </span>
         </li>
         <li>
          <span>
           Type
           <span>
            <span style="font-weight: bold;">
             wget pendrivelinux.com/downloads/u8/initrd.gz
            </span>
           </span>
          </span>
         </li>
         <li>
          <span>
           <span style="font-weight: bold;">
            Reboot your computer and set your system BIOS boot priority to boot from the USB stick
           </span>
           <span style="font-weight: bold;">
            .
           </span>
          </span>
         </li>
        </ol>
        <p>
         <span>
          If everything has gone as it should, you should now be able to boot Ubuntu from the USB flash device and it should save your changes, restoring them on boot.
         </span>
        </p>
        <p>
         <span style="color: rgb(255, 0, 0);">
          Notes
         </span>
         : If your having trouble getting Ubuntu to boot, your memory stick may have a corrupted MBR. To repair the MBR of your USB device, at the terminal type
         <span style="font-weight: bold;">
          sudo apt-get install lilo
         </span>
         then type
         <span style="font-weight: bold;">
          lilo -M /dev/sd
          <span style="color: rgb(255, 0, 0);">
           x
          </span>
         </span>
         (replacing x with the letter of your flash device).
        </p>
        <p>
         The
         <span style="font-weight: bold;">
          good
         </span>
         thing is that you can boot up your OS from USB stick and
         <span>
          learn a lot of things for Linux, and then decide to install it on your hard drive.
         </span>
         <br/>
         <span>
          <br/>
          The
          <span style="font-weight: bold;">
           negative
          </span>
          side is the speed (is a little slowly) and protection  becouse is open session everyone who is using your usb can enter in your computer  for example.
         </span>
        </p>

           <p>
            Thanks. This one is related with yours:
           </p>
           <p>
            <a href="http://ubuntuforums.org/showthread.php?t=1734965" rel="nofollow">
             http://ubuntuforums.org/showthread.php?t=1734965
            </a>
            :
           </p>
           <p>
            ==== Safe &amp; tidy live USB, with personal files, persistent or fresh boot, no CD, any BIOS ====
           </p>
           <p>
            This tutorial shows how to create a bootable USB flash drive with both Ubuntu and personal data in it.
           </p>
           <p>
            Features:
            <br/>
            + The operating system files are kept appart from the personal ones
            <br/>
            + On booting you can choose to do it in a persistent or fresh way (loading or not saved configuration or session)
            <br/>
            + No CD needs to be burnt (only it the BIOS is old )
            <br/>
            + If the BIOS of your computer cannot boot from USB (for example because its old) a solution is given so it can (see at bottom: Appendix A))
            <br/>
            + If you dont have a (free) pendrive an alternative is given (see at bottom: Appendix B))
           </p>
           <p>
            Advantages:
            <br/>
            + We have 2 partitions:
            <br/>
            ++ The first partition is for our personal files or archives (songs, photos, videos, documents, )
            <br/>
            ++ The second partition is for Ubuntu (the operating system), including the persistence file, casper-rw
            <br/>
            + This way the pendrive can be used for 3 things:
            <br/>
            ++ Boot Ubuntu in any computer (in Live mode)
            <br/>
            ++ Install Ubuntu in any computer
            <br/>
            ++ Use our personal files, both from Ubuntu Live (for example the one in the same pendrive), an installed Ubuntu, other Linux distro, Windows, 
            <br/>
            + As our personal files are appart from the operating system ones:
            <br/>
            ++ Our personal data is easily accessed and managed by any operating system (for example, booting the Ubuntu Live of the same USB flash drive you dont need to use sudo neither search within the file system; and you can edit, delete,  them)
            <br/>
            ++ The operating system files are safe (for example Windows XP cannot access them: it only can access the first partition, where the personal data is. If everything were in a single partition, with any operating system, all the files would be accessable and for example when trying to delete a personal file we could delete by accident an operating system file).
            <br/>
            + Its useful to have the option to boot the live USB in fresh mode:
            <br/>
            ++ The booting is faster (the persistence file is not used)
            <br/>
            ++ We can use it to boot the live USB in a computer different to ours, which probably will have a different hardware (even if we can boot in persistent mode on it -I think that normally yes- after finishing the external computers configurations would be saved and when booting back again on our computer probably some of the configurations would be different to the ones we like to use)
            <br/>
            + As we dont burn any CD (only it the BIOS is old ) we help to keep or room or office tidy, we save money, pollute less, 
            <br/>
            + We can have a live USB even if we dont have a pendrive but yes a memory card of a photo camera and an external card reader
           </p>
           <p>
            Howto:
            <br/>
            (If you have Ubuntu installed in your computer or in another live USB jump to B))
            <br/>
            A) You have Windows installed in your computer but not Ubuntu
            <br/>
            0. Boot Windows
            <br/>
            1. Download
            <a href="http://www.pendrivelinux.com/downloads/Universal-USB-Installer/Universal-USB-Installer-1.8.4.1.exe" rel="nofollow">
             http://www.pendrivelinux.com/downloads/Universal-USB-Installer/Universal-USB-Installer-1.8.4.1.exe
            </a>
            <br/>
            2. Insert your USB flash drive
            <br/>
            3. Copy the files you want to keep to another drive
            <br/>
            4. Run the program downloaded in 1. and follow the instructions of
            <a href="http://www.pendrivelinux.com/universal-usb-installer-easy-as-1-2-3/" rel="nofollow">
             http://www.pendrivelinux.com/universal-usb-installer-easy-as-1-2-3/
            </a>
            (by now dont use the persistence option)
            <br/>
            (If you have another USB flash drive jump to B))
            <br/>
            5. Reboot the computer and press the key stated by the BIOS to go to the boot menu (F12, F11, , then use the arrows to select the USB flash drive and then press Enter). If no boot menu, go to the BIOS main menu (with F2 or Del ), then to boot options 
            <br/>
            6. Install Ubuntu in your computer. More info in
            <a href="https://help.ubuntu.com/community/Installation/FromUSBStickQuick" rel="nofollow">
             https://help.ubuntu.com/community/Installation/FromUSBStickQuick
            </a>
           </p>
           <p>
            B) You have Ubuntu installed in your computer (or in another live USB)
            <br/>
            Ill do it with this example:
            <br/>
            + Desktop -or laptop- computer, not netbook
            <br/>
            + USB flash drive of 4 GB
            <br/>
            + Personal files will take up more than half of the memory or space
            <br/>
            + Ubuntu version for the USB flash drive: 10.04, 32-bit (for 32-bit or 64-bit computers)
            <br/>
            0. Boot Ubuntu (the one installed in your computers hard drive or the one in an extra live USB)
            <br/>
            1. Download the required ISO file:
            <a href="http://www.ubuntu.com/desktop/get-ubuntu/download" rel="nofollow">
             http://www.ubuntu.com/desktop/get-ubuntu/download
            </a>
            (and choose and click) or
            <a href="ftp://swtsrv.informatik.uni-mannheim.de/pub/linux/distributions/ubuntu-release/lucid/ubuntu-10.04.2-desktop-i386.iso" rel="nofollow">
             ftp://swtsrv.informatik.uni-mannheim.de/pub/linux/distributions/ubuntu-release/lucid/ubuntu-10.04.2-desktop-i386.iso
            </a>
            (or from other country )
            <br/>
            2. Plug the USB flash drive
            <br/>
            3. Copy the files you want to keep to another drive
            <br/>
            4. Hold Alt key and press F2 . Type gnome-terminal and press Enter
            <br/>
            5. Type sudo bash and press Enter. Enter your password
            <br/>
            6. Type ls /dev/disk/by-id/*usb* -l and press Enter . At the end of the first line there should be something like sdb or sdc or sdd  In my case sdb
            <br/>
            7. Type fdisk -l and press Enter . After /dev/sdb I see /dev/sdb1 (there is only 1 partition; no divisions in my USB flash drive)
            <br/>
            8. Type umount /dev/sdb1 and press Enter
            <br/>
            9. Type fdisk /dev/sdb and press Enter
            <br/>
            10. Type m and press Enter to see the options
            <br/>
            11. Type p and press Enter to see the partitions (in my case /dev/sdb1 )
            <br/>
            12. Type d and press Enter (fdisk selects the only partition I have)
            <br/>
            13. Type p and press Enter (now there is no partition so no /dev/sdb1 is shown)
            <br/>
            14. Type n and press Enter then p and Enter then 1 (number of partition) and Enter then Enter (to use the default beginning: 1) then 600 (something more than the half) and Enter
            <br/>
            15. Type p and press Enter. I see /dev/sdb1 with Id of 83 (Linux file system)
            <br/>
            16. We want change it to FAT32: Type l and press Enter . We see that c is our option (W95 FAT32 (LBA) file system)
            <br/>
            17. We change it: Type t and press Enter (it selects the only partition up to now). Type c and press Enter
            <br/>
            18. Type p and press Enter . We see Id of c (W95 FAT32 (LBA) file system)
            <br/>
            19. Type n and press Enter then p and Enter then 2 and Enter then Enter (to use the default: just after first partition) then Enter (to use the default: all the free space, to the end)
            <br/>
            20. Type p and press Enter . We see also /dev/sdb2 . With Id of 83
            <br/>
            21. Type t and press Enter then 2 and Enter then c and Enter
            <br/>
            22. Type p and press Enter . We see Id of c bor both
            <br/>
            23. Type w and press Enter. Changes are written on disk
            <br/>
            24. Type fdisk -l and press Enter . Now we see the 2 partitions.
            <br/>
            25. Now we format the first one: type mkfs.vfat /dev/sdb1 and press Enter
            <br/>
            26. Now we format the second one: type mkfs.vfat /dev/sdb2 and press Enter
            <br/>
            27. Close the terminal or console
            <br/>
            28. Hold Alt key and press F2 . Type gnome-control-center and press Enter
            <br/>
            29. Click on USB Startup Disk Creator (under Hardware)
            <br/>
            30. Click on the button named Other.. (up-right) and choose and open the downloaded ISO file
            <br/>
            31. Under Disk to use click on /dev/sdb2
            <br/>
            32. Down, select Stored in reserved extra space
            <br/>
            33. Move the How much button almost to the limit. In my case the limit is 876.0 MB but I put it in 851.0 (just in case I need to modify some file of booting, )
            <br/>
            34. Click on the button Make Startup Disk
            <br/>
            35. When it finishes click on Exit
            <br/>
            36. Edit the file syslinux.cfg (inside the Live USBs syslinux folder): delete all the content and copy there this code:
            <br/>
            prompt 1
            <br/>
            timeout 50
            <br/>
            default 1
            <br/>
            say -
            <br/>
            say Enter the number of the desired option
            <br/>
            say 
            <br/>
            say -
            <br/>
            label 1
            <br/>
            say 1 Try Ubuntu without installing
            <br/>
            kernel /casper/vmlinuz
            <br/>
            append noprompt cdrom-detect/try-usb=true persistent file=/cdrom/preseed/ubuntu.seed boot=casper initrd=/casper/initrd.lz quiet splash 
            <br/>
            label 2
            <br/>
            say 2 Try Ubuntu without installing and fresh
            <br/>
            kernel /casper/vmlinuz
            <br/>
            append noprompt cdrom-detect/try-usb=true file=/cdrom/preseed/ubuntu.seed boot=casper initrd=/casper/initrd.lz quiet splash 
            <br/>
            label 3
            <br/>
            say 3 Install Ubuntu
            <br/>
            kernel /casper/vmlinuz
            <br/>
            append noprompt cdrom-detect/try-usb=true persistent file=/cdrom/preseed/ubuntu.seed boot=casper only-ubiquity initrd=/casper/initrd.lz quiet splash 
            <br/>
            label 4
            <br/>
            say 4 Check disc for defects
            <br/>
            kernel /casper/vmlinuz
            <br/>
            append noprompt boot=casper integrity-check initrd=/casper/initrd.lz quiet splash 
            <br/>
            label 5
            <br/>
            say 5 Test memory
            <br/>
            kernel /install/mt86plus
            <br/>
            label 6
            <br/>
            say 6 Boot from first hard disk
            <br/>
            localboot 080
           </p>
           <p>
            FINISHED !!!
           </p>
           <p>
            Now if we open the terminal and we use fdisk -l we see that there is a * for Boot in /dev/sdb2 . So the second partition is bootable (where Ubuntu GNU/Linux, the operating system, is located)
           </p>
           <p>
            To see this working reboot the computer and press the key stated by the BIOS to go to the boot menu (F12, F11, , then use the arrows to select the USB flash drive and then press Enter). If no boot menu, go to the BIOS main menu (with F2 or Del ), then to boot options .
           </p>
           <p>
            The first time in the middle of the boot we have to choose the language and click on Try Ubuntu v 10.04.2 LTS
           </p>
           <p>
            If we click on Places menu and then on 2.4 GB Filesystem the first partition is mounted and the Nautilus File Browser opens there. We can copy, create, delete, edit, open  our personal files: songs, photos, videos, documents 
           </p>
           <p>
            ENJOY !!!
           </p>
           <p>
            Appendix A) If the BIOS of your computer cannot boot from USB you can download
            <a href="http://download.plop.at/files/bootmngr/plpbt-5.0.11-2.zip" rel="nofollow">
             http://download.plop.at/files/bootmngr/plpbt-5.0.11-2.zip
            </a>
            , uncompress the file and burn a CD (a Business card CD is enough, or the easier to find Mini CDs) from the image (file) plpbt.iso (for example the one in the root folder). You insert the CD and the pendrive with Ubuntu in the computer, reboot or switch it on, and a menu appears where you select USB, so the computer boots from the USB flash drive. It may happen that the BIOS were configured to look the hard drive before the CD or DVD drive: in this case you just need to change the boot order of the BIOS. To do this, just after rebooting or turning on the computer, press the key stated to go to the BIOS main menu (F2 or Del ), then to boot options  The newer BIOSes also have a boot menu, accessable with F12, F11,  But these modern BIOSes normally allow to boot via USB. (The Plop Boot Manager allows to boot from any USB flash drive, no matter the operating system in it)
           </p>
           <p>
            Appendix B) If you dont have an USB flash drive you can have Ubuntu in the memory card of your photo camera (SD, xD, ). And the memory card can hold both the operating system, your personal data and the photos you take with the camera. To boot Ubuntu from this card you just need to insert it in an external reader (for example one of those that have several sockets for different formats) connected via USB to the computer (if you put the card in the internal reader of the computer probably you wont be able to boot Ubuntu -if someone can do it, please, explain it-)
           </p>
