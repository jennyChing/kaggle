     <h3>
      <a class="anchor" href="#QIM" id="QIM" name="QIM">
       Q
      </a>
      IM
     </h3>
     <p>
      QIM is a Hanyu Pinyin input method with support for both Simplified and Traditional Chinese.
     </p>
     <p>
      QIM is simply the best Pinyin input method available for Mac OS X. Its underlying data structure supports the powerful
      <a href="http://www.bjhm.com.cn/software/znsr/znsr_1.htm">
       Heima shenpin
      </a>
      [] technology. Currently, this provides two steps forward. First is intelligent sentence parsingthe ability to type long sentences as input strings while maintaining a high rate of accuracy in transforming them into hanzi. Second is a very large and highly efficient database structureit contains more than 22 million entries across the entire spectrum of professional usage. The
      <a href="http://www.sogou.com/pinyin/">
       Sogou
      </a>
      [] search engine's dictionary has also been included, providing a frequently-updated database of contemporary terms.
     </p>
     <p>
      Other useful features include  the ability to limit the display of candidates to standard hanzi in the Xinhua Dictionary [], optional support for filtering candidates by Pinyin tone, and support for all keyboard layouts, like French, German, etc.
     </p>
     <p>
      You can also inspect and edit the User Dictionary with import and export capabilities via the included QIM Dictionary Manager (QDM) application.
     </p>
     <p>
      Free. If you can afford to do so, we strongly recommend you make a donation to support QIM's ongoing development. Localized for both Chinese and English.
     </p>
     <p>
      <a href="http://www.macqim.com/wordpress/">
       http://www.macqim.com/wordpress/
      </a>
     </p>
     <h3>
      <a class="anchor" href="#DerYi" id="DerYi" name="DerYi">
       DerYi
      </a>
     </h3>
     <p>
      DerYi [, originally  as Macintosh shareware] is based in Taiwan. First developed in 1994, its main input modes are Zhuyin, Pinyin, and Cangjie, along with user-defined phrase input. Like
      <a href="old/tcim_x4.html#Hanin">
       Hanin
      </a>
      , it uses a rolling approach to converting input strings as you type, but DerYi gives the user more control over the process.
     </p>
     <p>
      Version 4xxx (OS X 10.4 and above) comes with a Unicode option, along with Big5 and GB.
     </p>
     <p>
      <a href="http://www.smartinside.com.tw/">
       http://www.smartinside.com.tw/
      </a>
     </p>

      MacKEY
     </h3>
     <p>
      MacKEY's built-in input method can convert long strings of Pinyin text, with tones optional. For best results, you should parse the Pinyin into words and phrases as you type. See
      <a href="language.html#MacKEY">
       Language Tools
      </a>
      .
     </p>
     <h3>
      Wenlin
     </h3>
     <p>
      Wenlin's built-in input method allows you to input any character or compound (word or phrase) in the dictionary, with tones optional. See
      <a href="language.html#wenlin">
       Language Tools
      </a>
      .
     </p>

      <a href="http://rescomp.stanford.edu/~domingo2/Chinese.html">

     <h2>
      <a class="anchor" href="#frameworks" id="frameworks" name="frameworks">
       IM Frameworks
      </a>
     </h2>
     <p>
      Input-method frameworks generate input methods from data tables. In the past, these source files had to be converted into a format the framework could handle, like a binary data file with the ".dat" extension. More recent frameworks, like Mac OS X 10.5+ and OpenVanilla (see below), can handle plain-text data tables directly.
     </p>
     <h3>
      <a class="anchor" href="#cin" id="cin" name="cin">
       CIN plug-ins
      </a>
     </h3>
     <p>
      In OS X 10.5 and above, you can create a plain-text source file  using the CIN plug-in data format described below, change the file extension to ".cin", and then place it in the /Library/Input Methods folder or your Home ~/Library/Input Methods folder.
     </p>
     <p>
      For samples, see:
      <a href="http://openvanilla.googlecode.com/svn/trunk/Modules/SharedData/">
       http://openvanilla.googlecode.com/svn/trunk/Modules/SharedData/
      </a>
     </p>
     <ul>
      <li>
       The plain-text source file must be in
       <a href="character_sets.html#Unicode">
        UTF-8
       </a>
       encoding.
       <ul>
        <li>
         Windows-style
         <code>
          CR+LF
         </code>
         newlines are not allowed.
        </li>
        <li>
         Mac OS X and UNIX-style
         <code>
          /n
         </code>
         newlines are required.
        </li>
       </ul>
      </li>
      <li>
       For comments, use # at the start of a line and it will be ignored.
       <ul>
        <li>
         Comments are not allowed in the
         <code>
          %chardef
         </code>
         block.
        </li>
       </ul>
      </li>
      <li>
       Blank lines are ignored.
      </li>
      <li>
       <code>
        %gen_inp
       </code>
       <ul>
        <li>
         Required. Tells the input-method framework what to do.
        </li>
       </ul>
      </li>
      <li>
       <code>
        %cname
       </code>
       <ul>
        <li>
         Determines the name that will appear in the Input menu.
        </li>
        <li>
         Only
         <code>
          %cname
         </code>
         is valid in OS X 10.5.
        </li>
        <li>
         <code>
          %cname
         </code>
         ,
         <code>
          %ename
         </code>
         ,
         <code>
          %tcname
         </code>
         , and
         <code>
          %scname
         </code>
         are all valid in
         <a href="#OpenVanilla">
          OpenVanilla
         </a>
         .
        </li>
       </ul>
      </li>
      <li>
       <code>
        %encoding
       </code>
       <ul>
        <li>
         <code>
          UTF-8
         </code>
         is required for both OS X 10.5 and OpenVanilla.
        </li>
       </ul>
      </li>
      <li>
       <code>
        %selkey
       </code>
       <ul>
        <li>
         Defines keys used to select candidates.
        </li>
        <li>
         In Mac OS X 10.5, this setting is ignored and defaults to 123456789.
        </li>
       </ul>
      </li>
      <li>
       <code>
        %endkey
       </code>
       <ul>
        <li>
         <code>
          %endkey
         </code>
         is designed to provide punctuation and other symbols in input methods.
        </li>
        <li>
         Keys declared here also need to be defined in both the
         <code>
          %keyname
         </code>
         and
         <code>
          %chardef
         </code>
         blocks.
        </li>
        <li>
         See OpenVanilla's
         <a href="http://openvanilla.googlecode.com/svn/trunk/Modules/SharedData/">
          bpmf.cin
         </a>
         [] and
         <a href="http://openvanilla.googlecode.com/svn/trunk/Modules/SharedData/">
          cj.cin
         </a>
         [] for examples of how this works. Also
         <a href="romanization.html#Biaoyin">
          Biaoyin
         </a>
         .
        </li>
       </ul>
      </li>
      <li>
       <code>
        %keyname begin
       </code>
       <ul>
        <li>
         Required. Key definitions begin on the next line.
        </li>
        <li>
         These definitions map keys typed to characters displayed in the inline/input window. It is used mostly to display radicals in radical-based input methods like CangJie, Dayi, and Wubi, but it can also be used in other ways.
        </li>
       </ul>
      </li>
      <li>
       <code>
        %keyname end
       </code>
       <ul>
        <li>
         Required. Key defintions end on the previous line.
        </li>
       </ul>
      </li>
      <li>
       <code>
        %chardef begin
       </code>
       <ul>
        <li>
         Required. The data table begins on the next line.
        </li>
        <li>
         Unlike the Apple format, the CIN format does not use a delimiter. Instead, each entry is listed on a new line, with a space (or tab) between the input string and the output string.
        </li>
        <li>
         Input strings are not case-sensitive.
        </li>
       </ul>
      </li>
      <li>
       <code>
        %chardef end
       </code>
       <ul>
        <li>
         Required. The data table ends on the previous line.
        </li>
       </ul>
      </li>
     </ul>
     <h3>
      <a class="anchor" href="#inputplugin" id="inputplugin" name="inputplugin">
       Apple plug-ins
      </a>
     </h3>
     <p>
      Plain-text source files in  the Apple plug-in data format described below can be used with the Input Method Plug-in Converter utility in
      <a href="old/os9.html#utilities">
       OS 9
      </a>
      , and OS X
      <a href="old/osx3.html#utilities">
       10.3
      </a>
      and
      <a href="old/osx4.html#utilities">
       10.4
      </a>
      . In OS X 10.5 and above, you simply change the file extension to ".inputplugin" and then place the file in the /Library/Input Methods folder or your Home ~/Library/Input Methods folder.
     </p>
     <p>
      For samples, see the Apple-format Cantonese plug-ins listed
      <a href="#Cantonese">
       above
      </a>
      .
     </p>
     <ul>
      <li>
       In OS X 10.3 and above, the plain-text source file must be in
       <a href="character_sets.html#Unicode">
        UTF-16
       </a>
       encoding.
      </li>
      <li>
       In  OS 9, it must be in
       <a href="character_sets.html#GB">
        GB 2312
       </a>
       (Simplified Chinese only) or
       <a href="character_sets.html#Big5">
        Big Five
       </a>
       (Traditional Chinese only) encoding.
      </li>
      <li>
       For comments, use # at the start of a line and it will be ignored.
      </li>
      <li>
       Blank lines are ignored.
      </li>
      <li>
       Don't leave fields blank. Either omit the line (if it is not required and the default meets your needs) or define a value.
      </li>
      <li>
       <code>
        METHOD:
       </code>
       <ul>
        <li>
         <code>
          TABLE
         </code>
        </li>
        <li>
         Required. Defines the format of the data.
        </li>
       </ul>
      </li>
      <li>
       <code>
        ENCODE:
       </code>
       <ul>
        <li>
         <code>
          SC
         </code>
         [Simplified Chinese]
        </li>
        <li>
         <code>
          TC
         </code>
         [Traditional Chinese]
        </li>
        <li>
         <code>
          Unicode
         </code>
         [Unicode,
         <code>
          UTF-16
         </code>
         also valid]
        </li>
        <li>
         Required. Determines the character set and script/framework used in the plug-in.
        </li>
        <li>
         Legacy values, like
         <code>
          GB2312
         </code>
         and
         <code>
          BIG5
         </code>
         , are depreciated.
        </li>
       </ul>
      </li>
      <li>
       <code>
        PROMPT:
       </code>
       <ul>
        <li>
         Required. Determines the name that will appear in the Input menu.
        </li>
        <li>
         32 characters maximum.
        </li>
       </ul>
      </li>
      <li>
       <code>
        VERSION:
       </code>
       <ul>
        <li>
         For the version number of the data.
        </li>
        <li>
         8 characters maximum
        </li>
       </ul>
      </li>
      <li>
       <code>
        DELIMITER:
       </code>
       <ul>
        <li>
         Required. Determines the character used to delimit multiple output strings for the same input string.
        </li>
       </ul>
      </li>
      <li>
       <code>
        MAXINPUTCODE:
       </code>
       <ul>
        <li>
         Limits the length of input strings.
        </li>
        <li>
         In OS 9 to OS X 10.4, the maximum length of input strings is seven (the default).
        </li>
        <li>
         In OS X 10.5, there is no maximum length of input strings.
        </li>
       </ul>
      </li>
      <li>
       <code>
        VALIDINPUTKEY:
       </code>
       <ul>
        <li>
         Required. Determines the characters that are valid in input strings.
        </li>
        <li>
         Input strings are not case sensitive.
        </li>
       </ul>
      </li>
      <li>
       <code>
        BEGINCHARACTER
       </code>
       <ul>
        <li>
         Required. The body of the data table begins on the next line.
        </li>
        <li>
         For lines of data, the tab key is the delimiter between the input string and the output string(s).
        </li>
        <li>
         Output strings can vary in length, up to 100 characters. [?]
        </li>
       </ul>
      </li>
      <li>
       <code>
        ENDCHARACTER
       </code>
       <ul>
        <li>
         Required. The data table ends on the previous line.
        </li>
       </ul>
      </li>
     </ul>
     <h3>
      <a class="anchor" href="#OpenVanilla" id="OpenVanilla" name="OpenVanilla">
       OpenVanilla
      </a>
     </h3>
     <p>
      OpenVanilla is a free, open-source Unicode-based Chinese input method framework based in Taiwan. Supports data tables in the
      <a href="#cin">
       CIN
      </a>
      format for its generic input module. A variety of tables are available, including data for CangJie [], Dayi [], Jianyi [], Wubizixing [], and Four-corner [], among others. The tables can easily be edited, and other tables can be adapted for use in OpenVanilla. For example, tables from the popular but proprietary
      <a href="http://www.liu.com.tw">
       Boshiamy
      </a>
      [] input method can be installed. See
      <a href="http://docs.google.com/View?docid=ah6d8th954vw_201fd5dkx">
       here
      </a>
      for a more detailed discussion of this topic (in Chinese).
     </p>
     <p>
      OpenVanilla also includes more sophisticated modules, like the open-source, intelligent
      <a href="http://docs.google.com/View?docid=ah6d8th954vw_219fx6c6f">
       Chewing
      </a>
      [] input method. It is similar to
      <a href="old/tcim_x4.html">
       Hanin
      </a>
      , but is Zhuyin-only and supports an even wider array of keyboards. The well-known
      <a href="http://docs.google.com/View?docid=ah6d8th954vw_218c7c5wn">
       Array
      </a>
      [] input method comes with a dedicated input module to use instead of the generic one.
      <a href="http://docs.google.com/View?docid=ah6d8th954vw_220f63v82">
       Taiwanese
      </a>
      (POJ) and
      <a href="http://docs.google.com/View?docid=ah6d8th954vw_224c57x9t">
       Tibetan
      </a>
      (Sambhota and TCC) also have their own modules, among others.
     </p>
     <p>
      OS X 10.4 and above.
     </p>
     <p>
      <a href="http://openvanilla.org/">
       http://openvanilla.org/
      </a>
     </p>
