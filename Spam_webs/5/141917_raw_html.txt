                    <h3>
                     Building an Internet radio recorder with VLC Player
                    </h3>
                    <div class="xar-article-details">
                     By
                     <a href="http://www.nothickmanuals.info">
                      Dmitri Popov
                     </a>
                     on
March 30, 2005 (9:00:00 AM)
                    </div>
                    <p style="text-align: right">
                     <!-- ADDTHIS BUTTON BEGIN -->
                     <script type="text/javascript">

                      </b>

                     <a href="http://archive09.linux.com/articles/43721?theme=print" style="text-decoration: none" title="Printer friendly page">
                      <img alt="" src="/themes/linux/images/tango-icon-theme/16x16/devices/printer.png" style="vertical-align: middle;"/>
                      <b style="text-decoration: underline; font-size: 85%">
                       Print
                      </b>
                     </a>
                     <a href="http://archive09.linux.com/articles/43721#commentthis" style="text-decoration: none" title="Comment on this article">

                     </a>
                    </p>
                    <div class="xar-clearleft">
                     It's nice to be able to listen to a net radio station directly on your computer. But wouldn't it be great if you could record it on your hard disk? Better yet, what if you could schedule recordings, so you don't miss your favourite programmes? Using
                     <a href="http://www.videolan.org/vlc/">
                      VLC Player
                     </a>
                     and a simple batch file you can turn your computer into a radio recorder.
                    </div>
                    <div class="xar-align-left" id="featurecontent">
                     <p>
                      VLC Player is a core application of the VideoLAN project. This versatile software can handle a wealth of video and audio formats and stream content over a local network. Moreover, VLC Player is able to play net streams and record them to the hard disk, which makes it a perfect tool for our project.
                     </p>
                     <p>
                      Listening to a net radio (or any net stream for that matter) using VLC Player is not that difficult. However, to be able to record it using VLC Player, you have to make sure that you have a direct link to the stream. Normally the links on radio stations' Web sites refer to .pls playlist files, which contain the stream information. Like any other player, VLC Player can use the playlist files to play the stream, but not to record it. To solve this problem you have to extract a direct link to the stream from the .pls file.
                     </p>
                     <p>
                      Let's say you want to record the Virgin Radio Classic Rock stream. Point your browser to
                      <a href="http://www.virginradio.co.uk/thestation/listen/streams.html">
                       the page containing the streaming links
                      </a>
                      . If you use the Firefox browser, right-click on the desired link (an MP3 modem or broadband is always the safest bet), and select Save Link As to save the playlist file.
                     </p>
                     <p>
                      Open the saved file in a text editor and locate the line containing the direct link to the stream. In this case it is:
                     </p>
                     <p>
                      <code>
                       File2=http://vruk.sc.llnwd.net:12265
                      </code>
                     </p>
                     <p>
                      Check that the link works properly, then copy it into the Clipboard. Launch VLC Player and choose File &gt; Open Network Stream. Select the HTTP/FTP/MMS option, and enter the link into the URL field. To avoid hiccups during playback, you might want to enable caching. To do this, tick the Caching check box and type a value in milliseconds into the Caching field (5,000 works fine in most cases). Finally, click OK. If the link works, VLC Player will play the stream.
                     </p>
                     <p>
                      The next step is to configure VLC player to record the current stream. Once again, choose File &gt; Open Network Stream, select the HTTP/FTP/MMS option, and enter the link into the URL field. This time, though, tick the Stream output check box, and press the Setting button. This will open the Stream Output dialogue window. If you want to listen to the stream while it's being recorded, tick the Play locally check box.
                     </p>
                     <p>
                      Use the Browser button next to the Filename field to give the recorded stream file a name and choose where it will be stored. For Encapsulation Method, select Raw. In the Transcoding options section, tick the Audio codec check box and select MP3 from the list. Select a bitrate from the Bitrate list (128 is a good compromise between audio quality and file size). Since you are going to record a stereo stream, select 2 from the Channels list. Finally, press OK to save the settings and close the window.
                     </p>
                     <p>
                      Press OK to start recording. To make sure that everything works properly, close VLC Player and try to play the recorded file on your favourite MP3 player.
                     </p>
                     <p>
                      Although VLC Player can now play and record the net stream, there are still a couple of things you can do to improve the way it works. First of all, if you quit VLC Player, you'll lose all your settings. Luckily, you can launch VLC Player using a command with different parameters. This means that you can create a batch file (or a script on Linux) that you can use to launch VLC Player with predefined settings.
                     </p>
                     <p>
                      Create a new text file and enter the following string, if you're running Windows:
                     </p>
                     <p>
                     </p>
                     <pre style="margin:0; padding:2px; border: none; width:500px; height:auto; overflow:auto">
                      "C:\Program Files\VideoLAN\VLC\vlc.exe" --http-caching 5000 http://vruk.sc.llnwd.net:12265 :sout=#transcode{acodec=mp3,ab=128,channels=2}:duplicate{dst=display,dst=std{access=file,mux=raw,url="C:\VLCOutput.mp3"}}
                     </pre>
                     <p>
                      In this string,
                      <code>
                       "C:\Program Files\VideoLAN\VLC\vlc.exe"
                      </code>
                      points to the VLC Player's executable. Linux users can replace this part with the appropriate file location.
                     </p>
                     <p>
                      <code>
                       --http-caching 5000
                      </code>
                      enables caching and sets it to 5,000 milliseconds.
                     </p>
                     <p>
                      <code>
                       http://vruk.sc.llnwd.net:12265
                      </code>
                      points VLC Player to the net stream.
                     </p>
                     <p>
                      The part of the command that starts with
                      <code>
                       :sout
                      </code>
                      contains recording settings. Typing this sting manually can be somewhat laborious. Luckily, you can simply copy the string from the Destination Target field in the Stream Output dialogue window and paste it in the text file. Finally, save the file  as a shell script, or with the .bat extension on Windows.
                     </p>
                     <p>
                      The last part of the project is to create a schedule that will execute the batch file at a predefined time. You can use cron on Linux or the Scheduled Tasks tool on Windows to do this. On Windows, go to Accessories &gt; System Tools &gt; Scheduled Tasks and double-click on Add Scheduled Task. This will launch the Schedule Task wizard that will guide you through the process of creating a task. When the wizard prompts you to choose a program file, press the Browse button and point to the batch file. Configure the rest of the settings, and save the task. That's it! Your radio recorder is ready for use.
                     </p>

                       <br/>
                       <br/>
                       I made 2 files to solve the problem, a batch file to start recording which saves a file in the format "record_MMDDYYYY_HR_MN_SS.SS.mp3":
                       <br/>
                       <br/>
                       FOR&lt;nobr&gt; &lt;wbr&gt;&lt;/nobr&gt;/f "tokens=2 delims= " %%D in ('echo %DATE:/=%') do SET DATEN=%%D
                       <br/>
                       FOR %%T in (echo %TIME::=_%) do SET TIMEN=%%T
                       <br/>
                       "C:\...\vlc.exe" --http-caching 5000 &lt;a href="mms://aninternetaddresshere/" title="aninternetaddresshere"&gt;mms://aninternetaddresshere/&lt;/a aninternetaddresshere&gt;&lt;nobr&gt; &lt;wbr&gt;&lt;/nobr&gt;:sout=#transcode{acodec=mp3,ab=64,channels=2}:dup&lt;nobr&gt;l&lt;wbr&gt;&lt;/nobr&gt; icate{dst=std{access=file,mux=raw,url="C:\..\reco&lt;nobr&gt;r&lt;wbr&gt;&lt;/nobr&gt; d_%DATEN%_%TIMEN%.mp3"}}
                       <br/>
                       <br/>
                       <br/>
                       Then, I made a vbs file to kill the application:
                       <br/>
                       <br/>
                       set wmi = getobject("winmgmts:")
                       <br/>
                       wql = "select * from Win32_Process "   &amp; " where name='vlc.exe'"
                       <br/>
                       set results = wmi.execquery(wql)
                       <br/>
                       for each app in results
                       <br/>
                       app.terminate
                       <br/>
                       next
                       <br/>
                       <br/>
                       <br/>
                       So now I can record overseas shows overnight and listen to them the next morning.  It's pretty cool.
