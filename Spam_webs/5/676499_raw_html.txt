     <p>
      <b>
       Finger binary
      </b>
      is a system for
      <a class="mw-redirect" href="/wiki/Finger_counting" title="Finger counting">
       counting
      </a>
      and displaying
      <a class="mw-redirect" href="/wiki/Binary_numeral_system" title="Binary numeral system">
       binary numbers
      </a>
      on the
      <a href="/wiki/Finger" title="Finger">
       fingers
      </a>
      of one or more
      <a href="/wiki/Hand" title="Hand">
       hands
      </a>
      . It is possible to count from 0 to 31 (2
      <sup>
       5
      </sup>
      1) using the fingers of a single hand, or from 0 through 1023 (2
      <sup>
       10
      </sup>
      1) if both hands are used.
     </p>

     <h2>
      <span class="mw-headline" id="Mechanics">
       Mechanics
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Finger_binary&amp;action=edit&amp;section=1" title="Edit section: Mechanics">
        edit

       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <div class="hatnote">
      Further information:
      <a class="mw-redirect" href="/wiki/Binary_numeral_system" title="Binary numeral system">
       Binary numeral system
      </a>
     </div>
     <p>
      In the binary number system, each
      <a href="/wiki/Numerical_digit" title="Numerical digit">
       numerical digit
      </a>
      has two possible states (0 or 1) and each successive digit represents an increasing
      <a href="/wiki/Power_of_two" title="Power of two">
       power of two
      </a>
      .
      <b>
       Note:
      </b>
      What follows is but one of several possible schemes for assigning the values 1, 2, 4, 8, 16, etc. to fingers, not necessarily the best. (see below the illustrations.): The rightmost digit represents two to the
      <a class="mw-redirect" href="/wiki/Zeroth_power" title="Zeroth power">
       zeroth power
      </a>
      (i.e., it is the "ones digit"); the digit to its left represents two to the first power (the "twos digit"); the next digit to the left represents two to the second power (the "fours digit"); and so on. (The
      <a href="/wiki/Decimal" title="Decimal">
       decimal number system
      </a>
      is essentially the same, only that powers of ten are used: "ones digit", "tens digit" "hundreds digit", etc.)
     </p>
     <p>
      It is possible to use
      <a href="/wiki/Digit_(anatomy)" title="Digit (anatomy)">
       anatomical digits
      </a>
      to represent
      <a href="/wiki/Numerical_digit" title="Numerical digit">
       numerical digits
      </a>
      by using a raised finger to represent a binary digit in the "1" state and a lowered finger to represent it in the "0" state. Each successive finger represents a higher power of two.
     </p>

     <p>
      The values of each raised finger are added together to arrive at a total number. In the one-handed version, all fingers raised is thus
      <b>
       31
      </b>
      (16 + 8 + 4 + 2 + 1), and all fingers lowered (a fist) is 0. In the two-handed system, all fingers raised is
      <b>
       1,023
      </b>
      (512 + 256 + 128 + 64 + 32 + 16 + 8 + 4 + 2 + 1) and two fists (no fingers raised) represents 0.
     </p>
     <p>
      It is also possible to have each hand represent an independent number between 0 and 31; this can be used to represent various types of paired numbers, such as
      <a href="/wiki/Month" title="Month">
       month
      </a>
      and
      <a href="/wiki/Day" title="Day">
       day
      </a>
      , X-Y
      <a class="mw-redirect" href="/wiki/Coordinate" title="Coordinate">
       coordinates
      </a>
      , or sports scores (such as for
      <a href="/wiki/Table_tennis" title="Table tennis">
       table tennis
      </a>
      or
      <a href="/wiki/Baseball" title="Baseball">
       baseball
      </a>
      ).
     </p>

     <h2>
      <span class="mw-headline" id="Negative_numbers_and_non-integers">
       Negative numbers and non-integers
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Finger_binary&amp;action=edit&amp;section=5" title="Edit section: Negative numbers and non-integers">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <div class="hatnote">
      For more details on this topic, see
      <a class="mw-redirect" href="/wiki/Binary_numeral_system#Representing_real_numbers" title="Binary numeral system">
       Binary numeral system  Representing real numbers
      </a>
      .
     </div>
     <p>
      Just as fractional and negative numbers can be represented in binary, they can be represented in finger binary.
     </p>
     <h3>
      <span class="mw-headline" id="Negative_numbers">
       Negative numbers
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Finger_binary&amp;action=edit&amp;section=6" title="Edit section: Negative numbers">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      Representing negative numbers is extremely simple, by using the leftmost finger as a
      <a href="/wiki/Sign_bit" title="Sign bit">
       sign bit
      </a>
      : raised means the number is negative, in a
      <a class="mw-redirect" href="/wiki/Sign-magnitude" title="Sign-magnitude">
       sign-magnitude
      </a>
      system. Anywhere between -511 and +511 can be represented this way, using two hands. Note that, in this system, both a positive and a negative zero may be represented.
     </p>
     <p>
      If a convention were reached on palm up/palm down or fingers pointing up/down representing positive/negative, you could maintain 2^10 - 1 in both positive and negative numbers (-1023 to +1023, with positive and negative zero still represented).
     </p>
     <h3>
      <span class="mw-headline" id="Fractions">
       Fractions
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Finger_binary&amp;action=edit&amp;section=7" title="Edit section: Fractions">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      There are multiple ways of representing fractions in finger binary.
     </p>
     <h4>
      <span class="mw-headline" id="Dyadic_fractions">
       Dyadic fractions
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Finger_binary&amp;action=edit&amp;section=8" title="Edit section: Dyadic fractions">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h4>
     <p>
      Fractions can be stored natively in a binary format by having each finger represent a fractional power of two:
      <img alt="\tfrac{1}{2^x}" class="mwe-math-fallback-image-inline tex" src="//upload.wikimedia.org/math/7/e/d/7ed68125c51416a69847be82843cdc5e.png"/>
      . (These are known as
      <a class="mw-redirect" href="/wiki/Dyadic_fraction" title="Dyadic fraction">

      </a>
      .)
     </p>

      Using the left hand only:

     <div class="thumb tright">
      <div class="thumbinner" style="width:178px;">
       <a class="image" href="/wiki/File:Chinesische.Zahl.Acht.jpg">
        <img alt="" class="thumbimage" data-file-height="150" data-file-width="176" height="150" src="//upload.wikimedia.org/wikipedia/commons/9/94/Chinesische.Zahl.Acht.jpg" width="176"/>

        3/4, in fractional finger binary
       </div>
      </div>
     </div>
     <p>
      The total is calculated by adding all the values in the same way as regular (non-fractional) finger binary, then dividing by the largest fractional power being used (32 for one-handed fractional binary, 1024 for two-handed), and
      <a class="mw-redirect" href="/wiki/Equivalent_fractions" title="Equivalent fractions">
       simplifying the fraction
      </a>
      as necessary.
     </p>
     <p>
      For example, with thumb and index finger raised on the left hand and no fingers raised on the right hand, this is (512 + 256)/1024 = 768/1024 = 3/4. If using only one hand (left or right), it would be (16 + 8)/32 = 24/32 = 3/4 also.
     </p>
     <p>
      The simplification process can itself be greatly simplified by performing a
      <a class="mw-redirect" href="/wiki/Bit_shift" title="Bit shift">
       bit shift
      </a>
      operation: all digits to the right of the rightmost raised finger (i.e., all trailing zeros) are discarded and the rightmost raised finger is treated as the ones digit. The digits are added together using their now-shifted values to determine the
      <a class="mw-redirect" href="/wiki/Numerator" title="Numerator">
       numerator
      </a>
      and the rightmost finger's original value is used to determine the
      <a class="mw-redirect" href="/wiki/Denominator" title="Denominator">
       denominator
      </a>
      .
     </p>
     <p>
      For instance, if the thumb and index finger on the left hand are the only raised digits, the rightmost raised finger (the index finger) becomes "1". The thumb, to its immediate left, is now the 2s digit; added together, they equal 3. The index finger's original value (1/4) determines the denominator: the result is 3/4.
     </p>
     <h4>
      <span class="mw-headline" id="Rational_numbers">
       Rational numbers
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Finger_binary&amp;action=edit&amp;section=9" title="Edit section: Rational numbers">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h4>
     <p>
      Combined
      <a href="/wiki/Integer" title="Integer">
       integer
      </a>
      and fractional values (i.e.,
      <a href="/wiki/Rational_number" title="Rational number">
       rational numbers
      </a>
      ) can be represented by setting a
      <a href="/wiki/Radix_point" title="Radix point">
       radix point
      </a>
      somewhere between two fingers (for instance, between the left and right pinkies). All digits to the left of the radix point are integers; those to the right are fractional.
     </p>

      <span class="mw-editsection">

        edit

       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      <a class="mw-redirect" href="/wiki/Dyadic_fraction" title="Dyadic fraction">
       Dyadic fractions
      </a>
      , explained above, unfortunately have limited use in a society based around decimal figures. A simple non-dyadic fraction such as 1/3 can be approximated as 341/1024 (0.3330078125), but the conversion between dyadic and
      <a class="mw-redirect" href="/wiki/Decimal_fraction" title="Decimal fraction">
       decimal
      </a>
      (0.333) or
      <a class="mw-redirect" href="/wiki/Vulgar_fraction" title="Vulgar fraction">
       vulgar
      </a>
      (1/3) forms is complicated.
     </p>
     <p>
      Instead, either decimal or vulgar fractions can be represented natively in finger binary. Decimal fractions can be represented by using regular integer binary methods and dividing the result by 10, 100, 1000, or some other power of ten. Numbers between 0 and 102.3, 10.23, 1.023, etc. can be represented this way, in increments of 0.1, 0.01, 0.001, etc.
     </p>
     <p>
      <a class="mw-redirect" href="/wiki/Vulgar_fraction" title="Vulgar fraction">
       Vulgar fractions
      </a>
      can be represented by using one hand to represent the
      <a class="mw-redirect" href="/wiki/Numerator" title="Numerator">
       numerator
      </a>
      and one hand to represent the
      <a class="mw-redirect" href="/wiki/Denominator" title="Denominator">
       denominator
      </a>
      ; a spectrum of rational numbers can be represented this way, ranging from 1/31 to 31/1 (as well as 0).
     </p>
     <h2>
      <span class="mw-headline" id="Finger_ternary">
       Finger ternary
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Finger_binary&amp;action=edit&amp;section=11" title="Edit section: Finger ternary">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      In theory, it is possible to use other positions of the fingers to represent more than two states (0 and 1); for instance, a
      <a href="/wiki/Ternary_numeral_system" title="Ternary numeral system">
       ternary numeral system
      </a>
      (
      <a class="mw-redirect" href="/wiki/Number_base" title="Number base">
       base
      </a>
      3) could be used by having a fully raised finger represent 2, fully lowered represent 0, and "curled" (half-lowered) represent 1. This would make it possible to count up to 59,048 (3
      <sup>
       10
      </sup>
      1) on two hands. In practice, however, many people will find it difficult to hold all fingers independently (especially the middle and ring fingers) in more than two distinct positions.
     </p>
