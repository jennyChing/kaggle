jQuery(function($) {
    // Debug flag
    var debugMode = false;
    // Default time delay before checking location
    var callBackTime = 100;
    // # px before tracking a reader
    var readerLocation = 150;
    // Set some flags for tracking &amp; execution
    var timer = 0;
    var scroller = false;
    var endContent = false;
    var didComplete = false;
    // Set some time variables to calculate reading time
    var startTime = new Date();
    var beginning = startTime.getTime();
    var totalTime = 0;
    // Get some information about the current page
    var pageTitle = document.title;
    // Track the aticle load
    if (!debugMode) {
        _gaq.push(['_trackEvent', 'Reading', 'ArticleLoaded', '', , true]);
    } else {
        alert('The page has loaded. Woohoo.');    
    }
    // Check the location and track user
    function trackLocation() {
        bottom = $(window).height() + $(window).scrollTop();
        height = $(document).height();
        // If user starts to scroll send an event
        if (bottom &gt; readerLocation &amp;&amp; !scroller) {
            currentTime = new Date();
            scrollStart = currentTime.getTime();
            timeToScroll = Math.round((scrollStart - beginning) / 1000);
            if (!debugMode) {
                _gaq.push(['_trackEvent', 'Reading', 'StartReading', '', timeToScroll]);
            } else {
                alert('started reading ' + timeToScroll);
            }
            scroller = true;
        }
        // If user has hit the bottom of the content send an event
        if (bottom &gt;= $('.entry-content').scrollTop() + $('.entry-content').innerHeight() &amp;&amp; !endContent) {
            currentTime = new Date();
            contentScrollEnd = currentTime.getTime();
            timeToContentEnd = Math.round((contentScrollEnd - scrollStart) / 1000);
            if (!debugMode) {
                _gaq.push(['_trackEvent', 'Reading', 'ContentBottom', '', timeToContentEnd]);
            } else {
                alert('end content section '+timeToContentEnd);
            }
            endContent = true;
        }
        // If user has hit the bottom of page send an event
        if (bottom &gt;= height &amp;&amp; !didComplete) {
            currentTime = new Date();
            end = currentTime.getTime();
            totalTime = Math.round((end - scrollStart) / 1000);
            if (!debugMode) {
                if (totalTime &lt; 60) {
                    _gaq.push(['_setCustomVar', 5, 'ReaderType', 'Scanner', 2]);
                } else {
                    _gaq.push(['_setCustomVar', 5, 'ReaderType', 'Reader', 2]);
                }
                _gaq.push(['_trackEvent', 'Reading', 'PageBottom', pageTitle, totalTime]);
            } else {
                alert('bottom of page '+totalTime);
            }
            didComplete = true;
        }
    }
    // Track the scrolling and track location
    $(window).scroll(function() {
        if (timer) {
            clearTimeout(timer);
        }
        // Use a buffer so we don't call trackLocation too often.
        timer = setTimeout(trackLocation, callBackTime);
	pec_ga_event = 1;
    });
});

      <div class="entry-content">
       <p dir="ltr">
        <strong>
         Google Analytics reporting views
        </strong>
        offer different ways to look at the website or application traffic data, which may help online business owners and marketers better understand how to attract, retain, and convert more site visitors.
       </p>
       <p>
        When a user creates in Google Analytics a new property (i.e., website), Google Analytics automatically shows a single, unfiltered default view that displays all of the data collected. Unfortunately, this is where many marketers and small business owners stop, but each Google Analytics property may have many views or many ways, if you will, of segregating the data in order to get a better understanding or a more accessible insight.
       </p>
       <p>
        A quick example should make the point. Many ecommerce platforms use an index page to route traffic. In some cases, it will be possible to access a page with or without that index page being referenced. In the example below notice the index.php in one URL and not the other. Google Analytics would count these two pages separately, even though they point to exactly the same content.
       </p>
       <div class="wp-caption alignnone" id="attachment_70260" style="width: 580px">
        <img alt="Google Analytics would count the index and non-index pages separately because of the differences in the URLs." class="size-large wp-image-70260 " height="142" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-example-indexphp-570x142.png" title="Google Analytics would count the index and non-index pages separately because of the differences in the URLs." width="570"/>
        <p class="wp-caption-text">
         Google Analytics would count the index and non-index pages separately because of the differences in the URLs.
        </p>
       </div>
       <p>
        A Google Analytics user could create a new view and filter that would ignore index.php and report, perhaps, more accurately that the product page had 300 page views for the time period in view.
       </p>
       <h3>
        3 Things to Remember about Google Analytics Views
       </h3>
       <p dir="ltr">
        Before getting down to the business of creating and using views, there are three things that must be understood.
       </p>
       <ul>
        <li>
         <p dir="ltr">
          <strong>
           Views are not retroactive.
          </strong>
          Google Analytics will only track data for a particular view from the moment it is created; it will not include historical data from other views.
         </p>
        </li>
        <li>
         <p dir="ltr">
          <strong>
           Always keep the raw data.
          </strong>
          Views may be most effective when filters are used to exclude, include, or change data. But filters can have unexpected errors, so always keep at least one clean view that has the raw data Google Analytics collected.
         </p>
        </li>
        <li>
         <p dir="ltr">
          <strong>
           Test before implementing.
          </strong>
          When working with views, it is a good idea to add new filters to test views, ensuring that the filter is working as expected before adding it to an important view. Although a filter can be removed, its impact cannot be reversed.
         </p>
        </li>
       </ul>
       <h3>
        Create a New Google Analytics Reporting View
       </h3>
       <p dir="ltr">
        To create a new reporting view in Google Analytics, navigate to the administration tab by clicking the word Admin in the main navigation.
       </p>
       <div class="wp-caption alignnone" id="attachment_70259" style="width: 580px">
        <img alt="Locate the administration section in Google Analytics' main navigation." class="size-large wp-image-70259 " height="210" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-admin-link-570x210.png" title="Locate the administration section in Google Analytics' main navigation." width="570"/>
        <p class="wp-caption-text">
         Locate the administration section in Google Analytics main navigation.
        </p>
       </div>
       <p dir="ltr">
        Select the desired Account and Property. Remember it is possible for an Account to have more than one property.
       </p>
       <div class="wp-caption alignnone" id="attachment_70258" style="width: 580px">
        <img alt="It is possible to have several accounts and properties in Google Analytics." class="size-large wp-image-70258 " height="154" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-account-property-570x154.png" title="It is possible to have several accounts and properties in Google Analytics." width="570"/>
        <p class="wp-caption-text">
         It is possible to have several accounts and properties in Google Analytics.
        </p>
       </div>
       <p dir="ltr">
        Open the view dropdown menu and click the Create New View link to open a new page.
       </p>
       <div class="wp-caption alignnone" id="attachment_70257" style="width: 580px">
        <img alt='The "Create new view" link is below the list of existing views in the view dropdown menu.' class="size-large wp-image-70257 " height="280" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-create-new-view-570x280.png" title='The "Create new view" link is below the list of existing views in the view dropdown menu.' width="570"/>
        <p class="wp-caption-text">
         The Create new view link is below the list of existing views in the view dropdown menu.
        </p>
       </div>
       <p dir="ltr">
        Google Analytics currently supports two kinds of views: mobile app and website. Mobile app views will store data about application errors or the Google Play report while the website view include Site Content reports.
       </p>
       <p dir="ltr">
        Name the report and set the time zone it should use, and click create to add this new view.
       </p>
       <div class="wp-caption alignnone" id="attachment_70256" style="width: 580px">
        <img alt="Google Analytics allows website and mobile app views, which have slightly different reports." class="size-large wp-image-70256 " height="359" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-create-view-page-570x359.png" title="Google Analytics allows website and mobile app views, which have slightly different reports." width="570"/>
        <p class="wp-caption-text">
         Google Analytics allows website and mobile app views, which have slightly different reports.
        </p>
       </div>
       <p dir="ltr">
        Any views settings are available by clicking view settings in the administration tab. Remember this view will begin tracking data from when it is created (going forward), but it will not have historical data from any other view.
       </p>
       <h3>
        Add a Filter to a View
       </h3>
       <p dir="ltr">
        Views get most interesting when you customize them with filters. To get started, click the Filters link from the View column of the administration tab.
       </p>
       <div class="wp-caption alignnone" id="attachment_70255" style="width: 580px">
        <img alt='The "Filters" link is in the View column of the Admin section.' class="size-large wp-image-70255 " height="341" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-filters-link-570x341.png" title='The "Filters" link is in the View column of the Admin section.' width="570"/>
        <p class="wp-caption-text">
         The Filters link is in the View column of the Admin section.
        </p>
       </div>
       <p dir="ltr">
        A new dialogue will open, click the New Filter button to get started customizing the view.
       </p>
       <p dir="ltr">
        Google Analytics filters can be Predefined or Custom. The predefined filters try to make it easy to add very common filters. In fact, when the filter configuration page loads it may display a filter that is frequently used to exclude internal traffic from the view. Simply put, this filter will ignore all traffic from a particular domain. At traditional companies where all of the employees work in the same building with the same Internet Service Provider domain, the filter will not count these internal users, preventing them from impacting the data. A more refined version of the same filter targets Internet Protocol (IP) addresses or address ranges.
       </p>
       <div class="wp-caption alignnone" id="attachment_70253" style="width: 580px">
        <img alt="Google Analytics filters can be Predefined or Custom. The predefined filters try to make it easy to add very common filters." class="size-large wp-image-70253" height="352" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-filter-settings-570x352.png" title="Google Analytics filters can be Predefined or Custom. The predefined filters try to make it easy to add very common filters." width="570"/>
        <p class="wp-caption-text">
         Google Analytics filters can be Predefined or Custom. The predefined filters try to make it easy to add very common filters.
        </p>
       </div>
       <p dir="ltr">
        Custom filters allow users to develop their own unique ways of including and excluding data. As an example, here is a filter that would combine the URLs
        <em>
         http://some-store.com/index.php/product
        </em>
        and
        <em>
         http://some-store.com/product
        </em>
        , aggregating page views.
       </p>
       <div class="wp-caption alignnone" id="attachment_70252" style="width: 580px">
        <img alt='This filter would replace "/index.php" with just a slash, effectively combining data for two URLs.' class="size-large wp-image-70252 " height="551" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-filter-remove-indexphp-570x551.png" title='This filter would replace "/index.php" with just a slash, effectively combining data for two URLs.' width="570"/>
        <p class="wp-caption-text">
         This filter would replace /index.php with just a slash, effectively combining data for two URLs.
        </p>
       </div>
       <p dir="ltr">
        Notice that the filter type is custom, and it is conducting a search and replace. The string to be replaced is /index.php, which includes the leading slash.
       </p>
       <h3>
        Filter Order Matters
       </h3>
       <p dir="ltr">
        If you apply more than one filter to a view, it is important to remember that Google Analytics will process filters in the order applied. So the second filter used will not see all of the data, but rather the output from the first filter.
       </p>
       <p dir="ltr">
        In its documentation, Google Analytics explains this concept with the example of a filter for U.S. visitors and a second filter for Canadian visitors. This combination of filters would result in, well, no results. The first filter would remove all visitors except those from the United States and the second filter would remove all visitor data except for data from Canadian visitors. The trouble is that there would be no Canadian visitors, since the previous filter had excluded them, and the current data set of only U.S. visitors, would not pass through the second filter.
       </p>
       <p dir="ltr">
        Rather, a single filter would be needed to include traffic from both the United States and Canada in a single step.
       </p>
       <div class="wp-caption alignnone" id="attachment_70251" style="width: 580px">
        <img alt='The "" character acts to include both U.S. and Canadian visitors.' class="size-large wp-image-70251 " height="514" src="http://www.practicalecommerce.com/files/2014/07/070114-ga-country-filter-570x514.png" title='The "" character acts to include both U.S. and Canadian visitors.' width="570"/>
        <p class="wp-caption-text">
         The  character acts to include both U.S. and Canadian visitors.
        </p>
       </div>
       <p dir="ltr">
        Applied correctly, the filter and its associated view will only collect data from American and Canadian visitors, providing a clearly defined subset of data.
       </p>
       <h3>
        Let the Experiments Begin
       </h3>
       <p>
        With a basic understanding of how reporting views and filters work, it is time to experiment. Here are some filter ideas.
       </p>
       <ul>
        <li>
         <p dir="ltr">
          Visitors from Email Marketing Campaigns Only
         </p>
        </li>
        <li>
         <p dir="ltr">
          Returning Visitors Only
         </p>
        </li>
        <li>
         <p dir="ltr">
          Visitors from Sites with Banner Advertising
         </p>
        </li>
        <li>
         <p dir="ltr">
          Visitors from Your Social Media Pages
         </p>
        </li>
       </ul>
