         <h1>
          Announcing Acire
         </h1>
         <p class="post-meta">
          by
          <a href="http://www.jonobacon.org/author/admin/" rel="author" title="Posts by jono">
           jono
          </a>
          | Dec 30, 2009 |
          <a href="http://www.jonobacon.org/category/acire/" rel="category tag">
           Acire
          </a>
          ,
          <a href="http://www.jonobacon.org/category/community/" rel="category tag">
           Community
          </a>
          ,
          <a href="http://www.jonobacon.org/category/desktop/" rel="category tag">
           Desktop
          </a>

          <a href="http://www.jonobacon.org/category/ubuntu/" rel="category tag">

          |
          <span class="comments-number">
           <a href="http://www.jonobacon.org/2009/12/30/announcing-acire/#respond">

             21 comments
            </span>
           </a>
          </span>
         </p>
         <div class="entry-content">
          <p>
           After a wonderful week in England with family celebrating Christmas, Erica and I flew home to the East Bay. We were sat at Heathrow having a cup of coffee and I was thinking of what I occupy myself with on the plane ride over. Unfortunately,
           <a href="http://wiki.ubuntu.com/Lernid">
            Lernid
           </a>
           hacking was out of the question as I had no net connection on the plane, so I got to thinking of something else. After some busy hacking time at 35,000 feet I am proud to show of the results of my labor: a little program called
           <em>
            Acire
           </em>
           .
          </p>
          <p>
           Let me provide a bit of an introduction.
          </p>
          <p>
           I love Python and I love PyGTK and as I have waxed lyrical about recently, I am passionate about how tools such as
           <a href="http://wiki.ubuntu.com/Quickly">
            Quickly
           </a>
           are lowering the bar for opportunistic programmers to scratch their itch. The problem is, much as Python and PyGTK are incredible for writing apps, like any other tool they still need to be learned, and the best way to learn is by downloading, running and playing with code snippets. Traditionally this has involved finding scattered bits of sample code online and stuffing it into a directory. Not only do we download other peoples sample code but we often write our own little code snippets as we learn new parts of these tools, and these would be perfect to share with others. Unfortunately, many of us never end up putting them anywhere online.
           <em>
            Acire
           </em>
           is a first step in making this whole process easier.
          </p>
          <p>
           Right now,
           <em>
            Acire
           </em>
           is a tool that allows you to browse snippets easily. It looks like this:
          </p>
          <p>
           <img src="http://farm3.static.flickr.com/2798/4227106057_03730b2ae2_o.jpg" width="600"/>
          </p>
          <p>
           The combo box in the top left provides different categories of snippets. In the screenshot above you can see the general category
           <code>
            PyGTK
           </code>
           , but there is no reason why we cant have sub-categories such as
           <em>
            PyGTK -&gt; TreeViews
           </em>
           . When you select a category, the list underneath it shows all the current snippets. Clicking on a snippet not only loads the code in the view to the right, but also provides a simple description and location of the snippet underneath the code. You can then run the snippet by clicking the
           <em>
            Execute
           </em>
           button or copy the full code to the clipboard by clicking the
           <em>
            Copy
           </em>
           button.
          </p>
          <p>
           To make snippets work in
           <em>
            Acire
           </em>
           , I have created some simple metadata tags that snippet authors should put at the top of the snippet in the code. Here is an example:
          </p>
          <pre>
           <code>
            # [SNIPPET_NAME: Actions]
# [SNIPPET_CATEGORIES: PyGTK]
# [SNIPPET_DESCRIPTION: Demonstrates using Actions with a gtk.Action and gtk.AccelGroup]
           </code>
          </pre>
          <p>
           To test this out I have already added this meta data content to all the examples from the
           <a href="http://www.pygtk.org/tutorial.html">
            PyGTK Manual
           </a>
           . This functionality already makes browsing available snippets, running them and looking at the code an absolute doddle.
          </p>
          <p>
           I have just got in from a day of traveling, I am exhausted, and headed to bed. I will get the code online tomorrow and talk more about plans for
           <em>
            Acire
           </em>
           .
           <img alt=":-)" class="wp-smiley" src="http://www.jonobacon.org/wp-includes/images/smilies/simple-smile.png" style="height: 1em; max-height: 1em;"/>
          </p>
         </div>
