        <h2 id="iredmail-mail-server-with-ldap-postfix-roundcube-dovecot-clamav-spamassassin-amavisd-on-debian-lenny-">
         iRedMail: Mail Server With LDAP, Postfix, RoundCube, Dovecot, ClamAV, SpamAssassin, Amavisd On Debian (Lenny) 5.0.1
        </h2>
        <p>
         Last edited 03/31/2011
         <br/>
        </p>
        <p>
         iRedMail is a shell script that lets you quickly deploy a full-featured mail solution in less than 2 minutes. Since iRedMail 0.5, it supports Debian 5.0.1 (it supports both i386 and x86_64). Its object is to make a Linux mail server installation and configuration simple and easy to use. iRedMail supports both OpenLDAP and MySQL as backends for storing virtual domains and users.This tutorial shows how to use OpenLDAP as the backend.
        </p>
        <div style="width:336px;float:left; margin: 10px 15px 10px 0px;background-color:white;">
         <script type="text/javascript">
          GA_googleFillSlot('howtoforge_com_article_rectangle_a_300x250');
         </script>
        </div>
        <p>
         Now iRedmail provides two webmail program, RoundCube and SquirrelMail, and both support the MySQL and OpenLDAP backends and can change the password.
        </p>
        <ul>
         <li>
          Main components used in this solution:
          <a href="http://code.google.com/p/iredmail/wiki/Main_Components" mce_real_href="http://code.google.com/p/iredmail/wiki/Main_Components" target="_blank">
           http://code.google.com/p/iredmail/wiki/Main_Components
          </a>
         </li>
         <li>
          Discussion forum:
          <a href="http://www.iredmail.org/forum/" mce_real_href="http://www.iredmail.org/forum/" target="_blank">
           http://www.iredmail.org/forum/
          </a>
         </li>
         <li>
          Project home page:
          <a href="http://code.google.com/p/iredmail/" mce_real_href="http://code.google.com/p/iredmail/" target="_blank">
           http://code.google.com/p/iredmail/
          </a>
         </li>
        </ul>
        <p>
        </p>
        <h3 id="-requirements">
         1 Requirements
        </h3>
        <p>
         It is strongly recommended that you use the 'netinst' version to install a minimal Debian system, and then deploy mail service via iRedMail solution. e.g.:
        </p>
        <p>
         To install such a system you will need the following:
        </p>
        <ul>
         <li>
          debian-501-amd64-netinst.iso or debian-501-i386-netinst.iso ,
          <a href="http://www.debian.org/CD/http-ftp/" mce_real_href="http://www.debian.org/CD/http-ftp/" target="_blank">
           download Debian CD/DVD images via HTTP/FTP
          </a>
         </li>
         <li>
          Internet connection.
         </li>
        </ul>
        <p>
        </p>
        <h3 id="-preliminary-note">
         2 Preliminary Note
        </h3>
        <p>
         In this tutorial we use:
        </p>
        <ul>
         <li>
          Hostname
          <span class="system">
           mail.example.com
          </span>
         </li>
         <li>
          IP address
          <span class="system">
           192.168.1.10
          </span>
         </li>
         <li>
          LDAP suffix (root dn)
          <span class="system">
           dc=example,dc=com
          </span>
         </li>
         <li>
          The first virtual domain:
          <span class="system">
           example.com
          </span>
         </li>
        </ul>
        <p>
        </p>
        <br/>
        <h3 id="-setting-hostname">
         4 Setting hostname
        </h3>
        <p>
         In Debian (Lenny), hostname is set in these two files:
        </p>
        <p>
         *
         <span class="system">
          /etc/hostname
         </span>
         : hostname setting
         <br/>
         *
         <span class="system">
          /etc/hosts
         </span>
         : hostname &lt;=&gt; IP address mapping
        </p>
        <p>
         Map this hostname to '127.0.0.1' (Warning: List the FQDN hostname as first item.)
        </p>
        <p class="command">
         # vi /etc/hosts
        </p>
        <table align="center" bgcolor="#cccccc" border="1" bordercolor="#000000" cellpadding="2" cellspacing="0" class="" width="90%">
         <tbody>
          <tr>
           <td class="">
            <pre>
             #127.0.0.1       localhost
#127.0.1.1      mail.example.com        mail
127.0.0.1   mail.example.com mail localhost localhost.localdomain
            </pre>
           </td>
          </tr>
         </tbody>
        </table>
        <p class="command">
         # vi /etc/hostname
        </p>
        <table align="center" bgcolor="#cccccc" border="1" bordercolor="#000000" cellpadding="2" cellspacing="0" class="" width="90%">
         <tbody>
          <tr>
           <td class="">
            <pre>
             mail
            </pre>
           </td>
          </tr>
         </tbody>
        </table>
        <p>
         Verify the FQDN hostname:
        </p>
        <p class="command">
         # hostname --fqdn
        </p>
        <p class="system">
         mail.example.com
        </p>

        <br/>
        <h3 id="-start-iredmail-installation">
         5 Start iRedMail Installation
        </h3>
        <p>
         Switch to root user, we must execute further commands as root user without sudo.
        </p>
        <p class="command">
         $ sudo su -
        </p>
        <p>
         Download the new version of iRedMail:
        </p>
        <p class="command">
         # wget http://iredmail.googlecode.com/files/iRedMail-0.7.0.tar.bz2
         <br/>
         # tar jxvf iRedMail-0.7.0.tar.bz2
        </p>
        <br/>
        <br/>
        <p>
         Start the iRedMail installer:
        </p>
        <p class="command">
         # cd iRedMail-0.7.0
         <br/>
         # bash iRedMail.sh
        </p>
        <p>
         Welcome page:
        </p>
        <p>
         <a class="fancybox" href="https://www.howtoforge.com/images/iredmail_debian_lenny/big/iredmail_01.png" id="img-iredmail_01">
          <img height="253" mce_real_src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_01.png" src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_01.png" width="550"/>
         </a>
        </p>
        <p>
         All emails should be stored in this directory:
        </p>
        <p>
         <a class="fancybox" href="https://www.howtoforge.com/images/iredmail_debian_lenny/big/iredmail_02.png" id="img-iredmail_02">
          <img height="257" mce_real_src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_02.png" src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_02.png" width="549"/>
         </a>
        </p>
        <p>
         iRedMail supports both OpenLDAP and MySQL as backends to store virtual domains and users. OpenLDAP is recommended.
        </p>
        <p>
         <a class="fancybox" href="https://www.howtoforge.com/images/iredmail_debian_lenny/big/iredmail_03.png" id="img-iredmail_03">
          <img height="253" mce_real_src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_03.png" src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_03.png" width="550"/>
         </a>
        </p>
        <p>
         LDAP suffix, normally it's your domain name, such as:
         <span class="system">
          dc=example,dc=com
         </span>
         <br/>
         iRedMail will put all virtual domains/users in 'o=domains,dc=example,dc=com', it's called 'LDAP base dn':
         <span class="system">
          o=domains,dc=example,dc=com
         </span>
         .
        </p>
        <p>
         <a class="fancybox" href="https://www.howtoforge.com/images/iredmail_debian_lenny/big/iredmail_04.png" id="img-iredmail_04">
          <img height="255" mce_real_src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_04.png" src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_04.png" width="550"/>
         </a>
        </p>
        <p>
         Set a password for the LDAP server admin. The admin account is defined in
         <span class="system">
          /etc/ldap/slapd.conf
         </span>
         , normaily it's:
         <span class="system">
          cn=Manager,dc=example,dc=com
         </span>
         .
        </p>
        <p>
         <a class="fancybox" href="https://www.howtoforge.com/images/iredmail_debian_lenny/big/iredmail_05.png" id="img-iredmail_05">
          <img height="254" mce_real_src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_05.png" src="https://www.howtoforge.com/images/iredmail_debian_lenny/iredmail_05.png" width="550"/>
         </a>
        </p>
