    <div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en">
     <p>
      <b>
       Advanced Vector Extensions
      </b>
      (
      <b>
       AVX
      </b>
      ) are extensions to the
      <a href="/wiki/X86" title="X86">
       x86
      </a>
      <a class="mw-redirect" href="/wiki/Instruction_set_architecture" title="Instruction set architecture">
       instruction set architecture
      </a>
      for
      <a href="/wiki/Microprocessor" title="Microprocessor">
       microprocessors
      </a>
      from
      <a class="mw-redirect" href="/wiki/Intel_Corporation" title="Intel Corporation">
       Intel
      </a>
      and
      <a href="/wiki/Advanced_Micro_Devices" title="Advanced Micro Devices">
       AMD
      </a>
      proposed by Intel in March 2008 and first supported by Intel with the
      <a href="/wiki/Sandy_Bridge" title="Sandy Bridge">
       Sandy Bridge
      </a>
      processor shipping in Q1 2011 and later on by AMD with the
      <a href="/wiki/Bulldozer_(microarchitecture)" title="Bulldozer (microarchitecture)">
       Bulldozer
      </a>
      processor shipping in Q3 2011. AVX provides new features, new instructions and a new coding scheme.
     </p>
     <p>
      <b>
       AVX2
      </b>
      expands most integer commands to 256 bits and introduces
      <a class="mw-redirect" href="/wiki/Fused_multiply-accumulate" title="Fused multiply-accumulate">
       FMA
      </a>
      .
      <a href="/wiki/AVX-512" title="AVX-512">
       AVX-512
      </a>
      expands AVX to 512-bit support utilizing a new
      <a href="/wiki/EVEX_prefix" title="EVEX prefix">
       EVEX prefix
      </a>
      encoding proposed by Intel in July 2013 and first supported by Intel with the
      <a class="mw-redirect" href="/wiki/Knights_Landing_(microarchitecture)" title="Knights Landing (microarchitecture)">
       Knights Landing
      </a>
      processor scheduled to ship in 2015.
      <sup class="reference" id="cite_ref-reinders512_1-0">
       <a href="#cite_note-reinders512-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>

     <h2>
      <span class="mw-headline" id="Advanced_Vector_Extensions">
       <span id="AVX1">
       </span>
       Advanced Vector Extensions
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Advanced_Vector_Extensions&amp;action=edit&amp;section=1" title="Edit section: Advanced Vector Extensions">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      The width of the
      <a href="/wiki/SIMD" title="SIMD">
       SIMD
      </a>
      register file is increased from 128 bits to 256 bits, and renamed from
      <a href="/wiki/Streaming_SIMD_Extensions#Registers" title="Streaming SIMD Extensions">
       XMM
      </a>
      0XMM7 to YMM0YMM7 (in
      <a href="/wiki/X86-64" title="X86-64">
       x86-64
      </a>
      mode, YMM0YMM15). In processors with AVX support, the legacy
      <a href="/wiki/Streaming_SIMD_Extensions" title="Streaming SIMD Extensions">
       SSE
      </a>
      instructions (which previously operated on 128-bit XMM registers) can be extended using the
      <a href="/wiki/VEX_prefix" title="VEX prefix">
       VEX prefix
      </a>
      to operate on the lower 128 bits of the YMM registers.

     <p>
      AVX introduces a three-operand SIMD instruction format, where the destination register is distinct from the two source operands. For example, an
      <a href="/wiki/Streaming_SIMD_Extensions" title="Streaming SIMD Extensions">
       SSE
      </a>
      instruction using the conventional two-operand form
      <i>
       a=a+b
      </i>
      can now use a non-destructive three-operand form
      <i>
       c=a+b
      </i>
      , preserving both source operands. AVX's three-operand format is limited to the instructions with SIMD operands (YMM), and does not include instructions with general purpose registers (e.g. EAX). Such support will first appear in AVX2.
      <sup class="reference" id="cite_ref-avx2_2-0">
       <a href="#cite_note-avx2-2">
        <span>
         [
        </span>
        2
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      The
      <a href="/wiki/Data_structure_alignment" title="Data structure alignment">
       alignment
      </a>
      requirement of SIMD memory operands is relaxed.
      <sup class="reference" id="cite_ref-intel_vol1_3-0">
       <a href="#cite_note-intel_vol1-3">
        <span>
         [
        </span>
        3
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      The new
      <a class="mw-redirect" href="/wiki/VEX_coding_scheme" title="VEX coding scheme">
       VEX coding scheme
      </a>
      introduces a new set of code prefixes that extends the
      <a href="/wiki/Opcode" title="Opcode">
       opcode
      </a>
      space, allows instructions to have more than two operands, and allows SIMD vector registers to be longer than 128 bits. The VEX prefix can also be used on the legacy SSE instructions giving them a three-operand form, and making them interact more efficiently with AVX instructions without the need for VZEROUPPER and ZEROALL.
     </p>
     <p>
      The AVX instructions support both 128-bit and 256-bit SIMD. The 128-bit versions can be useful to improve old code without needing to widen the vectorization, and avoid the penalty of going from SSE to AVX, they are also faster on some early AMD implementations of AVX. This mode is sometimes known as AVX-128.
      <sup class="reference" id="cite_ref-4">
       <a href="#cite_note-4">
        <span>
         [
        </span>
        4
        <span>
         ]
        </span>
       </a>
      </sup>
      AVX-128 instructions that do not use YMM registers are also safe to use on operating systems without AVX-support
      <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
       [
       <i>
        <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
         <span title="This claim needs references to reliable sources. (April 2015)">
          citation needed
         </span>
        </a>
       </i>
       ]
      </sup>
      , since AVX-support in operating systems refers to handling YMM register state.
      <sup class="reference" id="cite_ref-intel_vol1_3-1">
       <a href="#cite_note-intel_vol1-3">
        <span>
         [
        </span>
        3
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <h3>
      <span class="mw-headline" id="Applications">
       Applications
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Advanced_Vector_Extensions&amp;action=edit&amp;section=2" title="Edit section: Applications">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <ul>
      <li>
       Suitable for
       <a href="/wiki/Floating_point" title="Floating point">
        floating point
       </a>
       -intensive calculations in multimedia, scientific and financial applications (
       <a href="/wiki/Integer" title="Integer">
        integer
       </a>
       operations are expected in later extensions).
      </li>
      <li>
       Increases parallelism and throughput in floating point
       <a href="/wiki/SIMD" title="SIMD">
        SIMD
       </a>
       calculations.
      </li>
      <li>
       Reduces register load due to the non-destructive instructions.
      </li>
      <li>
       Improves Linux RAID software performance (required AVX2, AVX is not sufficient)
       <sup class="reference" id="cite_ref-5">
        <a href="#cite_note-5">
         <span>
          [
         </span>
         5
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
     </ul>
     <p>
      <a href="/wiki/Prime95" title="Prime95">
       Prime95
      </a>
      /MPrime, the software used for
      <a class="mw-redirect" href="/wiki/GIMPS" title="GIMPS">
       GIMPS
      </a>
      , started using the AVX instructions since version 27.x.
     </p>

        </code>
       </td>

       </td>

        <code>
         VZEROALL
        </code>

     <h2>
      <span class="mw-headline" id="Advanced_Vector_Extensions_2">
       <span id="AVX2">
       </span>
       Advanced Vector Extensions 2
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Advanced_Vector_Extensions&amp;action=edit&amp;section=7" title="Edit section: Advanced Vector Extensions 2">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Advanced Vector Extensions 2 (AVX2), also known as
      <b>
       Haswell New Instructions
      </b>
      ,
      <sup class="reference" id="cite_ref-avx2_2-1">
       <a href="#cite_note-avx2-2">
        <span>
         [
        </span>
        2
        <span>
         ]
        </span>
       </a>
      </sup>
      is an expansion of the AVX instruction set introduced in Intel's
      <a href="/wiki/Haswell_(microarchitecture)" title="Haswell (microarchitecture)">
       Haswell microarchitecture
      </a>
      . AVX2 makes the following additions:
     </p>
     <ul>
      <li>
       expansion of most vector integer SSE and AVX instructions to 256 bits
      </li>
      <li>
       three-operand general-purpose bit manipulation and multiply
      </li>
      <li>
       three-operand
       <a href="/wiki/FMA_instruction_set" title="FMA instruction set">
        fused multiply-accumulate
       </a>
       support (FMA3)
      </li>
      <li>
       <a href="/wiki/Gather-scatter_(vector_addressing)" title="Gather-scatter (vector addressing)">
        Gather
       </a>
       support, enabling vector elements to be loaded from non-contiguous memory locations
      </li>
      <li>
       DWORD- and QWORD-granularity any-to-any permutes
      </li>
      <li>
       vector shifts.
      </li>
     </ul>

     <h2>
      <span class="mw-headline" id="AVX-512">
       AVX-512
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Advanced_Vector_Extensions&amp;action=edit&amp;section=10" title="Edit section: AVX-512">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <div class="hatnote relarticle mainarticle">
      Main article:
      <a href="/wiki/AVX-512" title="AVX-512">
       AVX-512
      </a>
     </div>
     <p>
      <i>
       AVX-512
      </i>
      are 512-bit extensions to the 256-bit Advanced Vector Extensions SIMD instructions for x86 instruction set architecture proposed by
      <a class="mw-redirect" href="/wiki/Intel_Corporation" title="Intel Corporation">
       Intel
      </a>
      in July 2013, and scheduled to be supported in 2015 with Intel's
      <a class="mw-redirect" href="/wiki/Knights_Landing_(microarchitecture)" title="Knights Landing (microarchitecture)">
       Knights Landing
      </a>
      processor.
      <sup class="reference" id="cite_ref-reinders512_1-1">
       <a href="#cite_note-reinders512-1">
        <span>
         [
        </span>
        1
        <span>
         ]
        </span>
       </a>
      </sup>
     </p>
     <p>
      AVX-512 consists of multiple extensions not all meant to be supported by all processors implementing them. Only the core extension AVX-512F (AVX-512 Foundation) is required by all implementations.
     </p>
     <p>
      The instruction set consists of the following:
     </p>
     <ul>
      <li>
       AVX-512 Foundation  expands most 32-bit and 64-bit based AVX instructions with EVEX coding scheme to support 512-bit registers, operation masks, parameter broadcasting, and embedded rounding and exception control
      </li>
      <li>
       AVX-512 Conflict Detection Instructions (CDI)  efficient conflict detection to allow more loops to be vectorized, supported by Knights Landing
       <sup class="reference" id="cite_ref-reinders512_1-2">
        <a href="#cite_note-reinders512-1">
         <span>
          [
         </span>
         1
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       AVX-512 Exponential and Reciprocal Instructions (ERI)  exponential and reciprocal operations designed to help implement transcendental operations, supported by Knights Landing
       <sup class="reference" id="cite_ref-reinders512_1-3">
        <a href="#cite_note-reinders512-1">
         <span>
          [
         </span>
         1
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       AVX-512 Prefetch Instructions (PFI)  new prefetch capabilities, supported by Knights Landing
       <sup class="reference" id="cite_ref-reinders512_1-4">
        <a href="#cite_note-reinders512-1">
         <span>
          [
         </span>
         1
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       AVX-512 Byte and Word Instructions (BW)  extends AVX-512 to cover 8-bit and 16-bit integer operations
       <sup class="reference" id="cite_ref-reinders512b_20-0">
        <a href="#cite_note-reinders512b-20">
         <span>
          [
         </span>
         20
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       AVX-512 Doubleword and Quadword Instructions (DQ)  enhanced 32-bit and 64-bit integer operations
       <sup class="reference" id="cite_ref-reinders512b_20-1">
        <a href="#cite_note-reinders512b-20">
         <span>
          [
         </span>
         20
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
      <li>
       AVX-512 Vector Length Extensions (VL)  extends most AVX-512 operations to also operate on XMM (128-bit) and YMM (256-bit) registers
       <sup class="reference" id="cite_ref-reinders512b_20-2">
        <a href="#cite_note-reinders512b-20">
         <span>
          [
         </span>
         20
         <span>
          ]
         </span>
        </a>
       </sup>
      </li>
     </ul>
