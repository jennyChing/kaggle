JavaScript code should be executed in 
"strict mode".
        </p>
        <hr/>
        <h2>
         The "use strict" Directive
        </h2>
        <p>
         The "use strict" directive is new in JavaScript 1.8.5 (ECMAScript version 5).
        </p>
        <p>
         It is not a statement, but a literal expression, ignored by earlier versions 
of JavaScript.
        </p>
        <p>
         The purpose of "use strict" is to indicate that the code should be executed in "strict mode".
        </p>
        <p>
         With strict mode, you can not, for example, use undeclared variables.
        </p>
        <table class="lamp">
         <tr>
          <th style="width:34px">
           <img alt="Note" src="/images/lamp.jpg" style="height:32px;width:32px"/>
          </th>
          <td>
           Strict mode is supported in:
           <br/>
           Internet Explorer from version 10. Firefox from version 4.
           <br/>
           Chrome from version 13. Safari from version 5.1. Opera from version 12.
          </td>
         </tr>
        </table>
        <hr/>
        <h2>
         Declaring Strict Mode
        </h2>
        <p>
         Strict mode is declared by adding "use strict"; to the beginning of a 
JavaScript file, or a JavaScript function.
        </p>
        <p>
         Declared at the beginning of a JavaScript file, it has global 
scope (all code will execute in strict mode).
        </p>
        <p>
         Declared inside a function, it has local scope (only the code inside the function is 
in strict mode).
        </p>
        <p>
         Global declaration:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          x = 3.14; // This will 
 cause an error
          <br/>
          myFunction(); // This will also cause an error
          <br/>
          <br/>
          function myFunction() {
          <br/>
          x = 3.14;
          <br/>
          }
          <br/>
         </div>
         <br/>
         <a class="tryitbtn" href="tryit.asp?filename=tryjs_strict_global" target="_blank">
          Try it Yourself 
         </a>
        </div>
        <p>
         Local declaration:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          x = 3.14; // This will not cause an error.
          <br/>
          myFunction(); // This will cause an error
          <br/>
          <br/>
          function 
 myFunction() {
          <br/>
          "use strict";
          <br/>
          x = 3.14;
          <br/>
          }
          <br/>
         </div>
         <br/>
         <a class="tryitbtn" href="tryit.asp?filename=tryjs_strict_local" target="_blank">
          Try it Yourself 
         </a>
        </div>
        <hr/>
        <h2>
         The "use strict"; Syntax
        </h2>
        <p>
         The syntax, for declaring strict mode, was designed to be compatible with 
older versions of JavaScript.
        </p>
        <p>
         Compiling a numeric literal (4 + 5;) or a string literal ("John Doe";) in a 
JavaScript program has no side effects. It simply compiles to a non existing 
variable and dies.
        </p>
        <p>
         So "use strict"; only matters to new compilers that "understand" the meaning 
of it.
        </p>
        <hr/>
        <h2>
         Why Strict Mode?
        </h2>
        <p>
         Strict mode makes it easier to write "secure" JavaScript.
        </p>
        <p>
         Strict mode changes previously accepted "bad syntax" into real errors.
        </p>
        <p>
         As an example, in normal JavaScript, mistyping a variable name creates a new 
global variable. In strict mode, this will throw an error, making it impossible 
to accidentally create a global variable.
        </p>
        <p>
         In normal JavaScript, a developer will not receive any error feedback 
assigning values to non-writable properties.
        </p>
        <p>
         In strict mode, any assignment to a non-writable property, a getter-only 
property, a non-existing property, a non-existing variable, or a non-existing 
object, will throw an error.
        </p>
        <hr/>
        <h2>
         Not Allowed in Strict Mode
        </h2>
        <p>
         Using a variable (property or object) without declaring it, is not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          x = 3.14; // This 
 will cause an error (if x has not been declared)
         </div>
        </div>
        <p>
         Deleting a variable, a function, or an argument, is not allowed.
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          x = 3.14;
          <br/>
          delete x;  // This 
 will cause an error
         </div>
        </div>
        <p>
         Defining a property more than once, is not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          var x = {p1:10, p1:20}; // This will 
 cause an error
         </div>
        </div>
        <p>
         Duplicating a parameter name is not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          function x(p1, p1) {}; // This will 
 cause an error
         </div>
        </div>
        <p>
         Octal numeric literals and escape characters are not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          var x = 010; // This 
 will cause an error
          <br/>
          var y = \010; // This 
 will cause an error
         </div>
        </div>
        <p>
         Writing to a read-only property is not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          var obj = {};
          <br/>
          obj.defineProperty(obj, "x", {value:0, writable:false});
          <br/>
          <br/>
          obj.x = 3.14;  // This 
 will cause an error
          <br/>
         </div>
        </div>
        <p>
         Writing to a get-only property is not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          var obj = {get x() 
	{return 0} };
          <br/>
          <br/>
          obj.x = 3.14; // This 
 will cause an error
         </div>
        </div>
        <p>
         Deleting an undeletable property is not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          delete Object.prototype; // This will cause an error
         </div>
        </div>
        <p>
         The string "eval" cannot be used as a variable:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          var eval = 3.14; // This will cause an error
         </div>
        </div>
        <p>
         The string "arguments" cannot be used as a variable:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          var arguments = 3.14; // This will 
 cause an error
         </div>
        </div>
        <p>
         The with statement is not allowed:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          with (Math){x = cos(2)}; // This will cause an error
         </div>
        </div>
        <p>
         For security reasons, eval() is not allowed to create  
variables in the scope from which it was called:
        </p>
        <div class="example">
         <div class="example_code notranslate jsHigh">
          "use strict";
          <br/>
          eval ("var x = 2");
          <br/>
          alert (x) // This 
 will cause an error
         </div>
        </div>
        <p>
         In function calls like f(), the this value was the global object. In strict mode, it is now undefined.
        </p>
        <p>
         Future reserved keywords are not allowed. These are:
        </p>
        <ul>
         <li>
          implements
         </li>
         <li>
          interface
         </li>
         <li>
          package
         </li>
         <li>
          private
         </li>
         <li>
          protected
         </li>
         <li>
          public
         </li>
         <li>
          static
         </li>
         <li>
          field
         </li>
        </ul>
        <hr/>

         Watch Out!
