        <p>
         WordPress is easy to configure and install, ready to go as soon as you publish your blog.
        </p>
        <p>
         But relying on WordPress default options might result in a loss on your behalf:
        </p>
        <ul>
         <li>
          Less functionality
         </li>
         <li>
          Bored users (who see the same stuff on every WP sites they stumble upon)
         </li>
         <li>
          Drops in search engine rankings.
         </li>
        </ul>
        <p>
         You have a flexible tool in your hands, that you can tweak to turn it into something pleasant to use and efficient to rely upon. Why not spice things up for your users (and yourself)?
        </p>
        <p>
         This article wants to help you make your WordPress-based website fun and helpful for your users by adding interesting
         <strong>
          code snippets
         </strong>
         that will also enhance your SEO.
        </p>
        <p>
         Give them a try and let me know how they worked out for you! ;)
        </p>
        <h2>
         9 Codes To Enhance Your WordPress Blog
        </h2>
        <p>
         I will offer you two types of code snippets in this post:
        </p>
        <ol>
         <li>
          Codex codes
         </li>
         <li>
          Code snippets and widgets
         </li>
        </ol>
        <p>
         Codex codes are PHP snippets I wrote myself (with the help of my fiance, Simone) strictly using the
         <a href="http://codex.wordpress.org/" target="_blank">
          WordPress.org Codex
         </a>
         as a reference.
        </p>
        <p>
         Code snippets and widgets are ready-made pieces of PHP code (or WordPress plugins) that are freely available on other websites and on WordPress.org, but I also included snippets my fiance and I wrote from scratch using open and Codex references.
        </p>
        <p>
         Both types of codes are easy to implement and install, but you may want to start with widgets and plugins if youre unsure how to manipulate code.
        </p>
        <p>
         I understand that 9 code snippets may not be enough for those of you who like to experiment with lots and lots of different code, so Im sure our Robert Ludroskys
         <a href="http://www.webhostingsecretrevealed.net/blog/web-design/25-handy-code-snippets-for-wordpress-developers/">
          25 handy code snippets for WordPress Developers
         </a>
         will satiate your hunger (at least for a while). ;)
        </p>
        <h2>
         Codex Codes
        </h2>
        <h3>
         1. Differentiate Sticky Posts
        </h3>
        <p>
         To make WordPress check if your current post is a sticky post  and display it accordingly  you can use the Codex boolean function:
        </p>
        <p>
         <code>
          &lt;?php is_sticky(); ?&gt;
         </code>
        </p>
        <p>
         The function alone will only return TRUE or FALSE values, so what you can do here if write a conditional construct (if/else) to manage your sticky posts. An example:
        </p>
        <pre>
         &lt;?php if is_sticky() {
  the_title();
  the_time('M, d, Y');
  the_excerpt();
}
else {
  include 'post-template.php';
}
        </pre>
        <p>
         In this sample usage, I displayed the sticky post as a box containing only the title, the post date and the excerpt (not the entire post), while the standard template for the other posts is contained in post-template.php.
        </p>
        <p>
         Each type of post will have its own CSS stylesheet, but were not defining styles here; only templates.
        </p>
        <p>
         Single.php is your default template for single posts (youll find it included with every default WP theme).
        </p>
        <p>
         If you have developed your own WP theme but you have setup no templates for single posts, you can follow the Codex guide
         <a href="http://codex.wordpress.org/Theme_Development#Single_Post_.28single.php.29" target="_blank">
          here
         </a>
         .
        </p>
        <h3>
         2. Displaying Post URL
        </h3>
        <p>
         This one might be fun to add to your template. If you want your readers to get the URI for each of your pages (or posts), just use the following tag within your single.php, page.php or even index.php template:
        </p>
        <pre>
         &lt;a href="&lt;?php echo get_page_link();?&gt;"&gt;&lt;?php echo get_page_link(); ?&gt;&lt;/a&gt;
        </pre>
        <p>
         Instead of using the_permalink, thats the most obvious choice and works for all cases, WP allows you to use two alternative forms for posts and pages:
        </p>
        <pre>
         &lt;?php echo get_post_permalink(); ?&gt;
         <code>
          &lt;?php echo
          <code>
           get_page_link();
          </code>
          ?&gt;
         </code>
        </pre>
        <p>
         For example, your page template can include:
        </p>
        <pre>
         &lt;p&gt;Link: &lt;a href="&lt;?php echo get_page_link();?&gt;"&gt;&lt;?php echo get_page_link(); ?&gt;&lt;/a&gt;&lt;/p&gt;
        </pre>
        <h3>
         3. Retrieve and Display Post ID Number
        </h3>
        <p>
         Even when you setup your permalink structure to be user- and SEO-friendly, you can still show your visitors the post ID by adding this simple function to your meta line:
        </p>
        <pre>
         <code>
          <code>
           &lt;?phpthe_ID();?
          </code>
          &gt;
         </code>
        </pre>
        <p>
         Sample usage:
        </p>
        <pre>
         <code>
          &lt;p class="meta"&gt;Posted by Author's Name. Post ID is &lt;?phpthe_ID();?&gt;&lt;/p&gt;
         </code>
        </pre>
        <h3>
         4. List Categories By ID
        </h3>
        <p>
         The WordPress standard function is:
        </p>
        <p>
         <code>
          &lt;?phpget_all_category_ids()?&gt;
         </code>
        </p>
        <p>
         Here is an example I used on one of my blogs, written using code from the
         <a href="http://codex.wordpress.org/Function_Reference/get_all_category_ids" target="_blank">
          Codex
         </a>
         and
         <a href="http://stackoverflow.com/questions/14816552/wordpress-get-category-link" target="_blank">
          a topic on StackOverflow
         </a>
         :
        </p>
        <pre>
         &lt;?php
 $category_ids = get_all_category_ids();
 foreach($category_ids as $cat_id) {
   $cat_name = get_cat_name($cat_id);
 $category_link = get_category_link($cat_id); //we need the cat link for the URL to work!
 echo "&lt;a href=\"{$category_link}\"&gt;{$cat_id}&lt;/a&gt;: {$cat_name}&lt;br/&gt;";
 }
 ?&gt;
        </pre>
        <p>
          and how its displayed:
        </p>
        <div style="text-align: center;">
         <img alt="category WP code" height="354" src="http://whsr.webrevenueinc1.netdna-cdn.com/wp-content/uploads/2014/12/categorycode.png" width="286"/>
        </div>
        <p>
         Note: get_all_category_ids() is now a deprecated function, but you can still use it without problems, like I do on my blog (screenshot above). However, if you wish to use the new function for this snippet, visit the
         <a href="http://codex.wordpress.org/Function_Reference/get_terms" target="_blank">
          get_terms()
         </a>
         page of the Codex.
        </p>
        <p>
         <strong>
          What does this code do?
         </strong>
        </p>
        <p>
         The code gets all category IDs and the category name for each ID, then it links the ID to the category link, while it displays the category name after the :  hence the
         <tt>
          echo()
         </tt>
         expression
         <tt>
          &lt;a href=\"{$category_link}\"&gt;{$cat_id}&lt;/a&gt;: {$cat_name}&lt;br/&gt;
         </tt>
         .
        </p>
        <h4>
         5. A Detailed Users/Authors Page For Your Blog
        </h4>
        <p>
         Have you ever wanted to build a custom page that would display all your blog authors or users without having to rely on a plugin?
        </p>
        <p>
         Well, you
         <em>
          can
         </em>
         create your own custom Authors/Users page with only a text editor, an FTP uploader and your beloved WordPress Dashboard.
        </p>
        <p>
         First of all, you need to create a copy of your page.php file. Rename your copy into any name  I named mine users.php.
        </p>
        <p>
         Add
         <a href="http://codex.wordpress.org/Page_Templates" target="_blank">
          the template tag
         </a>
         to the top of this template for WordPress to recognize it as a template, then go to your Dashboard
         <em>
          -&gt; Pages -&gt; Add New
         </em>
         and create a page for your list of Users/Authors. Return to the Pages list, click Quick Edit under your new page title and select your new template from the Template drop-down menu. Save your changes.
        </p>
        <p>
         Now open your users.php (or whatever you named it) file and apply the following code:
        </p>
        <pre>
         <code>
          &lt;?php
 $result=count_users();
 echo'Thereare',$result['total_users'],'totalusers';
 foreach($result['avail_roles']as$role=&gt;$count)
 echo',',$count,'are',$role,'s';
 echo'.';
 ?&gt;
         </code>
        </pre>
        <p>
         This code uses the
         <a href="http://codex.wordpress.org/Function_Reference/count_users" target="_blank">
          count_users
         </a>
         function is listed as default usage in the Codex. You can view a live example of this code at
         <a href="http://robocity.in/users/" target="_blank">
          http://robocity.in/users/
         </a>
         .
        </p>
        <p>
         After that code, lets add something about the blog admin:
        </p>
        <pre>
         &lt;?php printf( __( 'Number of posts published by user "Leaders": %d', 'text-dom-here' ), count_user_posts(1) ); ?&gt;
        </pre>
        <p>
         I used the
         <a href="http://codex.wordpress.org/Function_Reference/count_user_posts" target="_blank">
          count_user_posts
         </a>
         function here, following the usage format suggested by the Codex.
        </p>
        <p>
         The function
         <a href="http://codex.wordpress.org/Function_Reference/get_userdata" target="_blank">
          get_userdata
         </a>
         is good to show the correlation between a certain username and the real name of the person using it. See below:
        </p>
        <pre>
         &lt;?php $user_info = get_userdata(1);
 $username = $user_info-&gt;user_login;
 $first_name = $user_info-&gt;first_name;
 $last_name = $user_info-&gt;last_name;
 echo "$first_name $last_name logs into their WordPress site with the username of $username.";
?&gt;
        </pre>
        <p>
         Now, this is the most interesting function you can use on your Authors/Users page  the
         <a href="http://codex.wordpress.org/Function_Reference/get_users" target="_blank">
          get_users
         </a>
         function:
        </p>
        <pre>
         &lt;?php
$blogusers = get_users( 'blog_id=1&amp;orderby=nicename&amp;role=administrator' );
// Array of WP_User objects.
foreach ( $blogusers as $user ) {
 echo '&lt;span&gt;' . esc_html($user-&gt;user_login) . '&nbsp;-&nbsp;' . esc_html($user-&gt;user_email) . '&lt;/span&gt;&lt;br/&gt;';
}
?&gt;
        </pre>
        <p>
         This function will retrieve a list of users according to certain parameters that you can configure in the code. In the example above (taken from the Codex page and edited to fit my Users page at Robocity.in), I want to get a list of administrators for blog ID 1 (current blog) ordered by nicename, and for each user, I want to display username and email in a sentence of the type Username  name@example.com.
        </p>
        <p>
         If you like to show logged in users something about themselves, you may use the
         <a href="http://codex.wordpress.org/Function_Reference/get_currentuserinfo" target="_blank">
          get_currentuserinfo
         </a>
         function to retrieve, say, the users name and address. I didnt use this function in my example, but creativity doesnt know limits, right? ;)
        </p>
        <h2>
         Code Snippets &amp; Widgets
        </h2>
        <h3>
         6. Alternative Archives!
        </h3>
        <p>
         <img alt="nicer blog archives" class="alignright size-full wp-image-11902" height="137" src="http://whsr.webrevenueinc1.netdna-cdn.com/wp-content/uploads/2014/12/nicearchives.png" width="216"/>
         Imagine if you could replace your old sidebar Archives list  that keeps growing as your blog grows and forces users to scroll  with a snippet like the one you can see here on the right.
        </p>
        <p>
         Wouldnt it make things easier for your readers to catch your blog age and depth of archives at a glance? :)
        </p>
        <p>
         My fianc and I wrote this simple snippet using the get_post from the Codex and a simple dropdown menu for the list of archives. We created our own functions to make the idea work out well.
        </p>
        <pre>
         &lt;?php
 function formatPostDate($postId, $format = '%F') {
 $post = get_post($postId);
 $output = strftime($format, strtotime($post-&gt;post_date));
 return $output;
 }

function getLastPostID() {
 list($post) = get_posts(array('posts_per_page' =&gt; 1));
 return $post-&gt;ID;
 }
 ?&gt;

&lt;p&gt;This blog has posts since &lt;?php echo formatPostDate(1, '%B %e, %Y'); ?&gt;.&lt;br/&gt;
 Last post was published on &lt;?php echo formatPostDate(getLastPostID(), '%B %e, %Y'); ?&gt;.&lt;/p&gt;

&lt;p&gt;Do you wish to visit a specific archive?&lt;/p&gt;

&lt;select name="archive-dropdown" onchange="document.location.href=this.options[this.selectedIndex].value;"&gt;
 &lt;option value=""&gt;&lt;?php echo esc_attr( __( 'Select Archive' ) ); ?&gt;&lt;/option&gt;
 &lt;?php wp_get_archives( array( 'type' =&gt; 'monthly', 'format' =&gt; 'option', 'show_post_count' =&gt; 1 ) ); ?&gt;
 &lt;/select&gt;
        </pre>
        <p>
         <strong>
          What does this code do?
         </strong>
        </p>
        <p>
         The function formatPostDate takes the posts ID and date format and returns a date in a format of your choice. The second function, getLastPostID, takes an array of posts containing only one post and returns its ID.
        </p>
        <p>
         To get the first post of the blog  generally with ID=1, we write echo formatPostDate(1, %B %e, %Y) in the first paragraph: this code uses the formatPostDate function and takes as parameters the post ID #1 and a standard English format for the date, and returns the date.
        </p>
        <p>
         To get the last post of the blog, we use formatPostDate(getLastPostID(), %B %e, %Y) to get the ID of the last post and a date format (again, in English) and print the date on screen.
        </p>

        <h3>
         8. Use the if/else construct to manipulate your page sidebar content
        </h3>
        <p>
         This code snippet is based on WordPress Codex, but I placed it in this section because it allows for much creativity and you may be able to find ready-made snippets on the Web.
        </p>
        <p>
         Suppose that you want to use multiple sidebars on your site. You may fill your sidebar with divs and sections, but the more that file grows (even if you use widgets) the heavier it gets.
        </p>
        <p>
         The Codex comes to you rescue with the
         <a href="http://codex.wordpress.org/Function_Reference/get_sidebar" target="_blank">
          get_sidebar($name)
         </a>
         function. The usage is simple:
        </p>
        <ol>
         <li>
          Create (say) a sidebar for your Quotes snippets. Name it sidebar-quotes.php
         </li>
         <li>
          In the main sidebar.php file (or the header or index file, according to your WP theme structure) write get_sidebar(quotes); to include the sidebar-quotes.php file.
         </li>
        </ol>
        <p>
         You will get a slimmer sidebar with calls to other sub-sidebars, that you can optimize with if/else constructs if you know a bit of PHP coding for WordPress.
        </p>
        <p>
         You may not know this, but when you use the generic get_sidebar() code, that will include sidebar.php because you didnt add a $name argument. This is the default usage. When you add a -name after sidebar*.php, you can call that name addon with the $name arg.
        </p>
        <p>
         If you have ads, special offers or sidebar functions that you want to display on certain pages of your blog or code that will overload your standard sidebar, you will find this code snippet a life saver for UX.
        </p>
        <h3>
         9. A tool for integrating code snippets into your WordPress site
        </h3>
        <p>
         <img alt="codesnippets screenshot" class=" size-large wp-image-11901 aligncenter" height="371" src="http://whsr.webrevenueinc1.netdna-cdn.com/wp-content/uploads/2014/11/codesnippets-750x371.png" width="750"/>
        </p>
        <p>
         <a href="http://wordpress.org/extend/plugins/code-snippets/" target="_blank">
          Code Snippets
         </a>
         is a free WordPress plugin created by Shea Bange that allows you to easily add code snippets to your blog.
        </p>
        <p>
         The nice thing about this plugin is that you no longer have to edit your
         <tt>
          functions.php
         </tt>
         file, but you can add custom code directly from your WordPress Dashboard. Basically, you can add code snippets like you would add a new post or page.
        </p>
        <p>
         <a href="http://premium.wpmudev.org/blog/wordpress-code-snippets/" target="_blank">
          Tom Ewer at WPMUDev.org
         </a>
         wrote an interesting introduction to this plugin, but the most useful resource I found so far for Code Snippets is
         <a href="https://wordpress.org/support/plugin/code-snippets" target="_blank">
          the plugin support forum at WordPress.org
         </a>
         .
