          <h1>
           <a aria-hidden="true" class="anchor" href="#basic-coding-standard" id="user-content-basic-coding-standard">
            <span class="octicon octicon-link">
            </span>
           </a>
           Basic Coding Standard
          </h1>
          <p>
           This section of the standard comprises what should be considered the standard
coding elements that are required to ensure a high level of technical
interoperability between shared PHP code.
          </p>
          <p>
           The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD",
"SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be
interpreted as described in
           <a href="http://www.ietf.org/rfc/rfc2119.txt">
            RFC 2119
           </a>
           .
          </p>
          <h2>
           <a aria-hidden="true" class="anchor" href="#1-overview" id="user-content-1-overview">
            <span class="octicon octicon-link">
            </span>
           </a>
           1. Overview
          </h2>
          <ul>
           <li>
            <p>
             Files MUST use only
             <code>
              &lt;?php
             </code>
             and
             <code>
              &lt;?=
             </code>
             tags.
            </p>
           </li>
           <li>
            <p>
             Files MUST use only UTF-8 without BOM for PHP code.
            </p>
           </li>
           <li>
            <p>
             Files SHOULD
             <em>
              either
             </em>
             declare symbols (classes, functions, constants, etc.)
             <em>
              or
             </em>
             cause side-effects (e.g. generate output, change .ini settings, etc.)
but SHOULD NOT do both.
            </p>
           </li>
           <li>
            <p>
             Namespaces and classes MUST follow an "autoloading" PSR: [
             <a href="https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-0.md">
              PSR-0
             </a>
             ,
             <a href="https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-4-autoloader.md">
              PSR-4
             </a>
             ].
            </p>
           </li>
           <li>
            <p>
             Class names MUST be declared in
             <code>
              StudlyCaps
             </code>
             .
            </p>
           </li>
           <li>
            <p>
             Class constants MUST be declared in all upper case with underscore separators.
            </p>
           </li>
           <li>
            <p>
             Method names MUST be declared in
             <code>
              camelCase
             </code>
             .
            </p>
           </li>
          </ul>
          <h2>
           <a aria-hidden="true" class="anchor" href="#2-files" id="user-content-2-files">
            <span class="octicon octicon-link">
            </span>
           </a>
           2. Files
          </h2>
          <h3>
           <a aria-hidden="true" class="anchor" href="#21-php-tags" id="user-content-21-php-tags">
            <span class="octicon octicon-link">
            </span>
           </a>
           2.1. PHP Tags
          </h3>
          <p>
           PHP code MUST use the long
           <code>
            &lt;?php ?&gt;
           </code>
           tags or the short-echo
           <code>
            &lt;?= ?&gt;
           </code>
           tags; it
MUST NOT use the other tag variations.
          </p>
          <h3>
           <a aria-hidden="true" class="anchor" href="#22-character-encoding" id="user-content-22-character-encoding">
            <span class="octicon octicon-link">
            </span>
           </a>
           2.2. Character Encoding
          </h3>
          <p>
           PHP code MUST use only UTF-8 without BOM.
          </p>
          <h3>
           <a aria-hidden="true" class="anchor" href="#23-side-effects" id="user-content-23-side-effects">
            <span class="octicon octicon-link">
            </span>
           </a>
           2.3. Side Effects
          </h3>
          <p>
           A file SHOULD declare new symbols (classes, functions, constants,
etc.) and cause no other side effects, or it SHOULD execute logic with side
effects, but SHOULD NOT do both.
          </p>
          <p>
           The phrase "side effects" means execution of logic not directly related to
declaring classes, functions, constants, etc.,
           <em>
            merely from including the
file
           </em>
           .
          </p>
          <p>
           "Side effects" include but are not limited to: generating output, explicit
use of
           <code>
            require
           </code>
           or
           <code>
            include
           </code>
           , connecting to external services, modifying ini
settings, emitting errors or exceptions, modifying global or static variables,
reading from or writing to a file, and so on.
          </p>
          <p>
           The following is an example of a file with both declarations and side effects;
i.e, an example of what to avoid:
          </p>
          <div class="highlight highlight-php">
           <pre>
            <span class="pl-pse">
             &lt;?php
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // side effect: change ini settings
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-c1">
              ini_set
             </span>
             (
             <span class="pl-s">
              <span class="pl-pds">
               '
              </span>
              error_reporting
              <span class="pl-pds">
               '
              </span>
             </span>
             ,
             <span class="pl-c1">
              E_ALL
             </span>
             );
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // side effect: loads a file
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              include
             </span>
             <span class="pl-s">
              <span class="pl-pds">
               "
              </span>
              file.php
              <span class="pl-pds">
               "
              </span>
             </span>
             ;
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // side effect: generates output
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-c1">
              echo
             </span>
             <span class="pl-s">
              <span class="pl-pds">
               "
              </span>
              &lt;html&gt;
              <span class="pl-cce">
               \n
              </span>
              <span class="pl-pds">
               "
              </span>
             </span>
             ;
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // declaration
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              function
             </span>
             <span class="pl-en">
              foo
             </span>
             ()
            </span>
            <span class="pl-s1">
             {
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // function body
             </span>
            </span>
            <span class="pl-s1">
             }
            </span>
           </pre>
          </div>
          <p>
           The following example is of a file that contains declarations without side
effects; i.e., an example of what to emulate:
          </p>
          <div class="highlight highlight-php">
           <pre>
            <span class="pl-pse">
             &lt;?php
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // declaration
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              function
             </span>
             <span class="pl-en">
              foo
             </span>
             ()
            </span>
            <span class="pl-s1">
             {
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // function body
             </span>
            </span>
            <span class="pl-s1">
             }
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // conditional declaration is *not* a side effect
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              if
             </span>
             (
             <span class="pl-k">
              !
             </span>
             <span class="pl-c1">
              function_exists
             </span>
             (
             <span class="pl-s">
              <span class="pl-pds">
               '
              </span>
              bar
              <span class="pl-pds">
               '
              </span>
             </span>
             )) {
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              function
             </span>
             <span class="pl-en">
              bar
             </span>
             ()
            </span>
            <span class="pl-s1">
             {
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // function body
             </span>
            </span>
            <span class="pl-s1">
             }
            </span>
            <span class="pl-s1">
             }
            </span>
           </pre>
          </div>
          <h2>
           <a aria-hidden="true" class="anchor" href="#3-namespace-and-class-names" id="user-content-3-namespace-and-class-names">
            <span class="octicon octicon-link">
            </span>
           </a>
           3. Namespace and Class Names
          </h2>
          <p>
           Namespaces and classes MUST follow an "autoloading" PSR: [
           <a href="https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-0.md">
            PSR-0
           </a>
           ,
           <a href="https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-4-autoloader.md">
            PSR-4
           </a>
           ].
          </p>
          <p>
           This means each class is in a file by itself, and is in a namespace of at
least one level: a top-level vendor name.
          </p>
          <p>
           Class names MUST be declared in
           <code>
            StudlyCaps
           </code>
           .
          </p>
          <p>
           Code written for PHP 5.3 and after MUST use formal namespaces.
          </p>
          <p>
           For example:
          </p>
          <div class="highlight highlight-php">
           <pre>
            <span class="pl-pse">
             &lt;?php
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // PHP 5.3 and later:
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              namespace
             </span>
             <span class="pl-en">
              Vendor\Model
             </span>
             ;
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              class
             </span>
             <span class="pl-en">
              Foo
             </span>
            </span>
            <span class="pl-s1">
             {
            </span>
            <span class="pl-s1">
             }
            </span>
           </pre>
          </div>
          <p>
           Code written for 5.2.x and before SHOULD use the pseudo-namespacing convention
of
           <code>
            Vendor_
           </code>
           prefixes on class names.
          </p>
          <div class="highlight highlight-php">
           <pre>
            <span class="pl-pse">
             &lt;?php
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-c">
              // PHP 5.2.x and earlier:
             </span>
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              class
             </span>
             <span class="pl-en">
              Vendor_Model_Foo
             </span>
            </span>
            <span class="pl-s1">
             {
            </span>
            <span class="pl-s1">
             }
            </span>
           </pre>
          </div>
          <h2>
           <a aria-hidden="true" class="anchor" href="#4-class-constants-properties-and-methods" id="user-content-4-class-constants-properties-and-methods">
            <span class="octicon octicon-link">
            </span>
           </a>
           4. Class Constants, Properties, and Methods
          </h2>
          <p>
           The term "class" refers to all classes, interfaces, and traits.
          </p>
          <h3>
           <a aria-hidden="true" class="anchor" href="#41-constants" id="user-content-41-constants">
            <span class="octicon octicon-link">
            </span>
           </a>
           4.1. Constants
          </h3>
          <p>
           Class constants MUST be declared in all upper case with underscore separators.
For example:
          </p>
          <div class="highlight highlight-php">
           <pre>
            <span class="pl-pse">
             &lt;?php
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              namespace
             </span>
             <span class="pl-en">
              Vendor\Model
             </span>
             ;
            </span>
            <span class="pl-s1">
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              class
             </span>
             <span class="pl-en">
              Foo
             </span>
            </span>
            <span class="pl-s1">
             {
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              const
             </span>
             <span class="pl-c1">
              VERSION
             </span>
             <span class="pl-k">
              =
             </span>
             <span class="pl-s">
              <span class="pl-pds">
               '
              </span>
              1.0
              <span class="pl-pds">
               '
              </span>
             </span>
             ;
            </span>
            <span class="pl-s1">
             <span class="pl-k">
              const
             </span>
             <span class="pl-c1">
              DATE_APPROVED
             </span>
             <span class="pl-k">
              =
             </span>
             <span class="pl-s">
              <span class="pl-pds">
               '
              </span>
              2012-06-01
              <span class="pl-pds">
               '
              </span>
             </span>
             ;
            </span>
            <span class="pl-s1">
             }
            </span>
           </pre>
          </div>
          <h3>
           <a aria-hidden="true" class="anchor" href="#42-properties" id="user-content-42-properties">
            <span class="octicon octicon-link">
            </span>
           </a>
           4.2. Properties
          </h3>
          <p>
           This guide intentionally avoids any recommendation regarding the use of
           <code>
            $StudlyCaps
           </code>
           ,
           <code>
            $camelCase
           </code>
           , or
           <code>
            $under_score
           </code>
           property names.
          </p>
          <p>
           Whatever naming convention is used SHOULD be applied consistently within a
reasonable scope. That scope may be vendor-level, package-level, class-level,
or method-level.
          </p>
          <h3>
           <a aria-hidden="true" class="anchor" href="#43-methods" id="user-content-43-methods">
            <span class="octicon octicon-link">
            </span>
           </a>
           4.3. Methods
          </h3>
          <p>
           Method names MUST be declared in
           <code>
            camelCase()
           </code>
           .
          </p>
