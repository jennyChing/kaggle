   <h1 class="firstHeading" id="firstHeading" lang="en">
    ISO/IEC 8859-15
   </h1>
   <div class="mw-body-content" id="bodyContent">
    <div id="siteSub">
     From Wikipedia, the free encyclopedia

    <div id="contentSub">

    <div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en">
     <p>
      <b>
       ISO/IEC 8859-15:1999
      </b>
      ,
      <i>
       Information technology  8-bit single-byte coded graphic character sets  Part 15: Latin alphabet No. 9
      </i>
      , is part of the
      <a href="/wiki/ISO/IEC_8859" title="ISO/IEC 8859">
       ISO/IEC 8859
      </a>
      series of ASCII-based standard
      <a href="/wiki/Character_encoding" title="Character encoding">
       character encodings
      </a>
      , first edition published in 1999. It is informally referred to as
      <b>
       Latin-9
      </b>
      (and was for a while called Latin-0). It is similar to
      <a href="/wiki/ISO/IEC_8859-1" title="ISO/IEC 8859-1">
       ISO 8859-1
      </a>
      , and thus generally intended for Western European languages, but replaces some less common symbols with the
      <a href="/wiki/Euro_sign" title="Euro sign">
       euro sign
      </a>
      and some letters that were now deemed missing in part 1 for the target use.
     </p>
     <p>
      <b>
       ISO-8859-15
      </b>
      is the
      <a href="/wiki/Internet_Assigned_Numbers_Authority" title="Internet Assigned Numbers Authority">
       IANA
      </a>
      preferred charset name for this standard when supplemented with the
      <a href="/wiki/C0_and_C1_control_codes" title="C0 and C1 control codes">
       C0 and C1 control codes
      </a>
      from
      <a class="mw-redirect" href="/wiki/ISO/IEC_6429" title="ISO/IEC 6429">
       ISO/IEC 6429
      </a>
      .
     </p>
     <p>
      Microsoft has assigned
      <b>
       code page 28605
      </b>
      aka
      <b>
       Windows-28605
      </b>
      to ISO-8859-15.
     </p>
     <p>
      There were attempts to make ISO-8859-15 the default character set for 8-bit communication, but it was never able to supplant the popular ISO-8859-1. However, it did see some use as a character set for terminal or textual programs under Linux when the Euro sign was needed, but the use of full
      <a href="/wiki/UTF-8" title="UTF-8">
       UTF-8
      </a>
      (
      <a href="/wiki/Unicode" title="Unicode">
       Unicode
      </a>
      ) was not practical. All the printable characters from both ISO/IEC 8859-1 and ISO/IEC 8859-15 are also found in
      <a href="/wiki/Windows-1252" title="Windows-1252">
       Windows-1252
      </a>
      .
     </p>

     </p>

     <h2>
      <span class="mw-headline" id="Coverage">
       Coverage
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=ISO/IEC_8859-15&amp;action=edit&amp;section=2" title="Edit section: Coverage">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      ISO 8859-15 encodes what it refers to as "
      <a href="/wiki/Latin_alphabet" title="Latin alphabet">
       Latin alphabet
      </a>
      no. 9". This character set is used throughout the
      <a href="/wiki/Americas" title="Americas">
       Americas
      </a>
      ,
      <a href="/wiki/Western_Europe" title="Western Europe">
       Western Europe
      </a>
      ,
      <a href="/wiki/Oceania" title="Oceania">
       Oceania
      </a>
      , and much of
      <a href="/wiki/Africa" title="Africa">
       Africa
      </a>
      . It is also commonly used in most standard romanizations of East-Asian languages.
     </p>
     <p>
      Each character is encoded as a single eight-bit code value. These code values can be used in almost any data interchange system to communicate in the following languages:

     <h3>
      <span class="mw-headline" id="Coverage_of_punctuation_signs_and_apostrophes">
       Coverage of punctuation signs and apostrophes
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=ISO/IEC_8859-15&amp;action=edit&amp;section=3" title="Edit section: Coverage of punctuation signs and apostrophes">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <div class="hatnote">
      See also:
      <a href="/wiki/Latin_alphabets" title="Latin alphabets">
       Latin alphabets
      </a>
     </div>
     <p>
      For some languages listed above, the correct typographical
      <a href="/wiki/Quotation_mark#Table" title="Quotation mark">
       quotation marks
      </a>
      are missing, since only ,, ", and ' are included.
     </p>
     <p>
      Also, this encoding does not provide the correct character for the apostrophe, and oriented single high quotation marks, although some texts use the spacing grave accent and spacing acute accent, which are both part of ISO 8859-1, instead of the 6-shaped/9-shaped quotations marks or apostrophes (and this works reliably with some font styles, where all these characters are displayed as slanted wedge glyphs).
     </p>
