        <h2 id="how-to-configure-ssh-keys-authentication-with-putty-and-linux-server-in-quick-steps">
         How To Configure SSH Keys Authentication With PuTTY And Linux Server In 5 Quick Steps
        </h2>
        <p>
         This tutorial explains how you can replace password-based SSH authentication with key-based authentication which is more secure because only the people that own the key can log in. In this example, we're using
         <a href="http://www.chiark.greenend.org.uk/%7Esgtatham/putty/" target="_blank">
          PuTTY
         </a>
         as our SSH client on a Windows system.
        </p>
        <div style="width:336px;float:left; margin: 10px 15px 10px 0px;background-color:white;">
         <script type="text/javascript">
          GA_googleFillSlot('howtoforge_com_article_rectangle_a_300x250');
         </script>
        </div>
        <p>
        </p>
        <h3 id="-get-the-zip-file-with-all-putty-binaries">
         1. Get the zip file with all PuTTY binaries
        </h3>
        <p>
         <a href="http://the.earth.li/%7Esgtatham/putty/latest/x86/putty.zip">
          http://the.earth.li/~sgtatham/putty/latest/x86/putty.zip
         </a>
         <br/>
         <br/>
        </p>
        <h3 id="-generate-a-private-and-public-key-pair">
         2. Generate a private and public key pair
        </h3>
        <p>
         Open PuTTYgen.exe, press
         <span class="highlight">
          Generate
         </span>
         button, move mouse. Once the keys are generated, type your
         <span class="highlight">
          key passphrase
         </span>
         (choose a "hard to guess" one).
Save Public key. Save Private key.
         <br/>
         <br/>
         <img alt="Screen shot: PuTTY key generator" height="471" src="/images/ssh_key_authentication_putty_linux/putty-key-generator.png" width="483"/>
         <br/>
         <br/>
        </p>
        <h3 id="-configure-your-linux-server-create-user-save-public-key">
         3. Configure your Linux server (create user, save public key)
        </h3>
        <p>
         For this guide let's assume you regular login name is
         <span class="system">
          autotimesheet
         </span>
         (replace it with one that you use regularly).
        </p>
        <p>
         As root, on the shell, type:
        </p>
        <p class="command">
         adduser autotimesheet --disabled-password
        </p>
        <p>
         You will be asked to fill in some details such as the user's real name (empty string is fine).
        </p>
        <p>
         Now type:
        </p>
        <p class="command">
         su autotimesheet
         <br/>
         cd /home/autotimesheet
         <br/>
         mkdir .ssh
         <br/>
         chmod 700 .ssh
         <br/>
         cd .ssh
        </p>
        <p>
         Then in that folder, create and edit a file called
         <span class="system">
          authorized_keys2
         </span>
         .
        </p>
        <p>
         In there, cut/paste your public ssh key, on
         <span class="highlight">
          ONE LINE
         </span>
         (That is very important!!!)
        </p>
        <p>
         <u>
          Do not add
         </u>
         the
         <a class="__cf_email__" data-cfemail="ec89818d8580ac89948d819c8089c28f8381" href="/cdn-cgi/l/email-protection">
          [emailprotected]
         </a>
         <script cf-hash="f9e31" type="text/javascript">
          /* &lt;![CDATA[ */!function(){try{var t="currentScript"in document?document.currentScript:function(){for(var t=document.getElementsByTagName("script"),e=t.length;e--;)if(t[e].getAttribute("cf-hash"))return t[e]}();if(t&amp;&amp;t.previousSibling){var e,r,n,i,c=t.previousSibling,a=c.getAttribute("data-cfemail");if(a){for(e="",r=parseInt(a.substr(0,2),16),n=2;a.length-n;n+=2)i=parseInt(a.substr(n,2),16)^r,e+=String.fromCharCode(i);e=document.createTextNode(e),c.parentNode.replaceChild(e,c)}}}catch(u){}}();/* ]]&gt; */
         </script>
         at the end of the line.
         <br/>
         <u>
          Do not add
         </u>
         the
         <span class="highlight">
          BEGIN PUBLIC KEY
         </span>
         or
         <span class="highlight">
          END PUBLIC KEY
         </span>
         .
         <br/>
         <u>
          Do not add
         </u>
         the
         <span class="highlight">
          rsa-key-20090614
         </span>
         at the end.
         <br/>
         <u>
          Make sure
         </u>
         , there is
         <span class="highlight">
          ssh-rsa
         </span>
         at the beginning.
        </p>
        <div style="width:336px;float:left; margin: 10px 15px 10px 0px;background-color:white;">
         <script type="text/javascript">
          GA_googleFillSlot('howtoforge_com_article_rectangle_b_300x250');
         </script>
        </div>
        <p>
         It should be something like:
        </p>
        <p class="system">
         ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAIBzPeFQv+295iKzuuPH0jA9449pSHVpCwXvCR9EstmYh...
        </p>
        <p>
         Type:
        </p>
        <p class="command">
         chmod 600 authorized_keys2
        </p>
        <p>
        </p>
        <h3 id="-add-private-key-to-putty-ssh-authentication-agent">
         4. Add Private key to PuTTY SSH authentication agent
        </h3>
        <p>
         Start the PuTTY SSH authentication agent PAGEANT.EXE. When started, right-click its syspanel icon...
        </p>
        <p>
         <img alt="Screen shot: Pageant - syspanel icon" height="29" src="/images/ssh_key_authentication_putty_linux/pageant-icon.png" width="20"/>
        </p>
        <p>
         ... and select "add key".
Select your private key file, and type in your passphrase when you add the key.
The Pageant works as a passphrase keeper.
        </p>
        <p>
        </p>
        <h3 id="-use-putty-to-connect-to-your-server">
         5. Use PuTTY to connect to your server
        </h3>
        <p>
         Open PuTTY and connect as
         <span class="system">
          autotimesheet@yourserver
         </span>
         :
        </p>
        <p>
         <img alt="Screen shot: PuTTY - connect to a server" height="442" src="/images/ssh_key_authentication_putty_linux/putty.png" width="426"/>
        </p>
        <p>
         See that you log in without a password ...
        </p>
        <p>
         <img alt="Screen shot: PuTTY - Authenticating with public key from agent" height="179" src="/images/ssh_key_authentication_putty_linux/putty-autotimesheet.png" width="543"/>
        </p>
