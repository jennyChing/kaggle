         <p>
          <img alt="think-problem-solution-work-ss-1920" class="has-border aligncenter wp-image-218908 size-large" height="450" src="http://searchengineland.com/figz/wp-content/seloads/2015/04/think-problem-solution-work-ss-1920-800x450.jpg" width="800"/>
         </p>
         <p>
          Does this scenario resonate with you?
         </p>
         <p>
          Youre about to launch a PPC campaign, and youve ensured everything on your end is streamlined and ready to go. You publish your campaigns and wait in suspense to see how many conversions are rolling in only to hear crickets chirping.
         </p>
         <p>
          You did everything you were supposed to on your end, including giving the web developer the AdWords tracking code for conversions to install. Suddenly, it hits you it wasnt anything you did. The conversion code wasnt installed correctly.
         </p>
         <p>
          You shake your head. Who is supposed to be the tech person in this relationship?
         </p>
         <p>
          Unfortunately, sometimes the IT folks or web developers dont take the time to understand proper implementation of the AdWords conversion code, and this is where you can step in to troubleshoot and fix the problem fast.
         </p>
         <p>
          In this post, Ill outline tips you can keep on hand to diagnose the no-conversions problem next time it arises even if youre a non-techie PPC professional.
         </p>
         <h2>
          4 Common Problems With AdWords Conversion Tracking
         </h2>
         <p>
          There are several possiblereasons whyconversions arent showing up after a PPC campaign goes live. Some of the common problems we see with implementation areactually quite simple to fix:
         </p>
         <ol>
          <li>
           The code is placed on the correct page, but is not copied exactly as it should be.
          </li>
          <li>
           The code is not actually placed on the page. (They thought they did it, but alas, its not there.)
          </li>
          <li>
           Additional JavaScript code has been added via the CMS or application delivery infrastructure, whichchanged the AdWords code. (We recently had a client who installed a page speed plugin that somehow integrated with the AdWords conversion code.)
          </li>
          <li>
           The code is for remarketing, not conversions. (The AdWords remarketing and conversion codes look very similar.)
          </li>
         </ol>
         <h2>
          Troubleshooting Guide: 4 Steps To Solve The No-Conversions Issue
         </h2>
         <p>
          First things first: If you are seeing conversions within 24 hours, there shouldnt be a problem with tracking. We usually spot-check with the client, however, and ask if they are seeing some of the same activity we are in terms of conversions or sales, just to be sure.
         </p>
         <p>
          However, knowing how to assess the situation if youre not seeing conversions is crucial. If you find yourself in a bind, here are four steps you can take to research the problem.
         </p>
         <h2>
          1. Test The PPC Conversion Yourself
         </h2>
         <p>
          Do a test the old-fashioned way and act as a customer. Find the ad in search, click on the ad, and follow through with the conversion.
         </p>
         <p>
          If its a product you or your client sells, consider how to handle a test purchase maybe you want to use a discount code that puts the product at $0.
         </p>
         <p>
          For other types of conversions, you can filter out your tests from the data by searching for the ad with a specific query, like the keyword youre targeting plus your name.
         </p>
         <p>
          For example, Id trigger the ad from search by searching for keyword + Pauline. This will record that conversion with that keyword.
         </p>
         <h2>
          2. Use ADiagnostic Tool
         </h2>
         <p>
          I love the handy Tag Assistant tool. Its a Google Chrome extension that validates the implementation of Google tracking on any Web page. You can download it from the Chrome Web store
          <a href="https://chrome.google.com/webstore/detail/tag-assistant-by-google/kejbdjndbnbjgmefkgdddjlbokphdefk" target="_blank">
           here
          </a>
          .
         </p>
         <p>
          The extension works by color-coding the status of the implementation. From the
          <a href="https://support.google.com/tagassistant/answer/2947093?hl=en" target="_blank">
           support page
          </a>
          , heres an explanation:
         </p>
         <blockquote>
          <ul>
           <li>
            A grey icon with an x indicates that no code was found on the page
           </li>
           <li>
            A green indicator will show if a valid tag was found. The number in the icon indicates the number of tags that were found
           </li>
           <li>
            A blue indicator will show there are suggestions on improving overall tagging health
           </li>
           <li>
            A yellow indicator will show that a tag was found with minor implementation issues
           </li>
           <li>
            A red indicator will show that a tag was found with critical implementation issues
           </li>
          </ul>
         </blockquote>
         <p>
          Sometimes, the code will be implemented just fine, but the Tag Assistant may suggest a few nit-picky things that might make it even better. Heres an example of what you might see if your code falls into the blue category (which is comprised of suggestions on improving overall code health):
         </p>
         <p>
          <img alt="blue-conversion-tracking" class="aligncenter wp-image-218465 size-full" height="626" src="http://searchengineland.com/figz/wp-content/seloads/2015/04/blue-conversion-tracking1.png" width="465"/>
         </p>
         <p>
          Getting to green is the goal, but you can assess it on a case-by-case basis. In the example above, one thing we didnt do was set the conversion value, and the extension is alerting us of that.
         </p>
         <p>
          Heres a video overviewof the extension for more info:
         </p>
         <div class="fit-embeded">
          <iframe allowfullscreen="" frameborder="0" height="281" src="https://www.youtube.com/embed/4AqanTBA9X4?feature=oembed" width="500">
          </iframe>
         </div>
         <h2>
          3. Confirm The Code Has Been Set Up Accurately
         </h2>
         <p>
          In order for the tracking code to work, you first need to ensure that the client has some sort of thank you page that renders after a conversion, whether its a product purchase, a sign-up form, or something else.
         </p>
         <p>
          Look at the Web page to see if the code has been placed in the right spot between the opening and closing body tags (&lt;body&gt; and &lt;/body&gt;) on the page. Then, confirm with the developer that he or she sees it set up properly, too.
         </p>
         <p>
          AdWords gives a step-by-step guide on how to set the codeup, and you can always share
          <a href="https://support.google.com/adwords/answer/1722054" target="_blank">
           this document
          </a>
          with anyone who needs to better understand implementation.
         </p>
         <p>
          From that help file is an example of where to place the code. Ive taken a screenshot and highlighted the body tags here:
         </p>
         <p>
          <img alt="code-body-tags" class="aligncenter wp-image-218466 size-full" height="757" src="http://searchengineland.com/figz/wp-content/seloads/2015/04/code-body-tags1.jpg" width="600"/>
         </p>
         <h2>
          4. View Source Code &gt; Copy, Paste
         </h2>
         <p>
          If something is still wrong and youre not sure what it is, you can go the Web page itself and right click to view the source code, as illustrated here:
         </p>
         <p>
          <img alt="google-view-page-source" class="aligncenter size-full wp-image-218467" height="548" src="http://searchengineland.com/figz/wp-content/seloads/2015/04/google-view-page-source1.jpg" width="600"/>
         </p>
         <p>
          Identify the conversion tracking code on the HTML side, copy it and paste into a document. Then, go to your AdWords account, copy the original tracking code and paste it into the same doc.
         </p>
         <p>
          Next, start comparing. Are they identical? Something as small as a misplaced forward slash (/) canbreak the code, causing it not to work. After identifying anydiscrepancies,you can highlight the problem area and send to the developer.
         </p>
         <p>
          These four steps are usually enough to identify the problem. If all else fails, check out
          <a href="https://support.google.com/adwords/answer/1722021" target="_blank">
           this thorough troubleshooting guide
          </a>
          from AdWords, which covers everything from confirming and verifying the code to troubleshooting the code snippet and debugging it.
         </p>
         <p>
          Do you have any tips to ensure the tracking gets implemented smoothly? Share with us in the comments if so.
         </p>
